# 00-总控入口

## 总控职责

总控只做四件事：

1. 建立当前任务模型。
2. 选择最窄的下游模块。
3. 要求所有结论绑定证据。
4. 输出可执行产物，并决定是否重新路由。

总控不直接替代所有模块。它像竞赛沙盒总控一样，先确认主导证据面，再进入专项模块。

## 默认工作模型

把任意网文问题先视作一个“读者承诺是否被持续兑现”的系统问题。不要先问“爽点够不够”，而是问：

1. 这本书承诺给读者什么体验？
2. 读者为什么现在要继续看？
3. 当前段落在满足哪个承诺？
4. 满足后有没有新承诺接上？
5. 主角、事件、人设、金手指、社交关系是否一起服务这个承诺？

## 最小可验证路径

每次分析只先证明一个最小闭环：

```text
读者需求 -> 戏剧性契机 -> 主角目标 -> 阻碍/代价 -> 期待形成 -> 情绪拉扯 -> 兑现 -> 新期待
```

如果这个闭环无法在一章、一个单元或一个开篇中被证明，不要扩展到全书理论；用户明确要求全书精拆时，先建立章节索引和覆盖台账，再逐卷证明。

## 证据优先级

当来源冲突时，按以下顺序判断：

1. 原文章节实际呈现
2. 读者真实反馈和追读变化
3. 同题材榜单样本
4. 作者大纲、设定、章纲
5. 拆书表和二次分析
6. 教程概念
7. 作者主观解释

教程概念不能压过文本事实。拆书表不能压过读者真实阅读路径。

## 路由规则

1. 如果问题还模糊，留在总控，先做任务建模。
2. 如果问题集中在题材和卖点，进入 `module-题材与读者承诺.md`。
3. 如果问题集中在第一单元，进入 `module-开篇与钩子.md`。
4. 如果问题集中在爽点、追读、断期待，进入 `module-期待感与情绪.md`。
5. 如果问题集中在单元事件，进入 `module-单元故事起承转合.md`。
6. 如果问题集中在卷纲、换地图、高潮后掉读，进入 `module-长线结构与不断档.md`。
7. 如果需要从别人作品中取证，或明确要求完整拆一本书，进入 `module-拆书取证.md`。
8. 如果问题集中在某个题材的长期训练，进入 `genre-modules/` 对应模块。
9. 如果用户是新人且目标不清，进入 `curriculum/新人等级路径.md`。

## 重路由规则

出现以下情况时回到总控：

- 当前模块无法解释主要问题。
- 发现问题根因在上游，例如开篇问题其实是题材承诺不清。
- 发现问题根因在下游，例如单元故事能成立，但长线期待断档。
- 拆书越拆越散，无法回答一开始的问题。
- 产物无法指导下一章或下一轮修改。
- 题材模块给不出事件，返回案例库重新取同题材证据。

## 标准输出格式

每次总控结束时输出：

```text
当前任务：
主导问题：
已进入模块：
关键证据：
判断：
产物：
下一步：
```
