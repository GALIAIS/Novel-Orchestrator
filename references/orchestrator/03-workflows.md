# 03-workflows

## Workflow A：新人入门训练

目标：建立网文基础判断力，不急着追求复杂技巧。

阶段：

1. 题材识别：选一个明确题材，不跨题材乱学。
2. 样本收集：找 5 本同题材近期有效样本。
3. 开篇取证：每本只拆第一个单元故事。
4. 六要素归纳：主角、身份、行动、困境/目标、社交关系、金手指。
5. 期待链路归纳：读者期待什么、什么时候形成、什么时候兑现。
6. 仿写训练：不搬设定，只仿结构功能。
7. 自审复盘：用开篇十问和单元故事卡检查。

产物：

- `同题材样本表`
- `第一单元拆解表`
- `我的开篇六要素`
- `仿写训练稿`

## Workflow B：题材定位

目标：确认这本书承诺给谁看、看什么、为什么继续看。

步骤：

1. 写出一句话题材承诺。
2. 写出主角起点状态和终局想象。
3. 写出读者最想看到的 5 个场面。
4. 写出同题材不可缺席元素。
5. 写出本书反常点或差异化表达。
6. 检查书名、简介、第一章是否同向。

产物：

- `读者承诺卡`
- `卖点一致性检查`

## Workflow C：开篇诊断

目标：修复第一单元吸引力不足。

步骤：

1. 以第一个单元故事为范围，不机械限制三章。
2. 填六要素。
3. 标出第一章末尾的短期期待。
4. 标出两万字内是否存在两长一短。
5. 检查反常行为、危机、金手指、背景信息是否服务同一承诺。
6. 检查有没有过度填空式堆要素。
7. 输出删改清单。

产物：

- `开篇诊断报告`
- `第一单元修复清单`

## Workflow D：期待感与爽点修复

目标：让“目标”变成“读者想看的目标”，让“爽点”变成“期待兑现”。

步骤：

1. 写出主角当前目标。
2. 写出目标达成后的具体画面。
3. 写出失败代价。
4. 写出谁会误判主角。
5. 写出主角凭什么打破误判。
6. 写出兑现时的多层反应：反派、旁观者、亲友、势力、主角自身。
7. 写出兑现后接上的新期待。

产物：

- `期待链路表`
- `爽点兑现表`

## Workflow E：单元故事构建

目标：把一个事件写成有因果、有拉扯、有转折、有兑现的故事单元。

步骤：

1. 起：确定主角将得失什么。
2. 承：设计主角选择与执行，而不是被剧情拖走。
3. 转：增加意料之外但合逻辑的新信息、阻碍、代价或反转。
4. 合：兑现得失，回收铺垫，写出收获和反应。
5. 扣：在合之前或合之后接上下一个目标。

产物：

- `单元故事卡`
- `章节功能表`

## Workflow F：长线不断档

目标：解决高潮后掉读、换地图断裂、支线散乱。

步骤：

1. 列出当前短线目标。
2. 列出已有长线期待。
3. 在当前目标兑现前，提前露出下一个目标线索。
4. 保留或迁移人际关系网，让见证体系不断裂。
5. 将支线设计成源流：最终能流回主线。
6. 每个阶段收获都改变主角状态，并开启更高层问题。

产物：

- `期待库存表`
- `循环衔接表`
- `换地图风险表`

## Workflow G：拆书取证

目标：从样本中提取当前问题的证据，而不是泛读感想。

步骤：

1. 先写拆书问题。
2. 只选与问题相关的章节范围。
3. 记录原文证据。
4. 抽象结构功能。
5. 判断是否可迁移到自己的题材。
6. 形成一个训练动作。

产物：

- `拆书取证表`
- `可迁移写法清单`
- `下一次仿写任务`

## Workflow G2：全书精拆

目标：把一本书拆成可复查的章节、单元、卷级和全书结构文件，而不是输出几段读后感。

步骤：

1. 建立输出文件夹：`<书名>-拆书/`。
2. 生成 `00-项目说明/00-拆书说明.md`：书名、来源文件、目标问题、拆解粒度、已知缺章或乱码。
3. 生成 `01-章节索引/01-章节索引.md` 和 `01-章节索引/02-coverage-ledger.md`。
4. 先扫一遍全书，写 `02-全书总览/01-全书承诺与核心循环.md`。
5. 建立大类文件夹：`03-题材承诺/`、`04-世界观设定/`、`05-力量金手指/`、`06-大纲主线/`、`07-卷章结构/`、`08-角色拆解/`、`09-关系势力/`、`10-情节冲突/`、`11-期待情绪/`、`12-伏笔信息/`、`13-叙事技法/`、`14-商业包装/`、`15-迁移训练/`、`18-媒介专项/`。每个大类继续按条目建子文件夹和独立 md，禁止只写三个粗类或把所有内容塞进总报告。
6. 按自然卷或每 20-50 章切块，逐块写入 `16-卷级精拆/vol-xx-卷级精拆.md`。
7. 每个章节至少记录：章节功能、主角目标、阻碍/信息差、期待、兑现/压制、章尾钩子。
8. 每个单元至少记录：起承转合、爽点兑现、收获、关系变化、下一单元接法。
9. 每卷结束写卷复盘：卷级承诺、核心矛盾、高潮兑现、断档风险、可迁移写法。
10. 角色必须按 `08-角色拆解/01-主要角色/`、`02-重要配角/`、`03-反派角色/`、`04-功能角色/` 分文件夹；每个角色一个独立 md，至少拆到基础档案、外貌识别、性格心理、成长变化、剧情功能、关系网络、表演证据、写作拆解。
11. 设定、势力、事件、伏笔、期待、爽点、场景等也按条目独立建 md；每条至少拆边界、代价、变化、关联角色/事件/伏笔、风险、可迁移写法，总表只做索引。
12. 轻小说补角色属性、对白密度、日常/主线平衡、插画节点、卷尾钩子；漫画补分镜、页翻点、格子节奏、视觉钩子、对白气泡、高潮页。
13. 最后写 `99-总报告/99-总报告.md`：全书结构图、强项、弱点、可迁移模板、仿写训练。

## Workflow H：题材专项训练

目标：在一个题材内形成稳定事件生产能力。

步骤：

1. 进入 `genre-modules/` 对应题材模块。
2. 从 `case-library/按题材/` 选 30 条案例。
3. 每条只拆核心循环、开篇必备、期待来源和断档风险。
4. 汇总本题材 10 个高频场面。
5. 用自己的设定改写 3 个事件。

产物：

- `题材专项拆解表`
- `高频场面清单`
- `同题材仿写稿`

## Workflow I：新人课程路径

目标：把学习拆成等级、训练、验收。

步骤：

1. 打开 `curriculum/新人等级路径.md`。
2. 从 L0 开始判断自己是否达标。
3. 未达标则做对应 `训练任务库.md` 任务。
4. 用 `验收标准.md` 检查产物。
5. 通过后再进入下一等级。

产物：

- `等级训练计划`
- `阶段验收文件`
