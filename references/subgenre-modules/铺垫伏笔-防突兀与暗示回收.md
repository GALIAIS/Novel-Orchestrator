# 铺垫伏笔-防突兀与暗示回收

依据案例：`batch-072` 案例 358。

## 读者承诺

这个模块用于卷纲、章纲里的前置信息设计。它承诺的是用铺垫防止后文突兀，用伏笔制造期待和回收惊喜。

## 核心循环

```text
后文关键内容 -> 前文铺垫自然化 -> 关键细节暗示 -> 读者猜测 -> 回收呼应
```

## 开篇必备

- 先确定后文关键人物或事件。
- 铺垫负责让它不突兀。
- 伏笔负责提示但不说透。
- 回收位置提前安排。
- 反转必须合理。

## 期待来源

- 铺垫给合理性。
- 伏笔给期待感。
- 猜到开头带来参与感。
- 猜不到结尾带来惊喜。
- 回收让读者相信作者有设计。

## 断档风险

- 把所有前置信息都叫伏笔。
- 铺垫太重，读者提前猜透。
- 伏笔没有回收。
- 后文人物突然出现。
- 只追求反转，不保证合理。

## 训练任务

设计一个后文反转，写 2 条铺垫和 1 条伏笔。铺垫只负责合理性，伏笔必须让读者产生错误但接近的猜测。
