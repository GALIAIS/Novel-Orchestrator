# 开篇场景扩张-信息预载与范围再约束

依据案例：`batch-412` 案例 2056

## 读者承诺

从变量受控的熟悉场景建立主角目标，只通过当前行动预载后续规则；意外扩大局势时保持目标和视角连续，并重新限制阵营、空间、时间和决策范围。

## 核心循环

```text
熟悉小场景与单一目标 -> 当前行动展示必要规则 -> 局部结果 -> 已铺垫意外扩大风险 -> 原目标升级或重选 -> 阵营按动作逐层进入 -> 大场面范围再约束 -> 事后小场景清点与续接
```

## 开篇必备

- 一个追随者、一个目标和一项主要阻碍。
- 熟悉场景中的独特异常。
- 只在当前行动中使用的预载规则。
- 扩张后仍连续的主角目标和视角。
- 有限活跃阵营、空间锚点和时间窗口。
- 高潮后消化状态变化的小场景。

## 期待来源

读者先在低负荷环境看懂人物与规则，局部事件随后升级为集体危机；场面虽扩大，主角仍有明确可执行目标，事后变化又被沉淀成下一轮承诺。

## 断档风险

- 把小场景理解成没有事件。
- 为未来大战提前讲百科。
- 扩张后突然切为全知战报。
- 一次引入所有阵营与历史。
- 主角在大场面中失去可影响结果。
- 离开小场景后不再限制范围。

## 训练任务

写两个各 1800 字的连续场景：前者只允许三名人物和三条被行动使用的规则，后者新增两个阵营，但保留同一视角、一个空间锚点和一个本章决策，结尾回到清点后果的小场景。
