# 爽文ABC情绪框架

依据案例：`batch-002` 案例 010。

## 读者承诺

用最小情绪单位生成爽文：A 负责期待蓄力，B 负责爽点释放，C 负责嵌套和突然变化，让章节不只是发生事件，而是持续操控读者期待。

## 核心循环

```text
A 动机/共情/压迫/期待 -> B 装逼/反应/释放 -> C 嵌套旧问题或突然新问题 -> 下一轮 A
```

## 开篇必备

- 主角为什么进入事件。
- 读者为什么共情或期待。
- 谁制造质疑、困难、误判或压迫。
- 主角如何释放第一波小爽点。
- 章末如何嵌入下一轮期待。

## 期待来源

- A 让读者想看主角解决。
- B 让读者获得释放。
- C 让读者在释放后仍然有新问题可追。
- 多个 A/B 可嵌套成小爽点和大高潮。

## 断档风险

- 没有 A，直接 B，爽点无铺垫。
- A 过长，B 太迟，读者只感到压抑。
- B 只有结果，没有他人反应。
- C 只是卖关子，没有和旧问题形成逻辑连接。

## 训练任务

用 A/B/C 标记重写一个章节大纲。要求至少有 A1、A2、B1、A3、B2、C 六拍，且每个 B 都能找到对应的前置 A。
