# 无限流-强制剧情参与驱动力

依据案例：`batch-163` 案例 813。

## 读者承诺

让无限流主角有必须进入剧情、接触人物、战斗和升级的硬理由，避免主角可以旁观却硬被作者推着走。

## 核心循环

```text
获得金手指 -> 外部危险不可逃 -> 成长资源绑定剧情 -> 不参与等于慢性死亡 -> 主动进入副本
```

## 开篇必备

- 明确不参与剧情的代价。
- 成长资源只能从剧情人物或副本事件中获得。
- 外部敌人要让躲藏失效。
- 每个副本都服务升级路线。

## 期待来源

读者期待主角每次参与副本都有生存压力和收益目标，不是为了看热闹。

## 断档风险

- 主角可以躲却不躲。
- 系统任务替代真实动机。
- 成长资源和剧情人物无关。
- 升级路线没有规划。

## 训练任务

设计 5 个副本参与理由，每个都写清外部压力、内部收益、不参与代价和主角主动选择。
