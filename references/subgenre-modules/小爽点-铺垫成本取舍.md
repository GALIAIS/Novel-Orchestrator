# 小爽点-铺垫成本取舍

依据案例：batch-270 案例 1348

## 读者承诺

让签到、开罐、抽奖、捡漏等小爽点根据重量选择是否铺垫：轻爽点直接给，重爽点加缺口、阻力和重要性。

## 核心循环

```text
奖励出现 -> 判断重量 -> 直接给爽/轻铺垫 -> 强化重要性 -> 收益兑现
```

## 开篇必备

- 先判断奖励对剧情的重量。
- 普通奖励不要过度铺垫。
- 关键奖励要说明主角缺口和奖励价值。
- 轻阻力能放大奖励爽感。

## 期待来源

期待来自收益和需求差。主角越缺，奖励越重要，小爽点越有效。

## 断档风险

- 每个小奖励都铺成长流程。
- 奖励重要性不明，爽感弱。
- 天降宝物砸给已经富足的主角。
- 为小爽点硬造大冲突。

## 训练任务

列 5 个奖励，标注“直接给”或“铺垫后给”。只给最关键的一个设计完整情绪下压。
