# 平台群像-强配角与真群像边界

依据案例：batch-349 案例 1742

## 读者承诺

平台标签里的群像多指配角塑造好；新人优先写强主角加好配角，不急着写没有唯一主角的真群像。

## 核心循环

```text
主角承载主线 -> 配角各有目标 -> 配角丰富世界 -> 主线不散 -> 群像感加分
```

## 开篇必备

- 明确唯一主角和主线承诺。
- 配角有目标、立场和记忆点。
- 配角与主角产生关系或冲突。
- 配角不能抢走追读主线。
- 真群像需多核心结构，新人慎碰。

## 期待来源

读者既能追主角爽点，又觉得世界里其他人也鲜活。

## 断档风险

- 把配角丰满误写成多主角散线。
- 每个配角都开长线，主线失焦。
- 主角存在感被稀释。
- 群像标签带来错误读者预期。
- 角色多但功能重复。

## 训练任务

为 5 个配角各写一个目标、一个冲突点、一个记忆动作。删掉与主线无关的配角长线。
