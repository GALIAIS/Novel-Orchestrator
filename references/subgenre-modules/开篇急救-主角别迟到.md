# 开篇急救-主角别迟到

依据案例：`batch-325` 案例 1621

## 读者承诺

急救、事故、神迹开篇可以提供强画面和强危机，但主角必须尽早进入视角中心，否则读者会把患者或旁人误认为主角。

## 核心循环

```text
外部危机 -> 读者寻找主角 -> 主角直接承压 -> 主角选择/能力介入 -> 危机回收或升级
```

## 开篇必备

- 前 300 字内点明主角。
- 危机要直接落到主角身上，或让主角承担明确后果。
- 群像议论只保留能推高主角压力的句子。
- 世界观只放一条当前事件必须知道的信息。
- 主角信念要和设定冲突直接相连。

## 期待来源

读者期待主角在强危机场景中做出选择，而不是旁观一群人解释设定。

## 断档风险

- 患者、护士、医生抢走主角第一印象。
- 主角出场太晚，代入被迫重置。
- 生死危机不在主角身上，冲突变间接。
- 多人对话标签重复，拖慢节奏。
- 世界观一次塞太满，读者只觉得枯燥。

## 训练任务

重写一个急救开篇，删掉一半旁人议论，让主角在第一屏承担直接压力并完成第一次动作。
