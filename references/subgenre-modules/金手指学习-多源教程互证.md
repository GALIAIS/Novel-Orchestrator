# 金手指学习-多源教程互证

依据案例：`batch-087` 案例 434。

## 读者承诺

学习金手指设置时，不只收藏单个教程，而是把多个教程的共同原理互相印证，筛掉暴论和情绪化表达，留下能检查自己作品的问题。

## 核心循环

```text
收集同主题教程 -> 对照共同观点 -> 比较解释差异 -> 转成检查问题 -> 回查自己的金手指
```

## 开篇必备

- 至少两个同主题资料来源。
- 只保留能互相印证的底层原则。
- 不把教程作者的流量话术当结论。
- 每条观点都转成作品检查问题。

## 期待来源

- 金手指能否第一章看懂。
- 是否和主线矛盾绑定。
- 是否能制造升级期待。
- 是否能持续带来收获。

## 断档风险

- 收藏链接不消化。
- 只听一家教程形成偏见。
- 把暴论当规律。
- 学完没有转成自己的检查表。

## 训练任务

找两个金手指教程，各摘 5 条观点。整理共同原则，转成 10 个检查问题，检查自己的金手指开篇是否过关。
