# 逻辑漏洞-情绪调动优先

依据案例：`batch-095` 案例 471。

## 读者承诺

读者不是来审现实论文的，但也不是完全不在意逻辑。这个模块处理的是：当桥段有漏洞时，判断它是否仍能靠情绪调动成立，以及漏洞是否会被目标读者注意到并变成毒点。

## 核心循环

```text
设计情绪桥段 -> 检查现实逻辑 -> 判断目标读者敏感度 -> 加强情绪收益 -> 遮蔽/修补漏洞 -> 用后续爽点带走注意力
```

## 开篇必备

- 明确本桥段要调动的情绪：愤怒、委屈、孤独、爽、恐惧或同情。
- 漏洞不能正好打断主角代入。
- 关键行为至少要满足虚构逻辑，不必满足所有现实细节。
- 读者一旦注意到漏洞，下一段必须有更强情绪或补丁。

## 期待来源

- 不公平伤害带来的愤怒。
- 人情冷暖带来的委屈。
- 主角被误解、被抛弃后的反弹。
- 后续打脸和成长对前期压抑的补偿。

## 断档风险

- 作者只追求情绪，导致主角双标或伪善。
- 行为逻辑太突兀，读者还没爽先出戏。
- 目标受众并不接受降智桥段。
- 拆爆款时只学漏洞，不学它真正的情绪收益。

## 训练任务

写一个“同伴逃跑背叛主角”的 600 字桥段。写完标注 3 个逻辑漏洞和 3 个情绪收益。若漏洞正好破坏主角代入，必须重写动机。
