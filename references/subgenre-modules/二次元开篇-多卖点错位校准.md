# 二次元开篇-多卖点错位校准

依据案例：`batch-084` 案例 420。

## 读者承诺

这个模块用于二次元、末世、恋爱、金手指等多标签开篇的卖点校准。它承诺让作者只保留清楚的主承诺，避免危机、卖肉、恋爱、人设和金手指互相拆台。

## 核心循环

```text
标签堆叠 -> 读者实际卖点识别 -> 承诺冲突排查 -> 主副卖点重排 -> 开篇校准
```

## 开篇必备

- 第一章只明确一个主卖点。
- 金手指复杂度要低。
- 危机不能被金手指直接消除。
- 卖肉描写要有代入前提。
- 二次元人设要匹配平台。

## 期待来源

- 读者知道自己该看什么。
- 危机保留张力。
- 人设、故事、金手指互相支撑。
- 恋爱对象不会先引发恶感。
- 平台受众能接住表达方式。

## 断档风险

- 末世危机被无限重生抵消。
- 黑丝美腿出现在主角濒死场景。
- 金手指数据太多。
- 日轻嘴炮错投中文网文受众。
- 恶役标签让恋爱喜剧难成立。

## 训练任务

列出第一章所有卖点承诺，只保留一个主承诺和两个副承诺。检查每个承诺是否互相强化；互相抵消的后移或删除。
