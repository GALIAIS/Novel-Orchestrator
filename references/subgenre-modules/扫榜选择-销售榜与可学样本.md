# 扫榜选择-销售榜与可学样本

依据案例：`batch-249` 案例 1241。

## 读者承诺

扫榜选择模块帮助新人从付费榜单里找到真正可学的成熟样本，而不是被大神光环或热闹数据带偏。

## 核心循环

```text
确定题材 -> 看销售榜/分类销售榜 -> 排除过高不可学样本 -> 锁定成熟作者好创意 -> 拆前三章兑现
```

## 开篇必备

- 明确自己要写的题材或子题材。
- 优先使用销售榜作为市场筛选器。
- 区分大神观摩样本和新人可学样本。
- 记录书名、简介、前三章卖点和第一个爽点。

## 期待来源

新人从榜单获得的是学习路径确定感：知道市场正在买什么，也知道哪些成熟处理方式可以拆出来训练。

## 断档风险

- 只看榜单热度，不拆文本结构。
- 直接学大神的个人能力和资源积累。
- 只抄设定，不学承诺和兑现。
- 跨平台、跨题材盲目迁移。

## 训练任务

在目标题材分类销售榜里选 5 本书，标出 2 本观摩样本和 3 本可学样本。只拆 3 本可学样本的开篇承诺、主角处境、第一冲突和第一爽点。
