# 套路复用-熟悉元素换角度

依据案例：`batch-327` 案例 1634

## 读者承诺

宗门大比、秘境夺宝、剑修这类成熟元素可以反复用；服务目标受众，并换目的、视角或回收对象，就有新鲜感。

## 核心循环

```text
熟悉元素 -> 目标受众识别 -> 换目的/视角/对象 -> 熟悉爽点回收 -> 形成新鲜感
```

## 开篇必备

- 明确服务哪类读者。
- 把核心类型口味直接摆出来。
- 保留熟悉爽点。
- 改变事件目的或视角。
- 避免照搬标志桥段和剧情顺序。

## 期待来源

读者期待喜闻乐见的装逼、夺宝、剑修爽点，同时看到一点不同。

## 断档风险

- 想讨好所有读者。
- 只搬表层事件，不改目的。
- 为了新奇瞎编，反而像小学生作文。
- 不敢筛选读者，开头口味模糊。

## 训练任务

选一个老套路，写三种换角度方案：主角设局、奖励是陷阱、反派误判。
