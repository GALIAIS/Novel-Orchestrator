# 爽感机制-矛盾冲突期待解决链

依据案例：`batch-182` 案例 910。

## 读者承诺

爽感来自矛盾白热化后的冲突、情绪代入、主角计划和解决兑现的连续传动。

## 核心循环

```text
矛盾 -> 冲突 -> 情绪波动 -> 代入 -> 主角计划 -> 期待 -> 解决 -> 爽感
```

## 开篇必备

- 先有根本矛盾。
- 冲突必须白热化。
- 情绪波动要能让读者代入。
- 主角要有解决计划。

## 期待来源

读者看到主角承压并形成解决路径，才会期待冲突被解决后的爽感。

## 断档风险

- 没矛盾，只写事件。
- 没冲突，只写解释。
- 没计划，读者只憋屈。
- 没解决，爽感落空。

## 训练任务

按八项链路检查一章：矛盾、冲突、情绪、代入、计划、期待、解决、爽感；缺哪项补哪项。
