# Notion资料库-写作数据库管理

依据案例：batch-273 案例 1362

## 读者承诺

用 database、表格、清单或文档管理写作资料，让人物、章节、伏笔、素材能被找回并回写正文。工具只服务执行，不替代写作。

## 核心循环

```text
资料混乱 -> 建最小数据库 -> 分类归档 -> 写作调用 -> 定期备份
```

## 开篇必备

- 先定正文入口，再定资料库。
- 只建人物、章节、伏笔等必要表。
- 明确工具缺点和备份方式。

## 期待来源

作者期待来自秩序感和可追踪性。读者端感受到的是伏笔不断、人物不乱、更新更稳。

## 断档风险

- 沉迷搭系统，正文没有推进。
- 工具加载慢、语言门槛或数据安全影响使用。
- 表太多、字段太多，维护成本高。

## 训练任务

建立三张最小表：人物、章节、伏笔。每张最多 5 个字段，用一周后再决定是否扩展。
