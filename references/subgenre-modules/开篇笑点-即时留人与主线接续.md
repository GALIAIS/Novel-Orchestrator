# 开篇笑点-即时留人与主线接续

依据案例：batch-382 案例 1907

## 读者承诺

笑点可以提供开篇即时正反馈，但必须同时推进人物、关系或剧情，并很快接入危机与爽点。

## 核心循环

```text
反差包袱 -> 即时笑点 -> 剧情同步推进 -> 接入危机/爽点 -> 基调续航
```

## 开篇必备

- 第一处人物反差。
- 不伤核心代入的笑点对象。
- 笑点造成的关系变化。
- 后续危机或爽点入口。

## 期待来源

- 无需长铺垫的即时反馈。
- 恋爱互动或隐藏实力推动剧情。
- 笑点为设定争取接收时间。
- 可重复的喜剧机制。

## 断档风险

- 单纯融梗不推剧情。
- 开篇搞笑，后续无法维持。
- 贬低主角或核心角色形成毒点。
- 作者自嗨而读者尴尬。

## 训练任务

写一个 800 字恋爱或无敌流开篇，前三百字给笑点，笑点必须改变关系或目标，结尾接入明确危机。
