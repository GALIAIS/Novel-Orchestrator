# 末世拆书-平台子类取样

依据案例：`batch-063` 案例 313。

## 读者承诺

这个模块用于末世、丧尸、废土、洪水、异常世界等题材的拆书选样。它承诺的是先按子类和目标平台筛样本，再拆近期有效开篇，避免拿不匹配的老经典误导新书架构。

## 核心循环

```text
确定末世子类 -> 查目标平台 -> 选近期样本 -> 补经典参考 -> 拆前三章危机/资源/规则 -> 迁移架构
```

## 开篇必备

- 明确是丧尸、洪水、废土还是异常世界。
- 先确认目标平台是否收这类文。
- 至少选一本近期有效样本。
- 经典老书只拆可迁移部分。
- 前三章重点看危机、资源、规则和目标。

## 期待来源

- 生存危机压迫。
- 资源争夺和安全区目标。
- 感染、灾变或怪物规则。
- 人际秩序崩坏。
- 主角建立新秩序或找到活路。

## 断档风险

- 泛泛找“末世文”，不分子类。
- 只学老书，不看近期平台。
- 目标平台不收稿。
- 开篇只有灾难氛围，没有资源和目标。
- 丧尸规则不清，危机无法升级。

## 训练任务

为自己的末世题材列出 5 个样本：2 本近期有效、2 本经典、1 本同平台。每本只拆前三章的灾变规则、资源压力、主角目标和下一步期待。
