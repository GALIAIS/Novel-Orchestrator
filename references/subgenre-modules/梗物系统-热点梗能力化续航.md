# 梗物系统-热点梗能力化续航

依据案例：`batch-200` 案例 997。

## 读者承诺

把网络梗变成超凡能力，可以获得持续更新的脑洞池；但梗必须服务冲突、升级和破局，不能只做段子展示。

## 核心循环

```text
热点梗 -> 能力化规则 -> 主角破局 -> 读者识别爽点 -> 新梗变体续航
```

## 开篇必备

- 交代梗物或梗能力的底层规则。
- 第一章给一个读者熟悉的梗。
- 让主角用梗解决具体问题。
- 同时给出限制或代价。

## 期待来源

读者期待熟悉的梗被改造成意外能力，并看主角用它在正经冲突中赢。

## 断档风险

- 只玩梗，不推进主线。
- 热点过时导致笑点失效。
- 梗门槛太高，读者不懂。
- 能力没限制，爽点变成乱开挂。

## 训练任务

选 5 个热点梗，分别写成超凡能力，并给每个能力加一个不能无脑赢的限制。
