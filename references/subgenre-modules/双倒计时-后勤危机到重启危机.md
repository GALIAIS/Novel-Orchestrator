# 双倒计时-后勤危机到重启危机

依据案例：`batch-123` 案例 615。

## 读者承诺

长线生存种田可以用双倒计时：前期用后勤断绝推动行动，中期再触发更大的重启或毁灭倒计时，既有紧迫感又不压爆科技进展。

## 核心循环

```text
封闭困境 -> 后勤短倒计时 -> 种田建立平衡 -> 触发系统级危机 -> 长倒计时开启 -> 新阶段行动
```

## 开篇必备

- 前期危机要贴近日常生存。
- 短倒计时提供立即行动理由。
- 长倒计时不要过早压死节奏。
- 触发长期危机要有事件和代价。
- 主角不能因合理行动背毒锅。

## 期待来源

- 读者先看今天怎么活。
- 种田成果带来阶段满足。
- 平衡被打破产生新期待。
- 更大倒计时给中期主线理由。

## 断档风险

- 一开篇就抛太大倒计时，科技进展不可信。
- 只写后勤危机，长线目标不足。
- 主角修复系统却背锅，读者反感。
- 二五仔动机不合理。

## 训练任务

设计一个生存种田双倒计时：短期危机 3 万字内解决，中期触发长期危机。写清触发者动机和主角不背毒锅的处理。
