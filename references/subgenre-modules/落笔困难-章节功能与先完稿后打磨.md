# 落笔困难-章节功能与先完稿后打磨

依据案例：batch-387 案例 1933

## 读者承诺

落笔时先完成目标、动作和结果，暂时跳过不会描写的位置；完整粗稿之后再集中补积累和画面。

## 核心循环

```text
章节功能 -> 朴素粗稿 -> 难点占位 -> 完成因果链 -> 定向打磨
```

## 开篇必备

- 本章唯一功能。
- 人物目标与关键动作。
- 明确结果和下一章期待。
- 第二遍才处理的画面难点。

## 期待来源

- 首稿持续向前推进。
- 完整因果链暴露真实短板。
- 拆书积累能对准具体场景。
- 第二遍画面逐步接近想象。

## 断档风险

- 每个词都要求一次成型。
- 首稿同时承担最终润色。
- 章节功能不清导致细节失控。
- 只收集华丽词汇不学动作顺序。

## 训练任务

九十分钟写一章：前六十分钟禁止回删，最后三十分钟只打磨三个最影响画面的段落。
