# 下行期待-拉扯必要条件

依据案例：`batch-017` 案例 084。

## 读者承诺

情绪下行的目的不是压低读者，而是制造期待。没有期待的下行，不是拉扯，是消耗。

## 核心循环

```text
下行压制 -> 读者在意 -> 解决期待 -> 上行兑现
```

## 开篇必备

- 下行对象要让读者在意。
- 解决者要明确。
- 解决方式要有可期待空间。
- 下行后必须接上行动或钩子。

## 期待来源

- 主角会反击。
- 真相会揭开。
- 误判会被打脸。
- 压制会被回收。

## 断档风险

- 下行没有对应期待。
- 读者不知道谁能解决。
- 压制过长，无上行信号。
- 只虐不爽。

## 训练任务

检查一个下行情节，写出它拉起的明确期待。写不出来就重构或删除。
