# 模拟器异能-主动破局与限制代价

依据案例：`batch-108` 案例 540。

## 读者承诺

模拟器文的爽点是先展示坏未来，再看主角用信息差主动改变命运。奖励不能只是系统到点自动发，必须来自主角思考、尝试和破局；金手指还要有限制、代价和阶段解锁，才能支撑长篇。

## 核心循环

```text
现实危机 -> 模拟惨败 -> 主角分析 -> 主动改变 -> 破局奖励 -> 代价/解锁下一轮
```

## 开篇必备

- 一个现实危机。
- 一段足够惨的模拟未来。
- 主角从模拟中提取关键信息。
- 主角主动做出改变。
- 模拟次数、代价或 debuff。

## 期待来源

- 读者知道未来很坏但可以改。
- 主角用聪明而不是天降礼包破局。
- 每次模拟都有新风险。
- 未解锁功能持续吊期待。

## 断档风险

- 主角被动等系统自动奖励。
- 模拟无限使用导致无压力。
- 升级太快，三五十万字写空。
- 只有预知，没有权衡和代价。

## 训练任务

写一个模拟器单元：模拟中主角死一次，现实中主角只改变一个选择并破局。最后必须付出一次代价或消耗一次模拟机会。
