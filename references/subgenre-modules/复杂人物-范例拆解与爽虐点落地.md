# 复杂人物-范例拆解与爽虐点落地

依据案例：`batch-218` 案例 1090。

## 读者承诺

先从优秀作品提取复杂人物和爽虐点范例，再用理论复盘并迁移到自己的桥段。

## 核心循环

```text
素材积累 -> 范例提取 -> 理论分析 -> 桥段应用 -> 写作验证
```

## 开篇必备

- 新人先保证剧情通畅和爽点合理。
- 理论用于解释案例，不直接生成角色。
- 爽虐点必须绑定具体剧情过程。
- 人物复杂来自处境压力和代价选择。

## 期待来源

读者看到人物被压力逼迫做选择，理解其不得已，同时期待后果爆发。

## 断档风险

- 用 MBTI/九型人格硬套人物。
- 只背抖音爽虐点，不找范例。
- 群像靠设定表串联，没有冲突链。
- 追求复杂导致主线不通。

## 训练任务

拆 3 个“被逼成坏人”的成熟案例，再写一条三人物冲突链。
