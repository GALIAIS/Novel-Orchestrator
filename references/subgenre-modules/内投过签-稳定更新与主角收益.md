# 内投过签-稳定更新与主角收益

依据案例：`batch-230` 案例 1150。

## 读者承诺

内投过签后，作者要按编辑反馈稳定执行：不断更、不乱炫技、主角不吃大亏，亏也要换来更大收益。

## 核心循环

```text
编辑反馈 -> 稳定更新 -> 控制主角亏损 -> 收益大于代价 -> 推荐期兑现
```

## 开篇必备

- 明确更新底线。
- 规划可爆更节点。
- 每次主角吃亏都有更大收益。
- 主角收益高于配角收益。

## 期待来源

读者持续看到主角稳步变强、资源不亏、处境越来越好，连载安全感稳定。

## 断档风险

- 过签后为了超常发挥乱改路线。
- 断更伤害推荐和读者信任。
- 主角吃亏没有足够补偿。
- 配角赚得比主角多，读者站队松动。

## 训练任务

检查最近十章大纲，列出主角每次亏损和收益。任一事件亏损大于收益，就重写收益或删减亏损，并制定一周更新底线。
