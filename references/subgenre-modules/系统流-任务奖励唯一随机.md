# 系统流-任务奖励唯一随机

依据案例：`batch-036` 案例 180。

## 读者承诺

系统流的基本承诺是任务带来奖励，奖励改变现实。唯一性保证稳定预期，随机性制造惊喜和下一轮期待。

## 核心循环

```text
困境 -> 任务 -> 完成 -> 奖励 -> 现实兑现 -> 新任务/大奖期待
```

## 开篇必备

- 系统出现前先有困境。
- 当前任务明确。
- 奖励能现实兑现。
- 暂未获得的大奖要拉期待。

## 期待来源

- 读者等任务完成。
- 读者等抽奖或升级奖励。
- 唯一规则让爽点稳定。
- 随机奖励让爽点放大。

## 断档风险

- 只有面板没有事件。
- 任务重复导致疲劳。
- 奖励不影响现实。
- 随机性太散，主线失控。

## 训练任务

设计 3 连任务：每个任务写清触发、完成方式、奖励、现实用途和下一轮更大奖励期待。
