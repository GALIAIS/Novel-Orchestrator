# 起点发书-封面签约更新准备

依据案例：`batch-081` 案例 405。

## 读者承诺

这个模块用于新人准备起点发书。它承诺把封面、发书方式、签约反馈字数和更新底线整理成一张可执行清单。

## 核心循环

```text
稿件准备 -> 封面准备 -> 内投/直发选择 -> 签约反馈 -> 稳定更新
```

## 开篇必备

- 至少准备前三章。
- 有一份简短大纲。
- 明确内投还是直接发书。
- 封面用作家 App 自带制作或外包定制。
- 每天 4000 字更新能力要先自证。

## 期待来源

- 作者知道发书流程。
- 签约反馈字数有心理预期。
- 更新底线具体。
- 编辑反馈能快速校准。
- 发书前焦虑变成清单项。

## 断档风险

- 没有稳定 4000 字能力就发书。
- 封面和简介临时凑。
- 不知道内投和直发区别。
- 只等后台提示，不准备大纲。
- 忽略基础红线和平台细规。

## 训练任务

做一张起点发书清单。内投版写到 1 万字，直发版写到 3 万字；两版都必须包含封面、简介、前三章、大纲和 7 天更新记录。
