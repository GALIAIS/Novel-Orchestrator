# 平台审核-敏感风险预期管理

依据案例：`batch-120` 案例 599。

## 读者承诺

平台审核会直接影响更新、可见性和收入。作者选平台时不只看流量，也要预估敏感边界、锁章概率、替代表达和被卡后的处理路径。

## 核心循环

```text
选择平台 -> 识别敏感边界 -> 写前避险 -> 发前自查 -> 被锁处理 -> 调整题材和表达
```

## 开篇必备

- 明确目标平台审核风格。
- 高风险桥段提前降敏。
- 敏感词准备替代表达。
- 存稿留出审核缓冲。
- 预备章节被锁后的沟通和修改方案。

## 期待来源

- 稳定更新带来读者信任。
- 风险预案降低断更恐慌。
- 平台规则清楚后题材选择更稳。
- 作者能在流量和宽松之间做取舍。

## 断档风险

- 题材踩线后频繁锁章。
- 不知道敏感边界，只能反复试错。
- 为宽松去小站，但低估流量损失。
- 敏感期被举报导致全书节奏中断。

## 训练任务

为目标平台列一个敏感自查表：10 个高风险词/桥段、10 个替代表达、3 个被锁章后的处理动作。
