# 内容预告-视频文字双版本

依据案例：`batch-163` 案例 812。

## 读者承诺

用完成状态、发布时间、主题和版本差异制造课程或拆书内容的等待期待。

## 核心循环

```text
完成确认 -> 时间预告 -> 主题说明 -> 视频版期待 -> 文字版增量 -> 多渠道发布
```

## 开篇必备

- 明确内容已经完成或接近完成。
- 给出发布时间。
- 一句话说明核心主题。
- 说明文字版或补充版有什么增量。

## 期待来源

读者知道什么时候能看到、能学到什么，以及为什么还要等文字版。

## 断档风险

- 只说“快发了”，不说主题。
- 预告讲太多，削弱正式内容。
- 视频版和文字版没有差异。
- 发布渠道不清。

## 训练任务

写一个 100 字内容预告，包含状态、时间、主题、案例、版本增量和发布渠道。
