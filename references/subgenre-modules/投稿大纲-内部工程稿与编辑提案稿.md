# 投稿大纲-内部工程稿与编辑提案稿

依据案例：batch-385 案例 1923

## 读者承诺

个人大纲用于长期执行，编辑提案用于快速判断项目。投稿前应按当前编辑要求，从工程稿提炼独立可读的外部版本。

## 核心循环

```text
内部工程稿 -> 提取卖点与主线 -> 压缩成长循环 -> 按要求投稿 -> 反馈回写工程稿
```

## 开篇必备

- 一句话题材和目标读者。
- 三个核心卖点。
- 主角目标与优势。
- 前三章兑现方式。

## 期待来源

- 编辑迅速理解项目。
- 长线循环证明可持续性。
- 反馈落到卖点和结构。
- 内部版本继续支持写作。

## 断档风险

- 把历史个例字数当通用标准。
- 直接发送满是私人缩写的工程稿。
- 只列世界观，不讲主线循环。
- 为简洁隐藏结局和长线方向。

## 训练任务

按目标编辑当前要求另写提案稿，讲清题材、卖点、主角、前三章、长线循环和结局，再让不知情者复述。
