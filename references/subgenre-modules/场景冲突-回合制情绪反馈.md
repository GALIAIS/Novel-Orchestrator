# 场景冲突-回合制情绪反馈

依据案例：`batch-257` 案例 1281。

## 读者承诺

场景冲突模块把人物行为拆成动作、情绪、反馈、结果的连续回合，解决“知道结局但不知道怎么写过程”的卡点。

## 核心循环

```text
冲突目标 -> 一方动作 -> 对方情绪反馈 -> 结果改变局面 -> 下一回合升级
```

## 开篇必备

- 先明确最终爽点。
- 每回合都要有具体动作。
- 情绪变化必须由结果触发。
- 素材骨架来自拉片和人设积累。

## 期待来源

读者看见双方动作互相影响，情绪和局面一轮轮变化，冲突就有推进感。

## 断档风险

- 只写情绪标签，没有动作。
- 回合结果不改变局面。
- 情绪变化不服务最终爽点。
- 素材只记表面桥段，没有提炼骨架。

## 训练任务

写 5 回合对峙表：谁先动作、带什么情绪、对方如何反馈、结果如何改变局面，再扩成 1000 字。
