# 番茄首秀-高潮卡点与早反馈

依据案例：`batch-231` 案例 1155。

## 读者承诺

首秀或推荐要尽量卡在能承接新读者的高潮上；新人早拿数据反馈，早修正问题。

## 核心循环

```text
十万字节点 -> 高潮承接 -> 首秀曝光 -> 数据反馈 -> 修正/给量
```

## 开篇必备

- 当前高潮能让新读者看懂。
- 下个高潮距离可控。
- 明白平台首秀/推荐规则。
- 接受早反馈而不是无限等完美。

## 期待来源

流量进入时正好遇到强情绪或强事件，数据能更快验证作品是否能承接读者。

## 断档风险

- 为等下个高潮错过当前窗口。
- 当前高潮只服务老读者，新读者看不懂。
- 把首秀当最终审判，不敢早反馈。
- 数据差却不根据反馈修正。

## 训练任务

写首秀决策表：当前字数、当前高潮、下个高潮距离、前三日数据、读者进入成本、可修改问题。当前高潮能承接新读者就优先上。
