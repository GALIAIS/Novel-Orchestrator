# 开篇重写-爽画面反推聚焦

依据案例：batch-287 案例 1433

## 读者承诺

开篇所有信息都朝主角最爽画面推进，让读者知道主角受什么压、想做什么、凭什么反击、反击后爽在哪里。

## 核心循环

```text
最爽画面 -> 出手理由 -> 主角目标 -> 金手指显影 -> 冲突压迫 -> 对比兑现 -> 新期待
```

## 开篇必备

- 一个具体最爽画面。
- 一个能让主角出手的理由。
- 主角短期目标。
- 可见金手指或能力优势。
- 敌人低估、伤害或阻拦主角。
- 第一章内的选择和收益。

## 期待来源

读者提前知道主角具备反击可能，又看到敌人或环境不断压迫，于是期待主角用信息差和能力差完成对比兑现。

## 断档风险

- 主角沉睡或被配角挤出中心。
- 背景设定过多，情节不动。
- 危机只有形式，没有带来目标和收益。
- 18 元素不发生在主角身上。
- 爽点落在怪物爆炸、设定展示等作者自嗨内容上。
- 没有仇恨和出手理由。

## 训练任务

把一个散乱开篇压成四段：主角受压、主角目标、金手指出手、敌人/围观者反应。每段只保留服务最爽画面的信息。
