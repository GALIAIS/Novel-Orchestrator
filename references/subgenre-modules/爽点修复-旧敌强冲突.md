# 爽点修复-旧敌强冲突

依据案例：`batch-021` 案例 102。

## 读者承诺

修复“套路正确但不爽”的问题。把突然出现的新敌人改成长期欺压主角的旧敌，把小打小闹改成无路可退的强冲突。

## 核心循环

```text
旧敌压迫 -> 不公平伤害 -> 公开舞台 -> 主角反击 -> 旧敌惨败 -> 收益兑现
```

## 开篇必备

- 旧敌和主角的长期矛盾。
- 主角必须反击的理由。
- 可公开见证的测试或比赛舞台。
- 主角打脸后的具体收益。

## 期待来源

- 读者想看长期欠账被清算。
- 围观者、宗门高层、亲友共同见证。
- 主角从炮灰身份逆袭。

## 断档风险

- 新敌突然出现，仇恨不足。
- 主角主动招事，情绪正当性弱。
- 主角被坑后反应太软。
- 只秒杀，没有收益和身份改变。

## 训练任务

写一个外门测试桥段：旧敌欺压主角三次，测试中主角公开反杀，并获得入门、资源或身份收益。
