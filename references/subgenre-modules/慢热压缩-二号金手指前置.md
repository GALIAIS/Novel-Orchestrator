# 慢热压缩-二号金手指前置

依据案例：`batch-022` 案例 107。

## 读者承诺

解决开篇流畅但慢、金手指薄弱的问题。把拖沓过渡合并，让第二个核心能力尽快觉醒。

## 核心循环

```text
检查卖点 -> 删除过渡 -> 合并两章 -> 第一次穿越/冒险 -> 二号金手指觉醒
```

## 开篇必备

- 第一金手指或主角优势。
- 第二金手指的出现条件。
- 第一次冒险或穿越。
- 能让读者继续看的新期待。

## 期待来源

- 读者明确知道主角靠什么变强。
- 慢热段被压缩后，能力反馈更早。
- 第二章不再只是找人帮忙。

## 断档风险

- 清单都打勾，但读者仍看不到核心卖点。
- 金手指太薄，情绪拉扯不够。
- 第二章只有过渡和找人。
- 二号金手指拖到第三章后才进入正题。

## 训练任务

把两章开篇压成一章：保留第一次能力反馈，删掉纯过渡段，并在章末让二号金手指出现。
