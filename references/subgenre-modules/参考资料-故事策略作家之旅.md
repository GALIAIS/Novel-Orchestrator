# 参考资料-故事策略作家之旅

依据案例：`batch-318` 案例 1590

## 读者承诺

写作资料库要记录版本、可读性和适用问题，并把理论书转译成网文动作，而不是只收藏 PDF。

## 核心循环

```text
收集资料 -> 区分版本/质量 -> 排阅读优先级 -> 提炼概念 -> 转成作品动作
```

## 开篇必备

- 文件名和版本记录清楚。
- 下载或缺失状态要标注。
- 同类资料要有优先级。
- 每本资料只绑定一个当前写作问题。
- 读后必须输出可执行笔记。

## 期待来源

作者期待从《作家之旅》《故事策略》等理论书里获得结构工具，并转成开篇、冲突、人物压力、章节节奏的实际改法。

## 断档风险

- 资料越收越多，实际不读。
- 不区分翻译质量和版本。
- 理论照搬，不适配网文节奏。
- 没有输出拆解卡，资料无法转成能力。

## 训练任务

列 3 本写作资料，每本只写一个当前问题、一个要摘出的概念、一个要落到作品里的动作。
