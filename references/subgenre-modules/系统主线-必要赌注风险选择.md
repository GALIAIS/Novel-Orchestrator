# 系统主线-必要赌注风险选择

依据案例：`batch-121` 案例 604。

## 读者承诺

系统主线可以强制，但不能让主角像被牵线。读者要看到主角评估赌注、风险、成本和收益后，主动选择执行方式。

## 核心循环

```text
主线任务出现 -> 成本/失败代价明示 -> 主角分析收益风险 -> 选择开启或执行方式 -> 金手指兑现 -> 能力上限提升
```

## 开篇必备

- 任务目标明确。
- 开启或执行有成本。
- 失败代价具体。
- 成功收益能改变处境。
- 主角有可判断、可取舍的空间。

## 期待来源

- 读者等主角如何规避风险。
- 成本让任务不是白送。
- 收益提升带来上行。
- 主角分析过程建立能动性。

## 断档风险

- 系统说什么主角做什么。
- 只有强制，没有主角思考。
- 任务奖励和当前困境无关。
- 失败代价含糊，读者感受不到赌注。

## 训练任务

设计一个系统主线任务，写清开启成本、失败代价、成功收益、主角可选方案。再用 600 字写主角权衡后行动。
