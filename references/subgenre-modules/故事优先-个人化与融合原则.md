# 故事优先-个人化与融合原则

依据案例：`batch-318` 案例 1586

## 读者承诺

先把宏大题材落成具体个人的故事，再在开篇明确基调和爽点，避免主题压过故事或桥段失去作用。

## 核心循环

```text
宏大题材 -> 个人化困境 -> 大纲确认故事 -> 开篇校准基调 -> 爽点稳定兑现
```

## 开篇必备

- 主角的私人困境。
- 一段具体关系或具体选择。
- 明确的开篇基调。
- 可感知的爽点类型。
- 删除不服务人物、情节、期待的桥段。

## 期待来源

读者期待先被人物困境牵住，再通过稳定基调理解更大的主题。

## 断档风险

- 主题先行变成说教。
- 只喊故事为王，却不知道开篇承诺什么爽点。
- 前中后期基调参差。
- 宏大设定没有落到个人压力。

## 训练任务

把一个“拯救世界”题材改写成一句个人化故事，只写主角和谁的关系出了什么问题，以及他必须做什么选择。
