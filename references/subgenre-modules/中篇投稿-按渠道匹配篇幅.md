# 中篇投稿-按渠道匹配篇幅

依据案例：`batch-333` 案例 1665

## 读者承诺

20 万字中篇要按渠道模型投稿，先匹配字数、收益方式、版权和自带流量。

## 核心循环

```text
中篇项目 -> 平台字数/收稿习惯 -> 收益方式 -> 投稿包装 -> 主渠道
```

## 开篇必备

- 明确作品字数和题材。
- 判断是否有自带流量。
- 比较短篇频道、中短篇平台、自引流平台。
- 看版权和结算方式。
- 按渠道改简介和卖点。

## 期待来源

作者把中篇投到适合的商业模型，而不是硬套百万字连载逻辑。

## 断档风险

- 长篇平台和中篇篇幅错配。
- 不看版权要求。
- 有粉丝却不考虑自引流。
- 只看千字价，不看平台声量。
- 投稿包装不适配渠道。

## 训练任务

为中篇项目做渠道表：起点短篇、晋江、磨铁、自引流平台，各列字数、题材、收益、版权风险。
