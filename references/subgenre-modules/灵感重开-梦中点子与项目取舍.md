# 灵感重开-梦中点子与项目取舍

依据案例：`batch-223` 案例 1111。

## 读者承诺

新灵感可以带来更强兴趣，但必须转成可验证的读者承诺，避免无限重开。

## 核心循环

```text
旧稿推进 -> 新灵感出现 -> 兴奋感比较 -> 核心承诺校验 -> 合并/替换/另开/继续
```

## 开篇必备

- 先写清旧项目核心卖点。
- 再写清新点子的核心卖点。
- 比较前三章钩子和长期冲突，而不是只比新鲜感。
- 给重开设置停止标准。

## 期待来源

作者获得的是项目取舍清晰度：热情保留，成本可控，练习目标明确。

## 断档风险

- 把梦中兴奋误判为成熟方案。
- 反复重开，没有完成任何一版。
- 旧稿可用资产被全部丢弃。
- 重开只是逃避当前难点。

## 训练任务

把旧项目和新点子各写成 200 字卖点，只比较读者承诺、前三章钩子、长期冲突和个人热情，决定合并、局部替换、另开新坑或继续旧稿。
