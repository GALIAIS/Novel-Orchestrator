# 新人基础-故事优先非脑洞

依据案例：`batch-104` 案例 518。

## 读者承诺

新人要写的是故事，不是脑洞、噱头或创意。脑洞只能增色，必须扩成起因、经过、结果，才能成为可连载内容。

## 核心循环

```text
放下偏见 -> 看懂样本优点 -> 稳定产出 -> 脑洞扩成故事 -> 大纲线 -> 匹配网站
```

## 开篇必备

- 一个完整故事起因。
- 一个可展开经过。
- 一个阶段结果。
- 心里有大纲线。
- 目标网站风格匹配。

## 期待来源

- 读者看的是事件发展。
- 大纲线保证持续性。
- 稳定更新建立信任。
- 网站匹配提高初期数据。

## 断档风险

- 找捷径和秘笈。
- 只看别人缺点，看不见优点。
- 脑洞不能扩成故事。
- 起点风发错平台。

## 训练任务

拿一个脑洞写成三段：起因、经过、结果。再写目标网站和为什么适合，不许只写“这个点子很有趣”。
