# 日常文-代入感功能分区

依据案例：`batch-082` 案例 409。

## 读者承诺

这个模块用于设计恋爱日常文。它承诺不硬套三幕式，而是用角色功能分区、生活事件和遗憾补偿维持代入感。

## 核心循环

```text
主角代入 -> 角色分区 -> 日常事件 -> 遗憾补偿 -> 关系升温
```

## 开篇必备

- 主角身份要容易代入。
- 给主角一个可补偿的遗憾。
- 每个女角色负责不同生活区域。
- 日常事件要有卖点功能。
- 对话和心理活动要推进关系。

## 期待来源

- 读者想过主角这一天。
- 校园、家庭、恋爱各有情绪区。
- 未来知识或能力弥补遗憾。
- 每段日常都产生关系变化。
- 生活连续感带来追读。

## 断档风险

- 日常只有流水账。
- 女角色功能重叠。
- 没有代入补偿。
- 只总结剧情，不研究卖点效果。
- 硬套三幕式导致日常变形。

## 训练任务

为 5 章日常文写功能分区表：每章写生活事件、角色功能、代入补偿、关系升温。没有关系变化的日常删掉或合并。
