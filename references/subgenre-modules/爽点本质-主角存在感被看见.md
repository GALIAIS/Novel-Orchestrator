# 爽点本质-主角存在感被看见

依据案例：batch-282 案例 1407

## 读者承诺

爽点可以理解为主角存在感被世界看见：队友和路人因好理由关注，敌人因威胁和损失关注，最终形成地位反馈。

## 核心循环

```text
主角行动 -> 世界反馈 -> 正面认可 -> 敌方忌惮 -> 存在感上升
```

## 开篇必备

- 主角做出值得关注的行动。
- 关注者分层。
- 每类关注都有理由。
- 关注带来后续影响。

## 期待来源

读者代入的是“我被看见、被承认、被忌惮”。存在感通过角色反应落地。

## 断档风险

- 只写旁白吹主角。
- 被坏理由关注却当爽点。
- 关注没有带来地位或行动变化。
- 缺少铺垫，旁人突然震惊。

## 训练任务

写一个主角刷存在感段落，必须出现队友认可、路人震惊、敌人忌惮三种反馈。
