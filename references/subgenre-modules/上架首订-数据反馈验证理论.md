# 上架首订-数据反馈验证理论

依据案例：`batch-122` 案例 610。

## 读者承诺

上架不是只报喜，而是用付费数据验证写法。收藏、首订、收订比、打赏和追读断点，都能反推读者承诺是否兑现。

## 核心循环

```text
上架前收藏 -> 设定首订目标 -> 获取首订反馈 -> 计算收订比 -> 对照开篇和追读 -> 调整后续或下一本
```

## 开篇必备

- 上架前记录收藏。
- 提前设定首订目标。
- 上架后记录首订和收订比。
- 把数据对应到具体写法假设。
- 区分平台和盗版等外部影响。

## 期待来源

- 作者知道付费转化是否达标。
- 数据能验证理论学习是否落地。
- 追读断点指向具体章节问题。
- 下一本选题和开篇有证据可依。

## 断档风险

- 只看收藏，不看首订。
- 把数据差泛化成“全都不好”。
- 忽略平台和盗版环境。
- 不把数据回连到开篇承诺和追读节点。

## 训练任务

做一张上架复盘表：收藏、首订、收订比、打赏、追读断点。每个数据只写一个对应写法假设和一个后续动作。
