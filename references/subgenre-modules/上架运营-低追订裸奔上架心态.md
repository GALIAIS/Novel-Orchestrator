# 上架运营-低追订裸奔上架心态

依据案例：`batch-193` 案例 963。

## 读者承诺

新人低追订、无推荐上架时，要把数据记录、心态稳定和后续复盘分开处理，避免用一次裸奔上架定义整本书或作者能力。

## 核心循环

```text
连载学习 -> 上架前追订偏低 -> 裸奔上架 -> 记录真实数据 -> 上架后复盘可改变项
```

## 开篇必备

- 记录上架前追订和推荐状态。
- 明确上架时间和更新计划。
- 区分平台流量问题和文本问题。
- 上架后观察订阅、追订、留存和评论反馈。

## 期待来源

作者获得真实商业反馈，并把焦虑转成下一轮选题、开篇、简介、节奏和更新策略的改进依据。

## 断档风险

- 低追订直接导致心态崩盘。
- 只求鼓励，不记录数据。
- 把裸奔上架当成个人失败。
- 不理解后台数据，复盘无从下手。

## 训练任务

做一张上架前后数据表，连续记录 14 天：更新字数、追订、均订、收藏、推荐状态、读者反馈和次日调整。
