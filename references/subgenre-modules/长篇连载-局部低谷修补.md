# 长篇连载-局部低谷修补

依据案例：`batch-148` 案例 738。

## 读者承诺

长篇前百章出现连续弱段，不等于整本毁掉。先判断弱在情绪压力还是节奏速度，能小改就修，修不了就继续写并积累经验。

## 核心循环

```text
发现弱段 -> 拆问题性质 -> 小改尝试 -> 不能修就继续写 -> 经验复盘
```

## 开篇必备

- 明确弱段位置。
- 连续几章。
- 弱在情绪还是节奏。
- 是否影响主线承诺。
- 能否小改。

## 期待来源

作者需要从灾难化判断回到可执行修补。读者需要的是后续体验回稳，而不是作者反复弃书重开。

## 断档风险

- 把局部问题说成全崩。
- 不拆情绪和节奏，只焦虑。
- 能小改却直接重写。
- 改不了也不继续积累。

## 训练任务

选连续 5-10 章，标出每章情绪压力点和推进事件。低谷章只做三处小改：删重复解释、提前冲突触发、章尾增加未解决压力。
