# 世界细节-人物绑定与功能性白描

依据案例：`batch-394` 案例 1970

## 读者承诺

用有限细节同时建立世界质感和人物辨识度：只具体描写会改变读者判断、人物选择、关系或世界规则的部分，首次建立锚点，后续短标签回收。

## 核心循环

```text
场景提出判断需求 -> 选诊断性细节 -> 细节进入动作关系或后果 -> 首次形成记忆锚 -> 后续短标签唤回 -> 回收为情节功能
```

## 开篇必备

- 明确本场需要读者判断身份、威胁、情绪还是规则。
- 先给总体印象，再给一到两个诊断性细节。
- 奇特食物、节日或服饰与人物行动绑定。
- 陌生且影响后续的特征得到足够说明。
- 熟悉且不影响判断的信息被概括。
- 未进入事件的世界资料暂留设定库。

## 期待来源

细节先提示人物或世界存在特殊规则，随后通过选择和后果证明；再次出现的短标签让读者获得识别、联想与回收满足。

## 断档风险

- 把稀奇程度误当叙事价值。
- 按身高、衣料、颜色逐项扫描。
- 配角和工具人获得同等描写预算。
- 细节删掉后人物与情节毫无变化。
- 非冲突章节只陈列风景，不改变任何状态。
- 世界设定增加一致性成本，却不参与故事。

## 训练任务

把一个奇特世界元素分别写成百科介绍和人物事件，事件版必须造成选择、误会或关系变化；再写同一配角的 80 字首登场与 20 字复现，只保留两个能预测其行动的记忆锚。
