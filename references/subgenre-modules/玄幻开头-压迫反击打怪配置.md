# 玄幻开头-压迫反击打怪配置

依据案例：`batch-177` 案例 882。

## 读者承诺

开头战斗必须提供爽点、奖励、反转、信息或情绪偿还，不能只用打怪制造热闹。

## 核心循环

```text
处境受压 -> 不公平伤害 -> 反击期待 -> 重点打怪 -> 爽点/奖励兑现
```

## 开篇必备

- 每个冲突都有明确功能。
- 同类打怪不要连续重复。
- 弱冲突可以删掉或合并。
- 不公平伤害要为后续反击蓄力。

## 期待来源

读者期待主角从受压到反击，战斗带来明确收益。重复打怪如果没有奖励和情绪偿还，只会稀释后面真正的大冲突。

## 断档风险

- 第一章就塞无功能打怪。
- 第二章重复同类事件。
- 战斗没有奖励、信息或反转。
- 看守冲突等人性压迫被普通怪物事件抢篇幅。

## 训练任务

列出开头前三章所有冲突，为每个冲突写一个功能。没有功能的冲突，删掉或并入功能更强的事件。
