# 仙侠模拟器-预知反杀与强敌压迫

依据案例：`batch-217` 案例 1084。

## 读者承诺

主角用模拟器提前看见死局，拆解强敌，现实布局，完成越级反杀。

## 核心循环

```text
危机出现 -> 模拟死局 -> 发现强点 -> 现实布置 -> 再模拟校验 -> 反杀兑现
```

## 开篇必备

- 冲突或金手指先行。
- 第一个敌人有压迫感。
- 模拟器参与解决危机。
- 背景信息打碎后置。

## 期待来源

读者期待主角靠预知和多重后手，把原本打不过的敌人一步步削弱到可反杀。

## 断档风险

- 单纯模拟没有新意。
- 黄纸前平铺太久。
- 敌人太简单，压不出爽点。
- 金手指只给情报，不参与翻盘。

## 训练任务

写冲突先行、金手指先行、模拟失败先行 3 版开头，每版包含死因、敌人强点、现实布置和保命后手。
