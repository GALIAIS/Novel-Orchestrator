# 读者需求-表达欲校准

依据案例：`batch-002` 案例 006。

## 读者承诺

允许作者写强表达、悲壮、虐感、孤独、疯批、换地图，但每次表达都必须回到读者需求：主角获得什么、守住什么、证明什么、改变什么。

## 核心循环

```text
题材需求确认 -> 作者表达冲动 -> 核心梗偏移检查 -> 表达降噪/收益补偿 -> 读者期待续接
```

## 开篇必备

- 明确题材读者最想得到的核心满足。
- 明确主角的可代入收益，而不是只给宏大使命。
- 明确关系网、目标、金手指或题材核心如何持续提供期待。

## 期待来源

- 读者相信主角的痛苦会换来意义和收益。
- 读者相信主角的成长能保护关系网。
- 读者相信题材核心不会被作者的自嗨表达冲毁。

## 断档风险

- 为悲壮而清空亲友。
- 为换地图而推倒前期投入。
- 为虐而虐、为刀而刀。
- 为致敬旧作或个人表达，破坏当前书的题材承诺。

## 训练任务

写一张“表达欲校准卡”：列出你最想写的一段强表达，再列出这段给读者的收益、补偿、关系变化和新期待。四项缺两项以上，就先改桥段。
