# 核心梗-书名简介开篇同向

依据案例：`batch-013` 案例 061。

## 读者承诺

书名、简介和开篇必须持续兑现同一个核心梗。读者为什么点进来，前三章就要为什么服务。

## 核心循环

```text
书名吸引 -> 简介强化 -> 开篇兑现 -> 支线回流 -> 核心梗反复变形满足
```

## 开篇必备

- 明确读者因哪个承诺点进来。
- 第一章就展示核心梗对应的情绪方向。
- 支线、事业线、人物关系都要服务核心梗。
- 主角行为不能反向背叛书名承诺。

## 期待来源

- 读者期待主角持续脱离旧压迫。
- 读者期待伤害者后悔、失败、被打脸。
- 读者期待核心梗换场景继续兑现。
- 读者期待承诺升级，而不是换题。

## 断档风险

- 前期写无关事业线，核心梗消失。
- 洗白核心矛盾对象，破坏读者站队。
- 需求空缺和高潮释放不对称。
- 作者被自己的人物感动，忘了读者是因承诺进来的。

## 训练任务

把书名、简介和前三章每章功能列成一张表。每章必须写出它兑现了哪个核心梗承诺，没有兑现的段落删改。
