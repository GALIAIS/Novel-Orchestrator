# 系统流开篇-危险任务与金手指可见

依据案例：`batch-209` 案例 1041。

## 读者承诺

系统流开篇要让主角被现实压力推入危险任务，并在冲突中看见金手指如何破局。

## 核心循环

```text
现实压力 -> 被迫接任务 -> 危险现场 -> 金手指生效 -> 收益/暴露钩子
```

## 开篇必备

- 主角必须接受冒险的硬理由。
- 第一章尽早进入危险任务。
- 金手指效果在事件中可见。
- 失败代价具体到钱、命、家人或身份。

## 期待来源

读者看到主角不是顺风顺水铺设定，而是在危险中靠能力获得活路和收益。

## 断档风险

- 前两章只有拾荒或日常。
- 金手指停留在设定说明。
- 冲突太晚或太轻。
- 主角没有非接不可的理由。

## 训练任务

写 800-1500 字开篇，让主角第一句就在危险任务现场，并用金手指解决一次具体危机。
