# 读者视角-作者信息差与商业自审

依据案例：`batch-253` 案例 1261。

## 读者承诺

读者视角模块让作者从自嗨信息里退出来，只按读者可见信息检查期待、吸引力和商业价值。

## 核心循环

```text
作者自审 -> 切读者视角 -> 切编辑视角 -> 找信息差和无聊点 -> 回改承诺
```

## 开篇必备

- 区分作者已知和读者可见。
- 每段都要有继续看的理由。
- 自问自己愿不愿意看。
- 再用编辑视角看签约价值。

## 期待来源

读者只凭正文决定要不要追，作者必须把脑内合理性转成可见承诺和情绪钩子。

## 断档风险

- 作者默认读者知道背景。
- 细纲完整但读者没有期待。
- 老书虫反馈被误当商业判断。
- 自我感动替代市场验证。

## 训练任务

用作者、读者、编辑三种视角检查一份细纲，每种视角列 5 个问题，并标出必须前置的信息。
