# 擦边卖点-审核风险与替代表达

依据案例：`batch-198` 案例 990。

## 读者承诺

擦边卖点会让读者等待违规兑现，长期审核风险高；更稳的是改成信任、保护欲、依赖和价值认同。

## 核心循环

```text
擦边钩子 -> 读者等待开车 -> 审核压制 -> 无法兑现 -> 关系变量替代
```

## 开篇必备

- 避免直白情欲数值。
- 关系变量可过审。
- 保留挫折和升级空间。
- 让价值认同推动关系。

## 期待来源

读者追的是关系变化、信任建立和共同目标，而不是单纯等尺度升级。

## 断档风险

- 卖点注定导向开车。
- 过审不代表长期安全。
- 一步到位导致写不长。
- 系统数值太直白。

## 训练任务

把好感/情欲系统改成信任/保护欲/价值认同系统，写前三章变量变化和挫折。
