# 写作工具-跨端安全选择

依据案例：`batch-142` 案例 709。

## 读者承诺

碎片化写作时，工具选择不只看手机电脑同步，还要按正文、灵感、大纲的重要性分层，确认导出、备份、平台稳定和隐私安全。

## 核心循环

```text
工作忙 -> 需要跨端码字 -> 列同步/大纲/安全需求 -> 区分写作资产 -> 选择工具并做备份
```

## 开篇必备

- 使用设备。
- 是否云同步。
- 正文存放方案。
- 大纲和灵感存放方案。
- 导出和备份方式。

## 期待来源

- 碎片时间可写。
- 手机电脑无缝接续。
- 核心大纲不被单点托管。
- 工具风险可控。

## 断档风险

- 只图方便，不做导出备份。
- 把核心大纲放在小众软件里。
- 小众工具停服。
- 隐私和稿件安全无法验证。

## 训练任务

改写一条工具求助帖，明确设备、平台、预算、是否云同步、是否离线备份。再列选择码字软件前必须确认的 5 个问题。
