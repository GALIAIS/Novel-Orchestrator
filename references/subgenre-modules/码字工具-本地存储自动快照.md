# 码字工具-本地存储自动快照

依据案例：`batch-321` 案例 1603

## 读者承诺

码字工具首先要不丢稿、少分心、可迁移；功能多少不如本地存储、自动快照和备份可靠。

## 核心循环

```text
选择设备 -> 本地存储 -> 自动快照 -> 外部备份 -> 减少分心 -> 稳定产出
```

## 开篇必备

- 明确主要码字设备。
- 确认可本地保存和导出。
- 有自动快照或版本回滚。
- 有外部备份。
- 工具停运时能迁移。

## 期待来源

作者期待工具降低摩擦，而不是因为云服务、停运或功能折腾影响写作。

## 断档风险

- 纯云存储无法本地保存。
- 软件停运导致付费和稿件流程受损。
- 工具太复杂，变成拖延。
- 手机码字让注意力分散。
- 没有备份和快照。

## 训练任务

检查自己的码字流程：本地副本、自动快照、外部备份、断网方案和迁移方案是否都有。
