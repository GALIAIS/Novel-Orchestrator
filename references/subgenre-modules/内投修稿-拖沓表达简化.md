# 内投修稿-拖沓表达简化

依据案例：`batch-145` 案例 722。

## 读者承诺

新人内投后发现稿件还能优化时，可以继续做低风险修稿，尤其是简化拖沓表达、删重复、压空泛旁白；流程焦虑不应阻止文本变好。

## 核心循环

```text
已投稿 -> 重读发现拖沓 -> 判断修改风险 -> 简化表达 -> 保留核心内容 -> 焦虑下降
```

## 开篇必备

- 已完成的投稿动作。
- 当前想改的问题。
- 修改是否改变核心设定或剧情。
- 是否需要等编辑明确要求。

## 期待来源

新人期待的是明确可执行的答复：能不能改、哪些能直接改、哪些需要同步。短而确定的答复能降低流程焦虑。

## 断档风险

- 把低风险文字简化拖成流程问题。
- 一边等编辑，一边停止自查。
- 修改核心卖点却不留版本。
- 把“简化表达”误改成删掉必要信息。

## 训练任务

拿一段 500 字旧稿，删改 10 处拖沓表达。只允许改词句密度，不允许改变剧情事实。最后写一句说明：改动是否影响核心设定、主线和人物动机。
