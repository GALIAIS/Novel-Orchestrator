# 支线回流-三成缓冲主线优先

依据案例：`batch-039` 案例 191。

## 读者承诺

支线用于缓冲和补强主线，不是另开无关故事。支线占比宜控制，完成后必须回到主线。

## 核心循环

```text
主线暂缓 -> 支线进入 -> 获得关系/资源/成长 -> 支线结束 -> 回主线
```

## 开篇必备

- 支线人物和主线有关。
- 支线功能清楚。
- 篇幅不喧宾夺主。
- 回主线触发点明确。

## 期待来源

- 支线带来人物关系。
- 支线提供主线所需资源。
- 主线紧迫感在回归时增强。
- 读者看到支线不是水文。

## 断档风险

- 支线与主线无关。
- 支线超过主线占比。
- 主线推进中硬插支线。
- 支线结束后拖着不回主线。

## 训练任务

写一个支线计划：说明主线为何暂缓、支线收益是什么、何时回主线。
