# 番茄拆榜-阅读追更取样

依据案例：`batch-029` 案例 141。

## 读者承诺

在免费平台拆书时，先找能代表读者行为的榜单入口，避免用付费畅销逻辑误选样本。

## 核心循环

```text
平台差异 -> 榜单识别 -> 老书看阅读 -> 新书看推荐/追更 -> 选样拆解
```

## 开篇必备

- 明确平台不是付费畅销逻辑。
- 区分阅读、追更、高分、推荐。
- 区分新书和老书。
- 标注样本可能的偏差。

## 期待来源

- 样本更接近真实读者选择。
- 新书能拆出当前平台推流偏好。
- 老书能拆出长期留存结构。
- 追更能观察持续钩子。

## 断档风险

- 只看评分不看样本阶段。
- 把推荐曝光误当稳定成绩。
- 老书新书混拆。
- 榜单入口错导致结论错。

## 训练任务

列 10 本番茄样本，按阅读、追更、高分、推荐分组，再分别决定拆开篇还是拆长线。
