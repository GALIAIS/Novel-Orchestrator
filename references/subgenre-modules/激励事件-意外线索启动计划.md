# 激励事件-意外线索启动计划

依据案例：batch-289 案例 1441

## 读者承诺

激励事件的核心是让主角不能继续停在原状态，意外线索和激烈冲突都可以承担这个功能。

## 核心循环

```text
原状态 -> 意外线索/冲突 -> 不得不行动 -> 形成计划 -> 执行受阻 -> 再次拉扯
```

## 开篇必备

- 主角原状态。
- 长期愿望或缺口。
- 打破现状的事件。
- 事件为何迫使行动。
- 行动计划第一步。
- 后续阻碍。

## 期待来源

读者看到主角从静止进入行动，并期待计划执行时如何遭遇阻碍和下行。

## 断档风险

- 把激励事件理解成只能写惨剧。
- 线索出现后主角仍能不动。
- 只有计划，没有后续阻碍。
- 定义抠太死，具体剧情应用反而僵。

## 训练任务

写一个“发现字条后出发”的开篇，三句话交代原状态、不得不行动的理由、第一处阻碍。
