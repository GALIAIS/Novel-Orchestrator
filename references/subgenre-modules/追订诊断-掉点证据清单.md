# 追订诊断-掉点证据清单

依据案例：batch-289 案例 1444

## 读者承诺

追订下滑要先收集掉点证据，只有一句“狂掉”时不能硬判原因。

## 核心循环

```text
发现追订下滑 -> 定位掉点章节 -> 对照剧情事件 -> 检查承诺断档/毒点 -> 修复并观察
```

## 开篇必备

- 掉订起始章节。
- 掉幅。
- 掉点前承诺。
- 掉点章节剧情。
- 掉点后新钩子。
- 读者反馈关键词。
- 修复验证指标。

## 期待来源

作者把模糊焦虑转成可诊断材料，才能知道是剧情断档、毒点、更新还是题材承诺问题。

## 断档风险

- 没数据就凭感觉改。
- 只问结果，不给正文和曲线。
- 把大神身份当诊断证据。
- 用通用建议替代掉点定位。

## 训练任务

补齐一张追订掉点表：章节、追订变化、剧情事件、读者评论、新期待是否接上。
