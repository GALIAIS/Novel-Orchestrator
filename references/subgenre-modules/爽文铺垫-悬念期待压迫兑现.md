# 爽文铺垫-悬念期待压迫兑现

依据案例：`batch-121` 案例 602。

## 读者承诺

爽文铺垫不是单一技巧，而是悬念、期待、压迫、爽点、反转的组合。每个工具都要回到情节折线，用拆书验证效果。

## 核心循环

```text
悬念留未知 -> 期待指定兑现 -> 压迫逼近危险 -> 爽点多角度释放 -> 反转改变局面
```

## 开篇必备

- 悬念要让读者知道现状会变。
- 期待要让读者知道等什么。
- 压迫要有必须马上成功的压力。
- 爽点要有结果和反应。
- 反转必须意外但合逻辑。

## 期待来源

- 未知事件带来的牵引。
- 读者等主角装逼或反击。
- 倒计时、强敌、陷害增强紧迫。
- 多方反应放大爽点。

## 断档风险

- 压迫拉太紧，读者只感到憋屈。
- 爽点角度单一，兑现不够。
- 反转只有意外，没有铺垫。
- 分类停在笔记，不用拆书验证。

## 训练任务

选一章正文，用五种标签逐段标注：悬念、期待、压迫、爽点、反转。缺哪一类，就从榜单书找同类桥段补一处。
