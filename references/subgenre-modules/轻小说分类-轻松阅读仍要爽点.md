# 轻小说分类-轻松阅读仍要爽点

依据案例：`batch-337` 案例 1681

## 读者承诺

轻小说可以偏日常、吐槽、恋爱和泛 ACG，但本质仍要提供低阻力阅读和明确爽点。

## 核心循环

```text
轻松风格 -> 讨喜人物 -> 小矛盾触发选择 -> 发糖/吐槽/脑洞/装逼兑现 -> 读者继续追关系
```

## 开篇必备

- 目标分类明确。
- 读者能感到轻松阅读。
- 第一章有爽点，不只是人设标签。
- 人物通过选择显形。
- 尺度红线先排掉。

## 期待来源

读者期待人物好玩、阅读轻快、情绪反馈快，并在轻冲突里看到糖点、吐槽、脑洞或装逼收益。

## 断档风险

- 把轻小说理解成无矛盾。
- 只写人物不写选择。
- 过度纠结日轻/国轻定义。
- 软色情或未成年暗示踩线。
- 人设只贴标签不落场景。

## 训练任务

列前三章的轻松点、爽点和人物选择；每章至少一处情绪反馈，删除所有尺度风险内容。
