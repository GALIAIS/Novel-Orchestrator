# 公开处刑修订-毒点逻辑与卖点适配

依据案例：`batch-250` 案例 1246。

## 读者承诺

公开处刑修订模块把读者反馈拆成可执行改项：冲突逻辑、爽点顺序、书名简介、文风适配和章节节奏。

## 核心循环

```text
读者反馈 -> 定位毒点和错位 -> 保留大剧情 -> 调整因果/爽点/包装 -> 再次验收
```

## 开篇必备

- 明确哪些大剧情保留。
- 列出读者具体卡点。
- 先修开局冲突因果。
- 让简介、书名、文风指向同一承诺。

## 期待来源

读者看到的是修订后的顺畅进入：冲突不再莫名其妙，卖点顺序更清楚，包装不再误导题材。

## 断档风险

- 只重写正文，不回应具体反馈。
- 大剧情保留，但开局毒点没拆掉。
- 简介太雅，读者看不出爽点。
- 字数变短，却没有提高事件密度。

## 训练任务

拿一个被反馈有毒的开头，列出 5 条反馈对应的 5 个修改动作。只改前三章事件顺序和包装，不改核心题材。
