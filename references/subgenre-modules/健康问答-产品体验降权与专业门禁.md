# 健康问答-产品体验降权与专业门禁

依据案例：`batch-396` 案例 1980

## 读者承诺

处理创作者健康问题时不把品牌体验伪装成医学建议：先收集症状和产品信息，分层标记证据，明显或持续异常交给合格医生或药师。

## 核心循环

```text
出现不适 -> 记录症状与使用经过 -> 个人体验降权 -> 核对标签但不自行诊断 -> 触发专业门禁 -> 回写非治疗性工作负荷调整
```

## 开篇必备

- 区分自我判断和已确认诊断。
- 记录开始时间、变化和单/双眼。
- 记录所用产品名称、成分与频率。
- 标出是否影响视力或伴随明显异常。
- 将网友舒适体验标为低等级证据。
- 明确停止社区试药并专业转介的条件。

## 期待来源

用户不再被品牌数量制造的假共识带偏，而是知道缺少哪些信息、哪些结论不能从帖子得出，以及下一步应交给谁判断。

## 断档风险

- 把自称无病当成诊断。
- 按评论出现次数制作品牌排名。
- 把舒服、清凉当成安全有效证据。
- 混淆产品类别、成分与适用范围。
- 用工作休息建议承诺治疗。
- 持续或加重异常仍继续自行试药。

## 训练任务

把健康帖评论分成个人体验、产品信息、风险主张、非药物建议和专业转介五栏；删除品牌优劣结论，重写成只收集信息、标证据和设置转介门禁的安全卡。
