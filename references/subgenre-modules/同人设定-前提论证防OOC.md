# 同人设定-前提论证防OOC

依据案例：batch-265 案例 1323

## 读者承诺

在同人、二创、群像辩论或价值观冲突中，让角色观点既有锋芒又不崩人设：每句判断都要回到设定事实、行动史和论证前提。

## 核心循环

```text
争论主题 -> 各方前提 -> 世界观事实 -> 行动史校验 -> 观点交锋 -> 不由作者强判输赢
```

## 开篇必备

- 先明确争论的是哪一个核心价值。
- 双方都要有能成立的前提。
- 角色的立场要能从经历中长出来。
- 作者不能偷换概念替某一方压人。

## 期待来源

期待来自熟悉角色的读者想看“他会怎么说、为什么这么说”。当每方都自洽，争论会增加人物厚度；当前提不成立，读者会立刻出戏。

## 断档风险

- 把角色偏见写成绝对真理。
- A 观点成立就强行判 B 错。
- 忽略原作行动史，导致 OOC。
- 辩论只有金句，没有推理过程。

## 训练任务

写一张角色争论证据表：每个角色列“立场、前提、原作证据、漏洞、不会说的话”。写完后再动笔写对话。
