# 输入转输出-开书条件评估

依据案例：`batch-079` 案例 392。

## 读者承诺

这个模块用于新人判断是否该从拆书、听课、看教程转向开书实战。它承诺用表达欲和时间两个硬条件，把“我还没准备好”的焦虑变成可执行判断。

## 核心循环

```text
持续输入 -> 出现空转感 -> 核对表达欲 -> 核对写作时间 -> 开书试错或继续训练
```

## 开篇必备

- 有明确想写的灵感。
- 有强烈表达冲动。
- 每天能拿出至少 4 小时写作。
- 已知道本次开书想验证哪种写法。
- 能接受试错后的复盘。

## 期待来源

- 拆书成果转成自己的题材方案。
- 开书目标从“成神”降到“验证方法”。
- 时间投入可持续。
- 试稿会暴露真实短板。
- 实战反馈反过来提升拆书效率。

## 断档风险

- 无限输入，永远不开写。
- 只有灵感，没有稳定时间。
- 只有时间，没有表达欲。
- 把第一本书当最终作品。
- 开书后不复盘，只靠热情硬冲。

## 训练任务

把最近拆书学到的一个方法迁移到自己的前三章。先写 300 字核心梗，再写 7 天试稿。7 天后只复盘一个问题：拆书方法是否真的进入了正文。
