# 书名简介-隐含信息悬念

依据案例：`batch-026` 案例 130。

## 读者承诺

书名用读者已有类型认知提供隐含信息，再抛一个反常悬念，让读者自动脑补爽点。

## 核心循环

```text
已有认知 -> 隐含信息 -> 反常点 -> 自动脑补 -> 简介确认方向 -> 正文兑现
```

## 开篇必备

- 读者能一眼识别的类型信号。
- 隐含的身份、金手指或场景。
- 一个“为什么”的反常悬念。
- 简介里确认爽点趋势。

## 期待来源

- 读者自己补空缺细节。
- 书名只给轮廓，简介引导方向。
- 反常点让读者想点开。
- 第一章兑现书名承诺。

## 断档风险

- 悬念没有认知基础，成了无根神秘。
- 简介写成设定说明。
- 作者觉得自己不看简介，就敷衍简介。
- 正文第一章不兑现书名期待。

## 训练任务

写 5 个书名，每个标出“隐含信息”和“反常问题”，再写一句简介确认读者应该脑补什么。
