# 书名简介-身份装逼方式显性化

依据案例：`batch-319` 案例 1591

## 读者承诺

书名和简介要让读者第一眼看出主角身份、反差悬念和装逼方式；仿写黄金三章时，结构必须服从自己的设定。

## 核心循环

```text
身份词入书名 -> 反差词制造疑问 -> 简介点出装逼方式 -> 开篇展示核心异常 -> 结构按本书设定重排
```

## 开篇必备

- 明确主角身份，不用泛泛的“我”让读者猜。
- 简介里给出核心爽点和装逼方式。
- 金手指名称与作用逻辑一致。
- 道德风险词要提前规避误判。
- 学范例只学原理，不搬不适配的结构。

## 期待来源

读者期待从书名简介直接判断这本书主角是谁、怎么爽、为什么有反差，前三章再验证这个承诺。

## 断档风险

- 身份不清，读者第一眼没抓手。
- 简介只有设定，没有爽点。
- 照搬爆款结构，反而压不住原创信息。
- 金手指逻辑怪，读者理解成本上升。
- 犯罪、洗白等词让读者误判主角底线。

## 训练任务

写 3 个含身份词的书名，再写 100 字简介，标出装逼方式、核心爽点和需要规避的道德误判。
