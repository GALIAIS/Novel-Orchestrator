# 开篇设定-危机中压缩

依据案例：`batch-156` 案例 776。

## 读者承诺

新人开篇不靠长背景吸引读者，而是把设定压缩到骨架，让危机、金手指展示和外部反馈先成立。

## 核心循环

```text
压缩背景 -> 危机入场 -> 金手指触发 -> 奖励脱险 -> 外部反馈
```

## 开篇必备

- 240 字以内的背景骨架。
- 当前危机。
- 不公平伤害或生死压力。
- 金手指触发点。
- 能力展示。
- 队友、领导或外界反馈。

## 期待来源

读者期待主角马上处理当前麻烦，并通过处理方式建立人设和能力承诺。

## 断档风险

- 前世、穿越原因、世界观解释占主菜。
- 危机出现太晚。
- 金手指只介绍不使用。
- 没有外部反馈给爽点。

## 训练任务

把第一章背景压到 240 字以内，再写一个包含危机、金手指触发、奖励脱险和外部反馈的开篇事件。
