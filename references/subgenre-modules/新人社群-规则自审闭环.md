# 新人社群-规则自审闭环

依据案例：`batch-154` 案例 766。

## 读者承诺

新人进入学习社群后，先读规则、搜旧帖、做自审，再求反馈；社群提供思路碰撞，不替作者完成判断。

## 核心循环

```text
读置顶 -> 搜关键词 -> 自审问题 -> 提交具体卡点 -> 讨论校准 -> 方法内化
```

## 开篇必备

- 明确学习规则。
- 明确已搜索的旧帖。
- 明确自己先做过的判断。
- 明确仍然卡住的具体点。
- 明确反馈后要沉淀的方法。

## 期待来源

期待来自作者从被动求评转为主动诊断：每次讨论都能提升自己的判断能力，而不是只拿一次性答案。

## 断档风险

- 不读规则直接求评。
- 不搜旧帖重复提问。
- 不自审，只让别人代工。
- 只记录结论，不沉淀方法。

## 训练任务

发帖前写一张自审表：已读规则、搜索关键词、旧帖结论、自判问题、已尝试改法、仍卡住的点、希望讨论的范围。
