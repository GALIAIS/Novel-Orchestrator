# 学习方法-去教条化实践试错

依据案例：batch-269 案例 1341

## 读者承诺

让教程和拆书服务写作实践，而不是把作者锁进唯一答案崇拜；通过多方法试错形成自己的构建方法论。

## 核心循环

```text
学习规则 -> 发现僵化 -> 放开唯一解 -> 多方法实践 -> 记录效果 -> 稳固个人方法
```

## 开篇必备

- 先识别自己正在死套哪条规则。
- 每次只试一个变量。
- 试错必须落到真实章节。
- 记录结果，不靠空想判断。

## 期待来源

期待来自作者重新获得弹性：同一个写作问题可以有多种解法，最终选择适合自己和作品的那一种。

## 断档风险

- 把教程当标准答案。
- 为了避错拒绝尝试。
- 只学习不写作。
- 每次失败都归因于没找到唯一解。

## 训练任务

选一个教程规则，写三版同场景：严格遵守、故意反向、折中改造。记录哪一版最适合当前作品。
