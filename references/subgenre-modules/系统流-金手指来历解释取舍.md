# 系统流-金手指来历解释取舍

依据案例：`batch-186` 案例 928。

## 读者承诺

系统来历默认不必解释；只有当系统是世界底层规则的一部分时，才值得投入篇幅解释。

## 核心循环

```text
系统功能 -> 判断是否绑定世界观 -> 不绑定则省略 -> 绑定则铺垫规则 -> 解释服务主线
```

## 开篇必备

- 系统功能先服务剧情。
- 不让来历解释拖节奏。
- 若解释，必须增加真相期待或爽点。
- 结尾轻解释可以接受。

## 期待来源

读者主要追过程剧情和功能兑现，不是为了系统说明书。

## 断档风险

- 解释一堆但不服务主线。
- 世界观闭环写不圆。
- 结局解释替代过程爽点。
- 设定说明压过人物行动。

## 训练任务

为一个系统写“不解释版”和“世界观绑定版”，比较哪个更能服务主线爽点。
