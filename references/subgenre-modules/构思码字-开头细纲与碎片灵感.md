# 构思码字-开头细纲与碎片灵感

依据案例：batch-381 案例 1903

## 读者承诺

前期构思不是偷懒，第一卷细纲和前 2-3 万字开头值得反复打磨。碎片灵感要落成细纲、正文或改稿点。

## 核心循环

```text
项目阶段 -> 构思分配 -> 开头细纲 -> 正文落地 -> 反馈修正
```

## 开篇必备

- 第一卷目标。
- 前 2-3 万字留存点。
- 每章冲突和钩子。
- 碎片灵感归档位置。

## 期待来源

- 开头承诺更准。
- 细纲降低正文阻力。
- 灵感被转成章节。
- 正文推进更稳。

## 断档风险

- 无限构思不落地。
- 硬性比例忽略项目阶段。
- 灵感只停在脑内。
- 开头不反复检查留存。

## 训练任务

记录一天：碎片时间只记灵感，正式写作时间把一个灵感转成细纲并写 1000 字正文。
