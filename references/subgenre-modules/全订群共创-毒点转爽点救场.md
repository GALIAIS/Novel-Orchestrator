# 全订群共创-毒点转爽点救场

依据案例：`batch-183` 案例 911。

## 读者承诺

全订群共创的承诺是用核心付费读者的熟悉度和脑洞，帮助作者解决卡文、转场、感情戏和毒点修补。

## 核心循环

```text
剧情卡点 -> 全订群提问 -> 读者共创解释 -> 作者筛选整合 -> 毒点转爽点
```

## 开篇必备

- 问题要具体：转场、感情、设定漏洞或毒点。
- 读者反馈要来自熟悉前文的核心读者。
- 作者保留取舍权。
- 修补后要在下一章验证。

## 期待来源

读者熟悉角色关系和前文承诺，可能提供作者没想到的情绪解释和戏剧转场。

## 断档风险

- 泛泛问“怎么写”。
- 被单个读者牵着走。
- 只收脑洞，不做结构整合。
- 把群聊整活直接塞进正文。

## 训练任务

列出一个转场卡点、一个感情戏短板和一个轻微毒点，让 3 条读者脑洞分别补救，再筛出一条改成下一章爽点。
