# 循环解密-真相欲望达成

依据案例：`batch-063` 案例 311。

## 读者承诺

这个模块用于《开端》式循环、重生试错、无限解密短中篇。它承诺的是让读者跟随主角反复失败、获得新信息、修正策略，最终完成生存、救人、真相揭露或遗憾修复。

## 核心循环

```text
灾难循环 -> 欲望未达成 -> 本轮行动失败 -> 新真相 -> 下轮策略变化 -> 阶段改变结果
```

## 开篇必备

- 循环或重来机制要尽快出现。
- 主角欲望必须具体：活下去、救人、阻止爆炸、找真相。
- 第一轮必须失败，并留下明确遗憾。
- 每一轮都要增加信息，而不是重复受苦。
- 读者要看见下一轮能改变什么。

## 期待来源

- 灾难为什么发生。
- 下一轮主角会换什么策略。
- 真相揭开后会发生什么。
- 主角能不能弥补上一轮遗憾。
- 人设在反复失败中成长。

## 断档风险

- 循环只重复同一事件。
- 解密没有行动压力。
- 真相太少，读者只看主角挨打。
- 短篇结构硬拖成长篇。
- 只学网剧外壳，不做网文章节钩子。

## 训练任务

写三轮循环梗概。每轮必须有失败点、新信息、策略变化和一个被改变的小结果；第三轮结束后接一个更大的新问题。
