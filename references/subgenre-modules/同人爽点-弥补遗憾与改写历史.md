# 同人爽点-弥补遗憾与改写历史

依据案例：`batch-242` 案例 1208。

## 读者承诺

同人文要让读者熟悉的原作遗憾、大事件和人物命运被主角重新改写，形成补偿爽点。

## 核心循环

```text
原作遗憾 -> 主角提前介入 -> 改写大事件 -> 角色命运变化 -> 原作粉补偿感
```

## 开篇必备

- 让读者认出原作基础。
- 标出将被改写的遗憾。
- 给主角信息差或介入能力。
- 保留原作大事件的期待压力。

## 期待来源

读者知道原本会发生什么，因此会期待主角如何提前布局、救人、改命或影响势力。

## 断档风险

- 只借 IP 名词，不改原作痛点。
- 改写太轻，读者没有补偿感。
- 主角介入点太晚。
- 原作人物反应不像原作。

## 训练任务

列 3 个原作遗憾，为每个写出原本结果、主角介入点和改写后的爽点。
