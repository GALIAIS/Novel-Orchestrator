# 类型定位-扫榜投稿别错类

依据案例：`batch-319` 案例 1593

## 读者承诺

先固定作品主类型，再去扫对应榜单、设置开篇信号和选择投稿分组，避免读者期待被题材乱窜打碎。

## 核心循环

```text
确定主类型 -> 扫同类榜单 -> 开篇释放类型信号 -> 投稿匹配分组 -> 稳定兑现读者期待
```

## 开篇必备

- 主类型明确。
- 副元素不抢主类型。
- 前三章出现足够强的类型信号。
- 混搭规则提前说明。
- 不拿幸存者偏差指导新人。

## 期待来源

读者会根据仙侠、玄幻、都市、科技、灵异等开篇信号生成期待，后文必须按这个期待提供爽点。

## 断档风险

- 扫榜扫错，学错节奏。
- 投稿投错组，入口不匹配。
- 缝合元素越来越多，大纲失控。
- 机器人外壳写成普通修仙，读者期待落空。
- 中期突然换题材，读者流失。

## 训练任务

给作品写类型承诺卡：主类型、副元素、前三章类型信号、扫榜对象、投稿分组和不能乱加的元素。
