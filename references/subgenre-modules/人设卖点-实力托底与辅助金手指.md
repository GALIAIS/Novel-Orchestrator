# 人设卖点-实力托底与辅助金手指

依据案例：`batch-221` 案例 1102。

## 读者承诺

主打人设的文也需要实力托底，让人设有发挥空间。

## 核心循环

```text
实力托底 -> 人设发挥 -> 冲突解决 -> 人设加深 -> 长期续航
```

## 开篇必备

- 明确主打人设。
- 设计不抢戏的实力来源。
- 能力服务人设，而不是反客为主。
- 第一次冲突用人设解决。

## 期待来源

读者期待主角明明有能力，却按独特人设处理问题，形成反差和记忆点。

## 断档风险

- 人设主角太弱，变成怂。
- 系统或任务抢走最大卖点。
- 辅助能力无存在价值。
- 标题简介和正文卖点错位。

## 训练任务

设计 3 个主打人设的主角，每个写实力托底、辅助能力和前三章人设解决冲突事件。
