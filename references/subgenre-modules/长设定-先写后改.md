# 长设定-先写后改

依据案例：`batch-139` 案例 691。

## 读者承诺

把复杂设定、规则边界或组织机制先写成可修改的文本，避免作者在脑内追求一次成型，最后因为篇幅和难度卡死。

## 核心循环

```text
发现设定很长 -> 先写完整粗稿 -> 标出读者马上需要的部分 -> 按剧情需求删改 -> 后置非必要说明
```

## 开篇必备

- 设定为什么必须存在。
- 主角第一次接触该规则的场景。
- 读者马上需要理解的一条限制。
- 该规则能制造的直接冲突。

## 期待来源

- 作者有可修改对象。
- 设定不再只停在脑内。
- 读者先获得最小可读信息。
- 后续规则能在剧情中逐步释放。

## 断档风险

- 写前就嫌长，导致设定永远不落地。
- 一次性把全部背景塞给读者。
- 只解释规则，不展示规则制造的冲突。
- 为了压缩删掉读者马上需要的信息。

## 训练任务

选一个世界观规则写 800 字粗稿。写完后用三种颜色或标签标注：马上要用、后续再说、可以删除。最终保留“马上要用”的内容，改成 200 字以内的剧情内说明。
