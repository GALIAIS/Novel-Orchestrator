# 原神同人-遗憾补完剧情向

依据案例：`batch-151` 案例 754。

## 读者承诺

剧情向同人也要满足同人核心爽感：原作角色与主角产生新互动，原作遗憾被弥补，设定空白被合理扩展，读者看到另一种可能。

## 核心循环

```text
原作遗憾 -> 主角介入 -> 角色互动 -> 遗憾补完 -> 奖励/人脉 -> 后续伏笔
```

## 开篇必备

- 熟悉原作的读者遗憾。
- 一个高人气角色登场互动。
- 不 OOC 的好感或关系推进。
- 原作设定空白的扩展点。
- 主角因此获得的奖励、人脉或伏笔。

## 期待来源

读者期待补偿：喜欢的角色有新可能，遗憾剧情被改写，原作没展开的角色能力、谋略、关系被补完。

## 断档风险

- 自称非桃文却完全忽视角色关系爽点。
- 只复刻原作，不提供新事件。
- 遗憾补完没有转成主角奖励。
- 过渡章无反转、无装逼、无关系推进。
- 扩展设定导致角色 OOC。

## 训练任务

列 3 个原作遗憾，每个设计一个事件：主角如何介入、角色如何反馈、遗憾如何改写、主角获得什么奖励或人脉。
