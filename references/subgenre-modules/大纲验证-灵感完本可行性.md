# 大纲验证-灵感完本可行性

依据案例：`batch-106` 案例 527。

## 读者承诺

大纲不是形式，而是发书前验证创意能否写长、写完、不崩的工具。过稿和写出开头都不等于应该马上发。

## 核心循环

```text
灵感 -> 核心梗 -> 简纲 -> 粗纲 -> 设定落地 -> 细纲 -> 可行性判断
```

## 开篇必备

- 一句话核心梗。
- 300-500 字简纲。
- 1000-5000 字粗纲。
- 世界观和人物线。
- 阶段事件和人物弧光。

## 期待来源

- 写前发现会不会崩。
- 判断创意是否能支撑完本。
- 减少开书后断档。
- 让正文更稳。

## 断档风险

- 过稿后急着发。
- 开头几万字能写，后面写空。
- 大纲做完就失去兴趣。
- 设定没有落到事件。

## 训练任务

把一个脑洞推进到粗纲。写完后如果不想写正文，就记录它为什么不足以成为商业长篇。
