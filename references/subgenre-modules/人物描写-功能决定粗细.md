# 人物描写-功能决定粗细

依据案例：`batch-030` 案例 147。

## 读者承诺

人物描写按角色功能和题材节奏取舍，工具人快速识别，重要角色才给特写。

## 核心循环

```text
角色功能 -> 场景重点 -> 总体印象 -> 动作结果 -> 必要特写
```

## 开篇必备

- 判断角色是否重要。
- 判断当前重点是剧情还是关系。
- 先给总体印象。
- 外貌只抓一个记忆点。

## 期待来源

- 读者扫一眼就知道人物作用。
- 重要角色通过重复特写加深记忆。
- 恋爱文用描写制造吸引力。
- 特殊见面场景提供长期回收点。

## 断档风险

- 形容词堆叠。
- 把外貌插进动作导致句子混乱。
- 工具人描写过细。
- 恋爱文重要人物描写太粗。

## 训练任务

同一人物写两版登场：快节奏剧情版不超过 50 字，恋爱重点版突出一个审美特写。
