# 不公平伤害-类型适配限度

依据案例：`batch-078` 案例 388。

## 读者承诺

这个模块用于判断开篇或分卷是否需要不公平伤害。它承诺把不公平伤害当工具，而不是铁律；适合就用，不适合就换成脑洞、金手指或目标驱动。

## 核心循环

```text
类型判断 -> 是否需要共鸣下行 -> 配合剧情使用 -> 上行兑现 -> 避免堆叠腻烦
```

## 开篇必备

- 明确题材是否适合拉仇恨。
- 不公平伤害要服务主线。
- 已有全文级伤害时少堆同类。
- 下行后必须有上行出口。
- 脑洞文可不用这套。

## 期待来源

- 读者替主角不平。
- 主角证明自己。
- 不公引出悬念。
- 伤害和主线目标绑定。
- 不用不公时，用设定新奇和金手指补期待。

## 断档风险

- 为伤害而伤害。
- 连续下行。
- 同类型不公堆太多。
- 类型不适配仍硬套。
- 只有仇恨，没有剧情用途。

## 训练任务

给一个脑洞开篇做适配判断：写“使用不公平伤害版”和“不使用版”，比较哪一版更符合目标平台与题材口味。
