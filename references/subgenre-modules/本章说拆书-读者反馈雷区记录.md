# 本章说拆书-读者反馈雷区记录

依据案例：batch-356 案例 1779

## 读者承诺

拆书时看本章说，可以捕捉读者真实反应。高讨论段落可能是毒点、共鸣或疑问，继续追踪作者处理方式，就能沉淀自己的雷区和补救方案。

## 核心循环

```text
章节阅读 -> 本章说高反应点 -> 判断毒点/共鸣 -> 追踪后续处理 -> 记录雷区和补救
```

## 开篇必备

- 章节和段落位置。
- 本章说集中反应。
- 毒点/共鸣/疑问分类。
- 作者后续处理方式。

## 期待来源

- 读者反馈暴露真实雷区。
- 大神的小问题也可学习。
- 共鸣点能反推有效写法。
- 补救方式可迁移到自己正文。

## 断档风险

- 只看正文，不看读者反应。
- 把本章说当无效噪音。
- 只记毒点，不看作者如何补救。
- 三本轮拆变成浅尝辄止，没有记录表。

## 训练任务

拆一本热书 5 章，每章记录 2 个本章说高反应点，标明毒点/共鸣/疑问，并写作者后续是否处理。
