# 开头平淡-穿越前置与十八标准自检

依据案例：`batch-237` 案例 1185。

## 读者承诺

编辑说开头平、没代入感时，先用开头 18 标准查结构；穿越前铺垫若删掉不影响主线，就从真正故事起点重开。

## 核心循环

```text
编辑反馈平淡 -> 18标准自检 -> 删除无效前置 -> 重定故事起点 -> 强化第一章承诺
```

## 开篇必备

- 明确第一章核心事件。
- 测试穿越前内容能否删除。
- 把主角困境、目标、变化提前。
- 用 18 标准逐项修开头。

## 期待来源

读者进入第一章就看到主角处境变化、核心冲突和后续期待，而不是先读一段可有可无的穿越前因果。

## 断档风险

- 把编辑反馈理解成文笔问题。
- 穿越前铺垫过长。
- 第一章删掉也不影响主线。
- 救人而死等通用前置占用太多篇幅。

## 训练任务

做一次删除测试：删掉穿越前全部内容，只从新世界第一个困境开始读；若故事成立，就把前置压缩到 0-300 字或删除。
