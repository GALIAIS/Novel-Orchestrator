# 故事架构-拆整本训练

依据案例：`batch-147` 案例 734。

## 读者承诺

故事架构不是速成模板，而是理论、拆书、章纲、反馈和实写循环训练出来的能力。拆整本书比自己写完整本更快，是高效训练入口。

## 核心循环

```text
理论学习 -> 对照教程 -> 大纲章纲 -> 公开反馈 -> 码字验证 -> 遇阻提问 -> 拆整本
```

## 开篇必备

- 先读基础理论。
- 做出可反馈的大纲和章纲。
- 找一本喜欢的书慢慢拆。
- 遇到阻碍再提问。
- 接受架构能力长期精进。

## 期待来源

新人期待的是可走的训练路径：先建立框架，再拆完整作品，再回到自己的章纲和正文里验证。

## 断档风险

- 只找万能教学，不产出大纲。
- 只看理论，不拆具体书。
- 只拆开头，不看整本承接。
- 写失控后不复盘结构。

## 训练任务

连续拆一本喜欢网文的 30 章，只记录每章目标、阻碍、冲突、结尾钩子和下章承接。拆完后写自己的前 10 章章纲。
