# 游戏穿越开篇-身份锚定与首战信息减负

依据案例：`batch-407` 案例 2031

## 读者承诺

可以暂缓确认游戏或现实真相，但先稳定主角当前信念、身体空间、立即危险和验证动作；首战只处理一个目标与一条敌人规则，用简洁胜负偿还开篇信息债。

## 核心循环

```text
陌生醒来 -> 当前信念与锚点 -> 玩家检查控制退出和旧经验 -> 一项规则失效 -> 单目标首战 -> 新能力简洁解法 -> 战后答案与分层设定
```

## 开篇必备

- 主角以为什么、读者看到什么分开。
- 身份、空间、危险和当前行动有锚点。
- 老玩家主动测试系统和经验差异。
- 行为偏离玩家常识时给出原因。
- 首战只有一个主要战术问题。
- 复杂历史、术语和反派塑造按需后置。

## 期待来源

现实真假保持悬念，当前行动却始终可懂；旧攻略失效把熟练玩家推入危机，新能力以一次清楚解法建立后续探索。

## 断档风险

- 身份、记忆、系统、怪物和世界同时未知。
- 只宣布老玩家却不展示检查顺序。
- 首战同时承担多线智斗和世界说明。
- 专有名词先于用途出现。
- 铺垫章节没有局部目标和回报。
- 机械复制他人铺垫与爽点章数比例。

## 训练任务

用六拍重写：醒来锚点、玩家测试、异常、单目标首战、经验失效、新能力解法；让陌生读者复述主角信念、验证过程和胜因，答不出就继续减负。
