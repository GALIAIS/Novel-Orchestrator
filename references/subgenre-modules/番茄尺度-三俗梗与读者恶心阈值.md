# 番茄尺度-三俗梗与读者恶心阈值

依据案例：`batch-315` 案例 1573

## 读者承诺

三俗梗要同时过平台审核和读者体验两道关。能写不代表该写，不违规也可能恶心读者。

## 核心循环

```text
想用低俗梗 -> 判断审核风险 -> 判断恶心阈值 -> 确认喜剧/人设功能 -> 保留或删除
```

## 开篇必备

- 先问梗是否服务剧情。
- 擦边亲密内容要考虑审核修改。
- 屎尿屁等生理梗要考虑读者反感。
- 没有功能的低俗梗直接删。

## 期待来源

读者期待轻松、爽、好笑，而不是被无意义生理梗打断阅读。

## 断档风险

- 把尺度试探当卖点。
- 作者自嗨，读者反胃。
- 审核能过，但留存下降。
- 低俗梗破坏人物质感。

## 训练任务

把一个三俗段落分成保留和删除两栏：保留项必须写明喜剧、人设或冲突功能。
