# 扫榜拆书-合情爽点优先于合理

依据案例：`batch-314` 案例 1567

## 读者承诺

扫榜拆书不是证明自己设定合理，而是校准读者真正会追的东西：流畅、反差、猎奇、新颖、爽点密度。

## 核心循环

```text
扫榜样本 -> 对照自家大纲 -> 删除过度解释 -> 保留合情流畅 -> 提高爽点密度
```

## 开篇必备

- 有突出的设定亮点。
- 有差异化反差。
- 反转服务爽点，不服务辩论。
- 必要逻辑点到为止。
- 解释不能挤占推进。

## 期待来源

读者期待故事顺畅推进并获得愉悦感。只要合情、不踩雷、爽点完成，细枝末节不必写成论文。

## 断档风险

- 设定四平八稳，没有钩子。
- 为了完全合理，把桥段解释拖慢。
- 作者沉迷解决漏洞，读者只觉得啰嗦。
- 反转合理但不刺激。

## 训练任务

选一个桥段，删掉所有“证明合理”的句子，只保留读者理解冲突和获得爽点所需的信息。
