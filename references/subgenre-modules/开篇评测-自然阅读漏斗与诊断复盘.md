# 开篇评测-自然阅读漏斗与诊断复盘

依据案例：`batch-414` 案例 2069

## 读者承诺

开篇评测同时提供真实行为证据和可执行诊断，却不让审稿任务污染自然阅读。第一阶段按目标受众分层，无说明、不要求读完地记录标题点击、第一屏继续、扫读、退出、复述和下一章意愿；第二阶段才询问感受、原因和修改建议。作者保留已经有效的标题、首句、人物和基调，每轮只修改一个主变量，再用相同漏斗复测。

## 核心循环

```text
声明目标读者和核心承诺 -> 分层招募目标与非目标样本 -> 无说明展示包装并记录瞬间联想 -> 无完成义务地盲读正文 -> 时间戳记录继续、扫读、退出、复述和追读 -> 第二阶段询问感受与原因 -> 将症状、解释和处方分栏 -> 锁定多人重复症状并保护有效资产 -> 单变量修改 -> 用同结构样本复测并与编辑门槛分开判断
```

自然行为回答“哪里失去读者”，诊断回答“可能为什么”，复测回答“修改是否真的有效”；三者不能由一份挑错清单替代。

## 开篇必备

- 明确的目标题材、目标受众和排斥元素。
- 测试者常读题材、阅读经验和当前阅读场景。
- 标题、封面、简介的第一联想与点击意愿。
- 第一屏和第一场景的继续阅读节点。
- 首次扫读位置、退出句、时间和即时情绪。
- 读后一句话复述及是否主动索要下一章。
- 症状、原因解释、修改处方三个独立字段。
- 不可误删的有效资产清单。
- 本轮唯一修改变量和复测样本结构。
- 编辑门槛与目标读者留存两套结果。

## 期待来源

- 包装联想能否在第一屏得到确认。
- 强首句建立的语气能否持续到第一场结果。
- 读者何时从精读转为扫读。
- 退出前最后理解的目标和问题是什么。
- 不同目标读者是否在同一位置重复流失。
- 单变量修改能否提高追读而不损坏点击率。
- 编辑式问题是否真的影响目标读者行为。

## 断档风险

- 要求测试者必须读完并给建议，行为数据失真。
- 不记录读者口味，把所有退出简单平均。
- 先问理论原因，测试者事后改写即时反应。
- 把单个读者的处方当作重复症状。
- 公开点评只找缺点，不记录有效资产。
- 一轮同时改标题、设定、人物和节奏，无法归因。
- 用编辑合格代替目标读者追读，或反过来。
- 直接要求新人复制名作整体，没有可执行中间机制。

## 训练任务

用五名目标读者和三名非目标读者完成一次两阶段测试。第一阶段不给说明和完成义务，只记点击、扫读、退出、复述和追读；第二阶段再分栏收集症状、解释和处方。选择一个多人重复症状做单变量修改，同时锁定两项有效资产，随后用同规模新样本复测。
