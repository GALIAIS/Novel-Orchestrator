# 矛盾网-主次升级

依据案例：`batch-004` 案例 018。

## 读者承诺

让长篇剧情不是换皮打怪，而是主要矛盾和次要矛盾交替主导、自然升级，形成可持续卷纲。

## 核心循环

```text
主要矛盾压迫 -> 次要矛盾铺垫 -> 主要矛盾解决 -> 次要矛盾接位 -> 新矛盾生长
```

## 开篇必备

- 当前主要矛盾。
- 主角无法后退的条件。
- 明确赌注和风险。
- 至少两个暗中升级的次要矛盾。

## 期待来源

- 读者知道主角必须面对冲突。
- 冲突能升级，且升级有逻辑。
- 每个矛盾解决后都会留下新局面。

## 断档风险

- 矛盾本身单一，变成换皮怪。
- 解决方式单一，读者提前猜完。
- 次要矛盾突然跳脸。
- 主角明明有更好选择，却被作者硬推。

## 训练任务

写一个 10 万字卷纲矛盾网：1 个主要矛盾、3 个次要矛盾、每个矛盾的升级点和接位点。
