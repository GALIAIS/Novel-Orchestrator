# 卷纲结构-三事件危机递进

依据案例：`batch-088` 案例 436。

## 读者承诺

一卷用三个递进事件持续兑现期待：每个事件都有短期危机、核心危机、伪胜利、下行和翻盘，卷首预告最终危机，卷尾集中兑现。

## 核心循环

```text
最终危机预告 -> 事件短危机 -> 伪胜利 -> 意外下行 -> 金手指翻盘 -> 对手和收获升级
```

## 开篇必备

- 卷首要露出最终危机。
- 每个事件都有可先解决的短期危机。
- 大危机不能被短期胜利消灭。
- 伪胜利后要有一次下行。
- 翻盘后收获必须逐层提高。

## 期待来源

- 强敌实力悬殊。
- 短期危机能否解决。
- 伪胜利后会怎样崩盘。
- 金手指如何真正翻盘。
- 下一事件对手和收获如何升级。

## 断档风险

- 框架太标准，写出来像模板。
- 只解决矛盾，忘了爽点延伸。
- 主线太密，缺少支线舒缓。
- 三个事件重复度太高。

## 训练任务

写一卷三事件大纲。每个事件写短期危机、核心危机、伪胜利、下行、翻盘和收获，再补两个支线舒缓点。
