# 日更追读-小篇幅爽点供给

依据案例：`batch-257` 案例 1284。

## 读者承诺

日更追读模块以天为单位安排小篇幅爽点，让读者每天点开都有即时满足。

## 核心循环

```text
每日更新 -> 小爽点兑现 -> 即时满足 -> 追读习惯稳定 -> 不养书
```

## 开篇必备

- 每天至少有一个可感知吸引点。
- 小爽点可在短篇幅内完成。
- 主线铺垫不能连续多天无回报。
- 旁人崇拜、惩戒恶人、优越感都可做日常爽点。

## 期待来源

读者每天打开更新，都能获得“今天没白看”的回报，于是维持阅读习惯。

## 断档风险

- 多日只有铺垫。
- 读者决定养书，习惯断裂。
- 小爽点和主角优势无关。
- 不爆更又不提供短回报。

## 训练任务

列一周 7 个小篇幅爽点，每个爽点 1000 字内完成，并写出当天读者获得的即时满足。
