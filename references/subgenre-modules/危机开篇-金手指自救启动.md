# 危机开篇-金手指自救启动

依据案例：batch-295 案例 1475

## 读者承诺

生存危机开篇让主角求助失败、被迫自救，并用金手指完成第一次破局，尽早展示核心卖点。

## 核心循环

```text
外部死亡压力 -> 第一计划失败 -> 主角主动行动 -> 金手指启动 -> 反杀/逃生 -> 能力前景展示
```

## 开篇必备

- 主角身处真实危险现场。
- 求人、找关系、等救援等方案失败。
- 主角有主动动作，不只是被动等待。
- 金手指解决当下问题，并暗示未来成长。

## 期待来源

- 读者看到主角在绝境里亲手破局。
- 金手指既有当前效果，也有后续升级空间。
- 前三章连续放大能力用途。

## 断档风险

- 危机只停留在背景介绍，主角不在现场。
- 主角靠别人给资格，主动权丢失。
- 金手指迟迟不出现，开篇卖点缺位。

## 训练任务

写三章开篇钩子：第一章求助失败并反杀，第二章用新能力扩大优势，第三章展示升级目标。
