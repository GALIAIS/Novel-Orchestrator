# 刺猬猫签约-首次审核等待

依据案例：`batch-138` 案例 687。

## 读者承诺

刺猬猫首次签约等待偏慢不一定异常。作者需要知道可能原因、能做的动作和哪些信息不能当官方时限。

## 核心循环

```text
首次签约 -> 编辑审核 -> 人手/审核压力 -> 等待 -> 后续直接沟通
```

## 开篇必备

- 是否首次签约。
- 当前等待时长。
- 编辑人手少。
- 审核忙可能拖慢。
- 后续动作不臆造。

## 期待来源

- 等待焦虑被解释。
- 作者知道下一步。
- 不把延迟误判为异常。
- 后续沟通路径更清楚。

## 断档风险

- 把三天当固定规则。
- 以为所有签约都一样慢。
- 编造官方时限。
- 审核期仍催错对象。

## 训练任务

写 200 字签约等待 FAQ：说明首次签约为什么可能慢、作者能做什么、哪些不能当官方承诺。
