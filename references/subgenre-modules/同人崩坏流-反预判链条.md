# 同人崩坏流-反预判链条

依据案例：`batch-322` 案例 1607

## 读者承诺

同人崩坏流靠打破原作秩序和连续反预判制造新鲜感，但必须建立在作者熟悉原套路、读者也熟悉原套路的前提上。

## 核心循环

```text
原套路 -> 读者预判 -> 作者反套路 -> 读者再预判 -> 再反转 -> 崩坏原作秩序
```

## 开篇必备

- 目标读者熟悉原作或套路。
- 作者知道读者会怎么预判。
- 反套路不能只是乱写。
- 书名或开篇先释放不按常理的信号。
- 只学局部操作，不盲目全盘复制高难度结构。

## 期待来源

老书虫期待自己被文本“秀一脸”，看到作者处理预判链的能力。

## 断档风险

- 作者不懂套路就反套路。
- 崩坏原作秩序但没有爽点回收。
- 操作太骚，普通读者跟不上。
- 新人照搬高难度反预判链条。

## 训练任务

写一个同人套路的四步反预判：原套路、读者第一预判、作者反套路、第二次反转。
