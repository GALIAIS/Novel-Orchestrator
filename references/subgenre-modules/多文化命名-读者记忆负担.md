# 多文化命名-读者记忆负担

依据案例：`batch-162` 案例 806。

## 读者承诺

在多文化世界观中降低姓名割裂感和读者记忆负担，让名字服务角色识别，而不是炫设定。

## 核心循环

```text
确定主命名体系 -> 少量引入异文化名 -> 绑定行为/关系/阵营 -> 降低记忆成本 -> 稳定识别
```

## 开篇必备

- 前几章优先使用一种主姓名体系。
- 异文化名字要短、好读、可记。
- 名字差异要对应文化、阶层、阵营或角色功能。
- 不要一次性抛出大量不同语系名字。

## 期待来源

读者期待通过名字快速判断角色位置、关系和气质，而不是反复停下来辨认谁是谁。

## 断档风险

- 世界观解释合理，但读者读起来费劲。
- 名字混杂却没有叙事功能。
- 异文化角色行为和名字来源不匹配。
- 多个复杂外文名同时出场。

## 训练任务

整理 20 个角色名，统一 80% 到主命名体系。剩余异文化名字必须写出识别功能，否则改名。
