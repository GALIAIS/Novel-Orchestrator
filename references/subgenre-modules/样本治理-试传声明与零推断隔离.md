# 样本治理-试传声明与零推断隔离

依据案例：`batch-414` 案例 2066

## 读者承诺

知识库使用者可以确认每个源样本都被人工查看，却不会因为批次必须产出内容而看到凭空制造的写作规律。作者明确声明“仅测试、无内容”时，系统保留路径、编号、媒体描述和排除理由，把“已查看”与“可归纳”分开；样本连续性不受破坏，方法库、训练集和推荐排序也不受空内容污染。

## 核心循环

```text
候选入队 -> 逐项查看正文、评论、附件与图片 -> 核对作者是否明确声明试传或无内容 -> 区分零内容、行政通知、资源索引、残缺证据和低质量观点 -> 记录可见证据与禁止推断 -> 保留来源和账本编号 -> 将零内容分析权重设为零并隔离出综合集 -> 新证据出现时重新开启审计
```

该循环的完成条件不是强行生成方法，而是内容状态可追溯、综合权重正确、任何排除都能由源证据解释。

## 开篇必备

- 源路径、topic_id、标题和账本编号。
- 正文、评论、附件、图片四类范围的人工查看记录。
- 作者对“测试、空内容或行政用途”的明确声明。
- 内容状态与证据状态两个独立字段。
- “已人工查看”和“可用于归纳”两个独立布尔值。
- 图片或表情的可见描述与帖内功能。
- 分析权重、排除用途和保留用途。
- 将来恢复审计所需的新证据条件。

## 期待来源

- 使用者能追溯编号为何存在而没有方法结论。
- 同一来源后来出现真实附件时能够恢复处理。
- 零内容不会被误算成反面教材或低质量观点。
- 表情图、标题和文件名不会被当作正文替代品。
- 综合统计只使用具备实质证据的样本。
- 人工流程能够明确说出“没有可拆内容”。

## 断档风险

- 为保持一篇一模块而把表情图扩写成创作理论。
- 删除空样本导致源顺序与账本编号无法对齐。
- 把零内容归为低质量作品，产生错误负面标签。
- 只看正文声明而不核对是否真的存在附件。
- 把“已查看”直接等同于“可训练、可推荐”。
- 排除样本却不写理由，后续无法复核。
- 新附件出现后仍沿用旧的零权重状态。
- 用脚本按字数自动判空，忽略资源索引和短而有效的观点。

## 训练任务

人工选取十条低信息资料，逐条查看四类证据，分别标成零内容、行政通知、资源索引、残缺证据或低质量观点。为每条填写权重、保留用途、排除用途和恢复条件；禁止依据字数自动分类，也禁止从标题、表情和未读附件补造结论。
