# 开篇危机-一次性设定用完即扔

依据案例：batch-290 案例 1449

## 读者承诺

稳定世界难造生死危机时，可以用一次性高压设定先把主角压进求生局，再后续淡化或补丁。

## 核心循环

```text
稳定世界 -> 一次性高压规则 -> 生死危机 -> 主角求生 -> 读者入局 -> 后续补丁
```

## 开篇必备

- 主世界稳定状态。
- 主角起点。
- 一次性高压势力/规则。
- 不公平伤害。
- 生死目标。
- 后续淡化或补丁方式。

## 期待来源

读者先被残酷规则和不公平伤害拉住，再跟着主角从危机里杀出来。

## 断档风险

- 一次性设定和主线世界观硬冲突。
- 用完不补丁却反复影响主线。
- 只制造危机，没有主角选择。
- 把开篇特例当长期规则。

## 训练任务

给一个稳定世界加一条边缘试炼规则，用 300 字写主角如何被迫进入生死局。
