# 开篇钩子-先抓人再布局

依据案例：`batch-330` 案例 1650

## 读者承诺

新人开头先用钩子抓住读者，再边讲故事边布局；没有当下，就没有后面的未来。

## 核心循环

```text
低耐心读者 -> 第一段钩子/惊爆点 -> 停留阅读 -> 故事推进 -> 布局展开
```

## 开篇必备

- 第一眼有异常、危机或惊爆点。
- 钩子制造明确疑问。
- 主角迅速被卷入。
- 解释和世界观后置。
- 第一章结尾再拉一次期待。

## 期待来源

读者立刻想知道“怎么回事、会怎样、主角怎么过关”，从而愿意进入后续故事。

## 断档风险

- 娓娓道来但没有当下吸引力。
- 把钩子误当成设定说明。
- 开头藏着掖着，读者等不到亮点。
- 只博眼球，不接故事。
- 书名简介和正文钩子不一致。

## 训练任务

重写第一千字：第一段出现异常/危机/惊爆点，背景解释全部延后到读者已经被钩住之后。
