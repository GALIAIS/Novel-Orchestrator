# 空间金手指开篇-封闭试炼与外界窗口

依据案例：`batch-413` 案例 2061

## 读者承诺

主角在一个既是牢笼、也是未来能力原型的封闭空间中醒来，必须在生存条件恶化前亲手找出规则并脱困。读者得到的不是系统说明书，而是一场可验证的生存解谜：穿越前技能改变观察和试验，错误产生真实代价，少量外界窗口持续预告更大的玄幻世界，出口选择则把本场后果直接交给外部冲突。

## 核心循环

```text
生存条件恶化 -> 观察一项异常 -> 依靠个人技能提出规则假设 -> 行动验证并支付成本 -> 反馈修正规则 -> 外界窗口增加身份或势力线索 -> 组合规则制造出口 -> 脱困后果引发外界冲突 -> 新场景继续检验已学能力
```

循环的稳定项是“目标—试验—反馈—选择”，不是“系统解释—主角照做”。空间每揭示一项功能，都要同时改变主角当前状态和读者对外部世界的判断。

## 开篇必备

- 一个即使没有金手指教学也成立的生存问题。
- 清楚的倒计时、资源损耗或失败后果。
- 至少三次可追溯的观察、假设、试验和反馈。
- 一次会留下伤势、资源损耗或禁区的失败试验。
- 穿越前技能在关键证据上的具体因果作用。
- 两个以内兼具当前用途与世界预告的外界窗口。
- 由主角作出的最终脱困选择及不可逆代价。
- 出口后能在短场景内接棒的外部人物或势力。

## 期待来源

- 空间到底遵守什么统一规则。
- 主角的旧技能还能识别哪些别人看不见的证据。
- 窗口中的声音、物件或痕迹来自谁。
- 金手指为何会困住使用者，它过去发生过什么。
- 最后一次试验会牺牲什么。
- 脱困暴露的坐标、身份或能力将引来谁。
- 空间规则进入外部世界后能解决何种更大问题。

## 断档风险

- 当前危机只为教学服务，读者不关心失败。
- 系统直接给答案，主角没有观察和选择。
- 试验无成本，规则因此没有可信重量。
- 穿越前技能只写在履历里，没有改变解题步骤。
- 封闭过久且没有外界窗口，类型世界迟迟不出现。
- 窗口数量过多，把小空间重新塞成设定百科。
- 出口由巧合或系统自动打开，主体性在高潮消失。
- 脱困后切到无关剧情，第一场积累全部失效。

## 训练任务

写一个 2500 字封闭空间开篇，只允许一名在场人物。设计三轮规则试验，其中一轮失败；让旧技能只贡献一个决定性观察，不能直接解题。放入两个外界窗口，并在结尾让主角以不可逆代价主动脱困，随后用 300 字证明该代价已经改变外部冲突。
