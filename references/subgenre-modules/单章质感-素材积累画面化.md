# 单章质感-素材积累画面化

依据案例：`batch-055` 案例 275。

## 读者承诺

这个模块解决新人单章像记叙文、单薄、缺画面感的问题。它承诺的不是靠堆细节变生动，而是通过作者视角积累设定、人物、冲突、动作和情绪模型，让一章从“百字概括”变成有过程、有画面、有拉扯的事件。

## 核心循环

```text
单章概括 -> 找出单薄点 -> 作者视角拆样本 -> 积累画面/设定/冲突素材 -> 填入事件过程 -> 单章质感提升
```

## 开篇必备

- 先写出本章 100 字概括，确认它到底在交代什么。
- 找出概括之外缺少的过程层：行动、观察、阻碍、反应、选择。
- 从同题材样本中拆画面素材、设定逻辑和矛盾触发。
- 细节必须服务事件，不做无功能铺陈。
- 每章至少有一个让读者能看见的具体变化。

## 期待来源

- 读者能感到主角在现场行动，而不是作者汇报结果。
- 设定通过问题和代价出现。
- 人物模型通过选择和反应显现。
- 矛盾触发让章节自然推进。
- 情绪拉扯来自事件过程，而不是情绪词。

## 断档风险

- 误以为“画面感”等于面面俱到细节描写。
- 只学三幕式，不补单章素材。
- 只读剧情，不拆作者如何展开过程。
- 脑中没有同题材设定和冲突模型。
- 扩写时堆废话，仍然没有行动和变化。

## 训练任务

拿一章自己的干瘪稿，先压缩成 100 字概括。再从样本书中各拆一个画面、一个设定逻辑、一个矛盾触发，把原章扩写到 1500 字，要求新增内容全部服务行动、阻碍、反应或情绪拉扯。
