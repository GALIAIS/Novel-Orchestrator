# 文道知识金手指-卷结构旁观者爽点

依据案例：`batch-175` 案例 873。

## 读者承诺

主角用文章、知识或文化资源解决大问题，金手指能力逐步揭露，并让旁观者、敌人和上层势力不断重新评价主角。

## 核心循环

```text
世界困境 -> 旁观者焦虑 -> 主角知识产出 -> 局面破开 -> 旁观者震惊 -> 金手指升级/新功能揭露
```

## 开篇必备

- 第一幕问题不能单调，要有阶段奖励。
- 金手指功能分批揭露，用到再说。
- 旁观者先有真实焦虑，再见证主角成果。
- 卷内设置相反情绪支线，避免一路平滑。

## 期待来源

读者期待主角拿出别人没有的文化、文章、知识或规则理解，把高层焦虑、敌人优势和世界困局打穿。爽点既来自破局，也来自别人逐步知道主角价值。

## 断档风险

- 只让读者知道主角厉害，书中人物没有反应。
- 金手指一次性解释完，后续缺少新鲜感。
- 卷纲情绪单调上升或下降。
- 一卷力量等级跨越过大，缺少缓冲。
- 人物过多，篇幅和笔力支撑不住。

## 训练任务

设计一卷知识型金手指剧情：列出 3 次功能揭露、2 个旁观者焦虑场景、1 条反向情绪支线和 1 个第一幕奖励在后文回收的节点。
