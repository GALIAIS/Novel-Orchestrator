# 配角模板-功能角色小主角化

依据案例：batch-271 案例 1354

## 读者承诺

用配角功能位增强主角表达、规则入口、侧面证明和剧情引线。好配角不是背景板，而是能让主角更好看、让事件自然发生的“小主角”。

## 核心循环

```text
主角需求 -> 配角功能位 -> 初遇事件 -> 信息差/反差 -> 关系升级 -> 配角引出新剧情
```

## 开篇必备

- 至少准备一个对话型配角，让信息不靠作者硬讲。
- 准备一个规则型或长辈型配角，承接世界规则和资源入口。
- 配角第一次出场要有事件功能，不只介绍身份。

## 期待来源

读者期待配角不断衬托主角：误判主角、信服主角、替主角背锅、把新麻烦带进来。配角越能制造反差和信息差，主角越容易被侧面证明。

## 断档风险

- 只借“小胖子、师妹、师伯”外壳，不借功能。
- 配角只聊天，不推动事件。
- 迪化角色没有前后反差，爽点不成立。
- 配角过多但职责重叠。

## 训练任务

为主角列四个配角功能位：对话、规则、迪化、引线。每个只写初遇事件和后续推动剧情的方式。
