# 一级结构-起承转合单元

依据案例：`batch-003` 案例 015。

## 读者承诺

给新人一个单元故事兜底框架：每个小剧情都要有进入、执行、升级、解决和后续反应，不能只堆设定或对话。

## 核心循环

```text
起：入局 -> 承：执行/铺垫 -> 转：升级/反转 -> 合：解决/收获/反应 -> 新钩子
```

## 开篇必备

- 主角为什么入局。
- 本单元要解决的问题。
- 参与人物的宽泛功能。
- 起承阶段的小爽点和铺垫。
- 转合阶段的大爽点和收获。

## 期待来源

- 读者想看主角如何解决当前问题。
- 读者想看起承阶段铺垫如何在转合阶段回收。
- 读者想看解决后各方反应。
- 读者想看收获如何进入下一单元。

## 断档风险

- 起承阶段爽点和转合抢戏。
- 合只有解决，没有收获和反应。
- 人物功能不清，剧情推进全靠作者硬推。
- 每个单元爽点同质化，没有细节差异。

## 训练任务

先写“合”的三类反应，再倒推“起”需要出现的人物、误判、目标和信息差。用这个方法写一个 3000 字单元。
