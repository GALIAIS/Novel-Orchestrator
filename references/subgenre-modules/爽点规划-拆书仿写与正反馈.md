# 爽点规划-拆书仿写与正反馈

依据案例：batch-362 案例 1806

## 读者承诺

爽点要从开头结构阶段规划，而不是写到后面硬塞。用拆书仿写学会“不可能选择、阻碍、行动、正反馈”的完整链条。

## 核心循环

```text
不可能选择 -> 阻碍 -> 主角行动 -> 正向反馈 -> 变强/得利
```

## 开篇必备

- 明确读者想代入的选择。
- 选择有现实不可得性。
- 阻碍具体。
- 反馈及时。

## 期待来源

- 主角做出现实中不敢做的事。
- 做完得到好处。
- 读者看见主角变牛。
- 爽点围绕事件推进。

## 断档风险

- 第五章才硬凑爽点。
- 先写剧情后找爽点。
- 只写赢了，不写正反馈。
- 不拆成熟案例就硬编。

## 训练任务

拆 3 个爽点，记录不可能选择、阻碍、行动、正反馈，再仿写一个 500 字桥段。
