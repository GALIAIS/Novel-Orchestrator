# 智能推选题-扫榜对标

依据案例：`batch-004` 案例 017。

## 读者承诺

用自身积累和平台智能推共同选题，找到真实存在的同类读者池，而不是只追榜首热词。

## 核心循环

```text
自身积累 -> 标签识别 -> 榜单样本 -> 智能推样本 -> 同水平拆书 -> 开篇承诺验证
```

## 开篇必备

- 题材必须是作者有阅读积累的方向。
- 书名、简介、正文前三章要让平台和读者识别标签。
- 至少有榜单样本和智能推样本两组对标。

## 期待来源

- 读者一眼知道这本书属于什么爽点池。
- 平台能把书推给正确读者。
- 作者能从同类样本获得可写长的素材。

## 断档风险

- 只看榜首，不看同类长尾样本。
- 有市场没积累，写不长。
- 有积累但标签不清，平台识别不到。
- 硬学高阶神书，忽略同水平解法。

## 训练任务

为一个题材找 3 本榜单样本和 3 本智能推样本，拆书名、简介、第一章承诺、前三章留存点。
