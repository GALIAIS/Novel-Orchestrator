# 连环小剧情-试探增量与暗线汇流

依据案例：`batch-398` 案例 1987

## 读者承诺

让主角面对暂时无解的强敌仍持续前进：每次试探虽未获胜，却分别带回信息、工具、位置或关系，并由跨事件变量把小剧情汇入共同暗线。

## 核心循环

```text
局部异常 -> 带假设试探 -> 小输换取增量 -> 共享变量复现 -> 新事件修正旧解释 -> 多次增量组合破局
```

## 开篇必备

- 每个小剧情有独立表面目标。
- 主角试探前提出可验证假设。
- 失败后至少一种状态向前推进。
- 三个事件共享可追踪变量。
- 后一事件改变前一事件的解释。
- 最终破局真实使用早期成果。

## 期待来源

读者期待三个异常究竟如何相连，也能看见能力不足的主角不断排除错误、积累条件，逐渐接近可执行答案。

## 断档风险

- 每次只冲上去失败。
- 共同黑幕只在结尾宣布。
- 小剧情没有局部结果。
- 失败既无代价也无新信息。
- 后续事件只是重复换皮。
- 最终靠突然升级清空早期积累。

## 训练任务

写三个“问题—试探—反馈—更新”小事件，分别获得信息、工具和关系；最终章只能组合这三项破局，不允许临时增加能力。
