# 起点新书期-推荐节点排期

依据案例：batch-290 案例 1448

## 读者承诺

起点新书期要把发书日期、推荐节点、日更字数、上架字数和入 V 爆更排成可执行计划。

## 核心循环

```text
发书 -> 推荐链路 -> 稳定日更 -> 三十万字左右上架 -> 入V爆更
```

## 开篇必备

- 发书日期。
- 日更最低字数。
- 预计推荐节点。
- 累计字数目标。
- 上架日期。
- 爆更存稿。
- 数据记录。

## 期待来源

作者用稳定更新承接推荐曝光，在上架节点用爆更放大收益。

## 断档风险

- 不排期，推荐期间断更或少更。
- 没存稿，上架不能爆。
- 把评论经验当官方规则。
- 只盯平台动作，忽略作品质量。

## 训练任务

从发书日开始列 8 周排期，每周写推荐节点、日更字数、累计字数和存稿目标。
