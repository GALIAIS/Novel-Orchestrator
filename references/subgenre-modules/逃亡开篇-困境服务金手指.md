# 逃亡开篇-困境服务金手指

依据案例：`batch-075` 案例 375。

## 读者承诺

这个模块用于灭国、追杀、逃亡类开篇。它承诺让危机直接压到主角身上，并让金手指尽早解决具体困境，而不是先用大段背景旁白消耗耐心。

## 核心循环

```text
主角当前麻烦 -> 第一矛盾 -> 逃亡压力 -> 金手指被需要 -> 能力解决问题 -> 新目标
```

## 开篇必备

- 前 500 字出现主角当前麻烦。
- 背景只保留理解冲突所需部分。
- 第一矛盾和主角身份或选择相关。
- 金手指不要拖到读者离开后才出现。
- 困境必须为金手指服务。

## 期待来源

- 主角被追杀但不是纯被动。
- 身份秘密带来追读。
- 金手指解决眼前问题，也暗示长期路线。
- 逃亡中不断获得选择和代价。
- 背景后置为阶段性真相。

## 断档风险

- 叛军攻城很热闹但和主角无关。
- 第一章大半都是可压缩背景。
- 逃亡只有危险，没有目标。
- 金手指太晚、太大或太小。
- 文字舒服但承诺迟到。

## 训练任务

把一个逃亡开篇改成三段：第一段主角麻烦，第二段身份导致的第一矛盾，第三段金手指解决具体困境。删掉所有不影响这三段理解的背景。
