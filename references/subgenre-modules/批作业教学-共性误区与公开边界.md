# 批作业教学-共性误区与公开边界

依据案例：`batch-248` 案例 1239。

## 读者承诺

批作业要提炼共性误区，比公开逐篇点名更稳，也更利于社群学习。

## 核心循环

```text
收集作业问题 -> 提炼共性误区 -> 匿名化示例 -> 制作教程 -> 作业水平提升
```

## 开篇必备

- 判断是否适合公开。
- 避免点名冒犯。
- 抽出共性问题。
- 给改法模板。

## 期待来源

新人能通过正反例看清问题，但不会把教学变成围观处刑。

## 断档风险

- 细节判断太主观。
- 公开批改冒犯作者。
- 只指出毒点不给改法。
- 没有抽象成通用方法。

## 训练任务

从 5 份作业里提炼 3 个共性问题，每个写错误表现和改法模板，不点名。
