# 同人开篇-主角先出场

依据案例：batch-302 案例 1506

## 读者承诺

同人也要先让读者跟随主角经历事件，原作热度和背景气氛不能替代主角入场。

## 核心循环

```text
原作热度 -> 主角入场 -> 当前事件/冲突 -> 原作信息随事件释放 -> 章末新钩子
```

## 开篇必备

- 主角尽早出场。
- 第一章有主角行动。
- 原作背景只保留当前必要信息。
- 创新角度通过事件兑现。

## 期待来源

- 原作熟悉感带来点击。
- 新主角/新角度带来追读理由。
- 主角行动让读者进入故事。

## 断档风险

- 开篇长时间没有主角。
- 写成实体感故事，网文承诺不足。
- 背景气氛堆太多，看不懂。

## 训练任务

把一个同人背景开篇改成主角先出场：第一段主角行动，第一章完成一次冲突。
