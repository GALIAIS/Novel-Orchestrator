# 系统任务-正反馈主动性

依据案例：`batch-134` 案例 668。

## 读者承诺

系统任务服务主角目标和主线冲突，提供奖励、提示和正反馈。主角不是任务傀儡，而是在权衡风险后主动选择。

## 核心循环

```text
主角目标 -> 主线冲突 -> 系统发布匹配任务 -> 主角权衡选择 -> 完成后正反馈
```

## 开篇必备

- 不急着解释系统来源。
- 主角先有自己的目标。
- 任务顺着剧情需要出现。
- 奖励补强主角行动。
- 主角有拒绝或选择空间。

## 期待来源

- 系统带来机会和奖励。
- 主角利用系统装逼破局。
- 任务推动已有冲突升级。
- 主角掌控感强。

## 断档风险

- 系统强推剧情。
- 主角无脑接任务。
- 花太多篇幅解释来源。
- 系统喧宾夺主，主角像工具人。

## 训练任务

写一个系统任务设计：先写主角目标、反派冲突、风险，再写任务和奖励。要求即使没有系统，主角也有理由行动。
