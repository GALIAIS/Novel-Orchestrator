# 公开处刑-要素表与多角度挑刺

依据案例：batch-305 案例 1522

## 读者承诺

新人提交作品求诊断时，用规则、置顶和要素表降低沟通成本，再通过多人视角筛查毒点，而不是只等某个老师给一句总评。

## 核心循环

```text
求看文 -> 读置顶规则 -> 填要素表 -> 公开处刑 -> 多角度挑刺 -> 汇总可修问题
```

## 开篇必备

- 提交前写明题材、平台、核心卖点。
- 写明前三章目标和主角当前目标。
- 标出作者自己最怀疑的问题。
- 接受不同评论者从开头、剧情构建、写作技巧等维度分开诊断。

## 期待来源

期待来自“我能获得可执行的问题清单”。要素表让反馈者知道该看什么，多人挑刺让作者看到自己盲区。

## 断档风险

- 只说“求大佬看看”，不给上下文。
- 不看规则，导致反馈者反复追问基础信息。
- 把单人意见当最终裁决。
- 混淆开头问题、剧情构建问题和文字技巧问题。

## 训练任务

把自己的作品整理成一页公开处刑提交单：作品名、题材平台、卖点、前三章目标、作者困惑、希望检查的三个维度。
