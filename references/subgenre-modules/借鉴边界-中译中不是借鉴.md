# 借鉴边界-中译中不是借鉴

依据案例：`batch-323` 案例 1611

## 读者承诺

借鉴热门作品只能学习功能，不能复用读者一眼能识别的核心梗、开场节奏和标志桥段；把原情节换说法，不会自动变成原创。

## 核心循环

```text
想学热门开篇 -> 识别不可碰的核心梗 -> 抽出类型功能 -> 重建自己的代价和冲突 -> 用新场景交付同类爽点
```

## 开篇必备

- 先列出读者会一眼认出的来源元素。
- 删除标志桥段、事件顺序和高度相似的关系结构。
- 保留的只能是类型功能，如循环、误会、打脸、危机。
- 给主角设计独立的损失、目标和代价。
- 如果写循环，第一章就交代不破局的坏结果。

## 期待来源

读者期待看到作者自己的冲突，而不是进入鉴别抄袭或洗稿的状态。

## 断档风险

- 认为只有几百字相似就没问题。
- 只改设定，不改核心梗和节奏。
- 用自己的话复述别人桥段。
- 循环设定没有不可逆坏结果。
- 读者先认出来源，后续新内容也失去信任。

## 训练任务

找一个想学的热门开篇，写出“不可复用清单”和“可学习功能清单”，再用完全不同人物、场景、代价重写 500 字。
