# 多线大纲-可视化支线管理

依据案例：`batch-074` 案例 367。

## 读者承诺

这个模块用于主线和多条支线并行的长篇规划。它承诺把线性文档外化成颜色、节点、连接线和备注，让作者在写作时一眼看见因果、情绪和支线交叉。

## 核心循环

```text
剧情线分色 -> 节点上墙 -> 连接因果 -> 标注情绪 -> 写作中移动/删改 -> 回看整体节奏
```

## 开篇必备

- 主线和每条支线有不同颜色或区域。
- 每个剧情节点只写一个核心变化。
- 节点之间标清前置和后续。
- 上行、下行、平缓要直接标注。
- 设定、伏笔、删改理由放在节点旁边。

## 期待来源

- 支线交叉让读者感觉世界在并行运转。
- 情绪曲线能避免连续平铺。
- 可视化节点能提前发现某条线太久没回收。
- 备注能保留伏笔用途，减少写到后面忘记。
- 动态改动让大纲服务正文，而不是束缚正文。

## 断档风险

- 只换工具，不拆清主线和支线职责。
- 节点太大，一个节点塞进多章内容。
- 连接线只表示顺序，不表示因果。
- 支线没有服务主线或人设。
- 大纲改动没有留下理由，后续复盘困难。

## 训练任务

把一个二十章故事拆成主线、感情线、成长线三种颜色。每五章检查一次：哪条线推进、哪条线停滞、哪条线和主线发生交叉。
