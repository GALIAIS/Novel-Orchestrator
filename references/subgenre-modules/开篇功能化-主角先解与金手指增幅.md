# 开篇功能化-主角先解与金手指增幅

依据案例：`batch-415` 案例 2072

## 读者承诺

开篇不会机械执行“几百字必须发生什么”，而会完成目标可见、危机成立、人物能动、能力承诺和章尾债务五项功能。主角先用自身观察、技能或关系取得局部进展，更高层阻碍从同一因果中暴露旧方法边界；已有信号的金手指再放大主角选择并带来代价，使外挂成为人物资产而非代打程序。

## 核心循环

```text
把固定公式还原为功能问句 -> 为每个功能生成三种实现 -> 选择最适合人物和题材的一版 -> 主角无外挂先取得局部进展 -> 同因果的高层阻碍暴露旧方法边界 -> 金手指响应前置信号并提供增幅 -> 主角选择使用方式并承担代价 -> 结算改变目标、资源、关系或身份 -> 新债务接入下一阶段
```

三解训练证明理论已被理解，主角先解保护主体性，同因果升级避免随机加敌，代价与状态变化则把开篇接入长线。

## 开篇必备

- 原公式背后的目标、阻碍、能动性、能力和债务功能。
- 每个功能至少三种不同场景实现。
- 主角在无外挂状态下的不可替代观察或行动。
- 局部进展及其对后续条件的真实改变。
- 与前序人物、组织或规则相连的升级阻碍。
- 金手指出现前的最小异常信号。
- 主角对能力用法的选择与即时代价。
- 目标、资源、关系或身份中的不可逆结算。
- 一条由本场结果自然生成的下一阶段债务。

## 期待来源

- 主角没有外挂时能做到什么。
- 局部近胜为何仍不足以解决高层问题。
- 金手指会放大哪项已有特质。
- 主角会选择哪种用法并牺牲什么。
- 第一场留下的资源或关系将怎样复用。
- 新债务会把能力带入何种更大冲突。

## 断档风险

- 把固定字数当成功能本身。
- 所谓三种实现只是同一模板换专名。
- 金手指替主角观察、决策和执行全部步骤。
- 局部进展在反转后被完全清零。
- 高层阻碍为章尾效果随机出现。
- 金手指毫无信号地自动救命。
- 能力没有代价、选择和后续债务。
- 反转很多，却没有状态发生不可逆变化。

## 训练任务

选一条常见开篇公式，改写成五个功能问句，并各写三种实现。完成一个 2500 字开篇：主角先靠自身能力改变局势，升级阻碍来自同一因果，金手指只在有信号后由主角选择使用；结尾列出四项状态变化和一条下一阶段债务。
