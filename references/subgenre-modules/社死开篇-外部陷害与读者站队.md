# 社死开篇-外部陷害与读者站队

依据案例：`batch-221` 案例 1104。

## 读者承诺

社会性死亡可以做开篇危机，但危机归因要让读者站主角。

## 核心循环

```text
已有成就 -> 外部嫉妒 -> 曝光绯闻 -> 社死危机 -> 主角反击
```

## 开篇必备

- 危机原因不让主角显蠢。
- 外部压力来源明确。
- 主角已有成就可招嫉妒。
- 社死后有反击和收益。

## 期待来源

读者看到主角被外部陷害或曝光，会期待澄清、反杀和名声逆转。

## 断档风险

- 主角主动犯蠢导致社死。
- 读者把主角判成过错方。
- 模仿样本只学表面事件。
- 社死危机没有后续收益。

## 训练任务

写两版社死开篇，对比主角主动犯错和外部陷害的读者站队，再把前者改成后者。
