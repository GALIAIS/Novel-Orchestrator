# 起承转合-意外变化结构

依据案例：batch-308 案例 1537

## 读者承诺

网文的“转”要靠意外带来变化。变化可以改目标、改障碍、改悬念，让读者进入下一轮追读。

## 核心循环

```text
起：建立问题 -> 承：努力/对抗/延宕 -> 转：意外改变问题 -> 合：解决或失败并生成新问题
```

## 开篇必备

- 起点必须有目标、障碍或悬念。
- 承接阶段要让读者看到推进。
- 转折必须改变问题性质。
- 合要给阶段结果，不拖成无意义延宕。

## 期待来源

期待来自问题变化。读者以为要解决 A，意外让 A 变成 B，旧期待没断，新期待又被点燃。

## 断档风险

- 转折只是插事，没有改变目标或障碍。
- 承接阶段没有推进，只是在原地解释。
- 悬念不变形，读者疲劳。
- 合不给结果，只有继续拖。

## 训练任务

把同一段剧情写成目标版、障碍版、悬念版起承转合，每版都标出“转后变化”。
