# 开头疲劳-激情恢复与重开

依据案例：`batch-059` 案例 292。

## 读者承诺

这个模块处理开头反复修改后失去激情的问题。它承诺的是用休息、输入、复盘、细纲硬写或低成本重开，把情绪低谷转成可执行决策。

## 核心循环

```text
开头试写 -> 多轮反馈修改 -> 激情下降 -> 休息输入 -> 复盘错漏和兴奋点 -> 继续硬写或重开
```

## 开篇必备

- 记录已写字数和修改次数。
- 判断修改后是否偏离自己想写的核心。
- 先休息一两天，不硬逼激情。
- 复盘错漏和仍想写的东西。
- 字数少时允许重开，别被沉没成本绑住。

## 期待来源

- 作者保留学习收益。
- 疲劳不被误判成能力崩溃。
- 细纲在低激情期承担推进功能。
- 重开能制造新激情。
- 旧稿错漏成为下一稿的防错清单。

## 断档风险

- 强迫自己立刻有激情。
- 反复修改到作品不像自己的。
- 一万字也舍不得放弃，越拖越痛苦。
- 重开但不总结错漏。
- 完全靠激情，不建立细纲和习惯。

## 训练任务

对当前开头写一张二选一表：继续写的理由 3 条，重开的理由 3 条。若继续写，按细纲写 3000 字；若重开，先写旧稿错漏清单。
