# 驱动方式-新人剧情驱动优先

依据案例：`batch-235` 案例 1173。

## 读者承诺

人物驱动可用但难控；新人先用剧情驱动托住结构，再逐步提升人物自主性。

## 核心循环

```text
先定事件 -> 安排人物功能 -> 人物选择推动事件 -> 检查因果 -> 避免主线失控
```

## 开篇必备

- 主线事件清楚。
- 人物为剧情承担功能。
- 动机服务事件推进。
- 不让人物自由行动冲散主线。

## 期待来源

读者能稳定看到事件推进、目标变化和因果闭环，同时人物不至于完全扁平。

## 断档风险

- 新人直接写人物驱动，因果失控。
- 人物选择制造剧情 bug。
- 误以为剧情驱动就不能有人物。
- 主线被角色闲聊带散。

## 训练任务

把人物驱动想法改成剧情驱动：先列 5 个必须事件，再给每个事件安排人物动机和选择。
