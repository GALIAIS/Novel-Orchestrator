# 剧情粗模-计划意外问答与口述转写

依据案例：`batch-412` 案例 2058

## 读者承诺

用可判定主线句约束方向，以人物计划、可信意外、有代价选择和状态结算生成单元；口述只负责快速捕捉，转写后压成场景卡并完成因果与连续性审计。

## 核心循环

```text
主线句 -> 单元进入状态 -> 人物计划与关键假设 -> 既有因素制造意外 -> 备选中的代价选择 -> 信息关系资源结算 -> 下一债务 -> 口述压卡与连续性审计
```

## 开篇必备

- 能判断成败的长线目标。
- 每单元明确进入和退出状态。
- 计划依赖的关键假设。
- 来自既有规则、对手或误判的意外。
- 不限于打脸得宝的多样结算。
- 转写后的主体、时间、因果和专名校对。

## 期待来源

人物计划先形成清楚预期，已铺垫因素让假设失效；选择暴露优先级，结算保证状态真变，本轮收获与代价自然成为下一计划的新条件。

## 断档风险

- 意外只是随机打断计划。
- 每轮固定反派丢脸、主角得宝。
- 提示问题过多成为新拖延。
- 口述稿直接当作细纲终稿。
- 转写重复、跳时间和代词不清未校对。
- 上一轮得失没有进入下一轮。

## 训练任务

五分钟口述一个计划、意外、选择、结算单元，再压成 300 字场景卡；意外必须来自已有条件，结算禁止打脸或得宝，下一单元强制使用上一轮的一项收获和一项代价。
