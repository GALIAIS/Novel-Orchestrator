# 公开处刑-章纲降维评审

依据案例：`batch-134` 案例 669。

## 读者承诺

把正文压缩成 5-10 章章纲，让评审者快速判断剧情方向、毒点和症结。先解决故事是否能跑，再讨论文风细节。

## 核心循环

```text
正文成本高 -> 章纲压缩 -> 快速判断剧情方向 -> 定位毒点 -> 再回正文查信息密度
```

## 开篇必备

- 每章核心事件。
- 人物变化。
- 冲突推进。
- 预期读者感受。
- 铺垫伏笔和下一章钩子。

## 期待来源

- 评审成本降低。
- 更多人愿意参与。
- 讨论集中到剧情构建。
- 作者填写时先自查一遍。

## 断档风险

- 直接丢长正文，没人愿意看。
- 章纲只写事件，不写作用。
- 过早陷入文风争执。
- 忽略正文里的信息跳级问题。

## 训练任务

把前 5-10 章正文压缩成章纲，每章写核心事件、人物变化、冲突推进、预期感受、伏笔和可能毒点。
