# 题材用户匹配-固定套路微创新

依据案例：`batch-045` 案例 224。

## 读者承诺

不同题材服务不同用户群。先识别固定套路和读者预期，再在兼容位置做微创新，才更容易被书名、简介和正文承接。

## 核心循环

```text
题材分类 -> 用户需求 -> 固定套路 -> 书名简介识别 -> 微创新
```

## 开篇必备

- 确定题材大类和特定子类。
- 明确目标用户。
- 书名简介与正文一致。
- 微创新不破坏固定预期。

## 期待来源

- 读者一眼知道是不是想看的书。
- 固定套路降低理解门槛。
- 金手指和题材适配。
- 微创新提供新鲜感。

## 断档风险

- 作者兴趣和用户需求错位。
- 过度创新导致题材识别失败。
- 高门槛题材低实力硬写。
- 书名简介吸来的人和正文不一致。

## 训练任务

选一个题材，写出用户群、固定套路、金手指适配、书名关键词和一个安全微创新。
