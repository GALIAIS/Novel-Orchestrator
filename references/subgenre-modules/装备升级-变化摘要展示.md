# 装备升级-变化摘要展示

依据案例：batch-284 案例 1418

## 读者承诺

升级装备展示重点写新增变化和当前用途，完整属性只在关键节点出现，避免反复复读旧设定。

## 核心循环

```text
装备升级 -> 摘要新变化 -> 当前事件试用 -> 关键节点亮全貌 -> 新期待
```

## 开篇必备

- 装备核心功能。
- 本次升级新增效果。
- 主角一句话判断。
- 实战测试场景。
- 完整属性展示时机。

## 期待来源

读者期待装备升级改变战斗、探索或选择，而不是重复阅读旧面板。

## 断档风险

- 每次升级都完整复读，显得水。
- 只说升级，不写用途。
- 完全不提醒旧功能，读者断片。
- 面板展示和事件无关。

## 训练任务

设计三次装备升级：前两次只写新增变化，第三次在高潮前完整展示，并让完整属性成为破局依据。
