# 关系毒点-开局女友与承诺落空

依据案例：batch-368 案例 1837

## 读者承诺

开局自带女友会天然建立强关系承诺；后期分手或死亡若无极大意义，容易让读者觉得被晃点。

## 核心循环

```text
女友标签 -> 关系承诺 -> 读者期待 -> 掉线/死亡 -> 毒点反噬
```

## 开篇必备

- 明确她是否真女主。
- 明确关系线功能。
- 明确死亡或分手是否必要。
- 可替代工具人先替代。

## 期待来源

- 稳定陪伴。
- 情感归属。
- 后续重要影响。
- 关系承诺兑现。

## 断档风险

- 用女友当工具人。
- 十章后随意掉线。
- 死亡没有极大意义。
- 作者以为情感描写少就无风险。

## 训练任务

把“开局女友后期死亡”改成工具人版、开局已死版、重大意义版，比较承诺风险。
