# 新人开篇-一根直线做减法

依据案例：`batch-330` 案例 1647

## 读者承诺

新人开篇先简单、直接、易懂，用一根直线讲清楚主线/金手指相关事件。

## 核心循环

```text
核心事件 -> 主角进入 -> 金手指/危机显现 -> 读者理解期待 -> 复杂内容后置
```

## 开篇必备

- 只有一个核心事件。
- 信息都指向主线或金手指。
- 人物少，支线少，设定少。
- 前三章形成完整小事件。
- 用删减优先级筛内容。

## 期待来源

读者快速看懂主角遇到什么、要做什么、凭什么值得继续看。

## 断档风险

- 一开头塞多条线。
- 伏笔和说明书压过期待。
- 配角、场景、概念过载。
- 世界观宏大但眼前事件不清。
- 新人没有能力托住复杂结构。

## 训练任务

列出前三章所有人物、概念、场景、支线，只保留服务主线/金手指的一根直线。
