# 编剧课工具-场景序列悬念

依据案例：`batch-076` 案例 379。

## 读者承诺

这个模块用于把编剧课笔记转成网文诊断工具。它承诺每个场景都要传信息、推剧情、揭变化，并放进序列节奏和悬念反转里检查。

## 核心循环

```text
开场兴趣 -> 激励事件 -> 场景推进 -> 转折/再转 -> 悬念失控 -> 高潮解决
```

## 开篇必备

- 主角价值与态度明确。
- 开场有强刺激或明确期待。
- 场景动作有回响。
- 每场至少有对抗或张力。
- 序列位置清楚。

## 期待来源

- 场景不断揭示新信息。
- 转折改变重要因素。
- 悬念制造观众或角色失控。
- 反转来自误导和隐藏关键信息。
- 高潮解决核心问题。

## 断档风险

- 场景只传信息，不推进剧情。
- 动作没有后果。
- 序列缺中点转折。
- 悬念只靠隐瞒，不制造压迫感。
- 高潮没有兑现前面伏笔。

## 训练任务

拿一个三千字开篇，逐场标注“信息、推进、变化、序列位置、悬念方式”。凡是五项里少两项以上的场景，删掉或合并。
