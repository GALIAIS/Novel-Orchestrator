# 系统终局-保留悬念与本源合理化

依据案例：batch-387 案例 1935

## 读者承诺

系统不必在结尾强行消失。只有前文建立来源承诺时才需结算；揭示本源则要与世界规则、主角天赋和控制权成长统一。

## 核心循环

```text
确定系统身份 -> 建立或不建立来源谜团 -> 同源伏笔 -> 控制权成长 -> 保留/留白/揭示
```

## 开篇必备

- 系统属于哪类叙事存在。
- 是否承诺解释来源。
- 与世界力量体系的同源点。
- 主角当前控制边界。

## 期待来源

- 外挂逐渐显示更深规则。
- 主角从使用者变成掌控者。
- 早期异常在终局回收。
- 主线完成后仍可保留适度余韵。

## 断档风险

- 终章临时宣布系统来源。
- 前文没问来源却强行复杂化。
- 留悬念影响主线闭环。
- 本源解释与世界规则冲突。

## 训练任务

为同一系统故事写保留、留白、揭示三版结尾纲，逐项补足伏笔和控制权变化并检查主线闭环。
