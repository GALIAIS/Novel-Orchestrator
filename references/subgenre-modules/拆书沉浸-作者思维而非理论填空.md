# 拆书沉浸-作者思维而非理论填空

依据案例：`batch-223` 案例 1115。

## 读者承诺

拆书的目标是进入成熟作者的写作思维，理解节奏、取舍和读者期待，而不是把章节硬塞进理论名词。

## 核心循环

```text
阅读章节 -> 记录读者期待 -> 观察作者推进 -> 比较自己写法 -> 提炼差距
```

## 开篇必备

- 理论只做辅助标尺。
- 每章记录真实功能。
- 关注节奏、信息投放、期待变化。
- 用长文本连续拆解验证方法是否有效。

## 期待来源

作者获得的是可迁移的写作判断：成熟作者为什么在这里推进、停顿、转向、压信息或放爽点。

## 断档风险

- 为套英雄之旅而拆书。
- 只记录术语，不记录章节功能。
- 不比较自己和样本作者的差距。
- 拆得很细，却没有形成写作动作。

## 训练任务

选一段 2-3 万字网文，不套英雄之旅术语，只记录每章读者期待、作者推进手段、节奏变化、信息投放，并写 300 字差距总结。
