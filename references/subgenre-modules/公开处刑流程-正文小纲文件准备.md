# 公开处刑流程-正文小纲文件准备

依据案例：`batch-212` 案例 1059。

## 读者承诺

公开处刑前要把正文、小纲、简介和表格放到上传入口能看见的位置。

## 核心循环

```text
材料准备 -> 本地可见目录 -> 统一命名 -> 上传测试 -> 提交反馈
```

## 开篇必备

- 正文和小纲分开命名。
- 文件保存到手机/电脑可见目录。
- 从上传入口测试能否选中。
- 保留备份版本。

## 期待来源

作者把反馈机会用于作品问题，而不是浪费在找不到文件上。

## 断档风险

- 文件只在 WPS 或 App 内部目录。
- 手机上传入口看不到。
- 文件名混乱。
- 没有版本号和备份。

## 训练任务

做公开处刑提交清单：正文、小纲、简介、问题说明、版本号、保存目录、备份目录、上传测试。
