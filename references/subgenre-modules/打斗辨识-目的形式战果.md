# 打斗辨识-目的形式战果

依据案例：`batch-082` 案例 407。

## 读者承诺

这个模块用于判断一场打斗是否值得写细。它承诺让打斗至少提供目的、角色痕迹、形式变化和战后收益，而不是只告诉读者谁赢谁输。

## 核心循环

```text
打斗必要性 -> 角色打法 -> 形式变化 -> 战果差异 -> 收获结算
```

## 开篇必备

- 开打前先说明为什么非打不可。
- 主角打法要符合性格、武器或传承。
- 不能连续使用同一种对波。
- 战斗结果要改变局势。
- 战后必须有收获、代价或新问题。

## 期待来源

- 读者知道这场不能跳。
- 主角用自己的方式打。
- 对手不是换皮木桩。
- 战斗形式有变化。
- 结果带来超额收益。

## 断档风险

- 换掉人名和招式仍然成立。
- 打斗只剩谁打谁、谁赢谁输。
- 能量波换皮重复。
- 过程写太长，收益太少。
- 无法写好就不肯略写。

## 训练任务

删除一场打斗里所有人名和招式名，检查还能否看出主角是谁。看不出就重写打法；只剩胜负就压缩过程，补打斗原因和战后收获。
