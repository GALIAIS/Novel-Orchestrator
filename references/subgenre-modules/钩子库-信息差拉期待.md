# 钩子库-信息差拉期待

依据案例：`batch-003` 案例 011。

## 读者承诺

当章节缺少追读感时，用可翻检的钩子库补足期待，而不是只说“这里要铺垫”。核心是把信息差、反差、危机、悬念落到具体章纲动作。

## 核心循环

```text
描述价值 -> 留出未知 -> 制造误判/反差 -> 延迟兑现 -> 兑现前补新钩子
```

## 开篇必备

- 至少一个读者知道但角色不知道的信息。
- 至少一个角色误判或低估。
- 至少一个费解行为、危险信号或未解释目标。

## 期待来源

- 读者想看真相揭开。
- 读者想看误判者反应。
- 读者想看主角费解行为背后的理由。
- 读者想看危机如何被解决。

## 断档风险

- 只铺信息，没有制造情绪缺口。
- 钩子太多但不兑现。
- 兑现后没有新钩子接上。
- 描述不足，读者不知道该期待什么。

## 训练任务

给一章正文补 3 个钩子：一个信息差、一个反差、一个费解行为。每个钩子必须写清楚兑现章节。
