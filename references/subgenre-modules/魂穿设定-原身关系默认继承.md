# 魂穿设定-原身关系默认继承

依据案例：`batch-330` 案例 1648

## 读者承诺

魂穿继承身体和记忆后，读者默认主角也要处理原身社会关系；拒绝身份必须有明确剧情回报。

## 核心循环

```text
魂穿融合记忆 -> 社会关系继承 -> 主角处理关系 -> 利用身份改命 -> 新身份形成爽点
```

## 开篇必备

- 说明是否融合记忆。
- 原身关系压力清楚。
- 主角接受或拒绝身份的理由清楚。
- 拒绝身份要绑定金手指或主线。
- 频道读者的毒点差异要提前判断。

## 期待来源

读者期待主角利用原身身份和现代/异界信息差改命，而不是无意义地否认关系。

## 断档风险

- 原身有社会关系，主角却硬说无关。
- 拒绝身份不产生剧情价值。
- 男频魂穿标配被写成拧巴伦理。
- 女频读者产生夺舍、替身、恋爱对象错位感。
- 花活没有爽点回报。

## 训练任务

列出原身所有关键关系，逐项判断主角拒绝身份会不会制造毒点，以及能否换来更大的剧情收益。
