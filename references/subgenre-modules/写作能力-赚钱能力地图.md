# 写作能力-赚钱能力地图

依据案例：`batch-012` 案例 060。

## 读者承诺

从写作者而不是普通读者的角度评估能力短板，把教程、案例和训练任务重新组织成能提升成绩的能力地图。

## 核心循环

```text
项目现状 -> 能力分组 -> 定位短板 -> 绑定案例/教程 -> 单项训练 -> 写作验证
```

## 开篇必备

- 不用泛泛评分作品好坏，先判断自己要补哪项能力。
- 爽点、期待、节奏应联动评估。
- 题材背景、设定框架、长期承诺可作为大设定能力组。
- 一次只选最影响成绩的短板。

## 期待来源

- 新人知道自己下一步该练什么。
- 已有笔记能被重新归档成能力组。
- 拆书不再散，能对应具体写作短板。
- 训练结果能回到作品数据或章节质量上验证。

## 断档风险

- 把能力地图当排名表，只制造焦虑。
- 同时补八项能力，最后没有一项落到写作。
- 只收藏教程，不绑定案例和训练任务。
- 用读者喜好代替写作者能力诊断。

## 训练任务

给当前项目做一次能力诊断，只选一个主短板，绑定 3 篇案例或教程，设计一周训练和一个可验证指标。
