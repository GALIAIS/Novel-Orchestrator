# 商业动画首集-设定事件化

依据案例：`batch-083` 案例 412。

## 读者承诺

这个模块用于学习商业作品第一集/第一章如何快速立角色和世界观。它承诺把设定挂到事件、道具、测试、事故和角色互动上，而不是纯讲设定。

## 核心循环

```text
角色先出场 -> 一分钟事件 -> 设定事件化 -> 记忆点强化 -> 高潮回收
```

## 开篇必备

- 开头先给强角色或强处境。
- 第一事件尽快出现。
- 设定必须通过场景功能展示。
- 主要角色用动作登场。
- 前置信息在高潮回收。

## 期待来源

- 观众先记住角色。
- 世界观通过异常事件显形。
- 每个角色都有目的。
- 日常小道具制造亲近感。
- 高潮同时兑现能力和人设。

## 断档风险

- 纯旁白讲设定。
- 事件出现太晚。
- 角色登场没有动作功能。
- 记忆点只靠介绍。
- 前置设定不回收。

## 训练任务

把第一章设定全部改成五类承载物：事故、道具、测试、追逐、对话。不能归类的设定先删。
