# 书名承诺-第一章要素配比

依据案例：`batch-069` 案例 344。

## 读者承诺

这个模块用于书名、简介和第一章一致性检查。它承诺的是让书名筛选目标读者，第一章用背景、金手指、危机、赌注和目标兑现书名期待。

## 核心循环

```text
书名给承诺 -> 读者形成期待 -> 第一章铺背景和金手指 -> 危机堵路 -> 目标接上
```

## 开篇必备

- 书名能看出题材和卖点。
- 第一章出现时间地点主角。
- 主角行为先动起来。
- 金手指篇幅适中。
- 危机、赌注、目标和路径齐全。

## 期待来源

- 书名让读者知道自己点的是什么菜。
- 金手指带来直接爽点想象。
- 新手礼包或初始收获提供短期满足。
- 长期目标形成暗钩。
- 危机和堵路让读者追下一章。

## 断档风险

- 书名和正文表里不一。
- 书名不知所云。
- 第一章缺少必要要素。
- 金手指介绍过长。
- 乱换地图导致承诺失焦。

## 训练任务

拿一个书名写第一章要素清单：300 字内写主角出现和行为，500 字内写金手指，500 字内写危机赌注和堵路。检查是否满足书名期待。
