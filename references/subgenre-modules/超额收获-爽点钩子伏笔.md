# 超额收获-爽点钩子伏笔

依据案例：`batch-049` 案例 243。

## 读者承诺

主角完成原目标后得到额外收益，当场提供爽点，后文提供钩子、伏笔或破局工具，让读者既爽又好奇“这个东西以后怎么用”。

## 核心循环

```text
原任务 -> 基础奖励 -> 额外收益 -> 当场爽点 -> 短期回收 -> 长期破局
```

## 开篇必备

- 先明确原任务和基础奖励。
- 超额收获要比基础奖励多一层惊喜。
- 至少一个超额收获要短期回收。
- 长期伏笔要让读者有记忆点。
- 收获数量不能多到读者记不住。

## 期待来源

- 多拿一层奖励带来即时爽。
- 读者好奇额外奖励后文怎么用。
- 小物件、小人情、小信息关键时刻回收，会带来布局感。
- 超额收获引发新意外，可以自然推进剧情。

## 断档风险

- 一次送太多，读者记不住。
- 只送不用，会像堆奖励。
- 所有收获都太适配，会显得机械。
- 完全不适配后文，会浪费篇幅。

## 训练任务

给一个任务桥段设计 3 个超额收获：一个立刻用，一个 10 章内用，一个卷末破局用，并标出各自功能。
