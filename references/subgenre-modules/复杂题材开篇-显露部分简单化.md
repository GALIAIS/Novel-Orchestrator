# 复杂题材开篇-显露部分简单化

依据案例：`batch-068` 案例 337。

## 读者承诺

这个模块用于玄幻谍战、权谋、悬疑、复杂反转类开篇。它承诺的是题材可以复杂，但开篇显露给读者的目标、任务和危险必须简单明快。

## 核心循环

```text
复杂设定后台运行 -> 前台只给简单任务 -> 读者看懂目标 -> 进入危险 -> 后续逐步展开反转
```

## 开篇必备

- 主角当前任务一句话能说清。
- 失败代价直接可见。
- 首个阻碍简单具体。
- 致敬桥段脱离原作也能成立。
- 复杂计划和多层反转后置。

## 期待来源

- 简单任务降低进入门槛。
- 明确危险让读者不用猜。
- 隐藏复杂计划制造后续反转。
- 主角执行先吸引，解释后补。
- 读者先顺读，再承受复杂度。

## 断档风险

- 开篇上来十几个步骤。
- 致敬只对少数人有效。
- 读者还没代入就被迫记设定。
- 每几句都要停下来理解。
- 作者炫结构，读者看不见故事。

## 训练任务

把一个复杂计划拆成“前台三句”和“后台十条”。开篇只能使用前台三句，后台信息必须等主角行动后逐条露出。
