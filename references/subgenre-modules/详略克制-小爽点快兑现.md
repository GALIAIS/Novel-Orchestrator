# 详略克制-小爽点快兑现

依据案例：`batch-044` 案例 216。

## 读者承诺

小爽点要快速兑现，不该抢主线主菜的篇幅。强主角碾压低级对手时，利索比详写更符合人设。

## 核心循环

```text
低级阻碍 -> 主角能力 -> 快速碾压 -> 反应确认 -> 转入主菜
```

## 开篇必备

- 明确这段是小冲突。
- 对手层级低于主角。
- 快速展示能力。
- 结尾接回真正目标。

## 期待来源

- 读者看到主角符合人设地解决麻烦。
- 见证者反应放大状态。
- 小冲突验证前置铺垫。
- 主线重点没有被打断。

## 断档风险

- 低级对手打得有来有回。
- 为了打戏字数牺牲主线。
- 爽点体量和对手层级不匹配。
- 不知道这一段只是开胃菜。

## 训练任务

写一个 400 字碾压桥段：只完成能力展示、对手败退、见证反应和主线承接。
