# 素材软件-长篇设定库与复杂度

依据案例：`batch-226` 案例 1127。

## 读者承诺

用工具集中管理角色、势力、地图、名词和大纲，降低长篇素材分散带来的空白焦虑。

## 核心循环

```text
素材缺口 -> 工具建库 -> 生成/管理设定 -> 写作调用 -> 人工筛选
```

## 开篇必备

- 先确认项目是否真的需要复杂设定库。
- 只启用当前有用功能。
- 每条素材写功能和冲突关系。
- 软件输出必须经过作者筛选。

## 期待来源

素材不再散落，长篇项目的人物、地图、势力、名词有统一入口。

## 断档风险

- 功能太复杂，反而不写正文。
- 把素材生成误当故事能力。
- 早期软件有 bug、UI 和平台限制。
- 推荐工具时像推销，缺适用场景。

## 训练任务

用任意工具或表格建立长篇素材库：10 个角色、5 个势力、5 个地点、20 个名词。每条只写功能、冲突关系、首次出场用途。
