# 公开处刑-设定过载与站队缺失

依据案例：batch-369 案例 1841

## 读者承诺

开局先让读者关心主角，再让读者理解世界；设定过载、低俗劝退点和不公平伤害缺失会让站队失败。

## 核心循环

```text
主角处境 -> 不公平伤害 -> 读者站队 -> 少量设定 -> 冲突推进
```

## 开篇必备

- 低阅读成本设定。
- 主角明确受压。
- 一个站队理由。
- 删掉抢先劝退的低俗点。

## 期待来源

- 主角被不公平对待。
- 读者想看他反击。
- 世界观不妨碍理解。
- 冲突先于设定解释。

## 断档风险

- 简介堆五六种自创体系。
- 节奏快但情绪没上来。
- 低俗桥段先打消兴趣。
- 文笔好但承诺不清。

## 训练任务

重写一个灵气复苏开局，只保留 1 个设定，前三页出现不公平伤害和主角选择。
