# 复杂设定-有事直说开篇

依据案例：`batch-025` 案例 125。

## 读者承诺

复杂设定开篇先说清主角状态、核心规则和当前事件，避免读者把基础预期猜错。

## 核心循环

```text
状态直说 -> 规则成体系 -> 单一事件 -> 铺垫行为 -> 冲突推进 -> 再补设定
```

## 开篇必备

- 主角是穿越、游戏、重生还是其他。
- 能否退出或回到现实。
- 本章唯一核心事件。
- 读者最可能误判的点。

## 期待来源

- 读者不用猜基础规则。
- 注意力集中到当前冲突。
- 背景信息服务行为和矛盾。
- 简单事件可以写出画面感。

## 断档风险

- 把主角状态藏成谜。
- 设定碎片左一脚右一脚。
- 第一章同时塞太多事件。
- 简介写成设定说明书。

## 训练任务

重写一个复杂设定第一屏：第一段直接排除最大误判，第一章只保留一个核心事件，其他设定全部后置。
