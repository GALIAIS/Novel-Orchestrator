# 金手指关系-人物强弱组合

依据案例：`batch-099` 案例 494。

## 读者承诺

金手指不仅是奖励发放器，也可以是伙伴、限制器、互坑对象、宠物或 CP 感来源。设计重点是主角与金手指的强弱、可靠度和互动收益。

## 核心循环

```text
主角定位 -> 金手指定位 -> 关系类型 -> 互动冲突 -> 奖励/限制/笑点 -> 成长
```

## 开篇必备

- 男频主角不能显弱智。
- 金手指可以坑，但要能被主角利用。
- 关系类型要服务题材承诺。
- 金手指限制主角时，要产生新玩法。

## 期待来源

- 强强联手的爽感。
- 互坑关系的喜感。
- 拖后腿限制器防止碾压。
- 互补关系带来陪伴和成长。

## 断档风险

- 系统骂主角，读者觉得憋屈。
- 主角被金手指牵着走，失去能动性。
- 金手指太强，主角没有发挥。
- 金手指太弱，只剩吐槽。

## 训练任务

同一题材设计四版金手指关系：强强、互坑、拖后腿、互补。每版写清主角如何拿到收益且不显弱智。
