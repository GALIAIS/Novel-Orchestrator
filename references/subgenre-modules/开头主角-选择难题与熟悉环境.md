# 开头主角-选择难题与熟悉环境

依据案例：batch-373 案例 1862

## 读者承诺

主角人设要在压力选择中显形。开头用熟悉环境降低代入成本，再用难题逼主角做选择。

## 核心循环

```text
熟悉环境 -> 难题压迫 -> 主角选择 -> 性格显形 -> 后续选择验证
```

## 开篇必备

- 读者熟悉的场景。
- 一个具体难题。
- 至少一个有压力的选择。
- 选择暴露的核心性格。

## 期待来源

- 主角怎么选。
- 主角为什么这样选。
- 选择带来的后果。
- 后续能否保持人设一致。

## 断档风险

- 用设定表介绍人设。
- 难题太轻，选择无压力。
- 前后选择互相打架。
- 环境陌生又解释过多。

## 训练任务

设计三个连续选择，压力逐级升高，但都指向同一核心性格。
