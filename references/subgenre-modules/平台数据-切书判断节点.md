# 平台数据-切书判断节点

依据案例：`batch-129` 案例 644。

## 读者承诺

早期收藏低会制造切书焦虑，但单一数据不能决定作品生死。推荐、签约反馈、上架 24 小时和收入目标，才是更可靠的判断节点。

## 核心循环

```text
早期数据焦虑 -> 平台经验校准 -> 等强信号节点 -> 对照收入目标 -> 决定继续或切书
```

## 开篇必备

- 记录字数和收藏。
- 区分同人/原创。
- 确认是否有推荐。
- 确认是否签约。
- 设定清楚的切书判断节点。

## 期待来源

- 作者有现实压力。
- 数据变化提供悬念。
- 不同建议形成判断冲突。
- 暂缓切书保留起飞窗口。

## 断档风险

- 只看裸收藏。
- 没上推荐就判死刑。
- 混用同人和原创标准。
- 收入目标没有数据验证路径。

## 训练任务

写一个作者早期数据求助片段：包含具体数据、收入目标、两个相反建议、一个暂缓切书判断节点。
