# 科幻进化样本-能力限制主角选择

依据案例：batch-267 案例 1333

## 读者承诺

在科幻、进化、异能样本中，把金手指限制设计成主角性测试：能力越稀缺，选择越能证明主角。

## 核心循环

```text
能力规则 -> 次数/代价限制 -> 任务失败 -> 更虚弱重试 -> 高代价救人选择
```

## 开篇必备

- 能力规则要清楚，尤其是次数和代价。
- 第一次使用要展示收益，也展示限制。
- 反派或小 boss 要有可理解的不公平伤害来源。
- 主角选择要高于普通人的自保逻辑。

## 期待来源

期待来自读者知道机会有限：每次使用能力都在消耗未来，每次重试都让主角更接近死亡或失败。

## 断档风险

- 时间回溯没有代价，紧张感消失。
- 主角选择只是被迫，不显主角性。
- 反派动机只有坏，没有情绪来源。
- 推荐样本只学气质，不学具体机制。

## 训练任务

设计一个三次使用的能力：第一次失败、第二次勉强成功、第三次会严重伤害主角。写清主角为什么仍然要用第三次。
