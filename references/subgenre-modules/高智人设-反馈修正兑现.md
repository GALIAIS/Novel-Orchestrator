# 高智人设-反馈修正兑现

依据案例：`batch-077` 案例 384。

## 读者承诺

这个模块用于借用高智、高专业或强原型人物时的连载修正。它承诺用数据和章节评确认落差，再把聪明具体写成洞察、布局、推理和破局。

## 核心循环

```text
高智承诺 -> 正文表现 -> 读者反馈 -> 定位落差 -> 补聪明场景 -> 再看反馈
```

## 开篇必备

- 明确读者对人设的预期能力。
- 世界观铺设不能替代能力展示。
- 高智必须通过行动证明。
- 章节评要按位置归类。
- 修正后继续观察反馈变化。

## 期待来源

- 借用强标签带来的天然期待。
- 主角比别人多看一步。
- 读者看到布局回收。
- 推理或信息差解决问题。
- 反馈被正文兑现。

## 断档风险

- 只说主角聪明，不写聪明行为。
- 铺世界观太久，主角能力迟迟不出。
- 忽视集中反馈。
- 用受众年龄解释问题。
- 修正只改台词，不改事件设计。

## 训练任务

给高智主角写三场证明：开场洞察、章中布局、章末破局。每场都必须让配角或事件结果证明主角比别人多看一步。
