# 新人签约-基础套路扫榜

依据案例：`batch-013` 案例 065。

## 读者承诺

新人想签约和拿成绩，先让编辑、算法和读者看懂你掌握了哪个流派的基础套路，再谈个人风格和创新。

## 核心循环

```text
扫榜 -> 扒流派骨架 -> 书名简介亮点 -> 黄金三章展示 -> 内投/发布 -> 反馈试错
```

## 开篇必备

- 选定目标平台和目标流派。
- 扫近期样本，不只看老经典。
- 扒背景、升级体系、经典事件。
- 书名简介先亮出标签和卖点。
- 前三章展示流派特征，不乱融花活。

## 期待来源

- 编辑一眼知道这本书大概怎么写。
- 算法和读者从书名简介开篇获得一致标签。
- 新人通过基础套路稳定上路。
- 每次试错能沉淀成自己的可复制经验。

## 断档风险

- 只学理论不写正文。
- 不知道自己写的流派，样本也不看。
- 过早追求花活，基础套路没站稳。
- 书名简介有亮点，正文前三章没有兑现。

## 训练任务

扫 3 本目标流派近期样本，整理共同骨架，再写书名、简介、6000 字正文和大纲，作为一次完整内投材料。
