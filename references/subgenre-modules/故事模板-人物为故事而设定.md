# 故事模板-人物为故事而设定

依据案例：`batch-235` 案例 1174。

## 读者承诺

人物设定要服务一句话故事、核心任务、危机和关系推进，而不是孤立炫设定。

## 核心循环

```text
一句话故事 -> 核心任务 -> 角色功能 -> 特殊性制造/解决危机 -> 关系推进
```

## 开篇必备

- 一句话故事清楚。
- 角色存在理由清楚。
- 特殊能力尽快展示用途。
- 人设反差能进入事件。

## 期待来源

读者看到每个角色都在推动故事成立，既有趣味，又有危机和解决方案。

## 断档风险

- 为设定而设定。
- 角色特殊性不参与剧情。
- 标题不能概括核心内容。
- 主角动机不自然。

## 训练任务

写一句话主线，再设计 3 个角色。每个角色写清故事为什么需要 TA、TA 制造什么危机、解决什么危机。
