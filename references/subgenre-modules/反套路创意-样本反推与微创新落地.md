# 反套路创意-样本反推与微创新落地

依据案例：`batch-220` 案例 1096。

## 读者承诺

反套路不是凭空硬想，而是总结原套路后反向、借鉴、修改，落成可写桥段。

## 核心循环

```text
原套路样本 -> 反向推演 -> 保留原爽点 -> 改造冲突 -> 可写变体
```

## 开篇必备

- 先写清原套路爽点。
- 反向后仍保留读者收益。
- 第一章能兑现新冲突。
- 不把点子焦虑当创作。

## 期待来源

读者既能识别熟悉类型，又能看到一个新变化带来的新冲突。

## 断档风险

- 为反套路删掉爽点。
- 没经验时硬想创新。
- 点子只有一句话，写不成长线。
- 依赖别人公开好点子。

## 训练任务

选 5 个套路，每个写原爽点、反向点、风险、保留爽点方式和前三章事件。
