# 平台决策-时效证据与风险矩阵

依据案例：batch-390 案例 1946

## 读者承诺

平台选择以明确目标、当前官方规则、近期可比样本和可承受损失为依据，不让过期传闻或孤注一掷口号代替决策。

## 核心循环

```text
明确目标与子类型 -> 收集带日期证据 -> 比较收益权利更新和流量 -> 设继续退出条件 -> 有限试错并复盘
```

## 开篇必备

- 现金流、反馈或品牌的优先级。
- 细化到读者体验的题材子类。
- 当前官方规则与近期同阶段样本。
- 最坏损失、继续阈值和退出条件。

## 期待来源

- 平台口号被拆成可核验变量。
- 作品与具体读者样本进行匹配。
- 合同和更新负担进入同一决策。
- 第一本失败仍能留下可复盘证据。

## 断档风险

- 把旧福利数字当现行规则。
- 只看头部成绩和平台总流量。
- 用传闻补齐关键未知。
- 把一次投稿变成身份成败赌注。

## 训练任务

为两个平台各建一页证据卡，给每条信息标来源、日期和置信度；缺少当前官方证据时只列待核验项，不下最终结论。
