# 创作状态-卡文休息与存稿缓冲

依据案例：`batch-201` 案例 1005。

## 读者承诺

突然不会写时先按状态问题处理：停下休息，用存稿兜底，恢复判断力后再定位真实卡点。

## 核心循环

```text
状态失控 -> 停止硬写 -> 存稿缓冲 -> 休息恢复 -> 回来定位卡点
```

## 开篇必备

- 先判断是疲劳还是剧情卡点。
- 有存稿时优先用存稿保更新。
- 不在混乱状态下推翻大纲。
- 休息后再处理具体场景。

## 期待来源

稳定输出来自可恢复的写作流程，而不是每天都靠满状态硬撑。

## 断档风险

- 状态崩时继续硬写，产出大量废稿。
- 没有存稿，单日状态直接影响更新。
- 把疲劳误判成作品整体失败。
- 临时大改导致问题扩大。

## 训练任务

做一张卡文急救清单：复读上一章目标，列下一场戏结果，写 200 字低质量草稿；仍不顺就停止正文写作。
