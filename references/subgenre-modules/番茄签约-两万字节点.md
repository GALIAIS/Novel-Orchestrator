# 番茄签约-两万字节点

依据案例：`batch-140` 案例 697。

## 读者承诺

给新人一个可记录的番茄签约验证节点：章节数、总字数、提交时间、等待状态和日更能力都要进入日志，而不是只凭感觉期待签约。

## 核心循环

```text
开书写到阶段字数 -> 提交签约 -> 记录等待状态 -> 观察反馈 -> 复盘下一次验证节点
```

## 开篇必备

- 平台和题材。
- 当前章节数。
- 当前总字数。
- 是否已提交签约。
- 日更字数或产能。

## 期待来源

- 新人有明确行动节点。
- 同伴签约案例提供参照。
- 日更能力被提前暴露。
- 等待过程有记录可复盘。

## 断档风险

- 只祈祷签约，不记录过程。
- 写太多才发现方向不行。
- 不知道自己日更能力能否支撑后续。
- 把单个案例误当平台硬规则。

## 训练任务

写一份签约节点日志，记录平台、题材、章节数、总字数、日更字数、提交时间、等待状态。再写 100 字说明：若未过签，下一次最早在哪个字数节点验证。
