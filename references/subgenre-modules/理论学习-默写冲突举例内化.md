# 理论学习-默写冲突举例内化

依据案例：`batch-074` 案例 368。

## 读者承诺

这个模块用于把教程、课程、拆书笔记转成可调用的写作能力。它承诺通过间隔默写、对照冲突、复看修正和自家举例，让理论从“听懂了”变成“写作时能用”。

## 核心循环

```text
输入理论 -> 间隔默写 -> 对照原文 -> 区分记错/不同意 -> 自己举例 -> 写作卡点调用
```

## 开篇必备

- 每条理论必须有明确来源。
- 默写时不能看原笔记。
- 对照时要记录差异，不只改答案。
- 冲突要判断原因：记错、理解偏差、理念不同。
- 最后必须落到自己的大纲或章节例子。

## 期待来源

- 默写暴露真实掌握程度。
- 冲突分析让理论变成自己的判断。
- 自家例子降低照搬模板的风险。
- 卡文时复盘理论，比空刷教程更有效。
- 长期积累后形成个人方法库。

## 断档风险

- 只摘抄金句，不做间隔回忆。
- 默写错了只改字面，不查原因。
- 把理念不同误判为自己全错。
- 举例仍然使用教程原例，没有迁移到自己的作品。
- 学习复盘不进入写作流程。

## 训练任务

建立一张“理论默写卡”。每张卡包含来源、隔日默写、差异、冲突判断、自家例子、下一次使用位置。连续做十张后，筛出三条真正能改善当前大纲的理论。
