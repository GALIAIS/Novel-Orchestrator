# 情绪拉扯-预期反转与需求积累

依据案例：`batch-017` 案例 082。

## 读者承诺

笑话式预期反转和网文式情绪拉扯不同。网文拉扯更重持续需求积累：读者知道主角能赢，但作者合理延迟兑现。

## 核心循环

```text
主角优势已知 -> 兑现被阻隔 -> 需求积累 -> 再阻隔 -> 最终集中满足
```

## 开篇必备

- 读者要先知道主角有优势。
- 阻隔必须合理。
- 每次阻隔增加一个新需求。
- 最终释放要回收所有需求。

## 期待来源

- 主角被误解。
- 主角有能力却暂不出手。
- 旁人替主角做无效尝试。
- 最后所有人一起被打脸。

## 断档风险

- 只有一次预期反转，没有持续积累。
- 阻隔不合理，读者觉得作者强行拖。
- 主角优势读者不知道，变成憋屈。
- 释放只满足最后一层，没有回收前面需求。

## 训练任务

写一场主角有能力但暂不出手的剧情，至少两次合理阻隔，最后一次释放同时满足两层期待。
