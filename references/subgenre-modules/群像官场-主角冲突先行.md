# 群像官场-主角冲突先行

依据案例：batch-264 案例 1318

## 读者承诺

古代官场、群像和轻松日常都可以写，但开篇必须先让读者抓住主角、任务、冲突，再逐步展开设定。

## 核心循环

```text
主角登场 -> 任务降临 -> 难点明确 -> 主角行动 -> 官场设定补入
```

## 开篇必备

- 第一段主角。
- 一个具体任务。
- 任务为什么不可能。
- 本章可见冲突。
- 需要后置的设定清单。

## 期待来源

读者期待看主角如何用口才、道理或现代认知应付官场难题，而不是先背设定。

## 断档风险

开篇介绍士人、朝廷、国朝概念过多，会劝退；群像没有主角入口，读者找不到故事。

## 训练任务

把一个官场设定开头删到 1000 字：主角、任务、难点必须前三段出现。
