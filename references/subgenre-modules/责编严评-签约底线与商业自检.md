# 责编严评-签约底线与商业自检

依据案例：`batch-217` 案例 1081。

## 读者承诺

把严厉编辑反馈转化为商业自检，判断作品是否达到最低可卖标准。

## 核心循环

```text
投稿作品 -> 编辑快判 -> 商业底线不足 -> 拒稿/严评 -> 自检改稿
```

## 开篇必备

- 明确读者为什么点进来。
- 前三章有清楚故事和卖点。
- 判断最低均订预期是否有依据。
- 把刺耳反馈拆成可修改问题。

## 期待来源

作者不再靠自我感觉判断作品，而是回到编辑和读者的最低商业要求。

## 断档风险

- 只记住骂人话术，不拆问题。
- 把责编性格当签约规律。
- 作品没有脑洞、故事和节奏底线。
- 用“想被骂醒”代替改稿。

## 训练任务

用 8 个问题做编辑 30 秒自检：凭什么签、凭什么有 500 均、前三章凭什么追、最大拒稿点是什么。
