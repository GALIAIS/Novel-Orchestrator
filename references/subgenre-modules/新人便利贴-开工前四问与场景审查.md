# 新人便利贴-开工前四问与场景审查

依据案例：batch-360 案例 1797

## 读者承诺

新人把踩过的坑写成开工前可见清单，反复提醒主角目的、阻碍、不行动代价和场景功能，减少低级结构错误。

## 核心循环

```text
写前看清单 -> 明确目的/代价/阻碍 -> 写场景小故事 -> 写后审查删改
```

## 开篇必备

- 主角为什么做。
- 不做会付出什么代价。
- 阻碍是什么。
- 本场目的是什么。
- 这个场景讲了哪个小故事。

## 期待来源

- 主角行动清楚。
- 冲突和阻碍持续存在。
- 每场都有变化。
- 一个场景承载多个有效信息。

## 断档风险

- 靠记忆写，忘掉核心问题。
- 场景只解释不变化。
- 主角目的和阻碍不清。
- 写完不复查。

## 训练任务

给下一章每场戏写一张便利贴：目的、代价、阻碍、小故事。写完逐项打勾，不过关就删改。
