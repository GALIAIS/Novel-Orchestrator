# 扒文进阶-先总后分精扒

依据案例：`batch-031` 案例 151。

## 读者承诺

扒文先拉全文框架，再按问题分层取证，避免处处都扒导致半途而废。

## 核心循环

```text
明确目的 -> 选对口样本 -> 全文框架 -> 专项分层 -> 自文验证
```

## 开篇必备

- 本次扒文目的。
- 样本与自己方向对应。
- 第一遍只做总框架。
- 后续每遍只解决一个问题。

## 期待来源

- 从混乱全扒转为问题驱动。
- 能逆推作者布局。
- 每次精扒都有可迁移结论。
- 结论能回到自己的文验证。

## 断档风险

- 一章扒几千字。
- 看什么都觉得重要。
- 只拆细节不看框架。
- 只收集方法不用自己文验证。

## 训练任务

选一本同方向作品，第一遍只写全文框架，第二遍只拉伏笔，第三遍只拉配角，不许混在一起。
