# 维权证据链-全量扫描比对

依据案例：`batch-064` 案例 320。

## 读者承诺

这个模块用于教程、视频、拆书和创作资料被系统性搬运时的维权。它承诺的是从单条发现升级到全账号扫描、已删内容保存、逐条原文比对和公开证明，减少只有情绪没有证据的低效控诉。

## 核心循环

```text
单条提醒 -> 全量扫描 -> 保存视频/截图 -> 原文逐条比对 -> 证明系统性搬运 -> 投诉/公开维权
```

## 开篇必备

- 记录提醒来源。
- 扫描疑似账号全部相关内容。
- 保存已删或可能被删的视频证据。
- 每条都对应原文来源和相同句段。
- 区分转发、引用、洗稿和照搬。

## 期待来源

- 全量证据提高投诉成功率。
- 逐条比对让围观者看懂问题。
- 保存删除前证据防止对方灭证。
- 正常转发不被误伤。
- 社群形成反剽窃共识。

## 断档风险

- 只截标题，不保存视频内容。
- 只证明一条，没证明系统性。
- 没有原文对照。
- 把正常转发当侵权。
- 对方删稿后证据不足。

## 训练任务

建立一份 5 条证据目录：搬运标题、搬运链接、截图路径、原文链接、相同句段、相似类型、是否已备份。目录完成前不写长篇控诉。
