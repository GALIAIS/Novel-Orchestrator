# 飞卢开局-编辑入库与上架验证

依据案例：`batch-345` 案例 1723

## 读者承诺

飞卢开局要把编辑沟通、入库曝光、首日反馈、合同和上架爆更作为同一套流程处理。

## 核心循环

```text
联系编辑 -> 样稿确认 -> 发书入库 -> 24小时反馈 -> 合同节点 -> 上架爆更
```

## 开篇必备

- 发书前先联系编辑。
- 准备开头样稿。
- 发书后给编辑书号书名。
- 小说带封面。
- 上架前提前处理合同。

## 期待来源

作者期待作品进入平台流量入口；没有入库曝光，开局数据无法真实反映作品能力。

## 断档风险

- 自己直接发导致没曝光。
- 不带封面。
- 到上架字数合同没处理。
- 首日反馈不看。
- 上架后没有爆更计划。

## 训练任务

写一份飞卢开书流程表，从联系编辑到上架爆更，每项标明完成证据。
