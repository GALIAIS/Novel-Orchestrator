# 双金手指整合-动机前置开篇

依据案例：`batch-042` 案例 208。

## 读者承诺

开篇可以有前世身份、系统、传承等多个资源，但读者真正需要的是一个清晰核心：主角靠什么破局，为什么必须继续变强。

## 核心循环

```text
书名承诺 -> 单核心金手指 -> 第一困境破局 -> 深层动机 -> 长线成长
```

## 开篇必备

- 书名卖点尽早展示或强预告。
- 金手指最好合成一个主核心。
- 第一困境必须由核心能力解决。
- 主角长期目标要明确。

## 期待来源

- 读者知道核心能力如何成长。
- 主角动机不只是活下去。
- 原身关系或前世执念影响当前选择。
- 第一章能看到卖点兑现。

## 断档风险

- 两个金手指抢戏。
- 主角名字和身份太晚出现。
- 旁白人格化破坏沉浸。
- 留在当前环境的理由太浅。

## 训练任务

把一个双金手指开篇改成单核心版本：删除或合并一个能力，并补出主角的深层动机。
