# 读者冒犯-情节设定与隐性踩雷

依据案例：`batch-249` 案例 1242。

## 读者承诺

读者冒犯模块用于检查情节是否在作者无意识中踩到读者的尊严、公平感、代入身份或价值底线。

## 核心循环

```text
情节功能设计 -> 读者立场复核 -> 找出隐性冒犯点 -> 调整责任和表达 -> 保留冲突但不断信任
```

## 开篇必备

- 主角站队要清楚。
- 冒犯性行为的责任归属要清楚。
- 冲突对象不能误伤目标读者的代入身份。
- 下行情节要给读者继续看的信任锚点。

## 期待来源

读者愿意忍受冲突，是因为相信文本站在他们愿意代入的一边；一旦情节冒犯了他们，后续爽点会提前失效。

## 断档风险

- 作者只看情节工具性，不看读者感受。
- 把羞辱当成铺垫，把冒犯当成冲突。
- 解释太晚，读者已经弃读。
- 修复时只补说明，没有改变立场和责任归属。

## 训练任务

选一段下行情节，分别从主角粉、同类身份读者、价值观敏感读者三个角度写出可能被冒犯的点，再给每处设计一个保留冲突的降冒犯改法。
