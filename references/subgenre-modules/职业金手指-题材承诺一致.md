# 职业金手指-题材承诺一致

依据案例：batch-267 案例 1331

## 读者承诺

让职业流或身份流主角的金手指稳定服务题材承诺，避免能力杂乱导致读者不知道自己在看什么爽点。

## 核心循环

```text
职业身份 -> 核心能力 -> 能力边界 -> 事件池筛选 -> 同类爽点连续兑现
```

## 开篇必备

- 第一章就明确主角职业/身份承诺。
- 金手指先围绕职业能力展开。
- 事件要能自然展示职业优势。
- 越界能力必须有强限制或强代价。

## 期待来源

期待来自读者知道“这本书会反复怎么爽”。神医就期待治病、救人、医术反转和病患身份变化；能力乱飞会打断这种稳定期待。

## 断档风险

- 金手指塞太多无关能力。
- 事件池和职业无关，像大纲崩盘。
- 能救死人却没有代价，边界崩掉。
- 对话堆太多，核心事件不推进。

## 训练任务

给一个职业主角列“可写事件池/禁止事件池”。每个可写事件标注对应能力，每个禁止事件标注破坏哪条题材承诺。
