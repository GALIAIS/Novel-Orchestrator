# 日轻开篇-废话取舍平台适配

依据案例：`batch-121` 案例 603。

## 读者承诺

日轻开篇可以轻快、吐槽、恋爱和异世界混搭，但第一章仍要交代主角、主线、世界和危机。平台受众不匹配时，再多噱头也会变成毒点。

## 核心循环

```text
轻小说卖点 -> 平台受众判断 -> 第一章减废话 -> 信息落点明确 -> 危机推动主线 -> 保留轻快风味
```

## 开篇必备

- 主角身份一句话说清。
- 主线目标一句话说清。
- 世界规则给当前必要信息。
- 第一章必须有有效事件。
- 平台要匹配日轻受众。

## 期待来源

- 读者期待卖点快速展开。
- 异世界和角色关系制造轻快冲突。
- 危机让吐槽不空转。
- 平台受众认可文风后才愿意追。

## 断档风险

- 心理描写多但不推动行动。
- 前三章仍看不懂主线和世界。
- 信息跳级，人物出场无功能说明。
- 把刺猬猫、SF、日轻受众混为一谈。

## 训练任务

把前三章压成五句：主角、主线、世界、危机、卖点。删掉不能服务这五句的开篇废话，再判断目标平台是否适配。
