# 推荐反馈-切书止损节点

依据案例：`batch-135` 案例 675。

## 读者承诺

推荐位反馈差时，不凭情绪立刻切书。先确认推荐位置和数据变化，再归因作品内因，最后设置继续观察或止损节点。

## 核心循环

```text
上推荐 -> 数据未起 -> 校验推荐价值 -> 复盘作品问题 -> 设定止损节点
```

## 开篇必备

- 推荐名称和端口。
- 推荐前后数据。
- 作者第一反应。
- 作品内因判断。
- 继续观察或切书节点。

## 期待来源

- 数据让判断具体。
- 推荐期间继续吃曝光。
- 失败转成能力复盘。
- 切书有节点，不是情绪崩溃。

## 断档风险

- 一上推荐没涨就立刻切。
- 不区分网页端和手机端推荐。
- 只怪推荐，不复盘爽点。
- 没有明确止损线。

## 训练任务

写 100 字推荐复盘：曝光机会、数据未达预期、情绪反应、一个判断依据、一个继续观察或止损节点。
