# 校园爽点-不公平伤害与系统格斗

依据案例：batch-365 案例 1822

## 读者承诺

校园危机爽点要先让读者站队，再让系统格斗兑现压倒性胜利；被帮助对象越无辜，主角出手越顺应民意。

## 核心循环

```text
朋友遇险 -> 解决希望 -> 隐情反转 -> 被堵羞辱 -> 系统格斗 -> 围观敬佩
```

## 开篇必备

- 低门槛校园危机。
- 被保护对象值得保护。
- 对方仗势欺人。
- 主角金手指能即时兑现。

## 期待来源

- 每次上行都被更强下行打断。
- 混混身份逐步抬高。
- 主角先被羞辱再反击。
- 围观者见证实力反转。

## 断档风险

- 被保护对象理亏，读者不想主角帮。
- 只堆敌人数量，不升级压力。
- 金手指爆发前缺羞辱燃料。
- 胜利后没有围观反馈和关系收益。

## 训练任务

把一场校园堵人戏改成五段上行/下行链，每一段都要加强读者对主角出手的支持。
