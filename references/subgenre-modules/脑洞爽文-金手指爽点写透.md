# 脑洞爽文-金手指爽点写透

依据案例：`batch-108` 案例 538。

## 读者承诺

脑洞爽文不是把多个爽点赶进前三章，而是把第一个金手指爽点写透。读者要先共情主角遭遇的不公平，再看到主角用金手指改变局面，并通过场面、对手和观众反应确认爽感。

## 核心循环

```text
不公平伤害 -> 金手指触发 -> 主角出手 -> 对手被压低 -> 观众反应 -> 限制留钩子
```

## 开篇必备

- 原身或主角受到的不公平伤害。
- 金手指一句话卖点。
- 第一个装逼点完整写透。
- 主角出手过程有画面。
- 金手指有边界，防止开局无敌崩盘。

## 期待来源

- 读者想替主角讨回公道。
- 百分百触发带来的确定爽感。
- 对手强度先被衬托，再被主角压过。
- 旁观者震惊放大打脸效果。

## 断档风险

- 三章塞太多装逼点。
- 爽点蜻蜓点水。
- 世界观和金手指解释太草。
- 完全无敌没有限制，后续写不长。

## 训练任务

把一个脑洞金手指开篇缩到一个爽点：补齐伤害、触发、出手、对手反应、观众反应和限制，其他爽点全部后移。
