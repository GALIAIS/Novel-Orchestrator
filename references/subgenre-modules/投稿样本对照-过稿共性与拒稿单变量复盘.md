# 投稿样本对照-过稿共性与拒稿单变量复盘

依据案例：`batch-406` 案例 2026

## 读者承诺

用同一作者、相近时期的通过稿和拒稿稿建立个人能力基线；先排除渠道等混杂变量，再按开篇承诺和状态变化比较，只改一个变量复测并写完首个单元。

## 核心循环

```text
收集通过与拒稿样本 -> 记录渠道要求排除混杂 -> 同维度比较开篇状态 -> 建立竞争拒稿假设 -> 单变量改稿 -> 同渠道或目标读者复测 -> 成立机制写入首个单元
```

## 开篇必备

- 记录渠道、编辑、方向和提交时间。
- 比较书名简介、人物目标和首个变化。
- 慢与快都用状态变化而非字数判断。
- 无明确反馈时保留多个竞争解释。
- 通过稿作为自身可达基线。
- 反馈提交遵守规则并明确问题。

## 期待来源

拒稿不再是无从理解的否定；作者从自己的通过稿中找到可复现差异，经单点复测后把开篇能力推进到完整单元。

## 断档风险

- 附件不可读仍评价具体文本。
- 不同渠道结果直接比较。
- 作者喜欢等同于编辑会通过。
- 把传统题材或慢调当作唯一原因。
- 同时更换题材、简介和人物。
- 连续写七个开头却不训练中段。

## 训练任务

用两份通过稿和一份拒稿填写同一张对照表，列三个竞争假设，只改证据最强的一项；复测后选择成立机制写到首个完整单元。
