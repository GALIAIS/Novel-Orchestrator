# 开头审核-清单边界使用

依据案例：`batch-172` 案例 857。

## 读者承诺

开头审核清单用于降低新人风险，但不能机械打勾；必须结合书名、简介、题材、受众期待和作者阶段判断。

## 核心循环

```text
书名简介期待 -> 开篇承接 -> 清单自查 -> 题材修正 -> 优先改最大追读问题
```

## 开篇必备

- 先写清读者进书前被什么吸引。
- 冲突最好影响主角或至少波及主角。
- 金手指/优势要带来阶段性收益。
- 无敌文不一定有危机，但必须有冲突。

## 期待来源

读者期待开头兑现书名简介承诺，并在前几万字内看到主角、冲突、优势和情绪拉扯形成追读理由。

## 断档风险

- 把清单当绝对公式。
- 只打勾，不判断读者为什么继续看。
- 慢开局没有提前用书名简介拉期待。
- 新人借大神例外逃避基础要求。

## 训练任务

拿一个开头 2 万字逐项自查，每项写“已兑现、延后兑现、无法兑现”及原因。最后只改最影响追读的 3 项。
