# 作者身份保护-笔名作品谨慎公开

依据案例：`batch-067` 案例 331。

## 读者承诺

这个模块用于作者运营和投稿安全。它承诺的是把作品曝光控制在必要范围内，避免举报、冒名、红眼病和平台完读数据污染。

## 核心循环

```text
需要交流/反馈 -> 判断公开必要性 -> 脱敏或小范围展示 -> 避免身份攻击面 -> 保护作品数据和权益
```

## 开篇必备

- 作者身份和读者入口分离。
- 陌生群聊不直接给笔名和作品名。
- 求诊断时优先脱敏样章。
- 成绩可以说区间，不给可定位信息。
- 熟人推广先评估是否会低完读。

## 期待来源

- 作者获得交流和反馈。
- 作品不被恶意举报。
- 投稿身份不被冒用。
- 平台定向推荐不被非目标读者污染。
- 社交展示不伤害长期收益。

## 断档风险

- 为了炫耀公开笔名。
- 把陌生群友当精准读者。
- 让朋友随手点开又快速弃读。
- 向不可信对象暴露投稿身份。
- 求反馈时没有做脱敏。

## 训练任务

写一份作者身份边界表，列出“可公开、仅熟人、仅编辑、永不公开”四栏。再把一段求反馈话术改成不暴露笔名和作品名的版本。
