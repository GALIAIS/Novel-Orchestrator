# 校园霸凌开局-过去伤害与系统反杀

依据案例：`batch-254` 案例 1266。

## 读者承诺

校园霸凌开局模块把过去伤害转化为读者站队和反杀期待，避免把正文开场写成持续受虐。

## 核心循环

```text
过去霸凌 -> 不公平伤害 -> 当前危机逼近 -> 系统提前给能力 -> 反杀打脸
```

## 开篇必备

- 霸凌经历尽量用简短回忆交代。
- 当前剧情放在危机即将再次发生前。
- 金手指必须先于危机落下出现。
- 主角低位要服务后续打脸，不要长期受辱。

## 期待来源

读者因为过去的不公平伤害站到主角一边，又因为系统出现看到主角有能力处理即将到来的危机，于是期待反杀兑现。

## 断档风险

- 开局现场长时间写主角被打。
- 读者还没看到系统，就先被憋屈劝退。
- 霸凌只提供痛苦，不提供报复目标。
- 反杀后没有世界观和主线延展。

## 训练任务

写一个 800 字校园系统开局：过去伤害不超过 150 字，系统必须在当前危机爆发前出现，并在结尾抛出第一轮反杀对象。
