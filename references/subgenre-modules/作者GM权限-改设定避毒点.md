# 作者GM权限-改设定避毒点

依据案例：batch-294 案例 1469

## 读者承诺

作者有权修改设定，设定一旦逼出毒点，就改前置规则而不是替毒点开脱。

## 核心循环

```text
设定逼出毒点 -> 使用GM权限 -> 改前置规则 -> 保住人设和爽点 -> 补逻辑
```

## 开篇必备

- 当前设定。
- 导致的毒点。
- 可删或可改规则。
- 主角人设优先级。
- 读者需求。
- 最小逻辑补丁。

## 期待来源

读者看到角色行为顺、人际关系不毒，而不是看作者炫复杂设定。

## 断档风险

- 被自己设定绑架。
- 舍不得删毒点。
- 为毒点开脱。
- 设定合理压过人物关系和主角人设。

## 训练任务

找一个毒点设定，写三种改法，选最能保住读者爽点的一种。
