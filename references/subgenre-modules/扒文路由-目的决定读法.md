# 扒文路由-目的决定读法

依据案例：`batch-060` 案例 299。

## 读者承诺

这个模块用于把扒文从无目标阅读变成取证工作。它承诺的是先确定目的，再决定样本、阅读范围、记录字段和输出物。

## 核心循环

```text
确定目的 -> 选样本 -> 设阅读范围 -> 记录证据 -> 提炼框架 -> 用下一本验证
```

## 开篇必备

- 本次扒文只能有一个主目的。
- 市场认知看热题材共性。
- 短板修复看同题材解决方案。
- 成绩原因必须看评论区。
- 输出物要能用于下一次写作或验证。

## 期待来源

- 作者知道为什么看这本书。
- 记录内容和目的匹配。
- 共性在下一本得到验证。
- 短板有可迁移框架。
- 市场口味和个人口味差异被看见。

## 断档风险

- 全文看完但没有问题意识。
- 把特殊桥段当通用框架。
- 不看评论区就判断成绩原因。
- 大目标太大导致扒文痛苦。
- 只收藏规律，不用下一本验证。

## 训练任务

做一次三本书扒文。每本只填同一张表：本次目的、阅读范围、共性爽点、独特点、读者评论证据、可迁移框架。
