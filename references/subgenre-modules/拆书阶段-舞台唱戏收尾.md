# 拆书阶段-舞台唱戏收尾

依据案例：`batch-074` 案例 366。

## 读者承诺

这个模块用于拆解和设计“主角显圣型”事件。它承诺先把问题、负面情绪和期待摆好，再让主角登台解决真实难题，最后用震惊、奖励、关系和新信息完成收尾。

## 核心循环

```text
舞台准备 -> 期待悬置 -> 登台唱戏 -> 场内信服 -> 演出收尾 -> 下一事件
```

## 开篇必备

- 明确地点和配角。
- 配角身上有负面情绪或未解问题。
- 主角有欲望、任务或被卷入的理由。
- 读者知道主角可能解决，但场内角色暂时不知道。
- 问题必须具体到能被解决、证明或反转。

## 期待来源

- 别人的焦虑等主角来消解。
- 主角能力和场内误判之间形成落差。
- 读者提前知道一点，场内角色滞后知道。
- 解决问题后有评价、资源或关系回报。
- 收尾还能给下一场戏留下新问题。

## 断档风险

- 准备阶段只讲设定，没有任务压力。
- 登台阶段只装逼，不解决真实难题。
- 旁观者反应空泛，不能证明主角价值。
- 世界观被干讲，没有成为破局钥匙。
- 收尾只有震惊，没有奖励、关系或新钩子。

## 训练任务

设计一个“主角第一次在专业场合被认可”的事件。写出舞台准备、登台唱戏、演出收尾三段，每段至少三条，并注明读者在哪一刻最期待主角出手。
