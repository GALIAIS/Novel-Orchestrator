# 微创新反套路-金手指优先创新

依据案例：`batch-196` 案例 978。

## 读者承诺

新人做微创新，优先改金手指和首章卖点；成熟套路底盘可以保留，反套路必须服务亮点。

## 核心循环

```text
老套路底盘 -> 金手指新意 -> 首章亮点 -> 读者反馈 -> 小方向调整
```

## 开篇必备

- 开书大方向先定稳。
- 金手指有可感知新意。
- 首章就能看到亮点。
- 其他结构按成熟套路推进。

## 期待来源

读者在熟悉框架里看到新鲜机制，既不需要重新学习整套规则，又能获得亮点刺激。

## 断档风险

- 为反套路而反套路。
- 大方向频繁变化。
- 新人全盘创新导致叙事失控。
- 亮点不在前三章兑现。

## 训练任务

选一个老套路题材，只创新金手指。写出保留套路、创新机制、前三章兑现点和不能改变的基调。
