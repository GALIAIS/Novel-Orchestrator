# 码字工具-三端同步与草稿安全感

依据案例：`batch-251` 案例 1253。

## 读者承诺

码字工具模块优先解决电脑、手机、平板之间的同步断裂，让作者随时记录和续写。

## 核心循环

```text
多设备割裂 -> 选择云同步工具 -> 降低转场成本 -> 定期备份 -> 稳定产出
```

## 开篇必备

- 确认主要写作设备。
- 选择基础云同步可用的工具。
- 设置本地或导出备份。
- 建立每天固定入口。

## 期待来源

作者不再因为设备不同步丢失状态，碎片时间和整块时间能接上同一个草稿。

## 断档风险

- 为工具功能花样分心。
- 被低概率盗文焦虑卡住。
- 只云同步不备份。
- 软件迁移成本大于写作收益。

## 训练任务

用同一个项目文件做三端同步测试：电脑写 200 字、手机补 100 字、平板检查大纲，连续 3 天记录问题。
