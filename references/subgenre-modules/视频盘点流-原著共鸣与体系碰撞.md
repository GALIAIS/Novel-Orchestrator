# 视频盘点流-原著共鸣与体系碰撞

依据案例：`batch-201` 案例 1004。

## 读者承诺

视频盘点流承诺的不是传统剧情推进，而是给熟悉原著的读者名场面回忆、遗憾补完、梗识别和跨作品体系争议。

## 核心循环

```text
盘点主题 -> 原著记忆点 -> 遗憾补完 -> 跨作品对照 -> 体系碰撞 -> 读者站队
```

## 开篇必备

- 选读者熟悉度高的原著或名场面。
- 明确盘点视角和评价标准。
- 给出一个遗憾点或争议点。
- 用弹幕/观众反应放大共鸣。

## 期待来源

读者期待看到自己熟悉的角色、桥段和战力体系被重新解释，并参与“谁更强、谁更遗憾、怎么补”的讨论。

## 断档风险

- 原著太小众，读者 get 不到梗。
- 作者自嗨超过读者共鸣。
- 只复述原著，没有新评价。
- 用传统网文标准误判盘点流。

## 训练任务

写 800 字盘点片段：包含原著记忆点、遗憾点、改写判断、跨作品对照和弹幕反应。
