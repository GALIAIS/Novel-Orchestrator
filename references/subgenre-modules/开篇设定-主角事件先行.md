# 开篇设定-主角事件先行

依据案例：`batch-081` 案例 401。

## 读者承诺

这个模块用于压缩新人开篇的设定堆叠。它承诺让主角、事件和危机先抓住读者，再把背景拆成可带来刺激的分层揭示。

## 核心循环

```text
主角出场 -> 事件启动 -> 危机抓眼 -> 设定后置 -> 分层揭示
```

## 开篇必备

- 前 300 字出现主角。
- 第一场必须有正在发生的事件。
- 题材特性尽快显形。
- 金手指或核心异常不能拖太晚。
- 背景设定只留当场必须理解的部分。

## 期待来源

- 读者先关心主角在做什么。
- 危机让读者继续看。
- 设定每次揭示都带来“原来如此”。
- 概念不跳级，理解成本低。
- 类型承诺不靠简介，靠正文兑现。

## 断档风险

- 前几百字像设定集。
- 主角出场晚。
- 金手指出来晚。
- 同时抛出多个陌生概念。
- 照搬影视或日轻开篇节奏。

## 训练任务

重写第一章前 800 字：删掉 80% 背景设定，保留主角、事件、危机、题材特性。被删的设定拆成 3 次后续揭示，每次必须对应一个剧情刺激。
