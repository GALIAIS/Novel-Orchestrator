# 脑洞投稿开篇-核心事件压缩

依据案例：`batch-046` 案例 230。

## 读者承诺

脑洞文投稿开篇要快进核心事件。书名要能 YY，前 500 字要把金手指、第一世界切入和一个小爽点立住。

## 核心循环

```text
书名承诺 -> 核心事件 -> 金手指关联 -> 小冲突兑现 -> 差异化说明
```

## 开篇必备

- 书名能让读者想象爽点。
- 第一章直接进入核心事件。
- 系统来源和事件有关。
- 主角人设稳定。

## 期待来源

- 读者迅速知道脑洞玩法。
- 金手指不是天上掉下来的。
- 主角能用任务机制装逼。
- 同类文差异明确。

## 断档风险

- 前面写成记叙文。
- 第一章无关内容太多。
- 系统来源承接弱。
- 主角人设突然跃迁。

## 训练任务

把一个脑洞文第一章压到 500 字：保留主角身份、金手指来源、第一世界小冲突和核心爽点。
