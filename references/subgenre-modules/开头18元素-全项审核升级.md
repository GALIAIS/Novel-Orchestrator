# 开头18元素-全项审核升级

依据案例：`batch-167` 案例 831。

## 读者承诺

用全项审核和示范案例校准新人开头，避免作者误以为自己理解标准、误以为开头已经符合。

## 核心循环

```text
开头要素 -> 原文证据 -> 示范对照 -> 缺项补写 -> 全项复查
```

## 开篇必备

- 每一项都要找到原文证据。
- 不理解的条目必须看示范案例。
- 不能用“感觉差不多”代替符合。
- 缺项直接进入补写清单。

## 期待来源

读者期待第一章入口清晰，不需要替作者补主角、目标、困境、关系、金手指和期待。

## 断档风险

- 把基础门槛当加分项。
- 作者自判符合但没有文本证据。
- 只背概念，不会对照原文。
- 示例不足导致新人持续误判。

## 训练任务

拿一章开头逐项找证据。没有证据的项写“缺项”，并补一个具体句段，不允许写“隐含符合”。
