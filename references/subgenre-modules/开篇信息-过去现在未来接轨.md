# 开篇信息-过去现在未来接轨

依据案例：batch-302 案例 1510

## 读者承诺

第一章让读者迅速接轨故事：主角从哪来、现在怎样、接下来要面对什么危机。

## 核心循环

```text
过去 -> 现在 -> 未来危机 -> 第一场事件 -> 章末期待
```

## 开篇必备

- 一码事先写清。
- 变化不要过于频繁。
- 必要信息不能跳过。
- 金手指来历可后置。

## 期待来源

- 读者知道主角在哪、要干嘛。
- 当前危机明确。
- 后续信息有承接位置。

## 断档风险

- 为塞满要素导致信息乱跳。
- 读者还没接轨就连续变化。
- 推理无伏笔，内鬼无线索。

## 训练任务

把一个信息过载开头压成一章一件事：用三句话分别交代过去、现在、未来危机。
