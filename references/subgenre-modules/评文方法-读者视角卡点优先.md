# 评文方法-读者视角卡点优先

依据案例：`batch-339` 案例 1693

## 读者承诺

帮人看文先用普通读者视角快速读，标出想停、卡住、不爽的位置，再用作者视角解释原因。

## 核心循环

```text
一目十行读稿 -> 标卡点 -> 判断是否想继续 -> 作者视角找因 -> 给最小修改建议
```

## 开篇必备

- 第一遍不咬文嚼字。
- 标出想点叉的位置。
- 区分流畅度、爽点和个人相性。
- 文字问题只说影响阅读的。
- 作者视角只补关键原因。

## 期待来源

作者期待知道普通读者何时离开，而不是得到一堆炫技式细枝末节挑错。

## 断档风险

- 只从作者角度逐句挑。
- 把个人相性当普遍毒点。
- 忽略流畅度卡点。
- 不说想不想继续看。
- 反馈太散，作者无从改。

## 训练任务

看 3000 字开头，第一遍只标想点叉的位置，第二遍只解释前三个卡点原因。
