# 人物称谓变焦-心理距离与指代清晰度

依据案例：`batch-413` 案例 2062

## 读者承诺

人物称谓会像叙述镜头一样改变关系距离，但变化始终建立在指代清晰和视角权限上。姓名、职务、关系称呼、评价性称号和私密昵称分别选择人物的一层身份；只有在场合、知识、关系或情绪发生可见变化时改口，读者才能从称谓本身读出身份揭露、疏远、亲近、背叛或和解。

## 核心循环

```text
确定视角已知信息与双方关系 -> 建立稳定叙述锚点 -> 场合、身份或情绪发生变化 -> 选择有权限且能表达新状态的称谓 -> 保留同一人识别线索 -> 称谓改变读者对距离和评价的理解 -> 状态稳定后维持新层级或回到主锚
```

称谓变焦的目标不是避免重复，而是用最小语言记录关系状态变化。没有新增信息和情绪的同义替换，应优先删除。

## 开篇必备

- 每名主要人物一个稳定、易识别的叙述锚点。
- 明确当前视角是否知道姓名、职务和隐藏身份。
- 对话称呼与叙述指代各自的使用权限。
- 公共称呼、关系称呼、评价性称号和昵称的来源。
- 第一次称谓变化的可见触发动作。
- 改口后仍能确认是同一人的上下文线索。
- 一处能用称谓替代关系说明的关键节点。

## 期待来源

- 谁知道人物的真名或私密昵称。
- 公开场合与私下场合为何使用不同称呼。
- 敌我双方对同一称号作何相反解释。
- 一直用职务的人何时第一次叫出姓名。
- 亲近者突然叫全名意味着何种决裂。
- 被隐藏的称谓是否会暴露旧身份和关系史。
- 最终改口会确认怎样的新关系。

## 断档风险

- 一段内为避免重复轮换多个标签，读者误以为出现新人。
- 机械按物理距离决定称谓，忽略关系和场合。
- 视角使用自己尚不知道的姓名或绰号。
- 称号没有评价来源，只是华丽别名。
- 昵称人人都能使用，失去亲密和权限价值。
- 改口没有触发，关系变化显得突兀。
- 每次称谓变化都由旁白解释，文本失去含蓄。
- 稳定关系仍频繁变焦，增加认知负担而无增量。

## 训练任务

写一场 1200 字的合作破裂对话。给双方各设一个稳定指代，全场只允许三次称谓变化，并分别由公开身份、秘密揭露和主动决裂触发。删除解释性旁白后交给冷读者，仅凭称谓顺序判断关系曲线；无法判断的改口重新设计。
