# 选项系统-反套路选择

依据案例：`batch-008` 案例 040。

## 读者承诺

系统给出选项，读者形成预判，主角用符合人设的方式偏离预判，最后产生意想不到但合理的爽点、喜剧或反转。

## 核心循环

```text
选项出现 -> 读者预期 -> 主角选择/回避 -> 结果错位 -> 爽点兑现
```

## 开篇必备

- 系统选项规则清楚。
- 主角底线清楚。
- 第一次选择要展示主角性格。
- 结果反差必须合理，不是作者强行拧。

## 期待来源

- 读者想看主角会不会按系统诱导走。
- 主角越坚持自己的逻辑，反差越有趣。
- 结果越能反向达成目标，喜剧和爽感越强。
- 选项会不断暴露系统、世界和主角之间的错位。

## 断档风险

- 为反套路而反套路，主角选择不符合人设。
- 主角突破底线却没有代价，直接崩人设。
- 选项结果不可预测到像随机事件，读者无法期待。
- 女主或配角优点反复展示，但不推进剧情。

## 训练任务

设计三组选项：一组主角不按套路选，一组主角回避目标却反向达成，一组主角用非常规手段但不破底线。每组写出读者预期和实际结果。
