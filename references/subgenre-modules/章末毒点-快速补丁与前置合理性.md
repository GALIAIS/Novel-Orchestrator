# 章末毒点-快速补丁与前置合理性

依据案例：`batch-047` 案例 233。

## 读者承诺

章末冲突不能牺牲人物逻辑。无脑挑衅如果必须写，要提前给身份、年龄、利益或认知局限，不能总靠下一章补丁。

## 核心循环

```text
章末争议 -> 读者不满 -> 快速补丁 -> 合理性回补 -> 风险复盘
```

## 开篇必备

- 挑衅行为有原因。
- 角色智商不被硬降。
- 章末不只靠套路制造冲突。
- 必要补丁足够快。

## 期待来源

- 读者理解冲突来源。
- 挑衅背后有年龄或利益限制。
- 下一章能立刻止损。
- 角色逻辑被保住。

## 断档风险

- 无脑挑衅太老套。
- 读者带着不满过夜。
- 补丁像临时找借口。
- 新人误以为补丁能替代设计。

## 训练任务

检查一个章末冲突，提前写出挑衅者的年龄、利益、认知局限和行为合理性。
