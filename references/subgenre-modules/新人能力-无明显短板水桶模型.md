# 新人能力-无明显短板水桶模型

依据案例：`batch-223` 案例 1112。

## 读者承诺

作品先不能有明显短板劝退读者，再用强项形成记忆点。

## 核心循环

```text
读者点开 -> 基础不劝退 -> 爽点留人 -> 痛点共鸣 -> 强项记忆
```

## 开篇必备

- 行文、爽点、痛点、自我定位至少都过线。
- 找出当前最大劝退点。
- 只选一个强项做主卖点。
- 把补短板动作落实到前三章。

## 期待来源

读者在可替代品很多的环境里，不愿忍受明显缺陷；无短板是留下来的前提，亮点是追读的理由。

## 断档风险

- 只追求一个亮点，忽略基础短板。
- 把痛点写成卖惨。
- 不理解市场和读者需求。
- 作者定位模糊，什么都想写。

## 训练任务

拿前三章做四项体检：行文、爽点、痛点、定位。每项写一个最大短板和一个最小修复动作，再选一个强项作为本书主卖点。
