# 开篇装逼-先打赢再解释

依据案例：`batch-320` 案例 1598

## 读者承诺

开篇读者先想看主角把事干成，再关心为什么强；先打赢、再解释，常比先解释、再打赢更有爽感。

## 核心循环

```text
普通/善良铺垫 -> 危机压迫 -> 主角直接打赢 -> 再解释强大原因 -> 人设与爽点成立
```

## 开篇必备

- 公开处刑最好正文和小纲一起发。
- 前面铺主角普通、卑微或善良。
- 危机对象保持合理智商。
- 少写人体结构和发力说明。
- 装逼结果先兑现，解释后置。

## 期待来源

读者期待先看到主角开干，产生“怎么这么猛”的惊讶，再通过解释获得“原来如此”的顺滑感。

## 断档风险

- 读者正期待装逼时，作者转去解释。
- 快节奏删掉人设底座。
- 怪物为了配合主角变蠢。
- 只发正文不发小纲，点评者难判断后续。

## 训练任务

把一段“解释主角为什么强”改成“先三招打赢敌人，再用 100 字解释强大原因”。
