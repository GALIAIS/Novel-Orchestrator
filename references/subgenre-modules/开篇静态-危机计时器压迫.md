# 开篇静态-危机计时器压迫

依据案例：`batch-120` 案例 596。

## 读者承诺

静态开篇可以有氛围和设定，但必须让读者知道主角正在被什么逼迫、必须在多久内做什么。危机计时器能把独角戏从说明状态拉回行动状态。

## 核心循环

```text
静态困境 -> 外部压迫靠近 -> 时间限制出现 -> 主角主动判断 -> 金手指/能力解围 -> 进入下一行动
```

## 开篇必备

- 第一页就有当前危险。
- 主角短期目标明确。
- 有一个倒计时或外部逼近信号。
- 设定只保留当前求生必需项。
- 第一章末必须推进到新场景或新选择。

## 期待来源

- 读者等主角如何逃出当前困境。
- 外部威胁让独角戏变成对抗。
- 时间限制制造翻页压力。
- 金手指首秀带来第一次收益。

## 断档风险

- 氛围好但剧情不展开。
- 伏笔太多，读者无耐心记忆。
- 主角长期自言自语，没有外部反应。
- 金手指只展示用法，没有爽点或收益。

## 训练任务

把一个静态开篇改成危机版：加一个外部敌人、一个倒计时、一个主角主动选择。删掉不服务这三项的设定说明。
