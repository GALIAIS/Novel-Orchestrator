# 同人题材-自带流量版权风险

依据案例：`batch-062` 案例 308。

## 读者承诺

这个模块用于同人题材选择。它承诺的是利用公共 IP 的熟悉感降低开篇门槛，用改命、补遗憾、抢资源、名场面介入制造爽点，同时提前承认版权、平台规则、热度过期和 OOC 风险。

## 核心循环

```text
公共认知 -> 熟悉感入场 -> 名场面介入 -> 改命/补遗憾 -> 原作角色反应 -> 新剧情分叉
```

## 开篇必备

- 原作 IP 必须有足够公共认知。
- 第一章尽快出现读者熟悉的人、地点、规则或名场面。
- 主角介入点要改变原作走向。
- 明确补哪类遗憾或抢哪类资源。
- 写清 OOC 边界和版权风险。

## 期待来源

- 读者自动带入原作世界。
- 熟人角色对新变化的反应。
- 原剧情被改写的爽感。
- 主角用信息差提前布局。
- 原作遗憾被补偿。

## 断档风险

- 开篇解释太多原作常识。
- 角色 OOC 破坏公共认知。
- 只蹭 IP，没有新事件。
- 选择过气或读者不熟的 IP，却按热门同人写。
- 忽视版权和平台规则，把同人当长期稳定资产。

## 训练任务

选一个 IP，写出公共认知、名场面、遗憾补偿、主角介入方式、OOC 红线、版权风险六项。第一章必须让至少三项进入事件。
