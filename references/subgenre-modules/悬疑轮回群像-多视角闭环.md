# 悬疑轮回群像-多视角闭环

依据案例：`batch-200` 案例 998。

## 读者承诺

悬疑轮回结构要用高危开局、智商在线的群像和多视角信息增量，建立“坑都能合理回收”的作者信任。

## 核心循环

```text
高危开局 -> 规则揭示 -> 多视角补全 -> 旧坑回收 -> 新一轮危机
```

## 开篇必备

- 第一章立刻给危机。
- 明确第一层规则或异常。
- 埋下可公平回收的坑。
- 角色不能靠降智推动剧情。

## 期待来源

读者期待每次视角切换都带来新信息，并让同一事件出现更深一层真相。

## 断档风险

- 多视角只是重复同一件事。
- 坑多但回收不公平。
- 角色降智削弱悬疑可信度。
- 节奏快但规则松散。

## 训练任务

设计同一事件三视角表：每个视角新增 1 条关键信息，第三视角必须推翻第一视角判断。
