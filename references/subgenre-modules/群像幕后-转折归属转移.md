# 群像幕后-转折归属转移

依据案例：`batch-165` 案例 825。

## 读者承诺

在群像或幕后流中，把局部矛盾的关键转折交给最适合的角色完成，让读者从单主角代入扩展到阵营和关系网代入。

## 核心循环

```text
多角色铺垫 -> 配角独立压力 -> 局部矛盾逼近 -> 配角完成转折 -> 阵营收益 -> 主线推进
```

## 开篇必备

- 先让读者认识配角的欲望和困境。
- 配角能力要提前展示。
- 转折要匹配角色，而不是临时抢戏。
- 明确主角是直接参与、间接布局，还是暂时让位。

## 期待来源

读者期待自己关注的角色、阵营或布局获得胜利；爽感可以来自团队推进，而不只来自主角亲自打脸。

## 断档风险

- 单主角承诺未转向，突然让配角解决核心矛盾。
- 配角没有铺垫，关键时刻工具化。
- 主角长期缺席，读者误以为换书。
- 多线转折不能流回主线。

## 训练任务

设计一个群像单元：列出 3 个配角各自能解决的局部矛盾，并说明每次转折如何让主线向前推进。
