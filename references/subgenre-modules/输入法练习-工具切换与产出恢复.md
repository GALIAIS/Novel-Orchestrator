# 输入法练习-工具切换与产出恢复

依据案例：batch-375 案例 1871

## 读者承诺

输入法练习要服务长期稳定产出。新工具降速期不能打断写作节奏，当前瓶颈不是输入时不要钻工具牛角尖。

## 核心循环

```text
识别瓶颈 -> 试练新工具 -> 记录降速 -> 设恢复线 -> 保住稳定产出
```

## 开篇必备

- 当前输入法。
- 切换理由。
- 练习期速度损失。
- 是否影响写作节奏。

## 期待来源

- 更低手指负担。
- 更顺输入节奏。
- 稳定恢复速度。
- 不再被工具打断。

## 断档风险

- 把构思慢误判成输入慢。
- 新工具不熟导致思路中断。
- 为理论速度牺牲更新。
- 没有止损线。

## 训练任务

记录 7 天构思、输入、修改和疲劳数据，只在输入成为主要瓶颈时切换工具。
