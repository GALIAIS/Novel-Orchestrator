# 附件投稿-正文缺失与证据边界

依据案例：`batch-239` 案例 1191。

## 读者承诺

附件不可读时不做伪拆解；只记录可见证据、缺失范围和后续补证方式。

## 核心循环

```text
作者投稿 -> 内容在附件 -> 附件不可读 -> 标注证据限制 -> 等可读正文补拆
```

## 开篇必备

- 记录正文可见内容。
- 记录附件名、ID、大小。
- 尝试本地定位和下载。
- 明确不可判断项。

## 期待来源

系统保持可信度：读过才拆，没读到就说没读到。

## 断档风险

- 根据文件名臆测题材。
- 没有正文也强行评价节奏。
- 忽略附件下载失败。
- 后续拿到正文后忘记补拆。

## 训练任务

为任一附件投稿建立证据表，列出可读/不可读内容；不可读正文不进入技法判断。
