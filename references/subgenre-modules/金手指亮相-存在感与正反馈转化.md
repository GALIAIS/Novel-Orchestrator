# 金手指亮相-存在感与正反馈转化

依据案例：batch-346 案例 1730

## 读者承诺

金手指首次亮相要让读者看见差异、潜力和后续期待；如果只展示存在感，必须足够新奇，否则应尽快转成解决事端或装逼反馈。

## 核心循环

```text
获得能力 -> 展示差异 -> 读者理解潜力 -> 改变局势/评价 -> 形成下一次应用期待
```

## 开篇必备

- 能力的新奇点要一眼可感。
- 首次展示要让主角和平常环境拉开差距。
- 老套能力必须绑定具体事端。
- 影视式视听冲击要转译成文字里的氛围、评价和结果。
- 展示后要留下下一步可期待的应用方向。

## 期待来源

读者期待“这个能力还能怎么用”，并期待主角凭它改变处境。

## 断档风险

- 只写设定说明，没有场景反馈。
- 能力老套却不解决问题。
- 作者觉得酷，读者无法感知差异。
- 借鉴影视镜头，却忘记小说需要阅读理解。
- 展示能力后旁人、局势、主角目标都没有变化。

## 训练任务

写 3 个金手指亮相场景：纯展示、解决小事、公开装逼。每个场景结尾都写出读者下一步期待。
