# 爽文扫榜-大事件望远镜与平台差异

依据案例：`batch-247` 案例 1234。

## 读者承诺

神豪、赘婿、兵王这类爽文需要事件串、大事件爽点和望远镜，不只是零碎小爽点。

## 核心循环

```text
小爽点连续 -> 望远镜指向大事件 -> 事件逐步升级 -> 阶段性装逼兑现
```

## 开篇必备

- 说明大的装逼目标。
- 每章安排小爽点。
- 让事件有衔接。
- 区分平台节奏。

## 期待来源

读者知道什么时候会大爆，才愿意吃前面的小爽点。

## 断档风险

- 只有小爽点，没有大事件。
- 没有望远镜。
- 平台样本不匹配目标节奏。
- 爽了但不上不下。

## 训练任务

设计 5 章事件串，第 1 章给出第 5 章大事件望远镜，每章都有小爽点。
