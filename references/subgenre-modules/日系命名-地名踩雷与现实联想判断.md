# 日系命名-地名踩雷与现实联想判断

依据案例：`batch-220` 案例 1098。

## 读者承诺

判断名称是否踩雷，看公众熟悉度、敏感程度和负面联想强度，而不是只看现实是否同名。

## 核心循环

```text
候选名称 -> 现实同名 -> 熟悉度判断 -> 敏感度判断 -> 保留/替换
```

## 开篇必备

- 重要地名先查现实同名。
- 避开高敏政治、历史、宗教符号。
- 判断联想是否会压过作品语境。
- 替换成本低时优先换名。

## 期待来源

作者保留日系氛围，同时避免读者被现实敏感联想拉出故事。

## 断档风险

- 现实有同名就过度焦虑。
- 忽略高敏符号。
- 名称联想强过作品设定。
- 没有低风险备选名。

## 训练任务

设计 10 个日系地名，按公众熟悉度、敏感程度、负面联想和替换成本评分。
