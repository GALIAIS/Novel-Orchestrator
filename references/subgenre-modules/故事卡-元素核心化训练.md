# 故事卡-元素核心化训练

依据案例：`batch-009` 案例 042。

## 读者承诺

给定元素不是摆设，而是把故事卡逼成具体桥段。读者能看到元素改变目标、危机、能力、规则或情绪。

## 核心循环

```text
限定元素 -> 故事卡 -> 元素核心化 -> 危机产生 -> 行动解决 -> 情绪变化
```

## 开篇必备

- 元素至少一个成为规则、倒计时、金手指或关键道具。
- 故事卡要有明确情绪链。
- 主角救人必须有动机。
- 被救者前后态度要变化。

## 期待来源

- 读者想看限定元素如何派上用场。
- 元素越不可替换，故事越像“长出来”的。
- 同一故事卡多题材复用，能快速积累桥段。
- 点评会暴露自己对套路的误解。

## 断档风险

- 元素只是背景，换成别的词也不影响故事。
- 救人过程一笔带过。
- 被救者没有反应和好感变化。
- 为凑元素强行机械降神。

## 训练任务

用“地点、时间、故事卡”三项组合写 5 个梗概。每个梗概后写一句：删掉哪个元素故事会崩，为什么。
