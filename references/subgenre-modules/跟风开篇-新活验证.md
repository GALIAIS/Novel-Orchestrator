# 跟风开篇-新活验证

依据案例：`batch-014` 案例 066。

## 读者承诺

跟风题材已经有读者和模板，开篇必须在已知承诺上给出可记忆的新切入点，让读者觉得熟悉但不重复。

## 核心循环

```text
热题材 -> 扫同类 -> 提取共同承诺 -> 设计新活 -> 第一章展示 -> 编辑/章评验证
```

## 开篇必备

- 明确跟风对象和热标签。
- 知道同类读者已经期待什么。
- 第一章就体现差异化切入点。
- 人设、事件、金手指至少一处有新活。

## 期待来源

- 读者知道这是熟悉题材。
- 读者看到开篇有不一样的处理。
- 读者期待这个新活如何持续兑现。
- 编辑能从前几章判断题材和差异点。

## 断档风险

- 只复制外壳，没有新切入点。
- 新活只是作者自嗨，不符合读者章评偏好。
- 开篇和标签不一致，算法/读者都识别困难。
- 差异点藏太晚，前面已经像劣质跟风。

## 训练任务

扫 3 本同类开篇，写出共同模板，再写一个第一章就能看见的新切入点，并用 3 条读者可能评论验证是否成立。
