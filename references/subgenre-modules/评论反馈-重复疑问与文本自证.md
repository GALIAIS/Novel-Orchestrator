# 评论反馈-重复疑问与文本自证

依据案例：`batch-230` 案例 1147。

## 读者承诺

两个以上读者问同一个问题，且作者需要长篇解释，通常说明正文没有把关键信息讲清楚。

## 核心循环

```text
重复评论 -> 归并问题 -> 定位正文缺口 -> 文内补一句/一段 -> 减少解释成本
```

## 开篇必备

- 收集重复问题。
- 不把读者当成理解力问题。
- 找到正文中最早该补的位置。
- 用文本自证替代评论区解释。

## 期待来源

读者能在正文里自然理解动机、规则、因果和主线，不需要作者额外说明。

## 断档风险

- 作者在评论区越解释越多，正文却不改。
- 把单个恶意评论当成普遍反馈。
- 没有曝光时误以为没人问就是没问题。
- 修改只补设定，不补读者真正卡住的因果。

## 训练任务

整理最近 20 条评论，列出重复两次以上的问题。每个问题只允许用正文内一句话或一个动作补救，禁止写作者解释。
