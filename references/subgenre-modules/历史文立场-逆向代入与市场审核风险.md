# 历史文立场-逆向代入与市场审核风险

依据案例：`batch-205` 案例 1024。

## 读者承诺

历史文高度依赖读者身份和文化立场代入，逆向攻占中原类脑洞容易同时触发市场反感和审核风险。

## 核心循环

```text
逆向历史脑洞 -> 代入断裂 -> 立场反感 -> 市场扑街/审核风险 -> 此路不通
```

## 开篇必备

- 先确认读者默认代入立场。
- 主角身份不能切断爽点入口。
- 历史创新要保留文化安全感。
- 高风险脑洞要替换表达。

## 期待来源

历史文读者期待站在认可的文化立场上改写遗憾、获得改命爽点。

## 断档风险

- 把逆向思维误当创新。
- 主角立场让主流读者反感。
- 政策和市场风险叠加。
- 失败样本消失，被误判为空白机会。

## 训练任务

评估一个历史脑洞：读者默认立场、主角立场、反感点、审核风险、替代方案。
