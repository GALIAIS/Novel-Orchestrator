# 人物关系网-工具选型

依据案例：`batch-170` 案例 850。

## 读者承诺

人物关系网工具的目标是维护长篇人物、阵营、关系变化和旧伏笔，最终让读者感到关系清楚、变化可信、前后不忘。

## 核心循环

```text
角色增加 -> 关系复杂 -> 工具可视化/双链化 -> 章节依据回写 -> 后续冲突调用
```

## 开篇必备

- 先区分静态关系图和长期人物档案。
- 每条关键关系要有来源章节和变化节点。
- 工具要方便修改，不要只追求图好看。
- 关系网要服务冲突、阵营和信息差。

## 期待来源

读者期待长篇人物关系前后清楚，旧关系能在新冲突中被重新调用，而不是写到后面互相遗忘。

## 断档风险

- 工具很复杂，但没有回写到剧情。
- 只有角色名连线，没有利益、秘密和变化。
- 静态图不更新，越写越失真。
- 角色关系缺少章节依据，复盘时无法追溯。

## 训练任务

为 20 个角色建立关系网，至少写 10 条关键关系的来源章节、当前状态、利益绑定、隐瞒信息和后续冲突用途。
