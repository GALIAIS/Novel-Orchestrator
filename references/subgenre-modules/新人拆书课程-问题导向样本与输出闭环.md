# 新人拆书课程-问题导向样本与输出闭环

依据案例：`batch-410` 案例 2050

## 读者承诺

先确定一个具体学习问题，再选锚点、当前和对照三本差异样本；用同一结构卡人工拆解，只迁移一项机制并经冷读验证，证据不足时才扩展书单。

## 核心循环

```text
定义学习问题 -> 选择三类差异样本 -> 审计作品阶段与阅读范围 -> 统一结构卡拆解 -> 比较共同机制和变量 -> 原创迁移练习 -> 冷读验证 -> 决定扩样或进入下一问题
```

## 开篇必备

- 本轮唯一学习问题。
- 锚点、当前和对照三类样本。
- 作品时间、作者阶段、平台口径和阅读范围。
- 统一的前三章与首弧结构卡。
- 一项共同机制和一项差异变量。
- 原创练习与陌生读者验证。

## 期待来源

庞大书单被收束为一个问题，三个样本给出可比较证据；共同机制在原创场景中首次复现，冷读结果让阅读从收藏变成已验证能力。

## 断档风险

- 每个类别先收藏几十本却没有问题。
- 用排行榜代替样本审计。
- 三本采用不同拆解标准无法比较。
- 只读开篇便评价全书。
- 一次迁移多个变量看不出效果。
- 没有原创输出和冷读反馈。

## 训练任务

选择一个学习问题和三本差异样本，逐本完成同一结构卡；只迁移一项共同机制写 1500 字原创场景，让三名未读样本者复述承诺、冲突和回报，仍不清楚时再增加第四本。
