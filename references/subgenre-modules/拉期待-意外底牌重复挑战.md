# 拉期待-意外底牌重复挑战

依据案例：`batch-322` 案例 1609

## 读者承诺

拉期待可以用意外、新变量、身份冲突定义、新目标、敌人底牌、直接疑问和重复挑战来触发。

## 核心循环

```text
稳定局面 -> 期待触发器 -> 读者产生疑问 -> 小高潮回收 -> 指向大高潮
```

## 开篇必备

- 先建立一个稳定预期。
- 触发器要改变当前局面。
- 新人物或物品必须对事件发生作用。
- 重复挑战要让读者记得第一次失败。
- 小高潮结束还要指向大高潮。

## 期待来源

读者期待看到新变量如何改变局面，主角如何应对旧失败的新版本。

## 断档风险

- 只抛疑问，没有事件作用。
- 新物品只露面，不改变局势。
- 重复挑战没有升级。
- 小高潮结束后没有更大目标。

## 训练任务

为一段剧情设计 5 个期待触发器，并写出每个触发器后读者会问的问题。
