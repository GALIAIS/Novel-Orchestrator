# 脑洞系统-基础风格取舍

依据案例：`batch-234` 案例 1169。

## 读者承诺

脑洞系统文先定基础风格；严肃和欢快可以混，但必须有主次，不能互相抢读者观感。

## 核心循环

```text
金手指气质 -> 选择主风格 -> 辅助风格服务 -> 统一第一章观感 -> 再谈节奏
```

## 开篇必备

- 金手指名称和文本气质一致。
- 严肃/欢快比例明确。
- 宏大场面服务主风格。
- 第一章给出稳定观感。

## 期待来源

读者看到搞怪系统，就知道自己要读轻松脑洞还是严肃玄幻，不会被割裂感打断。

## 断档风险

- 把风格取舍误认为润色问题。
- 金手指搞怪，叙事却庄严。
- 场面太大导致代入感弱。
- 第一章观感摇摆。

## 训练任务

给脑洞系统开头标注严肃/欢快比例。若欢快为主，把一个宏大场面改成吐槽、误导、错愕驱动的轻松版本。
