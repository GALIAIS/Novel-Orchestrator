# 主角视角-感官边界控制

依据案例：`batch-168` 案例 839。

## 读者承诺

通过限制信息来源，让叙述只呈现主角能看到、听到、感到和判断的内容，保持代入连续。

## 核心循环

```text
确定视角人物 -> 检查信息来源 -> 删除他人心理 -> 转成主角感官 -> 保持视角连续
```

## 开篇必备

- 明确当前视角人物。
- 每句话检查主角能否感知。
- 非主角心理不能直接写。
- 他人反应要写成主角看到的表情和动作。

## 期待来源

读者期待跟着主角经历冲突，而不是在上帝视角和多个角色心理之间来回跳。

## 断档风险

- 直接写反派心理。
- 用旁观者视角描述主角。
- A 说/B 说机械切换导致视角漂移。
- 动作戏里危险来源分散，代入感降低。

## 训练任务

改写 5 句上帝视角句子，把信息全部转成主角可见、可听、可感的版本。
