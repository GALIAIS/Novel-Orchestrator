# 成熟套路创新-反惯性细节

依据案例：`batch-049` 案例 244。

## 读者承诺

保留成熟套路的低理解成本，在细节、能力用法、战斗方式、装逼路径和结果上反惯性，让读者猜得到开头，猜不到结尾。

## 核心循环

```text
成熟框架 -> 默认预期 -> 细节新开发 -> 剧情证明 -> 惊喜结果
```

## 开篇必备

- 先让读者认出熟悉框架。
- 不完全打破构架，避免解释成本过高。
- 新意放在具体情节里证明。
- 反转结果必须是惊喜，不是狗血。
- 草稿后修时，可根据人物行为反推性格和铺垫。

## 期待来源

- 熟悉套路让读者轻松进入。
- 细节变化让读者产生新鲜感。
- 猜得到方向但猜不到落点，会增强追读。
- 经典框架被重新开发，会带来竞争力。

## 断档风险

- 完全照搬经典，会被前作压住。
- 完全打破框架，读者理解成本过高。
- 意料之外如果伤害读者情绪，会变成背刺。
- 新意只靠说明，不靠剧情证明，会站不住。

## 训练任务

选一个成熟桥段，写出读者默认预期，再设计 3 个不同的细节变体，要求每个变体都不增加额外设定解释。
