# 血腥尺度-审查风险与风格预警

依据案例：`batch-109` 案例 543。

## 读者承诺

血腥暴力尺度没有稳定精确的安全线。作者要在保留威胁感的同时，降低审核、举报和读者不适风险，用风格预警、荒诞化表现和敏感词规避保护作品。

## 核心循环

```text
重口需求 -> 判断风险 -> 改写表现 -> 提前预警 -> 控评避雷
```

## 开篇必备

- 明确场景必须提供的威胁感。
- 删除非必要血腥细节。
- 避开高风险词。
- 标签和简介提前筛选读者。
- 评论区及时处理恶意反馈。

## 期待来源

- 恐怖或战斗氛围仍然成立。
- 读者知道作品口味。
- 审核风险下降。
- 争议被控制在可处理范围。

## 断档风险

- 把尺度当成固定规则。
- 过度渲染分尸、虐杀、切面等细节。
- 重口内容没有前置预警。
- 和恶意评论纠缠，影响创作节奏。

## 训练任务

把一个血腥场面改成“结果化版本”：只写后果、反应和威胁，不直接写高风险生理细节。
