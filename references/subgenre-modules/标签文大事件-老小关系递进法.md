# 标签文大事件-老小关系递进法

依据案例：`batch-225` 案例 1125。

## 读者承诺

用同类题材的成熟大事件承载一卷主线，再用可信关系网完成冲突升级。

## 核心循环

```text
题材标签 -> 大事件库 -> 阶段主线 -> 小冲突 -> 老人物出头 -> 舞台升级
```

## 开篇必备

- 先列同类文常见大事件。
- 每个大事件有开始和结束边界。
- 关系网能自然引出更高层人物。
- 创新放在背景、金手指、人物表达和事件变体上。

## 期待来源

读者看到熟悉题材事件后，立刻知道当前阶段目标，并期待一轮轮晋级和更大舞台。

## 断档风险

- 从零硬造，导致卷主线发散。
- “小的来了老的”没有合理理由。
- 只有套路表面，没有题材承诺。
- 排斥套路，错过成熟剧情内核。

## 训练任务

为一个题材列 10 个常见大事件；任选一个写开始条件、结束条件、奖励/损失；设计三层冲突升级：小人物、背后老人物、老人物出头理由。
