# 正文先写-搭架子再修

依据案例：batch-302 案例 1508

## 读者承诺

准备材料太多但正文不好看时，先把故事架子写出来，再回头补细节、对照理论优化。

## 核心循环

```text
准备过多 -> 正文卡顿 -> 先写粗稿 -> 回头梳理 -> 修改补强
```

## 开篇必备

- 当前场景目标一句话说清。
- 暂停翻大设定。
- 先写完整事件链。
- 回修时只补目标、冲突、情绪。

## 期待来源

- 作者有可修改对象。
- 粗稿降低启动压力。
- 理论用于优化，而不是阻止动笔。

## 断档风险

- 大纲、导图、人设越做越大。
- 细节记不住，正文反而写不动。
- 第一遍不好就怀疑全部方法。

## 训练任务

用 800 字写完一个卡住场景，只要求事件完整。写完后再补三处情绪起伏。
