# 简介诊断-设定转期待感冲突链

依据案例：`batch-189` 案例 944。

## 读者承诺

简介不能只陈列设定，要把设定转成损失、敌人、目标和反击路径，读者才会产生期待。

## 核心循环

```text
主角原身份 -> 被剥夺 -> 获得金手指 -> 锁定敌人 -> 夺回/反杀期待
```

## 开篇必备

- 主角原本拥有什么。
- 现在失去了什么。
- 谁造成损失。
- 金手指如何改变局面。

## 期待来源

读者知道主角惨在哪里、敌人是谁、要夺回什么、爽点何时可能兑现。

## 断档风险

- 只有设定，没有冲突。
- 双金手指没有对抗关系。
- 矿工等低位身份没有剥夺感。
- 长线幕后太多，第一层冲突不清。

## 训练任务

拿一个设定简介，补齐“原身份、损失、反派、金手指、第一目标”五项，再压缩成 150 字简介。
