# 过场故事化-场景增值与套娃止损

依据案例：batch-389 案例 1942

## 读者承诺

把必要过场写成有目标、冲突、选择和结果的小故事，但每场必须增值并立即回绑父目标，以退出条件阻止无限套娃。

## 核心循环

```text
父目标遇门槛 -> 过场形成局部冲突 -> 选择带来增值结果 -> 局部闭环 -> 回绑父目标
```

## 开篇必备

- 无法轻易跳过的必要门槛。
- 局部角色自己的目标和阻力。
- 线索之外的第二项场景价值。
- 明确的嵌套层数与退出条件。

## 期待来源

- 无聊流程被转成可行动的近程问题。
- 小故事同时塑造人物或世界。
- 阶段结果让主线更近、更难或更重要。
- 局部闭环后迅速返回长期目标。

## 断档风险

- 作者强设门槛冒充必然性。
- 每个 NPC 都追加一条取钥匙任务。
- 临时伏笔只为证明过场有用。
- 支线规模膨胀到遮住父目标。

## 训练任务

把一个问路或取证过场写成局部故事，要求交付线索、人物选择和第二项增值；删除测试后压缩任何可一句替代的部分。
