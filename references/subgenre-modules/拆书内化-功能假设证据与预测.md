# 拆书内化-功能假设证据与预测

依据案例：batch-389 案例 1945

## 读者承诺

拆书能力形成于提出功能假设、引用证据、比较替代解释和验证预测的过程；表格只保存推理痕迹，不提供标准答案。

## 核心循环

```text
观察效果 -> 提出功能假设 -> 证据与替代解释竞争 -> 预测或改写验证 -> 迁移内化
```

## 开篇必备

- 明确到段落的功能假设。
- 文字与结构位置证据。
- 至少一个替代解释。
- 可由后文或改写检验的预测。

## 期待来源

- 模糊好恶变成可复核判断。
- 过度解读被证据和预测筛除。
- 独立拆解暴露自己的注意盲区。
- 通过验证的机制能迁移到原创文本。

## 断档风险

- 把作者意图与文本效果混为一谈。
- 直接抄别人表格产生会了的错觉。
- 没有证据却写唯一正确解释。
- 表格很完整但从不仿写验证。

## 训练任务

为三个连续段落各写功能假设、证据、替代解释、后文预测和置信度，读完后验证，再迁移一个成立机制到原创场景。
