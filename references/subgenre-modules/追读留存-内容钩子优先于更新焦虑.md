# 追读留存-内容钩子优先于更新焦虑

依据案例：batch-347 案例 1733

## 读者承诺

更新稳定很重要，但读者留下的根本理由是想看后面的内容。更新量要服务期待，而不是替代期待。

## 核心循环

```text
读者快速消费 -> 爽点未完全兑现 -> 章尾留下期待 -> 更新承接期待 -> 追读留存
```

## 开篇必备

- 每章有清楚爽点或问题推进。
- 章尾留下下一步期待。
- 更新量与内容质量一起看。
- 暴更用于承接强期待。
- 数据波动先检查故事吸引力，再看客观更新因素。

## 期待来源

读者不是因为“作者更得多”而留下，而是因为“我想知道下一步”。

## 断档风险

- 把收藏掉落全归因于渣更。
- 章尾只是停住，不是断章。
- 追求暴更但内容无聊。
- 单个读者评论被当成平台铁律。
- 正爽时没有后续承接。

## 训练任务

重写 5 个章节结尾，每个结尾都写清读者下一章期待：目标、危机、秘密、反转或爽点兑现。
