# 套路可预测-样本验证爽感优先

依据案例：`batch-115` 案例 573。

## 读者承诺

套路可预测不等于失败。网文读者常常知道大方向，仍会为过程爽感、节奏和熟悉承诺付费。新人要区分“节奏套路”和“卖点陈旧”。

## 核心循环

```text
使用成熟套路 -> 小样本质疑没新意 -> 区分节奏与卖点 -> 保留节奏 -> 微创新卖点 -> 用真实读者样本验证
```

## 开篇必备

- 快速出主角。
- 少抛设定。
- 点明金手指或核心亮点。
- 人物不要太多。
- 直接进入关键剧情。

## 期待来源

- 读者能猜到方向，形成期待。
- 过程够爽，愿意继续看。
- 微创新放在人设、金手指、切入点。
- 推荐后的真实样本比零散评价更可靠。

## 断档风险

- 把所有意见都当目标读者反馈。
- 为了不可预测破坏节奏。
- 只改套路壳，不增强爽点。
- 老白评价压过目标受众。

## 训练任务

拿一个熟悉套路开篇，只改一个微创新点：主角人设、金手指或切入事件。其他节奏不动，再检查爽点是否更清楚。
