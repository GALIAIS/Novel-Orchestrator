# 连载留存-追订下滑新爽点补位

依据案例：`batch-178` 案例 886。

## 读者承诺

付费连载必须持续给读者追更理由，旧爽点结束后要尽快抛出新爽点。

## 核心循环

```text
推荐曝光 -> 追订变化 -> 定位掉订章节 -> 检查拖点/毒点 -> 新爽点补位
```

## 开篇必备

- 上架后看追订，不只看均订。
- 掉订最快的章节优先复盘。
- 检查情节是否拖沓。
- 检查是否有毒点或爽点断档。

## 期待来源

读者期待每章都推进目标、冲突和奖励。拖剧情会先伤追订，再拖累均订。

## 断档风险

- 只看总均订，不查章节曲线。
- 旧冲突结束后迟迟没有新目标。
- 作者体感“还能铺”，读者已经不追。
- 毒点集中在几章却没有及时修正。

## 训练任务

复盘最近 20 章追订，找出掉订最大的 3 章，逐章写出拖点、毒点、新爽点缺口和补位方案。
