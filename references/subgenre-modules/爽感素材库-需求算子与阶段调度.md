# 爽感素材库-需求算子与阶段调度

依据案例：`batch-418` 案例 2089

## 读者承诺

爽点素材不会停留在打脸、获利、智斗、拯救和搞笑的平铺清单，而会按底层需求、事件操作、兑现证据、故事阶段和后续代价组合。每场只选择一个主需求和至多一个副需求，主角在压力下主动行动，结果由资源、危险、关系、权威或世界状态的真实变化证明，满足后的责任再生成下一轮剧情。

## 核心循环

```text
从类型承诺选择主需求和可选副需求 -> 用个人处境建立迫切性与失败损失 -> 选择获得、保护、击败、揭露、解谜、创造或改革等事件操作 -> 阻碍和成本迫使主角作选择 -> 主角用能力、关系或信息完成行动 -> 以资源、危险、关系、权威或世界状态变化提供兑现证据 -> 配角反应补充但不替代结果 -> 满足规模与当前故事阶段匹配 -> 代价和责任制造下一轮需求
```

需求算子提供情绪根，事件操作变成可写动作，兑现证据让结果可验证，阶段调度控制升级，代价余波连接长线。

## 开篇必备

- 类型承诺及开局应优先满足的需求。
- 生存、资源、能力、尊重、归属、公正、求知、创造或价值中的主需求。
- 至多一个副需求。
- 主角此刻在意的个人原因和失败损失。
- 明确的满足阈值。
- 获得、守住、夺回、保护、击败、揭露、说服、解谜、创造或改革操作。
- 主角需要承担的成本和主动选择。
- 状态变化形式及配角反应的验证功能。
- 当前处于开局、前期、中期、后期或终局。
- 满足后的代价、责任和下一需求。

## 期待来源

- 主角当前最缺的东西是什么。
- 不满足会造成怎样的个人损失。
- 主角将选择哪种高风险操作。
- 能力、关系或信息如何在行动中被验证。
- 状态变化是否达到预设满足阈值。
- 他人改观会确认哪项真实成果。
- 同一核心需求如何从个人扩大到组织和制度。
- 本次满足会制造什么新责任。

## 断档风险

- 把所有爽感都写成打脸。
- 将需求、动作、结果、反应和规模混成一张平铺列表。
- 一场同时满足过多需求，任何一项都没有重量。
- 只有人物自称满足，没有状态变化。
- 用围观者震惊替代行动结果。
- 开局直接追求世界级规模，缺少个人锚点。
- 只增加数值，不扩展关系和制度压力。
- 素材没有标注前置条件和故事阶段。
- 每次满足都没有代价或余波。
- 将单个评论中的平台偏好当作普遍规律。

## 训练任务

把一个只标注“打脸”的旧情节改写为能力证明、关系归属、公正复仇、资源控制和创造改造五版。每版标出主需求、失败损失、事件操作、兑现证据、故事阶段和余波；选择最符合当前类型承诺的一版扩写到 2000 字，禁止只靠旁观者反应完成结算。
