# 开篇冷启动-有效冲突与角色认知限流

依据案例：`batch-408` 案例 2037

## 读者承诺

第一章让陌生读者迅速辨认主角、当前欲望和迫近问题；用目标与现实的不一致制造有效冲突，并把需要同时记忆的活跃角色限制在可组织范围内。

## 核心循环

```text
主角欲望建立中心 -> 异常、阻碍或期限打破常态 -> 少量活跃角色以不同动作加压 -> 主角作出选择 -> 选择产生局部结果 -> 下一问题显形 -> 按因果需要扩展角色
```

## 开篇必备

- 明确的主要追随者。
- 一个正在变化的局面。
- 不依赖打斗的有效冲突。
- 主角、压力源、关键关系三个核心单位。
- 姓名与独特动作绑定。
- 章节末由主角选择产生的后果。

## 期待来源

读者先知道跟谁走，再看到他必须现在回应的偏差；少量角色分别改变信息、压力和选择，名字因此随因果被记住。

## 断档风险

- 把冲突等同争吵或战斗。
- 多人轮流报姓名却不改变局面。
- 镜头平均分配导致主角中心漂移。
- 同时加载姓名、关系、立场和世界设定。
- 复制旧作而不审计发布年份与作者声望。
- 用核心粉丝耐心替代匿名冷读测试。

## 训练任务

把一个六人以上开篇压到三个活跃单位，其余角色延迟姓名或合并功能；让匿名读者复述主角、问题与选择，任一项不清就继续限流。
