# 作者自拆-睡前复盘闭环

依据案例：`batch-030` 案例 150。

## 读者承诺

写完后用冷视角拆自己的书，把漏洞和小毒点记录成次日修改清单。

## 核心循环

```text
当日码字 -> 睡前回放 -> 发现问题 -> 记录清单 -> 次日定点修复
```

## 开篇必备

- 当天有明确产出。
- 睡前只记录，不情绪化大修。
- 每次最多抓 3 个关键问题。
- 第二天先修最影响追读的一项。

## 期待来源

- 作者从产出视角切到读者视角。
- 漏洞、小毒点、动机问题更容易浮出。
- 问题被外化成清单。
- 高频小修提升稳定性。

## 断档风险

- 睡前无限焦虑。
- 发现问题但不记录。
- 第二天全部推翻重写。
- 只挑文字，不看期待断点。

## 训练任务

连续 7 天睡前自拆，每天写下 3 个问题，第二天只优先修最影响读者继续看的 1 个。
