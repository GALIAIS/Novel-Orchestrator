# 计划意外-连续转机拆事件

依据案例：`batch-073` 案例 363。

## 读者承诺

这个模块用于开篇危机、破案、翻案和自救事件。它承诺的是每次计划被意外阻断后，都给出新转机，让主角越走越接近兑现。

## 核心循环

```text
总目标 -> 计划 -> 意外阻断 -> 转机补路 -> 新计划 -> 新意外 -> 新转机 -> 兑现
```

## 开篇必备

- 主角总目标明确。
- 每个计划都要具体。
- 意外必须阻断当前路径。
- 转机来自已有角色、信息或金手指。
- 结尾要有超额收获。

## 期待来源

- 阻断制造压力。
- 转机证明主角能动。
- 多轮计划形成推进感。
- 金手指在关键处发挥。
- 超额收获补偿读者。

## 断档风险

- 意外只让主角被动挨打。
- 转机凭空出现。
- 计划之间没有递进。
- 只完成目标，没有额外收获。
- 照搬高手结构导致过复杂。

## 训练任务

设计一个三轮自救事件。每轮写计划、意外、转机，最后写目标兑现和一个超额收获。
