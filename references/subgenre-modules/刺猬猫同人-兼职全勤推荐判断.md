# 刺猬猫同人-兼职全勤推荐判断

依据案例：batch-306 案例 1527

## 读者承诺

想从飞卢转刺猬猫写同人时，先评估题材、日更能力、推荐表现和收益结构。平台不是避难所，迁移要靠机制判断。

## 核心循环

```text
原平台吃力 -> 目标平台调研 -> 编辑确认机制 -> 前期推荐看收藏 -> 活动放大或及时止损
```

## 开篇必备

- 先明确自己是同人、原创还是轻小说向。
- 估算稳定日更能力。
- 查清签约、全勤、推荐、活动规则。
- 把前三轮推荐收藏作为早期验证点。

## 期待来源

期待来自题材适配和推荐验证。签约不难不等于收益稳定，真正的判断点是推荐期是否获得足够收藏和后续均订基础。

## 断档风险

- 只看月入过万人数，忽略大多数作者生态。
- 把全勤当主要收入来源。
- 不上推荐仍期待自然流量。
- 飞卢写不动就直接换平台，没有检查题材适配。

## 训练任务

写一张平台迁移表：原平台痛点、目标平台读者、日更要求、推荐节点、收藏阈值、活动策略、撤退条件。
