# 叙事诊断-概念过大先贴文本

依据案例：batch-353 案例 1763

## 读者承诺

叙事问题必须落到具体文本才能诊断。抽象问“怎么增强叙事”只能得到通用资料；贴出样稿后，才能判断是事件顺序、视角、信息量、动作连续性还是人物目标出了问题。

## 核心循环

```text
抽象不适感 -> 提供样稿 -> 拆叙事字段 -> 定位单点故障 -> 小范围修改验证
```

## 开篇必备

- 样稿段落。
- 本段想达成的效果。
- 主角当前目标。
- 读者可能卡住的位置。

## 期待来源

- 作者从模糊焦虑转成具体修改。
- 每次只改一个叙事字段。
- 修改后阅读阻力明显下降。
- 诊断标准逐渐内化。

## 断档风险

- 没有文本，只谈概念。
- 同时改文笔、设定、人设，无法判断效果。
- 把所有问题都归为叙事。
- 不说明段落目的，导致诊断失焦。

## 训练任务

贴出 500 字样稿，先只标注事件顺序和视角位置，不改句子。确认问题后再重写一版。
