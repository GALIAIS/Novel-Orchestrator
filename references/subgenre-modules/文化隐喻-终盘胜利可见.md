# 文化隐喻-终盘胜利可见

依据案例：`batch-056` 案例 278。

## 读者承诺

这个模块用于处理主题、隐喻和爽点冲突。它承诺的是：可以保留象征层表达，但终盘必须让目标读者看见主角赢在哪里；读者看不懂的文化隐喻不能替代情绪兑现。

## 核心循环

```text
主题承诺 -> 终盘对抗 -> 主角可见行动 -> 反派可见失败 -> 群众/关系反馈 -> 象征层补强主题
```

## 开篇必备

- 明确目标读者理解哪套文化符号。
- 主题表达要能落到主角行动。
- 反派计划成功条件必须提前展示。
- 主角成长后要产生可见效果。
- 象征层胜利不能覆盖行动层失败。

## 期待来源

- 读者看到主角在终盘主动改变局势。
- 反派至少有一部分计划被主角打断。
- 群众、同伴或敌人承认主角带来的变化。
- 主题通过行动被证明，而不是靠台词解释。
- 隐喻让爽点更厚，不替代爽点。

## 断档风险

- 作者觉得主角象征性赢了，读者只看到主角一直输。
- 文化背景门槛太高，目标读者无法接收。
- 反派成功像卡 bug，没有充分铺垫。
- 终盘只剩救灾和善后，没有胜利证据。
- 主题台词和剧情结果互相打架。

## 训练任务

拿一个主角“精神胜利”的结尾，补三条可见胜利：主角阻止了什么、救下了什么、让反派失去了什么。保留隐喻，但不能让隐喻承担主要爽点。
