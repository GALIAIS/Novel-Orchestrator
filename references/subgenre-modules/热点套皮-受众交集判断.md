# 热点套皮-受众交集判断

依据案例：`batch-042` 案例 209。

## 读者承诺

跟热点前先判断热度属于谁：IP、平台、游戏、同人圈、题材圈可能完全不同。只有目标读者和热点受众有交集，套皮才有意义。

## 核心循环

```text
外部热点 -> 受众拆分 -> 交集判断 -> 套皮选择 -> 原创承诺
```

## 开篇必备

- 明确热点真实受众。
- 明确小说目标读者。
- 确认可借的是皮还是核心。
- 第一章兑现原创读者承诺。

## 期待来源

- 热点元素降低识别门槛。
- 同人或套皮满足已有兴趣。
- 原创承诺提供持续阅读理由。
- 读者知道这不是纯蹭热度。

## 断档风险

- 把游戏热度误判成题材热度。
- 把二次元受众误判成科幻受众。
- 只有皮，没有网文核心爽点。
- 原创不如直接写同人。

## 训练任务

选一个热门 IP，拆出真实受众、可借元素、小说目标读者和最大错位风险。
