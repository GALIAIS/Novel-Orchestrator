# 瓶颈恢复-扫片翻书触发

依据案例：`batch-076` 案例 380。

## 读者承诺

这个模块用于题材反复推翻、设定凑不完整、缺乏动笔灵感的阶段。它承诺停止在空白里硬凑，改用大量扫片、翻书和重读教科书恢复触发。

## 核心循环

```text
设定卡死 -> 外部输入 -> 摘创意桥段 -> 教科书触发问题 -> 回填当前开篇 -> 最小动笔
```

## 开篇必备

- 写清当前题材为什么被否定。
- 列出缺失的设定块。
- 输入目标只看创意和桥段。
- 教科书复读要带问题。
- 每次输入后必须回填一个最小改动。

## 期待来源

- 外部桥段打破原地空想。
- 教科书重新激活结构问题。
- 创意和功能模块重新组合。
- 小改动降低重新动笔门槛。
- 连续输入形成素材池。

## 断档风险

- 继续否定题材，不补输入。
- 看作品只评价好坏，不摘桥段。
- 收集素材后不回填当前项目。
- 迷信完整设定，迟迟不写最小片段。
- 用风向判断压死个人兴趣。

## 训练任务

三天内扫 5 个电影或小说开头。每个只摘一个创意、一个桥段、一个可迁移冲突；最后把其中一个冲突塞回当前项目，写 500 字试运行。
