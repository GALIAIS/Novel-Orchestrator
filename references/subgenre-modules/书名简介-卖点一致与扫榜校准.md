# 书名简介-卖点一致与扫榜校准

依据案例：`batch-257` 案例 1285。

## 读者承诺

书名简介模块让标题、简介第一屏、金手指和正文开篇同向，避免包装承诺和实际内容错位。

## 核心循环

```text
扫榜样本 -> 提炼同题材包装逻辑 -> 书名承诺卖点 -> 简介第一屏确认 -> 开篇兑现
```

## 开篇必备

- 至少扫 30 个同题材样本。
- 书名要明确核心题材和卖点。
- 简介第一屏前置金手指和主冲突。
- 删除会误导读者的副卖点。

## 期待来源

读者从书名判断题材，从简介确认卖点，点进正文后立刻看到同一承诺。

## 断档风险

- 书名像争霸，简介像种田。
- 金手指出现太晚。
- 卖点混杂，读者无法判断。
- 特殊设定和普通设定差异没突出。

## 训练任务

找 30 本同题材榜单书，记录书名、简介第一句、金手指位置、主冲突承诺，再写 3 组自己的书名简介，保留与开篇同向的一组。
