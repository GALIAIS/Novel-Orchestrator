# 写作工具-稳定同步防丢稿

依据案例：`batch-180` 案例 899。

## 读者承诺

写作软件首先要稳定、同步、防丢稿，其次才是拼字、大纲、起名等辅助功能。

## 核心循环

```text
写作场景 -> 选择主工具 -> 自动保存/同步 -> 备份确认 -> 辅助工具补充
```

## 开篇必备

- 主写作工具必须可靠。
- 多端写作用云同步。
- 小众软件可辅助，不承担唯一稿件存储。
- 每天结束确认备份。

## 期待来源

作者期待工具减少损耗，而不是制造丢稿风险。顺手和安全比功能堆叠更重要。

## 断档风险

- 为花哨功能放弃稳定。
- 小众软件闪退丢字。
- 没有云端或本地备份。
- 花太多时间试工具，不写正文。

## 训练任务

固定一个主写作工具和一个备份方案，连续 7 天写作，每天确认至少一份可恢复备份。
