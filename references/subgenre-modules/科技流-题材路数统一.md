# 科技流-题材路数统一

依据案例：`batch-024` 案例 119。

## 读者承诺

科技流开篇要让读者看到主角用技术解决具体问题，其他设定都服务第一次技术验证。

## 核心循环

```text
具体威胁 -> 技术能力 -> 装置制造 -> 实测解决 -> 观众反馈 -> 下一项技术目标
```

## 开篇必备

- 主类型是科技流。
- 第一威胁清楚。
- 主角技术能力有证据。
- 第一项装置能当章验证。

## 期待来源

- 技术方案压过蛮力。
- 直播或观众能看懂技术成果。
- 怪兽、灾变或外星威胁给科技升级压力。
- 每次验证带出下一项更强技术。

## 断档风险

- 巨兽、守护者、蜘蛛侠、粉丝值、灵气复苏同时抢戏。
- 无关噱头与核心爽点无关。
- 事件突然发生，缺铺垫。
- 主角实力和阴谋线一笔带过。

## 训练任务

删改一个科幻开篇：只保留第一威胁、第一装置、第一次验证和一次观众反馈。删掉暂时无关的世界观名词。
