# 长篇升级流-同路不同级后期重复控制

依据案例：`batch-178` 案例 889。

## 读者承诺

长篇升级流允许核心循环重复，但每一轮要通过等级、敌人、场景、奖励和代价升级，降低疲劳。

## 核心循环

```text
固定基调 -> 核心循环重复 -> 等级抬升 -> 敌人/副本变化 -> 奖励升级 -> 继续追读
```

## 开篇必备

- 明确不可丢的核心基调。
- 每轮事件保持读者熟悉的爽点。
- 用等级和规模制造差异。
- 不为新花样轻易跳出承诺。

## 期待来源

读者期待熟悉循环中的升级变化。完全跳出基调可能比重复更伤读者。

## 断档风险

- 同级同路，只有复读。
- 为避免重复乱换类型，读者流失。
- 敌人变强但奖励不变。
- 规模升级了，情绪没有升级。

## 训练任务

列出 5 个相似大事件，比较敌人等级、场景规模、奖励、主角代价和情绪差异，判断是否做到同路不同级。
