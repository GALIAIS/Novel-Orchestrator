# 综漫开篇-先入第一个世界

依据案例：`batch-323` 案例 1612

## 读者承诺

综漫同人系统流开篇要先让读者进入第一个世界的危机、名场面和主角行动，而不是先讲完原世界背景与系统说明。

## 核心循环

```text
异常场景醒来 -> 短倒叙长期目标 -> 融合本世界身份 -> 面对原作危机 -> 用系统/主角能力改写局面
```

## 开篇必备

- 第一个世界的具体场景。
- 主角当前处境和即时危机。
- 原世界目标只保留能解释动机的部分。
- 面向大众时补足看懂危机所需的信息。
- 面向粉丝时尽快触碰名场面或热点问题。
- 系统能力通过行动展示，少用说明书式介绍。

## 期待来源

读者期待主角以不同方式经历原作磨难，并组合多个主角金手指改写既有剧情。

## 断档风险

- 第一章讲述性文字过多。
- 穿越前经历喧宾夺主。
- 没看过原作的读者无法代入。
- 老粉看不到熟悉热点。
- 系统规则只被解释，没有推动事件。

## 训练任务

选一个动漫世界，设计“醒来场景、当前危机、原作信息、粉丝名场面、系统第一次生效”五项，每项不超过 80 字。
