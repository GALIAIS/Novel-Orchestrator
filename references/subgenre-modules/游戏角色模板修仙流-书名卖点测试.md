# 游戏角色模板修仙流-书名卖点测试

依据案例：`batch-195` 案例 971。

## 读者承诺

游戏角色模板修仙流的书名要让读者第一眼看出模板来源、变强方式和开局爽点，而不是只堆题材标签。

## 核心循环

```text
标题压缩卖点 -> 读者第一眼联想 -> 检查亮点缺口 -> 调整玩法词 -> 提升点击意愿
```

## 开篇必备

- 标题包含主角模板或能力特征。
- 标题给出可想象的升级动作。
- 开局场景兑现标题承诺。
- 不依赖读者知道过多原作单位名。

## 期待来源

读者看到标题就能想象主角如何用游戏机制在修仙世界破局，并期待第一章马上看到这种玩法。

## 断档风险

- 标题堆“系统、修仙、穿越、游戏”。
- 开局动作太弱，无法体现爽点。
- 原创世界和原作单位名冲突。
- 作者需要解释标题，读者才懂卖点。

## 训练任务

写 5 个标题，逐个标注“模板、变强方式、开局动作”。删掉需要额外解释才能懂的标题。
