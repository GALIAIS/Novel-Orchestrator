# 软科幻发书-平台样本贴合

依据案例：batch-283 案例 1413

## 读者承诺

软科幻新人发书要用平台同类样本做决策，而不是凭平台传闻下注。哪个平台的科幻类更贴近自己的节奏、设定密度和读者口味，就优先选择哪个。

## 核心循环

```text
扫描平台样本 -> 对比设定密度 -> 调整开篇节奏 -> 十万字验证 -> 复盘继续或止损
```

## 开篇必备

- 同类样本清单。
- 目标平台读者偏好的冲突速度。
- 设定投放上限。
- 主角目标和危机先于世界观百科。
- 十万字前的验证计划。

## 期待来源

读者想在可理解的危机和行动中逐步接收科幻设定，而不是先被宏大世界观挡在门外。

## 断档风险

- 新人一开篇堆战锤、群星、光环式大设定。
- 平台风格不匹配。
- 重写开头后不调整后文逻辑。
- 把平台传闻当成发书决策依据。

## 训练任务

各拆 3 本起点和番茄软科幻样本的前三章，记录冲突速度、设定投放、主角目标和评论关键词，再决定自己的开篇版本。
