# 拆书偏食-主角视角校准

依据案例：`batch-057` 案例 284。

## 读者承诺

这个模块处理拆书后把单一技巧过度放大的问题。它承诺的是把学到的技巧重新放回主角视角、代入感和基础叙事里，避免拆书反过来破坏写作。

## 核心循环

```text
拆到技巧 -> 回写正文 -> 检查主角视角 -> 检查技巧占比 -> 补主角心理/行动/收益 -> 配角反应辅助兑现
```

## 开篇必备

- 明确本章主角在想什么、要什么、做什么。
- 配角震惊只能辅助主角成长，不能替代主角成长。
- 主角的心理变化和行动选择必须先出现。
- 技巧占比过高时，先回到代入感检查。
- 槽点和毒点分开判断，先处理破坏承诺的问题。

## 期待来源

- 读者跟着主角体验成长和收益。
- 配角反应放大爽点，而不是抢走叙事中心。
- 主角行动导致别人态度变化。
- 技巧服务基础叙事，爽点更稳。
- 拆书成果能直接改善正文，而不是制造偏科。

## 断档风险

- 拆某一技巧太细，写作时只盯这一项。
- 配角视角过多，主角代入裂开。
- 只写别人眼中的主角厉害，不写主角自身变化。
- 学高手局部技巧，忘了高手的基础盘。
- 把所有不合理都叫毒点，忽略真正伤害代入的问题。

## 训练任务

选一段配角震惊爽点，标出每一句属于主角心理、主角行动、配角反应、作者说明。配角反应超过主角心理和行动总量时，重写到主角视角占优。
