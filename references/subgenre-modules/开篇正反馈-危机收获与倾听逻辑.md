# 开篇正反馈-危机收获与倾听逻辑

依据案例：batch-378 案例 1890

## 读者承诺

网文开篇要让危机和主角有关，并在处理危机后给明确收获。读者想看处理方式和收益，不想先吞设定。

## 核心循环

```text
主角目标 -> 相关危机 -> 处理危机 -> 明确收获 -> 下一章期待
```

## 开篇必备

- 主角目标。
- 和主角有关的危机。
- 处理危机的动作。
- 危机后的收获。

## 期待来源

- 读者知道为什么继续看。
- 金手指或能力当场起效。
- 正向反馈带来粘性。
- 下一章有更大问题。

## 断档风险

- 信息量过载。
- 第一章没有期待感。
- 危机和主角无关。
- 只有表达逻辑，没有倾听逻辑。

## 训练任务

把一章高信息量开头删掉 70% 设定说明，保留目标、危机、金手指使用、收获和章末期待。
