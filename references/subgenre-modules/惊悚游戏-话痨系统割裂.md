# 惊悚游戏-话痨系统割裂

依据案例：batch-309 案例 1542

## 读者承诺

惊悚游戏开篇要先给规则、危险和压迫感。系统可以存在，但语气必须服务惊悚，而不是话痨抢戏。

## 核心循环

```text
主角进入惊悚游戏 -> 系统提示规则 -> 危机压迫升级 -> 主角行动 -> 副本信息逐步清晰
```

## 开篇必备

- 系统语气贴合题材。
- 副本等级和规则要清楚。
- 主角危机要明确。
- 吐槽和骂系统不能破坏主角讨喜度。

## 期待来源

期待来自危险和规则。系统越冷，副本越清晰，读者越能进入惊悚游戏。

## 断档风险

- 话痨系统破坏恐怖氛围。
- 待机空间气氛不对。
- S 级副本情况讲不清。
- 主角一直骂无恶意系统，显得烦。

## 训练任务

写同一个系统登场两版：轻小说吐槽版、惊悚冷感版。判断哪版更符合你的平台和题材。
