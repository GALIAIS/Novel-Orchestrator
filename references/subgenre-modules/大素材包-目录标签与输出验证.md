# 大素材包-目录标签与输出验证

依据案例：batch-378 案例 1889

## 读者承诺

大素材包只有被目录化、标签化、按问题检索并转成输出，才不是硬盘负担。

## 核心循环

```text
保存素材 -> 建目录标签 -> 带问题检索 -> 抽取可用点 -> 写入正文
```

## 开篇必备

- 素材来源。
- 一级分类。
- 高频标签。
- 当前写作问题。

## 期待来源

- 资料储备更厚。
- 写作卡点能快速检索。
- 场景和设定更具体。
- 输出验证资料价值。

## 断档风险

- 收藏即满足。
- 链接失效不备份。
- 没有目录难以使用。
- 素材不绑定写作问题。

## 训练任务

给一个大素材包建立 20 个标签，用其中 3 个标签各写一个 300 字场景。
