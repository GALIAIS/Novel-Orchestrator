# 平台写作-外部编辑与草稿防丢

依据案例：`batch-208` 案例 1036。

## 读者承诺

创作者写长文时要把平台输入框当发布窗口，而不是写作容器；先外部保存，再复制发布。

## 核心循环

```text
长文内容 -> 外部编辑 -> 自动/手动保存 -> 复制发布 -> 保留备份
```

## 开篇必备

- 明确平台输入框可能没有可靠草稿。
- 写清外部编辑器和保存路径。
- 规定手动保存频率。
- 发布前保留本地版本。

## 期待来源

创作者相信投入的时间不会因为误点退出、网页崩溃或入口不明而归零。

## 断档风险

- 直接在网页输入长文。
- 不确认草稿箱入口。
- 没有自动保存或本地备份。
- 丢稿后只靠情绪恢复，不改流程。

## 训练任务

写一份“防丢稿 SOP”：覆盖章节正文、大纲设定、拆书感想三类文本，每类写清编辑器、保存频率、发布步骤和恢复路径。
