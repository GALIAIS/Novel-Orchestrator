# 创作状态-兴奋感需延迟复检

依据案例：`batch-331` 案例 1654

## 读者承诺

写作兴奋感是好信号，但不是质量验收；好章节要经得起冷却后复看。

## 核心循环

```text
写时兴奋 -> 完成并留档 -> 冷却72小时 -> 复看情绪是否仍在 -> 补铺垫或删自嗨
```

## 开篇必备

- 记录当场兴奋点。
- 不当场过度修改。
- 延迟复看。
- 区分文本情绪和脑内补完。
- 只改传达不到位的部分。

## 期待来源

作者把真实热爱转成读者可感的情绪，而不是只停留在自我感动。

## 断档风险

- 写时兴奋误判为必然好文。
- 过几天发现情绪没写出来。
- 当场乱改毁掉初稿能量。
- 只为赚钱丢掉情绪燃料。
- 只为理想不做读者转译。

## 训练任务

封存一个兴奋片段 72 小时，复看标出仍有情绪的句子和只靠脑补的句子，只改后者。
