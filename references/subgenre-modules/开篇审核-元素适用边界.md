# 开篇审核-元素适用边界

依据案例：`batch-019` 案例 092。

## 读者承诺

让新人用开篇清单降低失误，但不机械迷信清单。重点是主角聚焦、金手指可见、冲突成立、期待能拉住读者。

## 核心循环

```text
书名简介承诺 -> 主角聚焦 -> 冲突触发 -> 金手指显形 -> 期待/爽点验证
```

## 开篇必备

- 读者从书名简介得到的期待。
- 主角第一时间进入视野。
- 冲突影响主角。
- 金手指或核心优势能被读者感知。

## 期待来源

- 主角将如何解决当前冲突。
- 金手指会带来什么收益。
- 书名简介提前承诺的慢开局问题会如何兑现。
- 无敌文不靠危机，靠冲突和反应。

## 断档风险

- 把清单当法律，硬塞所有元素。
- 慢开局没有书名简介预期托底。
- 冲突只落在配角身上。
- 思想品质不能产生可见收益，却硬当金手指。

## 训练任务

检查自己的第一单元，只问三件事：主角是否聚焦、金手指是否可见、冲突是否直接影响主角。三项任一不成立，重写事件入口。
