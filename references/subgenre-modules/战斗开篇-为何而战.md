# 战斗开篇-为何而战

依据案例：`batch-057` 案例 282。

## 读者承诺

这个模块适用于低武、玄幻、武侠、动作向开篇。它承诺的不是更多打斗，而是让每场战斗都有目标、紧迫性、收益和后续钩子。

## 核心循环

```text
战斗理由 -> 失败代价 -> 战前期待 -> 战中压制 -> 主角破局 -> 收益兑现 -> 超额回报接下一段
```

## 开篇必备

- 战斗前先让读者知道主角为什么必须打。
- 明确失败代价，不能只有“打起来了”。
- 战斗收益要提前可见。
- 队友不屑、敌人压制、怪物乱入都要服务主线。
- 战斗结束必须有收获、反应和新期待。

## 期待来源

- 读者想看主角获得尊敬、资源、底牌曝光或更高目标。
- 战斗前的望远镜让胜利值得期待。
- 战斗中误判和压制制造情绪拉扯。
- 战后超额收益钩住下一情节点。
- 主角用能力解决别人解决不了的问题。

## 断档风险

- 把打戏当卖点本身。
- 连续多章打斗，却没有收益升级。
- 危机看似严重，实际主角底牌太多，读者不信。
- 敌人和队友只是装逼工具，没有明确功能。
- 打完只赢了战斗，没有接上新期待。

## 训练任务

拆一场打斗，删掉所有漂亮动作，只保留“为什么打、输掉会怎样、赢了得什么、谁会改变看法、下一步钩子”。如果删完只剩动作，说明这场战斗不能放开篇。
