# 起承转合-剧情卡顿排障

依据案例：`batch-041` 案例 203。

## 读者承诺

起承转合不是固定格式，而是卡文时的排障工具。哪里缺行动、阻碍、转折或兑现，就补哪里。

## 核心循环

```text
起事件 -> 承目标 -> 转压力 -> 合结果 -> 新钩子
```

## 开篇必备

- 起要有事件。
- 承要有目标和行动。
- 转要有阻碍和代价。
- 合要有反馈和新期待。

## 期待来源

- 作者知道卡在哪。
- 剧情有推进方向。
- 读者等结果。
- 单元结束能接下一段。

## 断档风险

- 只套公式不写行动。
- 起承转合都写成设定说明。
- 合只解决问题没有新期待。
- 导图不结合具体项目。

## 训练任务

拿一个卡住的桥段填四栏：每栏必须写一个行动和一个读者期待。
