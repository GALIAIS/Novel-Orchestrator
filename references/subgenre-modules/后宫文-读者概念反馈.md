# 后宫文-读者概念反馈

依据案例：`batch-198` 案例 988。

## 读者承诺

后宫文读者提出的新概念，要转成卖点、雷点、关系排序或类型期待来判断。

## 核心循环

```text
读者原话 -> 类型作者讨论 -> 概念归类 -> 写法调整
```

## 开篇必备

- 保留读者原话。
- 明确所属类型问题。
- 找懂类型的人确认。
- 把概念落到关系结构。

## 期待来源

作者能用读者语言反推后宫文的真实阅读承诺，而不是凭自己想象处理关系线。

## 断档风险

- 忽略垂直读者语言。
- 把新概念当普通吐槽。
- 没有类型作者校准。
- 无展开时强行脑补。

## 训练任务

收集 5 条后宫文读者评论，分别归类为卖点、雷点、党争风险、角色排序或关系期待。
