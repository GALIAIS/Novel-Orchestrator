# 开篇诊断-金手指前置信号

依据案例：`batch-152` 案例 760。

## 读者承诺

开篇清单做到不等于好看。读者需要尽快看到金手指信号、主角优势、冲突和情绪拉扯，否则文字流畅也会显得慢。

## 核心循环

```text
开篇异常 -> 主角目标 -> 金手指信号 -> 冲突拉扯 -> 章尾钩子 -> 正式觉醒
```

## 开篇必备

- 第一屏异常。
- 主角短期目标。
- 金手指第一信号。
- 一场明确冲突。
- 情绪拉扯或爽点。
- 压缩世界观说明。

## 期待来源

读者期待快速确认“这本书好看在哪里”。如果二号金手指第三章才出现，前两章必须用强冲突或明确信号撑住。

## 断档风险

- 金手指太薄弱或太晚。
- 第二章只做功能桥段。
- 世界观说明脱离事件。
- 慢热但没有情绪拉扯。
- 读者看完觉得流畅但不爽。

## 训练任务

把两章慢热开头压成一章：保留异常、目标、冲突、金手指信号、章尾钩子五项，删掉只解释不推进的桥段。
