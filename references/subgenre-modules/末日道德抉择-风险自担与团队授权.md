# 末日道德抉择-风险自担与团队授权

依据案例：`batch-403` 案例 2011

## 读者承诺

末日主角可以坚持仁慈或不杀，但必须先控制威胁、提出可执行替代方案，并让实际承担风险的队友进入决策；坚持少数意见者亲自承担额外成本。

## 核心循环

```text
敌人造成致命威胁 -> 团队夺回控制 -> 核验规则和替代方案 -> 明确决策权与成本承担 -> 提出带条件的仁慈 -> 团队授权或分裂 -> 敌人行为接受验证 -> 长期后果检验价值
```

## 开篇必备

- 敌人的行为、能力和再犯概率有证据。
- 杀或不杀所依据的规则经过核验。
- 先解除武装、隔离并控制信息。
- 直接受害者与风险承担者拥有发言权。
- 不杀方案包含资源、看守和违约后果。
- 主角的道德底线在此前选择中已有铺垫。

## 期待来源

读者面对没有干净答案的生存难题，看主角能否既守住价值又不把代价转嫁给同伴，并等待仁慈最终被合作或背叛验证。

## 断档风险

- 用“圣母”或“冷血”替代方案分析。
- 可怜身世自动洗掉当前威胁。
- 主角单方面替队友承担致命风险。
- 不杀却没有拘束、交换和监控。
- 临时出现无代价第三条路。
- 犹豫反复循环而始终不决断。
- 后续不检验仁慈产生的真实后果。

## 训练任务

为高危俘虏设计处决、拘束交换和主角自担风险三案，逐项写决策权、成本与后果；再写一场限时团队会议，所有价值立场都必须落到可执行方案。
