# 开篇主线-不可拒绝理由先行

依据案例：`batch-329` 案例 1642

## 读者承诺

开篇每个关键事件都要让读者明白主角为什么非做不可，否则危机、冲突和伏笔都会显得僵硬。

## 核心循环

```text
硬目标 -> 不行动有代价 -> 主角被迫选择 -> 冲突升级 -> 主线持续
```

## 开篇必备

- 主角当前目标清晰。
- 第一章危机不能拒绝。
- 第二章冲突有明确因果。
- 第三章关系推进有必要性。
- 前中后期目标属于同一动力线。

## 期待来源

读者期待主角在压力中选择，而不是被作者随意推到下一个情节。

## 断档风险

- 主角可以拒绝却偏要接受。
- 冲突爆发理由不足。
- 伏笔脱离主目标。
- 前期救急、中期变强、后期终极目标彼此不连。
- 参考爆款只抄事件，不抄压力结构。

## 训练任务

给前三章所有选择补一句“主角不做会怎样”；写不出代价的事件直接删改。
