# 穿越开篇-设定后置与喜好优先

依据案例：`batch-342` 案例 1708

## 读者承诺

穿越文开篇先拉当前危机和期待，设定、原主身份和世界说明用到什么说什么。

## 核心循环

```text
穿越身份 -> 当前危机 -> 必要信息三句话 -> 期待启动 -> 设定后续穿插
```

## 开篇必备

- 作者自己写得顺手。
- 第一观感优先。
- 原主身份只给当前剧情必需部分。
- 世界设定穿插在事件中。
- 详细前史后续展开。

## 期待来源

读者期待新身份、新危机和新优势马上产生事件，而不是先看世界说明书。

## 断档风险

- 穿越后强迫症式堆设定。
- 为证明真实感打断期待。
- 详细设定全展示。
- 土著或穿越选择只靠空想。
- 技巧研究过头导致开篇别扭。

## 训练任务

把 1000 字穿越说明压成 3 句话，只保留当前危机、原主身份和主角优势，其余全部标注后置。
