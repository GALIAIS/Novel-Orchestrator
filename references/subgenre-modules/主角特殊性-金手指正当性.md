# 主角特殊性-金手指正当性

依据案例：`batch-161` 案例 801。

## 读者承诺

让读者相信主角为什么非他不可，且这个独一无二的理由从开篇到结局保持稳定。

## 核心循环

```text
确立特殊性 -> 展示不可替代 -> 持续兑现 -> 升级为延展 -> 结局回收正当性
```

## 开篇必备

- 前三章展示主角特殊性。
- 说明没有这个特殊性，主角走不到当前关键节点。
- 金手指、性格、血统或身份只能选一个主根基。
- 后续补设定不能推翻读者最初接受的理由。

## 期待来源

读者期待主角靠同一个核心特质不断破局，而不是中途突然换成另一种天命来源。

## 断档风险

- 中途改变主角特殊性的根基。
- 出现多个同款特殊者，稀释主角独一无二。
- 后期升级否定前期努力。
- 金手指只是强，不承担主角正当性。

## 训练任务

写一张主角正当性卡：主角独一无二来源、前三章展示方式、中期强化方式、结局回收方式。删掉所有会推翻根基的后期补丁。
