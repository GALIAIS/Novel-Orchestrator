# 低俗段子-屎尿屁梗风险边界

依据案例：`batch-236` 案例 1180。

## 读者承诺

屎尿屁梗默认不写；只有和主角人设、作品调性高度匹配时，才少量使用。

## 核心循环

```text
低俗笑点 -> 检查人设/调性 -> 可替代则替代 -> 必要时少量保留
```

## 开篇必备

- 判断是否必要。
- 判断是否符合主角人设。
- 判断是否破坏文本气质。
- 优先找更干净的笑点。

## 期待来源

读者看到幽默但不出戏，不会因为低俗梗质疑作者品味和作品调性。

## 断档风险

- 新人用高风险梗试探读者。
- 梗不服务人设和剧情。
- 出现频率过高。
- 脑洞/惊悚文误以为什么低俗都能包容。

## 训练任务

列出稿中所有低俗梗，逐条检查：服务人设、推动剧情、有无干净替代。任一为否就删改。
