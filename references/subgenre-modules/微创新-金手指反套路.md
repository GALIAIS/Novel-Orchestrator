# 微创新-金手指反套路

依据案例：`batch-004` 案例 020。

## 读者承诺

在熟悉套路上做小幅反转，制造新鲜感，同时不破坏读者价值观、题材逻辑和长篇延展能力。

## 核心循环

```text
识别旧套路 -> 选择反转点 -> 保留核心满足 -> 生成新矛盾 -> 检查写长能力
```

## 开篇必备

- 读者熟悉的原套路。
- 一个明确反转点。
- 反转后的收益和矛盾。
- 3 个以上可连续展开的单元。

## 期待来源

- 读者识别出旧套路。
- 读者发现主角用不同方式打破常识。
- 反套路不是噱头，而是持续产剧情。

## 断档风险

- 为反而反，违背读者价值观。
- 只有一个脑洞，不能写长。
- 和普通扮猪吃虎混淆，没有独特机制。
- 反转后失去原套路核心爽点。

## 训练任务

拿一个系统流、签到流、听劝流或不听劝流，写出原套路、反转点和 3 个连续单元。写不出第三个单元就先不立项。
