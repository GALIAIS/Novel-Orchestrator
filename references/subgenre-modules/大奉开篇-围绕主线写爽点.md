# 大奉开篇-围绕主线写爽点

依据案例：`batch-101` 案例 505。

## 读者承诺

前十章的爽点必须围绕主线组织。名作拆书不要泛拆全书，而要限定范围，看每个爽点是否推进同一个主线问题，并用章节功能反查理论。

## 核心循环

```text
限定前十章 -> 抓主线问题 -> 每章安排爽点 -> 爽点推进主线 -> 第十章回收或升级
```

## 开篇必备

- 明确前十章主线。
- 每章有一个主线动作。
- 爽点必须回答或推进主线问题。
- 第五章和第十章要有阶段性回收。
- 理论只作解释，不替代文本判断。

## 期待来源

- 主线问题清晰。
- 每章爽点不离题。
- 阶段回收让读者确认收益。
- 新问题接上后续。

## 断档风险

- 每章都有爽点，但互不服务。
- 为了技巧而加离题桥段。
- 拆书范围过大，抓不住开篇功能。
- 理论符合，主线却没有推进。

## 训练任务

列出自己前十章的主线问题，再给每章写一句“本章爽点如何推进主线”。无法回答的章节，合并、删掉或改成主线事件。
