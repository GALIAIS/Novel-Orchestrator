# 飞卢金手指爽文-短主线续航

依据案例：`batch-199` 案例 991。

## 读者承诺

飞卢金手指爽文的主线要服务高频爽点循环，不能抢走前期读者买账的花活口味。

## 核心循环

```text
金手指花活 -> 数据上涨 -> 阶段目标线 -> 爽点变体续航 -> 避免主线撕裂
```

## 开篇必备

- 明确前期核心卖点。
- 主线只做简单目标线。
- 每阶段有金手指变体。
- 数据变化后检查口味是否断裂。

## 期待来源

读者持续看到金手指玩出新花样，而不是突然被带进另一种宏大主线。

## 断档风险

- 主线替代原卖点。
- 大纲过度宏大。
- 爽点密度下降。
- 万字更新下复杂主线拖垮执行。

## 训练任务

设计 20 万字飞卢续航表：4 个阶段目标、12 个金手指花活、4 个阶段反派。
