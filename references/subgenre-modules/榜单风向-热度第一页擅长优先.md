# 榜单风向-热度第一页擅长优先

依据案例：`batch-094` 案例 467。

## 读者承诺

这个模块解决“看榜单选题”的降噪问题。榜单不是命令，而是候选池；热度、作品数、热门标签分别代表不同信息，最终只能选择自己能拆、能写、能持续更新的方向。

## 核心循环

```text
收集榜单 -> 分清排序口径 -> 排除宽泛标签 -> 排除新人高危标签 -> 识别头部拉权重 -> 选擅长方向 -> 拆样本验证
```

## 开篇必备

- 选题前先确认榜单来源和排序方式。
- 不把作品数多当成近期机会。
- 不把群像、赛博等高热高难标签直接当新人入口。
- 只从热度第一页里挑能写开篇、能拆样本的标签。

## 期待来源

- 平台近期读者点击和收藏偏好。
- 标签对应的成熟爽点和读者熟悉感。
- 热度第一页提供的候选方向。
- 同标签样本书的首章和数据反馈。

## 断档风险

- 误把宽泛标签当具体题材。
- 被白金或头部作品拉高的权重误导。
- 选择自己不擅长的高热标签，开篇就写不动。
- 只看榜单不拆正文，无法知道读者到底为何追读。

## 训练任务

找一张平台标签榜，按“热度第一页”列出 12 个标签。逐个标注：宽泛/具体、难度、是否头部拉权重、我是否擅长、可拆样本。最后只保留 3 个可写方向，并各写一句读者承诺。
