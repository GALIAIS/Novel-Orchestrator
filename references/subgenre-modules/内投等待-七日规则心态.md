# 内投等待-七日规则心态

依据案例：batch-278 案例 1386

## 读者承诺

把新人内投后的焦虑等待变成流程管理：按 3 天到 7 个工作日观察，超期礼貌确认，等待期间继续改稿和备投。

## 核心循环

```text
投稿发出 -> 记录日期/编辑 -> 等待窗口 -> 超期确认 -> 改稿/备投不中断
```

## 开篇必备

- 投稿日期和目标编辑。
- 平台或经验回复周期。
- 第 7 个工作日后的确认动作。
- 等待期改稿清单。

## 期待来源

作者不再把沉默当即时失败，而是把不可控审稿变成有边界的等待。安全感来自流程，不来自反复刷新。

## 断档风险

- 一两天没回复就自判失败。
- 等待期停止写作。
- 月底、编辑差异、漏看等变量被误读成作品必死。
- 催问太频繁，破坏职业感。

## 训练任务

做一张内投等待表：日期、编辑、平台周期、每日改稿任务、第 7 个工作日确认话术、备选投稿渠道。
