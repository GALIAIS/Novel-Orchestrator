# 投稿反馈-拒稿自诊路径

依据案例：`batch-059` 案例 293。

## 读者承诺

这个模块用于投稿被拒但没有具体意见的场景。它承诺的是降低对编辑详评的依赖，用有限反馈、特定评判渠道和自诊清单推进修改。

## 核心循环

```text
投稿 -> 笼统拒稿 -> 降低编辑详评预期 -> 找反馈渠道 -> 自查书名简介/开篇/金手指/期待 -> 定点修改
```

## 开篇必备

- 接受“未达签约标准”通常没有细节。
- 不把编辑当免费诊断师。
- 找有评判传统的渠道时带完整材料。
- 自查范围先限于书名、简介、前三章。
- 每个修改假设都找同题材样本验证。

## 期待来源

- 作者不再卡在“编辑为什么拒我”。
- 自诊清单给出下一步动作。
- 有反馈渠道提供外部校验。
- 同题材样本补足证据。
- 修改变成可执行项目。

## 断档风险

- 等编辑主动讲问题。
- 到处投稿只收集拒稿语。
- 没有具体问题就大改全书。
- 把简单评判误当深度诊断。
- 不看同题材样本。

## 训练任务

用六项自诊表检查被拒稿：书名、简介、第一章、前三章、金手指、期待链。每项写一个问题假设和一个对标样本。
