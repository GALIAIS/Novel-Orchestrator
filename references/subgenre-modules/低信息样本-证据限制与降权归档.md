# 低信息样本-证据限制与降权归档

依据案例：batch-375 案例 1872

## 读者承诺

低信息社交帖要如实降权归档。人工精拆的可信度，来自不把寒暄玩笑强行提炼成创作理论。

## 核心循环

```text
人工查看 -> 判断信息密度 -> 标注证据限制 -> 保留轻量价值 -> 降权归档
```

## 开篇必备

- 样本类型。
- 正文信息量。
- 评论信息量。
- 是否有可拆技法。

## 期待来源

- 知识库更干净。
- 检索时知道样本权重。
- 避免伪理论污染。
- 保留社群生态信息。

## 断档风险

- 为了凑格式硬编方法。
- 把玩笑当原则。
- 不标证据限制。
- 低价值样本占据高权重位置。

## 训练任务

找 5 条低信息帖，分别写明“可保留价值”和“不可强行提炼内容”。
