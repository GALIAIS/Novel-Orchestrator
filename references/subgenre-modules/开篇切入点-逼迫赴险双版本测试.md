# 开篇切入点-逼迫赴险双版本测试

依据案例：`batch-179` 案例 891。

## 读者承诺

被迫赴险型开头要让读者立刻理解主角的压力、代价和破局问题。

## 核心循环

```text
威胁逼迫 -> 死亡代价 -> 主角选择 -> 高危行动 -> 破局期待
```

## 开篇必备

- 明确不去会怎样。
- 明确去了有什么危险。
- 逼迫者要形成压迫。
- 行动开场和威胁开场都可写，最终看成稿效果。

## 期待来源

读者期待主角在两难中活下来或反制。切入点服务压力建立，不是越晚越好。

## 断档风险

- 威胁开场写成解释，拖慢节奏。
- 行动开场靠闪回补太多前因。
- 主角代价不清，读者不关心。
- 不写成稿，只在脑内纠结。

## 训练任务

同一设定写两个 800 字开头：威胁逼迫版和执行任务版。比较哪版更快给出代价、目标和追读问题。
