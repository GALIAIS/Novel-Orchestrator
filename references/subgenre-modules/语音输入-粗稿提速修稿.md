# 语音输入-粗稿提速修稿

依据案例：batch-276 案例 1377

## 读者承诺

用语音输入解决手速跟不上脑子的粗稿问题，再通过修稿把口述内容变成可读网文。

## 核心循环

```text
口述粗稿 -> 转文字 -> 删口语 -> 补动作描写 -> 正文成稿
```

## 开篇必备

- 先用语音写小纲或对话。
- 保留二次修稿时间。
- 用有效成稿字数判断效率。

## 期待来源

作者期待输入阻力下降，思路先落地。读者端只看最终章节是否顺，不看原始口述字数。

## 断档风险

- 口语化严重，不修直接发。
- 原始字数高，但可用成稿少。
- 环境噪音和识别错误拖慢效率。

## 训练任务

语音口述 800 字，修成 500 字正文，记录总耗时和可用成稿字数。
