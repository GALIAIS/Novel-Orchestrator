# 双主角互衬-稚嫩误判老手识破

依据案例：batch-268 案例 1340

## 读者承诺

用一方的善意稚嫩衬托另一方的经验、警惕和明智，同时制造关系张力和危险验证。

## 核心循环

```text
伪装求助 -> 稚嫩角色想救 -> 老手发现异常 -> 阻止行动 -> 危险验证 -> 老手被衬托
```

## 开篇必备

- 伪装危险要有迷惑性。
- 稚嫩角色误判要来自善意或经验不足。
- 老手要指出可观察异常。
- 危险真相必须随后验证。

## 期待来源

期待来自判断差异：读者先理解善意，再看到经验的价值，从而认可老手角色。

## 断档风险

- 稚嫩角色被写成纯蠢。
- 老手只靠作者钦定正确。
- 危险不验证，显得老手冷血。
- 反复使用会让稚嫩角色变拖后腿毒点。

## 训练任务

写一个末世或江湖求助陷阱：年轻角色要救，老手发现三处异常并阻止，最后用事实验证危险。
