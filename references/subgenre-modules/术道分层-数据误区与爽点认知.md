# 术道分层-数据误区与爽点认知

依据案例：`batch-220` 案例 1100。

## 读者承诺

套路保底入门，认知决定上限；吸量数据不等于订阅付费。

## 核心循环

```text
套路学习 -> 实践写作 -> 爽点/不爽点认知 -> 持续付费 -> 数据复盘
```

## 开篇必备

- 区分吸量点和订阅点。
- 记录爽点和不爽风险。
- 用套路保底，但不把套路当上限。
- 每写一段都复盘付费理由。

## 期待来源

作者从数据焦虑转向持续制造爽感和控制整书节奏。

## 断档风险

- 把收藏点击当订阅能力。
- 只背套路，不形成认知。
- 只谈道，不练术。
- 忽略“不爽”导致断追。

## 训练任务

拆 3 本同题材书的吸量点和付费点，写一份不爽清单，并每 1 万字复盘一次续订理由。
