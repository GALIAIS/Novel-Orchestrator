# 投稿首屏-书名简介筛选

依据案例：`batch-075` 案例 372。

## 读者承诺

这个模块用于投稿前的书名、简介和首章承诺校准。它承诺让编辑或读者在点开正文前，就能看见题材、爽点和差异。

## 核心循环

```text
书名识别题材 -> 简介压缩爽点 -> 首章兑现承诺 -> 编辑/读者继续看
```

## 开篇必备

- 书名能看出题材方向。
- 书名或简介能看出核心爽点。
- 至少有一个差异化信号。
- 首章不能背离包装承诺。
- 投稿前先假设对方不会耐心翻正文。

## 期待来源

- 包装降低识别成本。
- 书名先筛出目标读者。
- 简介提供验证欲。
- 首章承接包装，形成信任。
- 热门渠道里，首屏强度决定是否获得正文机会。

## 断档风险

- 书名文艺但看不出题材。
- 简介只讲设定，不讲冲突和爽点。
- 正文优点藏得太深。
- 包装承诺和前三章不一致。
- 只靠“死磕”不修首屏。

## 训练任务

给当前项目写 10 个书名和 3 版简介。逐个标注题材、爽点、差异，只保留能在 5 秒内被识别的版本。
