# 投稿流程-内投等待撤回纪律

依据案例：batch-266 案例 1329

## 读者承诺

让新人投稿时用流程管理焦虑：不一稿多投，等待、询问、拒稿、撤回、转投都留清楚记录。

## 核心循环

```text
投递 -> 等待 -> 礼貌询问 -> 明确状态 -> 撤回/拒稿 -> 转投下一家
```

## 开篇必备

- 投递前记录编辑、平台、邮箱或 QQ。
- 投递后记录日期和预计等待期。
- 超过等待期先询问，不直接乱投。
- 转投前必须明确拒稿或撤回。

## 期待来源

期待来自作者对投稿流程有控制感，减少因为几天没回复就反复猜测、乱撤、乱投造成的损耗。

## 断档风险

- 一稿多投破坏基本投稿纪律。
- 没记录投递时间，靠情绪判断是否凉了。
- 周末或编辑节奏差异被误判为拒稿。
- 撤回后又后悔，浪费机会。

## 训练任务

建一张投稿台账，字段只保留：作品、编辑、投递时间、询问时间、当前状态、下一步。每次操作前先更新台账。
