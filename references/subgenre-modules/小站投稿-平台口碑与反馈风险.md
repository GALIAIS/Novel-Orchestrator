# 小站投稿-平台口碑与反馈风险

依据案例：`batch-079` 案例 391。

## 读者承诺

这个模块用于判断是否把新书投给小站。它承诺让作者先看平台生态，而不是被个别名家入驻、单本点击或编辑态度带着走。

## 核心循环

```text
发现平台 -> 查外部口碑 -> 核对主打题材 -> 判断保底/反馈 -> 决定投入强度
```

## 开篇必备

- 明确平台主打题材。
- 查作者社区真实口碑。
- 看代表作收藏、点击、评论和更新。
- 问清保底和分成条件。
- 判断自己是否能承受低反馈写作。

## 期待来源

- 平台题材和作品方向匹配。
- 编辑能提供有效反馈。
- 收入底线可预期。
- 读者反馈能支撑长期连载。
- 作者知道最坏情况是什么。

## 断档风险

- 只看大神在写就跟投。
- 点击高但收藏和评论弱。
- 题材不匹配平台读者。
- 没保底还期待商业回报。
- 反馈太少导致写作动力断裂。

## 训练任务

拿一个小站做投稿评估，写出“题材匹配、保底、反馈密度、外部口碑、最坏情况”五项结论。缺一项就暂缓投稿。
