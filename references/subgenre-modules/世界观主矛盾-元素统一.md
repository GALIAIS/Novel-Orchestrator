# 世界观主矛盾-元素统一

依据案例：`batch-069` 案例 343。

## 读者承诺

这个模块用于解决世界观模糊和元素冲突。它承诺的是先找出每个元素背后的主要矛盾，再用组织关系把它们统一成一个读者能理解的故事世界。

## 核心循环

```text
列元素 -> 找各自矛盾主体 -> 判断冲突或同向 -> 设定组织关系 -> 主角进入主矛盾
```

## 开篇必备

- 明确世界主要敌人或主要矛盾。
- 宗门、鬼怪、妖兽等元素要有组织关系。
- 环境描写不能替代主矛盾说明。
- 主角第一章要进入这个矛盾。
- 不用只换名词解决问题。

## 期待来源

- 读者知道故事主要看人斗人还是人斗怪。
- 宗门存在有明确原因。
- 怪物影响社会秩序。
- 主角目标与世界矛盾相连。
- 元素统一后，背景不再像拼贴。

## 断档风险

- 鬼怪、妖兽、宗门各讲各的。
- 只靠环境描写堆氛围。
- 编辑反馈词不问清就乱改。
- 把鬼怪改名妖兽但功能不变。
- 宗门不是解决主矛盾的组织。

## 训练任务

写一个“宗门为什么存在”的设定句。必须说明敌人是谁、宗门靠什么对抗、主角第一章为什么被卷入。
