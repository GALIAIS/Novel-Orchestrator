# 挑剔读者-弃读理由自审

依据案例：`batch-034` 案例 169。

## 读者承诺

每章都要经得起挑剔读者审问：他没耐心、会忘伏笔、只认书名承诺、讨厌水文偏题，发现足够理由就会换书。

## 核心循环

```text
假设挑剔读者 -> 列弃读理由 -> 找最致命三项 -> 本章堵破绽 -> 留住读者
```

## 开篇必备

- 书名承诺和正文同向。
- 本章有希望、爽点或惊喜。
- 解释不压过事件。
- 配角和支线有功能。

## 期待来源

- 读者每章都知道为什么继续。
- 伏笔有提醒，回收不突兀。
- 包装、金手指和开篇几百字有诱惑。
- 套路服务情绪，而不是机械套表。

## 断档风险

- 只检查套路，不检查读者体验。
- 伏笔拖太久不提醒。
- 支线不回主线。
- 精彩剧情还没来，读者已经走了。

## 训练任务

拿一章正文列 20 个弃读理由，选出最致命 3 个，在同章补出希望、爽点或悬念。
