# 穿越开篇-原主伤害即时危机化

依据案例：batch-306 案例 1529

## 读者承诺

穿越文可以让原主先遭受不公平伤害，但主角醒来后必须立刻接上危机，让旧账变成当前主角自己的生存、尊严或利益问题。

## 核心循环

```text
原主受害 -> 主角穿越接盘 -> 危机继续压迫 -> 不行动就完蛋 -> 金手指/能力反击
```

## 开篇必备

- 原主旧伤害要有明确压迫者。
- 主角穿越时危机仍在现场或马上追来。
- 不行动的后果要具体。
- 第一反击要回收一部分憋屈。

## 期待来源

期待来自“这笔账现在也压在主角身上”。读者不需要主角替陌生人义愤，而是看主角为了自己活下去、站起来、讨回来。

## 断档风险

- 只写原主惨，主角没有即时压力。
- 反派不继续出手，报复动机悬空。
- 穿越和金手指来历解释太长，冲淡开篇。
- 不公平伤害没有回收，变成单纯添堵。

## 训练任务

把一个“原主被欺负死”的设定改成即时危机：写出穿越时刻、当前压迫、不行动后果、第一反击和后续报复线。
