# 人物状态-转人设

依据案例：`batch-158` 案例 789。

## 读者承诺

“穷”这类状态本身不是稳定人设，只有转化为欲望、行为、选择和他人评价，才成为可用人物标签。

## 核心循环

```text
当前状态 -> 稳定欲望 -> 反复行为 -> 关键选择 -> 他人评价 -> 人设成立
```

## 开篇必备

- 人物当前状态。
- 状态造成的欲望。
- 稳定行为。
- 说话习惯。
- 他人评价。
- 关键选择。

## 期待来源

读者期待从人物状态预测人物反应，而不是只记住一个静态标签。

## 断档风险

- 把穷、弱、病、丑等状态当完整人设。
- 状态消失后人物也失去辨识度。
- 没有行为证明。
- 人设不能影响选择。

## 训练任务

把“穷、病、丑、弱、孤独”各写成一个人设，每个补稳定欲望、稳定行为和关键选择。
