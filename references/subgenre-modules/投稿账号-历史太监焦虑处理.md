# 投稿账号-历史太监焦虑处理

依据案例：batch-276 案例 1379

## 读者承诺

把历史太监、下架、旧账号焦虑快速处理掉，注意力回到当前文本质量。签约核心看作品水平。

## 核心循环

```text
账号焦虑 -> 判断真实影响 -> 马甲/原号二选一 -> 优化文本 -> 投稿
```

## 开篇必备

- 不把旧失败当作不投稿理由。
- 当前开篇和卖点优先。
- 不放心就新开马甲。

## 期待来源

作者期待来自行动确定性。伪问题越快处理，越能把精力放回作品。

## 断档风险

- 用账号历史解释当前文本问题。
- 一直担心规则，拖延投稿。
- 换马甲后文本仍不过关。

## 训练任务

列出投稿前焦虑点，分为可控和不可控。不可控项只保留一个处理动作，可控项变成改稿清单。
