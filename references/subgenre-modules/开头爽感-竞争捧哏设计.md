# 开头爽感-竞争捧哏设计

依据案例：`batch-163` 案例 815。

## 读者承诺

通过竞争者、阻碍、围观和打脸认证，增强主角能力展示的爽感。

## 核心循环

```text
主角被看低 -> 竞争者抢先 -> 竞争者失败 -> 主角展示能力 -> 围观震惊/拜服
```

## 开篇必备

- 主角能力要先被误解或看低。
- 竞争者要带来实际阻碍。
- 竞争者失败要证明问题难度。
- 主角解决后要有见证者认证。

## 期待来源

读者期待主角能力不只是成功，而是在被压低、被质疑后完成反向证明。

## 断档风险

- 情节太顺，没有阻碍。
- 能力展示没人见证。
- 捧哏只惊叹，不承担竞争功能。
- 前期压抑不足，自言自语破坏人设。

## 训练任务

给一个能力展示桥段补上竞争性捧哏五步：嘲笑、抢先、失败、被救、拜服。
