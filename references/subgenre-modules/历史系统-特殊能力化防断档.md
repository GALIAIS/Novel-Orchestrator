# 历史系统-特殊能力化防断档

依据案例：`batch-068` 案例 336。

## 读者承诺

这个模块用于历史、三国、争霸文里想加入系统但又不想写任务系统的项目。它承诺的是把系统改造成持续有效的特殊能力，避免前期系统爽文、后期普通争霸的基调断裂。

## 核心循环

```text
历史目标 -> 特殊能力识别/掠夺 -> 小胜成长 -> 能力分层解锁 -> 争霸推进 -> 后期仍有能力价值
```

## 开篇必备

- 先明确主承诺是历史争霸还是系统爽文。
- 金手指不能只报数，要推动行动。
- 能力分层推出，不一次讲完。
- 后期仍要保留作用。
- 避免前期靠系统、后期忽然平推种田。

## 期待来源

- 三国名将和数值带来识别乐趣。
- 掠夺属性带来成长爽点。
- 分层解锁维持长期期待。
- 历史争霸提供主线目标。
- 特殊能力比低存在感系统更少承诺错位。

## 断档风险

- 标题承诺系统，正文系统只是工具。
- 系统读者等任务奖励却等不到。
- 历史读者嫌系统破坏代入。
- 主角无敌后能力失效。
- 前后十万字基调完全不同。

## 训练任务

写一个“三国特殊能力”金手指，设计 5 次升级。每次升级都必须改变主角争霸策略，而不是只增加数值。
