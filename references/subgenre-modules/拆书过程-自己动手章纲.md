# 拆书过程-自己动手章纲

依据案例：`batch-066` 案例 326。

## 读者承诺

这个模块用于新人拆高手开篇。它承诺的是让学习收益发生在“自己拆”的过程中：先自己写章纲和疑点，再请别人校正，而不是等别人给标准答案。

## 核心循环

```text
选样本 -> 自己拆章纲 -> 标出疑点 -> 请求校正 -> 修正方法 -> 仿写训练
```

## 开篇必备

- 先明确拆书问题。
- 只选可承受范围，如前三章或前十章。
- 每章自己写 4 句以内章纲。
- 写出不确定点。
- 问题只问校正，不让别人代拆。

## 期待来源

- 自己动手暴露理解漏洞。
- 反馈能针对真实产物。
- 焦虑变成可执行步骤。
- 方法假设能被验证。
- 拆书成果可直接转仿写。

## 断档风险

- 伸手要别人拆。
- 看答案以为自己会了。
- 自创方法不验证适用范围。
- 方向错还越拆越用力。
- 只收藏教程，不产出章纲。

## 训练任务

自己拆三章，每章 4 句以内。每章后写一个“我不确定为什么这样写”的问题，再拿这些问题去求反馈。
