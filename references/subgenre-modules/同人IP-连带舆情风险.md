# 同人IP-连带舆情风险

依据案例：`batch-161` 案例 803。

## 读者承诺

在写同人前评估原作热度收益和连带风险，避免作品因原作舆情、立场争议或平台审核被动下架。

## 核心循环

```text
借原作热度 -> 获得同人流量 -> 原作争议发酵 -> 平台风险上升 -> 同人内容被牵连
```

## 开篇必备

- 判断同人是否强绑定原作名词。
- 评估原作历史争议和当前平台环境。
- 准备原创化替代方案。
- 不把“现在没人举报”当成长期安全。

## 期待来源

作者期待借热度获得启动流量，但也要接受绑定原作后产生的审核和舆情成本。

## 断档风险

- 热度高但争议也高。
- 去掉原作名词后故事无法成立。
- 原作联动撤退带动同人清理。
- 完结多年旧作被翻旧账。

## 训练任务

选一个同人 IP，写 3 项流量收益、3 项连带风险和 1 个原创化壳。若原创化后核心爽点消失，标为高风险项目。
