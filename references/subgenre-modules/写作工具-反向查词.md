# 写作工具-反向查词

依据案例：`batch-163` 案例 814。

## 读者承诺

用反向词典从含义、描述或大类反查词语，缓解词穷、术语不准和同义词贫乏。

## 核心循环

```text
表达卡住 -> 输入含义 -> 获得候选词 -> 筛选词性/相关性 -> 人工判断语境 -> 写入正文
```

## 开篇必备

- 先写清想表达的意思。
- 查到候选词后按语境筛选。
- 检查人物是否会这么说。
- 只替换真正更准确的词。

## 期待来源

作者期待通过工具找到更准确、更贴合语境的表达，而不是堆砌生僻词。

## 断档风险

- 机械替换同义词。
- 候选词不符合人物口吻。
- 为了华丽损害可读性。
- 工具结果未经人工判断。

## 训练任务

找 10 个词穷位置，每处查 5 个候选词，只替换 3 个真正更准确的位置。
