# 开局提速-倒叙风险改写

依据案例：`batch-128` 案例 638。

## 读者承诺

网文开局优先承诺正在发生的危机、主角行动、金手指和爽点。倒叙会把读者从当下冲突拉回背景说明，留存未稳时尤其伤代入。

## 核心循环

```text
当下危机开场 -> 主角行动 -> 金手指介入 -> 行动中补信息 -> 爽点成立 -> 断章留期待
```

## 开篇必备

- 少用开头倒叙。
- 先给当下问题。
- 主角必须尽快行动。
- 金手指尽快展示。
- 背景信息拆进行动。

## 期待来源

- 读者先被危机抓住。
- 主角行动带出代入。
- 金手指提供翻盘期待。
- 爽点比背景说明更早成立。

## 断档风险

- 前三章都在解释过去。
- 读者未代入就被要求理解设定。
- 为提速堆事件但没有主角行动。
- 断章不在爽点前一秒。

## 训练任务

把一个倒叙开头改成两版：一版先交代主角和金手指再进危机；一版直接从危机现场开写。两版都在 800 字内完成危机、行动、金手指、爽点闭环。
