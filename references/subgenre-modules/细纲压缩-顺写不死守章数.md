# 细纲压缩-顺写不死守章数

依据案例：`batch-117` 案例 582。

## 读者承诺

细纲服务正文，不是正文服务细纲。实际写作中两章内容一章写完，只要事件功能完成、节奏顺，就继续往下写。

## 核心循环

```text
细纲预估 -> 正文实写 -> 篇幅偏差 -> 检查事件功能 -> 顺写后续
```

## 开篇必备

- 细纲点服务事件功能。
- 正文完成后检查因果。
- 不为章数硬灌水。
- 不因篇幅偏差停住。
- 后续章节自然接上。

## 期待来源

- 作者减少无效焦虑。
- 正文节奏优先。
- 缺什么补什么。
- 不被细纲形式绑死。

## 断档风险

- 为凑两章强行加废话。
- 细纲过细导致写作卡死。
- 忽略情绪和因果缺口。
- 篇幅变化后不更新后续安排。

## 训练任务

写完一章后，用细纲逐项打勾。只补没完成的事件、情绪或因果，不补“原本计划的章数”。
