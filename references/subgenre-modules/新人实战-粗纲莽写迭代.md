# 新人实战-粗纲莽写迭代

依据案例：batch-287 案例 1431

## 读者承诺

新人先达到能开书的最低结构标准，再用真实写作反馈迭代，而不是长期停在学习和准备里。

## 核心循环

```text
教程入门 -> 两三卷粗纲 -> 结局预设 -> 开书执行 -> 反馈暴露问题 -> 修改下一轮
```

## 开篇必备

- 已学完一套基础教程。
- 两三卷可执行粗纲。
- 一个明确结局方向。
- 每日更新计划。
- 第一轮反馈观察点。

## 期待来源

作者知道故事暂时能跑，读者也能连续看到主线推进；真实连载反馈会暴露节奏、完读、设定和情绪问题。

## 断档风险

- 无限学习不动笔。
- 粗纲没有结局，写到中途散掉。
- 没存稿导致心理压力过大。
- 平台节奏不匹配，起点过慢或番茄过快。
- 用拉片和找灵感替代正文产出。

## 训练任务

给一个新题材写三卷粗纲和一句结局，然后安排 7 天试写计划，每天记录字数、卡点和一个需要迭代的问题。
