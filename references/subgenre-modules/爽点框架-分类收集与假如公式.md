# 爽点框架-分类收集与假如公式

依据案例：batch-367 案例 1831

## 读者承诺

爽点可以分类收集，但最终要用“假如”公式转化成题材和情节，让读者获得生存、地位或情绪优势。

## 核心循环

```text
爽点原理 -> 分类归档 -> 假如公式 -> 情节设计 -> 优势兑现
```

## 开篇必备

- 一个明确爽点类别。
- 读者能感到的优势。
- 成本和收益差。
- 可展开的假如前提。

## 期待来源

- 神眷偏爱。
- 信息差优势。
- 小劳大获。
- 现代意识吊打土著。

## 断档风险

- 只背分类不转化。
- 信息差破坏主角代入。
- 爽点太抽象，场景不可写。
- 每次都从原理重推，效率低。

## 训练任务

把 10 个爽点素材归类，再各写一个“假如……”句子，挑 3 个扩成开篇事件。
