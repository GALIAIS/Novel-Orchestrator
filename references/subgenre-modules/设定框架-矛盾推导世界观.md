# 设定框架-矛盾推导世界观

依据案例：`batch-112` 案例 557。

## 读者承诺

设定不是百科清单，而是从核心矛盾里推导出来的组织、资源、地图和力量规则。先知道谁压迫谁、争什么，再补门派、功法和等级。

## 核心循环

```text
核心矛盾 -> 资源争夺 -> 阶层组织 -> 地图区域 -> 力量规则
```

## 开篇必备

- 先写核心矛盾。
- 设定服务冲突。
- 只提前固定会导致崩盘的规则。
- 细节边写边补。
- 每个组织都有利益来源。

## 期待来源

- 世界观有压迫和反抗。
- 组织不是凭空出现。
- 升级体系有边界。
- 后期战力不容易崩。

## 断档风险

- 设定太细，没开书就耗尽激情。
- 设定太粗，后期崩坏。
- 功法只剩名字。
- 地图门派和剧情矛盾无关。

## 训练任务

用一个核心矛盾推导世界观：写出 3 个组织、3 种资源、3 个地图区域和 3 条力量限制。
