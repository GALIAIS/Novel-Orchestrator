# 现代敏感词-真名规避系统延展

依据案例：`batch-050` 案例 250。

## 读者承诺

现代都市题材先规避品牌、地名、真人艺人等真名风险；系统文后期升级要围绕标题亮点扩展，不能写着写着变成普通升级打怪。

## 核心循环

```text
现实真名 -> 替代表达 -> 平台上传校验 -> 系统亮点 -> 围绕亮点升级 -> 后期不跑题
```

## 开篇必备

- 品牌、地名、人名优先虚构或模糊化。
- 目标平台敏感词要上传校验。
- 标题亮点前三章必须兑现。
- 系统升级方向提前设计。
- 每次升级都服务原卖点。

## 期待来源

- 读者被标题亮点吸引。
- 前期快速兑现能建立信任。
- 后期持续围绕亮点扩展，读者会觉得没跑题。
- 系统升级带来新场景、新关系和新代价。

## 断档风险

- 现代真名靠太近，容易触发风险。
- 平台敏感词规则差异大，不能只凭经验。
- 后期改写普通升级打怪，会背离标题承诺。
- 系统升级如果替换卖点，不如不升级。

## 训练任务

给一个都市系统文设计三阶段升级：每阶段写清仍然服务哪个标题亮点，并列出需要虚构化的品牌、地名和人名。
