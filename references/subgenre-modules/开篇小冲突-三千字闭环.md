# 开篇小冲突-三千字闭环

依据案例：batch-278 案例 1388

## 读者承诺

第一章用 2000-3000 字完成世界困境、金手指显形、事件压迫和主角即将出手的小闭环，先写完整再删慢点。

## 核心循环

```text
困境介绍 -> 金手指展示 -> 事件压迫 -> 主角行动/将行动 -> 断章承接
```

## 开篇必备

- 500-800 字内交代困境。
- 500 字内让金手指显形。
- 事件必须和主角困境有关。
- 结尾给出主角行动期待。

## 期待来源

读者迅速知道主角为什么难、凭什么破局、马上要怎么做。期待来自行动前的压迫，而不是设定说明。

## 断档风险

- 第一章只解释世界观。
- 金手指太晚出现。
- 小冲突拖到 3000 字外仍没有行动期待。
- 断章断在信息未讲完，而不是主角要出手。

## 训练任务

写一个 2500 字开篇：困境 700 字、金手指 500 字、事件压迫 1000 字、出手断章 300 字。写完后删掉不服务四段功能的解释。
