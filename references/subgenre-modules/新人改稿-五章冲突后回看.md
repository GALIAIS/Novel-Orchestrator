# 新人改稿-五章冲突后回看

依据案例：`batch-079` 案例 393。

## 读者承诺

这个模块用于阻止新人把第一章反复修碎。它承诺把修改单位从“几千字第一章”提升到“首个冲突闭环”，先写够五章再回看。

## 核心循环

```text
第一章不安 -> 暂停细修 -> 写够五章 -> 解决首个冲突 -> 放置回看 -> 定点修改
```

## 开篇必备

- 第一章先保证能读，不追求完美。
- 明确首个冲突是什么。
- 五章内让主角行动解决一次冲突。
- 放置 24-48 小时再回看。
- 回看时同时看文字、事件和期待。

## 期待来源

- 作者从完整小单元判断问题。
- 主角行动线更清楚。
- 冲突是否成立能被验证。
- 修改不再被即时焦虑支配。
- 回看本身训练作者判断力。

## 断档风险

- 第一章刚写完就反复删改。
- 还没解决冲突就求完整诊断。
- 把朋友“不错”当成有效商业反馈。
- 只改句子，不看事件。
- 截图字体、排版干扰真实阅读。

## 训练任务

锁定第一章不改，继续写到第五章。回看时只标三类问题：读者期待不清、主角行动不足、冲突没有兑现。其余句子级问题延后处理。
