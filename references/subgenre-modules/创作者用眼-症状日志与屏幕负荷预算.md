# 创作者用眼-症状日志与屏幕负荷预算

依据案例：batch-391 案例 1951

## 读者承诺

用预设离屏中断、完整屏幕负荷统计和症状日志保护长期创作；工作流只负责减负，持续、突发或加重异常交给合格专业人员评估。

## 核心循环

```text
预算近距离屏幕任务 -> 主动离屏中断 -> 记录症状诱因 -> 改用语音纸面或听读 -> 异常升级时专业评估
```

## 开篇必备

- 把码字、检索、聊天和娱乐合并统计。
- 写作开始前安排真正离屏的中断点。
- 记录症状时间、持续与重复情况。
- 明确不靠帖子自行诊断的升级边界。

## 期待来源

- 轻微不适在硬撑前被看见。
- 非必要看屏工作逐步被替代。
- 日志让高负荷环节可以调整。
- 专业评估及时接管高风险异常。

## 断档风险

- 把一篇经验帖当医疗处方。
- 从电脑换手机冒充休息。
- 用镜片营销掩盖正在加重的症状。
- 因赶更新延迟处理突然视力变化。

## 训练任务

连续七天记录全部近距离屏幕任务和眼部不适，将一项构思改为语音或纸面；出现突然、单眼、持续或加重异常时停止自行解释并联系专业机构。
