# 大纲正文-方向取舍校验

依据案例：`batch-151` 案例 751。

## 读者承诺

拆书模仿出来的开头可以帮助启动，但正文最终必须服务大纲方向。方向没错就修衔接，方向错了就删正文保大纲。

## 核心循环

```text
模仿开头 -> 正文偏移 -> 校验主线方向 -> 补桥/改纲 -> 方向错误则删重写
```

## 开篇必备

- 原大纲的核心方向。
- 开篇模仿来的卖点。
- 人物目标是否还一致。
- 第一冲突是否还服务主线。
- 接不上时的取舍标准。

## 期待来源

作者期待既保留模仿开头的有效爽点，又不丢掉长线承诺。读者期待开头的事件能自然通向后文，而不是孤立漂亮。

## 断档风险

- 只顾开头效果，写偏主线。
- 为了保留已写字数强行接大纲。
- 把大纲当死板细节，不允许正文合理变化。
- 方向错了还继续补丁，越写越散。

## 训练任务

拿一个开头和大纲做方向校验：人物目标、主线冲突、金手指用途、后续承诺各写一句。如果任意两项方向错，删正文重写。
