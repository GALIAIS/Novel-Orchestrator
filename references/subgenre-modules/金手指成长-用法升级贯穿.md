# 金手指成长-用法升级贯穿

依据案例：`batch-065` 案例 325。

## 读者承诺

这个模块用于金手指中后期续航。它承诺的是让金手指随着主角成长继续参与解决问题，既可以字面升级，也可以用法升级，避免中后期边缘化导致作品类型承诺丢失。

## 核心循环

```text
前期爽点 -> 主角变强 -> 旧用法疲劳 -> 升级/换用法 -> 更高难度问题 -> 类型承诺延续
```

## 开篇必备

- 前期明确金手指爽点。
- 预留成长或扩展空间。
- 中期准备组合用法。
- 后期准备高难度选择或高代价使用。
- 主角再强，金手指也要参与主线。

## 期待来源

- 读者想看金手指继续装逼。
- 旧功能在新场景里有新用法。
- 更高难度选择带来新刺激。
- 金手指和主角能力同步成长。
- 类型承诺不变。

## 断档风险

- 中后期忘了金手指。
- 主角解决方式突然换类型。
- 一直重复低难度用法。
- 主角成大神后系统无事可做。
- 读者为金手指入坑，却看不到金手指。

## 训练任务

为一个金手指写三阶段：前期低风险爽点，中期组合应用，后期高难度选择。每阶段写一个具体事件和读者期待。
