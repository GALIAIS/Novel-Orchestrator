# 首句入冲突-悬念升级与读者博弈

依据案例：batch-362 案例 1809

## 读者承诺

及格开头还不够，要让读者第一眼产生问题。首句进入冲突或异常，亮明主角身份，层层抛悬念，再用爽点和解释回收。

## 核心循环

```text
首句异常 -> 主角身份 -> 悬念升级 -> 冲突激化 -> 爽点兑现 -> 解释回收
```

## 开篇必备

- 第一段有异常或冲突。
- 读者知道这是主角。
- 主角行为让读者想问为什么。
- 第三段前后冲突继续升级。

## 期待来源

- 主角异常举动。
- 读者猜测意图。
- 悬念层层加码。
- 爽点后解释前因。

## 断档风险

- 只满足 200 字信息、500 字冲突的及格线。
- 主角出场像路人。
- 悬念没有回收。
- 只追求快，没人设、笑点、误会。

## 训练任务

把 500 字铺垫开头改成首句入冲突，第一段让读者问“为什么”，第三段升级冲突。
