# 双开准备-需求驱动与存稿

依据案例：`batch-062` 案例 310。

## 读者承诺

这个模块用于判断作者是否该双开。它承诺的是把“双开冲动”拆成真实需求、能力条件、读者更新承诺、存稿和切割方案；没有需求和准备，就优先单开、爆更或先写完一本再发。

## 核心循环

```text
双开念头 -> 真实需求 -> 主次取舍 -> 开头可签 -> 存稿/大纲准备 -> 数据验证 -> 保一本切一本
```

## 开篇必备

- 写清为什么必须双开，而不是一本一本写。
- 两本书要有主次，不要平均用力。
- 两个开头都要打磨到可签或至少可投。
- 每本准备精修 10-20 万字存稿。
- 大纲和卷纲要足够细。
- 提前写好最坏结果和切书条件。

## 期待来源

- 作者有明确商业或训练需求。
- 存稿降低断更风险。
- 编辑反馈降低开局失败概率。
- 主次策略避免两本一起拖死。
- 数据不好能快速止损。

## 断档风险

- 新人只因想写系列就双开。
- 两本都没开头质量和存稿。
- 当前书读者更新期待被破坏。
- 高速产能不稳定。
- 两本都扑却舍不得切。
- 本可单本爆更赚钱，却分散精力。

## 训练任务

写一张双开准备表：真实需求、两本主次、每本开头状态、每本存稿字数、卷纲完成度、每日产能、读者更新承诺、切书条件。任一项空白，就先不双开。
