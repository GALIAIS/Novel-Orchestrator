# 冲突结构-乱子两个关口

依据案例：batch-304 案例 1517

## 读者承诺

用一个乱子和两个关口锁住主角退路，让冲突从“有事发生”变成“主角无法退出”。

## 核心循环

```text
乱子 -> 关口1封退路 -> 第二幕对抗 -> A/B故事交汇 -> 关口2进入决战
```

## 开篇必备

- 平静生活被不平常事件打破。
- 关口 1 让主角有进无回。
- B 故事承载主题和人物弧。
- 关口 2 推向最后决战。

## 期待来源

- 读者担心主角无法退出。
- 第二幕麻烦持续升级。
- A/B 合一带来主题兑现。

## 断档风险

- 主角只是好奇进入麻烦。
- 退路没有被封死。
- 把 B 故事误当普通暗线。

## 训练任务

写一个冲突骨架：乱子、关口 1、A 线目标、B 线主题、关口 2，各一句。
