# 开篇倒叙-高潮前置与回溯断爽风险

依据案例：`batch-210` 案例 1047。

## 读者承诺

高潮前置可以抓人，但第二章不能立刻长回溯，把刚启动的爽点和悬念切断。

## 核心循环

```text
高潮开场 -> 读者上钩 -> 承接当前冲突 -> 短补前史 -> 继续兑现
```

## 开篇必备

- 第一章的当前悬念要清楚。
- 第二章继续承接当前结果。
- 前史只补必要信息。
- 起源设定拆进行动和对话。

## 期待来源

读者被精彩场面吸引后，想看的是冲突如何继续，而不是从多年以前重新开始。

## 断档风险

- 第一章刚爽，第二章倒回起源。
- 回溯篇幅过长。
- 当前悬念暂停太久。
- 前置高潮变成骗点开。

## 训练任务

写两版第二章：一版长回溯，一版继续当前冲突并只用 3 个细节补前史，对比断爽点。
