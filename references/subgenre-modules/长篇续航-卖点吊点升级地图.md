# 长篇续航-卖点吊点升级地图

依据案例：`batch-077` 案例 383。

## 读者承诺

这个模块用于防止中后期写崩。它承诺开篇噱头之后，用吊点接续、升级体系、地图体系和人际关系维持，让读者有持续追读理由。

## 核心循环

```text
开篇噱头 -> 核心吊点 -> 阶段兑现 -> 新吊点 -> 升级/地图推进 -> 长篇续航
```

## 开篇必备

- 主线大方向明确。
- 第一吊点清楚。
- 吊点兑现后有新期待。
- 升级体系或地位体系可持续。
- 地图或阶段提前规划。

## 期待来源

- 吊点让读者想验证。
- 小高潮和大高潮保持节奏。
- 地图扩大影响力。
- 等级或地位带来可见成长。
- 熟悉人际关系减少换地图断裂。

## 断档风险

- 一个噱头写完无后续。
- 卖点缺失或突然转移。
- 主线目标消失。
- 换地图导致人物关系断裂。
- 过度听单个读者建议改坏方向。

## 训练任务

给一个脑洞开局写 50 万字续航表：每 10 万字一个阶段，每阶段写吊点、地图、等级或地位、高潮和下一阶段新期待。
