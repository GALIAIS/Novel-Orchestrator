# 番茄试错-十万首秀反馈闭环

依据案例：batch-348 案例 1740

## 读者承诺

达到签约标准后，十万字首秀是低成本实战反馈。作者应边写边学，用数据和评论定位真实流失点。

## 核心循环

```text
签约正反馈 -> 写到首秀 -> 获得流量和评论 -> 看完读/跟读流失 -> 下本迭代
```

## 开篇必备

- 承认签约是阶段正反馈。
- 制定十万字完成计划。
- 首秀后看完读率、跟读率和评论。
- 把读者喜欢/讨厌/流失点分开记录。
- 下本只改最关键的三类问题。

## 期待来源

作者不再用想象判断作品，而用真实读者反馈修正下一步。

## 断档风险

- 永远等到自己满意才投稿。
- 把十万字试错看得过重。
- 数据反馈来了却不复盘。
- 评论全信或全不信。
- 写得少，永远学不会取舍和删除。

## 训练任务

写一份十万字首秀计划：每日最低字数、首秀日期、要看数据、评论分类、下本迭代清单。
