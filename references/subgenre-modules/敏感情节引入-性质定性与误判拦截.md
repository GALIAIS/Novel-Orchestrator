# 敏感情节引入-性质定性与误判拦截

依据案例：`batch-414` 案例 2068

## 读者承诺

联姻、背叛、能力失效、送女、失忆和换主角等敏感词出现时，作品会在读者退出前提供足以判断事件性质的最小锚点。锚点只排除最违背核心合同的错误分支，不泄露执行结果、代价和真相回收；读者知道“这是一场假联姻或计谋”，仍然不知道它会不会失败以及关系将付出什么。

## 核心循环

```text
确认主导卖点和关系底线 -> 为敏感关键词列出读者解释树 -> 标出目标分支与高退出风险分支 -> 找到首次误判窗口 -> 用一句信息和一个人物动作完成最小定性 -> 保留执行、代价和暴露方式的未知 -> 让表面分支误导对手而非误导核心受众 -> 真实后果升级目标分支 -> 回收时验证早期锚点
```

方法的关键是区分“性质”和“结果”：性质决定读者是否仍在同一合同内，结果才承担悬念。

## 开篇必备

- 作品主导卖点与核心关系、能力底线。
- 即将进入的敏感关键词。
- 三至五种目标读者可能形成的解释。
- 作品真实目标分支和最危险误判分支。
- 误判会导致退出的最早时间窗口。
- 一句最小性质锚点及其信息来源。
- 一个由主角态度或关系动作给出的底线确认。
- 尚未泄露的执行结果、代价和暴露方式。
- 回收时能验证锚点而非临时改口的证据。

## 期待来源

- 计谋如何在已知是伪装的情况下仍然失败。
- 主角确认底线后需要为此支付什么。
- 对手为何会相信表面分支。
- 既有关系会否因伪装产生真实裂缝。
- 敏感词背后的规则限制还能造成何种新问题。
- 回收是否能证明前置线索公平而非作者补丁。

## 断档风险

- 把危险误判当作廉价悬念，等反转时才解释。
- 用“后面会好起来”替代可见锚点。
- 一次性公布完整计划，结果也失去未知。
- 旁白保证角色不会违约，人物行动却没有确认。
- 锚点出现在目标读者已经退出之后。
- 伪装没有真实代价，敏感情节只是空壳。
- 只误导读者、不误导故事内对手，计谋缺少因果。
- 回收推翻早期信息而不是解释早期信息。

## 训练任务

为联姻、背叛和能力失效各画一棵三分支解释树，标出首次误判窗口。每个情节只允许一句定性信息和一个人物动作排除最危险分支；写三段各 800 字的开场，确保读者知道性质却不知道结果，并让伪装对故事内对手产生实际作用。
