# 商业写作心态-读者需求优先

依据案例：`batch-324` 案例 1616

## 读者承诺

想靠网文获得报酬，就要把读者需求放在自我表达之前；作者表达可以保留，但必须折进读者愿意付费的情节体验里。

## 核心循环

```text
自我表达 -> 读者反馈不匹配 -> 抗拒修改 -> 接受商业目标 -> 调整承诺 -> 稳定交付
```

## 开篇必备

- 明确这本书优先服务哪类读者。
- 把作者想表达的东西压缩成情节功能。
- 识别读者当前最想看的爽点、情绪或关系。
- 遇到冲突时，先保证读者承诺清晰。
- 保留个人表达的前提是它不破坏阅读期待。

## 期待来源

读者期待文本持续交付自己想看的内容，而不是被迫理解作者的表达欲。

## 断档风险

- 把爱好写作心态带进商业连载。
- 明知小改更顺读者期待，却因自尊抗拒。
- 还没积累读者，就要求读者先听作者表达。
- 把“我很重要”误解为“读者需求不重要”。

## 训练任务

列出一章里 3 个作者表达点和 3 个读者期待点，删改最冲突的一处，让读者承诺更明确。
