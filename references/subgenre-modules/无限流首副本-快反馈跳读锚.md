# 无限流首副本-快反馈跳读锚

依据案例：`batch-149` 案例 742。

## 读者承诺

第一个副本要短、快、有效：让主角尽快展示特质，拿到后续关键能力或装备，同时让读者明白主神空间的基础玩法。

## 核心循环

```text
进入首副本 -> 快速任务 -> 主角亮点 -> 关键奖励 -> 规则暗示 -> 信息锚回钩主线
```

## 开篇必备

- 首副本字数和章数上限。
- 主角第一次破局方式。
- 后续会反复使用的技能或装备。
- 空间规则可利用的信号。
- 原作/副本背景的最低理解补充。
- 段首段尾的信息锚。

## 期待来源

读者期待开篇立刻看到“这个主角怎么赢”和“这个空间怎么玩”，并在过渡段跳读后仍能接住主线。

## 断档风险

- 第一个副本写成八十章长卷。
- 原作背景不解释，非粉丝读者看不懂。
- 关键奖励太晚出现。
- 过渡段信息埋在长段中，跳读者断链。
- 回主线时不提示前文关键结论。

## 训练任务

把一个首副本压缩成 12-20 章章纲，每章只写任务推进、主角亮点、奖励线索三项。另写 5 个段首/段尾信息锚，保证跳读者也能跟上。
