# 强期待任务设计

依据案例：`batch-198` 案例 987。

## 读者承诺

强期待来自提前公开危机、代价、限制、唯一解法和高收益，让读者等主角破局。

## 核心循环

```text
关系绑定 -> 高代价危机 -> 常规无效 -> 主角唯一解 -> 高收益兑现
```

## 开篇必备

- 危机和主角关系密切。
- 代价足够大。
- 解决方法少。
- 主角被压制或被误解。
- 收获明确且不拖太远。

## 期待来源

读者知道只有主角能解，并提前看见破局后的崇拜、资源、地位或关系收益。

## 断档风险

- 危机和主角关系弱。
- 代价不清楚。
- 谁都能解决。
- 收获太小或太远。

## 训练任务

设计一个任务，填“关系、危机、代价、唯一解、压主、差异感、收获、时间”八项。
