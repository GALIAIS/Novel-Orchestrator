# 倒叙纠错-生死危机前置

依据案例：`batch-104` 案例 517。

## 读者承诺

把后续生死危机或中心冲突放到开头，再回到正序补背景，就是倒叙。它的价值是先制造悬念，再让背景补足变得有必要。

## 核心循环

```text
中心危机前置 -> 悬念形成 -> 顺叙补背景 -> 接回危机 -> 兑现或升级
```

## 开篇必备

- 前置片段必须是中心危机。
- 读者能产生明确疑问。
- 后续补背景要按问题顺序来。
- 倒叙和顺叙转换要有过渡。

## 期待来源

- 主角为什么陷入危机。
- 危机如何解决。
- 背景补足中的因果。
- 接回危机后的新推进。

## 断档风险

- 术语混乱导致写法误判。
- 前置片段不够重要。
- 顺叙补背景太散。
- 没有过渡，时间线混乱。

## 训练任务

写一个 600 字倒叙开头，前 200 字放中心危机，后 400 字回到起点。标出转场句和三个待解问题。
