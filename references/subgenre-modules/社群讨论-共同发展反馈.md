# 社群讨论-共同发展反馈

依据案例：`batch-126` 案例 629。

## 读者承诺

好社群不是单向喂答案，而是让成员通过提问、分析、反馈和复盘共同发展。

## 核心循环

```text
提出问题 -> 获得讨论 -> 修正认知 -> 复盘沉淀 -> 反向贡献案例
```

## 开篇必备

- 带具体问题进入讨论。
- 记录别人指出的关键点。
- 把反馈转成自己的动作。
- 有收获后也要贡献案例。
- 感谢之外要复盘方法。

## 期待来源

- 同伴案例补充视角。
- 讨论环境提高学习效率。
- 反馈能让盲区显性化。
- 共同发展形成长期积累。

## 断档风险

- 只消费答案，不贡献分析。
- 感谢但不复盘。
- 讨论结论不落到改稿。
- 把社群当情绪安慰而非训练场。

## 训练任务

每次讨论后写 5 行：收到什么反馈、改了什么认知、下一步怎么做、可复用方法、下次贡献什么。
