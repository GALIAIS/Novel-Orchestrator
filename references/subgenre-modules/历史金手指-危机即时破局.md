# 历史金手指-危机即时破局

依据案例：`batch-059` 案例 295。

## 读者承诺

这个模块适用于历史、飞卢向历史脑洞、金手指即时建设类开篇。它承诺的是先把危机推到读者最想看破局的位置，再让金手指用现场效果兑现，而不是先解释规则。

## 核心循环

```text
历史危机场景 -> 主角窘迫 -> 危机逼近顶点 -> 金手指现场生效 -> 敌我震惊 -> 新能力期待
```

## 开篇必备

- 先用读者看得懂的危机钩住。
- 主角要在危机里显得需要破局。
- 金手指不要过早长篇解释。
- 第一次展示必须是现场效果。
- 章末留下能力上限或下一次使用期待。

## 期待来源

- 城破、炮声、追杀、审判等危机制造紧迫。
- 读者期待主角如何在历史现场破局。
- 金手指出现时敌人和主角同时震惊。
- 即时效果替代说明。
- 装逼建立在前面压制和危机之上。

## 断档风险

- 危机刚起来就转去讲金手指设定。
- 读者没看懂金手指就弃书。
- 自我吐槽太多破坏历史代入。
- 简介使用冷门人物，读者关系链不清。
- 装逼没有铺垫，情绪拉不起来。

## 训练任务

写一个 2000 字历史危机开篇。前 1400 字只升级危机，后 600 字让金手指第一次现场生效，解释规则不得超过 100 字。
