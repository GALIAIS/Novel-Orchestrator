# 伪血缘关系-平台尺度与编辑确认

依据案例：`batch-250` 案例 1247。

## 读者承诺

伪血缘关系模块把“无血缘妹妹”等关系卖点控制在平台、年龄、行为和编辑确认的边界内。

## 核心循环

```text
关系卖点 -> 平台语境判断 -> 年龄血缘切边界 -> 编辑确认 -> 克制兑现福利感
```

## 开篇必备

- 说明无血缘或收养关系边界。
- 明确角色年龄。
- 亲密互动以陪伴、信任、保护为主。
- 平台不确定时先问编辑。

## 期待来源

读者期待的是亲近关系、保护欲和日常福利，不是冒着违规风险的露骨描写。

## 断档风险

- 把二次元标配误判为全平台安全。
- 未成年角色色情化。
- 伪血缘关系写成真禁忌。
- 编辑尺度未确认就开书。

## 训练任务

写一个无血缘妹妹设定表：平台、年龄、可写互动、不可写内容、编辑确认问题各列 3 条。
