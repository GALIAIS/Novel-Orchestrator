# 听书自审-耳朵读者视角

依据案例：`batch-258` 案例 1287。

## 读者承诺

听书自审模块把作者从视觉补全里拉出来，用耳朵检查流畅度、画面感和尴尬点。

## 核心循环

```text
写完放空 -> 转听书 -> 闭眼接收 -> 标记不适处 -> 集中修改
```

## 开篇必备

- 写完后先隔一段时间。
- 闭眼听，不边看边补。
- 只记录问题点，不立刻打断修改。
- 复听验证修改效果。

## 期待来源

作者像读者一样被动接收文本，更容易听出拗口、节奏断裂和画面感不足。

## 断档风险

- 边听边看，仍被作者视角补全。
- 听到问题马上碎改，丢失整体判断。
- 只查错字，不查节奏和画面。
- 把听书当结构诊断万能工具。

## 训练任务

把一章稿子转听书，闭眼听完，只记录 5 个最明显问题，再集中修改并复听。
