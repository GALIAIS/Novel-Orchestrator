# 谜语人开篇-网文承诺直给

依据案例：`batch-041` 案例 202。

## 读者承诺

书名、简介和前三章必须让读者知道主线、金手指、人设、冲突和爽点。谜语人包装会直接劝退。

## 核心循环

```text
直给承诺 -> 明确主线 -> 主角讨喜 -> 冲突启动 -> 爽点出现
```

## 开篇必备

- 书名能理解。
- 简介说清主线。
- 三章内有金手指和冲突。
- 主角有可代入的正面特质。

## 期待来源

- 读者知道看什么。
- 主角有行动能力。
- 设定不全藏。
- 第一单元有爽点。

## 断档风险

- 主线、金手指、身世全都不说。
- 主角懦弱歇斯底里。
- 画面感断层。
- 开篇没有网文爽点。

## 训练任务

把一个谜语人开篇改成直给版：主角、目标、金手指、第一冲突和第一爽点必须在简介或第一章出现。
