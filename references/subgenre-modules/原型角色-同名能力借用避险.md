# 原型角色-同名能力借用避险

依据案例：`batch-089` 案例 444。

## 读者承诺

借鉴知名游戏角色时，别同时照搬名字、能力、性格和识别性设定。把原型拆成“功能爽点”，再换名、换背景、换能力表现，重组为自己的角色。

## 核心循环

```text
识别原型 -> 拆功能爽点 -> 改名 -> 改能力表现 -> 改背景关系 -> 原创角色卡
```

## 开篇必备

- 先列出原型可识别元素。
- 名字不要直接照搬。
- 能力表现要换机制。
- 背景和关系线要原创。
- 只保留一个核心功能爽点。

## 期待来源

- 读者感到熟悉但不出戏。
- 能力在新世界有新用法。
- 角色关系不依赖原作。
- 原型气质转化成原创魅力。

## 断档风险

- 架空世界但角色仍高度可识别。
- 只改同音字，其他全照搬。
- 借鉴变成替换名字的同人。
- 法务风险和读者出戏同时出现。

## 训练任务

选一个游戏角色，列名字、外观、能力、性格、关系五个识别点。至少改四个，只保留一个功能爽点，写成原创角色卡。
