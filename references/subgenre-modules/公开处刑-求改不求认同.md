# 公开处刑-求改不求认同

依据案例：`batch-320` 案例 1600

## 读者承诺

公开处刑默认是商业写作反馈机制：找毒点、改问题、提高成绩；只想自我表达或找认同，就不该占用求改场景。

## 核心循环

```text
提交公开处刑 -> 社区指出毒点 -> 作者采纳修改 -> 下一版更好 -> 更接近商业目标
```

## 开篇必备

- 提交前明确自己是求改，不是求夸。
- 写清最想解决的问题。
- 愿意听不舒服的反馈。
- 区分商业目标和自我表达保留区。
- 下一版要能体现修改。

## 期待来源

点评者期待自己的时间能帮助作者改出更能赚钱的版本，而不是陷入认同争执。

## 断档风险

- 作者只想被认可，不想改。
- 反馈者指出毒点后被反喷。
- 自我表达目标和商业反馈目标混在一起。
- 没有下一版迭代，点评价值断掉。

## 训练任务

发公开处刑前写一段声明：本次求改的 3 个问题、会优先采纳的建议、保留自我表达的边界和下一版提交时间。
