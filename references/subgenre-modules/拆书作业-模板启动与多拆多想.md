# 拆书作业-模板启动与多拆多想

依据案例：batch-365 案例 1824

## 读者承诺

新人拆书先套模板完成第一版，再通过提交、反馈和重复拆解形成判断；模板的价值是启动训练，不是替代思考。

## 核心循环

```text
选择对象 -> 套用模板 -> 提交反馈 -> 保留证据 -> 多拆多想 -> 形成判断
```

## 开篇必备

- 明确拆书对象。
- 写清所用模板。
- 只设定少量观察重点。
- 保留原文证据和不确定问题。

## 期待来源

- 从不知道怎么拆到能写出第一版。
- 每次反馈都能修正观察顺序。
- 重复拆解让模板变成个人能力。
- 先完成再迭代。

## 断档风险

- 只收藏模板不动手。
- 把模板当标准答案。
- 没有原文证据。
- 第一次拆不完美就停止训练。

## 训练任务

选一本书的前三章套模板拆一版，再单独写出 3 个已看懂机制和 3 个还说不清的问题。
