# 恶人必死-主角参与清算

依据案例：`batch-120` 案例 598。

## 读者承诺

爽文里的十恶反派承诺的是恶有恶报。读者要看的不是抽象正义，而是主角一步步逼近、付出代价、亲手推动清算。

## 核心循环

```text
反派作恶 -> 读者愤怒积累 -> 主角介入 -> 反派反扑 -> 主角付代价击败 -> 清算兑现
```

## 开篇必备

- 反派罪行要具体。
- 受害者和主角立场要绑定。
- 主角必须有清算动机。
- 反派不能轻易被路人或意外带走。
- 结局代价要匹配罪行。

## 期待来源

- 读者等待反派付出代价。
- 主角越被压，清算越爽。
- 反派反扑强化 BOSS 感。
- 最终惩罚回收前期愤怒。

## 断档风险

- 反派悔改没有足够代价。
- 反派死亡和主角无关。
- 用意外或机制跳过清算过程。
- 高级主题压过爽文读者的情绪账。

## 训练任务

给一个反派写“罪行、受害者、主角动机、反扑、最终代价”五项。确认最终代价由主角行动直接造成。
