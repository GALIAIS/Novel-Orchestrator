# 微创新-基调与亮点

依据案例：`batch-005` 案例 022。

## 读者承诺

保住成熟题材的基本盘，只在一个可控点做亮点，让读者觉得新鲜但不失去原本想看的东西。

## 核心循环

```text
老套路定基调 -> 单点微创新 -> 读者反馈验证 -> 小亮点调整 -> 基调不乱
```

## 开篇必备

- 开头几万字先定清楚全书基调。
- 微创新要在第一章或前三章可见。
- 新人优先创新金手指，其他结构按成熟套路走。

## 期待来源

- 读者熟悉基本套路，降低理解成本。
- 微创新提供新鲜感。
- 小亮点可以根据反馈强化。

## 断档风险

- 为反套路破坏题材承诺。
- 基调中途大改，筛掉开篇吸来的读者。
- 新人同时创新太多位置。
- 亮点太隐蔽，读者前三章看不到。

## 训练任务

设计一个老套路开篇，只改一个亮点。写出“不能改的基调”和“可以根据反馈调整的小亮点”。
