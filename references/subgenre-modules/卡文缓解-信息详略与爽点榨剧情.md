# 卡文缓解-信息详略与爽点榨剧情

依据案例：`batch-256` 案例 1280。

## 读者承诺

卡文缓解模块在素材不足时，用明确爽点、信息详略、对比衬托和叙述模式先把正文推进起来。

## 核心循环

```text
素材不足 -> 明确核心信息 -> 安排详略对比 -> 调整叙述顺序 -> 扩成可读段落
```

## 开篇必备

- 先写清要表达的爽点。
- 主信息和衬托信息分开。
- 重点信息详写，辅助信息略写。
- 倒叙、插叙只为突出信息。

## 期待来源

读者能接收到明确重点，而不是看作者为了凑字数随机加戏。

## 断档风险

- 爽点不明确就硬扩。
- 详略平均用力，重点不突出。
- 叙述模式变花活。
- 临时榨剧情替代长期素材积累。

## 训练任务

为一个卡住场景写 8 行信息表：核心爽点、主信息、衬托信息、略写信息、详写反应、叙述顺序、补素材方向、段落目标，再扩成 800 字。
