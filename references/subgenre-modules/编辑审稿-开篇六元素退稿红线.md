# 编辑审稿-开篇六元素退稿红线

依据案例：`batch-100` 案例 498。

## 读者承诺

用编辑初筛标准检查开篇是否有继续读的最低资格：句子能读、主角尽早出现、矛盾能发展、金手指服务当前矛盾、主角亲自解决问题，并在解决后留下新矛盾。

## 核心循环

```text
基础可读性 -> 主角出现 -> 矛盾产生 -> 矛盾发展 -> 金手指破局 -> 主角打脸兑现 -> 衍生新期待
```

## 开篇必备

- 前 500 字出主角。
- 矛盾产生、发展、解决、衍生都要有。
- 世界观和人际关系只做最小交代。
- 金手指不能说明过长，要能服务眼前问题。
- 主角必须行动，不能让反派单方面推动。

## 期待来源

- 矛盾让读者想知道怎么解决。
- 金手指让读者预见破局方式。
- 主角行动制造爽点归属。
- 矛盾衍生让下一章不断档。
- 单一领域无敌带来噱头，未满足需求带来长线。

## 断档风险

- 病句错字、大长句、大长段。
- 景物和人物描写抢开篇位置。
- 第一人称误用。
- 题材太冷。
- 主角所有需求开局已满足，后续无事可写。

## 训练任务

把第一章前 1500 字切成六段：矛盾产生、关系/世界观、矛盾发展、金手指、解决打脸、矛盾衍生。哪一段没有文本，就补一场动作；哪一段超过两倍篇幅，就删到只保留当前矛盾需要的信息。
