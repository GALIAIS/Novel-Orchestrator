# 平台短篇转型-单篇闭环与渠道适配

依据案例：`batch-409` 案例 2042

## 读者承诺

平台短篇在一次阅读内完成一个核心问题和主要情绪结算；用高差异事件开场、判断变化维持中段、不可逆选择完成高潮，并以当前一手样本和征稿合同验证渠道适配。

## 核心循环

```text
审计当前渠道样本 -> 收束单一问题与情绪 -> 异常事件建立叙述者和利害 -> 信息持续改变判断 -> 不可逆选择使真相关系并线 -> 高潮回答 -> 后果闭环 -> 按征稿与合同要求投递
```

## 开篇必备

- 一个能在单篇内回答的核心问题。
- 一段主关系、一个秘密和一次关键选择。
- 高差异事件与立即利害。
- 由悬念来源决定的叙述视角。
- 中段至少两次判断变化。
- 结尾的情绪和因果后果。

## 期待来源

异常事件先制造点击动力，新信息不断推翻读者解释；高潮把秘密、关系和选择合并，结尾不依赖续集便完成主要回报。

## 断档风险

- 把长篇前三章直接当成短篇。
- 同时承诺多条世界线和关系线。
- 用第一人称口语掩盖因果跳步。
- 热点只作标签，不进入人物选择。
- 依据旧评论猜测当前渠道偏好。
- 未核验征稿、授权和费用条款便宣称适配。

## 训练任务

把长篇中一个已闭合的小弧压成 6000 至 10000 字，每 1000 字必须改变一次读者判断；只保留主关系、秘密和不可逆选择，并另做当前征稿证据表。
