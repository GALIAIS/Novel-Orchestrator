# 配角精炼-先写出来再取舍

依据案例：`batch-323` 案例 1615

## 读者承诺

戏份不重的配角不必在设定阶段一次精炼到位；先写出来暴露功能不足，再决定删、并、改，才有可操作依据。

## 核心循环

```text
配角先落到章节 -> 暴露鸡肋点 -> 判断信息/冲突/情绪功能 -> 删改合并 -> 调整大纲衔接
```

## 开篇必备

- 先写出配角实际出场的场景。
- 标明他在每场戏承担的功能。
- 判断删掉后是否影响信息、冲突、情绪或转折。
- 接受配角修改会牵动大纲。
- 以剧情精炼、衔接流畅、逻辑成立为准。

## 期待来源

作者期待通过实际修改，把看似鸡肋的配角转化为有功能的人物，或果断释放篇幅。

## 断档风险

- 因为点子辛苦构思过就舍不得删。
- 只凭喜好保留配角，不看章节功能。
- 修改配角后不更新大纲衔接。
- 把第一版当成成品，错过修正机会。
- 旧灵感过期后仍强行塞进新剧情。

## 训练任务

列一个配角的三次出场，每次只写“他改变了什么”。如果没有改变，就给出删掉、合并或改成关键转折人物的方案。
