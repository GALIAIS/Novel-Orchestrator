# 红楼同人-对比立flag与暗爽抬咖

依据案例：batch-378 案例 1886

## 读者承诺

同人爽点可以利用读者已有认知，通过对比和连续立 flag 把主角价值显形。flag 越明确，兑现越爽。

## 核心循环

```text
已有认知 -> 低估主角 -> 疯狂立flag -> 反差兑现 -> 抬咖升级
```

## 开篇必备

- IP 读者已有认知。
- 对比对象。
- 主角被低估点。
- 可兑现的 flag。

## 期待来源

- 读者等别人被打脸。
- 主角和大人物互动。
- 信息差暗爽。
- 底牌或撑腰曝光。

## 断档风险

- 立 flag 不兑现。
- 对比对象不清楚。
- 同人只借壳，不用读者认知。
- 爽点来源太单一。

## 训练任务

写一个熟 IP 场景，让三个角色连续低估主角并立 flag，主角一次行动完成反差兑现。
