# 穿越开篇-前世线索后置

依据案例：batch-295 案例 1473

## 读者承诺

穿越文开篇尽快兑现穿越后的主承诺，前世人设和伏笔改到正文里用短线索补出。

## 核心循环

```text
穿越后主线开场 -> 新世界压力 -> 前世线索触发 -> 少量回忆补信息 -> 当前事件继续推进
```

## 开篇必备

- 第一章进入穿越后核心场景。
- 前世信息只保留一个必要钩子。
- 伏笔尽量变成穿越后的物品、习惯、技能或关系。
- 不用番外打断早期节奏。

## 期待来源

- 读者尽快看到新世界、新身份、新冲突。
- 前世线索服务当前破局，而不是独立叙事。
- 旧信息逐步补出，形成持续理解。

## 断档风险

- 开篇长写前世，像挂羊头卖狗肉。
- 番外过早出现，打断主线承诺。
- 倒叙太长，读者忘记当前危机。

## 训练任务

把一段前世重要剧情拆成三个正文线索：一个物品、一个技能、一句短回忆。每个都必须推动当前事件。
