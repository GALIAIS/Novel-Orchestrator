# 恐怖正常化-主观安全与客观异常

依据案例：batch-363 案例 1813

## 读者承诺

让角色判断正常，同时让读者看见明显异常，制造“读者先知危险”的恐怖期待。

## 核心循环

```text
角色说正常 -> 细节显异常 -> 读者先紧张 -> 角色靠近 -> 异常兑现
```

## 开篇必备

- 角色有安全判断。
- 客观异常足够可视化。
- 读者一眼看懂不对劲。
- 后续兑现恐怖。

## 期待来源

- 主观正常和客观异常反差。
- 读者替角色担心。
- 主角或配角继续进入危险。
- 日常场景掉 san。

## 断档风险

- 异常太模糊。
- 主角误判显蠢。
- 只给梗图式反差，不兑现危险。
- 角色淡定没有理由。

## 训练任务

写 5 个恐怖入口：角色说正常，同段给一个读者立刻能懂的异常细节。
