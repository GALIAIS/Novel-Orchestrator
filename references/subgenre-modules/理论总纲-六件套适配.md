# 理论总纲-六件套适配

依据案例：`batch-017` 案例 081。

## 读者承诺

期待感、爽点、铺垫、信息差、情绪拉扯、节奏是一组联动工具，必须按题材、受众和平台环境选择使用。

## 核心循环

```text
题材受众 -> 概念适配 -> 案例验证 -> 小范围迁移 -> 写作反馈
```

## 开篇必备

- 先判断当前主问题属于六件套哪一项。
- 确认教程适用的题材环境。
- 选同题材案例验证。
- 不把单一理论硬套所有题材。

## 期待来源

- 理论能解释当前文本问题。
- 同环境案例可迁移。
- 六件套联动后，诊断更准确。

## 断档风险

- 抽象概念互相打架。
- 不看题材差异。
- 只收藏总纲，不落到案例。
- 用错环境导致误判。

## 训练任务

给当前项目做一次六件套诊断，只选一个主问题，并绑定一个同题材案例。
