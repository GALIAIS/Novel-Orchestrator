# 题材路数-开篇承诺恒定

依据案例：batch-268 案例 1337

## 读者承诺

保持开篇筛选来的读者不被半路换书：探险解密就持续给线索、谜团、恐惧和破解，不要写着写着变成纯打 boss。

## 核心循环

```text
开篇承诺 -> 第一个事件定路数 -> 单元复查 -> 防止惯用套路回流 -> 稳定读者期待
```

## 开篇必备

- 明确前三章承诺的题材类型。
- 第一个事件要代表后续主要路数。
- 每个单元结束后检查是否换题材。
- 记录作者自己的惯用套路，防止自动回流。

## 期待来源

期待来自路数稳定。读者因某种类型进来，就希望后续持续满足同一种核心乐趣。

## 断档风险

- 开头探险解密，后面玄幻打斗。
- 作者只会一个套路，写着写着换皮回流。
- 第一个事件后失去题材约束。
- 类型不统一筛掉原读者。

## 训练任务

为作品写“禁止偏航表”：开篇承诺、必须保留的事件类型、禁止写成的类型、每 5 万字复查点。
