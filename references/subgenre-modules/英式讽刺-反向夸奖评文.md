# 英式讽刺-反向夸奖评文

依据案例：`batch-133` 案例 664。

## 读者承诺

用礼貌夸奖包住真实批评，让熟人社群在轻松语气中识别设定问题。它适合吐槽训练，不适合替代正式改稿意见。

## 核心循环

```text
真实缺陷 -> 套褒义词 -> 形成反差 -> 读者识别潜台词 -> 转回修改建议
```

## 开篇必备

- 真实问题要明确。
- 表面夸法要像真夸。
- 潜台词能被识别。
- 明确使用场合。
- 严肃反馈要另给改法。

## 期待来源

- 礼貌外壳和负面含义的反差。
- 社群内部懂梗。
- 读者从笑点里反查问题。
- 话术可改造成自检清单。

## 断档风险

- 只阴阳怪气，没有可改方向。
- 用在不熟的人身上伤人。
- 问题和夸法不对应。
- 把吐槽当成评文。

## 训练任务

写 5 条反向夸奖评文话术，并为每条补一版严肃修改建议。
