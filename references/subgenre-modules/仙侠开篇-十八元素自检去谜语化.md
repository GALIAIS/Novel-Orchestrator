# 仙侠开篇-十八元素自检去谜语化

依据案例：`batch-196` 案例 977。

## 读者承诺

仙侠开篇先让读者看清主角、视角、危机和期待点，再谈文笔、设定和群像布局。

## 核心循环

```text
主角视角统一 -> 危机明确 -> 期待点出现 -> 动作反馈跟上 -> 信息延后展开
```

## 开篇必备

- 主角身份和视角稳定。
- 当前危机或不公平伤害明确。
- 金手指或异常点尽早出现。
- 动作后有心理和旁人反馈。

## 期待来源

读者知道自己跟谁、看什么危机、等什么变化，不会被谜语句、代称和过早群像拖走注意力。

## 断档风险

- 人物出场太快。
- 元素堆砌但没堆到期待点。
- 主角视角和全知视角来回跳。
- 重要信息打哑谜。

## 训练任务

用开头十八元素自检一段仙侠开头，删掉暂时不用的人物和谜语句，补上危机、金手指线索和动作反馈。
