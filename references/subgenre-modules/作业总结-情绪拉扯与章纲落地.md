# 作业总结-情绪拉扯与章纲落地

依据案例：`batch-245` 案例 1221。

## 读者承诺

情绪拉扯训练把故事结构落到每段剧情的读者情绪变化，让章纲不再只是事件列表。

## 核心循环

```text
提交作业 -> 暴露平铺问题 -> 标情绪上行/下行 -> 定爽点位置 -> 回写章纲
```

## 开篇必备

- 先有传统结构节点。
- 再标每段情绪变化。
- 明确本段爽点。
- 记录仍需练的短板。

## 期待来源

作者知道每段要让读者期待什么，剧情就不再只是按事件顺序滑过去。

## 断档风险

- 只懂理论，不实践。
- 只有事件，没有情绪变化。
- 拉得出去，扯不回来。
- 忽略转场、对白、专业储备等执行短板。

## 训练任务

把一章大纲切成 4 段，为每段写上行/下行和爽点；没有情绪变化的段落重写。
