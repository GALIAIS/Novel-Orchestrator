# 关键物夺取-导师提示反杀

依据案例：`batch-144` 案例 717。

## 读者承诺

主角急需一个关键物，但不会轻易得到；他先失败、受压，再在伙伴或导师的提醒下亲自破局，最终把关键物变成战利品，并用结尾悬念牵出更大威胁。

## 核心循环

```text
需要关键物 -> 第一次失败 -> 敌方压制 -> 导师提醒 -> 主角行动 -> 获得关键物 -> 更高层悬念
```

## 开篇必备

- 关键物必须一眼能懂。
- 关键物和眼前危机强绑定。
- 第一次获取要失败。
- 导师只能提醒，不能代打。
- 获得关键物后要立刻带来局面变化。

## 期待来源

读者会追问：主角什么时候拿到它、怎么拿到、拿到后能不能反杀。关键物越具体，补偿感越强。

## 断档风险

- 关键物不重要，拿不拿都一样。
- 失败太轻，读者没有压力。
- 导师替主角解决，主角失去主动性。
- 拿到关键物后没有即时效果。
- 结尾只结束事件，不留下下一层问题。

## 训练任务

写一个 800 字以内场景：主角必须拿到一件道具或证据，第一次失败并被压制；导师只提醒一个方法；主角亲自反杀或破局；结尾出现更高层人物的注视。
