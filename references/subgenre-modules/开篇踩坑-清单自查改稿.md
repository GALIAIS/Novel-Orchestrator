# 开篇踩坑-清单自查改稿

依据案例：`batch-111` 案例 551。

## 读者承诺

开篇自查清单能把“感觉不对”变成具体问题。新人写完开头后，先对照高频坑位排查，再按影响读者留下的程度改稿。

## 核心循环

```text
写完开头 -> 对照坑表 -> 标出毒点 -> 排优先级 -> 改稿复查
```

## 开篇必备

- 先承认开头可能有坑。
- 每个问题要落到具体段落。
- 优先修影响留存的坑。
- 每轮只改 3 个核心问题。
- 改完再复查。

## 期待来源

- 模糊焦虑变成明确清单。
- 新人知道先改什么。
- 开头毒点逐轮减少。
- 修改有可验证依据。

## 断档风险

- 只自嘲不修改。
- 清单看完不定位到正文。
- 一轮想改完所有问题。
- 改了文风却没改读者承诺。

## 训练任务

对自己的开篇做一次不改稿自查，只标出 5 个坑和对应段落。第二轮再动手改。
