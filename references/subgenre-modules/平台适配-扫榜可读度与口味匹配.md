# 平台适配-扫榜可读度与口味匹配

依据案例：`batch-235` 案例 1175。

## 读者承诺

如果平台热门书大多看不下去，也理解不了爽点，就不要硬在该平台发书。

## 核心循环

```text
扫榜 -> 判断可读度 -> 判断能否理解爽点 -> 匹配自身热情 -> 选择平台
```

## 开篇必备

- 读目标平台热门书。
- 能理解平台读者爽点。
- 不捏着鼻子硬写。
- 找自己能学习的参考书。

## 期待来源

作者和平台读者口味接近，写出来的节奏、爽点和价值判断更不容易格格不入。

## 断档风险

- 平台书都看不下去还硬写。
- 逆平台读者口味。
- 迎合到自己热情耗尽。
- 只看收益，不看适配度。

## 训练任务

扫 3 个平台各 10 本榜单书，记录能否读 3 章、能否理解爽点、是否愿意模仿。选交集最高的平台。
