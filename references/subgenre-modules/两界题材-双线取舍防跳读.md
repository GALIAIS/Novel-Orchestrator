# 两界题材-双线取舍防跳读

依据案例：`batch-060` 案例 298。

## 读者承诺

这个模块用于两界穿梭、双世界、分身、双时间线题材。它承诺的是用双线交错维持节奏，同时避免任一世界变成读者跳读点。

## 核心循环

```text
主世界承诺 -> 副世界功能 -> 资源/信息流动 -> 高潮后切线 -> 另一线继续兑现 -> 回流主线
```

## 开篇必备

- 明确哪一界是主承诺。
- 另一界必须有清晰功能。
- 第一次切换要带来资源、信息或身份差。
- 两边都不能只是换地图。
- 每次切线前后都要有未完成期待。

## 期待来源

- 读者期待两边信息差套利。
- 一个世界高潮后，另一世界接住节奏。
- 两条线互相制造资源和危机。
- 身份差带来爽点。
- 回流时主线推进更快。

## 断档风险

- 某一界无聊，读者跳读。
- 双线没有主次，结构散。
- 只学《道诡异仙》表层双穿，学不到诡异核心。
- 新人尝试复杂双时间线写崩。
- 切换只打断高潮，不制造新期待。

## 训练任务

规划前三次世界切换。每次写清切换前未兑现期待、切换后获得的信息/资源、回到主线时解决了什么问题。
