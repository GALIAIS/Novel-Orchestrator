# 结尾NPC-压主角毒点

依据案例：`batch-134` 案例 667。

## 读者承诺

结尾要完成主角价值结算。强 NPC 可以客串，但不能在天赋、修为、情感收益上全面压过主角，导致读者觉得追随主角不值。

## 核心循环

```text
主角长期成长 -> 结尾价值结算 -> 强 NPC 入场 -> 服务主角抬高 -> 主角不可替代
```

## 开篇必备

- 主角最终价值明确。
- 强 NPC 有功能边界。
- NPC 不能全面抢爽点。
- 若是前作主角，要服务彩蛋或逼格传递。
- 当前主角必须完成不可替代成就。

## 期待来源

- 主角登顶或完成独特价值。
- 强者认可主角。
- 客串扩大世界格局。
- 结尾让读者觉得前文投入被奖励。

## 断档风险

- 结尾新增更年轻更强更爽 NPC。
- NPC 后宫和成就都压主角。
- 梦结局抹掉前文投入。
- 客串强者抢走当前主角结算。

## 训练任务

设计一个结尾强者客串桥段：NPC 可以强，但必须写清他如何抬高主角，而不是压低主角。
