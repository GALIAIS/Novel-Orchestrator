# 搞笑玄幻-人设笑点与梗时效

依据案例：`batch-394` 案例 1967

## 读者承诺

让玄幻主线持续产生轻松体验，而不是停下来讲笑话：人物性格负责稳定发动，任务情境负责变化，热梗只做可替换的短期增幅。

## 核心循环

```text
严肃任务 -> 角色按稳定缺陷作出偏差选择 -> 意外结果形成笑点 -> 关系资源或风险变化 -> 后果推动下一任务
```

## 开篇必备

- 主要搞笑角色有清楚欲望、缺陷和知识盲区。
- 笑点只能由这个角色自然触发。
- 严肃目标和失败代价仍然成立。
- 第一处笑点对任务或关系产生后果。
- 热梗删掉后桥段仍有基本趣味。
- 笑点密度与题材承诺一致。

## 期待来源

读者知道角色会“以他的方式”把事情办歪，却不知道这次会造成什么结果；熟悉的人设和变化的情境同时提供确定感与意外。

## 断档风险

- 所有人都使用同一种作者段子腔。
- 只复刻书单表面梗，不拆具体场景。
- 热梗承担关键因果，语境失效后剧情也失效。
- 危机核心时刻强行搞笑，削弱风险与赌注。
- 笑完局面完全不变，段落可整体跳过。
- 一次测试失败就把机制问题归结为没有天赋。

## 训练任务

围绕一场宗门考核写热梗、人设、情境三版，删除全部流行语后筛选仍成立的部分；再给主要角色设计五种压力下的错误解法，用三场任务验证能否产生不同后果。
