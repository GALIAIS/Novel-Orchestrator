# 大纲灵感-素材积累去抄袭焦虑

依据案例：`batch-067` 案例 335。

## 读者承诺

这个模块用于新人立项、大纲和素材积累。它承诺的是把“怕雷同、没灵感、不会建纲”的焦虑转成可执行的素材收集、拆书提纲和细节改写。

## 核心循环

```text
灵感焦虑 -> 同题材输入 -> 提取故事点 -> 去掉原设定 -> 换主角/代价/细节 -> 形成细纲
```

## 开篇必备

- 不在灵感枯竭时硬开长篇。
- 先承认常见梗会雷同。
- 拆书只提取功能，不照搬设定。
- 每卷细纲要比大纲更具体。
- 用自己的主角、代价和细节重讲故事点。

## 期待来源

- 常见套路降低理解门槛。
- 细节处理制造新鲜感。
- 素材池提供持续事件。
- 细纲让作者不卡文。
- 执行质量替代空想原创性。

## 断档风险

- 把雷同等同抄袭，导致不敢写。
- 只看设定，不提取故事功能。
- 素材太少，大纲撑不起一卷。
- 只写大方向，没有细纲节点。
- 直接搬桥段，没有换代价和人物关系。

## 训练任务

从 5 本同题材书各拆 3 个故事点，只写“功能句”。再把 15 个功能句改写成一卷细纲，每个节点必须替换主角身份、冲突代价或情绪细节。
