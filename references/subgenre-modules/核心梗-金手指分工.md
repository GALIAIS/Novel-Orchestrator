# 核心梗-金手指分工

依据案例：`batch-007` 案例 035。

## 读者承诺

分清工具和模板：金手指是主角的具体优势，核心梗是利用优势反复满足读者情绪的剧情循环。

## 核心循环

```text
金手指 -> 解决阻碍 -> 情绪满足 -> 形成模板 -> 换壳重复
```

## 开篇必备

- 金手指必须具体可见。
- 核心梗必须能重复。
- 每次重复都要产生同一类核心情绪。

## 期待来源

- 读者知道主角有什么工具。
- 读者期待工具如何解决新问题。
- 读者期待熟悉模板换壳后再次爽。

## 断档风险

- 只有金手指，没有可重复剧情模板。
- 核心梗太抽象，落不到事件。
- 同时塞多个核心梗，主承诺混乱。
- 金手指和核心梗不配套。

## 训练任务

写 5 组“金手指/核心梗”。每组必须包含具体外挂、阻碍、解决方式、情绪满足。
