# 模拟器流-世界观融合冒险

依据案例：batch-293 案例 1463

## 读者承诺

模拟器冒险必须和主世界互相证明，每次模拟都带回现实能兑现的信息、资源或风险。

## 核心循环

```text
模拟冒险 -> 奖励/信息 -> 现实兑现 -> 世界观连接 -> 下一轮探索
```

## 开篇必备

- 模拟入口。
- 模拟世界与主世界关系。
- 任务目标。
- 可带回奖励。
- 现实兑现点。
- 文案风格。

## 期待来源

读者看模拟不是为了水字数，而是期待它如何改变现实主线。

## 断档风险

- 系统模拟和主线世界割裂。
- 每次模拟只像签到。
- 奖励没有现实用途。
- 文案无趣，变成播报。

## 训练任务

设计一次模拟：带回一件道具、一条信息、一个风险，并在现实下一章全部兑现。
