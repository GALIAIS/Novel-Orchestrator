# 开篇设定-必要信息最小披露

依据案例：`batch-192` 案例 960。

## 读者承诺

开篇先解决危机，设定只披露读者理解下一段剧情所必需的最小信息。

## 核心循环

```text
危机推进 -> 判断必要设定 -> 二三句话解释 -> 继续动作 -> 后续再展开
```

## 开篇必备

- 当前剧情优先。
- 只解释必须理解的设定。
- 武力境界用于强弱对比。
- 设定嵌入动作或世界介绍中。

## 期待来源

读者不会被设定百科打断，又能理解角色强弱、危机难度和解决方式。

## 断档风险

- 开篇灌设定。
- 完全不解释导致读者看不懂。
- 设定出现太突兀。
- 把武力境界当成独立爽点。

## 训练任务

列出开篇 10 条设定，只保留 2 条“不知道就看不懂下一场戏”的设定，其余延后。
