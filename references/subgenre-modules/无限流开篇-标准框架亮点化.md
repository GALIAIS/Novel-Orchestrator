# 无限流开篇-标准框架亮点化

依据案例：`batch-031` 案例 154。

## 读者承诺

标准无限流可以稳定，但必须用书名、金手指说明、身份压迫回收和顺滑转场制造吸引力。

## 核心循环

```text
题材筛选 -> 金手指明确 -> 身份压迫 -> 支线承诺 -> 同场景觉醒 -> 反差爽点
```

## 开篇必备

- 题材加金手指式书名。
- 核心专有词尽早解释。
- 身份压迫不能白给。
- 觉醒转场放在同一压迫场景。

## 期待来源

- 读者一眼知道是不是自己的菜。
- 被压身份形成反差期待。
- 胎中之谜等概念有明确效果。
- 敌人下刀时主角觉醒，爽点顺滑。

## 断档风险

- 结构标准但没有亮点。
- 专有词不解释。
- 身份支线不回收。
- 分隔线硬切身世介绍。

## 训练任务

把一个无限流开篇改成同场景觉醒：敌人压近、身世压迫、金手指解释、反手装逼连续完成。
