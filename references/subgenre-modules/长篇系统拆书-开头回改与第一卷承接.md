# 长篇系统拆书-开头回改与第一卷承接

依据案例：`batch-220` 案例 1097。

## 读者承诺

拆开头必须连接第一卷和后文兑现，先写出长篇流程，再回头改开篇。

## 核心循环

```text
开头方向 -> 大纲设定矛盾网 -> 写五六万 -> 后文反看开头 -> 回改诊断
```

## 开篇必备

- 不只拆第一章。
- 补齐大纲、设定、矛盾网、第一卷提纲。
- 开头问题要连接后文承接。
- 先判断方向，再精修细节。

## 期待来源

作者不再被第一章困死，而是用后文兑现来验证开头是否正确。

## 断档风险

- 只看第一章技巧。
- 死磕开头导致不往后写。
- 金手指和后文主线脱节。
- 把细节问题误判为方向问题。

## 训练任务

拆一本目标书第一卷，标出开头伏笔、设定兑现、主要矛盾推进；自己的项目写满 5 万字后回改一次。
