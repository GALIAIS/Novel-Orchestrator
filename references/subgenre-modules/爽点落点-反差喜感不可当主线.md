# 爽点落点-反差喜感不可当主线

依据案例：`batch-073` 案例 362。

## 读者承诺

这个模块用于娱乐文、病弱主角、误会反差梗。它承诺的是把铺垫落到主角实力、认可、心疼和震惊上，而不是只靠“大家白担心”的喜感。

## 核心循环

```text
病弱/误会设定 -> 期待铺垫 -> 实力爆发 -> 旁观者震惊/心疼 -> 爽点兑现
```

## 开篇必备

- 反差喜感只能做调剂。
- 主线爽点要对应期待铺垫。
- 爆发后必须提高主角价值。
- 旁观者反应要让读者爽。
- 煽情要控制度。

## 期待来源

- 绝症困境带来心疼。
- 坚持梦想带来尊重。
- 唱功强带来震惊。
- 乐坛关心带来地位爽。
- 路人羡慕形成外部见证。

## 断档风险

- 只写大家被骗。
- 读者觉得浪费表情。
- 主角像卖惨。
- 喜感重复过多。
- 铺垫和爆发不对应。

## 训练任务

写一个误会梗，先按喜感版写 100 字，再改成爽点版。爽点版必须让主角获得认可、震惊或心疼中的至少一种。
