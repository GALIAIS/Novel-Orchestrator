# 第一章切入-早冲突与金手指反证

依据案例：batch-371 案例 1853

## 读者承诺

第一章优先让主角早出场、冲突早发生、金手指在使用中被理解，少用配角谈话和修炼倒豆子。

## 核心循环

```text
主角早出场 -> 冲突目标 -> 上来就打/出事 -> 金手指使用 -> 限制反证规则
```

## 开篇必备

- 第一屏见主角。
- 前三页有冲突。
- 金手指通过动作展示。
- 限制让读者推断规则。

## 期待来源

- 荒野危机。
- 护卫不屑和打脸。
- 主角解决问题。
- 金手指规则可感。

## 断档风险

- 配角谈话 800 字后主角才出现。
- 修炼开头变背景说明。
- 金手指靠解释不靠事件。
- 纠结方案太久不动笔。

## 训练任务

把一个介绍型开头改成事件型：第一屏主角出场，前三页冲突，金手指只用一次动作和一次限制展示。
