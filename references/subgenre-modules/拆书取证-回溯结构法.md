# 拆书取证-回溯结构法

依据案例：`batch-009` 案例 043。

## 读者承诺

拆书先服务写作，不服务笔记好看。通过合书回溯，作者能抓住故事骨架，再回到原文验证自己漏掉了什么。

## 核心循环

```text
读完 -> 合书回溯 -> 写节点 -> 翻书校验 -> 修正模型 -> 仿写迁移
```

## 开篇必备

- 先读完整段或整本。
- 先问哪里最值得学。
- 不翻书写结构节点。
- 再翻书验证，不用彩笔堆符号。

## 期待来源

- 学到的是故事骨架，不是字句标本。
- 反复校验能看见自己真正没看懂的地方。
- 同题材市场文优先，难度不会过高。
- 结构内化后才能换血肉仿写。

## 断档风险

- 一开始抠字句和自然段。
- 对标过高，拆经典却无法迁移。
- 抄开篇三章，以为掌握整本书。
- 没有写作反馈，只做收藏笔记。

## 训练任务

读完 30 章后合书写结构回溯。第二天翻原文校对，标出遗漏的目标、阻碍、转折、收获。
