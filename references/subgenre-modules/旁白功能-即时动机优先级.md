# 旁白功能-即时动机优先级

依据案例：batch-308 案例 1539

## 读者承诺

旁白第一任务是让读者明白角色为什么现在这么做。动机清楚后，氛围、伏笔、设定、期待才有承载空间。

## 核心循环

```text
角色行动 -> 旁白说明即时动机 -> 读者接受行为 -> 再叠氛围/伏笔/期待
```

## 开篇必备

- 每个关键行为先回答“为什么”。
- 必要信息优先于漂亮说明。
- 伏笔和氛围不要抢即时动机的位置。
- 大段说书要有逻辑链和爽点支撑。

## 期待来源

期待来自顺滑理解。读者不被角色行为卡住，才有余力接收伏笔和长期布局。

## 断档风险

- 旁白很多，却没解释直接动机。
- 伏笔优先，当前行为生硬。
- 设定展开没有爽点支撑。
- 说书式旁白缺少逻辑链。

## 训练任务

标注一段旁白：即时动机、必要信息、氛围、伏笔、期待、可删。先补齐即时动机，再压缩其他项。
