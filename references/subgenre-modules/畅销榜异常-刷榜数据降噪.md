# 畅销榜异常-刷榜数据降噪

依据案例：`batch-120` 案例 600。

## 读者承诺

榜单能提供市场信号，但异常排名不能直接当趋势。章节极少、标题蹭热梗、排名突兀、社区怀疑刷数的样本，应先降噪观察。

## 核心循环

```text
发现榜单样本 -> 检查异常信号 -> 标记疑似噪声 -> 连续观察 -> 决定排除/纳入拆书
```

## 开篇必备

- 记录榜单时间和位置。
- 看章节数和更新频率。
- 查作者历史和评论区反馈。
- 判断是否借 IP、热梗或打赏冲榜。
- 至少连续观察 3 天。

## 期待来源

- 稳定样本帮助判断真实读者需求。
- 异常样本提醒市场噪声存在。
- 降噪后拆书更可靠。
- 长期榜单趋势比单日奇观更有用。

## 断档风险

- 看到异常高排名就跟风开题。
- 把刷榜当作自然需求。
- 只看榜单，不看章节和读者反馈。
- 用单日截图替代持续观察。

## 训练任务

扫一个畅销榜，挑出 3 个异常样本。给每个写“异常点、证据、观察期、是否纳入拆书”四项，三天后再判断。
