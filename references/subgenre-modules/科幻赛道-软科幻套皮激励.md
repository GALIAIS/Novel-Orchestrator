# 科幻赛道-软科幻套皮激励

依据案例：`batch-054` 案例 266。

## 读者承诺

科幻可以写，但网文优先选择软科幻或科幻套皮，用科学奇观服务人物目标、危机、升级和爽点，而不是硬拼硬科幻门槛。

## 核心循环

```text
科幻元素 -> 读者可理解卖点 -> 当前危机 -> 科幻能力破局 -> 爽点兑现
```

## 开篇必备

- 明确目标平台是否有科幻激励。
- 区分硬科幻和软科幻。
- 第一章给出读者能懂的科学奇观。
- 科普解释不能压过人物冲突。
- 科幻设定要能产生即时爽点。

## 期待来源

- 科学奇观带来新鲜感。
- 平台激励降低试水成本。
- 软科幻套皮更容易被网文读者接受。
- 设定用于破局，读者才觉得有用。

## 断档风险

- 硬科幻叫好不叫座。
- 科普太多会劝退。
- 只靠赛道激励，不等于有读者。
- 科幻元素不服务爽点，会变成空壳。

## 训练任务

把一个硬科幻设定改成软科幻开篇：保留奇观，增加主角危机、对手压力和一次破局爽点。
