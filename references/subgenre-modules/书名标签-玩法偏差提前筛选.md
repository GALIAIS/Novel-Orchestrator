# 书名标签-玩法偏差提前筛选

依据案例：`batch-326` 案例 1626

## 读者承诺

强标签书名会预设读者期待；如果玩法要偏离标签，必须在简介和第一章提前筛选，让偏差变成卖点。

## 核心循环

```text
书名标签 -> 读者默认期待 -> 玩法偏差 -> 提前说明 -> 筛选读者 -> 偏差变爽点
```

## 开篇必备

- 识别书名带来的默认期待。
- 在简介中交代玩法偏差。
- 第一章用世界规则再说明一次。
- 保留主标签的核心爽点。
- 把偏差写成底牌、反差或装逼层级。

## 期待来源

读者期待自己没有被标题骗进来，并能看到偏差玩法带来的新鲜爽点。

## 断档风险

- 标题承诺远程法爷，中期突然改成练剑。
- 简介不筛选，读者到正文才发现不是想看的书。
- 偏差玩法抢走主标签爽点。
- 反讽式书名没有从第一章开始布局。

## 训练任务

给一个强标签书名写简介和第一章设定句，把玩法偏差提前说清楚，并说明主标签爽点如何继续交付。
