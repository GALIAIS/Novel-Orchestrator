# 非爽点段落-具体期待自测

依据案例：batch-363 案例 1812

## 读者承诺

每个非爽点段落都必须让读者产生具体期待。写完要能回答“读者会期待什么”，答不出就删或压缩。

## 核心循环

```text
非爽点段落 -> 具体期待 -> 后续引爆 -> 无效内容压缩/删除
```

## 开篇必备

- 段落有明确功能。
- 期待能具体写出来。
- 信息后续能转为爽点。
- 字数不超过功能需要。

## 期待来源

- 具体暗示。
- 金手指新意。
- 未解问题。
- 能在高潮处回收的信息。

## 断档风险

- 老套路自以为有期待。
- 一百字能写完拖三百字。
- 信息密度过高压垮读者。
- 爽了一次但和主线无关。

## 训练任务

逐段检查开头，每段写“读者具体期待什么”。写不出来的段落删掉或压缩到 100 字以内。
