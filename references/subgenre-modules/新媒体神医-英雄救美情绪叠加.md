# 新媒体神医-英雄救美情绪叠加

依据案例：`batch-103` 案例 513。

## 读者承诺

新媒体无敌神医文的爽点靠直白卖点和情绪堆叠。多师父发装备是告诉目标读者“主角很牛”，英雄救美则靠受害者受苦的多层铺垫，让主角短时间兑现也足够爽。

## 核心循环

```text
卖点前置 -> 神装发放 -> 受害者受苦 -> 多层情绪叠加 -> 主角登场 -> 救人打脸
```

## 开篇必备

- 书名简介承诺无敌/神医/下山。
- 开头直接展示背景背书。
- 受害者苦难要有层次。
- 主角到场要快速兑现。
- 目标读者能扫读出“主角很牛”。

## 期待来源

- 多师父、多装备、多背景的夸张感。
- 英雄救美的保护欲。
- 受害者被虐的压抑。
- 主角到场后的强兑现。

## 断档风险

- 新人学大佬任性，介绍太多却无反转。
- 非目标读者觉得剧透和啰嗦。
- 铺垫没有情绪，只是拖字数。
- 救人兑现不够狠。

## 训练任务

写一个英雄救美段落，铺垫字数必须大于兑现字数。铺垫中至少包含听说、亲历、亲眼看到三层情绪。
