# 拆书取证-第一框架精拆

依据案例：`batch-011` 案例 051。

## 读者承诺

拆书先抓最精华的第一个完整框架，而不是从第一章机械拆到结尾。第一个框架通常已经展示核心技法和爽点循环。

## 核心循环

```text
开篇钩子 -> 第一个框架 -> 技法展示 -> 爽点循环 -> 后续重复变形
```

## 开篇必备

- 确认第一个框架起止。
- 标短中长期钩子。
- 标金手指首次完整展示。
- 标第一轮爽点循环。

## 期待来源

- 读者在第一个框架看到这本书能提供什么。
- 后续循环有可重复的核心动作。
- 金手指升级能防止重复疲劳。
- 长线结构保证后期不崩。

## 断档风险

- 面面俱到拆全文，最后抓不到重点。
- 只拆开头几章，看不到第一个框架闭环。
- 后续重复不变形。
- 第一个框架钩子解完后没有长线规划。

## 训练任务

拆一本书第一个框架，写出“钩子、金手指、爽点循环、框架收获、后续变形”五项。
