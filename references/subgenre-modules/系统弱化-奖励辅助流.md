# 系统弱化-奖励辅助流

依据案例：`batch-008` 案例 037。

## 读者承诺

系统不抢主线，只在关键节点提供主角异于常人的优势。读者看的重点仍是主角如何用天赋、职业经验、训练和判断推进故事。

## 核心循环

```text
题材主线目标 -> 主角主动行动 -> 系统奖励补强 -> 优势显现 -> 更难目标
```

## 开篇必备

- 先明确题材主线，而不是先展示复杂系统。
- 主角已有能力和系统奖励不能重复。
- 系统第一次出现要解决一个具体短板。
- 对标书中的核心爽点必须保留。

## 期待来源

- 主角主动努力带来奖励，而不是被任务牵着走。
- 弱系统让读者期待“什么时候拿出来用”。
- 奖励补足短板后，主角能完成别人做不到的操作。
- 主线事件来自世界和人物，不完全来自系统发布。

## 断档风险

- 系统太弱，读者忘了它存在。
- 系统太强，挤掉职业优势和天赋优势。
- 没有主对标书，多个设定拼接后主线失焦。
- 奖励机制不能转化成具体事件。

## 训练任务

选一本文内主对标，列出它的核心爽点。再写自己的弱系统奖励表，检查每个奖励是否只是在放大核心爽点，而不是换题材。
