# 番茄发书-目标优先试反馈

依据案例：`batch-037` 案例 184。

## 读者承诺

番茄发书先确定目标：保底、反馈、速度还是存稿稳定。内投和直接发都只是达成目标的路径。

## 核心循环

```text
目标确认 -> 内投/直发选择 -> 开头或存稿准备 -> 反馈/数据验证 -> 改稿或继续
```

## 开篇必备

- 明确是否追保底。
- 写出开头字数和存稿量。
- 准备一个大剧情闭环。
- 设定反馈和数据观察点。

## 期待来源

- 内投不过可先改。
- 直接发能快速试市场。
- 存稿保证更新稳定。
- 目标明确减少焦虑。

## 断档风险

- 不看目标只问路径。
- 存稿太多后发现方向错。
- 直接发无反馈不知道错哪。
- 把其他平台经验硬套番茄。

## 训练任务

写两套番茄发书方案：两万字内投试反馈、五万字直接发试市场，各列成本和观察点。
