# 末世净网-存稿止损转向

依据案例：batch-280 案例 1400

## 读者承诺

末世文遇到净网和平台风险时，按审核、推荐、首秀数据设止损线；上架前能改就改，验证失败就快速转废土、仙侠、修真或新题材。

## 核心循环

```text
存稿投入 -> 风向风险 -> 审核/推荐判断 -> 首秀验证 -> 改题材/切书
```

## 开篇必备

- 已写字数和平台。
- 末世敏感元素清单。
- 审核与推荐风险。
- 明确首秀或数据止损线。

## 期待来源

作者用最小成本验证题材是否还能活，避免上架后被封、限流或大改。

## 断档风险

- 被 5-8 万字存稿绑架。
- 末世血腥暴力和擦边不降级。
- 没有相邻题材替代方案。
- 首秀失败仍继续硬扛。

## 训练任务

为一部 8 万字末世文写三套方案：继续、改废土、切新书。每套写成本、风险、验证节点。
