# 双线时间-先提失窃再回溯过程

依据案例：`batch-252` 案例 1258。

## 读者承诺

双线时间模块用“先提示、后展开”的方式处理并行危机，避免频繁切镜头和硬回溯。

## 核心循环

```text
提示副线危机 -> 主线危机推进 -> 读者挂念副线 -> 主线处理完 -> 回溯副线过程
```

## 开篇必备

- 副线危机只先提一嘴。
- 主线精彩段不频繁切走。
- 回溯点要自然。
- 副线最终服务主角应对。

## 期待来源

读者知道家里出事但主角暂时回不去，会持续期待主角解决当前麻烦后的反应。

## 断档风险

- 两条线平均分镜，打断爽点。
- 事后倒回太久，解释成本高。
- 副线提示太多，抢走主线。
- 回溯只解释事实，不制造后续行动。

## 训练任务

写一个“主角外战、家中失窃”的三段结构：100 字提示失窃、400 字外战、200 字回溯失窃。
