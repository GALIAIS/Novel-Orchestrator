# 矛盾分解-铺垫扩展与水文边界

依据案例：batch-372 案例 1859

## 读者承诺

长篇可以把解决矛盾的步骤继续拆分，但每个子步骤都必须带来必要阻碍、必要收获或必要逻辑补全。

## 核心循环

```text
主目标 -> 前置条件 -> 子步骤冲突 -> 阶段收获 -> 回绑主线
```

## 开篇必备

- 明确终点。
- 阻挡终点的关键门槛。
- 子步骤存在的必要理由。
- 完成子步骤后的阶段奖励。

## 期待来源

- 读者知道主角正在接近终点。
- 每个门槛都有可见对抗。
- 阶段胜利带来成长或资源。
- 子步骤不断抬高最终决战意义。

## 断档风险

- 为拖长而制造可有可无困难。
- 子步骤删掉后主线无损。
- 支线不回绑主线。
- 分解过多导致主题松散。

## 训练任务

把一个三步主线拆成九步，每一步都写出必要理由、冲突事件、阶段奖励和删掉后的损失；删掉无法证明必要性的步骤。
