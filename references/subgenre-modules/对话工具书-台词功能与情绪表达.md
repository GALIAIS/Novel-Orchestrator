# 对话工具书-台词功能与情绪表达

依据案例：batch-346 案例 1728

## 读者承诺

对话训练不是追求漂亮句子，而是让台词承担冲突、关系、信息、节奏和情绪功能。

## 核心循环

```text
场景目标 -> 人物各有所求 -> 台词互相施压 -> 信息自然浮出 -> 关系/局势/情绪改变
```

## 开篇必备

- 每个对话人物都有即时目的。
- 每句台词至少承担一个功能。
- 设定信息尽量由问责、误会、试探、反驳带出。
- 吐槽必须基于真实槽点。
- 对话动作只用于改变节奏和暴露心理，不能机械填充。

## 期待来源

读者通过对白看见人物关系变化，而不是听作者借人物口解释背景。

## 断档风险

- 台词像说明书。
- 人物说话没有身份差异。
- 吐槽密集但没有真实笑点。
- 对话不改变局势。
- 工具书只看概念，没有转成修稿清单。

## 训练任务

把一段解释设定的旁白改成两人冲突对白。要求至少出现一次误解、一次反问、一次沉默或动作反应，并在结尾改变双方关系。
