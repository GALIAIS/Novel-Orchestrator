# 描写公式-六部件观察结构

依据案例：batch-352 案例 1758

## 读者承诺

新人不会写描写时，先把对象拆成可观察维度，再从对话、神态、动作、外形、心理、旁白六个部件里选最服务场景的几项。读者获得的不是华丽辞藻，而是清楚画面、人物状态和下一步行动压力。

## 核心循环

```text
确定描写对象 -> 找特殊性 -> 选观察维度 -> 选六部件中的少数几项 -> 按场景功能压缩
```

## 开篇必备

- 明确描写对象是人、物还是场景。
- 明确本段功能：出场、压迫、转场、伏笔、情绪。
- 至少两个具体观察维度。
- 删除不服务功能的部件。

## 期待来源

- 读者从细节判断身份和状态。
- 动作与神态让对白更有停顿。
- 五感让场景不再空。
- 旁白压缩信息，心理补足选择动机。

## 断档风险

- 六部件全塞，节奏变慢。
- 只写漂亮评价，没有可见细节。
- 描写和剧情功能脱节。
- 五感使用机械，像清单而不是场景。

## 训练任务

同一个角色写三版 80 字描写：初见惊艳、危机压迫、受伤疲惫。每版只选三种部件，并标注删掉了哪三种。
