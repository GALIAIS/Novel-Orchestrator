# 素材搬运-取证投诉边界

依据案例：`batch-063` 案例 315。

## 读者承诺

这个模块用于保护教程、拆书、创作笔记和知识库成果。它承诺的是发现疑似搬运时先取证、再判断、再投诉；同时提醒创作者学习别人方法时只吸收结构功能，不照搬原句和表达。

## 核心循环

```text
发现疑似搬运 -> 保存截图/链接 -> 对照原文 -> 区分引用/洗稿/照搬 -> 通知原作者 -> 平台投诉
```

## 开篇必备

- 保存原文发布时间和链接。
- 截图疑似搬运账号、标题、内容和发布时间。
- 对照相同句段或相同结构。
- 多个视频或文章要分别留证。
- 先证据后判断，避免只靠情绪。

## 期待来源

- 社群成员互相提醒。
- 证据链让投诉更可执行。
- 原作者能快速判断是否维权。
- 明确引用边界，减少误伤。
- 反向约束自己不搬运他人表达。

## 断档风险

- 只说抄袭，不保存证据。
- 只有截图没有链接。
- 把正常引用和照搬混为一谈。
- 投诉前没有原文对照。
- 自己学习时也复制原句、标题和讲法。

## 训练任务

给一篇自己的教程或拆书做保护包：原文链接、发布时间、核心句段、截图、备份。再写一条引用授权边界：哪些可以概述，哪些不能照搬。
