# 读者反馈-全互动指标与评论分流

依据案例：`batch-232` 案例 1158。

## 读者承诺

读者反馈包括点击、推荐、月票、订阅、完读率、评论等一切互动；评论要分流处理。

## 核心循环

```text
收集互动 -> 分类指标/评论 -> 判断真问题 -> 能改则改 -> 继续观察
```

## 开篇必备

- 不单看完读率。
- 数据和评论都纳入反馈。
- 无脑找茬删封。
- 真实错误正经回应，能改就改。

## 期待来源

作者不被单条评论牵着走，也不忽视真实问题，能用反馈修正作品。

## 断档风险

- 单点迷信完读率。
- 每条评论都当圣旨。
- 无脑找茬消耗心态。
- 真错误被当成恶评删掉。

## 训练任务

把一周反馈分成数据异常、真实错误、正常互动、无脑找茬四类。只对前两类安排文本或运营动作。
