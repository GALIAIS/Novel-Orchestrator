# 期待不断档-两长一短

依据精拆：`case-library/精拆/batch-001.md` 案例 001。

## 适用范围

所有长篇网文，尤其是升级、悬疑、恋爱、都市事业、无限副本、规则怪谈等追读依赖强的类型。

## 读者承诺

读者每一刻都知道“接下来还能看到什么值得期待的东西”，不会在一个爽点兑现后失去继续阅读理由。

## 核心循环

```text
短期期待推进 -> 长线期待挂住 -> 短期兑现前露出新目标 -> 旧期待兑现 -> 新期待转为当前目标
```

## 开篇必备

- 第一章末尾必须有短期期待。
- 两万字内至少建立两个长线期待。
- 第一个爽点不能把所有期待一次填完。
- 主角完成当前目标之前，要提前出现下个目标线索。

## 期待来源

- 身份变化后别人怎么看。
- 当前事件背后的更大势力。
- 金手指下一阶段能做什么。
- 人际关系会如何重估。
- 未解真相、旧仇、新地图、资源缺口。

## 断档风险

- 爽点写得很满，但没有新目标。
- 只有短线事件，没有长线牵引。
- 长线太远，阶段性没有小兑现。
- 换地图时旧人际网断裂。

## 操作卡

```text
当前短期期待：
长线期待 1：
长线期待 2：
当前事件兑现前的新线索：
兑现后转正的新目标：
见证体系：
```

## 训练任务

拿自己的前 2 万字，逐章标记读者期待。任何一章结尾如果没有“下一章为什么要看”，就补短期期待；任何一个高潮前如果没有新线索，就补长线扣子。
