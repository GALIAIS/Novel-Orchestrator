# 公开处刑-标准化提交材料

依据案例：`batch-172` 案例 860。

## 读者承诺

公开处刑材料要标准化提交，包含书名、简介、小纲、审核项和正文样本，让点评者能判断承诺与执行是否匹配。

## 核心循环

```text
求评 -> 填表 -> 放入正文样本 -> 添加话题 -> 聚焦点评 -> 回写修改
```

## 开篇必备

- 书名、简介、小纲放在正文前。
- 正文样本至少 6000 字。
- 明确自己最想被点评的问题。
- 添加 `#公开处刑#` 话题。

## 期待来源

作者期待得到有效反馈。标准化材料能减少点评者猜测，让问题更快落到题材承诺、主角、冲突、金手指和第一事件。

## 断档风险

- 只发裸正文，别人无法判断整体承诺。
- 样本太短，看不出第一事件。
- 没有具体求评问题，反馈发散。
- 把规则当大神通用公式，忽略它主要服务新人。

## 训练任务

整理一次公开处刑材料：200 字简介、1000 字小纲、6000 字正文、3 个求评问题，放入同一文档后再提交。
