# 书名简介-名人名著名场面

依据案例：`batch-057` 案例 283。

## 读者承诺

这个模块用于设计高识别度书名简介。它承诺的是把名人、名著、名场面压缩成读者一眼能懂的期待清单，让书名简介直接完成筛选和拉期待。

## 核心循环

```text
公共名人 -> 公共名著 -> 第一名场面 -> 身份反差 -> 熟人评价链 -> 格局升级 -> 点开正文
```

## 开篇必备

- 书名里至少有一个强公共认知对象。
- 简介第一句触发熟悉名场面。
- 身份反差要足够直观。
- 简介中列出主角将会碰撞的熟悉人物或势力。
- 第一章必须兑现书名承诺，不能只停在噱头。

## 期待来源

- 读者立刻脑补“这个人进那个世界会怎样”。
- 熟悉人物被主角重新评价或调度。
- 名场面被改写带来爽点。
- 个体穿越扩大到历史或世界格局。
- 简介本身像后文爽点目录。

## 断档风险

- 名人或名著读者不熟，钩子失效。
- 书名强，正文第一章不兑现。
- 简介只列设定，不列可期待场面。
- 只蹭公共认知，没有新视角。
- 熟悉人物人设写崩，引发反感。

## 训练任务

写 3 个书名简介方案，每个方案必须包含一个名人、一个公共故事世界、一个第一名场面和 5 个后续碰撞对象。写完后删除读者不认识的对象。
