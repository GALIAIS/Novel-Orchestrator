# 仁义人设-系统驱动与性格弧光

依据案例：batch-373 案例 1865

## 读者承诺

仁义人设要通过事件、收益和误会升级证明，不能只靠道德声明。自利者被迫做好事，比纯圣人开局更容易爽文化。

## 核心循环

```text
不平事件 -> 主角被迫出手 -> 系统奖励 -> 外界误会升级 -> 性格弧光
```

## 开篇必备

- 一件具体不平事。
- 主角不想管或不能管的理由。
- 迫使他出手的机制。
- 出手后的奖励和误会。

## 期待来源

- 主角如何被卷入。
- 正义行动的即时爽点。
- 奖励推动实力成长。
- 误会把小善举推成大格局。

## 断档风险

- 纯正义宣言显假。
- 人物塑造拖慢爽文节奏。
- 做好事没有收益和后果。
- 仁义变成鲁莽滥杀。

## 训练任务

写一章自利者被迫行义：一次打抱不平、一次系统奖励、一次外界误会，章末让他不得不继续做好事。
