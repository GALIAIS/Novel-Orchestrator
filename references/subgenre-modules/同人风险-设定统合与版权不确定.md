# 同人风险-设定统合与版权不确定

依据案例：`batch-094` 案例 466。

## 读者承诺

同人给读者的承诺是“熟悉原著的人物、世界、遗憾和爽点被重新打开”。但这个承诺天然带风险：原著 bug、战力表现不一致、官方吃书、版权方限制、原作热度变化，都会让作者失去主动权。

## 核心循环

```text
选择 IP -> 提取原著味 -> 识别设定 bug -> 决定修正/模糊/绕开 -> 用剧情爽点承接读者 -> 持续监控版权和平台风险
```

## 开篇必备

- 让读者立刻知道这是哪个原著味，而不是只堆设定考据。
- 先兑现一个公共认知爽点，证明你会写故事。
- 对原著 bug 只处理影响当前剧情的一两个点。
- 避免一开篇就跨作品战力论证，读者不是来看论文。

## 期待来源

- 原著角色、名场面、遗憾和未完成支线。
- 读者对原著 bug 的共同吐槽。
- 主角用新视角改写原著命运。
- 原著体系被更爽、更顺的剧情重新组织。

## 断档风险

- 作者沉迷统合战力，剧情停止前进。
- 修正过度导致原著味消失。
- 不修正又让硬核读者出戏。
- IP 风险不可控，作品越火越可能被限制。
- 官方设定频繁变动，作者大纲被迫重写。

## 训练任务

选一个同人 IP，写一张风险表：3 个原著味、3 个必须避开的版权/平台风险、3 个设定 bug。每个 bug 只给一个处理动作：修正、模糊、绕开。最后写 500 字开篇，只允许使用一个 bug。
