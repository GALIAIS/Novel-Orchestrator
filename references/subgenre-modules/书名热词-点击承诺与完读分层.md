# 书名热词-点击承诺与完读分层

依据案例：batch-392 案例 1959

## 读者承诺

书名用真实身份、关系和异常动作帮助目标读者准确脑补故事，前三章及时兑现；热词只负责识别，点击与完读必须分层复盘。

## 核心循环

```text
提取真实卖点 -> 组成一句话事件书名 -> 读者形成准确预期 -> 前三章兑现标题事件 -> 分层复盘点击追读和完读
```

## 开篇必备

- 主角身份、核心关系与异常事件。
- 两至三个确实存在于正文的熟悉标签。
- 目标读者只看书名后的故事预判。
- 前三章可见的标题承诺兑现。

## 期待来源

- 熟悉标签降低识别成本。
- 关系和动作立即形成具体画面。
- 点击后迅速确认标题没有骗人。
- 标题事件的新结果转成追读问题。

## 断档风险

- 热词堆满却没有清楚事件。
- 猎奇醒目被误当持续吸引力。
- 高曝光低完读仍只优化标题。
- 横向截图被误读为因果实验。

## 训练任务

为同一故事写题材词、身份关系、完整事件句三组书名，让目标读者预测前三章，淘汰错配版本并分别记录点击与留存。
