# 灵感设定-取舍服务主线

依据案例：batch-298 案例 1488

## 读者承诺

灵感既要记录，也要取舍；能服务核心矛盾和爽点的立刻用，不能服务当前主线的放进素材库。

## 核心循环

```text
记录灵感 -> 按核心矛盾筛选 -> 正文使用必要设定 -> 素材库搁置剩余点子 -> 后续复用
```

## 开篇必备

- 明确当前核心矛盾。
- 大纲围绕金手指和主角人设。
- 只细化等级、势力、爽点相关设定。
- 正文避免一次性说明文。

## 期待来源

- 读者看到冲突推进，而不是设定堆砌。
- 好点子及时转化成爽点。
- 素材库缓解后续枯竭。

## 断档风险

- 灵感堆成沙子，不成系统。
- 舍不得删无关设定。
- 把好点子一直藏着不用。

## 训练任务

列 10 条设定，按“立刻用、改后用、搁置、删除”分类，并写出与核心矛盾的关系。
