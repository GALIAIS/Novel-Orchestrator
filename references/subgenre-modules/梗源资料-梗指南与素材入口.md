# 梗源资料-梗指南与素材入口

依据案例：`batch-244` 案例 1217。

## 读者承诺

用梗前先有稳定素材入口，理解梗的含义、语境和传播人群，再决定是否入文。

## 核心循环

```text
追踪梗源 -> 理解语义 -> 判断适用场景 -> 放入角色/旁白 -> 读者接梗互动
```

## 开篇必备

- 记录梗来源。
- 写清梗含义。
- 判断读者识别门槛。
- 标注适合的角色和情绪。

## 期待来源

读者识别共同语境后，会在段评里接梗，形成轻松互动。

## 断档风险

- 不懂含义就硬塞。
- 旧梗过期。
- 梗和正文情绪冲突。
- 直接搬运科普内容。

## 训练任务

做 10 个梗的筛选表，只保留 3 个最贴角色、最不破坏剧情的梗。
