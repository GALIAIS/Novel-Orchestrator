# 名著爽点-善恶报应加工

依据案例：`batch-166` 案例 829。

## 读者承诺

从名著中抽取善恶报应、报恩报仇、做好事不留名等情绪原型，再压缩节奏并改造成网文爽点。

## 核心循环

```text
名著场景 -> 情绪原型 -> 善恶报应 -> 多次揭晓 -> 网文章节化加工
```

## 开篇必备

- 先拆情绪机制，不照搬名著情节。
- 明确善方、恶方和见证者。
- 把长铺垫压缩到网文可承受范围。
- 至少设计两次以上回响。

## 期待来源

读者期待善行被看见、恶人被惩罚、受助者反过来赞美或回报主角。

## 断档风险

- 名著节奏太慢，直接搬运劝退读者。
- 只报仇没有报恩，情绪单薄。
- 做好事不留名没有后续揭晓。
- 善恶报应没有见证者，爽点不扩散。

## 训练任务

拆一个名著爽点，改写成三段式网文兑现：第一次行动爽，第二次背后赞美爽，第三次身份揭晓爽。
