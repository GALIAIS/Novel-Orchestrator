# 借鉴边界-创意功能化防中译中

依据案例：batch-266 案例 1327

## 读者承诺

让拆书和借鉴停留在创意、功能、结构层面，避免把原作句子、段落、场景顺序和剧情链换皮复述。

## 核心循环

```text
看到好桥段 -> 抽取叙事功能 -> 丢掉原句原链路 -> 换人物规则因果 -> 写成新事件
```

## 开篇必备

- 先写“这个桥段有什么功能”，不要先复述剧情。
- 换掉人物欲望、场景规则、冲突对象和结果。
- 保留情绪机制，不保留表达和事件顺序。
- 写完反查是否仍像原作。

## 期待来源

期待来自作者能把别人的成熟方法转成自己的题材生产力，而不是把读者熟悉的场景换名字搬过来。

## 断档风险

- 只替换词语，不替换因果。
- 连续多个事件与原作同序。
- 人物关系和场景调度过于相似。
- 拆书记录写成可直接套用的原文模板。

## 训练任务

每次借鉴前写一张四栏表：原作功能、禁止保留、我的替换、最终差异。四栏填完再动笔。
