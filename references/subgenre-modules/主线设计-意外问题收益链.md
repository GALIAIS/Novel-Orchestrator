# 主线设计-意外问题收益链

依据案例：`batch-164` 案例 817。

## 读者承诺

让主角在完成原目标时遭遇意外问题，并通过解决问题获得超额收益，推动主线自然升级。

## 核心循环

```text
原目标 -> 意外问题 -> 伏笔合理化 -> 金手指解决 -> 原目标完成 -> 额外收益
```

## 开篇必备

- 主角原目标要具体。
- 意外问题不能随机，要有前文伏笔。
- 意外要围绕主角金手指设计。
- 结尾既完成原目标，也给额外收益。

## 期待来源

读者期待主角每次出门不只完成任务，还会因解决更大麻烦而赚到新资源和新矛盾。

## 断档风险

- 意外突兀，像作者硬拐。
- 原目标被意外挤没。
- 收益和金手指无关。
- 每卷意外无法接到下一卷。

## 训练任务

写三卷小纲，每卷包含原目标、意外问题、伏笔、金手指解法、额外收益和下一卷矛盾。
