# 核心卖点护栏-能力兑现与书名契约

依据案例：`batch-414` 案例 2070

## 读者承诺

标题、简介和前三章共同建立的核心卖点会被持续交付，而不是在作者需要制造困难时被临时关闭。能力型卖点通过对象、目标、组合、成本和对手适应产生变化；书名突出的人物保持足以影响关键选择的中心性。作品允许失败、限制和转型，但限制来自既有规则，转型先交付新承诺并完整结算旧承诺。

## 核心循环

```text
把包装和开篇压成一句可验证卖点 -> 设定每个单元的最低兑现节拍 -> 用对象、目标、组合、成本和社会反应重复交付 -> 对手依据既有规则学习反制 -> 主角以更聪明或更昂贵的方式继续使用核心资产 -> 禁止临时关能力或撤销包装中心性 -> 必须转型时先展示新乐趣并结算旧线 -> 用目标读者期待复述和流失点校准护栏
```

护栏保护的是读者来此阅读的理由，不是让主角永不失败。最有效的困难会迫使核心资产产生新用法，而不是让它暂时不存在。

## 开篇必备

- 标题造成的第一联想、简介明示和前三章实证。
- “目标读者反复来看主角用 X，以 Y 获得 Z 情绪”的卖点句。
- 每章或每单元的最低兑现节拍。
- 对象、环境、目标、组合和成本五类变化轴。
- 世界规则内可以成立的范围、精度、资源或附带伤害限制。
- 对手观察能力后可采取的真实反制。
- 两项以上未经铺垫不得使用的禁止否定。
- 书名中心人物影响关键选择的最低频率。
- 旧承诺退出、新承诺接棒和旧线结算条件。

## 期待来源

- 熟悉能力在新目标下会产生何种陌生用法。
- 对手怎样学习而非靠作者临时改规则。
- 主角为继续兑现核心能力会支付什么。
- 能赢战斗的能力为何无法直接解决关系或制度。
- 包装中心人物将在何处改变主角选择。
- 新阶段如何升级而非否定原有乐趣。
- 一次失败会给下一次兑现增加何种资产。

## 断档风险

- 为制造危机突然宣布核心能力在该环境无效。
- 临时限制与此前展示的规则直接冲突。
- 主角失去能力后只等待外力恢复，没有残余策略。
- 标题突出角色只负责吸引点击，正文迅速边缘化。
- 用“作者没承诺”否认包装已经造成的稳定联想。
- 每章重复同一展示，没有对象、成本和后果变化。
- 以平台或读者标签为逻辑漏洞开脱。
- 撤走旧卖点后才开始解释新方向。
- 只统计评论骂声，不测目标读者实际期待复述。

## 训练任务

为一个能力型开篇写十章兑现表，每章至少标出展示、升级、组合、代价或社会反应中的一项。设计资源、保护目标和对手适应三种规则内限制，禁止让能力直接失效；列出两条禁止否定。若书名突出具体人物，让她至少三次改变关键决策，并写出转型时新旧承诺的交接场景。
