# 开篇金手指来源-压缩入事件

依据案例：batch-271 案例 1352

## 读者承诺

开篇快速让读者看到主角“用能力改变局面”，而不是长篇解释能力来历。金手指来源可以有，但必须服从第一章事件、爽感和追读钩子。

## 核心循环

```text
来源设定 -> 压缩必要信息 -> 外部事件袭来 -> 主角主动使用能力 -> 爽点/优势兑现
```

## 开篇必备

- 第一章要有明确外部事件，不只写苏醒、疼痛、回忆。
- 主角必须尽快从被动承受转成主动选择。
- 金手指第一次使用要解决一个看得见的问题。

## 期待来源

读者期待的是“这个能力能带来什么优势”。来源越长，期待越容易断；来源越快转成事件，读者越容易确认书的爽点。

## 断档风险

- 连续数章都在讲金手指怎么来的。
- 主角长期被实验、污染、敌人推动，缺少主动性。
- 危机很多，但没有爽感兑现。
- 设定解释完整，故事吸引力反而不足。

## 训练任务

把一个复杂来源压缩成四段话，再写一个主角用能力解决外部麻烦的第一章事件。
