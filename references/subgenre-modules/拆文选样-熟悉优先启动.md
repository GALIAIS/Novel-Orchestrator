# 拆文选样-熟悉优先启动

依据案例：`batch-033` 案例 161。

## 读者承诺

拆文先选自己熟悉且能拆动的榜单样本，别卡在找最完美书单上。

## 核心循环

```text
明确问题 -> 榜单取样 -> 熟悉优先 -> 开始拆 -> 输出结论
```

## 开篇必备

- 当前要学的问题。
- 目标平台榜单。
- 自己熟悉的候选书。
- 前 3 章可拆点。

## 期待来源

- 降低启动焦虑。
- 熟悉作品更容易看出节奏。
- 天榜样本自带市场验证。
- 拆得动比样本完美更重要。

## 断档风险

- 一直找书单，不开始拆。
- 硬拆完全看不懂的作品。
- 目标问题不明确。
- 拆书变成收藏样本。

## 训练任务

从天榜选 3 本熟悉作品，每本只拆前 3 章的目标、冲突、爽点和章末钩子。
