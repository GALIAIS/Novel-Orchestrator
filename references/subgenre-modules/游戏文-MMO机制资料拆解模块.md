# 游戏文-MMO机制资料拆解模块

依据案例：`batch-216` 案例 1078。

## 读者承诺

大型网游文要让职业、等级、装备、道具、任务等机制可信运转。

## 核心循环

```text
机制缺口 -> 模块拆分 -> 资料查询 -> 规则转化 -> 爽点冲突
```

## 开篇必备

- 拆出职业、等级、装备、道具、任务模块。
- 参考成熟玩家社区或资料工具。
- 把规则转成剧情选择。
- 考虑游戏文小众市场风险。

## 期待来源

读者相信这个游戏世界可玩、可升级、可协作，主角操作有规则依据。

## 断档风险

- 只堆游戏名词，不理解机制。
- 单机 RPG 经验硬套 MMO。
- 资料没有转成冲突和奖励。
- 小众题材缺少头部竞争意识。

## 训练任务

列职业表、等级表、装备表、道具表、任务链表，每表至少 10 条，并标注对应剧情冲突或爽点。
