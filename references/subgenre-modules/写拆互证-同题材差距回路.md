# 写拆互证-同题材差距回路

依据案例：`batch-080` 案例 398。

## 读者承诺

这个模块用于把拆书成果变成自己的写作能力。它承诺用“自己写、同题材对比、带问题再拆”的回路，避免只懂理论不会落笔。

## 核心循环

```text
拆书获得方法 -> 自己实写 -> 同题材同爽点对比 -> 发现差距 -> 带问题重拆
```

## 开篇必备

- 先有自己的正文或章纲。
- 对标样本必须同题材。
- 最好选择同类爽点或同类冲突。
- 对比只看少数关键项。
- 差距要反过来生成下一次拆书问题。

## 期待来源

- 理论能进入正文。
- 对标差距清楚。
- 拆书目标更窄。
- 修改任务更具体。
- 能力通过循环内化。

## 断档风险

- 一直拆别人，不写自己。
- 拆书问题太泛。
- 跨题材对比导致误判。
- 只说别人好，不说自己差在哪。
- 发现差距后不改写。

## 训练任务

拿自己 2000 字和同题材样本 2000 字做三栏表：期待来源、情绪推进、兑现方式。每栏只写一条差距，再据此重拆样本一次。
