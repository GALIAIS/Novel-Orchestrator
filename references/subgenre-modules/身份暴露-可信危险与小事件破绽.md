# 身份暴露-可信危险与小事件破绽

依据案例：batch-371 案例 1855

## 读者承诺

显性金手指普通化时，可以用身份暴露风险拉期待，但危险必须可信，最好由第一个小事件后的破绽引发。

## 核心循环

```text
小爽点 -> 行为异常 -> 身份破绽 -> 可信危险 -> 隐藏/周旋
```

## 开篇必备

- 身份秘密。
- 暴露后果。
- 读者能立刻相信的危险。
- 小事件制造破绽。

## 期待来源

- 主角解决问题。
- 解决方式暴露异常。
- 被识破风险。
- 与识破者斗智。

## 断档风险

- 穿越者暴露危险空口说。
- 新编规则没实践证明。
- 金手指普通但没有替代期待。
- 身份风险和主角行动无关。

## 训练任务

写一个穿越者开头：先给小爽点，再让行为留下身份破绽，引出一个读者立刻相信的暴露后果。
