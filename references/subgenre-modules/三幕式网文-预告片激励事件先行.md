# 三幕式网文-预告片激励事件先行

依据案例：`batch-086` 案例 427。

## 读者承诺

第一幕像预告片，先把最高能的激励事件、危机和金手指承诺摆出来；建置、导师和世界说明可以后置，第二幕展开，第三幕收拢。

## 核心循环

```text
高能预告 -> 激励事件抓人 -> 过程补建置 -> 多线展开 -> 中点收束 -> 集中爆发
```

## 开篇必备

- 第一章要有足够强的惊人意外或激励事件。
- 主角必须立刻有外部压力和内部动机。
- 金手指第一幕展示，并给出可升级承诺。
- 关键人物尽量第一幕出现，最迟中点前出现。
- 可后置导师、世界观说明和常规生活铺垫。

## 期待来源

- 主角能不能从激励事件中活下来或翻身。
- 金手指下一档会带来什么爽点。
- 第一幕埋下的多条线如何在第二幕展开。
- 第三幕能否把预告过的爽点集中兑现。

## 断档风险

- 死卡教科书步骤，第一幕过长。
- 先讲设定和导师，激励事件迟迟不到。
- 金手指只出现名称，没有可升级诱惑。
- 中点后继续加入核心人物，故事线无法收束。
- 预告片只有刺激，没有后续展开空间。

## 训练任务

把现有开篇按三幕功能重排：列出第一幕高能事件、金手指展示、可后置建置、第二幕展开线和第三幕集中爆发点。
