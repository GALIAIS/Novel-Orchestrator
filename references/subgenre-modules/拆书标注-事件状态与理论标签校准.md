# 拆书标注-事件状态与理论标签校准

依据案例：`batch-403` 案例 2012

## 读者承诺

把拆书从情节复述推进到四层标注：可核对证据、人物与读者状态差、带对象的功能标签、机制与失效边界；再用删除、换序和原创迁移检验理解。

## 核心循环

```text
按状态变化切单元 -> 记录动作与信息证据 -> 写人物和读者前后差 -> 添加带对象的功能标签 -> 提出机制与边界 -> 删除换序校验 -> 跨题材仿写验证
```

## 开篇必备

- 单元以目标、知识、能力、关系或风险变化为界。
- 证据层只写文本可核对事实。
- 人物状态与读者状态分开记录。
- “赌注、悬疑、共情”等标签写明具体对象。
- 每个机制保留替代解释和失效条件。
- 多章汇总问题链、风险链和兑现链。

## 期待来源

模糊好恶逐步变成有证据、可反驳、能迁移的结构判断；拆解者能说明一段为何有效，也能预测删改后会失去什么。

## 断档风险

- 逐段复述却没有状态差。
- 用理论名词替代文本证据。
- 标签不写服务对象。
- 把作者意图当作唯一解释。
- 表格完整却从不做反事实测试。
- 只能复刻原作名词和情节。
- 六章拆完仍没有整体问题递进。

## 训练任务

人工标注六个连续事件单元，每个只给一个主功能标签和两条证据；完成后做删除、换序测试，再以不同题材写一段原创场景验证同一机制。
