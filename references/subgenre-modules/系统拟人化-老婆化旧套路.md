# 系统拟人化-老婆化旧套路

依据案例：`batch-169` 案例 843。

## 读者承诺

系统可以有人格或形象，但它必须首先是规则机制和剧情发动机；伴侣化、消费化写法会让系统流从金手指承诺滑向过时旧套路。

## 核心循环

```text
系统人形化 -> 仍保留权限/规则/代价 -> 拒绝越权 -> 逼主角选择 -> 机制推动主线
```

## 开篇必备

- 先交代系统规则，再展示人格魅力。
- 人形系统要有权限边界，不能无条件满足主角。
- 拟人互动必须带任务、限制、误导或代价。
- 不要用伴侣消费替代金手指兑现。

## 期待来源

读者可以接受有性格的系统，但期待它让剧情更紧，而不是让主角停下来消费系统形象。

## 断档风险

- 系统变成女主/男主替代品，金手指机制失焦。
- 旧尺度、旧网感不适配当前平台环境。
- 开局重点落在私欲满足，主线目标迟迟不启动。
- 系统过度万能，主角选择压力消失。

## 训练任务

设计一个人形系统的第一章：必须写出一条不可违背的规则、一次系统拒绝主角、一次奖励二选一。禁止写无关主线的暧昧陪聊。
