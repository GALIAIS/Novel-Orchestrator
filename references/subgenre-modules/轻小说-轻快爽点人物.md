# 轻小说-轻快爽点人物

依据案例：`batch-023` 案例 111。

## 读者承诺

给读者轻松、短快、泛 ACG 或日常吐槽感，同时保留网文爽点和人物魅力。

## 核心循环

```text
轻松语气 -> 讨喜人物 -> 小冲突逼选择 -> 爽点/糖点兑现 -> 人物标签加深
```

## 开篇必备

- 主角视角清楚。
- 第一爽点或第一糖点清楚。
- 人物魅力标签清楚。
- 三章内有小情节闭环。

## 期待来源

- 轻松阅读降低门槛。
- 人物在选择中变鲜明。
- 泛 ACG 元素提供风味。
- 恋爱、吐槽、系统或日常事件持续给反馈。

## 断档风险

- 把轻小说当成不要爽点。
- 只做人设展示，不让人物做选择。
- 视角不清，读者误判主角。
- 用危险毒点伪装轻小说卖点。

## 训练任务

写一个轻小说开篇三章小纲，每章都标出一个爽点或糖点，以及一次能证明人物标签的选择。
