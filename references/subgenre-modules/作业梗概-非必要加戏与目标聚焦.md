# 作业梗概-非必要加戏与目标聚焦

依据案例：`batch-242` 案例 1207。

## 读者承诺

作业梗概要先完成题目指定的爽点，不让背景铺垫、旁支解释和见证人加戏稀释核心冲突。

## 核心循环

```text
题目目标 -> 冲突对象 -> 见证人 -> 能力对照 -> 爽点兑现
```

## 开篇必备

- 明确主角能力。
- 明确被打脸对象。
- 明确见证人功能。
- 删除不承担冲突的背景。

## 期待来源

读者期待看到主角用能力解决眼前轻视，而不是先读过多社交关系说明。

## 断档风险

- 叙述太散，作业目标不突出。
- 见证人抢走主冲突。
- 能力展示误伤友方角色。
- 背景解释压过爽点。

## 训练任务

把 8 点梗概压缩成 5 点，每点对应冲突、误解、见证、展示、兑现之一。
