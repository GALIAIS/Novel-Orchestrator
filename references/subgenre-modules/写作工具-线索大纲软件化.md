# 写作工具-线索大纲软件化

依据案例：batch-271 案例 1355

## 读者承诺

把码字工具和图示工具分工：正文工具负责持续产出，思维导图或图示软件负责线索大纲、人物关系、伏笔回收。工具存在的意义是减少断线，不是替代写作。

## 核心循环

```text
创作需求 -> 工具分工 -> 线索可视化 -> 章节回写 -> 正文持续推进
```

## 开篇必备

- 先定正文入口，保证每天打开就能写。
- 线索大纲只记录会影响章节的关系、伏笔和回收。
- 每条线索都要有出现章节和预计回收节点。

## 期待来源

作者端期待来自秩序感：线索清楚，卡文减少，伏笔不丢，章节执行更稳。读者端感受到的是结构更清晰、更新更稳定。

## 断档风险

- 沉迷整理工具，把大纲扩展当成写作进度。
- 图示越画越复杂，却无法回写正文。
- 多工具分散，资料找不到。
- 没有定期清理无效线索。

## 训练任务

建立“正文入口”和“线索大纲”两个文件。线索大纲每条都写出现章节、回收章节、影响人物，写不出来就删除。
