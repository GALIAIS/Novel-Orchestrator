# 直播文-通用技法套壳

依据案例：`batch-150` 案例 746。

## 读者承诺

直播文不是另起炉灶的写法，底层仍是爽点、剧情构架、人物设定。直播元素负责把事件放到镜头前，用弹幕和观众反馈放大情绪。

## 核心循环

```text
通用爽点 -> 直播场景 -> 主角行动 -> 观众即时反应 -> 弹幕放大 -> 后续关注
```

## 开篇必备

- 主角的主播身份或被直播处境。
- 一个可见的事件目标。
- 明确阻碍或对手。
- 观众能立刻理解的爽点。
- 弹幕只做反馈，不替代事件。

## 期待来源

读者期待主角在直播镜头下完成专业展示、装逼、打脸、救场或搞笑，旁观者即时反馈让爽点更响。

## 断档风险

- 只写弹幕热闹，没有真实事件。
- 把直播文当成全新体系，忽视基础结构。
- 主角人设不清，观众反应无从成立。
- 爽点本身弱，弹幕再多也撑不住。

## 训练任务

把一个普通打脸桥段改成直播文：先写原事件目标、阻碍、爽点，再添加镜头位置、观众误判、弹幕反应和事件后续热度。
