# 爽点机制-额外收入与求而不得

依据案例：`batch-208` 案例 1040。

## 读者承诺

额外收入型爽点不是单纯捡漏，而是让主角只求 A 却顺手拿走 B/C/D，让反派为 B/C/D 付出后全盘落空。

## 核心循环

```text
主角只求A -> 反派追B/C/D -> 冲突结算 -> 主角额外收入 -> 反派钱人目标皆失
```

## 开篇必备

- 主角原始目标必须清楚。
- 反派追求的资源要提前标价。
- 额外所得要对主角有用。
- 反派损失要有金钱、人员或目标清算。

## 期待来源

读者看见主角付出有限却收获翻倍，同时看见反派投入巨大却损失惨重，获得收益差和惩罚差的双重爽感。

## 断档风险

- 额外收入没有提前价值铺垫。
- 反派损失太轻。
- 主角总靠被动捡漏，显得不主动。
- 同类桥段使用过频，收益贬值。

## 训练任务

设计一个拍卖、秘境或资源争夺场景，写出主角所求、主角额外所得、反派所求、反派下场四列表。
