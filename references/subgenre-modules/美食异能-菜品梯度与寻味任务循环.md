# 美食异能-菜品梯度与寻味任务循环

依据案例：`batch-395` 案例 1975

## 读者承诺

把品尝获得能力写成可持续的探索升级：普通菜建立小收益标尺，名菜提供专门能力，失传菜触发质变，找菜、取材和制作本身就是主线任务。

## 核心循环

```text
当前危机提出能力缺口 -> 获得菜谱或食材线索 -> 争取制作条件 -> 首次完整品尝 -> 能力跨过旧标尺即时破局 -> 副作用和下一道菜续接
```

## 开篇必备

- 第一种食物能激发食欲或探索欲，而非无准备的生理厌恶。
- 普通食物先展示可预测的小收益。
- 菜品等级由食材、工艺、稀有度等多项决定。
- 首次生效、品质阈值和冷却规则清楚。
- 第一项质变能力立即解决当前危机。
- 破局后出现资源竞争或能力副作用。

## 期待来源

读者既期待下一道菜是什么，也期待主角如何找到、完成并把能力用于眼前困境；美食发现和能力成长在同一事件中兑现。

## 断档风险

- 猎奇进食先制造主角入口厌恶。
- 只按价格决定能力，财富直接击穿系统。
- 菜品只是菜单展示，不产生事件。
- 奖励随机，缺少普通菜标尺。
- 获得能力后没有即时用途。
- 觉醒后全部风险消失。

## 训练任务

设计普通菜、工艺名菜、失传菜各三道，写清门槛、能力和限制；再将一道失传菜写成“线索—争取—制作—品尝—破局—副作用”三章任务链。
