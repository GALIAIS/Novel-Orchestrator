# 信息跳级-作者已知与读者未知

依据案例：batch-372 案例 1858

## 读者承诺

读者只会根据正文已经给出的信息理解剧情。作者脑内已知但正文未交代的因果，不能默认读者自动补齐。

## 核心循环

```text
检查读者已知 -> 找出缺失台阶 -> 用行动/反应补信息 -> 读者理解因果 -> 期待继续
```

## 开篇必备

- 主角当前目标。
- 行动的必要原因。
- 关键规则的最小展示。
- 读者判断局势所需的信息。

## 期待来源

- 因果清楚带来的跟读感。
- 读者能判断角色选择。
- 规则被事件证明。
- 信息逐步补足后的悬念。

## 断档风险

- 作者默认读者知道设定。
- 角色突然做决定但原因缺失。
- 规则只在作者脑内成立。
- 用大段解释补救，拖慢节奏。

## 训练任务

拿一段自己写的转折，分别列出“作者已知”和“读者已知”，把缺失信息改成行动、环境反馈或他人反应。
