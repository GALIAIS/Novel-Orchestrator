# 资源剧情-牵强断链拉片修复

依据案例：`batch-125` 案例 625。

## 读者承诺

资源剧情卡住时，不要继续硬凑资源。先查因果链：需求、来源、阻碍、选择、后果、下一需求是否连上。

## 核心循环

```text
资源需求 -> 资源来源 -> 阻碍出现 -> 主角选择 -> 代价后果 -> 下一资源需求
```

## 开篇必备

- 主角缺什么要明确。
- 资源来源不能凭空出现。
- 阻碍要自然挡住目标。
- 主角选择要带代价。
- 后果要接出下一步。

## 期待来源

- 读者理解主角为什么要资源。
- 阻碍让获取过程有戏。
- 选择代价带来情绪。
- 下一需求让剧情不断链。

## 断档风险

- 只围绕资源名词写，没有事件链。
- 剧情牵强，情节接不上。
- 分不清剧情尬还是笔力尬。
- 不找样本拉片，闭门硬憋。

## 训练任务

把卡住的 2000 字拆成六格：需求、来源、阻碍、选择、后果、下一需求。缺格就补；满格但难看就先写到 3 万字再修。
