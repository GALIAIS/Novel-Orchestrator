# 灵草好感-金手指早兑现

依据案例：`batch-133` 案例 662。

## 读者承诺

让读者在前三章明白系统规则、奖励价值和主角短期目标。灵草好感的趣味必须尽早变成可见奖励，而不是只停留在设定解释。

## 核心循环

```text
系统明牌 -> 攻略灵草 -> 好感达标 -> 获得有价值奖励 -> 用奖励脱困/装逼
```

## 开篇必备

- 书名和简介直给卖点。
- 系统名不抢戏。
- 第一章说明好感奖励规则。
- 第一份奖励价值要明确。
- 奖励尽快用于剧情。

## 期待来源

- 灵草也能攻略的新鲜感。
- 好感条带来的短期目标。
- 奖励从低到高升级。
- 主角用人无我有的能力破局。

## 断档风险

- 金手指半遮半掩。
- 奖励太廉价或价值没写清。
- 段落过长，慢感被放大。
- 书名不够标题党，入口弱。

## 训练任务

重写一个系统文开篇入口：3 个书名、1 版简介、1 个系统名、100 字内金手指规则，以及第一份奖励的使用场景。
