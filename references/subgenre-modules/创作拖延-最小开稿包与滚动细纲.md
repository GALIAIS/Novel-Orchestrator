# 创作拖延-最小开稿包与滚动细纲

依据案例：`batch-408` 案例 2036

## 读者承诺

用有退出条件的最小开稿包替代“全部想好再写”，再以限时探索稿获得真实证据、按场景或三章回填必要细纲，使规划与正文形成短周期闭环。

## 核心循环

```text
识别无限准备目标 -> 压成首个可验证单元 -> 限时完成最小开稿包 -> 写探索场景 -> 记录发现与结构债 -> 回填下一检查点细纲 -> 完成局部兑现 -> 滚动进入下一轮
```

## 开篇必备

- 一句核心读者承诺。
- 主角当前欲望和第一阻碍。
- 首个不可逆选择。
- 前三章局部回报。
- 暂定终局方向和三条硬规则。
- 准备、写作与复盘各自的结束条件。

## 期待来源

启动阻力被压到一场戏后，作者每天能看见稿件增长；真实正文持续暴露下一项最值得解决的问题，规划因此从无限想象变成定向回填。

## 断档风险

- 用“感觉准备好了”作为开稿标准。
- 把所有世界设定都列为前置条件。
- 以纯即兴逃避结构决策。
- 探索稿没有字数和复盘出口。
- 每次回填都重做已经兑现的承诺。
- 用懒惰标签代替具体障碍诊断。

## 训练任务

用 90 分钟完成最小开稿包、800 至 1500 字探索场景和十分钟复盘；只记录三个发现、两个结构洞和下一场唯一目标，次日据此滚动细纲。
