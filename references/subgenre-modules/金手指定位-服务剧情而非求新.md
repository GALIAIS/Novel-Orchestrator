# 金手指定位-服务剧情而非求新

依据案例：batch-353 案例 1761

## 读者承诺

金手指的任务不是让作者显得新颖，而是让主角在具体困境中产生新的行动路线。读者期待的是能力如何解决眼前矛盾、留下限制、打开下一阶段目标。

## 核心循环

```text
剧情困境 -> 能力嵌入 -> 当场验证 -> 留下限制/新问题 -> 下一阶段目标
```

## 开篇必备

- 一个清楚的主角困境。
- 一个能被能力改变的行动选择。
- 一次可见验证。
- 一个能力无法完全解决的限制。

## 期待来源

- 老能力在新困境中的新用途。
- 能力解决一半、制造一半问题。
- 主角主动使用能力，而不是能力自动替他赢。
- 拆非主流强书获得拉期待方式。

## 断档风险

- 为新而新，能力和剧情脱节。
- 开头只有设定展示，没有矛盾。
- 能力太万能，后续没有目标。
- 只看榜单趋势，越看越焦虑。

## 训练任务

用同一个“每日抽奖系统”，分别服务都市、玄幻、悬疑三个开局困境，每个写 150 字验证场景。
