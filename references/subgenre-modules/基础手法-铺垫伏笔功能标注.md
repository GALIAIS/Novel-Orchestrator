# 基础手法-铺垫伏笔功能标注

依据案例：batch-347 案例 1732

## 读者承诺

把铺垫、伏笔、钩子、误导等表现手法从概念变成文本标注和后文回收检查。

## 核心循环

```text
前置信息 -> 当下读者感受 -> 后文使用方式 -> 可信/回收/反转效果 -> 修正信息功能
```

## 开篇必备

- 明确每条前置信息的主功能。
- 铺垫要服务后续可信度。
- 伏笔要有可回收线索。
- 钩子要制造继续阅读问题。
- 误导要能在反转后解释得通。

## 期待来源

读者回看时能感觉“原来早有痕迹”，而不是作者临时补丁。

## 断档风险

- 只背定义，不做文本标注。
- 把所有提前信息都叫伏笔。
- 伏笔没有回收。
- 铺垫太明显变成剧透。
- 无效信息伪装成高级设计。

## 训练任务

拆一章文本，给每条提前信息标注功能和回收位置。没有回收的伏笔，要么删除，要么补回收。
