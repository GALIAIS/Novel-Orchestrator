# 资料求助-冷门书PDF证据边界

依据案例：`batch-313` 案例 1561

## 读者承诺

为研究型创作提供资料链路管理：当作者需要冷门书、民俗书、宗教史、怪谈资料但暂时拿不到原文时，把求助、封面、版本、外链和未验证内容分层记录，避免把“想找资料”误当作“已经掌握资料”。

## 核心循环

```text
题材缺口 -> 锁定书名/版本 -> 获取受阻 -> 记录证据边界 -> 等原始资料验证 -> 再转入设定
```

## 开篇必备

- 先写清楚资料要解决哪个创作问题。
- 标注书名、作者、译者、出版社、版本线索。
- 区分已读内容、图片可见内容、社群转述、未验证猜测。
- 未读原书前，不把观点写入世界观硬设定。

## 期待来源

作者获得的是研究秩序感：即使资料暂时不可得，也知道下一步查什么、哪些内容不能用、哪些设定需要等待证据。

## 断档风险

- 把求助帖当资料正文。
- 只凭封面、书名或评论玩笑生成设定。
- PDF 来源不明，版本和译者不清。
- 没有记录未验证状态，后续误用。

## 训练任务

建立一个“待查资料三栏表”：资料名、当前证据、进入设定前必须补查的内容。
