# 结构关联-逻辑时间混合

依据案例：`batch-014` 案例 067。

## 读者承诺

让长篇故事既紧凑又不累，靠的是在一级结构之间合理切换逻辑关联、时间关联和混合关联。

## 核心循环

```text
场景有效 -> 一级结构闭环 -> 逻辑线推进 -> 时间线舒缓 -> 新逻辑线开启
```

## 开篇必备

- 每个场景要有人物、情节、环境。
- 每个场景至少推进期待或爽点。
- 一级结构之间要标明连接方式。
- 连续逻辑线要适时用时间关联舒缓。

## 期待来源

- 逻辑关联让读者期待因果结果。
- 时间关联让读者在主角连续行动中放松。
- 混合关联既接因果又保持现场连续。
- 远距离回收用剧情提醒读者旧因。

## 断档风险

- 场景没有期待点或变化，变流水账。
- 逻辑关联太多，读者必须记住太多前文。
- 时间关联太多，故事松散无骨架。
- 隔太久回收旧线，没有提醒，读者忘了。

## 训练任务

把 8 个一级结构排成表，标注逻辑/时间/混合关联。连续逻辑超过 5 个时，插入一次时间关联缓冲。
