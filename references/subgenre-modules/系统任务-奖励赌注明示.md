# 系统任务-奖励赌注明示

依据案例：`batch-119` 案例 591。

## 读者承诺

系统流开篇要让读者相信主角“为什么听系统”和“凭什么变强”。任务只是入口，奖励、拒绝代价和现实困境才是读者愿意跟着看的赌注。

## 核心循环

```text
现实困境 -> 系统任务 -> 奖励/代价明示 -> 主角主动选择 -> 奖励兑现 -> 困境上行
```

## 开篇必备

- 主角当前困境要清楚。
- 系统第一次任务要和困境相关。
- 奖励必须具体到能改变处境。
- 拒绝或失败的代价要能解释服从动机。
- 下行铺垫要短，尽快进入上行。

## 期待来源

- 读者期待奖励如何兑现。
- 读者期待主角拿奖励反击谁。
- 风险让系统任务不是白送好处。
- 每次任务都能暴露下一层主线。

## 断档风险

- 系统只发布任务，不提供有效奖励。
- 主角无理由听系统安排。
- 下行情节写太细，压过反击爽感。
- 任务和主线冲突脱节。

## 训练任务

写一张系统任务卡：任务、奖励、拒绝代价、主角选择、第一次反击对象各一句。确认奖励能在 1500 字内产生可见效果。
