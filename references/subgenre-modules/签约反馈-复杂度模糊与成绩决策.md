# 签约反馈-复杂度模糊与成绩决策

依据案例：`batch-251` 案例 1251。

## 读者承诺

签约反馈模块把“太复杂”这种模糊意见转成继续写、观察成绩、再决定是否重开的流程。

## 核心循环

```text
模糊反馈 -> 追问复杂来源 -> 已签约先执行编辑建议 -> 上推荐看成绩 -> 数据决策
```

## 开篇必备

- 记录编辑原话。
- 拆分设定、剧情、人物、表达四类复杂度。
- 明确编辑建议的下一步。
- 设置成绩观察节点。

## 期待来源

作者不被一句模糊负反馈打乱，而是用连载和推荐数据获得真实市场反馈。

## 断档风险

- 反馈模糊就立刻重开。
- 自己脑补所有问题。
- 已签约项目没等数据就放弃。
- 只听鼓励，不设复盘节点。

## 训练任务

写一个签约后观察表：编辑反馈、可疑复杂点、继续写目标、推荐数据、重开触发线。
