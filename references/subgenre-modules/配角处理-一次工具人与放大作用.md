# 配角处理-一次工具人与放大作用

依据案例：`batch-325` 案例 1622

## 读者承诺

配角鸡肋时，不是机械删除，而是判断他该做一次性工具人、被合并、放大功能，还是暂时离开主角队伍。

## 核心循环

```text
配角占字数 -> 功能弱 -> 判断角色类型 -> 一次用完/合并/放大/调离 -> 节奏变紧
```

## 开篇必备

- 标明配角当前承担的信息、冲突、战力或情绪功能。
- 一次性工具人可以只出现一次。
- 协战小弟必须有持续参与事件的作用。
- 后期主要配角前期可以暂时离队。
- 角色出场频率要匹配功能强度。

## 期待来源

读者期待跟随主角的角色能参与事件，而不是被反复提一嘴。

## 断档风险

- 主要配角后期才有用，前期却一直陪跑。
- 工具人被迫长期维护。
- 同伴不参与冲突，只消耗字数。
- 作者舍不得删，导致框架越来越肿。

## 训练任务

给一个鸡肋配角写三种方案：只出现一次、合并到已有角色、放大成协战小弟；选最省节奏的一种。
