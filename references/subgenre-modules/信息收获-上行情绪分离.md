# 信息收获-上行情绪分离

依据案例：`batch-066` 案例 328。

## 读者承诺

这个模块用于判断上行下行和情绪折线。它承诺的是把“获得信息的事件功能”和“角色得知后的情绪反应”分开，避免把坏消息自动判成下行。

## 核心循环

```text
当前目标 -> 主动行动 -> 获得信息 -> 判断条件增加 -> 情绪反应另开节点
```

## 开篇必备

- 先确认角色当前目标。
- 判断信息是否帮助目标。
- 获得信息和情绪反应分节点。
- 主角主动行动要单独计入上行。
- 坏消息不等于结构下行。

## 期待来源

- 信息缺口被填补。
- 主角处境变清晰。
- 新判断带来新选择。
- 后续情绪产生新拉扯。
- 读者理解事件功能。

## 断档风险

- 把消息好坏等同上行下行。
- 当前目标不清就判断情绪。
- 后续怅然若失混进信息收获。
- 不拆小单元。
- 忽略主角主动行动带来的上行。

## 训练任务

找一个角色得知坏消息的段落，拆成“获得信息”和“情绪反应”两个节点。分别写它们对当前目标的作用。
