# 拉片台词-检索片段取材

依据案例：`batch-063` 案例 312。

## 读者承诺

这个模块用于拉片、影视取材、台词和镜头学习。它承诺的是用台词或画面关键词快速定位片段，再把片段拆成信息、情绪、转折和人物关系，而不是凭记忆空想或直接搬运台词。

## 核心循环

```text
写作问题 -> 关键词检索 -> 定位片段 -> 查看上下文 -> 拆功能 -> 改写成网文表达
```

## 开篇必备

- 先确定要解决的写作问题。
- 用台词、画面、动作或情绪关键词检索。
- 必须查看片段前后文。
- 只记录结构功能，不抄原句。
- 标注不能照搬的版权内容。

## 期待来源

- 快速找到可学习片段。
- 台词上下文帮助理解人物关系。
- 画面搜索帮助拆场景调度。
- 片段功能能迁移到章节设计。
- 多样样本能避免闭门造车。

## 断档风险

- 收藏工具但不拆功能。
- 只截一句台词，忽略上下文。
- 把影视镜头硬搬进网文。
- 直接复制台词或桥段。
- 检索关键词太泛，无法服务当前问题。

## 训练任务

用一个关键词找 3 个影视片段。每个片段只写四项：人物关系、情绪变化、转折点、可迁移到网文的功能。
