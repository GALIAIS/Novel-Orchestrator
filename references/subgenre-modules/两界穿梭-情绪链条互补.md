# 两界穿梭-情绪链条互补

依据案例：`batch-076` 案例 378。

## 读者承诺

这个模块用于两界穿梭、分身、未来信息流和双世界资源流。它承诺两界不是随便切地图，而是用资源、信息、权限和矛盾互相推动，并尽量保持情绪链条接近。

## 核心循环

```text
副世界获取资源/信息 -> 主世界兑现优势 -> 主世界改变反馈副世界 -> 权限/局势升级 -> 下一轮穿梭
```

## 开篇必备

- 两界各自有明确噱头。
- 两界受众口味不能割裂太严重。
- 至少一界提供资源或信息。
- 另一界负责兑现爽点。
- 两界之间要有反馈循环。

## 期待来源

- 资源差带来装逼。
- 信息差带来纠错和改命。
- 当前行动改变未来评价。
- 权限提高带来更多情报。
- 两界共同主线让读者不只想看一边。

## 断档风险

- 两界情绪味道差太大。
- 副世界只是刷资源，没有剧情吸引。
- 主世界兑现太弱，资源没爽点。
- 读者只想看其中一界。
- 支线占比过高，主线被稀释。

## 训练任务

写一个两界三轮循环。每轮必须包含副世界获得、主世界兑现、反馈升级，并标明两界情绪链条是否同类。
