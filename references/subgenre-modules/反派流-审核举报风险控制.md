# 反派流-审核举报风险控制

依据案例：`batch-070` 案例 348。

## 读者承诺

这个模块用于反派流、黑暗流和高争议主角设计。它承诺的是保留反派身份、抢夺和杀伐爽点，同时避免把恶行正当化成公共价值反面，降低举报和审核风险。

## 核心循环

```text
反派身份卖点 -> 识别价值导向风险 -> 删除正当化恶行 -> 保留爽文行为 -> 避开举报点
```

## 开篇必备

- 区分反派身份和反社会价值。
- 不把自私自利包装成真性情。
- 不贬低牺牲、公共责任和正向价值。
- 避免现实政治或组织映射。
- 高争议桥段提前降风险。

## 期待来源

- 抢机缘带来爽点。
- 腹黑和杀伐果断带来效率感。
- 反派身份提供差异。
- 风险控制保证连载安全。
- 争议不压过故事本身。

## 断档风险

- 认为“反派文”天然安全。
- 人气越高越树大招风。
- 粉丝吹捧引发对立和举报。
- 敏感期仍写高风险表达。
- 后期修改救不回前期价值导向。

## 训练任务

写一个反派主角夺宝桥段，列出 5 个可能被举报的表达，再改成低风险版本：保留夺宝结果，删除价值观反向正当化。
