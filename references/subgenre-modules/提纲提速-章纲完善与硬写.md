# 提纲提速-章纲完善与硬写

依据案例：batch-292 案例 1459

## 读者承诺

先用简单提纲降低临场决策，再边写边完善章纲，卡文时看情节点硬写。

## 核心循环

```text
简单提纲 -> 写作暴露问题 -> 补全章纲 -> 情节点可视化 -> 卡文硬写 -> 速度提升
```

## 开篇必备

- 本章目标。
- 角色欲望。
- 冲突点。
- 三个情节点。
- 结尾钩子。
- 写后补充问题。
- 用时和字数记录。

## 期待来源

作者不用边写边决定所有剧情，写作阻力下降，产出更稳定。

## 断档风险

- 无提纲全靠临场发挥。
- 提纲一次性求完美。
- 章内情节点不清。
- 卡文时不回看计划。

## 训练任务

写下一章 7 行章纲，完成正文后回填哪里卡住、提纲该补什么。
