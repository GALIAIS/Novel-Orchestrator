# 新人仙侠-穿越系统与低位面杠杆

依据案例：batch-383 案例 1912

## 读者承诺

穿越加系统能为新人快速提供共同视角、信息差、目标和显性反馈。土著天赋流同样可写，但要用更强执行补足这些功能。

## 核心循环

```text
穿越信息差 -> 系统目标 -> 低位面优势 -> 阶段反馈 -> 升级滚雪球
```

## 开篇必备

- 主角与读者的共同视角。
- 可解释的初始优势。
- 清楚的系统目标和反馈。
- 去掉系统后的替代机制。

## 期待来源

- 读者迅速看懂领先原因。
- 系统给出短期目标。
- 反馈形成升级刻度。
- 优势在更高层世界继续变化。

## 断档风险

- 把常见组合误当签约硬规则。
- 结构简单就降低完成质量。
- 土著流缺少目标和反馈替代物。
- 只幻想成绩，不做投稿验证。

## 训练任务

为同一创意写穿越系统版和土著天赋版三章纲，比较目标、反馈、解释成本和优势来源后再选择。
