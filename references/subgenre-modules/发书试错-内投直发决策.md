# 发书试错-内投直发决策

依据案例：`batch-037` 案例 181。

## 读者承诺

发书路径是试错工具，不是成绩保证。直接发省事，内投成本低且标准高，保底和分成目标不同，选择也不同。

## 核心循环

```text
目标网站 -> 选择内投/直发 -> 控制字数成本 -> 看反馈 -> 继续/切书/复盘
```

## 开篇必备

- 先分网站。
- 明确分成还是保底。
- 写清准备字数。
- 设置继续和切书条件。

## 期待来源

- 作者知道什么时候判断。
- 内投用低字数测试。
- 直发用站短和数据验证。
- 失败能回到下一本修正。

## 断档风险

- 一稿多投。
- 把签约当终点。
- 五万字无反馈还硬拖。
- 只问路径，不复盘开篇问题。

## 训练任务

为一本新书写内投和直发两套试错计划：成本、观察节点、继续条件、切书条件必须齐全。
