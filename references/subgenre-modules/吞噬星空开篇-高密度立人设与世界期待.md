# 吞噬星空开篇-高密度立人设与世界期待

依据案例：batch-353 案例 1762

## 读者承诺

强开篇可以在一章内同时立主角、立关系、立敌人、立世界和立目标。信息密度来自多视角事件证明，不来自作者直接解释。

## 核心循环

```text
日常事件 -> 路人/朋友/敌人多视角证明主角 -> 小危机展示能力 -> 大世界露出 -> 阶段目标落地
```

## 开篇必备

- 两个贯穿事件。
- 至少三个视角承担不同证明功能。
- 一个阶段挑战，如考试、考核、任务。
- 一个打开世界尺度的小危机。

## 期待来源

- 主角优秀但仍受出身和目标压力限制。
- 敌人恨意有因，后续报复必然。
- 日常世界突然露出超凡尺度。
- 阶段目标连接长期改命。

## 断档风险

- 多视角只是换镜头，没有承担证明功能。
- 世界设定用解释堆出，不由危机展示。
- 敌人只会嘲讽，缺少行动必然性。
- 主角没有升级意愿，目标不聚焦。

## 训练任务

设计一章“学院放学”开头，必须用路人、朋友、敌人三种视角分别证明主角的口碑、真实实力和潜在冲突。
