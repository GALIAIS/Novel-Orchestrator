# 古代军衔-资料外链证据卡

依据案例：batch-301 案例 1502

## 读者承诺

古代军衔和军制资料要先做来源卡，再转成读者能理解的等级差、军功和调兵权限。

## 核心循环

```text
资料入口 -> 核对朝代/体系 -> 抽取层级 -> 设计军功升级 -> 正文少量使用
```

## 开篇必备

- 记录来源、链接、提取码或书名。
- 标明适用朝代。
- 只抽当前剧情需要的三到五级。
- 称谓服务权力差，不堆名词。

## 期待来源

- 军职让上下级压迫清楚。
- 军功升级带来身份反馈。
- 调兵权限成为真实爽点。

## 断档风险

- 不分朝代混用军衔。
- 正文堆资料，读者看不懂。
- 未读资料却假装已考据。

## 训练任务

做一张军衔资料卡：来源、适用范围、三层军职、一个主角升迁事件。
