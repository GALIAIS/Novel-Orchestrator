# 签约执行-保底流程沟通

依据案例：`batch-092` 案例 456。

## 读者承诺

保底签约卡流程时，不靠群友猜测。把时间节点和卡点整理清楚，直接问编辑，确认材料、审核环节和预计完成时间。

## 核心循环

```text
提签 -> 等待超时 -> 记录节点 -> 询问编辑 -> 确认卡点 -> 补材料/继续等
```

## 开篇必备

- 记录提起电子签约时间。
- 明确已提交材料。
- 问当前在哪个流程节点。
- 问预计完成时间。
- 保存沟通记录。

## 期待来源

- 流程不确定性降低。
- 作者知道是否需要补材料。
- 签约时间有预期。
- 后续催办有依据。

## 断档风险

- 群里猜流程。
- 编辑交流障碍却不整理问题。
- 不留聊天记录。
- 合同问题凭经验判断。

## 训练任务

写一条 150 字内的签约进度询问，包含提签时间、等待天数、当前状态、是否补材料、预计完成时间。
