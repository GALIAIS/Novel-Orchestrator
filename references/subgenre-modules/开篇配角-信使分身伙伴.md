# 开篇配角-信使分身伙伴

依据案例：`batch-069` 案例 341。

## 读者承诺

这个模块用于开篇背景介绍、主角关系网和配角功能设计。它承诺的是让配角不只是讲解设定，而是同时承担信使、资源、互动、背锅和主角分身功能。

## 核心循环

```text
主角状态 -> 配套配角 -> 对话出信息 -> 配角承担功能 -> 转伙伴或形成克制
```

## 开篇必备

- 配角类型与主角出身匹配。
- 对话同时展露信息和关系。
- 配角提供资源或行动理由。
- 不符合主角逻辑的行为交给配角。
- 配角后续去留提前判断。

## 期待来源

- 配角降低说明感。
- 死党或亲人快速建立生活面。
- 配角小目标制造互动和冲突。
- 主角分身让台词和戏更好写。
- 信使转伙伴形成长期关系期待。

## 断档风险

- 配角只负责解释设定。
- 配角私事不服务主线。
- 主角和配角功能重复。
- 配角犯错后主角没有收获。
- 配角中期消失但前期承诺太重。

## 训练任务

给主角设计一个“信使型配角”，写清他带来的信息、资源、麻烦和后续伙伴路线。再写一个版本，让他的私事短暂克制主角大目标。
