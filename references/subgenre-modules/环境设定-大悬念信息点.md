# 环境设定-大悬念信息点

依据案例：`batch-050` 案例 246。

## 读者承诺

开篇环境不是堆细腻描写，而是用大悬念、小环境压力和密集有效信息点，让读者知道世界已经乱了，主角还在日常里不自知。

## 核心循环

```text
大悬念 -> 小环境日常 -> 信息点铺设 -> 主角不自知 -> 读者担心 -> 后期转化爽点
```

## 开篇必备

- 先抛一个能覆盖全书基调的大悬念。
- 给主角一个有压力的小环境。
- 信息点必须推主线或立人设。
- 让读者比主角多知道一点危险。
- 删除不能转化的细腻描写。

## 期待来源

- 世界焦急与主角安逸形成反差。
- 读者替主角担心，会愿意继续看。
- 信息点后期转化为爽点，会产生布局感。
- 小环境压力能立住主角处境。

## 断档风险

- 只写细腻生活，没有大悬念，会显得平。
- 信息点不能转化，就是水。
- 开头基调过欢脱，会影响全书预期。
- 新人掌控不了信息点转化，容易写成慢热。

## 训练任务

写一个 1000 字开篇设计表：列出大悬念、主角小环境、10 个信息点，并标注每个信息点的后续用途。
