# 信息差-碎片谜团链

依据案例：`batch-012` 案例 059。

## 读者承诺

通过控制读者、角色、作者和势力之间的信息量差异，制造悬念、压迫、误导、反转和真相揭开后的释然感。

## 核心循环

```text
完整真相 -> 切碎片 -> 给入口 -> 收集碎片 -> 建立假说 -> 验证推翻 -> 揭示真相
```

## 开篇必备

- 先写完整真相，再切碎片。
- 明确谁知道、谁不知道、谁误会。
- 入口碎片要足够异常，让角色必须追查。
- 谜团要有未知性、可能性和压迫感。

## 期待来源

- 读者想知道电话、行为、物品、规则背后的真实原因。
- 角色不知道危险，读者替他紧张。
- 多个假说同时成立，读者等待验证。
- 真相带来巨大好处，或错误判断带来严重后果。

## 断档风险

- 只隐藏信息，不提供可推理碎片。
- 谜团没有压迫感，角色不查也没损失。
- 误导碎片无法回收，变成作者耍赖。
- 新手把开篇写成纯探索，缺少爽点和目标。

## 训练任务

把一个世界观设定改成谜团链：写完整真相、5 个碎片、1 个误导碎片、2 个错误假说、1 个验证动作和最终揭示收益。
