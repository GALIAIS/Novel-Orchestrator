# 章节断点-微目标闭合与后果牵引

依据案例：`batch-418` 案例 2087

## 读者承诺

每章都会完成一次有意义的推进，并留下由本次推进自然产生的下一问题。断点不由字数决定，也不靠机械截断动作制造假悬念；本章的微目标、选择或认知发生局部闭合，资源、关系、位置、风险或目标出现状态差，新后果再把读者推向下一章。

## 核心循环

```text
定义本章唯一主问题 -> 记录章初资源、关系、认知、位置和风险 -> 人物主动执行一个可完成动作单元 -> 阻碍迫使其选择或付出代价 -> 主问题获得答案、部分答案或不可逆改写 -> 至少一项状态发生可见变化 -> 章末显露由结果产生的新后果 -> 读者能说出下一章要验证的问题 -> 下一章直接承接后果
```

本章问题提供聚焦，微目标闭合提供获得感，状态差证明推进，新后果制造可信牵引，直接承接则避免重复结尾和伪悬念。

## 开篇必备

- 本章唯一主问题。
- 章初资源、关系、认知、位置和风险。
- 人物主动采取的动作单元。
- 阻碍和被迫支付的代价。
- 本章必须完成的微目标。
- 主问题如何被回答、部分回答或改写。
- 章末至少一项可见状态差。
- 动作完成、答案翻转、选择落定或代价到账的断点类型。
- 由本章结果产生的新后果。
- 下一章可用一句话表达的验证问题。

## 期待来源

- 本章人物能否完成明确的微目标。
- 阻碍会迫使人物作出什么选择。
- 行动结果会改变哪项状态。
- 得到答案后原问题是否发生翻转。
- 本章收益附带什么代价。
- 新后果为何要求人物继续行动。
- 下一章会如何直接处理本章结果。

## 断档风险

- 字数够了才临时寻找切点。
- 把任何故事强压成固定三章。
- 每章只有动作，没有状态变化。
- 连续在半句话、挥刀或开门处机械切断。
- 用信息密度替代因果重点。
- 一个大事件没有嵌套微目标。
- 章末补一句预告，却没有真实后果。
- 下一章重演上一章最后一幕。
- 所有章节都使用同一种悬崖钩子。

## 训练任务

将一个长事件拆成五个可独立完成的微目标，为每章填写章初状态、本章问题、主动动作、状态差、新后果和下一章问题。分别用动作完成、答案翻转、选择落定和代价到账写四个 300 字章末，禁止用字数到点、半句话或重复结尾作为唯一牵引。
