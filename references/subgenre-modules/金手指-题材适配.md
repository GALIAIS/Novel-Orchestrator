# 金手指-题材适配

依据案例：`batch-006` 案例 029。

## 读者承诺

金手指必须服务题材核心爽点，能持续制造该题材读者想看的互动、事件和收益循环。

## 核心循环

```text
题材核心 -> 互动需求 -> 金手指功能 -> 事件生产 -> 收益循环
```

## 开篇必备

- 明确题材读者想看什么。
- 金手指第一功能要能直接推动该题材事件。
- 收益要能循环，不只解决前期。

## 期待来源

- 同人读者期待原作人物反应。
- 文娱读者期待名气、作品、资源继续转化。
- 种田读者期待建设渐进推进。
- 全民/副本读者期待私人副本、杀敌、成长路径。

## 断档风险

- 金手指强但和题材不互动。
- 前期收益有用，中后期边际递减。
- 金手指把题材写偏。
- 同人文没有强互动，只剩主角自嗨升级。

## 训练任务

为一个题材写“核心爽点 -> 金手指功能 -> 三个连续事件 -> 收益循环”，检验适配度。
