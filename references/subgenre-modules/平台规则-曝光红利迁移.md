# 平台规则-曝光红利迁移

依据案例：`batch-071` 案例 352。

## 读者承诺

这个模块用于发书平台选择和新人曝光判断。它承诺的是把“旧红利还能不能用”先查清楚，再决定更新量、题材包装和是否换站。

## 核心循环

```text
旧规则 -> 平台改权重 -> 新人曝光变化 -> 调整发书策略 -> 小成本试错
```

## 开篇必备

- 确认当前曝光入口。
- 判断更新量是否仍能换曝光。
- 看头部和老书权重是否上升。
- 判断题材包装和质量权重。
- 准备备用平台或备用节奏。

## 期待来源

- 平台规则清楚后，试错成本下降。
- 新人知道哪里有曝光窗口。
- 更新策略不再靠旧经验。
- 题材包装匹配当前推荐机制。
- 平台变化可转成决策依据。

## 断档风险

- 只听过去经验。
- 把爆更当万能解。
- 忽略头部资源集中。
- 规则变了还按旧节奏发书。
- 不准备备用路径。

## 训练任务

给目标平台写一张发书规则卡：曝光入口、更新要求、字数节点、头部权重、新人风险。再写一条换站预案。
