# 诸天换图-爽点阈值接力

依据案例：`batch-081` 案例 403。

## 读者承诺

这个模块用于诸天流从低武、武侠换到仙侠或更高世界时防止爽点断档。它承诺让每次换地图都有更高阈值或更强期待的新优势接力。

## 核心循环

```text
当前爽点建立 -> 超额收益沉淀 -> 换地图削弱旧优势 -> 新体系差异 -> 新爽点覆盖
```

## 开篇必备

- 明确第一世界卖的核心爽点。
- 判断该爽点是主菜还是点缀。
- 提前埋一个可带走的超额收益。
- 下一世界必须有不同主流体系。
- 主角至少保留一个别人没有的优势。

## 期待来源

- 读者知道主角优势不会归零。
- 新世界有新的比较对象。
- 旧爽点变成背景收益。
- 新爽点阈值高于旧爽点。
- 主角离开地图有明确时机。

## 断档风险

- 唯我独法突然失效。
- 新爽点弱于旧爽点。
- 换地图后主角优势被普通化。
- 第一个世界直接写到满级，后续没法升。
- 读者追的核心承诺被换掉。

## 训练任务

写三个地图的爽点接力：每个地图写旧优势、带走收益、新世界主流体系、新优势、离开时机。新优势不能低于旧优势。
