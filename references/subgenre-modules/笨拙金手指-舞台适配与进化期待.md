# 笨拙金手指-舞台适配与进化期待

依据案例：batch-351 案例 1755

## 读者承诺

金手指可以看似笨拙，但舞台、时机和进化机制必须让它在关键处决定局势。

## 核心循环

```text
表面低效 -> 特殊舞台放大 -> 虚晃积累期待 -> 关键反转 -> 进化换用法
```

## 开篇必备

- 先说明表面缺点。
- 快速给出适配舞台。
- 主角用时机和策略弥补低效。
- 关键场景必须让能力决定胜负。
- 留出进化机制。

## 期待来源

读者期待主角如何把别人看不起的能力用成杀招。

## 断档风险

- 真无效，和没有一样。
- 舞台设计也笨拙。
- 只挨打，不反转。
- 复仇回报不对等。
- 一种用法写到头却不进化。

## 训练任务

给一个笨拙能力设计三层舞台：无用、可用、封神。每层写出缺点如何变优势。
