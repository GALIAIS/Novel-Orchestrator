# 集团瓦解-横向零件拆解

依据案例：`batch-028` 案例 139。

## 读者承诺

面对强大集团，主角不是正面硬撞，而是拆功能、找裂缝、逐个击破，让庞然大物一步步失灵。

## 核心循环

```text
集团强大 -> 功能拆分 -> 利益裂缝 -> 选择小切口 -> 策反/同化/击破 -> 影响扩大
```

## 开篇必备

- 集团强在哪里。
- 集团由哪些功能零件组成。
- 内部有哪些利益矛盾。
- 第一阶段只打一个可验证切口。

## 期待来源

- 读者知道敌人不是铁板一块。
- 主角用小行动撬动大结构。
- 一个零件失灵牵连更大层级。
- 组织内部互相猜忌放大成果。

## 断档风险

- 把集团写成单个反派。
- 主角一上来直接打总部。
- 零件拆了但没有利益裂缝。
- 大纲追求一次性完美，正文无法调整。

## 训练任务

把一个强大组织拆成信息、资源、执行三个零件，各写一个可策反对象和一个第一卷击破点。
