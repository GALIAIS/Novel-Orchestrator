# 创作习惯-日更执行环境与细纲

依据案例：`batch-200` 案例 996。

## 读者承诺

日更执行的核心不是硬靠意志，而是用细纲、存稿和环境隔离，让每天写作变成可重复完成的填充任务。

## 核心循环

```text
大纲细纲 -> 当天填充 -> 排除干扰 -> 存稿缓冲 -> 稳定更新 -> 追更习惯
```

## 开篇必备

- 先把当天章节切成 2-3 个场景。
- 每个场景有明确目标和爽点。
- 写作前排除网络、床、沙发等干扰。
- 用存稿承接状态波动。

## 期待来源

读者相信作品会稳定更新，作者也因为每天可完成而减少卡文焦虑。

## 断档风险

- 没细纲导致每天临时想剧情。
- 只靠鸡血，状态一差就断。
- 写作环境里干扰太多。
- 单日目标过大，没有拆成小段。

## 训练任务

做 7 天日更执行表：每天 2 个场景、1 个爽点、1 个章末钩子，并写清写作前的断干扰措施。
