# 分身扮演流-视角目标完成再切换

依据案例：batch-357 案例 1782

## 读者承诺

分身扮演流要让每个视角先完成一个清楚目标，再切换到下一个点。读者不是怕多视角，而是怕每个视角都没看明白就被切走。

## 核心循环

```text
分身登场 -> 明确小目标 -> 完成信息/误导/爽点功能 -> 过渡 -> 切换下个视角
```

## 开篇必备

- 当前分身身份。
- 当前视角目标。
- 目标完成标志。
- 下个视角承接点。

## 期待来源

- 多个分身分别推动不同势力。
- 每次切换都有阶段成果。
- 幕后黑手布局逐步显形。
- 视角过渡带来局势升级。

## 断档风险

- 一个点没写完就切。
- 多视角平均分配篇幅。
- 分身目标不明确。
- 读者不知道当前段落有什么用。

## 训练任务

设计 4 个分身视角，每个视角写一个任务和一个完成标志。未完成标志前禁止切换。
