# 子流派找书-平台标签与榜单样本

依据案例：`batch-245` 案例 1223。

## 读者承诺

研究子流派时先找平台标签和榜单样本，比只等别人推荐单本书更稳。

## 核心循环

```text
输入子流派关键词 -> 点平台下拉标签 -> 看榜单排名 -> 建样本池 -> 拆共性承诺
```

## 开篇必备

- 明确目标子流派。
- 选定目标平台。
- 记录下拉标签。
- 优先看排名靠前样本。

## 期待来源

平台已经把读者搜索和分类习惯沉淀成标签，作者可以顺着标签找到真实样本。

## 断档风险

- 只凭印象找书。
- 单本推荐代表性不足。
- 不看开篇承诺。
- 忽略平台分类方式。

## 训练任务

用平台下拉标签找一个子流派榜单，记录前 10 本书的开篇承诺和共同爽点。
