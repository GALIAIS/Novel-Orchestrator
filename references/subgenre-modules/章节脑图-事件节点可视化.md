# 章节脑图-事件节点可视化

依据案例：batch-275 案例 1375

## 读者承诺

用脑图把一章的中心事件、人物行动、因果箭头和冲突升级可视化，最后回写成正文小纲。

## 核心循环

```text
中心事件 -> 行动节点 -> 因果连接 -> 冲突推进 -> 正文小纲
```

## 开篇必备

- 中心节点写事件，不写抽象概念。
- 每个分支必须是人物动作和结果。
- 节点数量控制在能执行的范围内。

## 期待来源

作者期待章节不散、线索不断、冲突有推进。工具只负责看清因果，不能代替正文。

## 断档风险

- 节点都是设定名词，无法落笔。
- 分支太多，执行成本过高。
- 脑图不回写正文，变成整理幻觉。

## 训练任务

画 12 节点以内章节脑图，每个节点必须是“人物+动作+结果”，再按箭头顺序写成正文小纲。
