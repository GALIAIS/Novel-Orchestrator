# 危机开头-信息锚点与金手指规则

依据案例：batch-350 案例 1748

## 读者承诺

危机开头要快，但必须保留主角目标、地点缘由、人物关系和金手指规则，否则读者只会混乱。

## 核心循环

```text
危机入场 -> 最少信息锚定 -> 金手指稳定发挥 -> 主角利用规则破局 -> 读者期待
```

## 开篇必备

- 300 字内交代主角目标。
- 出场人物控制在可识别范围。
- 说明主角为何在此。
- 金手指主动/被动规则统一。
- 危机服务主角装逼或破局。

## 期待来源

读者既感到危险，又知道自己在跟谁、为什么、看什么能力。

## 断档风险

- 一上来就打，但不知道谁是谁。
- 人物太多。
- 缺少地点和目标锚点。
- 金手指规则前后矛盾。
- 为了快节奏硬塞信息。

## 训练任务

写 1000 字危机开头，人物不超过 3 个。前 300 字给目标和地点，前 700 字让金手指发挥一次。
