# 闪前钩子-最低锚点与下一章因果回收

依据案例：`batch-405` 案例 2022

## 读者承诺

章末先展示未来结果证据，但保留人物、动作和风险最低锚点，让读者能提出一个明确因果问题；下一章从前因走到同一节点并赋予旧细节新含义。

## 核心循环

```text
本章局部结果 -> 闪前给未来异常与锚点 -> 单一因果问题 -> 下一章正常时序推进 -> 到达节点但不机械重播 -> 新信息重释 -> 后果推出下一问
```

## 开篇必备

- 可识别的人物或视角。
- 一个具体动作和立即风险。
- 只拿走一项核心因果。
- 片段展示结果而非解决方法。
- 下一章在承诺距离内偿还信息债。
- 到达节点时新增解释而非原样重复。

## 期待来源

读者知道未来会发生什么，却不知道人物如何走到那里；正常时序中的每次选择都被未来结果重新加压，节点重合时产生顿悟。

## 断档风险

- 把闪前、倒叙和预告混为一谈。
- 身份、地点、术语和时间同时未知。
- 读者无法复述想问什么。
- 下一章整段复制而没有重释。
- 信息债拖太久且无时序信号。
- 用形式钩子替代人物与事件发动机。

## 训练任务

写无锚点、多未知、单问题三个闪前版本，让陌生读者各写一个问题；只保留问题高度一致的版本，再写下一章完成重释。
