# 日常爽点链-每章一爽

依据案例：`batch-143` 案例 715。

## 读者承诺

每章不只推进流程，还要把普通动作改造成即时爽点、人物认可、资源入口和未来回收点，让读者持续获得情绪反馈。

## 核心循环

```text
普通动作 -> 意外危机 -> 主角介入 -> 即时爽点 -> 新场景入口 -> 小反派埋线 -> 未来回收
```

## 开篇必备

- 原本普通动作。
- 意外危机。
- 主角出手理由。
- 即时奖励。
- 新资源或新场景。
- 小反派/规则压迫。

## 期待来源

- 一章至少一个可感知爽点。
- 日常也有事件入口。
- 配角认可主角价值。
- 小反派提供未来打脸期待。

## 断档风险

- 只积累人物和情节，不总结爽点。
- 过渡段没有情绪反馈。
- 新场景没有资源或冲突。
- 小反派只出现一次，不回收。

## 训练任务

连续拆 10 章同类型爽文，标出每章即时爽点和未来伏笔。把“主角去找兼职”改写成救美版、打脸版、资源入局版。
