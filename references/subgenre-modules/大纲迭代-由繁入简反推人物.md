# 大纲迭代-由繁入简反推人物

依据案例：`batch-071` 案例 355。

## 读者承诺

这个模块用于防止剧情写崩和角色犯病。它承诺的是先把大纲写全，再反复检查逻辑、信息差和人物动机，最后压缩成可执行卷纲。

## 核心循环

```text
脑洞 -> 长大纲写全 -> 多次阅读 -> 反推人物动机 -> 修剧情 -> 压缩卷纲
```

## 开篇必备

- 第一版允许啰嗦。
- 先把想写的事情交代清楚。
- 多次阅读找逻辑断点。
- 用大纲反问每个角色想要什么。
- 最后压缩成方便记忆和投递的版本。

## 期待来源

- 长大纲降低写崩风险。
- 信息差修正让剧情更顺。
- 人物动机反推新剧情。
- 简化后方便执行。
- 学习总结让试错不白费。

## 断档风险

- 大纲周期太长磨掉热情。
- 只有资料卡，没有剧情检验人物。
- 角色戏份不足却想塑造丰满。
- 只试错不总结。
- 嘲笑爆款但不研究算法和市场。

## 训练任务

写一个 1000 字长大纲，再压缩成 200 字卷纲。压缩前必须标出 3 个角色动机问题和 3 个由人物反推的新剧情。
