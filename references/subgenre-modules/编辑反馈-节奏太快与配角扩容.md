# 编辑反馈-节奏太快与配角扩容

依据案例：`batch-228` 案例 1139。

## 读者承诺

投稿后编辑反馈优先级高，节奏太快要拿大纲沟通，把补配角和扩后续变成可回收内容。

## 核心循环

```text
编辑反馈 -> 大纲沟通 -> 配角扩容 -> 节奏缓冲 -> 推荐机会
```

## 开篇必备

- 标明编辑反馈对应章节。
- 找出连续高强度主线节点。
- 补人物关系或配角视角。
- 每个补充场景后续必须回收。

## 期待来源

作者获得的不是自我猜测，而是符合平台读者和推荐机制的节奏修正。

## 断档风险

- 把补配角写成灌水。
- 不跟编辑具体沟通。
- 忽略编辑对推荐的影响。
- 节奏慢下来但主线断掉。

## 训练任务

拿前三万字大纲标出每章主线事件；在两个连续高强度事件之间补 1 个配角视角或关系推进场景，并写明后续在哪个冲突回收。
