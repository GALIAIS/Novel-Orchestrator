# 复杂小纲-伏笔显化与降难

依据案例：`batch-337` 案例 1682

## 读者承诺

复杂小纲里的伏笔、动机和环境规则必须显化；没写出来的逻辑，读者会按不存在处理。

## 核心循环

```text
标题承诺 -> 前三章因果 -> 硬理由显化 -> 反派合理反应 -> 金手指收益落地
```

## 开篇必备

- 标题承诺和正文承诺一致。
- 主角每个危险选择都有硬理由。
- 反派不降智。
- 关键伏笔在小纲或正文可见。
- 前期复杂度控制在可驾驭范围。

## 期待来源

读者期待在权谋、冒险和金手指成长里看到清楚因果，而不是靠作者事后补设定。

## 断档风险

- 关键信息藏在作者脑内。
- 拿到危险物不跑却没理由。
- 强势反派不查、不软禁、不灭口。
- 金手指机制太复杂但首章收益不清。
- 公开处刑只发小纲导致误判。

## 训练任务

把前三章压成因果表：目标、阻碍、硬理由、反派反应、金手指收益；所有需口头解释的点补进正文动作或对话。
