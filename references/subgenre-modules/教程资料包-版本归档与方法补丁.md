# 教程资料包-版本归档与方法补丁

依据案例：batch-363 案例 1811

## 读者承诺

把教程、案例、模板和补丁按版本归档，写作时按问题调用，而不是临时翻聊天记录找灵感。

## 核心循环

```text
资料更新 -> 主题归档 -> 方法补丁 -> 案例验证 -> 写作调用
```

## 开篇必备

- 资料按主题分类。
- 新方法标注适用场景。
- 大案例用于验证。
- 模板用于执行。

## 期待来源

- 遇到问题能快速查。
- 方法持续迭代。
- 教程和案例互相印证。
- 写作系统越来越可复用。

## 断档风险

- 只收藏不吸收。
- 文件按时间乱堆。
- 补丁和旧方法冲突不记录。
- 附件不可读却臆造内容。

## 训练任务

整理自己的写作资料为开头、情绪、人物、毒点、拆书五类，每类写一行适用场景。
