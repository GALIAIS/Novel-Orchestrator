# 玄幻桥段清单-灵感索引去机械化

依据案例：batch-267 案例 1334

## 读者承诺

把传统玄幻爽点清单当灵感索引使用，保留桥段功能，删除机械套壳，让旧套路服务当前主线和人物压力。

## 核心循环

```text
缺事件 -> 扫清单 -> 抽功能 -> 接入当前人物欲望 -> 重写因果 -> 爽点兑现
```

## 开篇必备

- 先确定当前主线压力，再选桥段。
- 每个桥段只保留一个功能。
- 旧壳、尴尬措辞、过时价值观要删。
- 桥段必须改变人物关系、资源或身份。

## 期待来源

期待来自熟悉爽点的新鲜兑现。读者接受宗门、宝物、家族、打脸、反转，但不接受一眼模板味。

## 断档风险

- 按清单顺序硬塞事件。
- 只复制爽点名，不重写因果。
- 桥段不服务当前主线。
- 旧套路价值观触雷。

## 训练任务

从传统玄幻清单选 5 条，把每条改写为“功能句”。再为自己的主角各设计一个新事件，不能出现原清单措辞。
