# 作者穿书-信息差舞台破局

依据案例：`batch-044` 案例 218。

## 读者承诺

作者穿书的核心爽点不是解释自己写过多少设定，而是在所有人无解时，用作者才知道的信息差现场破局。

## 核心循环

```text
众人绝望 -> 作者信息差 -> 现场条件 -> 验证解法 -> 获得认可
```

## 开篇必备

- 不用序章堆设定。
- 开场有具体危机。
- 主角知道关键规则。
- 规则能立刻解决问题。

## 期待来源

- 读者看到全知优势有用。
- 旁人从不信到震惊。
- 主角获得地位、资源或保护。
- 后续还有更多隐藏规则可用。

## 断档风险

- 四章解释设定。
- 全知只能看资料，不能行动。
- 修改能力强到破坏体系。
- 把穿越本身误当不公平伤害。

## 训练任务

写一个作者穿书危机：所有人都解决不了，主角用一条隐藏规则和现场条件完成反杀。
