# 打斗拆书-必要性收获优先

依据案例：`batch-078` 案例 386。

## 读者承诺

这个模块用于拆解长打斗。它承诺不把精力浪费在每个动作节拍上，而是抓打斗必要性、关键转折、战后收获和超额收益。

## 核心循环

```text
情绪铺垫 -> 必须开打 -> 关键转折 -> 胜负结果 -> 收获/超额收益 -> 新后果
```

## 开篇必备

- 写清为什么非打不可。
- 打斗前有情绪压力。
- 只记录关键局势变化。
- 战后必须有收获或代价。
- 结果要服务情绪上升。

## 期待来源

- 读者在乎打斗原因。
- 胜负关系到目标兑现。
- 关键转折带来紧张。
- 超额收益带来补偿。
- 负面后果接出下一段。

## 断档风险

- 逐动作拆太细，耗时过大。
- 打斗本身不服务情绪。
- 只有胜负，没有收获总结。
- 只有动作，没有后果。
- 拆书变成复刻原文。

## 训练任务

把一场长打斗压成 12 行拆解：必要性、欲望、阻力、三次转折、胜负、收获、超额收益、代价、下一钩子、可删动作。
