# 番茄首秀-五天数据复盘

依据案例：`batch-342` 案例 1706

## 读者承诺

番茄首秀后要把评分、在读、完读率当作早反馈系统，用来判断继续写、止损和下一本修正点。

## 核心循环

```text
十万字首秀 -> 评分/在读/完读率 -> 判断及格线 -> 继续或止损 -> 复盘下一本
```

## 开篇必备

- 记录首秀后的核心数据。
- 分清流量高光和留存能力。
- 完读率要和评分、在读一起看。
- 切不切看数据和学习收益。
- 下一本要有明确修正点。

## 期待来源

作者期待首秀给流量，真正有用的是首秀证明前十万字的读者承诺是否被读完。

## 断档风险

- 只看在读不看完读。
- 高光后下坡不复盘。
- 数据刚及格却盲目膨胀。
- 数据不佳只情绪崩溃。
- 不把首秀反馈带到下一本。

## 训练任务

拿一组首秀数据写三条复盘：有读者的证据、留不住的证据、下一本前十万字要改的结构问题。
