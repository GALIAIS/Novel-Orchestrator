# 现实商标-模糊代称处理

依据案例：`batch-167` 案例 832。

## 读者承诺

在现实感、潜在风险和年代感之间取舍，决定真实品牌名、模糊代称或“都懂”名称如何使用。

## 核心循环

```text
现实品牌 -> 情节必要性 -> 直接使用/模糊代称 -> 自然度检查 -> 年代感检查
```

## 开篇必备

- 先判断专名是否情节必需。
- 不是关键情节时可以模糊带过。
- 模糊代称要让读者秒懂。
- 不把经验建议当法律结论。

## 期待来源

读者期待生活场景自然可信，同时不被过时品牌、突兀代称或无关风险打断阅读。

## 断档风险

- “聊天软件”等泛称太生硬。
- 真实品牌让作品绑定具体年代。
- 作者因风险焦虑过度牺牲自然度。
- 专名抢走剧情注意力。

## 训练任务

把 5 个现实品牌分别写成直接名、模糊名和“都懂”代称，选择最自然且最耐久的一版。
