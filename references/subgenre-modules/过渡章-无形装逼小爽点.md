# 过渡章-无形装逼小爽点

依据案例：`batch-145` 案例 723。

## 读者承诺

过渡章不只是交代信息，也能顺手给一个轻爽点：旁观者看到主角与高位人物、资源或关系熟络，由旁观者产生佩服，主角再用低调话术完成无形装逼。

## 核心循环

```text
过渡任务 -> 接触高位资源 -> 旁观者注意 -> 旁观者佩服 -> 主角低调解释 -> 小爽点成立
```

## 开篇必备

- 一个必须完成的过渡功能。
- 一个能体现主角资源的对象。
- 一个旁观者视角。
- 一句轻描淡写的解释。

## 期待来源

读者期待非高潮段落也有奖励。小爽点来自旁人发现主角资源，而不是主角主动炫耀。

## 断档风险

- 过渡章完全流水账。
- 主角直接自夸，爽点变油。
- 旁观者反应不足，暗爽落空。
- 爽点太重，抢走本章过渡功能。

## 训练任务

写一个 500 字过渡段：主角赛后向负责人汇报，队友发现他和大人物熟络并产生佩服；主角只能用一句低调话解释关系。结尾回到下一步剧情。
