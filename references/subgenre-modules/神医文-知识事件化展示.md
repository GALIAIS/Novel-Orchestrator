# 神医文-知识事件化展示

依据案例：batch-309 案例 1545

## 读者承诺

神医文读者要看主角用医学知识解决事件，而不是看医学知识陈列。专业能力必须变成行动、风险、反转和结果。

## 核心循环

```text
专业知识 -> 具体病例/冲突 -> 主角判断 -> 行动验证 -> 打脸/救人/获利 -> 卖点兑现
```

## 开篇必备

- 先给主角要解决的病或麻烦。
- 医学知识只写当前事件用得上的部分。
- 用对手误判制造打脸空间。
- “邪医”书名必须体现邪、怪、偏门或反常规。

## 期待来源

期待来自知识产生结果。读者看不懂医学细节也没关系，只要看到主角判断准、行动狠、结果爽。

## 断档风险

- 写成病历式陈列说明。
- 药名和现实信息太实，带来风险。
- 书名说邪医，正文全是正派。
- 专业经历多，但没有事件爽点。

## 训练任务

把一段医学讲解改成 500 字事件：误诊、主角判断、验证动作、质疑者反应、结果反转。
