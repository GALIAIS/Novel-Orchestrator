# 末世解谜开篇-短程回应防跳级

依据案例：`batch-036` 案例 176。

## 读者承诺

末世解谜可以让主角懵，但不能让读者丢失第一个问题。开头抛出的声音、画面或异常，必须在读者短暂记忆内给出回应。

## 核心循环

```text
异常开场 -> 短程问题 -> 部分回应 -> 更大谜团 -> 当前危机推进
```

## 开篇必备

- 第一章吸引点清楚。
- 第一个问题 500 字内有回应。
- 环境描写服务危机或谜团。
- 解释不能打断读者等待。

## 期待来源

- 读者知道自己在等什么。
- 小问题被回应后信任作者。
- 大谜团继续拉长线。
- 主角懵而读者不迷路。

## 断档风险

- 抛出声音后长篇解释背景。
- 把信息跳级误认为伏笔。
- 环境描写只堆末世氛围。
- 第一章吸引点不明。

## 训练任务

写一个末世开篇：前 200 字抛异常，500 字内回应一半，1000 字内把回应转成更大谜团。
