# 题材学习-纵向样本与机制地图闭环

依据案例：`batch-403` 案例 2014

## 读者承诺

先完整体验多个有差异的题材样本，再区分核心承诺、共享惯例、独有方案和表面装饰；以机制地图组织资源、制度和事件，用短场景暴露缺口并定向回读。

## 核心循环

```text
完整体验多样本 -> 四层比较类型结构 -> 建立资源制度事件地图 -> 写三个关键场景 -> 记录具体缺口 -> 定向回读补证 -> 更新原创组合 -> 完成短单元
```

## 开篇必备

- 目标题材的核心乐趣和主角位置清楚。
- 样本之间具有可比较差异。
- 第一遍允许以普通读者身份完整体验。
- 机制地图覆盖资源、成长、制度、日常和卷目标。
- 世界细节连接具体使用者和事件。
- 学习后立即写获得、受限和兑现三个场景。

## 期待来源

作者不再追求一次掌握全部细节，而能解释一轮类型事件如何发生、实际写出，并把暴露的未知转成下一次定向学习任务。

## 断档风险

- 只刷榜单、简介和开头。
- 把单部作品特色当成类型规则。
- 细节库与资源制度因果脱节。
- 混合元素只换皮、不改变机制。
- 用“尚未完整掌握”无限延迟开写。
- 连续开新大纲，只训练兴奋期。
- 实践后不回读修正机制地图。

## 训练任务

完整阅读三个差异明显的开篇单元并制作四层对照表，再写获得资源、制度受限和首次兑现三个场景；只针对写作暴露的五个缺口定向补读。
