# 公开处刑-网文元素前置

依据案例：`batch-102` 案例 507。

## 读者承诺

开篇必须尽快让读者知道主角是谁、在哪、在做什么、遇到什么事、金手指有什么用、当前行动有什么收益。缺这些，编辑会判断为缺少网文元素或主线不清。

## 核心循环

```text
主角出现 -> 当下目标 -> 爆发事件 -> 金手指作用 -> 任务收益 -> 主线牵引
```

## 开篇必备

- 前 500 字出主角。
- 前几百字交代时间、地点、人物和事件。
- 景色描写不能压过故事。
- 金手指要在前三章明确价值。
- 任务必须有目的和收益。

## 期待来源

- 读者知道主角处境。
- 冲突发生得早。
- 金手指能解决眼前问题。
- 任务和主线挂钩。
- 读者知道继续看会得到什么。

## 断档风险

- 开头多段风景但没有主角。
- 刺客出现却不知道为什么。
- 大佬派任务但收益不明。
- 金手指存在却不强调，读者感受不到。

## 训练任务

重写第一章前 1000 字：删掉不服务主角处境的景色描写，把金手指第一次作用提前到当前冲突里，并写清任务收益。
