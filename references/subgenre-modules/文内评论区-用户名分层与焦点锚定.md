# 文内评论区-用户名分层与焦点锚定

依据案例：`batch-416` 案例 2079

## 读者承诺

文内评论区既有真实人群感，又能让读者立即识别必须记住的贯穿玩家。一级锚点账号稳定、独特并作出影响主线的行动；少量二级账号代表不同立场；其余群众用合并反应和数据变化表达。评论按触发、质疑、体验证据、反驳和群体转向组成微型场景，并为线上线下身份回收留痕。

## 核心循环

```text
先确定评论区必须改变的剧情状态 -> 给贯穿角色分配独特稳定的一级账号 -> 只保留少量二级立场账号 -> 合并三级群众反应 -> 按触发、质疑、体验证据、反驳和转向排列评论 -> 锚点账号采取影响后续的行动 -> 用语言、知识与时间痕迹连接现实身份 -> 后续揭露回收而不新增名字噪声
```

三级命名匹配记忆权重，微型场景防止评论墙停戏，锚点行动建立因果，身份痕迹把首章账号变成长线资产。

## 开篇必备

- 评论区必须改变的信息、态度或行动状态。
- 一个稳定、独特、易记的一级锚点账号。
- 锚点账号的声音、知识痕迹和本场行动。
- 三个以内承担不同立场的二级账号。
- 群众反应的合并方式与数据变化。
- 一条具体质疑和一项可验证体验证据。
- 群体转向及其对下载、传播或角色决定的影响。
- 线上线下身份的识别链与后续回收节点。
- 统一的“网名：评论”或拟真格式。

## 期待来源

- 首位体验者的网名对应哪个现实人物。
- 他为何比普通账号掌握更多信息。
- 怀疑者会如何验证游戏或产品。
- 一项真实数据能否改变群体态度。
- 锚点玩家会作出何种下一步行动。
- 语言和专业知识会怎样提前暴露身份。
- 线上身份揭露后会重写哪段关系。

## 断档风险

- 所有一次性评论者都有独特名字。
- 网名数量超过后续因果权重。
- 相似网名和数字串让读者误认。
- 评论只是段子墙，不改变剧情状态。
- 重要角色只靠排版突出，没有行动结果。
- 群众每条反应都单独命名，认知负荷失控。
- 线上线下身份只靠旁白硬揭。
- 多种括号、颜色和符号破坏阅读节奏。
- 锚点账号后续换号却没有声音痕迹。

## 训练任务

写一个 800 字游戏发布评论区，只允许一个一级账号和三个以内二级账号，其余群众合并呈现。必须完成质疑、体验证据、群体转向和锚点行动；再写 500 字线下场景，用语言或知识痕迹让读者认出锚点，不得直接宣布身份。
