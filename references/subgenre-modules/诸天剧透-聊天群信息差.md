# 诸天剧透-聊天群信息差

依据案例：`batch-144` 案例 719。

## 读者承诺

用聊天群作为固定结构容器，用诸天万界作为素材池，用剧透制造信息差。读者获得熟悉角色互动、群员震惊和原剧情被改变的轻爽感。

## 核心循环

```text
群员入场 -> 原世界困境 -> 主角剧透 -> 群员反应 -> 选择改变 -> 新结果 -> 下一个世界
```

## 开篇必备

- 聊天群规则简单直接。
- 群员来自读者容易识别的不同世界。
- 主角立刻展示一次剧透优势。
- 群员反应要清楚，不必复杂塑造人设。

## 期待来源

读者期待熟悉角色因为主角的一句话产生变化。爽点不在聊天形式本身，而在“我知道未来，所以我能改变你现在的选择”。

## 断档风险

- 群聊太散，没有明确困境。
- 剧透只是科普，没有改变选择。
- 群员反应平淡，信息差不爽。
- 世界太冷门，熟悉感不足。

## 训练任务

写一个 800 字开局：主角加入跨世界聊天群，出现 3 个群员；主角剧透其中一人的未来危机，让对方当场改变选择，并留下下一个世界的入场钩子。
