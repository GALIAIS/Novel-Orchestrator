# 黄金三章-悬念回收流

依据精拆：`case-library/精拆/batch-001.md` 案例 005。

## 适用范围

开篇训练、轻松悬疑、宫廷/权谋、都市奇遇、系统文、日记体或非传统叙事入口。

## 读者承诺

开篇通过高密度悬念、分批回收和新悬念接续，让读者在三章内感到“顺、好奇、有趣、想继续看”。

## 核心循环

```text
反差入口 -> 抛出多个小悬念 -> 回收一部分 -> 金手指/核心设定入场 -> 再抛更大期待
```

## 开篇必备

- 书名、身份或处境有反差。
- 第一章不能只是设定，要有可追的疑问。
- 悬念要能回收，不能只挖不填。
- 金手指或核心设定要在早期参与问题解决。
- 章末要有新期待。

## 期待来源

- 读者想知道主角为什么在这个身份里。
- 读者想知道日记/叙述里的异常是真是假。
- 小悬念回收带来信任感。
- 金手指展示带来“以后还能怎么用”的期待。

## 断档风险

- 机械套“黄金三章”：第一章受辱、第二章世界观、第三章金手指。
- 悬念太多不回收。
- 金手指讲得太晚或太虚。
- 日记体/旁白只搞笑，不推进事件。

## 操作卡

```text
反差入口：
悬念 1：
悬念 2：
悬念 3：
第一轮回收：
第二轮回收：
金手指参与：
章末新期待：
```

## 训练任务

给自己的第一章设计 3 个悬念，必须在前三章回收 2 个，并在第三章末尾抛出一个更大的长期待。
