# 金手指开篇-先用后讲来历

依据案例：batch-310 案例 1546

## 读者承诺

金手指开篇先展示效果，让读者看到它能改变局面，再解释来源、潜力和长期危机。

## 核心循环

```text
具体问题 -> 金手指出现 -> 当场解决 -> 补充来历 -> 拉长期危机
```

## 开篇必备

- 第一章给一个可解决的问题。
- 金手指必须当场产生结果。
- 第二章再讲来源和规则。
- 长远危机要和能力绑定。

## 期待来源

期待来自“这个能力真有用”。读者先看到收益，才愿意听设定。

## 断档风险

- 开篇先解释来历，迟迟不用。
- 金手指第一次使用没有改变局面。
- 第二章补设定太长，没有新危机。
- 能力潜力和主线无关。

## 训练任务

把一个金手指开篇改成两章结构：第一章只写使用与结果，第二章写来源、限制、潜力和危机。
