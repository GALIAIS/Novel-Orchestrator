# 大纲收束-故事线减法

依据案例：`batch-127` 案例 634。

## 读者承诺

整理大纲的价值不是保存所有灵感，而是暴露故事线过载并主动减法。线索收束后，读者更容易追踪主线，作者也更容易持续推进。

## 核心循环

```text
收到反馈 -> 整理大纲 -> 发现线索过多 -> 合并/删除支线 -> 主线更清楚
```

## 开篇必备

- 明确主线是什么。
- 列出全部故事线。
- 区分服务主线、服务人物和只是好玩。
- 合并同功能支线。
- 删除不回流主线的支线。

## 期待来源

- 主线更清楚。
- 读者不被无关支线分散。
- 作者减少贪多嚼不烂。
- 后续伏笔更容易兑现。

## 断档风险

- 工具变成堆线索的仓库。
- 每条灵感都舍不得删。
- 支线不回流主线。
- 公开复盘只剩日常闲聊，没有结构动作。

## 训练任务

列出当前大纲所有故事线，标注“服务主线/服务人物/只是好玩”。删除或合并所有“只是好玩”的线，并写 100 字说明主线因此更清楚在哪里。
