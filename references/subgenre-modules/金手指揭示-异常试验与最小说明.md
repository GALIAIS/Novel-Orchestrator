# 金手指揭示-异常试验与最小说明

依据案例：`batch-403` 案例 2015

## 读者承诺

把完整规则留在作者私有表中，正文只让主角在当前压力下发现异常、进行低风险试验、确认一个用途和一项限制，再用它改变眼前选择。

## 核心循环

```text
具体困境 -> 可观察异常 -> 谨慎试验 -> 有限结果 -> 误判或限制 -> 新决策 -> 后续事件分层揭示规则
```

## 开篇必备

- 先诊断作者是否认同金手指核心乐趣。
- 私有规则表与正文信息量分开。
- 主角通过观察、怀疑和测试获得知识。
- 第一次只证明一个当前用途。
- 成本或限制紧随首次成功出现。
- 完成发现、应用和代价小单元后再决定去留。

## 期待来源

读者与主角共同猜测并验证能力，首次成功立即改变处境，限制又保留后续探索；作者不用正面宣讲能力多么厉害。

## 断档风险

- 主角复述完整能力说明书。
- 系统提示替代人物试验和选择。
- 第一次出现同时加载全部规则。
- 只有成功，没有误判、成本和反制。
- 把常见机制误当成必须使用套话。
- 初稿未完成便因羞耻切书。
- 作者根本不认同核心乐趣却强迫续写。

## 训练任务

同一能力分别写纯说明、角色试验和危机误用三个版本；让陌生读者复述用途、限制和下一步选择，只保留证据最清楚的版本，再完成一个发现、应用、代价小单元。
