# 公开处刑心态-虚假夸夸不如读者反馈

依据案例：`batch-341` 案例 1702

## 读者承诺

公开处刑的价值是低成本暴露问题，虚假表扬不能代替编辑和真实读者反馈。

## 核心循环

```text
求表扬 -> 获得短期安慰 -> 无法验证市场 -> 接受批评 -> 提取改稿动作 -> 发书验证
```

## 开篇必备

- 区分情绪安慰和有效反馈。
- 批评先提取问题，不先反驳。
- 表扬只作为信心补充。
- 编辑和读者反馈权重最高。
- 公开处刑目标是求改。

## 期待来源

作者期待被认可，但作品真正需要的是能暴露签约、留存和读者期待问题的反馈。

## 断档风险

- 把公开处刑当夸夸群。
- 用尬夸延误修改。
- 只听舒服反馈。
- 被批评后只处理情绪。
- 忽略真实读者验证。

## 训练任务

把一段负面评文改写成 3 条可执行改稿清单，只保留问题和动作，不保留情绪判断。
