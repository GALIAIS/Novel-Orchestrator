# 平台试错-高门槛拒稿与首秀验证

依据案例：batch-370 案例 1849

## 读者承诺

新人阶段先选容错高的平台完成试错，用十万字首秀和后续数据验证能力，别死磕高门槛平台开头。

## 核心循环

```text
内投反馈 -> 判断平台门槛 -> 低门槛试错 -> 写到首秀 -> 数据决定去留
```

## 开篇必备

- 拒稿理由归档。
- 平台门槛判断。
- 试错平台选择。
- 首秀验证指标。

## 期待来源

- 快速获得完整反馈。
- 写到后续暴露真实问题。
- 数据比情绪更可靠。
- 避免变成开头鬼。

## 断档风险

- 把拒稿当编辑不认真。
- 死磕 boss 级平台。
- 无限重构开头。
- 期待编辑培养作者。

## 训练任务

写平台试错计划：列三家平台门槛、首秀指标、最多重构次数和继续/放弃条件。
