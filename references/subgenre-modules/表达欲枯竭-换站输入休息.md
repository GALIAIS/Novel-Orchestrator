# 表达欲枯竭-换站输入休息

依据案例：`batch-062` 案例 307。

## 读者承诺

这个模块用于作者没有表达欲、读不进热销书、不知道写什么的状态修复。它承诺的是先修复输入和感受，再谈输出；不要把单一平台热销风格当成全部网文，也不要在审美疲劳时硬逼自己开新书。

## 核心循环

```text
输入疲劳 -> 识别错位 -> 换平台/换风格 -> 找回阅读愉悦 -> 轻量拆样本 -> 恢复输出
```

## 开篇必备

- 先写清楚最近读不下去的具体原因。
- 区分“不喜欢这个平台风格”和“不适合写网文”。
- 至少换一个平台或一种题材输入。
- 优先找能恢复愉悦的作品，喜剧、强爽点、强表达都可以。
- 情况严重时允许休息几个月。

## 期待来源

- 作者重新发现自己愿意看的东西。
- 新输入带来新的表达欲。
- 喜剧或强表达样本降低疲劳。
- 轻量拆样本把兴趣转成可执行动作。
- 休息后恢复阅读感觉。

## 断档风险

- 强迫自己继续啃不喜欢的热销书。
- 把平台热榜等同于唯一市场。
- 休息时完全不记录任何触动点。
- 刚恢复一点兴趣就立刻开大坑。
- 只追自我表达，不回到读者承诺。

## 训练任务

列出 5 个读不下去的热销样本原因，再找 5 个不同平台或不同风格样本。每个只记一句“我为什么愿意继续读”，最后选 1 个做第一章轻拆。
