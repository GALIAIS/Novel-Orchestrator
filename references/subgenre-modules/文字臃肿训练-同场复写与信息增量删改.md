# 文字臃肿训练-同场复写与信息增量删改

依据案例：`batch-410` 案例 2047

## 读者承诺

用场景卡减少结构犹豫，用同场复写对齐剧情节拍，再按因果、信息增量和节奏声音三轮删改；分别记录规划、初稿和修改时间，把笼统臃肿转为可训练问题。

## 核心循环

```text
场景卡锁定目标阻碍转向 -> 连续初稿 -> 功能相同片段对照 -> 按节拍比较字数与职责 -> 因果轮删无状态变化 -> 增量轮删重复 -> 节奏轮恢复声音 -> 分项计时复练
```

## 开篇必备

- 本场目标、阻碍、关键动作和结束变化。
- 初稿与润色分开的时间盒。
- 一个功能相同而非词句相同的参照片段。
- 对齐后的建立、加压、选择和结果节拍。
- 当前最高频的一类冗余。
- 保留人物声音和必要停顿的标准。

## 期待来源

清楚场景任务后初稿不再逐句停顿；对照版暴露具体重复，删改让因果变清，而分项时间与版本差异持续证明下一次写得更快、更准。

## 断档风险

- 把结构犹豫误判成纯文字问题。
- 边写初稿边追求最终措辞。
- 同场复写变成复制原句。
- 只缩短字数，不检查状态变化。
- 删除所有停顿导致只剩剧情摘要。
- 不计时便笼统否定写作速度。

## 训练任务

读完一个 500 至 800 字高密度场景后合上原文，用原创人物复写同样四拍；逐拍对齐并压缩 25%，保留目标、转向、情绪和人物声音，同时记录规划、初稿和三轮修改时间。
