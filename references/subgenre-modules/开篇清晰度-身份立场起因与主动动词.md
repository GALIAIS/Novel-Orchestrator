# 开篇清晰度-身份立场起因与主动动词

依据案例：`batch-417` 案例 2085

## 读者承诺

读者在首屏知道当前焦点人物以什么身份进入什么事件、站在哪一边、为什么此刻受压、想得到或避免什么。直接起因把外部危机变成个人问题，主角用第一个主动选择暴露性格和能力；描写只保留会改变目标、威胁、情绪或行动的细节，并按移动端意义节拍分段。

## 核心循环

```text
用职位、关系或处境锚定当前主角 -> 说明他在事件中的角色与立场 -> 给出足以理解当下的直接起因 -> 把外部危机转成主角具体欲望 -> 主角基于已知信息主动选择 -> 描写只保留会改变目标、威胁、能力、情绪或行动的细节 -> 结果生成下一目标 -> 按观察、动作、反应和判断节拍分段 -> 在真实手机宽度复读删改
```

身份立场提供代入锚，起因与欲望赋予危机意义，主动选择建立人物，功能句降低负荷，移动端节拍保证主语和因果清楚。

## 开篇必备

- 当前焦点人物的职位、关系或处境。
- 他在当前事件中的角色与立场。
- 足以理解当下的直接起因。
- 可以延后揭示的幕后问题。
- 主角当前欲望与不行动的个人损失。
- 第一个基于信息的主动选择。
- 选择暴露的性格、能力和盲点。
- 只保留具备目标、威胁、能力、情绪或行动功能的描述。
- 观察、动作、反应、判断的分段节拍。
- 实际手机宽度下的名字密度与段落墙检查。

## 期待来源

- 主角为何会卷入当前危机。
- 他想活、翻盘或离开的具体方式。
- 第一个选择会暴露什么人物底色。
- 哪项幕后起因被合理保留为悬念。
- 阻碍的体征怎样直接改变路线。
- 行动结果会生成什么新目标。
- 清楚分段能否让读者持续跟住主语。

## 断档风险

- 只给姓名、代词或“主角”标签。
- 危机很大却不知道为何落到此人身上。
- 完整隐藏直接起因，读者无法理解当下行动。
- 主角只因“可以这样做”而行动。
- 身份锚定被误写成一页履历。
- 弱势身份被当成所有类型的固定公式。
- 外貌清单不影响任何选择。
- 机械规定每段不超过固定字数。
- 同段塞入多名人物、时间和焦点变化。
- 未在真实手机宽度进行阅读测试。

## 训练任务

重排一个 1500 字开篇，首屏回答身份、事件角色、直接起因和当前欲望，前三段出现主动选择。给每句标目标、威胁、能力、情绪或行动功能，删去无标注句；按观察、动作、反应和判断分段，并在真实手机宽度朗读检查。
