# 金手指扩容-阶段性新增玩法

依据案例：`batch-113` 案例 562。

## 读者承诺

初始金手指如果不成长，长篇就需要阶段性扩容。新增能力不能凭空掉落，最好由上一阶段能力带来资格、因果或资源。

## 核心循环

```text
初始优势 -> 资格打开 -> 新能力 -> 新玩法 -> 更高层敌人
```

## 开篇必备

- 初始金手指一句话清楚。
- 标明它的成长风险。
- 第一阶段能带来新资格。
- 新能力有因果来源。
- 每阶段都有新玩法。

## 期待来源

- 能力树持续扩展。
- 读者期待下一阶段获得什么。
- 老能力不废，新能力不突兀。
- 长线不断档。

## 断档风险

- 初始能力单点用尽。
- 硬塞新外挂。
- 新能力和旧能力无因果。
- 主角多才多艺但没有结构。

## 训练任务

给一个初始能力设计三段扩容，每段写清“上一能力如何带来下一能力”。
