# 暴论教程-原理优先反模板

依据案例：`batch-073` 案例 364。

## 读者承诺

这个模块用于辨别“三句话必过”“固定字数必签”式教程。它承诺的是保留有用提醒，反查背后原理，避免死套模板后不会迭代。

## 核心循环

```text
看到硬规则 -> 反问解决什么问题 -> 找适用边界 -> 改成原则 -> 实践验证
```

## 开篇必备

- 不直接迷信“必过”。
- 每条规则都要反查原理。
- 区分短期开头技巧和长篇能力。
- 记住模板不能替代积累。
- 慢就是快。

## 期待来源

- 硬规则能提醒新人注意节奏。
- 原理能支持后续变形。
- 适用边界避免生搬硬套。
- 实践反馈让知识内化。
- 长期能力替代捷径幻觉。

## 断档风险

- 只知道怎么套，不知道为什么。
- 只会开头，不会后续。
- 以为签约等于会写书。
- 被外行教程吸粉话术带偏。
- 规则太死导致剧情僵硬。

## 训练任务

列出 5 条教程硬规则，每条写三栏：真实问题、背后原理、不适用场景。写不出原理的规则先不用。
