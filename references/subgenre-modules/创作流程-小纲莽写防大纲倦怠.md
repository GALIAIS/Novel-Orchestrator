# 创作流程-小纲莽写防大纲倦怠

依据案例：`batch-193` 案例 961。

## 读者承诺

针对写完完整大纲后失去热情的作者，用方向大纲保留主线，用码字前小纲保留执行新鲜感。

## 核心循环

```text
方向大纲定边界 -> 当天小纲定动作 -> 正文保留探索 -> 写完再补下一段 -> 防止预演耗尽动力
```

## 开篇必备

- 先确定主角目标和长期压力。
- 不提前写死所有细节转折。
- 每天码字前只准备近几章小纲。
- 给下一章保留一个未知点或待探索问题。

## 期待来源

作者每次开写时仍有探索感，正文不是机械搬运已经看完的故事。

## 断档风险

- 全书大纲写得过细，提前获得完结感。
- 完全不规划，导致主线散掉。
- 频繁被新脑洞吸走。
- 只靠变现刺激，没有调整执行流程。

## 训练任务

把一个完整故事规划拆成“方向大纲+三章小纲”，三章以外只保留问题，不写具体解决方案。
