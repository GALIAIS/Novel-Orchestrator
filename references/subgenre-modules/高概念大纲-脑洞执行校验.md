# 高概念大纲-脑洞执行校验

依据案例：`batch-157` 案例 785。

## 读者承诺

高概念大纲的价值在于观察大脑洞如何承载设定、矛盾和推进方式，同时警惕执行粗糙和断更风险。

## 核心循环

```text
一句话脑洞 -> 设定规模 -> 核心矛盾 -> 推进方式 -> 正文验证 -> 执行风险
```

## 开篇必备

- 一句话脑洞。
- 主角处境。
- 核心矛盾。
- 设定规模。
- 第一惊奇点。
- 推进方式。
- 执行风险。

## 期待来源

读者期待一个足够大的设定把人物和世界绑在一起，并持续制造惊奇。

## 断档风险

- 脑洞大但执行粗糙。
- 大纲吸引，正文落地弱。
- 设定详细但主线不清。
- 连载持续性不足。

## 训练任务

选一部高概念作品，只看简介或大纲，写出脑洞、核心矛盾、推进方式和风险，再读前三章验证承诺是否落地。
