# 教程预告-版本追踪

依据案例：`batch-173` 案例 865。

## 读者承诺

教程预告不是教程正文。知识库整理时要记录版本、更新内容、是否已读正文和是否包含案例，避免把预告当方法吸收。

## 核心循环

```text
旧方法 -> 新版预告 -> 记录版本 -> 回访正文 -> 吸收案例
```

## 开篇必备

- 标明只是预告还是正文。
- 记录发布时间和新增内容。
- 标注可见渠道和是否收费。
- 等读到正文后再吸纳具体方法。

## 期待来源

学习者期待核心工具持续升级。版本追踪能防止重复学习旧版，也能提示后续回访。

## 断档风险

- 把预告当完整教程。
- 不记录版本，旧新混用。
- 没读案例就整理方法。
- 忽略正反案例的补充价值。

## 训练任务

建立一个教程版本表，列出工具名、版本、预告/正文状态、新增内容、是否已读、是否有案例和需要回访的链接。
