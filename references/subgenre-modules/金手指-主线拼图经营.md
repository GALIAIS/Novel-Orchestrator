# 金手指-主线拼图经营

依据案例：`batch-012` 案例 056。

## 读者承诺

复杂金手指不是一次性说明的设定，而是一套能持续推动主线的拼图。读者既看每一块拼图当场解决问题，也期待完整蓝图最终成型。

## 核心循环

```text
展示蓝图 -> 获得碎片 -> 解决近冲突 -> 产生近爽点 -> 暗示远形态 -> 推动下一事件
```

## 开篇必备

- 先让读者看见金手指潜力，不急着解释全貌。
- 第一块拼图必须能立刻解决冲突。
- 主角身份、主线目标和金手指使用场景要绑定。
- 至少埋一个远期蓝图，让读者知道后续还有更大收益。

## 期待来源

- 下一块拼图是什么。
- 新功能如何解决新冲突。
- 金手指完整形态会带来什么身份、资源或势力变化。
- 主线副本和日常经营能否双重兑现爽点。

## 断档风险

- 只展示设定，不解决冲突。
- 拼图碎片没有即时价值，读者只看到说明书。
- 金手指和主线脱节，只在打脸时临时拿出来。
- 蓝图太快讲完，远期期待消失。

## 训练任务

设计一个复杂金手指的 5 块拼图，每块写出获取方式、解决冲突、当场爽点、远期暗示和下一事件入口。
