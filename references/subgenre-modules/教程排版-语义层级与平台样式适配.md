# 教程排版-语义层级与平台样式适配

依据案例：`batch-395` 案例 1972

## 读者承诺

让教程在富文本、Markdown和纯文本中都保持同一阅读地图：先保存标题、结论、步骤和警告的意义，再由平台适配层决定字号、粗体与颜色。

## 核心循环

```text
标注内容语义 -> 建立少量固定层级 -> 应用统一样式 -> 映射平台控件 -> 纯文本降级检查 -> 发布后复核
```

## 开篇必备

- 一篇文章只有一个最高级标题。
- 二级标题可以独立概括各模块。
- 粗体只标局部结论或关键词。
- 颜色只绑定少数固定状态。
- 警告同时保留明确文字标签。
- 去掉全部样式后仍能顺序阅读。

## 期待来源

读者先扫标题获得结构地图，再快速定位结论、例子、步骤与证据边界，不需要从装饰中猜文章逻辑。

## 断档风险

- 用放大、加粗和变色代替逻辑分节。
- 整段粗体导致重点消失。
- 同一颜色承担多个含义。
- 只靠颜色提示风险。
- 把某个平台样式写死进内容源。
- 复制发布后不检查格式降级。

## 训练任务

把一篇教程先还原为无样式语义版，再输出富文本、Markdown和纯文本三版；请读者各扫十秒，三版都能复述主题、三项结论和一项警告才通过。
