# 开篇求评-附件缺失证据限制

依据案例：`batch-126` 案例 628。

## 读者承诺

前三章求评必须以正文为证据。正文不可见时，只能记录证据限制，并搭好观感和平台判断框架。

## 核心循环

```text
提交前三章 -> 查看正文 -> 判断开篇观感 -> 对照平台榜单 -> 给修改或投稿建议
```

## 开篇必备

- 正文必须可见。
- 书名简介要附上。
- 目标平台要说明。
- 同类对标书要列出。
- 求评问题要具体。

## 期待来源

- 作者想知道前三章留不留人。
- 平台建议需要正文风格支撑。
- 对标榜单能降低误判。
- 明确问题让反馈更有效。

## 断档风险

- 只有附件但别人无法查看。
- 只问“观感”不说目标。
- 只凭题材名推荐平台。
- 证据不足却给确定结论。

## 训练任务

准备求评包：正文、简介、目标平台、对标书、三个具体问题。缺一项先补齐再求评。
