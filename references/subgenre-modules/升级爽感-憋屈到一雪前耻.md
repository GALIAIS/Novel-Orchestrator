# 升级爽感-憋屈到一雪前耻

依据案例：`batch-027` 案例 135。

## 读者承诺

升级爽感来自没实力时寸步难行，提升后回到原阻碍点一雪前耻。

## 核心循环

```text
欲望 -> 阻碍 -> 憋屈压抑 -> 艰难提升 -> 克服阻碍 -> 一雪前耻
```

## 开篇必备

- 主角明确欲望。
- 阻碍具体。
- 没实力造成实际损失。
- 提升后能改变原局面。

## 期待来源

- 读者先感到憋屈。
- 提升不容易，显得有价值。
- 回到同一个压迫点兑现。
- 达成后世界对主角态度改变。

## 断档风险

- 只写数值提升，不写前后改变。
- 没实力没有造成痛感。
- 提升后不回收原阻碍。
- 一雪前耻太轻，爽点不落地。

## 训练任务

写一个升级单元：前面写主角因弱小受阻，中间写艰难提升，后面必须回到同一个人或事上完成反打。
