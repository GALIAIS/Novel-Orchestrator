# 模拟器开篇-玩法直给工具人收益

依据案例：`batch-085` 案例 425。

## 读者承诺

这个模块用于模拟器题材开篇。它承诺第一章直接展示玩法、选择、失败和收益，让读者立刻知道这本书爽在哪里。

## 核心循环

```text
玩法直给 -> 模拟选择 -> 工具人行动 -> 收益结算 -> 下一轮更优解
```

## 开篇必备

- 书名或简介点明模拟器。
- 300 字内进入模拟规则。
- 800 字内完成一次选择。
- 明确存活越久奖励越多。
- 留下下一轮模拟期待。

## 期待来源

- 读者立即看到玩法。
- 每轮模拟都有收益。
- 工具人表现影响主角收获。
- 失败会转化为下次期待。
- 主角能主动压榨或调度资源。

## 断档风险

- 先铺背景不进玩法。
- 模拟规则不清。
- 主角存在感过低。
- 工具人收益引发道德反感。
- 作者不喜欢模拟器还硬写。

## 训练任务

写模拟器开篇 800 字：300 字内出现界面，800 字内完成选择并结算一个小收益。结尾必须让读者想看下一轮。
