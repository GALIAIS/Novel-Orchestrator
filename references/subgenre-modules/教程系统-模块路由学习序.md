# 教程系统-模块路由学习序

依据案例：`batch-084` 案例 418。

## 读者承诺

这个模块用于把散乱教程整理成新人可走的学习路径。它承诺按主题模块和新人级别推荐阅读顺序，而不是让新人一次吃完整套理论。

## 核心循环

```text
教程拆块 -> 新人分级 -> 模块路由 -> 阅读顺序 -> 反馈迭代
```

## 开篇必备

- 至少拆出拆书、冲突、结构、开篇、复盘模块。
- 标明每个模块适合的新人级别。
- 给出先后顺序。
- 每个模块绑定练习。
- 根据反馈调整。

## 期待来源

- 新人知道先学什么。
- 超纲内容被延后。
- 每个主题能独立训练。
- 学习顺序更稳定。
- 反馈能推动教程更新。

## 断档风险

- 所有理论混在一起。
- 不分新人级别。
- 只读不练。
- 模块没有入口问题。
- 后续不按反馈迭代。

## 训练任务

把一个教程拆成 5 个模块，并给 0、1、2、3 级新人各排一条阅读顺序和一个练习任务。
