# 读者群运营-作者成本与失控风险

依据案例：`batch-229` 案例 1144。

## 读者承诺

读者群不是自动提升黏性的工具，而是需要作者长期投入、规则管理和风险兜底的社群资产。

## 核心循环

```text
开群目的 -> 时间投入 -> 规则边界 -> 管理分工 -> 风险预案 -> 决定开或不开
```

## 开篇必备

- 明确开群目的。
- 作者能投入足够时间。
- 有管理员和禁聊规则。
- 有群聊偏离作品后的处理方式。

## 期待来源

当群能稳定围绕作品、更新、角色和活动运转时，它才可能沉淀核心读者。

## 断档风险

- 作者不活跃，群变成小圈子。
- 群聊脱离作品，转向游戏、闲聊或争议话题。
- 群友搞事让作者背锅。
- 内鬼带节奏影响连载心态。
- 社恐作者为了“应该开群”硬开。

## 训练任务

写一份开群决策表：目的、每周投入时间、管理员、禁聊规则、作品活动、失控预案。任一核心项缺失，则默认暂不开群。
