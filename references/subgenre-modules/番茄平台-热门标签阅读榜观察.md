# 番茄平台-热门标签阅读榜观察

依据案例：`batch-183` 案例 912。

## 读者承诺

番茄平台观察应把热门标签和阅读榜分开看，用交叉信息辅助选题，而不是迷信单一榜单。

## 核心循环

```text
平台观察 -> 热门标签看趋势 -> 阅读榜看热度 -> 拆标杆书验证 -> 得出临时判断
```

## 开篇必备

- 明确观察目标。
- 记录热门标签。
- 记录阅读榜样本。
- 至少拆几本标杆前三章。

## 期待来源

作者通过榜单和标签确认平台当前读者注意力，再用具体书验证卖点落地方式。

## 断档风险

- 把榜单当绝对畅销榜。
- 只看标签不拆书。
- 忽视平台规则变化。
- 用旧经验做长期判断。

## 训练任务

记录一个题材的热门标签和阅读榜前 10，再拆 3 本前三章，只输出趋势和验证证据。
