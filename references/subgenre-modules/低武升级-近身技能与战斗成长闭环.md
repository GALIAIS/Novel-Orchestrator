# 低武升级-近身技能与战斗成长闭环

依据案例：`batch-397` 案例 1984

## 读者承诺

把近身战写成升级文的主要玩法：距离、体力、伤势和少量技能持续组合反制，每次升级增加战术选择，每场胜负改变人物目标与资源。

## 核心循环

```text
长期目标迫使参战 -> 敌人规则克制 -> 距离体力技能交换 -> 组合破局 -> 付出伤势资源代价 -> 升级新增战术选择
```

## 开篇必备

- 定义力量上限和不能做什么。
- 明确普通武器仍有多少威胁。
- 近身距离、体力和伤势可感。
- 主角有长期“为何而战”。
- 第一敌人克制主角一项旧办法。
- 胜利改变资源、关系或身份。

## 期待来源

读者等待主角如何在可理解的限制内拆解新敌人，并看新技能改变打法，而非只让伤害数字继续增长。

## 断档风险

- 只争论低魔低武标签。
- 世界规则无法解释枪械或远程能力。
- 技能都是不同颜色的伤害数字。
- 战斗没有目标和后果。
- 升级后仍重复同一种打法。
- 作者再次滑入不擅长的经营支线。

## 训练任务

写一页力量上限、距离、体力、伤势和五项技能交互，再设计三场递进战斗；每场只新增一个战术选择，并让目标、关系和资源同步变化。
