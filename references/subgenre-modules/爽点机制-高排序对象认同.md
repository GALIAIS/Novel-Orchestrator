# 爽点机制-高排序对象认同

依据案例：`batch-192` 案例 956。

## 读者承诺

爽点来自主角获得认同，尤其是强者、喜欢的人、漂亮有钱有地位的人或群体的承认。

## 核心循环

```text
被贬低 -> 高排序对象在场 -> 展示实力 -> 震惊反馈 -> 获得认同
```

## 开篇必备

- 有明确贬低或误判。
- 有围观者或高价值认同者。
- 主角展示能改变他人判断。
- 反馈落到认同和震惊。

## 期待来源

读者期待主角让看不起自己的人、强者和群体承认自己，从而获得社交认同爽。

## 断档风险

- 没有围观反馈。
- 认同者层级低于主角当前层级。
- 反派太工具人，承认不够爽。
- 主角升级后认同者不升级。

## 训练任务

写一个打脸桥段，列出普通群众、强者、主角在意的人三类认同者，并安排最强的一类在高潮给反馈。
