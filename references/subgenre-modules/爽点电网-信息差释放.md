# 爽点电网-信息差释放

依据案例：`batch-003` 案例 014。

## 读者承诺

爽点不是主角赢了就自动成立，而是信息差通过人际关系网释放后，形成多点反馈和情绪传导。

## 核心循环

```text
配置关系网 -> 配置信息差 -> 形成情绪缺口 -> 揭开信息差 -> 多节点反应 -> 爽点放大
```

## 开篇必备

- 谁知道真相。
- 谁误判真相。
- 谁有资格评价主角。
- 谁关心主角输赢。
- 谁会因主角表现改变立场。

## 期待来源

- 读者等待误判被纠正。
- 读者等待关系人重新评价主角。
- 读者等待隐藏身份、能力、贡献被公开。
- 读者等待信息差释放后的关系变化。

## 断档风险

- 信息差只有主角知道，没有人接收。
- 关系网太薄，爽点无法传导。
- 释放太快，情绪缺口不够。
- 释放太慢，读者忘记期待。

## 训练任务

写一个信息差释放表：至少 5 个关系节点，每个节点写“释放前认知、释放后反应、后续行动”。
