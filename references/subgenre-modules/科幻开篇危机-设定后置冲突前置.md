# 科幻开篇危机-设定后置冲突前置

依据案例：batch-272 案例 1358

## 读者承诺

科幻开篇先兑现事故、目标、阻力和主角选择，再补设定。读者先关心主角如何活下来、如何突破危险，之后才愿意接收飞船、AI、星球和文明背景。

## 核心循环

```text
科幻事故 -> 即时危险 -> 主角目标 -> 外部阻碍 -> 行动破局 -> 设定补充
```

## 开篇必备

- 第一章要有可见危险，如漏水、缺氧、追杀、舱门卡死。
- 主角要有明确行动目标。
- 设定介绍插在行动间隙，不要连续灌输。

## 期待来源

读者期待来自近距离压力：飞船会不会沉、主角能不能逃、AI 会不会伤人、第一个敌人如何出现。宏大世界观要服务这些即时问题。

## 断档风险

- 设定说明占掉开篇篇幅。
- 只有生存背景，没有爆发冲突。
- 五章没有完整事件，读者感到流水账。

## 训练任务

把科幻设定说明压缩成 100 字，再写一个事故逃生场景，每 200 字加入一次新阻力。
