# 主金手指-逆推冲突与前期兑现

依据案例：batch-374 案例 1866

## 读者承诺

开篇冲突要围绕主金手指或主角过人之处设计。主能力不能参与第一冲突时，通常说明冲突选错了。

## 核心循环

```text
主金手指 -> 可装逼场景 -> 逆推冲突 -> 阻碍资源 -> 前期兑现
```

## 开篇必备

- 主金手指或主角过人之处。
- 它能解决的问题类型。
- 第一冲突和它的因果关系。
- 副能力的辅助边界。

## 期待来源

- 读者知道主角凭什么特别。
- 第一冲突正好需要这个特别之处。
- 装逼场景被逆推出合理阻碍。
- 后续还能升级兑现。

## 断档风险

- 第一冲突靠副金手指解决。
- 主金手指只做背景说明。
- 冲突和能力没有资源关系。
- 新人模仿大神低显性金手指。

## 训练任务

先写 3 个主金手指装逼场景，再倒推其中一个第一章冲突，禁止用无关能力解决。
