# 首次拆书-证据标注与增量迭代

依据案例：`batch-412` 案例 2059

## 读者承诺

拆书文件明确作品版本、实际范围和学习问题；每条结论分为证据、解释、假设和验证，持续追加时同步覆盖表、双索引与变更日志，使核心结论保持唯一且可纠错。

## 核心循环

```text
声明版本范围问题 -> 定位章节事件证据 -> 标注结构功能 -> 提炼带边界假设 -> 原创或对照验证 -> 更新覆盖与索引 -> 事实校对分析复核 -> 主结论修订并记录变更
```

## 开篇必备

- 作品版本、来源与实际阅读范围。
- 未覆盖部分和唯一学习问题。
- 可定位但不大段复制的事件证据。
- 解释、机制假设和适用边界。
- 章节索引、机制索引和未决问题。
- 反馈状态与变更日志。

## 期待来源

每项结论都能回到具体事件，多个证据逐渐汇成机制；原创练习暴露边界，新章节和公开纠错不断修订唯一现状，文档随覆盖扩大而更可信。

## 断档风险

- 未读附件却根据标题补写内容。
- 不声明范围便评价全书。
- 把个人感受直接写成通用规则。
- 文件尾部不断追加却没有索引。
- 新旧结论冲突而不更新主结论。
- 装饰图被误当内容证据。

## 训练任务

为已读三章作品建立增量拆书文件，每章写三条证据、功能、假设并完成一个原创验证；请另一人只核对事实定位，自己另做分析复核，将全部修订写入变更日志。
