# 签约意向-样稿提审与生效状态机

依据案例：`batch-405` 案例 2025

## 读者承诺

把编辑兴趣、补样、提审、报价、合同、签署、生效、验收和结算分成独立状态；每道门只凭书面证据推进，并以样稿同时验证一个明确创作假设。

## 核心循环

```text
编辑兴趣 -> 明确补样大纲要求 -> 版本化交付 -> 审核结论与正式报价 -> 合同审阅澄清 -> 签署生效 -> 首单元技术验证 -> 验收数据结算分别留证
```

## 开篇必备

- 当前商务状态只记录已有证据。
- 补样字数、格式、期限和用途明确。
- 保存样稿、大纲和沟通版本。
- 提审结果与正式报价形式清楚。
- 完整合同替代口头或聊天承诺。
- 新项目只验证一个主要写作机制。

## 期待来源

模糊的“可能签”被拆成连续可执行步骤；尚未发生的结果不被提前宣布，商务通过与内容有效也不会互相冒充。

## 断档风险

- 询价或补样就宣称板上钉钉。
- 样稿用途和权利范围不清。
- 没有保存交付版本与正式渠道。
- 报价未落字便安排长期产能。
- 把签约当成写作方法已经成功。
- 一次实验同时更换多个创作变量。
- 验收、数据和结算混成一个结果。

## 训练任务

制作签约状态表，禁止使用“差不多、稳了”；每格只写书面证据和下一动作。把补样设计成单变量实验，分别定义审核反馈和读者反馈如何验收。
