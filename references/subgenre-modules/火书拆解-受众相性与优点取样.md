# 火书拆解-受众相性与优点取样

依据案例：`batch-336` 案例 1678

## 读者承诺

火书看不进去时，先判断目标受众和市场有效机制，再决定学局部、继续拆或换类别。

## 核心循环

```text
个人不喜欢 -> 找目标受众 -> 提取有效爽点 -> 拆金手指/人设/望远镜 -> 判断能否迁移
```

## 开篇必备

- 分清个人口味和市场结果。
- 找出目标读者是谁。
- 至少列出 5 个有效优点。
- 只学自己能驾驭的一点。
- 相性不符时及时换类。

## 期待来源

作者期待从火书里提取可用机制，而不是被“我不喜欢”挡住学习路径。

## 断档风险

- 觉得无聊就否定火书。
- 看不出优点还硬写同类。
- 只学表面开篇。
- 忽略萌宠、人设、轻松感等非冲突爽点。
- 不拆每章望远镜。

## 训练任务

选一本看不进去但成绩好的书，写出 5 条对目标读者有效的优点；写不出就暂时换赛道。
