# 底层规则冲突-资源互斥与行为必然性

依据案例：`batch-412` 案例 2060

## 读者承诺

用参与人数或行为对资源、效率和风险的可观察反馈，迫使不同身份调整策略；隐藏、垄断、合作、转修和制度反应互相预判，持续生成冲突、联盟与世界结构。

## 核心循环

```text
规则改变稀缺资源或效率 -> 不同身份观察不完全反馈 -> 采取保护收益的一阶策略 -> 他人预判并二阶反应 -> 信任组织知识制度重组 -> 主角利用边界并付代价 -> 结果改变人数资源信任 -> 新状态再生冲突
```

## 开篇必备

- 一句话可计算的底层规则。
- 输入变量、反馈、范围、阈值和延迟。
- 新人、既得利益者、传授者与监管者的差异策略。
- 不止杀戮的合作、转修和制度选择。
- 有成本的例外与主角可改变参数。
- 局部结果通向组织和政治结构的反馈链。

## 期待来源

局部效率下降先制造猜疑，规则真相让善意人物也利益相冲；主角寻找边界策略获得智力爽，却因改变参数而重写更大的资源、信任和制度网络。

## 断档风险

- 规则只存在于设定说明，不改变行为。
- 所有身份都选择同一种直接杀戮。
- 稳定合作没有监督和违约成本。
- 不写范围、阈值、误差和例外。
- 金手指无成本取消核心规则。
- 局部冲突不反馈到资源和制度。

## 训练任务

设计一条“参与者越多，收益越低”的规则，为五类身份各写一阶和二阶策略，禁止全部直接杀戮；再写一场同时改变人数、资源和信任的冲突，并推导一个新组织制度。
