# 番茄首秀-完读阈值与切书决策

依据案例：batch-375 案例 1874

## 读者承诺

首秀吸量不等于成功。完读率低说明内容承接不住流量，需要用数据判断改稿、继续或切书。

## 核心循环

```text
首秀流量 -> 完读率 -> 对照阈值 -> 评估后续给量/收益 -> 切书或改稿
```

## 开篇必备

- 首秀流量。
- 完读率。
- 参考及格线。
- 后续给量和收益。

## 期待来源

- 数据替代玄学判断。
- 及时止损。
- 找出章节流失点。
- 判断是否值得改。

## 断档风险

- 只看流量兴奋。
- 平台继续给量就误判健康。
- 低完读还无限硬撑。
- 不计算改稿成本。

## 训练任务

做首秀复盘表：流量、完读、流失点、评论、后续给量、每日收益、改稿成本，并给出继续/改稿/切书结论。
