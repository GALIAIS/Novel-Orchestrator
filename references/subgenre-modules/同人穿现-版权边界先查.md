# 同人穿现-版权边界先查

依据案例：`batch-050` 案例 248。

## 读者承诺

手游角色穿越现实能吃角色认知和现实碰撞，但动笔前先确认官方二创态度、平台同人生态和真名使用风险，避免剧情还没讨论就卡在版权边界。

## 核心循环

```text
原 IP 角色 -> 穿越现实 -> 悬疑/冲突 -> 查二创声明 -> 平台适配 -> 必要时原创化
```

## 开篇必备

- 确认原 IP 是否允许二创。
- 确认目标平台是否适合同人。
- 判断是否使用真名和专有术语。
- 角色能力可以保留核心感觉，但要准备原创化方案。
- 悬疑事件要独立成立，不能只靠 IP 认知。

## 期待来源

- 熟悉角色进入现实，天然有反差。
- 现实悬疑能提供持续事件。
- 原作能力在现实中重用，会产生新鲜感。
- 原创化后可降低版权依赖。

## 断档风险

- “平台上有人写”不等于一定安全。
- 官方没声明时，真名照搬风险高。
- 只靠 IP 角色，没有现实事件，会缺少主线。
- 附件稿件不可见时，不应脑补正文问题。

## 训练任务

把一个手游同人穿现设定做原创化：替换真名、组织、术语、外观标识，同时保留一个能力机制和一个现实悬疑事件。
