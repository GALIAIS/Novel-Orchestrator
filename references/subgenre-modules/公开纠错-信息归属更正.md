# 公开纠错-信息归属更正

依据案例：`batch-133` 案例 665。

## 读者承诺

发现平台、编辑、渠道、身份等信息有误后，及时删改并公开说明，维护社群信息可信度。

## 核心循环

```text
旧信息发布 -> 发现错误 -> 确认实际情况 -> 删除/更正 -> 公开说明边界
```

## 开篇必备

- 原说法是什么。
- 错误点在哪里。
- 实际情况是什么。
- 已经做了什么处理。
- 哪些信息仍不确定。

## 期待来源

- 读者及时更新认知。
- 作者主动承担信息错误。
- 错误传播被截断。
- 不扩展臆测，保留可信边界。

## 断档风险

- 只删帖不说明。
- 更正时继续夹带未经证实的信息。
- 评价当事人，偏离事实修正。
- 不说明影响范围。

## 训练任务

把一段含错误身份/渠道信息的旧动态改成公开更正说明，必须包含原说法、实际情况、处理动作和不扩展臆测。
