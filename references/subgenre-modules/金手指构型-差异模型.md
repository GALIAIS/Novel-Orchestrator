# 金手指构型-差异模型

依据案例：`batch-007` 案例 034。

## 读者承诺

用“差异”设计金手指：我知道你不知道，我有你没有，我能你不能，我更早、更快、更省成本地解决问题。

## 核心循环

```text
环境难题 -> 主角差异 -> 他人无法复制 -> 主角解决 -> 差异升级
```

## 开篇必备

- 明确差异类型：信息差、资源差、时间差、角色差、环境差、等级差、魅力差。
- 明确别人为什么做不到。
- 明确这个差异能解决什么题材难题。

## 期待来源

- 读者想看主角用差异降维打击。
- 读者想看别人误判差异。
- 读者想看差异扩大到更高层级。

## 断档风险

- 环境中其他人也有同类能力。
- 差异不能解决题材核心问题。
- 只会构建，不知道怎么使用。
- 差异一次用完，没有升级路径。

## 训练任务

把自己的金手指归类为一种差异模型，并设计 3 次差异升级事件。
