# 参考资料-素材功能改造

依据案例：`batch-071` 案例 351。

## 读者承诺

这个模块用于世界观、地图、制度和角色素材积累。它承诺的是不把参考资料当摘抄库，而是提取功能、问题和情绪，再改造成自己的世界规则和事件。

## 核心循环

```text
收集素材 -> 提取功能 -> 统一主题 -> 改造制度/角色/事件 -> 进入长篇结构
```

## 开篇必备

- 素材来源要多样。
- 每条素材只提取一个核心功能。
- 世界观要有统一主题。
- 地图制度服务同一问题。
- 角色参考要转成剧情功能。

## 期待来源

- 参考资料提供真实复杂度。
- 功能改造产生新鲜感。
- 统一主题让世界不散。
- 多地图由同一问题变体展开。
- 角色原型融合后更立体。

## 断档风险

- 素材只做装饰。
- 直接照搬人物外形和设定。
- 每张地图都像另一本书。
- 参考太多但没有主题统合。
- 动笔后才发现世界观不能改。

## 训练任务

找 5 条素材，分别来自榜单、段子、现实制度、角色原型和影视桥段。每条写“功能句”，再统一成一个世界观里的 5 个事件。
