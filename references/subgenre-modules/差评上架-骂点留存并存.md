# 差评上架-骂点留存并存

依据案例：`batch-152` 案例 756。

## 读者承诺

差评作品仍能上架时，不能只看骂声，要同时拆骂点和留存点。毒点、黑流量、猎奇标题、标签受众和行文可读性可能同时存在。

## 核心循环

```text
离谱设定 -> 评论骂点 -> 争议曝光 -> 标签筛选 -> 行文留存 -> 上架数据
```

## 开篇必备

- 明确主要骂点。
- 可见成绩或上架证据。
- 标题和标签承诺。
- 可能的黑流量来源。
- 行文或爽点残留价值。

## 期待来源

读者期待看反差：为什么一本文被骂得厉害，却仍有人订阅。作者要从反面案例里拆出“不能学什么”和“为什么还能动”。

## 断档风险

- 用“读者没脑子”替代分析。
- 只看差评，不看标签承诺。
- 把黑流量当长期能力。
- 新人照抄离谱毒点。

## 训练任务

找一部差评但有数据的作品，列 3 个骂点、3 个留存点、1 个不能模仿点、1 个可借鉴包装点。
