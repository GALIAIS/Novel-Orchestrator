# 开篇危机-金手指破局反转

依据案例：`batch-139` 案例 693。

## 读者承诺

开头快速把主角推入足够高风险的核心危机，再让金手指通过具体过程破局，给读者危机、拉扯、反转和释放。

## 核心循环

```text
事件前移 -> 风险抬高 -> 主角被逼入绝路 -> 金手指介入 -> 过程受阻 -> 反转破局
```

## 开篇必备

- 第一章末尾爆发核心事件。
- 反派目标能逼主角到绝路。
- 失败代价足够明确。
- 金手指破局有过程，不只给结果。
- 危机解决后留下新期待。

## 期待来源

- 读者担心主角如何活下来。
- 金手指能力第一次被验证。
- 反派压迫越强，破局越爽。
- 反转让危机解决显得不是平推。

## 断档风险

- 第二章才爆发真正事件。
- 冲突多而杂，但都不够致命。
- 反派动机太轻，逼不出主角绝境。
- 第五章后人物和事件不服务核心危机。

## 训练任务

把一个十章开头细纲改成危机版：第二章事件前移到第一章末尾，反派动机升级为生死或生存危机，删除不服务危机的人物，并写出“危机升级、尝试失败、关键反转、情绪释放”四步破局。
