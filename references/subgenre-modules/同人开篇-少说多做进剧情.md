# 同人开篇-少说多做进剧情

依据案例：`batch-317` 案例 1582

## 读者承诺

同人读者已经带着原作期待进入，开篇要少解释前世和设定，尽快让主角进入当前世界的危机、遗憾或信息差爽点。

## 核心循环

```text
标题标明IP -> 跳过冗余序章 -> 主角落入当前处境 -> 接轨原作事件 -> 信息差/补遗憾兑现
```

## 开篇必备

- 标题直接带 IP 热词。
- 前世、死亡、神明说明只留一句。
- 第一章交代主角从哪里来、现在怎样、马上面对什么。
- 人设通过事件做出来，不靠作者解释。
- 原作设定介绍只服务当前原创支线。

## 期待来源

读者期待看到原作世界里的新变量：主角怎样补遗憾、改暗线、造信息差，并逐步被原作角色看见价值。

## 断档风险

- 序章拖沓，读者没看到同人爽点。
- 神明或穿越解释讲太满，反而削弱伏笔。
- 对话不推动剧情。
- 原创支线乱改原作主线核心，引发粉丝抵触。
- 标题不带热词，目标读者搜不到也看不懂。

## 训练任务

写一个同人 500 字开头：不写序章，只写主角当前危机、原作相关钩子和一个将来可扩散的信息差。
