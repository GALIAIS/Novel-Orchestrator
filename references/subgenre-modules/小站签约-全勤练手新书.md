# 小站签约-全勤练手新书

依据案例：`batch-116` 案例 577。

## 读者承诺

小站签约上架后的低数据，不一定等于失败。它可以承诺练手、全勤、完结经验和下本准备，而不是承诺立刻起飞。

## 核心循环

```text
签约上架 -> 数据不佳 -> 判断平台背景 -> 当前书练手拿反馈 -> 完结复盘 -> 下本更稳
```

## 开篇必备

- 先确认平台流量基准。
- 不用大站比例直接套小站。
- 能拿全勤就稳定更新。
- 明确本书训练目标。
- 为下本积累开篇、节奏和完结经验。

## 期待来源

- 低数据被转化为学习样本。
- 全勤提供短期稳定收益。
- 完结经验提升下本可靠性。
- 平台适配判断更清楚。

## 断档风险

- 数据一差就全盘否定。
- 只看比例，不看平台流量。
- 小站练手却投入不匹配的野心。
- 不做下本复盘。

## 训练任务

写一份上架后复盘：平台流量、当前比例、全勤价值、本书训练到的能力、下本必须改的三件事。
