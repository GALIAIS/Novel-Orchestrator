# 老作者入库-目标差距自检

依据案例：`batch-058` 案例 290。

## 读者承诺

这个模块适用于已有写作经验但想转型、升级或补方法的作者。它承诺的是按当前目标重新定义短板，而不是纠结自己算新人还是老作者。

## 核心循环

```text
梳理已有成绩 -> 明确未达成目标 -> 找到能力缺口 -> 读取精品资料 -> 选择训练路径 -> 旧项目收尾/新项目转型
```

## 开篇必备

- 写清写作年限、累计字数和已有成绩。
- 写清当前未达成目标。
- 标出旧项目是否需要收尾。
- 先读系统精品资料。
- 把经验转化为当前训练任务。

## 期待来源

- 作者知道自己不是从零开始。
- 过往经验能变成优势。
- 新目标暴露的新短板被看见。
- 资料学习有入口，不靠零散求文件。
- 转型前先处理旧项目，降低烂尾风险。

## 断档风险

- 写过很多字就拒绝补基础。
- 自认新人却不说具体短板。
- 到处求失效资料，不查精品区。
- 旧项目没收尾就盲目开新坑。
- 把社交报道当成训练计划。

## 训练任务

写一份作者目标差距表：已有成绩、目标成绩、差距能力、要读资料、要做训练、旧项目处理方式。每项只写一句。
