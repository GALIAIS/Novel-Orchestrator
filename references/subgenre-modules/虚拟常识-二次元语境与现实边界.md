# 虚拟常识-二次元语境与现实边界

依据案例：`batch-250` 案例 1250。

## 读者承诺

虚拟常识模块区分作品类型语境和现实社交边界，避免把二次元福利桥段误当现实通用规则。

## 核心循环

```text
类型语境 -> 读者接受角色反应 -> 福利/喜剧成立 -> 出圈到现实 -> 边界重新校验
```

## 开篇必备

- 明确作品处于二次元或特定圈层语境。
- 角色反应要符合类型预设。
- 现实风险不能被文本常识掩盖。
- 涉及身体评价时要控制关系、年龄和场合。

## 期待来源

读者在作品内接受夸张角色属性和福利反应，是因为文本已经提供了安全语境和类型约定。

## 断档风险

- 用动漫认识现实。
- 把现实冒犯包装成作品福利。
- 忽略年龄、关系、场合差异。
- 圈层内可接受表达被泛化到所有人。

## 训练任务

选一个二次元福利梗，写出作品内成立条件、现实风险、可保留的角色魅力和降冒犯改法。
