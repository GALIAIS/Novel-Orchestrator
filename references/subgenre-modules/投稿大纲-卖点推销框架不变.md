# 投稿大纲-卖点推销框架不变

依据案例：`batch-126` 案例 630。

## 读者承诺

投稿大纲是给编辑判断潜力的推销材料，不是逐字执行合同。卖点和大框架要稳，细节允许随正文调整。

## 核心循环

```text
已有正文 -> 提炼卖点 -> 写投稿大纲 -> 标出大框架 -> 正文细节调整 -> 保持主线不漂移
```

## 开篇必备

- 先确定核心卖点。
- 大纲突出潜力。
- 前三章承诺要清楚。
- 大框架不能每天变。
- 细节桥段可以调整。

## 期待来源

- 编辑看见作品潜力。
- 作者减少无限修改焦虑。
- 大纲给正文提供方向。
- 框架稳定提升投稿效率。

## 断档风险

- 把大纲当不可改合同。
- 每天改大框架，迟迟不发。
- 大纲没有卖点，只是流水账。
- 正文偏离核心承诺。

## 训练任务

写一页投稿大纲：卖点、主线、前三章承诺、前三个高潮。再列“可改细节”和“不可改框架”。
