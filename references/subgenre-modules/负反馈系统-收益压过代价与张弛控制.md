# 负反馈系统-收益压过代价与张弛控制

依据案例：`batch-253` 案例 1265。

## 读者承诺

负反馈系统模块把扣血、倒计时、代价型系统控制成紧迫感，而不是长期折磨主角的毒点。

## 核心循环

```text
系统代价 -> 生死紧迫 -> 正面收益更大 -> 阶段解决危机 -> 张弛切换
```

## 开篇必备

- 系统先给出明确正面收益。
- 代价不能压过爽点。
- 危机要有阶段性解决节点。
- 系统不要长期强制惩罚主角。

## 期待来源

读者接受代价，是因为系统带来更大的能力、收获和命运掌控感，负压最终能转成爽点。

## 断档风险

- 系统一直负面反馈。
- 自我意识系统话太多。
- 系统左右主角生死，读者憋屈。
- 能力普通，没有新奇卖点。

## 训练任务

设计一个代价型系统：1 个代价、3 个收益、1 个危机缓解节点、1 条系统不能越界的规则。
