# 倒叙开篇-危机前置信息补足

依据案例：`batch-101` 案例 502。

## 读者承诺

先把章回事件中最刺激的危机或结局片段放到开头，让读者带着疑问进入正叙。后续铺垫必须逐项回答开头造成的信息跳级，最后接回危机并继续兑现。

## 核心循环

```text
危机前置 -> 信息跳级 -> 回到起点 -> 逐步补足原因 -> 接回危机 -> 兑现旧恨新仇
```

## 开篇必备

- 前置片段必须足够刺激。
- 前置后至少形成 2-3 个明确疑问。
- 正叙补足不能离题。
- 接回危机时，危机最好仍未彻底解决。
- 倒叙跨度以小单元为宜。

## 期待来源

- 生死危机或强冲突。
- 读者想知道“怎么到这一步”。
- 正叙铺垫变成解谜过程。
- 接回时间线后的清算。

## 断档风险

- 为炫技打乱时间线。
- 前置片段不够强，读者不愿等回忆。
- 回忆太长，忘掉开头危机。
- 危机已经解除才回忆，期待感下降。

## 训练任务

把一个小单元写成两版：正序版和倒叙版。倒叙版开头必须停在危机未解处，后续每一场回忆都标注它回答了哪个疑问。
