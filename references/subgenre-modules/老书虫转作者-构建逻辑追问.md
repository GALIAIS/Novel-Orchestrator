# 老书虫转作者-构建逻辑追问

依据案例：`batch-087` 案例 432。

## 读者承诺

老书虫转作者的关键不是多读几本，而是把“好看”拆成“作者为什么这样安排”。看懂剧情逻辑链，才能把自己脑中的情节排成可读顺序。

## 核心循环

```text
选爽点事件 -> 追问设计原因 -> 找逻辑链起点 -> 拆铺垫和伏笔 -> 记录人物变化 -> 迁移到自己故事
```

## 开篇必备

- 不评价文笔，先拆事件结构。
- 每个爽点都追问为什么发生在这里。
- 找出前面铺了什么，后面伏了什么。
- 标注人物目标和动机变化。

## 期待来源

- 从读者感受转到作者操作。
- 情节从碎片变成顺序。
- 自己能复用别人的构建方式。
- 写作时知道下一步为什么发生。

## 断档风险

- 只会说“文笔不错”。
- 情节很多但没有因果排序。
- 只拆结果，不拆前置铺垫。
- 忽略人物目标变化。

## 训练任务

拆一个熟书里的爽点事件，回答：为什么发生、前面铺了什么、谁的目标变了、谁的动机变了、后面伏了什么。写完再用同结构设计自己的一个事件。
