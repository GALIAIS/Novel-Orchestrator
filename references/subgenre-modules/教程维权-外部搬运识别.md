# 教程维权-外部搬运识别

依据案例：`batch-098` 案例 489。

## 读者承诺

教程、案例、文案也是创作资产。发现外部搬运时，不只靠情绪反应，要保存来源、截图、时间、账号和相似段落，形成可追溯证据链。

## 核心循环

```text
发现眼熟内容 -> 对照原始教程 -> 保存截图链接 -> 标记相似段落 -> 记录删除/互动证据 -> 举报或维权
```

## 开篇必备

- 先保存证据，再讨论。
- 外部链接、账号、时间要完整。
- 相似内容要逐段对照。
- 评论被删也要截图。

## 期待来源

- 知识资产被保护。
- 社群能快速识别搬运。
- 后续举报有证据。
- 自己整理资料时也会更重视来源。

## 断档风险

- 只发泄，不保存证据。
- 二次传播时不标来源。
- 外部评论被删后证据流失。
- 把合理引用误判成抄袭，缺少对照。

## 训练任务

建立一张搬运证据卡：外部链接、账号、截图、疑似段落、原文来源、相似理由、处理动作。每项不能为空。
