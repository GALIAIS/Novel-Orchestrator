# 穿越开局-当前困境优先与金手指工具化

依据案例：`batch-399` 案例 1995

## 读者承诺

先交付穿越后的身份、困境和选择，再让金手指提供可利用的新解法；前史按当前触发后置，系统不替主角决定主线。

## 核心循环

```text
当前困境 -> 最少身份信息 -> 主角先作选择 -> 金手指提供规则能力 -> 主动利用并付代价 -> 外部结果变化 -> 前史触发回收
```

## 开篇必备

- 主角立刻面对可行动的问题。
- 穿越确认过程只保留必要信息。
- 金手指出现前人物已有欲望和选择。
- 能力或规则立即改变解决办法。
- 奖励落到身体、资源、身份或关系。
- 前史只在解释当前因果时出现。

## 期待来源

读者先理解无能力时为什么难，再看主角怎样利用新规则反转外部局面，并从后置前史发现更深动机。

## 断档风险

- 醒来确认流程占满第一章。
- 梦境只是后置说明书。
- 系统任意任务夺走主角因果。
- 人格化系统长期盖过现实关系。
- 面板增长没有可观察结果。
- 主角只和系统斗嘴、不对外行动。

## 训练任务

把穿越开头压成困境、身份、选择、金手指、结果五段共 1800 字，前史限三句；再删一半系统对白，确保主角目标和外部行动仍完整。
