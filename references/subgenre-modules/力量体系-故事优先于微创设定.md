# 力量体系-故事优先于微创设定

依据案例：`batch-326` 案例 1628

## 读者承诺

东方玄幻力量体系要服务故事和金手指爽点；新人不要把微创设定复杂化到压过开篇噱头和人物困境。

## 核心循环

```text
故事困境 -> 金手指功能 -> 力量体系配合 -> 事件中展示规则 -> 爽点回收
```

## 开篇必备

- 先确定主角困境和目标。
- 金手指配合故事设计。
- 体系规则只展示当前事件必须用到的部分。
- 开篇先给噱头，再补设定。
- 参考书只拆体系如何服务剧情。

## 期待来源

读者期待看到主角用体系变强、破局和获得爽点，而不是先读完整设定说明。

## 断档风险

- 新人几万字打磨世界观，开头没有吸引力。
- 武魂、功法、世界生态只借一部分，变成四不像。
- 体系复杂但没有好点子、金手指或人设。
- 规则讲得严谨，读者却还没关心主角。

## 训练任务

把修炼体系压成 5 句，只保留第一章困境和第一次爽点需要的规则。
