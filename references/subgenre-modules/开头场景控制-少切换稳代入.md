# 开头场景控制-少切换稳代入

依据案例：`batch-256` 案例 1277。

## 读者承诺

开头场景控制模块让第一场景先稳住主角、困境和期待，再决定是否切换。

## 核心循环

```text
第一场景进入 -> 主角目标明确 -> 当前冲突推进 -> 留下未解问题 -> 切场景承接
```

## 开篇必备

- 第一场景先完成基本功能。
- 新地点、新人物、新设定不要同时堆。
- 切场景必须带着问题过去。
- 主角视角和第一冲突优先。

## 期待来源

读者在一个稳定场景里理解主角处境，期待没有被频繁换地点打断。

## 断档风险

- 开头场景变化太多。
- 代入刚建立就切走。
- 场景切换只为堆世界观。
- 读者换场景后忘记主角目标。
