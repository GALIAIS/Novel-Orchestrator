# 投稿大纲-模板自检与编辑理解成本

依据案例：`batch-232` 案例 1156。

## 读者承诺

投稿大纲模板的价值是降低编辑理解成本，并暴露作者项目设计缺口。

## 核心循环

```text
准备投稿 -> 填模板栏目 -> 暴露缺口 -> 补齐主线卖点 -> 再提交
```

## 开篇必备

- 一句话卖点清楚。
- 主角目标清楚。
- 金手指或核心机制清楚。
- 前期事件和后续规划能概括。

## 期待来源

编辑能快速判断题材、卖点、主线和可持续性，作者也能知道自己哪里没想清。

## 断档风险

- 把模板当万能投稿捷径。
- 正文有想法，大纲说不清。
- 链接资料不可读却硬填细节。
- 只写前三章，没有后续承诺。

## 训练任务

写一页投稿大纲自检表：卖点、主角目标、金手指、前三章、前十万字、后三十万字。填不出的栏先补设计。
