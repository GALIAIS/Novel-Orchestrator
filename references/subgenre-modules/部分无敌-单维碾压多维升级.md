# 部分无敌-单维碾压多维升级

依据案例：`batch-091` 案例 453。

## 读者承诺

主角可以在一个维度无敌，用来提供碾压爽点；同时在地位、资源、人脉、队伍、舞台等维度继续升级，保证长线内容不断。

## 核心循环

```text
单维无敌 -> 当前舞台碾压 -> 暴露其他短板 -> 非实力升级 -> 更大舞台 -> 再次相对无敌
```

## 开篇必备

- 明确主角无敌的是哪一维。
- 明确至少两个仍需升级的维度。
- 当前舞台要能让主角碾压。
- 下个舞台要比当前更大。
- 碾压后要有资源或地位收获。

## 期待来源

- 主角如何花式打脸。
- 反派失势后的情绪回报。
- 非实力维度如何补齐。
- 团队或势力如何升级。

## 断档风险

- 绝对无敌到没有舞台升级。
- 升级维度改变整本书主题。
- 群像转得太早。
- 碾压缺少铺垫和情绪回报。

## 训练任务

设计一个单维无敌主角，写三个舞台。每个舞台写一次碾压、一次反派失势、一次非实力升级。
