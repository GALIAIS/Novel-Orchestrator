# 游戏玄幻融合-先定主骨架

依据案例：`batch-328` 案例 1638

## 读者承诺

游戏和玄幻可以融合，但必须先定主骨架；市场少见往往意味着难写、少借鉴或有毒。

## 核心循环

```text
游戏入口 -> 玄幻规则理解 -> 标签风格冲突 -> 明确主骨架 -> 控制割裂感
```

## 开篇必备

- 选择主骨架：玄幻、双世界、无限副本。
- 说明游戏元素承担什么功能。
- 说明玄幻元素承担什么功能。
- 现代语言只在合适阶段使用。
- 找邻近样本拆结构。

## 期待来源

读者期待游戏机制帮助理解或攻略玄幻危机，而不是两套风格互相打架。

## 断档风险

- 以为市场少就是蓝海。
- 现实吐槽长期破坏玄幻氛围。
- 双世界都写不好。
- 没有可拆样本，新人试错成本过高。
- 游戏机制只是噱头，没有实际爽点。

## 训练任务

写三版融合设定：游戏过渡、双世界穿梭、玄幻副本化；每版列入口、爽点、最大风险。
