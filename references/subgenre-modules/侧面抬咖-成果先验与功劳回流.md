# 侧面抬咖-成果先验与功劳回流

依据案例：`batch-418` 案例 2088

## 读者承诺

主角的影响力先由独立成果证明，再由来源揭示完成抬升。主角留下的方法、工具或判断规则被有自身目标的配角适配使用，第三方在不知道来源时先检验和评价成果；熟悉痕迹随后暴露源头，功劳、信任、地位或警惕才回流主角，同时保留配角的劳动和收益。

## 核心循环

```text
主角留下具体可复用的方法、工具或规则 -> 配角在独立问题中选择采用并作适配 -> 配角承担风险产出可检验成果 -> 第三方在未知来源时先评价成果 -> 措辞、标记或操作暴露源头 -> 配角说明主角贡献并保留自身劳动 -> 声望、信任或警惕回流主角 -> 方法被更多人改造、竞争或误用 -> 主角处理影响力带来的责任和冲突
```

成果先验避免自夸，配角能动性提高可信度，延迟揭源提供发现爽，功劳与责任共同回流则把一次反应扩展成长期关系变化。

## 开篇必备

- 主角早期留下的具体方法、工具或规则。
- 传播物最初解决的问题和使用边界。
- 有独立目标的配角及其新问题。
- 配角选择采用的理由。
- 配角作出的适配、劳动和风险承担。
- 第三方可独立检查的成果。
- 第三方在不知道来源时的先行评价。
- 读者能提前认出的措辞、标记或操作痕迹。
- 延迟揭源和功劳分配方式。
- 回流后改变的关系、地位、敌意或责任。

## 期待来源

- 配角能否把主角方法迁移到新问题。
- 适配后的成果是否经得住第三方检验。
- 读者会比人物提前多久认出来源。
- 来源揭示后功劳如何在两人间分配。
- 主角会如何回应超出视野的影响。
- 新盟友或竞争者是否因此出现。
- 方法扩散后会产生什么变体和误用。

## 断档风险

- 只有围观者震惊，没有可检查成果。
- 配角只复读主角原话，没有自己的目标和劳动。
- 开头直接宣布来源，失去发现过程。
- 所有成功都被算给主角，配角沦为宣传工具。
- 主角亲口解释自己多有影响力。
- 功劳回流不改变任何关系或局面。
- 方法扩散永远正确，没有变体、竞争和责任。
- 把未经授权搬运包装成正常致敬。
- 将“主角不犯蠢”机械化为主角永不犯错。

## 训练任务

写一个 2500 字单元，主角在前半段不得出场。配角使用主角早先留下的具体方法解决新问题，并加入自己的改造；第三方先检验成果，后半段才通过一个痕迹揭示来源。结尾必须同时给予配角独立收益、让部分功劳回流主角，并产生一个由方法扩散引发的新竞争或责任。
