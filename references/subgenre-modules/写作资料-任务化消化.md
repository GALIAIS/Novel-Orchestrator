# 写作资料-任务化消化

依据案例：`batch-035` 案例 172。

## 读者承诺

资料库的价值不在收藏数量，而在每次解决一个写作问题。读一段理论，就要产出一张卡、一次修改或一个训练任务。

## 核心循环

```text
当前短板 -> 选择资料章节 -> 摘一个概念 -> 转成写作任务 -> 验收结果
```

## 开篇必备

- 先写当前要解决的问题。
- 一次只读一个章节或概念。
- 摘要必须落到网文场景。
- 当天有应用动作。

## 期待来源

- 资料不再压迫作者。
- 理论能立刻改正文或小纲。
- 读书路径服务短板。
- 每张卡都能复用。

## 断档风险

- 囤资料带来虚假进步。
- 同时读太多书，什么都没应用。
- 笔记只有摘抄，没有训练。
- 通用编剧理论不做网文转译。

## 训练任务

读一个写作概念后，写出“概念摘要、网文对应场景、今天修改动作、验收标准”四行。
