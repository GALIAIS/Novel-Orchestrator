# 开头处刑-冲突前置与读完率

依据案例：`batch-226` 案例 1130。

## 读者承诺

开头先让第一个故事抓住读者，再铺设定、日常和伏笔。

## 核心循环

```text
开头内容标记 -> 冲突前置 -> 压缩废话 -> 读完率验证 -> 局部回修
```

## 开篇必备

- 第一章前 1/3 出现明确矛盾、压力或悬念。
- 设定和日常并入冲突。
- 伏笔不能替代当前钩子。
- 用后台读完率判断流失点。

## 期待来源

读者先被第一个故事吸住，才愿意看后续世界和设定。

## 断档风险

- 第八九章才出真正矛盾。
- 前面全是设定、日常和伏笔。
- 作者体感不拖，读者已退出。
- 已签约后过度回头，影响继续更新。

## 训练任务

拿前三千字做“开头处刑”：标红多余描写，标黄设定/日常，标绿冲突；把第一个故事钩子提前到前 1/3 章。
