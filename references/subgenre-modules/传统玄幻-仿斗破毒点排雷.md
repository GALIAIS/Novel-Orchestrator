# 传统玄幻-仿斗破毒点排雷

依据案例：`batch-064` 案例 318。

## 读者承诺

这个模块用于传统玄幻、废柴退婚、家族开篇和仿斗破结构。它承诺的是只学习经典开篇的功能：悬念、压制、仇恨、金手指、反打节奏；不照搬剧情壳，更不把英雄救美、女友背叛、主角自作自受等毒点塞进第一章。

## 核心循环

```text
低位压制 -> 原因悬念 -> 不公平伤害 -> 金手指线索 -> 仇恨集中 -> 快速反击期待
```

## 开篇必备

- 书名或简介要看得出卖点。
- 金手指或核心异常尽早出现。
- 主角下行原因不能显得自作自受。
- 路人和对手压制要足够明确。
- 第一章设定只保留当前冲突需要的部分。

## 期待来源

- 退步原因悬念。
- 被不公平羞辱后的反击。
- 金手指揭露。
- 对手或退婚方承担仇恨。
- 主角状态反转。

## 断档风险

- 大段设定挤掉冲突。
- 英雄救美导致主角像圣母或大冤种。
- 已有女友背叛触发绿帽敏感。
- 只学斗破剧情，不学节奏功能。
- 极度下行后不立刻给反击希望。

## 训练任务

把一个传统玄幻开篇改成 800 字功能梗概：低位、压制、悬念、金手指线索、反击期待都要出现；删除所有当前冲突用不到的设定。
