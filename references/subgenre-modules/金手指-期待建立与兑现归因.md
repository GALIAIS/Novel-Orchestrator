# 金手指-期待建立与兑现归因

依据案例：batch-372 案例 1857

## 读者承诺

金手指出现后，读者会期待主角用它解决冲突。高潮可以绕路，但最终因果必须回到金手指或主角过人之处上。

## 核心循环

```text
金手指亮相 -> 建立解决预期 -> 冲突压迫 -> 金手指参与因果 -> 爽点兑现
```

## 开篇必备

- 金手指的可感知价值。
- 它能解决的问题类型。
- 暂时不能完全解决的限制。
- 第一次小兑现。

## 期待来源

- 主角不可替代性。
- 逆转大局的可能。
- 读者对能力成长的预判。
- 冲突和能力之间的因果吻合。

## 断档风险

- 高潮靠无关好运解决。
- 金手指只摆设，不参与关键因果。
- 主角换成配角也能完成爽点。
- 金手指太弱或太无趣，无法逆转局面。

## 训练任务

设计一个金手指和一个强敌冲突，写明读者第一眼期待，并让最终胜利必须由金手指直接或间接导致。
