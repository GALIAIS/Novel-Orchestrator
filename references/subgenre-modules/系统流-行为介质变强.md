# 系统流-行为介质变强

依据案例：`batch-007` 案例 033。

## 读者承诺

系统不是面板装饰，而是一条可重复运行的主线机制：做什么、得到什么、如何变强。

## 核心循环

```text
特定行为 -> 获得介质 -> 消耗/转化介质 -> 变强 -> 更高层行为
```

## 开篇必备

- 主线行为清晰。
- 介质或奖励清晰。
- 变强路径清晰。
- 行为具备正当性或读者认可的爽感。

## 期待来源

- 读者知道主角下一步该做什么。
- 每次行为都有收获。
- 系统机制能倒推出长线事件。

## 断档风险

- 系统只发奖励，不产生主线。
- 介质和变强无关。
- 行为缺乏正当性，读者不认。
- 系统替代太多步骤，主角失去主动性。

## 训练任务

给一个系统写“行为、介质、变强、正当性、下一轮事件”五项。缺一项就补设定。
