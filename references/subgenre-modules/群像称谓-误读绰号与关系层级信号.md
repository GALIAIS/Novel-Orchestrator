# 群像称谓-误读绰号与关系层级信号

依据案例：`batch-403` 案例 2013

## 读者承诺

让正式名、头衔、绰号、亲昵称和敌称分别对应使用者与场合；昵称由误读或共同事件生成，经人物反应设定边界，再以改称显示关系、权力和身份变化。

## 核心循环

```text
正式身份建立 -> 误读或共同事件产生绰号 -> 人物反应设定边界 -> 特定群体重复沉淀历史 -> 新人获得或失去使用资格 -> 危机改称暴露关系与身份
```

## 开篇必备

- 首次出现绑定正式姓名与一个主称谓。
- 绰号有可见起源事件。
- 被称呼者明确接受、纠正或拒绝。
- 记录谁能在公开和私下使用。
- 每次改称对应关系或场合变化。
- 活跃别名数量服从读者辨识。

## 期待来源

共享绰号制造群体历史和亲切感，同一个称呼在新人、挚友和敌人口中又拥有不同权利；突然改回全名会立即带来关系拉扯。

## 断档风险

- 人物表宣布绰号却从不展示形成。
- 所有人无差别使用内部称谓。
- 随机轮换别名造成辨识困难。
- 笑点没有角色反应和边界。
- 冒犯性称呼被自动解释成亲密。
- 称谓变化不对应任何状态改变。

## 训练任务

为五人群像制作称谓矩阵，并给每个内部称谓写起源事件、允许者和禁用者；用三段对话展示新人、挚友和敌人对同一角色的不同关系。
