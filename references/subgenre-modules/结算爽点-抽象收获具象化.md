# 结算爽点-抽象收获具象化

依据案例：`batch-025` 案例 121。

## 读者承诺

主角赢得抽象收益后，立刻让读者看到它换来了什么具体生活、资源、身份和他人态度变化。

## 核心循环

```text
胜利 -> 抽象奖励 -> 配角解释 -> 具象福利 -> 主角补充隐藏价值 -> 后续使用
```

## 开篇必备

- 冲突结果清楚。
- 奖励名称清楚。
- 具体好处清楚。
- 因果关系点明。

## 期待来源

- 配角震惊替读者估价。
- 具象福利让爽感落地。
- 主角知道更深一层，拉后续期待。
- 下一场冲突能使用这份收获。

## 断档风险

- 只写升官、权限、名望，不写具体变化。
- 福利出现但没有和奖励建立因果。
- 配角解释过长，像设定科普。
- 结算后收获不再发挥作用。

## 训练任务

写一个结算桥段：主角获得“内门名额”后，用住宿、资源、师姐态度、任务权限四个方面具象化它的价值。
