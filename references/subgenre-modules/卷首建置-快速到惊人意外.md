# 卷首建置-快速到惊人意外

依据案例：`batch-081` 案例 402。

## 读者承诺

这个模块用于处理新卷开头难写、数据下降、人物生涩的问题。它承诺把新卷第一幕压缩成快速建置，并尽快抵达第一个惊人意外。

## 核心循环

```text
旧期待收束 -> 新场景建置 -> 新人物入场 -> 新伏笔落点 -> 惊人意外
```

## 开篇必备

- 明确上一卷留下的读者惯性。
- 新卷第一章给出新场景。
- 新配角必须有一到两个难忘特点。
- 新伏笔不能只摆设，要服务惊人意外。
- 建置章节数要有上限。

## 期待来源

- 新卷目标尽快出现。
- 新人物有可记忆点。
- 读者知道新地图有什么问题。
- 惊人意外重新抬高追读。
- 旧配角离场后，新关系能接住情绪。

## 断档风险

- 新卷只做说明和铺排。
- 新配角没有特点。
- 伏笔多但没有近期刺激。
- 作者和角色不熟，导致对话生涩。
- 建置过长，读者从上一卷高潮掉下去。

## 训练任务

为新卷写 5 章建置表。每章必须有一个新刺激；第 3-5 章内必须出现惊人意外，否则压缩场景和人物介绍。
