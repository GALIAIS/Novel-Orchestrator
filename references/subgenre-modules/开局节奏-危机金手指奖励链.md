# 开局节奏-危机金手指奖励链

依据案例：`batch-191` 案例 953。

## 读者承诺

开局生死危机和金手指出现后，必须尽快用金手指解决危机并给奖励反馈，拉起后续期待。

## 核心循环

```text
危机 -> 金手指 -> 解决 -> 奖励 -> 后续装逼期待
```

## 开篇必备

- 第一章危机明确。
- 金手指出现带来希望。
- 第二章验证金手指价值。
- 解决后有奖励或能力反馈。

## 期待来源

读者看到金手指不只是设定，而是能解决问题、带来收益、支撑后续爽点。

## 断档风险

- 金手指出现后不验证。
- 解决危机没有奖励。
- 第三章开始讲设定，期待链断掉。
- 新地图铺垫替代当前爽点兑现。

## 训练任务

写前三章小纲：第一章危机和金手指，第二章解决和奖励，第三章用奖励引出下一事件。
