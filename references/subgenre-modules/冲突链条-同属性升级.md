# 冲突链条-同属性升级

依据案例：batch-274 案例 1367

## 读者承诺

让冲突升级沿着清晰链条递进：生存链、信任链、资源链、权力链可以交叉，但每条链内部要有自己的压力加码逻辑。

## 核心循环

```text
链条命名 -> 初级压力 -> 同属性加码 -> 更大代价 -> 本卷高压节点
```

## 开篇必备

- 先给主要冲突链命名。
- 区分阻碍和升级。
- 每次升级都要更大、更近或更难解决。

## 期待来源

读者期待看到主角解决一个问题后，同类压力变得更大，主角能力、地位和选择也随之升级。

## 断档风险

- 把任意新麻烦都当升级。
- 多条链混在一起，读者觉得散。
- 提纲里没有升级逻辑，卷中段乱跳。

## 训练任务

列出生存链和信任链各 4 级压力，再标出两条链在哪些章节交叉。
