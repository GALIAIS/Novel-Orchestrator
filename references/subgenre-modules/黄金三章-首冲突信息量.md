# 黄金三章-首冲突信息量

依据案例：`batch-054` 案例 270。

## 读者承诺

黄金三章不按机械字数判断，而按首个冲突是否闭环、信息量是否足、期待感是否成立判断。

## 核心循环

```text
引发事件 -> 金手指出场 -> 主角破局/装逼 -> 小结尾 -> 新期待
```

## 开篇必备

- 第一冲突最好一万字内结束。
- 开篇小剧场要有起承转合。
- 4000 字不长，没信息才长。
- 题材核心和情节爽点都要出现。
- 小结尾要让读者愿意看下一段。

## 期待来源

- 引发事件给危机。
- 金手指给新鲜感。
- 装逼破局给爽点。
- 小结尾给阶段满足。
- 新期待给追读理由。

## 断档风险

- 只纠结字数，不检查情节。
- 信息量不足，读者直接走。
- 光有核心设定没有情节爽点，会干。
- 光有情节爽点没有核心卖点，会散。

## 训练任务

规划一个 6000 字黄金三章：每 1000 字写一个信息点，并标注引发、金手指、装逼、小结尾的位置。
