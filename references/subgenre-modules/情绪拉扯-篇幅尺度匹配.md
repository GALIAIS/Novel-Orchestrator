# 情绪拉扯-篇幅尺度匹配

依据案例：`batch-018` 案例 086。

## 读者承诺

让作者知道情绪拉扯不是固定三章或固定五段，而是按照事件尺度决定篇幅。小情节追求紧凑，大情节追求分层推进。

## 核心循环

```text
判断事件尺度 -> 标出折线功能 -> 分配篇幅 -> 删除重复情绪 -> 接入下一期待
```

## 开篇必备

- 当前事件的得失。
- 读者已经形成的期待。
- 这次拉扯服务的是小爽点、阶段高潮还是大情节转折。
- 每次上行/下行带来的新增变化。

## 期待来源

- 读者想看主角解决当前压制。
- 读者想看多次反转后的兑现。
- 读者想看大情节里关系、资源、危险逐步升级。

## 断档风险

- 小情节被拖成多章，导致重复压制。
- 大情节压缩成一章，导致转折没有重量。
- 每条线只改情绪词，不改局面。
- 篇幅服务作者设计感，不服务读者期待。

## 训练任务

同一个五折线情节写两个版本：800 字紧凑版和 5000 字分层版。分层版每一段必须新增局面变化，不能只重复嘲讽和震惊。
