# 新人剧情-桥段素材库主导

依据案例：`batch-055` 案例 271。

## 读者承诺

这个模块解决新人“懂理论但写不出剧情”的问题。它承诺的不是立刻获得高级角色驱动能力，而是先用桥段素材库把剧情稳定跑起来：先有可调用事件，再谈人物复杂度。

## 核心循环

```text
同题材阅读 -> 拆出桥段功能 -> 记录成素材卡 -> 替换人物/设定/目标 -> 写出可推进事件 -> 回收为自己的桥段库
```

## 开篇必备

- 明确一个同题材方向，避免跨题材乱拆。
- 开篇先确定主角目标和第一个事件，不先追求角色自由发挥。
- 至少准备 5 个同题材可迁移桥段。
- 每个桥段写明触发条件、阻碍、转折和兑现。
- 人设先服务剧情功能，避免新人阶段主线失控。

## 期待来源

- 读者看到主角进入熟悉但有压力的事件。
- 桥段本身带有清晰的目标、阻碍和结果想象。
- 人物性格通过事件选择显现，而不是先靠设定表说明。
- 旧桥段被换成新设定后产生微创新。
- 每个事件都能把主线往前推一步。

## 断档风险

- 只记理论，不记录桥段。
- 只复述剧情，不拆功能。
- 人物一自由发挥，主线立刻偏离。
- 素材库太少，写几章后只能重复同一种冲突。
- 桥段替换时只换名词，没有换目标和阻碍。

## 训练任务

拆 20 个同题材桥段，每个桥段写成 100-200 字素材卡。任选 3 个桥段改成自己的主角和设定，连续写成 3 个小事件，检查主线是否仍然稳定前进。
