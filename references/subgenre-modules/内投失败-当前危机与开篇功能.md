# 内投失败-当前危机与开篇功能

依据案例：`batch-228` 案例 1137。

## 读者承诺

内投失败后优先检查开篇是否进入当前主角、冒险世界、危机和金手指期待。

## 核心循环

```text
失败复盘 -> 删前史 -> 聚焦当前危机 -> 出金手指期待 -> 形成第一选择
```

## 开篇必备

- 视角尽快落到穿越后的主角。
- 前一千字进入当前困境。
- 危机集中在主角身上。
- 第一章给金手指或明确期待。

## 期待来源

读者先要知道主角现在怎么危险、有什么翻盘可能，而不是先看长篇前史。

## 断档风险

- 用古典叙事替代网文开篇功能。
- 一千字还在讲穿越前。
- 视角不在主角当前处境。
- 文风讨论盖过故事不清。

## 训练任务

删掉投稿开头所有穿越前说明，用 500 字写出主角当前处境、生死压力、可见金手指期待和第一个选择。
