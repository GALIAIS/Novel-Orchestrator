# 迪化开篇-事件展示金手指

依据案例：`batch-111` 案例 554。

## 读者承诺

开篇不要先解释金手指原理，而要把金手指放进事件里展示。主角真实目的可以很简单，但外界过度解读，形成迪化和地位升级。

## 核心循环

```text
事件危机 -> 主角自救 -> 金手指显效 -> 旁人误解 -> 地位升级 -> 新误会
```

## 开篇必备

- 一个具体场景。
- 一个迫使主角行动的危机。
- 金手指在行动中展示。
- 主角真实目的和外界理解不同。
- 危机后果要能支撑动机。

## 期待来源

- 读者看事件而不是说明书。
- 误打误撞带来喜剧和爽感。
- 旁人迪化抬高主角。
- 地位升级自然开启下一轮。

## 断档风险

- 开头只解释金手指。
- 人物很多但没有行动焦点。
- 危机后果太模糊，主角动机不足。
- 迪化没有后续兑现。

## 训练任务

把一个金手指说明段改成事件：主角只做一件自救行为，至少两个旁观者给出错误但合理的高估解读。
