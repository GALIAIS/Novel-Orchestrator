# 口碑推荐样本-从候选到可拆证据

依据案例：batch-392 案例 1958

## 读者承诺

单句荐书先以低权重候选保存，把赞美词改成可检查问题；只有直接阅读获得连续场景证据后，才允许升级为技法案例。

## 核心循环

```text
记录书名口碑 -> 赞美标签问题化 -> 人工阅读限定范围 -> 保存直接证据 -> 验证后升级或继续降权
```

## 开篇必备

- 推荐者原始说法与未验证标签。
- 明确到章节的验证范围。
- 标签对应的频率、结构或效果问题。
- 成立、部分成立、不成立或缺证据的结论。

## 期待来源

- 口碑帮助安排阅读优先级。
- 验证问题防止先入为主扩写。
- 连续文本区分高光与稳定机制。
- 样本权重与真实证据匹配。

## 断档风险

- 把一句“写得好”扩成完整理论。
- 根据书名猜测具体玩法。
- 只看单个高光就评价全书。
- 未直接阅读便生成技法模块。

## 训练任务

为三条单句荐书把形容词各改成三个问题，人工读前三章后逐项判定；未完成验证的样本只进入候选队列。
