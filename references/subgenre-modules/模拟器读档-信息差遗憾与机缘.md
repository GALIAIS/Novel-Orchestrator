# 模拟器读档-信息差遗憾与机缘

依据案例：`batch-224` 案例 1119。

## 读者承诺

每轮读档都带来新信息、新选择、新遗憾修正或新机缘抢占，避免重复枯燥。

## 核心循环

```text
失败死亡 -> 保留信息 -> 重开选择 -> 修正遗憾/抢机缘 -> 通关奖励
```

## 开篇必备

- 明确读档触发条件和代价。
- 写清主角能保留什么信息。
- 每轮安排不同变量。
- 奖励要回到现实主线或长期成长。

## 期待来源

读者期待的是“这次不同”：上轮失败的信息如何变成这轮的优势。

## 断档风险

- 反复重走同一剧情。
- 只有试错，没有情感遗憾。
- 奖励单一，缺乏成长反馈。
- 主角只是被系统推着走。

## 训练任务

写一个 300 字模拟器设定，列出保留信息、重来代价、每轮新变化、奖励机制、避免重复方法；再写三轮读档差异。
