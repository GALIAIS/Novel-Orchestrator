# 设定虑点-焦虑爽点循环

依据案例：`batch-011` 案例 053。

## 读者承诺

设定必须制造焦虑，主角解决焦虑才会产生爽点。没有虑点的设定只是装饰。

## 核心循环

```text
设定 -> 虑点 -> 主角行动/金手指 -> 焦虑平复 -> 爽点 -> 新虑点
```

## 开篇必备

- 书名简介先承诺核心虑点。
- 正文前三章展示具体焦虑。
- 金手指必须能解决虑点。
- 解决后接新虑点。

## 期待来源

- 读者认可主角确实有压力。
- 读者想看压力如何被解决。
- 焦虑越合理，解决越爽。
- 虑点不断接力，期待不断档。

## 断档风险

- 设定很大，但不制造情绪缺口。
- 虑点读者不认可。
- 金手指和虑点无关。
- 解决后没有新焦虑。

## 训练任务

列 5 个设定，每个写“焦虑、解决方式、爽点、新焦虑”。写不出来的设定后置。
