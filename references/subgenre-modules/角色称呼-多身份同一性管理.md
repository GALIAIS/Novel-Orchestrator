# 角色称呼-多身份同一性管理

依据案例：`batch-110` 案例 548。

## 读者承诺

一个角色可以有本名、化名、称号和职位称谓，但读者必须始终知道这是同一个人。称呼增加沉浸感之前，先要保证识别清楚。

## 核心循环

```text
多身份角色 -> 设主称呼 -> 控制称呼切换 -> 给同一性锚点 -> 降低识别成本
```

## 开篇必备

- 每卷确定一个主称呼。
- 新称呼第一次出现要解释。
- 谁使用哪个称呼要固定。
- 不为炫设定频繁换名。
- 重要场景优先清楚。

## 期待来源

- 身份层次清楚。
- 读者不被名字绊住。
- 称号变化能体现成长。
- 化名和职位服务剧情。

## 断档风险

- 同一角色称呼过多。
- 读者误以为是不同人。
- 称号升级太频繁。
- 沉浸感变成阅读负担。

## 训练任务

给一个多身份角色做称呼清单：保留 1 个主称呼、2 个场景称呼，其余全部暂时不用。
