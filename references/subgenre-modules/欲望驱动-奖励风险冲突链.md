# 欲望驱动-奖励风险冲突链

依据案例：`batch-176` 案例 877。

## 读者承诺

情节用主角欲望牵引读者，再用奖励、风险、转折和结果形成可追读的冲突脊椎。

## 核心循环

```text
欲望 -> 奖励 -> 风险 -> 冲突升级 -> 转折 -> 爽点兑现 -> 后续影响
```

## 开篇必备

- 主角必须有明确想要的东西。
- 奖励让读者期待得到。
- 风险让读者担心失去。
- 结果要改变主角处境。

## 期待来源

读者期待主角带着强欲望穿过风险，最终拿到奖励或获得更大目标。欲望越原始，读者越容易理解。

## 断档风险

- 只有事件，没有欲望。
- 只有奖励，没有风险。
- 只有爽点，没有后续影响。
- 欲望不升级，长线主线断开。

## 训练任务

把一个情节拆成七项：欲望、奖励、风险、冲突、转折、结果、后续影响。缺哪项就先补哪项。
