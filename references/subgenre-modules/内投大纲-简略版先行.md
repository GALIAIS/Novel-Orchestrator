# 内投大纲-简略版先行

依据案例：`batch-158` 案例 787。

## 读者承诺

内投大纲先满足编辑判断项目方向，不必被复杂模板吓住；照已有范例写简略版即可。

## 核心循环

```text
格式焦虑 -> 参考样例 -> 简略整本大纲 -> 卷纲波折检查 -> 后续工具化
```

## 开篇必备

- 书名和题材。
- 核心卖点。
- 主角。
- 核心矛盾。
- 主线方向。
- 第一卷目标。
- 阶段剧情和结局方向。

## 期待来源

编辑期待快速判断项目是否成立，作者期待用低成本版本理清方向。

## 断档风险

- 把开头、卷纲、细纲、整本大纲混成一团。
- 照网上复杂模板堆字段。
- 后文没想完就完全不敢写。
- 只解决格式，不检查波折。

## 训练任务

用 500 字写一版简略整本书大纲，再把卷纲按“计划 -> 意外 -> 损失/收益 -> 新目标”重排。
