# 智能外挂-决策边界与主体性控制

依据案例：`batch-212` 案例 1058。

## 读者承诺

智能外挂可以陪伴、吐槽、辅助战斗，但不能替主角做关键选择。

## 核心循环

```text
智能外挂 -> 边界限制 -> 收益绑定 -> 辅助功能 -> 主角决策
```

## 开篇必备

- 写清外挂能做什么。
- 写清外挂不能做什么。
- 绑定主角收益和生死。
- 最终选择必须由主角完成。

## 期待来源

读者享受外挂陪伴和辅助，但仍相信主角是故事主体。

## 断档风险

- 外挂抢主角决策权。
- 类分身设定踩主体雷。
- 吐槽役盖过主角。
- 外挂功能过多变第二主角。

## 训练任务

写智能外挂边界协议：5 条能做、5 条不能做、3 条收益绑定、3 个越权风险。
