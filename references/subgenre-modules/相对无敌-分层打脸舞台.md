# 相对无敌-分层打脸舞台

依据案例：`batch-105` 案例 522。

## 读者承诺

主角在当前舞台永远能赢，但远处永远有更高层。读者既不被当前强者压抑，又知道长线还有上升空间。

## 核心循环

```text
当前舞台 -> 仇恨铺垫 -> 主角相对无敌 -> 高级打脸 -> 资源回收 -> 下一层舞台
```

## 开篇必备

- 当前敌人不能强过主角太多。
- 远处更强者只露影子。
- 两三章内铺垫仇恨。
- 一章内兑现打脸。
- 新获得资源马上用上。

## 期待来源

- 无压抑的当前爽感。
- 更高层级的远期期待。
- 密集打脸。
- 小弟和势力回流使用。

## 断档风险

- 强者提前下场压主角。
- 升级后没有舞台使用。
- 打脸只循环二代喊人。
- 扣小细节，忽略架构逻辑。

## 训练任务

设计三个舞台层级，每层写一个敌人、一个仇恨、一个打脸对象和一个回收资源。主角只面对当前层能赢的人。
