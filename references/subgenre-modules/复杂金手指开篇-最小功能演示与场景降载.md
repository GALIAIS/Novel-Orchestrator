# 复杂金手指开篇-最小功能演示与场景降载

依据案例：`batch-409` 案例 2041

## 读者承诺

复杂能力在开篇只演示一项核心功能，用主角的需要、选择、代价和可验证结果完成规则教学；场景限制地点、人物和危险，再由小危机逐步升级到大事件。

## 核心循环

```text
有限场景出现小危机 -> 常规办法不足 -> 主角测试一项核心功能 -> 输入处理代价随动作显形 -> 输出改变结果 -> 局部危机解决 -> 能力边界暴露 -> 更大需求开启下一阶段
```

## 开篇必备

- 金手指完整功能树和首轮唯一功能。
- 一个能立即验证输出的小问题。
- 主角主动测试而非被动接收。
- 输入、成本、结果和限制清楚。
- 一个地点、少量人物和单一危险。
- 成功后自然出现的下一项需求。

## 期待来源

读者先理解主角缺什么，再看常规方案失败和能力输出形成鲜明对照；当前危机得到回报，能力限制与更大灾难继续拉动开发期待。

## 断档风险

- 开篇一次解释完整功能树。
- 大场面同时加载死亡、复活、群战和世界史。
- 主角旁观别人解决核心冲突。
- 规则与当前选择和结果无关。
- 只分短段却不删除重复信息。
- 第一次成功没有成本、边界或后续需求。

## 训练任务

把一个大场面开篇压成 1500 字，只保留一项危险和能力的一项功能；逐段标注推进、判断、规则、情绪或后果中的唯一职责，删除不产生因果增量的说明。
