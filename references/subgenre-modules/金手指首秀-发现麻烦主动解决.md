# 金手指首秀-发现麻烦主动解决

依据案例：`batch-119` 案例 594。

## 读者承诺

金手指首秀要尽快证明“它能解决主角眼前的问题”。读者不需要先听完整说明，先看主角主动使用、获得结果、学会一条规则。

## 核心循环

```text
发现能力 -> 遭遇麻烦 -> 主角主动尝试 -> 规则生效 -> 麻烦解决 -> 留下升级期待
```

## 开篇必备

- 发现能力和遇到麻烦可以互换顺序。
- 麻烦必须足够近，最好就在当前场景。
- 主角要主动使用，不是被系统托管。
- 结果要让读者立刻看懂能力价值。
- 首秀应服务第一个单元故事。

## 期待来源

- 第一次使用带来的确认感。
- 读者期待能力还有哪些边界。
- 小麻烦解决后，引出更大矛盾。
- 主角主动选择建立代入感。

## 断档风险

- 六七千字还没看到能力有效。
- 首秀只是说明书，没有事件结果。
- 小矛盾和主线无关。
- 主角被动等待系统安排。

## 训练任务

用 300 字写一个金手指首秀：一句发现能力，一句眼前麻烦，三句主动使用和结果。写完检查读者是否能复述“这个能力有什么用”。
