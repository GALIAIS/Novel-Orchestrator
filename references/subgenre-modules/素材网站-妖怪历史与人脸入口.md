# 素材网站-妖怪历史与人脸入口

依据案例：`batch-238` 案例 1186。

## 读者承诺

妖怪、历史、人物外貌这类素材不必从零硬编；用垂类网站降低启动成本，再把资料转成可写细节。

## 核心循环

```text
素材卡壳 -> 找垂类入口 -> 提取具体细节 -> 转化设定/描写 -> 服务剧情
```

## 开篇必备

- 明确素材需求。
- 选择对应入口。
- 提取可写细节。
- 筛掉不匹配题材调性的资料。

## 期待来源

读者读到的不只是资料名词，而是带有传说感、历史质感或外貌差异的具体场景。

## 断档风险

- 只收藏网站不转化。
- 原样堆百科。
- 随机人脸替代人物性格。
- 历史素材脱离主角处境。

## 训练任务

选一个素材站条目，提取 3 个细节，分别写成设定句、人物反应句、剧情触发句。
