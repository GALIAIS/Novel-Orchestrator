# 末世变种-平台风险与包装

依据案例：batch-262 案例 1309

## 读者承诺

末世题材遇到平台风险时，不是简单放弃，而是核验证据、保留核心爽点、改成平台更能接受的变种包装。

## 核心循环

```text
风险传闻 -> 查平台证据 -> 保留生存爽点 -> 改包装 -> 决定投放
```

## 开篇必备

- 风险来源和日期。
- 编辑/征文/标签证据。
- 需要规避的现实映射。
- 可保留的资源争夺和生存压力。
- 变种包装方向。

## 期待来源

读者仍然想看危机、生存、资源和秩序重建；平台侧需要减少敏感现实感。

## 断档风险

把旧政策当永久结论会误导；完全无视题材风险会浪费存稿；变种后丢掉末世爽点也会失去读者。

## 训练任务

把一个现代末世设定改成废土、异灾、未来孤城三版，每版写清保留的核心资源爽点。
