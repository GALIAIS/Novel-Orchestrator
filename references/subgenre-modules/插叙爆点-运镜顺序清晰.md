# 插叙爆点-运镜顺序清晰

依据案例：`batch-034` 案例 168。

## 读者承诺

爆点前置要让读者先看懂当前危机，再逐步理解背景。插叙不能为了说明设定而打断读者最想看的生死应对。

## 核心循环

```text
当前危机 -> 主角反应 -> 少量必要信息 -> 金手指解决 -> 危机后补背景 -> 新目标
```

## 开篇必备

- 第一镜头清楚。
- 危机中不长篇解释。
- 回忆只补必要信息。
- 主角用决断改变局面。

## 期待来源

- 读者想知道主角怎么活。
- 金手指第一次使用有画面。
- 背景疑问在危机后顺势回答。
- 爽点后立刻接下一章目标。

## 断档风险

- 当前-回溯-当前切换过早。
- 读者还没共情就被迫听设定。
- 伏笔作者懂，读者看不出来。
- 主角目标不明，危机解决后不知道看什么。

## 训练任务

把一个插叙开篇改成镜头表：当前危机 5 镜、主角反应 3 镜、金手指 3 镜、危机后背景 2 镜。
