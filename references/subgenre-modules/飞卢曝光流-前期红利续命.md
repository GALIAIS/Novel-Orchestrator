# 飞卢曝光流-前期红利续命

依据案例：`batch-061` 案例 301。

## 读者承诺

这个模块用于飞卢曝光、揭秘、盘点、身份公开类写法。它承诺的是用连续曝光吃前期期待红利，并提前准备上架后的非曝光事件。

## 核心循环

```text
强噱头 -> 连续曝光 -> 震惊反应 -> 更大秘密 -> 上架验证 -> 新事件续命
```

## 开篇必备

- 第一章抛出足够强的曝光对象。
- 每次曝光都要制造更大的追问。
- 旁观者反应要放大震惊。
- 上架前要有足够信息库存。
- 上架后必须转入新事件，不能只重复揭秘。

## 期待来源

- 读者想知道下一个秘密是什么。
- 曝光对象身份越高，反应越强。
- 热点外壳降低理解成本。
- 每层秘密抬高主角或家族价值。
- 新事件承接曝光后的资源和敌意。

## 断档风险

- 曝光完就没东西写。
- 只有震惊，没有事件推进。
- 热点外壳过期。
- 同质化太快。
- 平台规则变化导致红利消失。

## 训练任务

列 12 次曝光，每次写“曝光内容、震惊对象、更大追问、带来的新事件”。写不出新事件的曝光删掉。
