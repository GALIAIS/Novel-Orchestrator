# 擦边内容-长期连载风险控制

依据案例：batch-274 案例 1366

## 读者承诺

把亲密张力转成关系推进和情绪拉扯，而不是研究绕审核。长期连载要优先保证作品安全、账号安全和持续收益。

## 核心循环

```text
关系张力 -> 阻碍延迟 -> 情绪升温 -> 选择代价 -> 关系变化
```

## 开篇必备

- 明确亲密场景服务哪条关系线。
- 只写可安全承载的情绪、犹豫和选择。
- 高风险动作和敏感表达直接删除。

## 期待来源

读者期待的是关系靠近、误会升级、身份阻隔和最终选择。短刺激不如长期关系线稳定。

## 断档风险

- 机器审核过了，但人工或举报不过。
- 政策变化导致整本书受损。
- 把擦边当卖点，本末倒置。

## 训练任务

把一个亲密段落改成关系张力段：删除敏感动作，只保留欲望、克制、阻碍和关系后果。
