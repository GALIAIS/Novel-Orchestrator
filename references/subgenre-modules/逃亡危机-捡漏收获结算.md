# 逃亡危机-捡漏收获结算

依据案例：`batch-071` 案例 353。

## 读者承诺

这个模块用于逃亡、追杀、灾变和秘境危机场景。它承诺的是把被动逃亡转成有目标、有底牌、有最终收获的单元，而不是让主角长时间挨打。

## 核心循环

```text
先赢强敌 -> 更高层危机 -> 逃脱目标 -> 底牌拉扯 -> 摆脱追击 -> 秘宝结算
```

## 开篇必备

- 逃亡前主角先有高质量胜利。
- 敌人必须明确高出当前层级。
- 逃亡目标不是躲，而是从强者手中逃脱。
- 过程里逐步夺回主动权。
- 最后用超额收获解释篇幅价值。

## 期待来源

- 先前胜利抵消窝囊感。
- 高层敌人制造压迫。
- 底牌尽出带来拉扯。
- 捡漏资源提供补偿。
- 秘宝对后续主线有用。

## 断档风险

- 新人写长逃亡。
- 主角一直被动。
- 中途捡漏打散紧张感。
- 逃亡无结算。
- 收获对后续剧情没用。

## 训练任务

写一个三节点逃亡：被追、反制、逃脱。结尾必须给一个后续主线必用的收获，且逃亡前先安排主角赢过一次。
