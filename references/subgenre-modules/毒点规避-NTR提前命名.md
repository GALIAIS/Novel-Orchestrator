# 毒点规避-NTR提前命名

依据案例：`batch-316` 案例 1576

## 读者承诺

轻度 NTR 风险可以通过系统、旁白或配角提前点名，转成吐槽和玩梗，降低读者误判成作者无意识踩雷的概率。

## 核心循环

```text
伪装/换脸互动 -> 读者产生NTR联想 -> 系统提前吐槽 -> 作者显得有数 -> 毒感下降
```

## 开篇必备

- 明确读者可能误解的点。
- 在读者开骂前点名风险。
- 吐槽必须轻，不要扩大真实伤害。
- 后续行为要证明边界安全。

## 期待来源

读者期待作者知道哪些内容危险，并能把敏感点控制在可玩梗范围内。

## 断档风险

- 真实背叛不能靠吐槽洗白。
- NTR 风险过重，提前命名也没用。
- 吐槽太晚，读者已经反感。
- 上下文不足，读者看不懂规避点。

## 训练任务

给一个换脸、误认或暧昧桥段写一句提前吐槽，说明它如何降低毒点。
