# 滚动章纲-显式衔接与存稿缓冲

依据案例：batch-390 案例 1950

## 读者承诺

章纲把经验作者脑内的目标、阻力、选择、结果和章节交接外显出来，再由正文反馈滚动修订，降低近期卡文而不锁死全书。

## 核心循环

```text
主线卷目标定向 -> 近期章纲外显因果 -> 正文执行 -> 发现偏差 -> 修订后续并保持缓冲
```

## 开篇必备

- 稳定的整本主矛盾和当前卷结果。
- 每章目标、阻力、选择与状态变化。
- 上一结果到下一目标的明确交接。
- 可随正文修改的近期章纲缓冲。

## 期待来源

- 远期方向被拆成可执行的近期动作。
- 衔接漏洞在动笔前暴露。
- 正文不必临时发明所有因果。
- 实写反馈持续校准纸面计划。

## 断档风险

- 把大纲当作不可修改合同。
- 只有事件标题，没有人物选择。
- 无限细分却迟迟不写正文。
- 用章纲承诺不现实的固定产速。

## 训练任务

做五章滚动章纲后写完前两章，根据实际人物行为修订后三章；未写完近期正文前，不扩张到远期逐章规划。
