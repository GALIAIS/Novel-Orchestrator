# 起书框架-题材情绪噱头金手指

依据案例：`batch-067` 案例 332。

## 读者承诺

这个模块用于新书立项和第一章框架。它承诺的是让题材、情绪卖点、噱头和金手指从书名、简介到第一章同向工作，快速筛出想看这类爽感的读者。

## 核心循环

```text
题材定向 -> 情绪卖点定味 -> 噱头制造好奇 -> 金手指提供路径 -> 书名简介承诺 -> 第一章危机兑现
```

## 开篇必备

- 题材明确，最好有热度。
- 主副情绪卖点清楚。
- 噱头能引发好奇或大众情绪。
- 金手指或能力能解决危机，并最好可升级。
- 第一章写出背景、危机、赌注、目标、路径。

## 期待来源

- 书名直接给出想看点。
- 简介扩大书名承诺。
- 危机让读者知道主角会失去什么。
- 唯一路径让读者期待主角如何破局。
- 金手指升级带来长期期待。

## 断档风险

- 题材和卖点不一致。
- 简介堆杂信息，不扩大承诺。
- 噱头只是概念，没有情绪。
- 金手指不能进入第一危机。
- 第一章只有背景，没有赌注和路径。

## 训练任务

给同一个题材写 3 套“题材、主情绪、副情绪、噱头、金手指”。每套再写一句书名承诺和第一章危机，检查是否同向。
