# 小白系统-先给好处

依据案例：`batch-027` 案例 133。

## 读者承诺

系统文开局先让系统帮主角打开局面，压力来自外部，系统不背刺主角。

## 核心循环

```text
主角低潮 -> 系统给好处 -> 外部压力 -> 主角使用系统 -> 局面扭转 -> 代价被补偿
```

## 开篇必备

- 主角低潮。
- 系统第一份积极作用。
- 外部第三方压力。
- 主角自己的判断。

## 期待来源

- 读者确认系统是主角助手。
- 好处大于代价。
- 主角地位高于系统。
- 主角能主动使用系统解决事。

## 断档风险

- 系统上来嘲讽、制约、浪费钱。
- 系统和主角产生短期解决不了的冲突。
- 主角像系统工具人。
- 反系统没有更大收益。

## 训练任务

写一个系统第一章：系统必须同章给主角带来可见好处，且主角要靠自己的判断用它解决外部麻烦。
