# 标签融合-正文兑现优先

依据案例：`batch-024` 案例 120。

## 读者承诺

多标签创意必须在前三章用事件证明标签爽点，不能只靠简介和作者分析。

## 核心循环

```text
标签承诺 -> 具体危机 -> 主角信息差 -> 改变结果 -> 游戏/现实反馈 -> 标签成立
```

## 开篇必备

- 一个主标签。
- 标签对应的第一爽点。
- 读者能理解的既定悲剧。
- 主角第一次改变结果。

## 期待来源

- 模拟器有前后结果对照。
- 游戏有量化反馈和高手交流。
- 全民穿越让现实世界看懂主角战绩。
- 信息差让主角领先。

## 断档风险

- 标签分析很多，正文前几段无有效内容。
- 冒险突兀，人物互动流水账。
- 日轻式铺陈投 CN 网文，冲突太慢。
- 游戏线和现实线一起开，基调混乱。

## 训练任务

把一个多标签设定压成三章：每章只能证明一个标签爽点。写不进事件的标签先删掉。
