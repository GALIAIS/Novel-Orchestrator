# 网文基础训练-小剧情爽点优先级

依据案例：`batch-199` 案例 993。

## 读者承诺

学习网文先练小剧情爽点，把单个桥段写爽，再逐步叠加主线、立意和复杂节奏。

## 核心循环

```text
理论很多 -> 选最小爽点 -> 写小剧情 -> 得到反馈 -> 再加结构
```

## 开篇必备

- 每个小剧情只有一个爽点目标。
- 读者能立刻知道为什么爽。
- 主角动作和反馈明确。
- 暂不贪多求全。

## 期待来源

读者先在单个桥段里获得即时满足，再愿意跟随更长主线。

## 断档风险

- 理论堆太多，不知道练什么。
- 主线宏大但单场无爽点。
- 立意替代可读性。
- 没掌握就追求全套结构。

## 训练任务

连续写 5 个 1000 字以内小剧情，每个只练一个爽点：打脸、获利、认可、反杀、关系推进。
