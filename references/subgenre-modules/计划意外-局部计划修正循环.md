# 计划意外-局部计划修正循环

依据案例：`batch-340` 案例 1699

## 读者承诺

计划-意外是主角执行计划后发现原计划无法达成目标，于是修正计划继续行动的局部推进单元。

## 核心循环

```text
目标 -> 当前计划 -> 执行 -> 意外证明计划失效 -> 获得新信息 -> 修正计划 -> 下一轮行动
```

## 开篇必备

- 主角先有明确目标。
- 计划必须具体可执行。
- 意外要破坏计划路径。
- 目标不一定变，路径必须变。
- 新计划要带来下一步行动。

## 期待来源

读者期待主角持续求解；每次意外都让主角知道更多、付出更多或逼近真相。

## 断档风险

- 没计划就出意外。
- 意外不改变行动。
- 主角只被动挨打。
- 把普通事故误当结构转折。
- 局部计划-意外和幕末惊人意外混用。

## 训练任务

为一个目标写 4 轮计划-意外，每轮都写清原计划、失效原因、新信息和修正后的下一步。
