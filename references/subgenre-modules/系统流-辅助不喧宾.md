# 系统流-辅助不喧宾

依据案例：`batch-025` 案例 124。

## 读者承诺

系统负责提示、奖励和留下期待，主角负责判断、选择和承担风险。

## 核心循环

```text
新规则 -> 系统提示 -> 主角判断 -> 完成任务 -> 奖励应对危险 -> 后续期待
```

## 开篇必备

- 世界规则为什么难懂。
- 系统给出的有限提示。
- 主角自己的选择。
- 当前奖励服务当前危险。

## 期待来源

- 系统帮助读者适应副本。
- 主角用正常逻辑打出神级操作。
- 奖励一个当前用，一个后面用。
- 选择、签到、任务等频率可控。

## 断档风险

- 系统替主角思考和决定。
- 系统奖励剧透太多，框死长线。
- 常识异世界也硬塞系统，限制主角骚操作。
- 系统出现太频繁，喧宾夺主。

## 训练任务

设计一个选择系统桥段：系统给三个选项，但主角必须分析局势后选出最有用的一项，并用奖励解决眼前危机。
