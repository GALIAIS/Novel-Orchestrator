# 方法论工具箱-素材总纲节奏三部门

依据案例：`batch-253` 案例 1263。

## 读者承诺

方法论工具箱模块把过多理论按写作工作流收纳，避免知道得越多越不敢写。

## 核心循环

```text
理论混乱 -> 分素材/总纲/节奏风格 -> 放入工具箱 -> 写作试用 -> 修正成自己的方法
```

## 开篇必备

- 先判断当前卡在什么部门。
- 每个部门只保留常用工具。
- 工具必须有使用场景。
- 写完一段后复盘工具效果。

## 期待来源

作者不再被方法论拖住，而是能按当前问题调用工具，继续推进故事。

## 断档风险

- 工具分得太细，边界模糊。
- 工具分得太粗，不可执行。
- 收藏理论，不在正文里试验。
- 忘记最终目标是故事。

## 训练任务

为一个项目建立三栏工具箱：素材、总纲设计、节奏与风格。每栏只放 5 个工具，并写清“什么时候用”。
