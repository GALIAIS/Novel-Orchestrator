# 主线目标-下一步任务提示

依据案例：`batch-078` 案例 390。

## 读者承诺

这个模块用于检查故事是否有主线。它承诺让读者始终知道主角下一步要做什么，而不是只看见装逼、升级和随机事件。

## 核心循环

```text
初始任务 -> 意外改写 -> 新任务 -> 行动推进 -> 下一步提示
```

## 开篇必备

- 第一章给出阶段目标。
- 目标要具体到下一步行动。
- 意外发生后立刻重定向。
- 升级和地位提升要服务目标。
- 章末保留下一步期待。

## 期待来源

- 读者知道主角去哪。
- 主角知道为什么做。
- 意外改变任务而不是打散故事。
- 长期目标统摄短期事件。
- 每章末有可预期行动。

## 断档风险

- 只有故事梗概，没有行动线。
- 主角太被动。
- 事件随机发生。
- 升级太笼统。
- 装逼后不知道下一步。

## 训练任务

给前三章写任务提示：每章写“主角当前任务、意外、新任务、读者下一步期待”。如果任一章写不出任务，重做主线。
