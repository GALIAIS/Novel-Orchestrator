# 上架运营-全订读者反馈闭环

依据案例：`batch-182` 案例 909。

## 读者承诺

上架后的全订读者反馈闭环，承诺用核心付费读者体感和后台订阅数据共同校准剧情。

## 核心循环

```text
后台数据 -> 全订群提问 -> 收集体感 -> 定位漏洞/毒点 -> 调整后续剧情
```

## 开篇必备

- 建立全订读者沟通渠道。
- 主动问最近剧情感受。
- 把反馈与订阅数据对照。
- 区分高频问题和个体偏好。

## 期待来源

高质量读者可能帮助作者发现设定漏洞、剧情毒点和可补救路径。

## 断档风险

- 只看后台数据，不知道原因。
- 只看群聊情绪，被单个声音带偏。
- 不敢问读者具体感受。
- 收到反馈后不验证效果。

## 训练任务

设计一次全订群反馈收集：分别问爽点、节奏、人物、设定漏洞和毒点风险，并写出对应的数据验证指标。
