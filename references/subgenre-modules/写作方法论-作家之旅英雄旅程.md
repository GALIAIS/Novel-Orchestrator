# 写作方法论-作家之旅英雄旅程

依据案例：`batch-175` 案例 871。

## 读者承诺

经典叙事方法论提供结构语言，帮助作者把主角出发、越界、试炼、奖励、代价和回归变化拆成可训练节点。

## 核心循环

```text
阅读方法论 -> 提炼结构模型 -> 对照作品标注阶段 -> 找出人物变化 -> 回到章节设计
```

## 开篇必备

- 明确主角的日常世界和缺口。
- 给出让主角不得不行动的召唤或压力。
- 设置越界门槛，让读者感觉故事进入新阶段。
- 让试炼服务人物成长，不只是堆事件。

## 期待来源

作者期待获得一套成熟结构工具，读者则期待主角在冒险中发生可感知变化。英雄旅程的价值在于把“事件推进”和“人物成长”绑在一起。

## 断档风险

- 只收藏资料，不做作品标注。
- 机械套用所有阶段，导致网文节奏变慢。
- 只看结构，不看题材爽点和平台读者承诺。
- 把理论当答案，忽略具体章节执行。

## 训练任务

用英雄旅程语言拆一部网文开篇卷，只标注真正生效的 5 个节点，并写清每个节点带来的读者期待变化。
