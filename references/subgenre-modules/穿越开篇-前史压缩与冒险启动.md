# 穿越开篇-前史压缩与冒险启动

依据案例：`batch-201` 案例 1003。

## 读者承诺

穿越开篇的前史、背景和伏笔必须服务当前冒险启动；能一句话带过的信息，不要占掉第一章的大段篇幅。

## 核心循环

```text
必要前史压缩 -> 当前处境 -> 金手指受限 -> 冒险启动 -> 伏笔跟随事件露出
```

## 开篇必备

- 尽早出现当前目标或危机。
- 穿越前信息只保留必要句。
- 金手指暂失要转成悬念。
- 主线伏笔嵌入当前行动。

## 期待来源

读者期待主角进入新处境后立刻行动，并通过能力受限、恢复线索和冒险事件形成追读钩子。

## 断档风险

- 原世界说明过长。
- “金手指鸡肋”削弱读者期待。
- 世界背景先行，冒险迟迟不开始。
- 胎穿多年却没有当前行动目标。

## 训练任务

把 1300 字前史压缩成一句话、150 字、300 字三版，每版都保留金手指暂失、主线伏笔和马上冒险的钩子。
