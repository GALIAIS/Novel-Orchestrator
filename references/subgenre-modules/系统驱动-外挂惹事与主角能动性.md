# 系统驱动-外挂惹事与主角能动性

依据案例：batch-346 案例 1729

## 读者承诺

系统可以推动剧情、制造麻烦、调度支线，但最终爽点必须落在主角的选择、判断和承担上。

## 核心循环

```text
系统任务/异常 -> 主角被推入麻烦 -> 主角判断取舍 -> 付出代价破局 -> 系统反馈升级或失控
```

## 开篇必备

- 系统的能力、限制、奖励和惩罚要明确。
- 主角必须有拒绝、曲解、利用或反制系统的空间。
- 系统制造的任务要撞上主角自身欲望。
- 破局不能全靠系统自动完成。
- 感情线和泪点不能像奖励道具一样派发。

## 期待来源

读者期待系统骚操作带来意外局面，再看主角怎样把被动麻烦变成主动收益。

## 断档风险

- 主角只执行任务，像木偶。
- 系统全知全能，所有冲突失去悬念。
- 奖励替代人物成长。
- CP、支线、主线都由系统硬塞。
- 系统越写越像作者懒得编剧情的借口。

## 训练任务

为同一章设计两版系统任务：一版系统强制主角做事，一版系统只给诱惑和误导。比较哪一版更能体现主角性格。
