# 码字工具-跨端丢稿备份

依据案例：`batch-092` 案例 459。

## 读者承诺

码字工具第一标准是稿件安全。跨端同步、云端检测、手机端编辑都可能出问题，必须用本地备份和多版本留存防止单点丢稿。

## 核心循环

```text
写作 -> 手动保存 -> 同步刷新 -> 本地导出 -> 云端备份 -> 版本留存 -> 可恢复
```

## 开篇必备

- 不把唯一稿件放在单一工具里。
- 手机端写作先用本地记事本兜底。
- 每天导出本地副本。
- 文件名带日期和章节。
- 定期检查能否恢复。

## 期待来源

- 丢稿后能恢复。
- 跨端同步不再是单点风险。
- 云端锁稿不会毁掉全部内容。
- 写作心态更稳。

## 断档风险

- 手机端和 PC 端不同步。
- 回收站也没有稿件。
- WPS 等云端内容检测锁稿。
- 没有纯文本备份。

## 训练任务

连续 7 天执行三份备份：码字软件、纯文本本地副本、云端或移动硬盘副本。每天随机打开昨天版本确认可恢复。
