# 素材库-琐碎命名与工具替代

依据案例：`batch-229` 案例 1141。

## 读者承诺

低价值但高频出现的命名和设定杂项，应由素材库提供候选，让作者把精力留给情节与人物选择。

## 核心循环

```text
素材需求 -> 工具生成候选 -> 作者筛选风格 -> 进入项目词库 -> 正文快速调用
```

## 开篇必备

- 明确哪些素材会反复消耗时间。
- 有固定工具或素材来源。
- 有风格筛选标准。
- 有项目级词库承接。

## 期待来源

作者不在城市名、龙套名、功法名上反复卡顿，章节推进稳定，文本风格也更统一。

## 断档风险

- 依赖单一素材网站，工具失效就停摆。
- 直接使用工具产物，导致风格混杂。
- 把素材整理当成逃避写正文。
- 没有禁用名单，俗套名反复出现。

## 训练任务

建立项目素材三栏表：人物名、地名、功法/招式名。每栏至少 20 个候选，标注风格、适用场景和禁用理由，正文中只调用筛过的条目。
