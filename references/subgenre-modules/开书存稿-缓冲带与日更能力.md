# 开书存稿-缓冲带与日更能力

依据案例：`batch-227` 案例 1135。

## 读者承诺

存稿是救急缓冲，不是生产能力；开书前要确认日常码字速度能跟上更新。

## 核心循环

```text
内投通过 -> 估算更新频率 -> 测真实日码 -> 准备缓冲存稿 -> 持续补充
```

## 开篇必备

- 记录真实单章耗时。
- 设最低缓冲天数。
- 存稿不要多到前文大改全损。
- 写完一章再局部改词。

## 期待来源

作者的安全感来自稳定产出，而不是囤一个很快会耗尽的数字。

## 断档风险

- 只问存稿数字，不练速度。
- 前文要大改导致大量存稿报废。
- 每个词都纠结，章节无法完成。
- 存稿被当成长期更新能力。

## 训练任务

连续 7 天记录有效码字时间、字数、卡顿原因；每天先写完一章，再允许改 3 处词句。计算真实日更能力和最低安全存稿量。
