# 加更承诺-书友群规则与评论区风险

依据案例：`batch-233` 案例 1165。

## 读者承诺

公开加更规则就是对读者的契约；完不成要提前沟通和补偿，避免群友玩梗外溢到评论区。

## 核心循环

```text
设加更规则 -> 监控进度 -> 提前沟通 -> 补偿兑现 -> 评论区控场
```

## 开篇必备

- 规则不要随口许诺。
- 触发条件和期限明确。
- 完不成提前低姿态说明。
- 有管理员和评论区应急方案。

## 期待来源

读者知道作者守约，哪怕延期也能得到解释和补偿，社群情绪不外溢。

## 断档风险

- 加更规则超出产能。
- 失败后不沟通。
- 群友玩梗刷屏影响新读者。
- 手机端无法及时删帖。
- 惊动编辑造成误解。

## 训练任务

写加更承诺表：触发条件、兑现期限、失败补偿、提前公告、管理员动作。没把握做到的规则不要公开。
