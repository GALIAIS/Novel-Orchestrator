# 卷前提-冲突递进证明立意

依据案例：`batch-048` 案例 239。

## 读者承诺

用一个具体“前提”统摄一卷冲突，让理念通过人物选择、矛盾升级和高潮结果被证明，而不是靠旁白或角色长篇说教。

## 核心循环

```text
确定前提 -> 设计一级冲突 -> 升级二级冲突 -> 推到最终冲突 -> 主角介入 -> 高潮证明前提
```

## 开篇必备

- 前提必须具体，不能是空泛正确的大话。
- 第一阶段冲突就要让读者看见前提的苗头。
- 每次冲突升级都要让前提更有说服力。
- 主角行动必须参与情绪上行。
- 高潮画面要让读者自己得出结论。

## 期待来源

- 读者看见错误欲望逐步造成代价。
- 冲突越升级，前提越被验证。
- 主角介入能把悲剧基调改造成网文基调。
- 高潮不是讲道理，而是让事件应验。

## 断档风险

- 前提太大，作者驾驭不了。
- 前提和主角行动无关，会变成支线自嗨。
- 角色直接讲理念，读者会有被教育感。
- 只追求立意不顾爽点，会失去网文节奏。

## 训练任务

为同一卷纲写 3 个前提；每个前提分别写一级冲突、二级冲突、最终冲突、主角介入点和高潮证明画面。
