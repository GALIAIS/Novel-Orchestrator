# 功能性楔子-危机移交与群像单锚切入

依据案例：batch-388 案例 1937

## 读者承诺

楔子只负责制造会移交给正文的危机和疑问；群像先用一个人物或共同事件集中注意，再让角色以不同选择进入。

## 核心循环

```text
楔子发生不可逆事件 -> 主线锚点接住后果 -> 配角用不同选择亮相 -> 共同危机绑定 -> 稳定后扩展视角
```

## 开篇必备

- 楔子中正在发生的事件，而非历史说明。
- 能进入第一章的具体因果资产。
- 一个清楚的认知锚或中心问题。
- 配角对同一压力的差异化反应。

## 期待来源

- 异常事件留下明确疑问。
- 第一章迅速接住异常后果。
- 多人立场在共同危机中碰撞。
- 每次扩展视角都提供新信息。

## 断档风险

- 楔子只负责抖设定。
- 楔子结束后正文重新开书。
- 群像开场平均分镜导致无人可抓。
- 角色靠名单和介绍登场。

## 训练任务

写 400 字危机楔子和 800 字单锚第一章，让两名配角以相反选择进入；删除楔子检查它是否真正移交了因果。
