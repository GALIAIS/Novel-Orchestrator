# 东京校园日常-角色撑起系统弱化

依据案例：`batch-148` 案例 736。

## 读者承诺

系统只是入口，真正撑起作品的是东京校园日常、特色女角色、笑点与泪点。读者为角色关系和陪伴感追看，而不是为强外挂追看。

## 核心循环

```text
穿越入口 -> 弱系统 -> 特色角色登场 -> 日常冲突 -> 笑点泪点 -> 关系推进
```

## 开篇必备

- 一个轻系统或入口设定。
- 一个可持续日常场景。
- 几个有市场验证感的人设。
- 角色间的稳定冲突和反馈。
- 阶段关系目标，防止后期断粮。

## 期待来源

读者期待“和这些角色继续生活下去”。系统存在感越弱，角色塑造和日常事件越要强。

## 断档风险

- 系统抢戏，破坏日常味。
- 角色只有原型，没有本书改造。
- 全靠灵感推进，后期不知道写什么。
- 日常没有阶段目标，变流水账。

## 训练任务

设计 3 个校园日常女角色，每个写明市场原型、本书改造点、和主角的日常冲突、一个笑点、一个泪点，以及 20 章阶段关系目标。
