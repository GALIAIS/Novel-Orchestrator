# 平台榜单拆解-存量偏差与新书样本分层

依据案例：`batch-410` 案例 2046

## 读者承诺

先识别榜单的统计窗口、排序指标和作品资格，再将长期完本、稳定连载、近期新发和未增长作品分层比较，避免用存量畅销榜直接推断新书生态。

## 核心循环

```text
记录榜单口径和日期 -> 判断生命周期偏向 -> 解释存量优势 -> 补取完本、连载、新发和未增长四层样本 -> 用统一指标拆解 -> 核验平台事实来源 -> 在证据范围内形成结论
```

## 开篇必备

- 明确的研究问题。
- 榜单名称、日期、窗口和排序指标。
- 作品上榜资格与状态。
- 四个生命周期样本层。
- 开篇与长线两组统一指标。
- 平台事实的一手来源和适用日期。

## 期待来源

老书占榜先制造市场疑问，口径和生命周期拆解消除误判；新书与未增长样本补回冷启动证据，使研究从平台印象转为具体机制比较。

## 断档风险

- 没有榜单口径便推断市场兴衰。
- 把累计畅销等同近期新书表现。
- 只看成功样本，不保留未增长对照。
- 用旧评论判断平台归属和频道定位。
- 只读开篇却评价全书结构。
- 混淆找新书与学习长线写法。

## 训练任务

人工抽取完本畅销、稳定连载、近期新发和未增长作品各五本，用同一表记录生命周期、开篇承诺和首弧回报；分别写出榜单能说明、不能说明和其他层补充的结论。
