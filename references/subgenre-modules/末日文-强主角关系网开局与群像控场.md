# 末日文-强主角关系网开局与群像控场

依据案例：`batch-194` 案例 968。

## 读者承诺

主角自带团队或关系网时，先用事件立住主角，再让配角按功能分批登场，避免开篇群像信息把读者压懵。

## 核心循环

```text
主角开脸 -> 关系网露面 -> 配角按事件分批展开 -> 金手指介入 -> 群像服务危机
```

## 开篇必备

- 第一场事件展示主角能力或价值观。
- 第二场事件展示主角手腕或黑暗面。
- 团队成员只做必要露面。
- 金手指或关键道具尽快解决问题。

## 期待来源

读者先确认主角值得追，再对他的关系网、团队能力和末日危机产生兴趣。

## 断档风险

- 一次性介绍大量人物。
- 用旁白解释关系，不用事件表现。
- 群像压过主角开脸。
- 前文道具和金手指拖太久不用。

## 训练任务

写一个三场开局：主角光明面、主角黑暗面、三个团队成员露面。每个配角只能用一个动作展示功能。
