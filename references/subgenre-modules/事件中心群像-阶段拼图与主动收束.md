# 事件中心群像-阶段拼图与主动收束

依据案例：`batch-415` 案例 2074

## 读者承诺

多个看似随机的人物路线围绕一个持续变化的中央事件逐步成图。每名角色有不依赖案件、灾难或危机也成立的个人目标，偶遇由时间、地点、职业、关系和物件提前安排；巧合只把线路送到同一入口，主角必须主动追查、组合和选择。每个阶段达到事实、证据、资源、关系与风险门槛后结算一个“面”，再重分配角色功能。

## 核心循环

```text
建立中央事件状态表 -> 设置多名角色的独立目标与路线 -> 可回溯的时空因果让线路相撞 -> 巧合只产生入口或新问题 -> 主角主动追查并组合碎片 -> 每场改变中央状态和人物关系 -> 若干状态达到阶段门槛 -> 完成阶段之面并局部结算 -> 角色功能按新状态重分配 -> 更高风险阶段继续收束
```

中央状态表保证方向，独立目标避免纯工具人，主动收束保护主角能动性，阶段门槛让横向群像拥有纵向进度。

## 开篇必备

- 中央事件的已知事实、未知问题和时间压力。
- 证据、资源、舆论、关系和风险五类状态。
- 本阶段完成所需的明确门槛。
- 至少四名角色各自独立的目标、资源和底线。
- 每条路线的时间、地点、职业、关系和物件安排。
- 巧合带来的单一入口或问题。
- 主角事后的观察、追查、组合和选择。
- 每场改变的一项中央状态和一条关系。
- 阶段结算后的角色功能重分配。
- 十阶段重复与膨胀压力测试。

## 期待来源

- 分散角色何时发现自己卷入同一事件。
- 两份看似冲突的碎片如何互相验证。
- 主角会否主动识别偶遇背后的模式。
- 私人目标怎样改变证据和联盟。
- 哪些状态达到门槛后会揭开更大结构。
- 原盟友为何在下一阶段变成阻碍。
- 所有散点最终能否组成可回溯图形。

## 断档风险

- 中央事件只有一句主线，没有状态表。
- 角色个人目标只是查案任务的换皮。
- 每次都靠新人偶然送来关键线索。
- 巧合直接给出完整答案，主角只负责接收。
- 阶段没有门槛，横向铺陈无限扩张。
- 角色功能从开篇到结尾永不变化。
- 把“事件是主角”当成剥夺人物主体性的许可。
- 十几个阶段仍重复同一种相撞机制。

## 训练任务

设计一个中央事件和四名独立目标角色，制作事实、证据、资源、舆论、关系与风险状态表。安排三次可回溯相撞，每次只给一个入口，必须由主角主动追查才推进；完成一组阶段门槛后重新分配角色功能，并推演后续十阶段的重复风险。
