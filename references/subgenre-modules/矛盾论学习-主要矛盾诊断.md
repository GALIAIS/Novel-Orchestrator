# 矛盾论学习-主要矛盾诊断

依据案例：`batch-102` 案例 506。

## 读者承诺

把抽象理论变成写作诊断工具：先找当前失败结果里的主要矛盾，再决定下一轮只改哪里。理论不能替代文本、签约反馈和读者反馈。

## 核心循环

```text
理论学习 -> 写作实践 -> 失败反馈 -> 找主要矛盾 -> 定向修改 -> 再实践
```

## 开篇必备

- 明确失败发生在哪个阶段。
- 签约失败优先看前三五万字。
- 对照文笔、剧情、人设、金手指、题材逐项排查。
- 扫榜样本必须同类型、同构建方式。

## 期待来源

- 理论能解释自身问题。
- 诊断能给出下一步改法。
- 同类型样本能提供可迁移结构。
- 实践和总结能形成闭环。

## 断档风险

- 只总结畅销书共同点，不看自己缺陷。
- 把不同构建类型硬放一起比较。
- 理论理解浅，套不上具体文本。
- 写了很多字但反馈没有进入下一轮修改。

## 训练任务

对自己的前三万字做一张主要矛盾表，只选一个最可能导致签约失败的问题。下一轮修改只围绕这个问题，不同时改十个方向。
