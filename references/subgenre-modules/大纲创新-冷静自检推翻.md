# 大纲创新-冷静自检推翻

依据案例：`batch-061` 案例 303。

## 读者承诺

这个模块用于大纲完成后的最低创新检查。它承诺的是在开写前用市场样本和冷静期筛掉“毫无新增变化”的扑街套路。

## 核心循环

```text
写完大纲 -> 冷静搁置 -> 自问是否想看 -> 对照同类套路 -> 找出新增变化 -> 保留或推翻
```

## 开篇必备

- 一句话说清本书和同类有什么不同。
- 第一章必须看得见这个不同。
- 冷静一天后再审大纲。
- 用近期市场样本对照。
- 每次推翻都记录原因。

## 期待来源

- 读者第一眼看到新鲜点。
- 创新不大但不可缺席。
- 同类套路被换出新角度。
- 作者自己仍然想看。
- 市场前沿提供判断标尺。

## 断档风险

- 大纲只是旧套路换名词。
- 作者自嗨期立刻开写。
- 反复推翻但不记录原因。
- 创新只在设定表里，第一章不出现。
- 只追求怪，不服务读者承诺。

## 训练任务

把大纲卖点写成一句话，再删掉其中的“新东西”。如果删掉后故事仍成立，说明创新没有进入结构，重做第一单元。
