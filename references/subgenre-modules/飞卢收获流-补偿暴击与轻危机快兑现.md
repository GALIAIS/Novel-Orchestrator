# 飞卢收获流-补偿暴击与轻危机快兑现

依据案例：batch-361 案例 1803

## 读者承诺

飞卢收获流靠机制清楚、反馈快速、收益密集留人。危机可以不重，但必须让读者马上知道有乐子和下一次更大收获。

## 核心循环

```text
赠送机缘 -> 补偿暴击 -> 超额收获 -> 小冲突验证 -> 下一轮收益期待
```

## 开篇必备

- 第一章说清机制。
- 第一次收益快速出现。
- 小冲突用于验证收益。
- 下一轮奖励预期明确。

## 期待来源

- 暴击倍率。
- 白送变暴赚。
- 轻危机被快速解决。
- 收获越来越大。

## 断档风险

- 危机轻但机制也不强。
- 说明太久，收益太晚。
- 只写收获不验证用途。
- 平台快节奏没跟上。

## 训练任务

拆一章收获流开头，记录机制说明、第一次收益、冲突验证、下一次期待四栏，检查间隔是否超过 500 字。
