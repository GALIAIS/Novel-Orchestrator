# 核心梗-故事卡

依据精拆：`case-library/精拆/batch-001.md` 案例 002。

## 适用范围

用于拆书、素材整理、桥段仿写、爽点复用、题材迁移。

## 读者承诺

同一个情绪链可以换不同外壳重复成立。作者不用抄具体情节，而是复用能稳定制造情绪的故事结构。

## 核心循环

```text
原型故事 -> 提取情绪链 -> 抽象最小结构 -> 换素材重组 -> 检查情绪是否复现
```

## 提取步骤

1. 选一个自己真实觉得好看的原型故事。
2. 展开为环境、人物、起因、经过、结果。
3. 标出真正触发情绪的特征。
4. 删除不必要外壳。
5. 用最少特征复现原情绪链。

## 示例结构：英雄救美

```text
对象遇到困难/危险 -> 主角出手解决 -> 对象态度变化/好感提升
```

重点不是“必须美女遇险”，而是“主角解决对方关键问题后，关系状态发生正向变化”。

## 断档风险

- 把核心梗误解成一句梗、一个设定或题材标签。
- 只收集具体桥段，不抽象情绪链。
- 抽象过度，只剩空话，不能指导写作。
- 只从整本书尺度看核心梗，忽视一级单元核心梗。

## 操作卡

```text
原型故事：
读者情绪：
环境特征：
人物功能：
起因特征：
经过方向：
结果特征：
最小核心梗：
换题材版本：
```

## 训练任务

从三个喜欢的桥段里各抽一个核心梗，再用自己的题材各写 300 字梗概，检查读者情绪是否仍然成立。
