# 新人开篇-主角前置与铺垫压缩

依据案例：`batch-213` 案例 1062。

## 读者承诺

新人开篇先让读者确认主角是谁，再逐步给设定和背景。

## 核心循环

```text
主角出现 -> 动作目标 -> 当前冲突 -> 必要设定 -> 继续推进
```

## 开篇必备

- 主角在前三句内出现。
- 立刻有动作、目标或异常。
- 前几百字压缩无效铺垫。
- 读者认识主角前不塞重设定。

## 期待来源

读者先抓住人物和当前问题，再愿意记住后续信息。

## 断档风险

- 先铺世界观再出主角。
- 设定信息读者记不住。
- 自查元素齐全但节奏仍拖。
- 投稿样本太短导致反馈不足。

## 训练任务

标注第一章前 500 字的信息类型，删除或后移无效铺垫，重写 300 字主角直入版。
