# 平台规则-六点更新死线

依据案例：`batch-130` 案例 649。

## 读者承诺

平台更新计数规则会塑造作者节奏。刺猬猫次日凌晨六点截止，能自然形成“六点死线”的赶稿压力和群聊氛围。

## 核心循环

```text
更新计数规则 -> 六点截止 -> 剩余字数倒计时 -> 群聊提醒 -> 发布最低可用版本
```

## 开篇必备

- 明确平台。
- 明确截止点。
- 明确剩余字数。
- 明确错过代价。
- 用群聊或闹钟制造死线感。

## 期待来源

- 具体时间点带来压迫。
- 作者执行状态可视化。
- 群聊提醒形成社群氛围。
- 最后一刻发布有小高潮。

## 断档风险

- 只说“快截止”，没有具体时间。
- 不说明错过代价。
- 规则说明太硬，不能进入场景。
- 死线压力和人物目标无关。

## 训练任务

写一个作者群聊片段：凌晨五点四十五分，主角还差八百字，群里不断提醒“六点死线”。规则必须从对话里自然显现。
