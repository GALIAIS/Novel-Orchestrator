# 开头投稿-主角引领与设定少量投放

依据案例：`batch-247` 案例 1232。

## 读者承诺

开头要让主角带读者进入故事，而不是让作者替主角解释世界观和行动理由。

## 核心循环

```text
异常行为钩子 -> 主角主动选择 -> 必要设定跟随事件 -> 人设在行动中显露
```

## 开篇必备

- 前 100 字给钩子。
- 主角有语言、动作、心理。
- 设定只够理解当前剧情。
- 章末有承接。

## 期待来源

读者先对主角产生兴趣，再愿意跟着他理解设定。

## 断档风险

- 背景介绍太多。
- 主角像系统工具人。
- 作者叙述替代主角选择。
- 章节像独立短故事。

## 训练任务

重写开头 500 字，把异常行为前置，并用主角动作和心理交代必要设定。
