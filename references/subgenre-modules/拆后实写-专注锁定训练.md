# 拆后实写-专注锁定训练

依据案例：`batch-083` 案例 415。

## 读者承诺

这个模块用于解决拆书后仍写不出来的问题。它承诺先排查灵感、大纲和注意力，再用小字数锁和专注工具恢复正文产出。

## 核心循环

```text
拆书懂感 -> 实写卡住 -> 排查原因 -> 小字数锁定 -> 复盘继续
```

## 开篇必备

- 不把开书当硬任务。
- 判断大纲是否让自己想写。
- 手机进入专注模式。
- 电脑使用小黑屋或字数锁。
- 第一次只锁 1000-2000 字。

## 期待来源

- 小目标降低启动阻力。
- 不改前文保证推进。
- 专注工具隔离干扰。
- 完成字数恢复信心。
- 复盘知道卡点在哪。

## 断档风险

- 拆书越多越有懂感，但不写。
- 一章反复死磕。
- 写几百字就玩。
- 大纲不满意却硬写。
- 第一次锁太多导致更抗拒。

## 训练任务

连续 7 天只做一件事：锁 1000-2000 字，不改前文，不开其他 App。每天记录卡点属于灵感、大纲还是注意力。
