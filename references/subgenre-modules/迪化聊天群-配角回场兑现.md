# 迪化聊天群-配角回场兑现

依据案例：`batch-125` 案例 624。

## 读者承诺

迪化聊天群的爽点不是群员脑补一句就结束，而是配角回到原剧情兑现改变，再带着崇拜回群反馈。

## 核心循环

```text
群员脑补主角 -> 主角给提示/道具 -> 配角回原世界行动 -> 命运改变 -> 配角爽到 -> 回群崇拜反馈
```

## 开篇必备

- 一次只聚焦少量群员。
- 群员原剧情困境要明确。
- 主角的改变要具体。
- 必须展示改变后的原世界结果。
- 回群反馈要完成装逼闭环。

## 期待来源

- 读者等配角如何误解主角。
- 配角命运改变带来跨世界爽点。
- 五体投地反馈放大主角逼格。
- 新群员接力形成循环。

## 断档风险

- 人物太多，读者认不住。
- 写成大纲，只说改变不展示。
- 迪化无敌却套热血临阵突破。
- 配角刚变强就切下一个，爽点没落地。

## 训练任务

只选一个群员写完整单元：原困境、主角提示、回场改变、爽点兑现、回群崇拜反馈。
