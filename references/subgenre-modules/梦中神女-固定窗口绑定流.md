# 梦中神女-固定窗口绑定流

依据案例：`batch-127` 案例 633。

## 读者承诺

低位主角通过固定梦中窗口接触高位女神，用现代知识补足女神的信息盲区，获得认可、资源和危机牵引。爽点来自身份反差、智识装逼、高位女性反馈和现实危机回流。

## 核心循环

```text
梦中相见 -> 主角提供信息差 -> 女神现实验证 -> 敌对势力察觉 -> 主角被卷入 -> 女神反哺主角
```

## 开篇必备

- 先交代主角魂穿后的身份和处境。
- 明确梦中相见规则。
- 女神的问题必须真有信息盲区。
- 主角装逼要靠有效方案，不靠女神降智。
- 结尾给出现实验证或敌对势力钩子。

## 期待来源

- 低位者影响高位者。
- 现代知识在异界产生反馈。
- 女神认可和资源反哺。
- 小建议引发大局变化。

## 断档风险

- 开局环境不清，读者无法代入。
- 视角切换生硬。
- 高位女神被写降智。
- 背景时代和制度常识冲突。
- 平台需要快节奏却写成慢热铺垫。

## 训练任务

写 1500 字第一次梦中相见：300 字内交代主角身份、阶层和危机；女神提出本世界难题；主角用现代常识给出方案；结尾女神现实验证有效。
