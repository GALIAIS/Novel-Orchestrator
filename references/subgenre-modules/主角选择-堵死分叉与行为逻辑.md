# 主角选择-堵死分叉与行为逻辑

依据案例：batch-354 案例 1770

## 读者承诺

主角选择必须像被局势、人设和信息共同逼出来。多条路线存在时，要让读者看见不选项为什么走不通，最终只剩一条活路，主线才清晰。

## 核心循环

```text
出现选择点 -> 展示多条路线 -> 逐一堵死不选项 -> 人设过滤 -> 唯一行动推进主线
```

## 开篇必备

- 明确选择点。
- 列出至少两条读者能想到的路线。
- 给每条不选路线一个限制。
- 最终选择符合主角人设。

## 期待来源

- 读者预判被作者正面回应。
- 主角行为显得严谨。
- 主线推进轻松不费解。
- 人设通过选择被再次确认。

## 断档风险

- 作者上帝视角强推路线。
- 读者想到 A/B，正文却无解释地走 C。
- 金手指多选奖励不分析不选项。
- 主角行为和人设不符。

## 训练任务

为一个开头危机写三条路线，正文中分别堵死两条，再让主角选择剩下一条。检查读者是否还会问“为什么不选另一个”。
