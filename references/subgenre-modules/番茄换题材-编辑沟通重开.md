# 番茄换题材-编辑沟通重开

依据案例：`batch-085` 案例 424。

## 读者承诺

这个模块用于扑街切书后换题材。它承诺把尴尬情绪转成编辑沟通和公告确认，避免不声不响跑路。

## 核心循环

```text
旧书扑街 -> 复盘题材问题 -> 新书方向 -> 问编辑/查公告 -> 重开
```

## 开篇必备

- 说清旧书题材和问题。
- 说明新书题材。
- 准备新书核心卖点。
- 查作者群公告。
- 直接问是否需要换编辑。

## 期待来源

- 流程清楚。
- 编辑能给方向。
- 切书不再内耗。
- 新题材避免旧错误。
- 沟通比失联更礼貌。

## 断档风险

- 因为尴尬不问编辑。
- 扑街后直接失联。
- 不查平台公告。
- 换题材仍沿用旧写法。
- 新书没有商业卖点说明。

## 训练任务

写 200 字编辑沟通稿：旧书为何停、新书题材、核心卖点、是否需要换编辑。写完直接发。
