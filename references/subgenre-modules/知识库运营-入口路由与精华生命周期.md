# 知识库运营-入口路由与精华生命周期

依据案例：`batch-406` 案例 2030

## 读者承诺

让新人从规则、阶段方法、案例搜索和反馈提交四个入口快速行动；把核心、案例、待验证、已替代与归档分层，防止“精华”无限堆积。

## 核心循环

```text
新人任务入口 -> 阶段核心路由 -> 关键词别名检索案例 -> 标准模板提交 -> 新知识进入待验证 -> 验证后升级 -> 合并标记归档 -> 搜索日志校正入口
```

## 开篇必备

- 首屏只给少量任务入口。
- 规则、核心方法、案例和实验分层。
- 每篇记录问题、阶段、题材、证据和状态。
- 搜索覆盖用户词汇和同义词。
- 重复文档合并并保留来源。
- 过期内容有提示、替代与重定向。

## 期待来源

知识库越大，新人仍能快速知道该先读什么；使用者能分辨当前推荐方法和历史讨论，维护者也能持续瘦身而不丢证据。

## 断档风险

- 要求新人先读完全部精华。
- 分类只使用维护者内部术语。
- 待验证思考直接升格为标准。
- 重复内容永久并列。
- 删除旧文却不保留来源与重定向。
- 不看零结果搜索和真实任务路径。

## 训练任务

抽 30 篇精华标记核心、案例、待验证、已替代或归档，合并三组重复并保留来源；让新人执行四个检索与提交任务，按真实路径修正入口。
