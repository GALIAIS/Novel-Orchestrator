# 番茄拆书-站内榜单优先

依据案例：batch-310 案例 1548

## 读者承诺

写番茄就优先拆番茄原创、免费书、阅读榜和同类型榜单书。目标平台决定样本池。

## 核心循环

```text
确定番茄投稿 -> 查站内榜单 -> 筛同类型样本 -> 拆前三章与长期赚钱能力 -> 再设计新书
```

## 开篇必备

- 不把起点样本当主要依据。
- 优先看番茄阅读榜和原创榜。
- 老长篇可拆长期赚钱能力。
- 新人优先选字数较短、仍在榜的样本。

## 期待来源

期待来自平台口味匹配。拆同平台正在赚钱的书，才能接近目标读者。

## 断档风险

- 写番茄却主要拆起点。
- 只按个人喜欢选样本。
- 嫌老榜书过时，却忽略它仍在赚钱。
- 学老书开头而不考虑新人不可复制。

## 训练任务

从番茄阅读榜找 10 本同类型书，筛 3 本字数较短且仍在榜的，拆前三章卖点和节奏。
