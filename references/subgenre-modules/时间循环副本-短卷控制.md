# 时间循环副本-短卷控制

依据案例：batch-276 案例 1380

## 读者承诺

把时间循环控制成短卷或无限流副本，用信息增量、反转和人物完善制造爽点，避免长篇主机制拖到读者腻。

## 核心循环

```text
死亡/失败 -> 回到过去 -> 信息增量 -> 新行动 -> 真相推进 -> 副本收束
```

## 开篇必备

- 首轮失败要强。
- 每轮循环必须带来新信息或新收益。
- 提前设定结束条件和字数上限。

## 期待来源

读者期待主角带着记忆改写结果，逐步接近真相。循环越短越要密集兑现。

## 断档风险

- 循环过多，读者疲劳。
- 重复情节只换细节，没有信息增量。
- 照搬影视节奏，不适配追更。

## 训练任务

设计 5 次主要循环，每次写新增信息、失败原因、新爽点和离真相推进的距离。
