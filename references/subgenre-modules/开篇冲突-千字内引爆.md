# 开篇冲突-千字内引爆

依据案例：`batch-326` 案例 1627

## 读者承诺

新人开篇必须尽快交代主角是谁、在哪、要干什么、面对什么冲突，并在第一章内引爆冲突。

## 核心循环

```text
主角登场 -> 身份/目标明确 -> 直接冲突出现 -> 系统或金手指有必要 -> 章末引爆
```

## 开篇必备

- 1000 字内交代主角处境和目标。
- 冲突必须直接影响主角。
- 系统出现要有必要性。
- 无冲突路程压成一句。
- 章末前让主角必须正面应对。

## 期待来源

读者期待第一章就知道主角为什么值得写，以及下一章要解决什么问题。

## 断档风险

- 主角一直呼叫系统，但没有危机。
- 系统无效却占据大量篇幅。
- 三章看完仍不知道主角要干什么。
- 主角对原主关系处理冷漠，形象变差。
- 冲突平淡，读者不等后文解释。

## 训练任务

把一个穿越开篇缩成 1000 字，只保留身份、目标、冲突、系统必要性和章末引爆点。
