# 系统智能-无意义对话毒点

依据案例：`batch-169` 案例 842。

## 读者承诺

系统流的核心承诺是任务、规则、奖励、限制和即时反馈；系统台词必须服务这些承诺，不能变成陪聊水字数。

## 核心循环

```text
系统发声 -> 给出规则/任务/限制 -> 主角做选择 -> 奖惩兑现 -> 新规则打开
```

## 开篇必备

- 系统第一次出现要让读者知道它能改变什么。
- 系统台词要带任务、奖励、警告、限制或误导。
- 主角不能长时间和系统无意义拌嘴。
- 智能语气可以有，但每次发声都要推进剧情。

## 期待来源

读者期待系统像发动机一样持续制造目标、反馈和选择压力，而不是像旁白角色一样消耗篇幅。

## 断档风险

- 系统和主角聊天很热闹，但剧情没有变化。
- 系统人格抢过主角、女主、导师或吐槽役功能。
- 任务规则不清，读者只记得系统在说话。
- 删除系统台词后剧情不受影响。

## 训练任务

写一章系统流开头后，逐句检查系统台词。每句标注“任务、奖励、限制、设定、推进、无效”六类之一，删掉或改写所有“无效”台词。
