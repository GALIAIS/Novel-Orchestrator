# 投稿资源-情报验证

依据案例：`batch-140` 案例 699。

## 读者承诺

把编辑联系方式、投稿渠道、更新入口等稀缺资料整理成可行动资源，同时标明来源、时效和验证状态，减少新人寻找成本。

## 核心循环

```text
发现外部资料 -> 标明来源和对象 -> 收藏备用 -> 补充更新渠道 -> 使用前验证时效
```

## 开篇必备

- 资料来源。
- 适用平台。
- 适用作者。
- 更新时间。
- 验证状态。
- 风险提示。

## 期待来源

- 新人快速找到投稿入口。
- 分散资料被集中保存。
- 持续更新渠道降低过期风险。
- 风险提示避免误当官方保证。

## 断档风险

- 转发图片不可见或不可查。
- QQ 清单过期。
- 编辑身份无法验证。
- 把资源情报误解成过稿捷径。

## 训练任务

写 5 条稀缺资料转发型动态，每条包含来源、资料对象、适用人群、收藏理由和缺失验证点。再扩写 1 条为投稿新人资源帖，补齐更新时间、平台、风险提示和后续更新渠道。
