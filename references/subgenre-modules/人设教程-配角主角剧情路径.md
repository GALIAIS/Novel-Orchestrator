# 人设教程-配角主角剧情路径

依据案例：`batch-077` 案例 382。

## 读者承诺

这个模块用于系统学习或设计人物。它承诺从人设基本功能出发，先有目的地创建配角，再塑造主要角色，最后让人设反过来推动剧情。

## 核心循环

```text
人设概述 -> 配角目的 -> 主角立体化 -> 人设碰撞 -> 剧情更精彩
```

## 开篇必备

- 人设不是标签，而是剧情功能。
- 配角要有服务对象和目的。
- 主角要有多层价值、欲望和行动。
- 人设必须在事件中被验证。
- 人物关系要制造冲突或帮助。

## 期待来源

- 配角功能清楚，剧情不散。
- 主角立体，读者能持续理解选择。
- 人设差异制造碰撞。
- 人物关系推动事件变化。
- 系列化学习路径让缺口可补。

## 断档风险

- 只做人设卡，不写事件。
- 配角无目的，只凑人数。
- 主角只有性格词。
- 人设不影响剧情。
- 教程学习停在概述，没进入实操。

## 训练任务

给一个主角配三个配角：助力、阻力、镜像。再写一个事件，要求三名配角的性格和立场都改变事件走向。
