# 保底买断-收入能力换算

依据案例：`batch-059` 案例 294。

## 读者承诺

这个模块用于作者拿到保底或买断报价后判断自身商业位置。它承诺的是用收入、更新量、参考均订和合同风险做粗估，而不是用单个报价给自己定级。

## 核心循环

```text
收到报价 -> 明确千字/日更/月收入 -> 对照分成均订 -> 评估现金流和上限 -> 检查合同风险 -> 决定接受
```

## 开篇必备

- 先弄清合同形式：保底、买断、分成、保底分成。
- 计算预计日更和月收入。
- 用同年份同平台数据做参照。
- 估算对应分成均订区间。
- 查版权、结算、违约、更新压力。

## 期待来源

- 作者知道报价大概对应什么商业水平。
- 合同选择服务现金流需求。
- 上限预期和稳定收入能比较。
- 不被“保底”两个字冲昏头。
- 能和下一本策略连接。

## 断档风险

- 用旧平台表格精确套现平台。
- 只看月收入，不看版权和上限。
- 忽略日更压力。
- 误把保底报价当作品真实上限。
- 不考虑同题材样本和平台生态。

## 训练任务

用一个保底报价做三档测算：日更 4 千、6 千、8 千。写出月收入、对应分成均订估计和你能否长期完成。
