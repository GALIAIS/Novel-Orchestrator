# 新人首签-合同审阅与首卷交付闭环

依据案例：`batch-405` 案例 2023

## 读者承诺

把首次终审通过视为商务流程起点：正式报价、完整合同、条款澄清、双方签署和生效逐项过门；签后以前三章、小单元和首卷完成稳定交付。

## 核心循环

```text
终审通过 -> 正式报价与合同 -> 条款矩阵审阅 -> 产能和权利评估 -> 签署生效 -> 前三章与单元交付 -> 首卷回收 -> 验收结算留证
```

## 开篇必备

- 区分终审、报价、签署、生效和付款状态。
- 获取完整正式合同而非依赖聊天截图。
- 核对授权、交付、结算、修改和解约。
- 模糊单位与触发条件获得书面澄清。
- 计划符合个人可持续产能。
- 首卷具备力量资源、阶段目标与卷末回收。

## 期待来源

商务状态逐步落字，创作状态逐步闭合；作者不靠“百万字”证明成长，而靠首卷按质交付、验收和回收建立能力证据。

## 断档风险

- 从聊天信息推断合同已经生效。
- 不知道价格单位、期限和触发条件。
- 忽略有声、出版、改编等权利范围。
- 签署后才发现更新义务超出产能。
- 把签约等同于长篇结构成功。
- 为固定字数延长已经结束的故事。

## 训练任务

用完整合同制作条款矩阵并列出待澄清项；把第一卷拆成前三章、首单元、卷末三个交付门槛，分别写验收证据和稳定更新量。
