# 404禁区-敏感剧情资料索引

依据案例：`batch-052` 案例 260。

## 读者承诺

写高风险题材前先做 404 预检：确认平台尺度、敏感桥段和替代表达，保留冲突张力但避免作品直接被下架。

## 核心循环

```text
刺激桥段 -> 风险识别 -> 平台校验 -> 表达转译 -> 保留冲突 -> 避免 404
```

## 开篇必备

- 明确目标平台。
- 列出可能触碰的敏感剧情。
- 检查是否有真实人物、组织、地名映射。
- 准备架空化、弱化化、替代化方案。
- 如果核心卖点必须靠禁区支撑，优先换题材。

## 期待来源

- 风险预检能保护长线更新。
- 转译后仍能保留冲突。
- 平台尺度清楚，作者写作更稳。
- 架空表达能降低现实联想。

## 断档风险

- 只追求刺激，不看平台尺度。
- 资料索引过期，仍需上传和平台规则校验。
- 现实映射太强，替换名词也未必安全。
- 删掉敏感点后卖点消失，说明题材根基不稳。

## 训练任务

做一张 404 预检表：列出题材、平台、3 个敏感桥段、3 个替代表达，以及替换后保留的冲突。
