# 悬念结构-信息差与倒计时

依据案例：batch-299 案例 1493

## 读者承诺

悬念先让读者关心角色，再用有限答案、信息差和倒计时制造紧张或好奇。

## 核心循环

```text
读者关心角色 -> 提出问题 -> 限定2-3个答案 -> 设置信息差/倒计时 -> 兑现答案
```

## 开篇必备

- 问题清楚。
- 可能答案不超过三项。
- 明确角色知道什么、读者知道什么。
- 区分危机悬念和秘密好奇。

## 期待来源

- 角色不知道读者知道，制造潜藏危机。
- 角色和读者都知道，制造确定危机。
- 角色知道读者不知道，制造扣留信息。

## 断档风险

- 答案太多，读者选择困难。
- 读者不关心角色，悬念无效。
- 角色和读者都不知道且无铺垫，容易变毒点。

## 训练任务

设计一个倒计时悬念，写出角色视角信息、读者视角信息、两个可能答案和最终兑现。
