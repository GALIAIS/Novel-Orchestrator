# 打斗场景-风格侧写替招式

依据案例：`batch-335` 案例 1673

## 读者承诺

打斗不一定要逐招拆解，可以用风格、武器差异、伤势、旁观反应和结果侧写出战斗的势。

## 核心循环

```text
确定打斗基调 -> 写资源/武器差异 -> 写伤势或失败结果 -> 旁观反应补强 -> 胜负趋势兑现
```

## 开篇必备

- 先定诙谐、冷峻、惨烈或潇洒基调。
- 武器和使用者差异绑定。
- 至少一处伤势或结果可视化。
- 旁观者反应服务气势。
- 胜负趋势清楚。

## 期待来源

读者期待看见强弱、压制和风格记忆点，而不只是招式名和动作流水账。

## 断档风险

- 招式堆砌但没有胜负变化。
- 风格和题材基调冲突。
- 伤势描写只为血腥。
- 旁观者只喊口号。
- 侧写过多导致战斗结果不清。

## 训练任务

写同一场打斗两版：一版用招式过程，一版只用武器差异、伤势、旁观反应和结果，比较哪版更符合本书基调。
