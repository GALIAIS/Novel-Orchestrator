# 网文语句流畅度-汉语表达

依据案例：`batch-159` 案例 795。

## 读者承诺

让读者快速扫过去就能理解句子含义，减少翻译腔、定语堆叠、长段和静态描述造成的阅读阻力。

## 核心循环

```text
扫读句子 -> 标出费力处 -> 改动作表达 -> 拆长句长段 -> 移动端复查 -> 阅读阻力下降
```

## 开篇必备

- 第一章句子要直接、清楚、顺口。
- 少用过长主语和多层定语。
- 描写尽量转成动作、反应和事件。
- 分段适配移动端阅读。

## 期待来源

流畅语句让读者无障碍接收剧情、爽点、信息差和情绪变化，不被表达方式拦在故事外面。

## 断档风险

- 英语式词序和翻译腔。
- 静态形容词连环堆叠。
- 一段几百字不分段。
- 读者需要回看才能明白句意。

## 训练任务

从正文中抽 20 个句子做扫读测试。凡是 2 秒内不能理解的句子，改成更短、更动态、更符合汉语习惯的表达，并在手机宽度下复读一遍。
