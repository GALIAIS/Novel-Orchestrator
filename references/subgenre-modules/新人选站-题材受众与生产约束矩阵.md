# 新人选站-题材受众与生产约束矩阵

依据案例：`batch-410` 案例 2048

## 读者承诺

用题材读者承诺、当前可比样本、稳定产能、商业模式、合同边界和作者语感共同选择试投平台；热点只在改变核心循环时融合，系统载体先按功能而非外观设计。

## 核心循环

```text
明确主承诺与次级风味 -> 建立当前成功和未增长样本 -> 实测更新存稿能力 -> 核验商业合同边界 -> 加权平台匹配 -> 设计系统功能接口 -> 写开篇首弧 -> 复评后试投
```

## 开篇必备

- 题材的核心读者承诺。
- 候选平台的当前成功与未增长样本。
- 连续四周实测的更新和修改产能。
- 商业模式、合同与版权底线。
- 系统接口真正需要完成的功能。
- 开篇和首弧完成后的复评节点。

## 期待来源

故事承诺与平台读者先形成初始匹配，作者产能保证持续交付；功能化系统组织群体并制造失控，当前样本和复评让选站成为可修正决策。

## 断档风险

- 寻找抽象的“新人最容易成功平台”。
- 只看老爆款，不看近期冷启动样本。
- 忽略自身稳定更新与修改能力。
- 用旧评论替代当前合同和推荐机制。
- 系统人格只换萌宠或圣兽外观。
- 热点贴在标题上却不改变故事循环。

## 训练任务

为三个候选平台分别建立五本当前可比作品和两本未增长样本，以八项矩阵评分并标注证据日期；完成 5000 字开篇和首弧小纲后重新评分一次。
