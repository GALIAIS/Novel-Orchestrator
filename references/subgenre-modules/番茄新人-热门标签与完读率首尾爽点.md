# 番茄新人-热门标签与完读率首尾爽点

依据案例：`batch-202` 案例 1006。

## 读者承诺

番茄新人先从平台热门标签、话题广场求书需求和热门书热度找方向，再用章节首尾爽点提高完读率。

## 核心循环

```text
热门标签 -> 求书需求 -> 热门书热度转译 -> 开头爽点 -> 结尾钩子 -> 完读率
```

## 开篇必备

- 读者一眼能识别热门口味。
- 第一章开头先给强刺激。
- 章尾给爽点或追读钩子。
- 借热度但不抄袭。

## 期待来源

读者通过标签和书名进入，期待立刻得到熟悉类型的爽点反馈。

## 断档风险

- 功底不足还强碰纯原创。
- 蹭热门变成抄袭。
- 章节中段拖沓，首尾没有刺激点。
- 标签热门但正文兑现不够。

## 训练任务

选一个番茄热门标签，写 5 个不抄袭书名，并给第一章设计开头爽点和结尾爽点。
