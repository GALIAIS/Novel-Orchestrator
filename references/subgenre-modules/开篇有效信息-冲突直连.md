# 开篇有效信息-冲突直连

依据案例：`batch-065` 案例 322。

## 读者承诺

这个模块用于新人开篇两万字删改。它承诺的是只保留能直接指向冲突爆发事件、人物、时间、地点的信息；隔两层以上的设定、地点、配角和金手指功能全部删除或后置。

## 核心循环

```text
提炼冲突四要素 -> 逐段连线 -> 直连保留 -> 一层压缩 -> 两层删除 -> 冲突更清楚
```

## 开篇必备

- 先写出冲突爆发事件。
- 写出关键人物、时间、地点。
- 每段信息都要能连到四要素。
- 金手指只详写当前用得到的功能。
- 无戏份配角不命名。

## 期待来源

- 读者快速理解当前冲突。
- 设定只在需要时出现。
- 人物、地点、时间都服务爆发点。
- 冲突推进不被解释打断。
- 信息密度变高但不乱。

## 断档风险

- 作者觉得必要，读者觉得水。
- 设定 A 引出 B，B 又引出 C。
- 开篇同时介绍多个暂时无关地点。
- 金手指全功能说明书。
- 删除时心疼，导致开头臃肿。

## 训练任务

把第一章每段标成 A/B/C：A 直连当前冲突，B 隔一层，C 隔两层以上。删除 C，压缩 B，保留 A。
