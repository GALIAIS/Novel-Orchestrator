# 阅读训练-扫读模式与主题拆书

依据案例：batch-351 案例 1753

## 读者承诺

写作者阅读要按目的切换速度：扫读套路、慢读变体、分析结构、主题对比。

## 核心循环

```text
确定阅读目的 -> 扫过熟悉模块 -> 慢读关键变体 -> 记录结构 -> 横向对比
```

## 开篇必备

- 区分消费、检视、分析、主题阅读。
- 网文常规模块可扫读。
- 新设定和关键爽点慢读。
- 拆书时必须记录结构。
- 主题研究要同时对比多本。

## 期待来源

作者读得更快，因为知道哪里该快、哪里必须慢。

## 断档风险

- 每本都精读，效率低。
- 每本都扫读，学不到结构。
- 不记录读后产出。
- 阅读目的不清。
- 模式识别不足时强行跳读。

## 训练任务

30 分钟检视一本同题材书，只记录开头承诺、金手指、第一爽点、主线循环。
