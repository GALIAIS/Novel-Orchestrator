# 人物功能-主角对手爱恋对象

依据案例：`batch-322` 案例 1608

## 读者承诺

主要人物要先服务故事功能：主角承受命运打破，对手坚定阻止主角，爱恋对象带来意外和改变。

## 核心循环

```text
故事需求 -> 构造人物功能 -> 对手制造险境 -> 爱恋对象改变主角 -> 冲突与情感同步推进
```

## 开篇必备

- 一卷先设计 5-7 个主要人物功能。
- 主角有独一无二性和命运缺口。
- 对手从主角彻底失败反推。
- 对手坚信自己必须阻止主角。
- 爱恋对象必须推动故事改变。

## 期待来源

读者期待每个主要人物都改变局势、压迫主角或让主角成长，而不是只占戏份。

## 断档风险

- 先堆人设，后找功能。
- 反派良知太多，险境不够狠。
- 对手不是聚合矛盾的独立个体。
- 爱恋对象只负责陪伴，不制造故事改变。
- 爱情线没有接近、共同完成、现实打击和过渡。

## 训练任务

列一卷 5 个主要人物，每人只写剧情功能和让主角发生的改变，不写外貌标签。
