# 飞卢系统-高频获得爽点

依据案例：`batch-022` 案例 109。

## 读者承诺

用高频获得道具、血脉、技能、职业制造基础爽点，适合文笔和人物暂时不强的新人先把爽点密度拉起来。

## 核心循环

```text
事件 -> 获得奖励 -> 立刻展示 -> 再获得 -> 能力配套 -> 控制层级
```

## 开篇必备

- 系统奖励规则。
- 高频获得入口。
- 能力归属的统一体系。
- 当章至少一次展示。

## 期待来源

- 读者不断看到主角变强。
- 每次战斗或事件都有收益。
- 多项能力配套服务同一主线。

## 断档风险

- 十万字只升级一两次。
- 能力获得后不用。
- 前期给太多导致后期崩。
- 熔炉整合功能过早出现，消融获得快感。

## 训练任务

写一章系统剧情，至少三次获得奖励。每个奖励必须标注所属体系、当章用途或后续支线用途。
