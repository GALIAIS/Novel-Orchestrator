# 热点梗-书名章节名拉期待

依据案例：`batch-196` 案例 976。

## 读者承诺

用热点梗做书名或章节名，可以快速压缩题材、冲突和第一爽点，但后续必须有金手指机制续航。

## 核心循环

```text
热点识别 -> 标题承诺 -> 简介补强 -> 第一爽点兑现 -> 机制续航
```

## 开篇必备

- 热点梗读者一眼能懂。
- 标题包含主角处境和爽点。
- 第一章兑现标题噱头。
- 后续有可持续金手指机制。

## 期待来源

读者带着对热点的既有情绪进入故事，期待看到熟悉梗被小说化、战斗化或系统化。

## 断档风险

- 只有标题好笑，正文没有机制。
- 开头爽完后疲软。
- 热点过期太快。
- 章节名噱头和正文内容脱节。

## 训练任务

选一个热点梗，写 3 个书名和 3 个章节名，并为每个标题补一个后续机制升级点。
