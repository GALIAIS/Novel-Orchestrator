# 聚会装逼-第三方抢先虚晃

依据案例：`batch-018` 案例 089。

## 读者承诺

在 KTV、宴会、同学会、生日局等聚会场景里，让主角不是立刻下场打脸，而是先被第三方抢走出手机会，再通过第三方失败把主角补位的期待拉高。

## 核心循环

```text
反派设局 -> 主角准备出手 -> 第三方抢先 -> 第三方被反压 -> 主角补位 -> 更高层爽点
```

## 开篇必备

- 聚会场的社交压力。
- 反派想控制或展示的对象。
- 主角已经准备上场的动作。
- 第三方抢先的动机和关系立场。

## 期待来源

- 读者想看主角对上反派。
- 第三方抢先让期待短暂落空。
- 第三方失败证明反派不是空壳。
- 主角补位时用更高维标准解决问题。

## 断档风险

- 第三方没有动机，只是工具人。
- 主角从头到尾旁观，没有“没上成”的落空。
- 反派反压第三方太弱，主角出手没有必要。
- 爽点只停留在技能赢，不改变关系和场面评价。

## 训练任务

设计一个聚会装逼单元：主角准备出手，闺蜜、同学或同事抢先失败，主角再用更高层判断破局。写清第三方为什么抢。
