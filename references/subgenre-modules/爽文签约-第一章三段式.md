# 爽文签约-第一章三段式

依据案例：`batch-128` 案例 640。

## 读者承诺

爽文第一章要快速让读者知道主角是谁、卡在哪、凭什么翻盘、马上要怎么爽。世界观只给最低限度，金手指早露面，事件必须撞回开局困境。

## 核心循环

```text
世界/困境压缩 -> 金手指早亮 -> 困境事件爆发 -> 男主即将出手 -> 断章拉期待
```

## 开篇必备

- 500-800 字内交代最低限度世界和困境。
- 500 字内亮出金手指或底牌。
- 第三段事件必须照应困境。
- 断在男主即将出手。
- 总字数控制在 2000-2500 字左右。

## 期待来源

- 读者快速代入。
- 困境制造憋屈。
- 金手指制造翻盘希望。
- 出手前断章制造下一章期待。

## 断档风险

- 开局铺设定太久。
- 金手指迟迟不出现。
- 第三段事件和开局困境无关。
- 行动已经写完才断章，期待被提前耗尽。
- 把爽文模板误套到非爽文。

## 训练任务

用不超过 2500 字设计第一章大纲：500-800 字交代主角困境和最低限度世界信息，500 字内亮出金手指，最后安排一个直接撞上困境的事件，并断在主角即将出手处。
