# 平淡剧情-主动情绪拉扯

依据案例：`batch-224` 案例 1118。

## 读者承诺

把原本干巴巴的主线事件加工成有等待、误判、代价、反应和释放的可读场面。

## 核心循环

```text
平淡事件 -> 找情绪点 -> 延迟/误解/压力 -> 反应放大 -> 爽点释放
```

## 开篇必备

- 写清主角想要什么。
- 给出阻力和代价。
- 保留读者期待，不要直接跳结果。
- 释放前安排旁人反应。

## 期待来源

读者不是嫌主线事件少，而是嫌情绪没有被写出来；拉扯让事件变得有重量。

## 断档风险

- 只顾赶主线，爽点草草带过。
- 进度快但情绪薄。
- 把拉扯误解成无意义拖字数。
- 没有读者期待，只有事件流水账。

## 训练任务

选一个主角拿奖励的平淡桥段，先写 100 字快速版，再改成 500 字拉扯版，必须加入等待、误判、代价、旁人反应、最终释放。
