# 跨平台投稿-文风错配扫榜

依据案例：`batch-083` 案例 414。

## 读者承诺

这个模块用于判断一本文是否适合换平台投稿。它承诺用扫榜判断文风、开头信息包和长线能力是否匹配目标站，而不是把原平台成绩直接搬过去。

## 核心循环

```text
原平台成绩 -> 目标平台扫榜 -> 文风适配 -> 长线审查 -> 投稿决策
```

## 开篇必备

- 先确认目标平台同题材样本。
- 比较前三章开头风格。
- 看信息包是否服务长线。
- 判断目标读者耐受度。
- 不用单本成绩证明跨站可行。

## 期待来源

- 作者知道自己文风属于哪里。
- 避免发错网站单机。
- 编辑门槛提前预判。
- 长线能力不被开头爆点掩盖。
- 平台选择基于证据。

## 断档风险

- 飞卢天榜经验直接投起点。
- 渠道向开头放错平台。
- 只为前 5-10 万字爆发堆信息包。
- 没有长线安排。
- 四处问平台，不自己扫榜。

## 训练任务

用自己的前三章对照三个平台各 5 本同题材开头。填“爆点密度、长线信息、读者耐受度”，选最匹配平台。
