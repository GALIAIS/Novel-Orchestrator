# 期待与代入-需求解法与题面完备

依据案例：`batch-418` 案例 2090

## 读者承诺

读者会看见一个值得解决的需求、一条可信但尚未完成的路径，以及足以参与人物判断的条件。类型标签先建立总体合同，人物意义、时间压力和失败损失让需求具体；题面提供信息、能力和限制，人物主动选择并承担成本，部分验证逐步证明路径，最终解法只重组已给条件完成公平兑现。

## 核心循环

```text
类型标签建立总体内容合同 -> 可理解人物出现重要需求 -> 个人意义、时间压力和失败损失增加迫切性 -> 题面给出已知、未知、能力和限制 -> 可见但未完成的解法路径出现 -> 人物主动选择并承担成本 -> 障碍检验路径而非任意取消规则 -> 小证据部分验证能力和世界逻辑 -> 新成本保持结果不确定 -> 高潮重组已给条件公平兑现 -> 结果开启下一层期待
```

类型合同决定期待方向，人物需求提供情绪动力，题面完备允许读者参与，可见路径制造未来感，部分验证维持可信度，公平兑现同时保护期待与代入。

## 开篇必备

- 类型标签与读者总体内容合同。
- 一个可理解或讨喜的焦点人物。
- 当前重要需求及其个人意义。
- 时间压力和失败的具体损失。
- 人物已知与未知信息。
- 可用能力、资源、关系和金手指边界。
- 时间、空间和规则限制。
- 读者能够形成的粗略解法。
- 一条可见但未完成的路径。
- 第一笔与类型一致的小额兑现。

## 期待来源

- 人物的重要需求能否得到满足。
- 读者的粗略解法与人物选择有何差异。
- 可见路径是否真的可达。
- 金手指会如何扩展合法解法空间。
- 每次部分验证会新增什么成本。
- 障碍是否迫使人物调整而不破坏规则。
- 高潮将如何重组已经出现的条件。
- 原需求兑现后会出现什么更高层问题。

## 断档风险

- 只有危机，没有任何可信解决路径。
- 只有远方目标，没有距离、成本和失败损失。
- 把需求规模误当成情绪强度。
- 用无条件天文回报替代可信状态变化。
- 题面缺少当前选择所需条件。
- 解法偏离读者模型，却没有能力或规则铺垫。
- 金手指取消全部限制和选择。
- 用悬崖钩子代替类型内容和人物魅力。
- 部分验证没有新增成本，结果过早确定。
- 高潮临时添加万能能力或规则。

## 训练任务

写一个 1800 字困境场景：前三百字交代人物需求、个人意义、时间压力和失败损失，随后给出足以让读者形成粗略解法的题面和一条可见路径。中段安排两次部分验证，每次都提高可信度并新增成本；高潮只能重组已出现的条件解决问题，结尾兑现原需求并生成下一层期待。
