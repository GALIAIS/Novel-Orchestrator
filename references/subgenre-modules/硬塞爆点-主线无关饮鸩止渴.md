# 硬塞爆点-主线无关饮鸩止渴

依据案例：`batch-117` 案例 581。

## 读者承诺

爆点必须改变主线、人物状态或后续目标。只为短期刺激硬塞的小爆点，会制造后续无法回收的结构债。

## 核心循环

```text
数据焦虑 -> 想加爆点 -> 检查主线关系 -> 无关则删除 -> 有关则安排回收
```

## 开篇必备

- 爆点服务当前主线。
- 爆点改变人物状态。
- 爆点不破坏既有设定。
- 爆点后有连续回收。
- 删除后剧情不能完全不变。

## 期待来源

- 读者看到爆点后期待后续影响。
- 爆点推动下一目标。
- 短期刺激和长期结构一致。
- 作者不用后期硬圆。

## 断档风险

- 只为拉订阅临时加戏。
- 小爆点和主线无关。
- 爆点破坏设定。
- 后续圆不回来导致切书。

## 训练任务

给一个临时爆点写三条后续影响。如果写不出来，就删掉；如果能写出来，把它并入主线节点。
