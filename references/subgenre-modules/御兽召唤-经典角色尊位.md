# 御兽召唤-经典角色尊位

依据案例：`batch-024` 案例 116。

## 读者承诺

召唤经典神话或名著角色时，读者要看到尊位、原典味和第一次能力救场，而不是普通宠物收服。

## 核心循环

```text
前置危机 -> 原典线索 -> 经典角色救场 -> 尊位确认 -> 主角建立关系 -> 冒险推进
```

## 开篇必备

- 被召唤角色的时期。
- 角色不能被降格的身份边界。
- 主角与角色的朋友、前辈或盟友关系。
- 一次能让读者认出的原典能力。

## 期待来源

- 读者自带原典记忆。
- 角色尊位高，出场有压迫感。
- 危机中救场直接证明价值。
- 主角如何与强者相处形成后续看点。

## 断档风险

- 把经典角色写成宠物或工具。
- 前面水考试、吐槽和设定，迟迟不进入冒险。
- 能力靠面板说明，没有剧情展示。
- 主角短期赚钱计划冒犯角色尊位。

## 训练任务

写一个 1500 字经典角色召唤开篇：用危机、线索、救场三步完成角色出场，并标明主角不能越过的尊重边界。
