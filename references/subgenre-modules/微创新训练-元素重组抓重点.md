# 微创新训练-元素重组抓重点

依据案例：batch-287 案例 1432

## 读者承诺

用成熟套路元素重新组合出一眼可见的小变化，并把新意落到人物、冲突、金手指或关系上。

## 核心循环

```text
识别老套反馈 -> 扫榜拆元素 -> 选择一个重组点 -> 删掉前置理论 -> 写出开篇卖点 -> 实践验证
```

## 开篇必备

- 明确一个基础套路。
- 明确一个替换或反转元素。
- 第一眼能看懂的新意。
- 人物欲望和冲突先出现。
- 世界观解释后置。

## 期待来源

读者先获得熟悉类型的安全感，再看到一个可理解的新组合，于是产生“这个套路会怎么变”的点击欲望。

## 断档风险

- 把微创新写成抽象理论。
- 开篇先讲宇宙构成。
- 元素换得太多，读者失去类型识别。
- 扫榜只收藏素材，不落地实践。
- 过度抠概念，忽略大点和重点。

## 训练任务

选一个常见套路，列出人物、金手指、冲突、场景、关系五个元素，只改其中一个，写出 300 字开篇卖点。
