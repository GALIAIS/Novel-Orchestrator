# 平台保底-渠道定位与自我要求

依据案例：batch-382 案例 1908

## 读者承诺

保底和全勤只能提供收入底线。平台选择还要核对当前合同、渠道受众、作品风格、更新成本和持续质量目标。

## 核心循环

```text
核对合同 -> 判断渠道匹配 -> 评估更新成本 -> 获得收入底线 -> 数据验证成长
```

## 开篇必备

- 当前官方规则。
- 目标渠道与受众。
- 作品风格匹配度。
- 数据和质量底线。

## 期待来源

- 短期现金流更稳定。
- 作品进入匹配渠道。
- 更新量可持续。
- 保底之外仍有成长反馈。

## 断档风险

- 使用过期金额决策。
- 混淆保底、全勤和分成。
- 只码够字数混日子。
- 忽略渠道风格和读者数据。

## 训练任务

用当前官方信息为两个平台各做决策表，比较受众、渠道、更新量、收入底线、质量目标和退出条件。
