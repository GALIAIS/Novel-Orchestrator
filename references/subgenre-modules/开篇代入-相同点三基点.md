# 开篇代入-相同点三基点

依据案例：`batch-012` 案例 057。

## 读者承诺

当主角是原创人物时，用一个目标读者熟悉的状态、事件或情绪，让读者在戏剧性正式启动前先和主角建立连接。

## 核心循环

```text
原创主角 -> 基础人设对齐 -> 熟悉事件/状态 -> 情绪类同 -> 羡慕点/切入点 -> 开篇启动
```

## 开篇必备

- 判断主角是否需要相同点：熟悉人物可不加，原创主角更需要。
- 年龄、心理性别、性取向、相貌、名字先符合目标读者和题材。
- 相同点必须是读者熟悉且有情绪需求的具体事件或状态。
- 相同点必须指向羡慕点或切入点。

## 期待来源

- 读者看到“他像我”或“他是我想象中更强的我”。
- 主角做了读者想做但不敢做的事。
- 相同点触发金手指、关系变化或主线事件。
- 读者期待主角继续替自己完成情绪补偿。

## 断档风险

- 相同点只是热梗，和主线没有关系。
- 人设基础不符合目标读者，后续事件再像也无法代入。
- 一个开篇堆多个相同点，导致切入点延迟。
- 相同点只有共鸣，没有羡慕点或戏剧性后果。

## 训练任务

为一个原创主角写一个相同点，列出目标读者、共同情绪、羡慕点、切入点。如果不能指向后两项，删除该相同点。
