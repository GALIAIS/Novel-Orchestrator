# 提问复盘-费曼自问自答

依据案例：`batch-105` 案例 521。

## 读者承诺

把提问当成复盘工具。为了让别人看懂，你会被迫整理条件、缺口和选择，很多问题会在发送前被自己解决。

## 核心循环

```text
想提问 -> 写清背景 -> 梳理条件 -> 发现逻辑缺口 -> 自己推导 -> 精简后再问
```

## 开篇必备

- 问题发生在哪。
- 已知条件是什么。
- 卡住的是哪个选择。
- 已试过什么方案。
- 还缺哪类外部判断。

## 期待来源

- 问题变清楚。
- 自己能找到答案。
- 外部回答更精准。
- 复盘成本低。

## 断档风险

- 脑子热就发问。
- 只求别人给答案，不整理条件。
- 问题太散，别人无法判断。
- 明明自己能解决却没写出来。

## 训练任务

每次提问前先写 8 格提问表。写完后删掉已解决部分，只留下真正需要别人判断的内容。
