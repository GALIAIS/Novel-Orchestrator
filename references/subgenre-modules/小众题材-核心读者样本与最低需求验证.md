# 小众题材-核心读者样本与最低需求验证

依据案例：`batch-397` 案例 1982

## 读者承诺

把作者的小众兴趣转成一类读者可识别的稀缺体验，通过高表现、稳定和失败样本加目标读者短测，确认方向值得投入而不稀释核心味道。

## 核心循环

```text
定义未满足需求 -> 建立三层样本矩阵 -> 分类章评 -> 标题简介短测 -> 三章验证 -> 控制成本并深化核心体验
```

## 开篇必备

- 一句话说清小众核心乐趣。
- 写出目标读者的必须项和拒绝项。
- 首章迅速证明核心味道存在。
- 相邻题材只负责提供清晰入口。
- 评论按爽点、毒点、困惑和争议分类。
- 项目长度与验证强度匹配。

## 期待来源

核心读者发现长期缺失的体验被准确理解，并相信作品不会中途为了讨好更大人群而丢掉最初承诺。

## 断档风险

- 把个人不合群当作市场定位。
- 只研究榜首幸存样本。
- 用评论数量直接判断爽点。
- 标签相似却核心乐趣不同。
- 五名读者喜欢就外推为大市场。
- 为扩大入口删掉小众核心。

## 训练任务

各选三本高表现、稳定和失败样本，记录核心乐趣、首轮兑现与断档点；再用一句话承诺和 3000 字短样测试五名目标读者，先验证他们能否复述同一体验。
