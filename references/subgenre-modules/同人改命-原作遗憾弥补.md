# 同人改命-原作遗憾弥补

依据案例：`batch-082` 案例 406。

## 读者承诺

这个模块用于判断同人设定是否有点击价值。它承诺把看点落在原作遗憾的改写上，而不是只做 IP 和角色拼接。

## 核心循环

```text
原作遗憾 -> 新变量介入 -> 角色改变 -> 关系重写 -> 遗憾弥补
```

## 开篇必备

- 第一章点出原作遗憾。
- 新变量必须立刻影响主角。
- 主角改变要能被读者看见。
- 原作关系要出现第一次重写。
- 书名筛选读者时，要知道门槛。

## 期待来源

- 读者想看原作命运被改。
- 被欺负、被压迫、被误解的角色获得补偿。
- 强人设导师或外力制造新路线。
- 原作关系产生反转。
- 熟悉原作的读者获得“如果当初”的爽感。

## 断档风险

- 只拼两个 IP，没有原作遗憾。
- 双 IP 门槛太高。
- 黑暗流、非穿越过度筛读者。
- 三观改造让原作粉不接受。
- 新人学冷门题材而不是学改命结构。

## 训练任务

写一个同人开篇方案，只填五项：原作遗憾、新变量、主角第一处改变、原作关系第一次反转、读者补偿感。填不出来就不要开长篇。
