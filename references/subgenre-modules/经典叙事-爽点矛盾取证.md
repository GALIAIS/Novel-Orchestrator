# 经典叙事-爽点矛盾取证

依据案例：`batch-055` 案例 273。

## 读者承诺

这个模块用于把经典作品、视频解析、文学评论转成网文可用训练材料。它不承诺照搬经典题材，而是提炼其中仍然有效的爽点、矛盾、人物关系压力和叙事技法。

## 核心循环

```text
选择经典资料 -> 辨别可迁移技法 -> 抽出矛盾与爽点 -> 舍弃过时时代语境 -> 改写成当代网文事件 -> 验证读者期待
```

## 开篇必备

- 先确定要学的是爽点、矛盾、人物关系，还是叙事节奏。
- 不因题材过时否定结构技法。
- 不因资料作者争议全盘否定内容。
- 每条观点都要能落到一个具体事件。
- 经典材料必须转成自己的训练题。

## 期待来源

- 经典矛盾通常经过长期验证，具备稳定戏剧性。
- 人物关系压力可迁移到现代题材。
- 爽点不是时代装饰，而是期待被压制后释放。
- 旧结构换新语境，会产生新鲜感。
- 视频资料能降低入门门槛，但必须二次取证。

## 断档风险

- 看完资料只记金句，不做事件改写。
- 迷信经典，照搬过时桥段。
- 只争论资料作者立场，不验证结构是否有用。
- 把文学性当成网文爽点，导致开篇慢热。
- 没有区分“可迁移技法”和“不可迁移时代语境”。

## 训练任务

选一个经典叙事分析资料，记录 3 个矛盾设计例子。每个例子改写成一个现代网文小单元，必须包含目标、阻碍、误判、转折、兑现和新期待。
