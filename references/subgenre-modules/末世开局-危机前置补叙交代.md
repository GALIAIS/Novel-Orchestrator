# 末世开局-危机前置补叙交代

依据案例：`batch-180` 案例 897。

## 读者承诺

末世开局优先给持续生存压力，弱第一幕可后置补叙，强危机可直接前置。

## 核心循环

```text
强危机前置 -> 主角即时目标 -> 逃生/反击 -> 后置信息补叙 -> 生存压力不断线
```

## 开篇必备

- 开局有当前生死危机。
- 主角有明确逃生或反击目标。
- 末世前史只补必要信息。
- 补叙不能打断当前压力。

## 期待来源

读者期待怪物、人性、资源和猎杀压力持续存在。时间顺序不如当前危机重要。

## 断档风险

- 第一章后长时间无危机。
- 补叙变成解释包。
- 只写小爽点，没有生存压力。
- 猎杀危机前置后，前史补得太重。

## 训练任务

把一个末世顺叙开头改成危机前置版，用一句对话、一个环境细节、一段短回忆补出前史。
