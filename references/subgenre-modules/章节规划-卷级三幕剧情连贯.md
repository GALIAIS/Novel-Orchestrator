# 章节规划-卷级三幕剧情连贯

依据案例：`batch-098` 案例 488。

## 读者承诺

三幕式主要服务卷级剧情波动，不是每章机械套模板。章节规划要跟整体情节走，一小段剧情结束就断章，爽点在卷级上形成一浪接一浪。

## 核心循环

```text
卷级三幕 -> 小段剧情 -> 自然断章 -> 章末期待 -> 卷纲调整 -> 爽点爆发
```

## 开篇必备

- 先有卷级目标和阶段。
- 每章服务当前小段剧情。
- 断章点来自情节推进，不来自机械字数。
- 上架、推荐节点可卡爽点，但不能破坏连贯。

## 期待来源

- 一小段剧情完成后的新悬念。
- 卷级波动中的不断上行。
- 章节结尾自然抛出下一步。
- 爽点爆发和平台节点配合。

## 断档风险

- 每章硬套三幕式。
- 每章硬塞爽点，导致节奏假。
- 卷纲不随正文调整。
- 为卡章牺牲剧情连贯。

## 训练任务

把一个卷纲切成 10 个剧情小段。每段标注目标、阻碍、转折、收获、断章点。再决定每段拆几章，不反过来先定章节数。
