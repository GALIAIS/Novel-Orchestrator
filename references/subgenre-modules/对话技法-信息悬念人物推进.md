# 对话技法-信息悬念人物推进

依据案例：batch-266 案例 1328

## 读者承诺

让对话同时承担推进剧情、传递信息、制造悬念和暴露人物，而不是让角色轮流替作者念设定。

## 核心循环

```text
人物目标 -> 信息差 -> 试探/隐瞒 -> 冲突升级 -> 新信息/新悬念 -> 行动推进
```

## 开篇必备

- 每段对话至少有一方带着目标。
- 双方掌握的信息不能完全相同。
- 不该明说的信息要通过试探、回避、误解露出。
- 对话结束后局势必须变化。

## 期待来源

期待来自读者听出话里有话：角色说的是一层，隐瞒的是一层，下一步行动又由这段话触发。

## 断档风险

- 对话只负责解释背景。
- 双方没有目标，只是闲聊。
- 冲突全靠吵架，没有信息推进。
- 人物说话方式没有身份差异。

## 训练任务

写一段两人对话，限定 800 字，不许旁白解释背景。结束后检查读者是否能知道“关系、麻烦、隐瞒”三件事。
