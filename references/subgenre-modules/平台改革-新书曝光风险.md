# 平台改革-新书曝光风险

依据案例：`batch-082` 案例 408。

## 读者承诺

这个模块用于平台机制变化期的开书判断。它承诺让作者先看当前曝光、推荐和付费门槛，再决定是否开新书。

## 核心循环

```text
机制变化 -> 新书曝光检测 -> 首订预期下调 -> 成本评估 -> 开书/暂缓/换平台
```

## 开篇必备

- 记录平台最近的榜单改动。
- 看新书曝光，不看老书神话。
- 判断是否需要付费买曝光。
- 估算首订下限。
- 明确自己的日更能力。

## 期待来源

- 作者知道反馈是否仍然及时。
- 新书有没有自然曝光。
- 中下层是否还有机会。
- 高更新能否换来追订。
- 机制风险提前计入开书计划。

## 断档风险

- 按旧平台经验开新书。
- 只看头部精品和 IP 方向。
- 低估推荐机制改动。
- 新人被氪金曝光门槛卡住。
- 首订低后情绪崩盘。

## 训练任务

开书前连续 7 天记录目标平台新书曝光入口、更新榜、推荐位、首订反馈案例。写出一个“不低于最坏情况”的开书判断。
