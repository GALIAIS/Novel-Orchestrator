# 大神热评段落-自然融梗与反应点拆解

依据案例：`batch-207` 案例 1031。

## 读者承诺

评论多的大神段落值得拆反应点，但学习的是触发机制，不是硬塞原梗。

## 核心循环

```text
热评段落 -> 拆读者反应 -> 记录触发点 -> 自然融入 -> 原创替代
```

## 开篇必备

- 记录评论为什么集中。
- 区分梗、反差、递进和槽点。
- 只留机制，不抄原句。
- 判断自己剧情是否能自然承接。

## 期待来源

读者在熟悉反差或可吐槽节点上产生参与感。

## 断档风险

- 为融梗而融梗。
- 绞尽脑汁硬靠大神片段。
- 只摘句子，不懂反应机制。
- 梗和当前剧情不贴合。

## 训练任务

找 5 个热评段落，拆出读者反应原因，并各写一个原创替代场景。
