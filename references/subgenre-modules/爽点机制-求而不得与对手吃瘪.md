# 爽点机制-求而不得与对手吃瘪

依据案例：`batch-208` 案例 1038。

## 读者承诺

“求而不得”爽点要让主角无心所得与对手主动失败同场发生，形成鲜明收益差。

## 核心循环

```text
对手强求 -> 行动加码 -> 对手弄巧成拙 -> 主角无心得利 -> 对手吃瘪被点明
```

## 开篇必备

- 明确对手强烈想要什么。
- 明确主角并未刻意追求。
- 让主角行为刚好踩中收益点。
- 同场写出对手失败或损失。

## 期待来源

读者看到对手越努力越狼狈，主角越轻松越得利，情绪会集中在对比带来的爽感上。

## 断档风险

- 只写主角得利，不写对手吃瘪。
- 对手投入不够，失败不疼。
- 主角得利完全无因，像硬送。
- 吃瘪没有画面或结果清算。

## 训练任务

写 3 组桥段，每组包含对手所求、主角无心动作、主角获得、对手损失、当场对比句。
