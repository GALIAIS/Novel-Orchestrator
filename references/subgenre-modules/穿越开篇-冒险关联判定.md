# 穿越开篇-冒险关联判定

依据案例：batch-277 案例 1385

## 读者承诺

判断穿越开篇是否该写前史：凡是进入冒险前、与后续主线关联弱的内容都压缩；第一章应直接进入危机，用事件承载身份、金手指、职业线和后续人脉。

## 核心循环

```text
前史设定 -> 关联度判断 -> 无关压缩 -> 当前危机开场 -> 能力/身份显形 -> 后续主线打开
```

## 开篇必备

- 主角进入故事后的即时危机。
- 危机与后续主线高度相关。
- 身份、能力、职业线至少展示其一，最好多项合一。
- 前史最多作为几句话解释。

## 期待来源

读者期待主角现在怎么活、怎么破局、能力有什么用、下一步会进入什么更大的局。地点是否是监狱、山洞或异世界不重要，事件功能才重要。

## 断档风险

- 开篇先写追兵和前世纠葛，后续却不用。
- 主角先折腾、下行、再昏迷，阅读动能被清零。
- 山洞解密缺乏生死危机或不公平伤害。
- 把身份介绍当成剧情。
- 开局事件解决后不能打开职业线、人脉线或主线。

## 训练任务

把一个 2000 字前史开篇压缩为 500 字以内：前史只留 3 句，剩余篇幅写当前危机。危机必须让读者同时知道主角身份、能力和下一步目标。
