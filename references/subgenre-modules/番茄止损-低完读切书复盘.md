# 番茄止损-低完读切书复盘

依据案例：batch-292 案例 1458

## 读者承诺

番茄书完读极低且作者已经写不下去时，优先止损复盘，为下一本保留精力。

## 核心循环

```text
低完读 -> 判断严重度 -> 作者执行力耗尽 -> 切书 -> 复盘 -> 新书迭代
```

## 开篇必备

- 当前字数。
- 每日读者。
- 完读率。
- 收益。
- 作者能否继续写。
- 切书复盘点。
- 下一本改进项。

## 期待来源

作者把失败样本变成下一本的避雷清单，而不是被沉没成本拖住。

## 断档风险

- 只看还有读者，不看完读。
- 因不甘心继续硬撑。
- 切书但不复盘。
- 把少量稿费当继续写的充分理由。

## 训练任务

为一本低完读旧书写 5 个问题和 3 个新书改进动作。
