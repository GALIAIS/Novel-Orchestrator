# 同人新爽点-原著资产变脑洞铺垫

依据案例：`batch-341` 案例 1703

## 读者承诺

同人文可以把原著人物、故事和读者记忆转成新脑洞铺垫，用身份错位制造新爽点。

## 核心循环

```text
原著强资产 -> 新身份/新场域 -> 读者误判期待 -> 原著记忆加压 -> 新冲突兑现
```

## 开篇必备

- 保留原著识别度。
- 给角色换一个有反差的新身份。
- 新场域要产生独立问题。
- 原著能力服务新冲突。
- 第一章避免复刻原著剧情。

## 期待来源

读者期待熟悉强角色进入陌生问题后，被低估、被误判，再用原著级资产打出新爽感。

## 断档风险

- 只重播原著。
- 新身份没有反差。
- 原著人物 OOC。
- 新问题太弱。
- 熟悉感压过新鲜感。

## 训练任务

给一个原著强角色设计 3 个新身份，每个身份写一个误判场景和一次不复刻原著的破局。
