# 新人拒稿-开篇避雷自审

依据案例：`batch-114` 案例 566。

## 读者承诺

新人开篇先承诺清晰阅读入口：读者第一时间知道谁是主角、主角在干什么、当前故事是什么。不要用大神慢铺、设定百科和多人物大场面替代主角代入。

## 核心循环

```text
主角前置 -> 名字确认 -> 当前事件启动 -> 少设定少人物 -> 读者抓住代入对象 -> 再逐步扩展世界
```

## 开篇必备

- 第一屏出现主角。
- 主角名字尽早出现。
- 人物数量少到读者能分清焦点。
- 用事件带设定，不先讲设定。
- 默认使用第三人称或稳定第一人称。

## 期待来源

- 读者迅速确认阅读核心。
- 主角处境产生短期期待。
- 设定只在当下事件需要时出现。
- 人物少，主角存在感强。

## 断档风险

- 开头先写配角或历史，主角迟迟不出现。
- 主角没有名字，读者对不上人。
- 人物和设定同时大量涌入。
- 第二人称增加理解难度。
- 拿大神慢热当新人开篇依据。

## 训练任务

检查第一章前 1000 字：标出主角出现位置、名字出现位置、设定段数量、出场人物数量。把多余人物和设定后移，保留一个主角事件。
