# 卖点亮点-反馈驱动作业设计

依据案例：`batch-189` 案例 941。

## 读者承诺

卖点亮点教学不能只停留在讲解，要根据反馈设计针对性作业，把概念转成可操作训练。

## 核心循环

```text
讲解卖点亮点 -> 收集反馈 -> 定位卡点 -> 设计作业 -> 复盘改进
```

## 开篇必备

- 明确训练主题。
- 先让学员看懂概念。
- 收集真实不会操作的点。
- 作业只针对一个具体能力。

## 期待来源

学员从“听懂概念”进入“能拆、能写、能改”的实际训练。

## 断档风险

- 只讲不练。
- 还没看反馈就堆作业。
- 作业太泛，无法验证能力。
- 把社群热闹误当成掌握方法。

## 训练任务

围绕一本标杆书，分别写出分类标签、核心卖点、前三章亮点和可借鉴动作，再用这些内容改一版自己的简介。
