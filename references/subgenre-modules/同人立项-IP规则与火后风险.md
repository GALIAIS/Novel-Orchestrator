# 同人立项-IP规则与火后风险

依据案例：batch-370 案例 1847

## 读者承诺

同人项目先确认 IP、平台和编辑规则；大 IP 没火时没事不等于安全，真火后可能被封。

## 核心循环

```text
同人灵感 -> IP规则确认 -> 平台/编辑反馈 -> 风险评估 -> 再写开头
```

## 开篇必备

- IP 权属。
- 是否大 IP。
- 平台政策。
- 编辑确认方式。

## 期待来源

- 熟悉原作延伸。
- 低风险授权或默许。
- 开头使命服务原作情绪。
- 投入不会因规则归零。

## 断档风险

- 先写开头再查规则。
- 大 IP 踩雷。
- 以没封当安全。
- 火后版权和平台规则介入。

## 训练任务

写同人立项表：列 IP 权属、平台政策、编辑确认、火后风险，再决定是否继续。
