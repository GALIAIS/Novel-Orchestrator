# 事故背锅-敌方责任转移

依据案例：`batch-152` 案例 759。

## 读者承诺

事故可以发生，但不能让主角显蠢，也不能让队友长期像拖油瓶。更稳的责任来源是敌人强、环境险、信息差大或规则压迫。

## 核心循环

```text
事故发生 -> 敌方/环境承担主因 -> 主角保持判断 -> 队友不固定拖累 -> 补救反击
```

## 开篇必备

- 一个必须出事的剧情节点。
- 敌方强度或聪明度。
- 信息差或规则限制。
- 主角的正确判断。
- 队友的合理局限。
- 后续补救动作。

## 期待来源

读者期待看到强敌或局势制造麻烦，而不是己方犯蠢制造剧情。事故应抬高敌人和风险，而不是贬低主角和队友。

## 断档风险

- 主角反复背蠢锅。
- 队友固定背锅，变成功能性累赘。
- 事故没有补救和反击。
- 敌人不强，问题全靠己方降智。

## 训练任务

写 5 个事故来源：敌人、环境、信息差、规则、代价选择。每个都写主角正确判断、失败原因和后续反击。
