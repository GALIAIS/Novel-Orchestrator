# 公开处刑-先自审再求评

依据案例：`batch-116` 案例 578。

## 读者承诺

公开求评不是把基础问题丢给别人，而是先用开篇自审表过滤硬伤，再让别人帮你看认知盲区和阅读取向差异。

## 核心循环

```text
准备求评 -> 填开篇自审表 -> 修改硬伤 -> 明确求评问题 -> 收集反馈 -> 二次修改
```

## 开篇必备

- 先找开篇自我审核表。
- 按 18 要素做基础检查。
- 标出自己已改过的问题。
- 求评时说明希望看什么。
- 把反馈转成下一轮修改动作。

## 期待来源

- 规则提高社群学习效率。
- 自审减少重复基础反馈。
- 他人意见集中在盲区。
- 多种阅读取向补足作者滤镜。

## 断档风险

- 没自审就求评。
- 把所有反馈混成一团。
- 基础硬伤反复占用讨论。
- 作者滤镜导致只听顺耳意见。

## 训练任务

求评前写 300 字自审说明：已按哪个表检查、改了哪些硬伤、还想让别人重点看哪三个问题。
