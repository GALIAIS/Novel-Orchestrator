# 开头规则-金手指爽点与玄幻索引

依据案例：`batch-238` 案例 1190。

## 读者承诺

开头规则不能机械理解为“多少字必须出金手指/爽点”；新人应先回到基础索引，用功能判断安排开头元素。

## 核心循环

```text
害怕开头写错 -> 追问固定字数 -> 回到基础框架 -> 按功能安排金手指/爽点/铺垫
```

## 开篇必备

- 明确主角目标。
- 尽早给出期待和矛盾。
- 判断金手指出场功能。
- 控制背景铺垫篇幅。

## 期待来源

读者在前 3000 字内看见能力、冲突、目标和继续读的理由，而不是先被背景解释挡住。

## 断档风险

- 只问第几字出爽点。
- 背景铺垫无功能。
- 金手指出场晚且无期待。
- 传统玄幻忽略升级压力和世界压迫。

## 训练任务

标注开头前 3000 字里的金手指、第一次爽点、主角目标、背景信息；说不出功能的铺垫删除或后移。
