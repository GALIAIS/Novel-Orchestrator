# 作家之旅-低成本内容优先

依据案例：`batch-341` 案例 1704

## 读者承诺

写作工具书先看内容是否完整可用，低成本版本能降低门槛，但必须转成作品检查项。

## 核心循环

```text
获取可读版本 -> 检查内容完整 -> 阅读一个概念 -> 转成网文功能位 -> 应用到当前作品
```

## 开篇必备

- 区分纸质差和内容不可用。
- 每章只抽一个可用概念。
- 概念要改写成角色、选择、代价。
- 不用版本焦虑替代训练。
- 读完要能改一处稿。

## 期待来源

作者期待获得方法论，真正的回报是把“伙伴”“试炼”“门槛”等概念变成当前作品的角色功能和剧情压力。

## 断档风险

- 买书拍照即结束。
- 纠结价格和版本。
- 读完只记名词。
- 概念不落到章节。
- 不检查内容是否缺页错页。

## 训练任务

读一页写作书，把一个概念转成“第几章、谁出现、逼主角做什么选择、代价是什么”的一句检查项。
