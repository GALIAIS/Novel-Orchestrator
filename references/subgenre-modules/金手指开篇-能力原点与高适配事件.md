# 金手指开篇-能力原点与高适配事件

依据案例：`batch-409` 案例 2045

## 读者承诺

先把金手指写成输入、主动动作、输出、上限和成本，再寻找没有能力也会自然发生的高适配危机；用常规方案作基线、主角选择触发能力、结果后果接入主线。

## 核心循环

```text
定义初期能力契约 -> 选择独立成立的危机 -> 常规尝试证明问题约束 -> 赌注上升 -> 主角付费使用能力 -> 可视输出改变结果 -> 限制阻止万能化 -> 关系资源变化生成下一需求
```

## 开篇必备

- 能力的输入、动作、输出、上限和成本。
- 无金手指时也成立的事件动机。
- 主角原本欲望和常规方案。
- 不行动的赌注与主动使用选择。
- 结果前公平呈现的当前规则。
- 成功后改变的关系、资源或敌我状态。

## 期待来源

常规方法的局限先证明问题真实，新能力再以独特解法改写约束；成本和上限保留压力，输出后的状态变化把一次演示变成主线第一步。

## 断档风险

- 只有能力名称，没有可验证契约。
- 场景所有元素都为演示而摆靶。
- 系统自动完成高潮，主角没有选择。
- 结果后才补充决定胜负的规则。
- 最适配被写成无成本、无阻力。
- 解决当前问题后世界状态完全不变。

## 训练任务

先用五行定义一个原创金手指，再设计无能力也会发生的危机；写 1200 字开篇，主角先尝试常规方案，随后主动支付成本使用能力，结尾必须改变一项关系或资源状态。
