# 快乐分类-爽点菜单设计

依据案例：`batch-045` 案例 222。

## 读者承诺

爽点不只有打脸。财富、技艺、友谊、声誉、权力、期待、想象、交往、解除痛苦等，都能成为读者舒爽来源。

## 核心循环

```text
痛苦缺口 -> 快乐类型 -> 阻碍放大 -> 兑现画面 -> 新期待
```

## 开篇必备

- 先确定读者缺失。
- 选择对应快乐类型。
- 设计明确阻碍。
- 兑现后有可见变化。

## 期待来源

- 痛苦被解除。
- 欲望被满足。
- 旧误判被改写。
- 新快乐类型接上后续目标。

## 断档风险

- 只写结果，不写缺口。
- 快乐类型和题材不匹配。
- 爽点菜单机械套用。
- 没有见证者反应。

## 训练任务

选 5 种快乐类型，各写一个爽点场景：必须包含痛苦缺口、阻碍和兑现画面。
