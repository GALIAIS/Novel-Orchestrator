# 御兽文-高频数值反馈养成流

依据案例：`batch-215` 案例 1074。

## 读者承诺

御兽开篇即使剧情密度低，也能靠投喂、加点、升级、共享收益形成稳定追读。

## 核心循环

```text
投喂操作 -> 属性上涨 -> 面板可视化 -> 宠物升级 -> 主角共享收益
```

## 开篇必备

- 核心养成动作足够简单。
- 每次操作都有即时反馈。
- 宠物成长和主角成长绑定。
- 数值变化形成阶段升级期待。

## 期待来源

读者持续看到确定收益曲线，享受低负担养成、数值上涨和主宠同步变强。

## 断档风险

- 剧情弱但反馈也弱。
- 面板变化重复，没有升级节奏。
- 宠物强而主角没有同步收益。
- 投喂动作长期无变化导致疲劳。

## 训练任务

写一个 3000 字御兽开头大纲，包含 5 次投喂反馈、2 次面板变化、1 次宠物升级、1 次主角共享收益。
