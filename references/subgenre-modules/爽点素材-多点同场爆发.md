# 爽点素材-多点同场爆发

依据案例：`batch-078` 案例 387。

## 读者承诺

这个模块用于把爽点素材表转成具体场景。它承诺不用单个爽点硬撑，而是让多个期待在同一冲突现场同时爆发。

## 核心循环

```text
多个铺垫 -> 同场汇聚 -> 主角行动 -> 多条反馈 -> 整体爽感放大
```

## 开篇必备

- 至少三条前置期待。
- 多个相关角色同时到场。
- 一个动作能影响多条线。
- 每条爽点都有反馈对象。
- 爆发后保留下一轮期待。

## 期待来源

- 身份曝光。
- 关系修罗场。
- 利益冲突。
- 旁观者震惊。
- 一次行动解决多重误判。

## 断档风险

- 只堆爽点名词，不做铺垫。
- 多条线没有同场理由。
- 每个爽点单独爆，缺少合力。
- 爽点彼此抵消。
- 爆完没有后续。

## 训练任务

写一个多点爆发清单：三条铺垫、一个共同舞台、一个主角动作、三组反馈。再写 500 字试场景。
