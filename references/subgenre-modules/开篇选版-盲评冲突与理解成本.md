# 开篇选版-盲评冲突与理解成本

依据案例：`batch-394` 案例 1966

## 读者承诺

面对同一故事的多个开篇版本，用目标读者的实际接收结果选择骨架：先单测书名点击门，再比较谁能以最低理解成本讲清主角、困境、冲突与下一步期待。

## 核心循环

```text
包装单测 -> 多版等字数盲读 -> 脱稿复述 -> 记录术语与风格误读 -> 选清晰冲突骨架 -> 同向合并 -> 再次盲测
```

## 开篇必备

- 书名能给出题材或核心趣味信号。
- 各版本使用相同测试字数和同类读者。
- 主角、当前困境、失败损失可以脱稿复述。
- 第一冲突给出明确行动方向。
- 陌生术语只投放当前事件需要的部分。
- 风格信号不会让读者反复重置预期。

## 期待来源

读者先确认“这是什么体验”，再看懂“谁现在必须解决什么”，并在章末留下一个具体问题。可复述的期待比作者解释的后续计划更可靠。

## 断档风险

- 把三个附件中的相对最佳误当成投稿合格。
- 只问喜欢哪版，不检查读者究竟看懂什么。
- 用作者已知设定替读者补全关系。
- 将各版亮点全部拼回导致信息更拥挤。
- 书名阻断点击，却只修改正文。
- 改稿后不复测。

## 训练任务

写情绪、悬念、冲突三版各 1000 字，交给五名目标读者回答“谁、遇到什么、怎么解决、还等什么”；按无法复述项选骨架，只迁入服务同一事件的材料，再做第二轮盲测。
