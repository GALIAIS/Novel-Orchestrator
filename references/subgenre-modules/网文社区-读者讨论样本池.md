# 网文社区-读者讨论样本池

依据案例：`batch-089` 案例 442。

## 读者承诺

读者社区不是教程库，而是样本池。作者去那里看读者怎么夸、怎么骂、争什么，从而反推题材卖点和真实阅读口味。

## 核心循环

```text
找活跃社区 -> 看热帖/吐槽/推书 -> 记录读者反应 -> 反查样本 -> 提炼卖点反馈
```

## 开篇必备

- 只把社区当读者反馈来源。
- 优先看高回复、高争议、高推荐帖。
- 记录具体夸点和骂点。
- 回到原书前三章验证。

## 期待来源

- 找到真实读者关心点。
- 发现同题材样本。
- 看到争议如何形成。
- 用反馈修正自己的卖点判断。

## 断档风险

- 把社区闲聊当写作教程。
- 只看结论，不读样本。
- 被单个观点带偏。
- 忽略读者社区和目标平台差异。

## 训练任务

在读者社区找 10 个同题材讨论帖，记录夸点、骂点、争议点和样本书名，再读其中 3 本前三章。
