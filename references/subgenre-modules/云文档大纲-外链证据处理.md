# 云文档大纲-外链证据处理

依据案例：batch-288 案例 1439

## 读者承诺

只有外链大纲时，系统只记录证据状态和补查入口，不臆造正文分析。

## 核心循环

```text
发现外链 -> 记录文件名 -> 检查访问状态 -> 标注不可见内容 -> 保留补查入口
```

## 开篇必备

- 文件名。
- 外链地址。
- 资源类型。
- 访问检查结果。
- 已知信息。
- 未知信息。
- 后续补查条件。

## 期待来源

资料治理保持诚实：哪些内容读过，哪些只是入口，系统能一眼区分。

## 断档风险

- 把外链当成已读正文。
- 在不可见大纲上硬拆。
- 不记录访问状态。
- 把资料不可见误判成资料无价值。

## 训练任务

给一个不可访问外链写证据卡，列出已知、未知、禁止臆断、补查动作四栏。
