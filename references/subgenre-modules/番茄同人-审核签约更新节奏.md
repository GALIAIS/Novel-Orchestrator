# 番茄同人-审核签约更新节奏

依据案例：`batch-175` 案例 872。

## 读者承诺

同人文依托既有 IP 或角色认知提供延展满足，但作者侧必须先处理平台审核、签约和更新节奏，避免流程焦虑打断创作。

## 核心循环

```text
同人开书 -> 安全审核 -> 等签约结果 -> 继续存稿/更新 -> 字数节点后重视稳定更新
```

## 开篇必备

- 确认平台是否允许对应同人类型。
- 查清安全审核和签约流程。
- 早期不要把审核等待误判成作品失败。
- 准备存稿缓冲，避免流程波动影响节奏。

## 期待来源

作者期待流程清晰、更新不白费；读者期待熟悉 IP 下的新事件、新关系和新爽点。平台流程必须服务持续供给，而不是消耗作者判断。

## 断档风险

- 未查规则就写敏感同人。
- 审核等待期焦虑停摆。
- 把 3 万字早期问题当成长期运营问题。
- 进入长字数阶段后仍没有稳定更新计划。

## 训练任务

给一个同人项目做投稿流程表：平台、IP 风险、安全审核、签约节点、断更风险节点、存稿天数和遇到审核延迟时的处理方案。
