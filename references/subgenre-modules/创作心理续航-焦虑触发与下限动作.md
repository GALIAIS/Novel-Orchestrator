# 创作心理续航-焦虑触发与下限动作

依据案例：`batch-418` 案例 2086

## 读者承诺

创作者不需要等焦虑消失才工作，也不会让一张高收入截图、一次拒绝或一次重读厌恶直接触发删稿、弃书和透支加码。系统把情绪、项目证据与人格判断分开：先识别触发，把全盘否定改写成一个可观察症状，暂停不可逆决定，完成当天最低行动，再在固定反馈窗口按个人基线决定恢复、转向或停止。

## 核心循环

```text
比较、数据或稿件缺陷触发焦虑 -> 记录触发源和反应 -> 将“我不行”改写为一个具体症状 -> 启动冷却期并暂停删稿、弃书和透支等不可逆动作 -> 隔离高刺激比较源 -> 完成最低骨架、字数或单点修复 -> 在固定窗口核对完成、技能、反馈、商业和成本基线 -> 只验证一个可控变量 -> 按预设恢复、转向或停止门槛决策 -> 归档本次处置证据
```

冷却期保护判断，下限动作保护连续性，症状级诊断恢复可行动性，多轨基线避免一项波动吞掉全部评价，预设门槛同时阻止冲动放弃和无限硬撑。

## 开篇必备

- 最常见的比较、数据、拒绝或重读触发源。
- 焦虑出现时的身体反应和重复思维。
- 容易形成的全盘身份判决。
- 将判决改写成具体症状的格式。
- 删稿、停更、借贷、辞职和通宵等不可逆动作清单。
- 明确的冷却时长和数据隔离方式。
- 三十分钟内可完成的最低行动。
- 完成、技能、反馈、商业和生活成本五轨基线。
- 固定反馈对象与检查频率。
- 恢复、转向、停止和健康财务红线。

## 期待来源

- 全盘否定能否被压缩为一个可处理问题。
- 冷却期能否挡住最坏的即时决定。
- 最低行动能否恢复当天控制感。
- 单点复写是否改变目标读者反应。
- 当前波动与个人长期基线是否一致。
- 项目会达到恢复、转向还是停止门槛。
- 下一次触发能否直接复用本次处置记录。

## 断档风险

- 把稿费截图当作行业完整分布。
- 用一次数据波动判断全部能力。
- 将局部缺陷升级为人格和天赋判决。
- 焦虑时立即删稿、停更或作财务决定。
- 全天刷新数据并反复寻找极端比较样本。
- 把最低行动误当作方向已经正确。
- 只讲心态，不定位任何技术症状。
- 用“坚持”越过睡眠、健康、金钱和债务红线。
- 停止一个项目时否定全部创作身份。

## 训练任务

制作一张真实触发卡：写出最近一次想删稿或弃书的触发源、全盘判决、可观察症状、24 小时禁做动作和 30 分钟最低行动。冷却后只复写一个相关场景并获取两名目标读者反馈，最后按事先定义的恢复、转向和停止条件处理，禁止使用他人收入截图替代个人基线。
