# 读者控制欲-评论越界与作者边界

依据案例：batch-346 案例 1726

## 读者承诺

作者能接收读者反馈，但不会把叙事主导权交给威胁式评论。该模块服务连载心态、评论区管理、粉丝群管理和长线剧情稳定。

## 核心循环

```text
读者追更 -> 形成个人期待 -> 评论表达不满 -> 作者识别有效卡点/控制欲命令 -> 采纳问题而非交出方向
```

## 开篇必备

- 明确作品主承诺和不可随意改写的主线。
- 给评论反馈建立分类：卡点、偏好、控制欲。
- 对威胁式、命令式、接管式评论保留删除边界。
- 记录读者情绪，但单独验证其是否代表多数阅读问题。

## 期待来源

读者相信作者有方向，也相信真实阅读问题会被听见。

## 断档风险

- 被少数高噪音读者牵着改主线。
- 把所有批评都当控制欲，错过有效问题。
- 粉丝群过度亲密后，读者把建议升级成指挥。
- CP、人物命运、爽点兑现被临时评论反复摇摆。

## 训练任务

整理最近 20 条评论，标出“读者情绪”“可验证问题”“读者要求改法”“是否采纳”。最后只保留可验证问题进入修稿清单。
