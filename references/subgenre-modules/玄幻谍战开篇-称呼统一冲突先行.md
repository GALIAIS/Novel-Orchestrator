# 玄幻谍战开篇-称呼统一冲突先行

依据案例：`batch-086` 案例 426。

## 读者承诺

读者要看的不是一堆陌生设定，而是一个身份稳定的主角正在执行高风险任务，并且他的能力、计划和代价会立刻推动局面。

## 核心循环

```text
稳定主角称呼 -> 明确任务目标 -> 设置倒计时/失败代价 -> 行动中展示能力 -> 留下谍战反应链
```

## 开篇必备

- 一个读者能记住的主角称呼。
- 一个具体任务：刺杀、潜入、窃取、护送、伪装。
- 一个明确阻碍：守卫、禁制、身份核验、内鬼、时间限制。
- 最多 3 个专有设定词，且必须服务行动。
- 金手指或特殊能力要通过当场解决问题展现。

## 期待来源

- 刺杀计划能否成功。
- 伪装身份会不会暴露。
- 主角能力有什么限制和升级空间。
- 任务背后谁在布局。
- 完成任务后会引出怎样的追杀或反制。

## 断档风险

- 主角称呼混乱，让读者误以为多人出场。
- 开篇先讲远古背景，任务压力迟迟不出现。
- 设定名词过密，读者没有画面。
- 场景切换太快，缺少目标和因果。
- 作者想制造身份谜团，但读者连代入对象都找不到。

## 训练任务

写一个玄幻谍战第一章前 1200 字。前 200 字内固定主角称呼和任务目标，前 600 字内出现倒计时或失败代价，全文专有设定词不超过 3 个。
