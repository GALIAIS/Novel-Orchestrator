# 读者群运营-建群时机与管理边界

依据案例：`batch-317` 案例 1585

## 读者承诺

读者群不是越热闹越好，而是要在平台规则允许、作者精力可控的前提下，服务粉丝沉淀、反馈测试、灵感收集或冷启动。

## 核心循环

```text
读者要求建群 -> 确认平台规则 -> 设定准入和功能 -> 控制作者在线边界 -> 收集反馈/灵感/冷启动
```

## 开篇必备

- 先确认平台是否允许放联系方式。
- 明确群用途，不做无边界陪聊。
- 设定准入门槛，如全订、自订或特定功能群。
- 作者可以低频出现，避免被群聊吞掉写作时间。
- 对有效灵感公开感谢，鼓励正向贡献。

## 期待来源

读者期待和作者产生连接，作者期待把这份连接转成可控反馈、毒点测试、灵感池和下本书的初始读者。

## 断档风险

- 普通闲聊群浪费大量精力。
- 少数读者意见绑架剧情。
- 平台规则不允许联系方式，引发风险。
- 只有热闹没有功能，粉丝资产无法沉淀。

## 训练任务

写一页读者群规则：入群门槛、群用途、作者在线时间、不接受的建议类型、可测试内容、灵感采纳感谢方式。
