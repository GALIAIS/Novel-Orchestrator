# 新人平台选择-风格适配验证

依据案例：`batch-086` 案例 430。

## 读者承诺

新人选平台不靠传闻下注，而是用扫榜、过稿难度、反馈速度、读者口味和自身题材风格做验证，先选能最快暴露问题、最低成本试错的平台。

## 核心循环

```text
听到平台传闻 -> 搜多方反馈 -> 扫同题材榜单 -> 判断风格适配 -> 小成本开书验证 -> 按数据复盘
```

## 开篇必备

- 明确自己的题材和文风。
- 至少对比两个平台同题材样本。
- 记录过稿难度、推荐反馈速度和读者口味。
- 不用“平台强弱”替代“我能不能写这个平台”。
- 第一本先以验证为目标，不把全部预期押在单站。

## 期待来源

- 哪个平台更快给反馈。
- 自己的题材在哪边样本更多。
- 哪边读者更接受自己的节奏和卖点。
- 试错后能否迅速复盘下一本。

## 断档风险

- 被“谁完爆谁”这类口号带节奏。
- 只看流量，不看过稿和推荐机制。
- 鄙视某平台读者，导致风格适配失败。
- 忽略收益规则变化。
- 不扫榜就凭印象投稿。

## 训练任务

起点和番茄各找 5 本同题材新书，记录标题、简介、前三章卖点、读者反馈和更新节奏。写 300 字平台选择判断，并给出第一本试错策略。
