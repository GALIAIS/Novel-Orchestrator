# 毒点复盘-案例索引避雷

依据案例：batch-284 案例 1420

## 读者承诺

毒点解析要沉淀成可复习的避雷索引，让作者在写作前能对照“不该出现、无脑人设、基调不符”等案例检查风险。

## 核心循环

```text
毒点案例 -> 原因解析 -> 索引复习 -> 桥段自检 -> 删除或改写
```

## 开篇必备

- 常见毒点分类。
- 对应案例入口。
- 写作前自检问题。
- 保留或删除判断。
- 改写方案。

## 期待来源

读者未必奖励作者避开毒点，但会因为毒点直接弃读；索引的价值是减少失误。

## 断档风险

- 看完解析不复盘。
- 只记结论，不转成检查项。
- 桥段基调和作品承诺冲突。
- 人设行为显得无脑却未修正。

## 训练任务

把前三章每个争议桥段放进毒点索引检查，写出保留理由、删除方案或改写方案。
