# 金手指事件-能力服务故事危机

依据案例：batch-259 案例 1295

## 读者承诺

金手指不是展示能力说明书，而是解决故事里的真实危机、争夺真实收益、引出更强敌人。速度、力量、隐身都必须落在生死、资源、追杀、秘境、反制和升级链上。

## 核心循环

```text
故事目标 -> 危机卡住主角 -> 能力解决一部分问题 -> 敌人或环境反制 -> 能力升级/组合后推进主线
```

## 开篇必备

- 当前故事目标。
- 能力的具体效果和限制。
- 能力要解决的危机。
- 使用能力后获得的收益。
- 使用能力引出的新敌人或新问题。

## 期待来源

读者期待能力每次都有新用法：速度从逃命到抢宝，力量从破防到破阵，隐身从躲避到火中取栗。能力价值要被反复验证。

## 断档风险

先列能力再硬找比赛舞台，事件会空；能力无代价碾压，读者会觉得作弊；能力和主线无关，就算场面热闹也不会推动长篇。

## 训练任务

用一个能力串三场戏：小胜、被针对、组合反打。每场必须写清楚危机、限制、收益和新钩子。
