# 召唤开头-束缚危机与等待逻辑

依据案例：batch-362 案例 1807

## 读者承诺

召唤系统开头可以有延迟和救援，但主角不能只被绑着等死。必须给反派不杀理由，并让主角主动争取时间。

## 核心循环

```text
危机 -> 召唤延迟 -> 主角主动拖延 -> 倒计时 -> 救援兑现
```

## 开篇必备

- 主角有行动空间。
- 反派不立刻杀人的理由明确。
- 延迟不是纯等。
- 召唤对象有卖点。

## 期待来源

- 倒计时压迫。
- 主角智斗拖延。
- 残损系统带来的不确定。
- 战损强者或知名武将登场。

## 断档风险

- 主角被完全束缚。
- 反派降智废话。
- 救援还在赶路，主角只能等。
- “就这一次”也丢读者。

## 训练任务

写 800 字等救援开头：主角必须有主动拖延动作、倒计时、反派不杀理由。
