# 卖点中心-打破中庸思维

依据案例：`batch-019` 案例 095。

## 读者承诺

解决“写什么都不到点上”的问题。让每个事件围绕本书卖点生产爽点，再用矛盾和对比放大亮点。

## 核心循环

```text
定卖点 -> 围绕卖点设事件 -> 用卖点造爽点 -> 扫榜记亮点 -> 加对比
```

## 开篇必备

- 一句话卖点。
- 卖点在第一单元出现。
- 主角通过卖点获得收益。
- 反派或环境对卖点产生误判。

## 期待来源

- 系统文看系统如何推进剧情。
- 御兽文看宠兽收服和对战。
- 升级文看强弱对比和资源跃迁。
- 读者点进来想看的东西持续出现。

## 断档风险

- 开篇卖点不明确。
- 后期换成别的题材，背叅读者入口期待。
- 只堆灵感，不围绕卖点组织。
- 没有矛盾和对比，亮点变平。

## 训练任务

每天扫榜 20 分钟，记录一个亮点。把亮点改写进自己的题材时，必须写清它服务哪个卖点，以及制造了哪组对比。
