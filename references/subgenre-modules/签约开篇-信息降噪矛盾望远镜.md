# 签约开篇-信息降噪矛盾望远镜

依据案例：`batch-087` 案例 435。

## 读者承诺

签约开篇先让读者和编辑看懂：主角遇到什么问题、矛盾为什么够重、金手指如何解决，并且当前解决会望见下一层更大的爽点。

## 核心循环

```text
少陌生信息 -> 问题直白 -> 矛盾核心对准 -> 金手指匹配 -> 解决后引出更大期待
```

## 开篇必备

- 陌生名词尽量少，必须出现就立刻给功能。
- 一句话能讲清主角当前问题。
- 矛盾核心不能被动作戏带偏。
- 金手指要匹配矛盾，而不是随便强。
- 当前事件后要能望见下一层人物、资源或危机。

## 期待来源

- 主角如何解决当前问题。
- 金手指能带来什么更大连锁反应。
- 反派背后的人会不会出场。
- 收获是否能升级主角处境。

## 断档风险

- 开局信息跳级，读者看不懂名词。
- 矛盾烈度低，只靠打斗热闹。
- 金手指和主线矛盾不匹配。
- 解决当前问题后没有下一层期待。

## 训练任务

拿自己的第一章做三检：删陌生名词、重写一句话矛盾、把金手指和矛盾重新配对。最后写出解决后引出的下一层人物或收益。
