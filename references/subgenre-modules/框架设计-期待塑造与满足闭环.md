# 框架设计-期待塑造与满足闭环

依据案例：batch-380 案例 1896

## 读者承诺

框架模块不是填空术语，而是期待管理工具。人设、激励事件、至暗时刻都要塑造期待并在后续满足或反转。

## 核心循环

```text
模块 -> 期待 -> 延迟 -> 满足/反转 -> 新期待
```

## 开篇必备

- 人物行动期待。
- 激励事件带来的回应期待。
- 当前阶段目标。
- 一个可见兑现点。

## 期待来源

- 读者预判人物会怎么做。
- 事件打破稳定状态。
- 挫败抬高反弹期待。
- 阶段胜利开启下一轮目标。

## 断档风险

- 只填模块名，不制造期待。
- 过早把“本质”理解成删框架。
- 只有塑造，没有满足。
- 满足后不开新期待。

## 训练任务

把 10 章小纲拆成五个模块，每个模块写两行：塑造什么期待，在哪里满足。
