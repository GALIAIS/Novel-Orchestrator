# 先知模拟-结果前置望远镜

依据案例：batch-259 案例 1292

## 读者承诺

先把未来结果、死亡危机或可见收益摆出来，让主角和读者同步知道“为什么必须行动”。读者追看的不是单纯预言，而是主角如何改变结果，以及改变后会产生什么新落差。

## 核心循环

```text
预见结果 -> 主角尝试改变 -> 过程倒推原因 -> 结果偏移 -> 新预见继续拉期待
```

## 开篇必备

- 一个明确、足够近的结果。
- 结果里包含死亡、损失、收益或秘密。
- 主角能尝试的多个行动选项。
- 未来不是固定命运的证据。
- 改变结果后的反噬或新线索。

## 期待来源

期待来自“这个结果为什么会发生”和“主角能不能改变它”。每次模拟、剧本或预言都要给读者一个更具体的目标。

## 断档风险

未来若 100% 按预告发生，会失去改变快感；若预见结果过于模糊，主角行动就仍然缺乏理由；若每次模拟只换死法不加线索，会显得水。

## 训练任务

设计一章三次模拟：第一次给死亡结果，第二次给延迟死亡方法，第三次给可疑生路线索。结尾用线索引出真正敌人。
