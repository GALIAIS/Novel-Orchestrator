# 期待感-信息差错位验证

依据案例：`batch-075` 案例 373。

## 读者承诺

这个模块用于设计开篇期待感。它承诺不把期待感局限为强穿弱，而是用真实状态、表面状态、配角误判和读者验证欲，制造持续追读。

## 核心循环

```text
真实状态 -> 表面状态 -> 配角误判 -> 读者猜测 -> 揭露验证 -> 新错位
```

## 开篇必备

- 主角真实能力或潜力明确。
- 表面状态和真实状态有差异。
- 配角基于表面状态做出误判。
- 读者提前掌握一点关键信息。
- 揭露场景能带来震惊或反差爽点。

## 期待来源

- 实力和身份错位。
- 实力和潜力错位。
- 弱主角配强系统。
- 世界观规则和主角特质相冲。
- 读者等待猜测被验证。

## 断档风险

- 只有强弱标签，没有信息差。
- 读者不知道该期待什么。
- 揭露太晚，误判变成拖沓。
- 反差特征有毒，破坏代入。
- 每轮揭露后没有新错位。

## 训练任务

设计三轮信息差：每轮写真实状态、表面状态、谁误判、读者等什么、揭露后谁震惊。三轮之间必须逐步升级。
