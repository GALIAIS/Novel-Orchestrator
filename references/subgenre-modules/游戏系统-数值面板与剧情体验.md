# 游戏系统-数值面板与剧情体验

依据案例：batch-365 案例 1823

## 读者承诺

游戏/系统文的数值面板要承担成长反馈和感官刺激，但不能用大段属性替代剧情；越偏剧情体验，面板越要轻量。

## 核心循环

```text
任务/等级 -> 成长反馈 -> 短促数值刺激 -> 剧情选择 -> 奖励兑现
```

## 开篇必备

- 明确系统类型。
- 少量关键字段。
- 一次可感知成长。
- 面板展示后立刻回到事件。

## 期待来源

- 等级提升。
- 奖励到账。
- 资源差距被量化。
- 系统功能影响主角选择。

## 断档风险

- 大篇幅水属性。
- 字段太多但没有剧情功能。
- 只展示面板不推动选择。
- 把游戏模拟写得比剧情更重却没有对应受众。

## 训练任务

设计一个只含 5 个字段的系统面板，并标注每个字段对应的剧情功能；删掉不能制造选择或成长反馈的字段。
