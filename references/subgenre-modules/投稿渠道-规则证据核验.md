# 投稿渠道-规则证据核验

依据案例：`batch-056` 案例 276。

## 读者承诺

这个模块处理“某题材能不能投某投稿口”的渠道问题。它不提供凭印象得出的收稿判断，只提供投稿前核验动作：先确认当期规则、编辑口径和材料要求，再决定题材包装。

## 核心循环

```text
准备投稿 -> 渠道不确定 -> 查官方/编辑口径 -> 匹配题材包装 -> 补齐材料 -> 投递并记录反馈
```

## 开篇必备

- 先区分这是文本问题还是渠道问题。
- 明确目标平台和目标投稿口。
- 查当期公告、编辑要求或可靠回复。
- 准备书名、简介、题材标签、前三章和大纲。
- 不确定项必须标“待确认”，不能写成结论。

## 期待来源

- 作者知道稿子应该投向哪里。
- 题材包装和投稿口径一致。
- 编辑能快速判断这是不是自己收的方向。
- 投递材料完整，减少无效沟通。
- 反馈可以回流到下一轮包装调整。

## 断档风险

- 拿过期规则当当前规则。
- 只问“能不能投”，不提供书名简介和题材定位。
- 把渠道问题误判成作品质量问题。
- 一个脑洞只有一种包装，错过更合适投稿口。
- 没有保存规则来源，后续无法复盘。

## 训练任务

给一个准备投稿的项目做投稿核验卡：列出 3 个可能投稿口，每个投稿口写明证据来源、可投理由、风险项和投递前必须补齐的材料。
