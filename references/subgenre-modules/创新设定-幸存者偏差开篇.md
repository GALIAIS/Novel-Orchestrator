# 创新设定-幸存者偏差开篇

依据案例：`batch-101` 案例 501。

## 读者承诺

创新设定只有在开篇能被读者理解、被主角处境承载、被冲突验证时才有价值。不要因为榜单上有创新爆款，就误以为“新”本身能带来留存。

## 核心循环

```text
新设定 -> 熟悉锚点 -> 主角困境 -> 规则造成麻烦 -> 冲突验证规则 -> 留下更深问题
```

## 开篇必备

- 一个读者熟悉的现实或类型锚点。
- 一个必须立刻理解的新规则。
- 一个被新规则影响的主角处境。
- 一个能验证设定的事件，而非纯说明。

## 期待来源

- 新鲜感。
- 规则如何影响主角命运。
- 主角如何利用或反抗新规则。
- 设定背后的更大世界。

## 断档风险

- 只有设定名词，没有人物和事件。
- 把成功创新文当普遍规律。
- 没有笔力和开篇承载力却硬堆复杂设定。
- 读者看懂之前就被信息量劝退。

## 训练任务

写一条创新设定，再写一个让主角立刻吃亏或占便宜的场景。若新设定不能改变主角行动，就删掉或换成更贴近冲突的设定。
