# 资料包索引-版本更新与证据边界

依据案例：batch-259 案例 1293

## 读者承诺

写作学习资料包能提供安全感和检索入口，但系统必须区分目录可见、正文可读、附件可得、配套视频缺失。不能因为看见压缩包名或截图目录，就默认已经吸收完整内容。

## 核心循环

```text
零散资料 -> 版本化打包 -> 目录截图确认范围 -> 可访问内容入库 -> 不可访问内容列为待补证据
```

## 开篇必备

- 资料包名称和发布日期。
- 可见目录或更新清单。
- 文件、图片、网盘、附件的访问状态。
- 配套视频或外部材料说明。
- 需要后续补读的内容。

## 期待来源

学习者期待通过目录快速定位短板：开头、金手指、毒点、故事策略、情绪拉扯、新人问题、反套路创新等主题都能回到对应资料。

## 断档风险

链接过期、附件 403、压缩包无法打开、视频片段缺失，都会造成证据不完整。若不标注边界，后续系统会把未读内容误当成已吸收内容。

## 训练任务

为一个资料文件夹建立证据表：每份材料标注“已读正文、仅见目录、附件不可得、需视频配合、待复查”，并按主题排序。
