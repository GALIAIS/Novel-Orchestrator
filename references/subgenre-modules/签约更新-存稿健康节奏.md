# 签约更新-存稿健康节奏

依据案例：`batch-049` 案例 245。

## 读者承诺

签约后把稳定更新当成生产系统：用存稿、大纲细纲、发稿节奏和身体管理抵抗状态波动，而不是只靠临时激情。

## 核心循环

```text
签约 -> 日更压力 -> 建立存稿 -> 细纲降启动成本 -> 状态好时补仓 -> 状态差时消耗缓冲
```

## 开篇必备

- 明确目标日更字数。
- 设置最低存稿线和安全存稿线。
- 推荐给量前写一部分发一部分。
- 状态好时多写，平时写四存一。
- 大纲和细纲提前准备，减少卡文。

## 期待来源

- 存稿带来安全感。
- 细纲让低状态日也能推进。
- 规律更新保护全勤和推荐节奏。
- 身体稳定，长期产能才稳定。

## 断档风险

- 零存稿直接发书，身体波动会放大成断更风险。
- 只靠读者鼓励，无法覆盖病痛和疲劳。
- 长期透支身体，会损失更多写作时间。
- 没有细纲，状态差时更容易停摆。

## 训练任务

做一个 14 天发稿表：每天写、发、存三栏，目标是在第 14 天达到至少一周更新量的安全存稿。
