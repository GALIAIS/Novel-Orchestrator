# 毒点转亮点-强人设边界

依据案例：`batch-061` 案例 304。

## 读者承诺

这个模块用于处理 NTR、夺人资源、反道德主角等争议元素。它承诺的是只有当争议元素和强人设一致时，毒点才可能转成亮点；越过价值边界仍会变剧毒。

## 核心循环

```text
强公共人设 -> 争议元素匹配 -> 读者预期兑现 -> 爽点成立 -> 价值边界检查
```

## 开篇必备

- 主角人设必须先被读者识别。
- 争议行为要符合人设。
- 读者要提前被筛选。
- 行为结果服务主线。
- 手段不能越过目标读者价值底线。

## 期待来源

- 读者期待曹操式夺人夺势。
- 毒点变成人设兑现。
- 收小弟、夺资源、夺关系都能形成爽点。
- 稳定人设带来追读。
- 被夺者反应放大情绪。

## 断档风险

- 人设不支持争议行为。
- 纯爱读者被错误吸引进来。
- 行为手段猥琐失格。
- 社会事件改变读者接受度。
- 人物弧光写成核心人设崩坏。

## 训练任务

选一个争议元素，写出“常规为什么毒、什么人设能承接、读者期待什么、绝不能怎么写”四行。
