# 大纲模板-检索填充启动

依据案例：`batch-035` 案例 171。

## 读者承诺

大纲模板不是答案，而是启动器。先确认当前需要书纲、卷纲、单元纲还是章纲，再检索已有模板并填入具体项目。

## 核心循环

```text
确认大纲层级 -> 检索模板 -> 填当前项目 -> 暴露缺口 -> 回到训练
```

## 开篇必备

- 明确要写哪一层大纲。
- 选匹配题材和阶段的模板。
- 先填主角目标、阻碍、转折和兑现。
- 不用宏大设定代替下一步剧情。

## 期待来源

- 作者知道接下来写什么。
- 模板让空白变成问题清单。
- 单元闭环先跑通，再扩到全书。
- 检索和填充形成知识导航。

## 断档风险

- 只找模板不填项目。
- 书纲、卷纲、章纲混用。
- 用世界观填满大纲，事件仍然没有。
- 每次都重新问，知识库不沉淀。

## 训练任务

只用一个模板写 3000 字单元纲：主角目标、阻碍、转折、兑现、新期待五栏必须填满。
