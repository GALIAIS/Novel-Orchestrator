# 刺猬猫投稿-入库签约更新节奏

依据案例：`batch-057` 案例 285。

## 读者承诺

这个模块用于刺猬猫新人投稿流程判断。它承诺的是区分入库、审核上线、签约申请、站短、签约编辑和责编，避免新人把“已入库”误解成“已签约”。

## 核心循环

```text
投稿 -> 入库/上线 -> 稳定更新 -> 达到申请字数 -> 电脑端申请签约 -> 站短/合同 -> 责编对接
```

## 开篇必备

- 先确认当前状态是入库、上线、签约中还是已签约。
- 记录当前字数、收藏、更新时间。
- 保持更新频率，不要一口气发完后断更。
- 到 3 万字或 4-5 万字附近关注签约按钮。
- 用电脑端申请签约，别反复盲加编辑。

## 期待来源

- 新人知道下一步该做什么。
- 稳定更新带来最近更新曝光。
- 签约申请有明确节点。
- 过审后站短和签约编辑处理合同。
- 流程清晰后，焦虑从“编辑不理我”转成“继续更新和申请”。

## 断档风险

- 入库后以为编辑一定会加 QQ。
- 未签约就反复加责编。
- 2 万字一次性发完后停更。
- 不看电脑端签约入口。
- 直接照搬老作者快速上架策略。

## 训练任务

给已入库作品做 7 天更新计划：每天发几章、每章多少字、何时查看签约按钮、何时申请签约、何时等待站短。计划里必须写明“不重复骚扰编辑”的截止动作。
