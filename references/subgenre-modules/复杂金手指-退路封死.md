# 复杂金手指-退路封死

依据案例：`batch-026` 案例 128。

## 读者承诺

复杂金手指第一次出场，要解决主角天赋、经验和常规手段都解决不了的问题。

## 核心循环

```text
目标 -> 逻辑困难 -> 常规能力失效 -> 退路封死 -> 主动用外挂 -> 解决并埋主线
```

## 开篇必备

- 主角已有能力边界。
- 第一冲突限制。
- 常规办法为什么不够。
- 金手指不可替代点。

## 期待来源

- 主角主动选择强化。
- 金手指不是替代主角，而是补足极限。
- 复合型金手指有总轮廓。
- 后续每卷都能继续发挥。

## 断档风险

- 天赋经验已经能解决，却硬用金手指。
- 第一块碎片看起来就是全部金手指。
- 冲突天降，和主角目的无关。
- 信息跳级，读者跟不上。

## 训练任务

写一个金手指使用前置表：列出不用金手指的三种办法，再逐一封死，最后让主角主动使用金手指。
