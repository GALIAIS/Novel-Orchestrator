## 766：强调一下咱们进星球以后一定先看...

人工查看范围：正文。无评论、无附件、无图片。

### 读者承诺

这篇不是具体小说技巧，而是在规定社群学习契约：进星球先看置顶、先懂发帖规则、先搜索旧帖，再提交问题。公开处刑的限制不是为了卡人，而是为了让作者在自审过程中掌握技巧和知识。

### 核心机制

```text
读置顶规则 -> 搜索旧帖 -> 自己先自审 -> 再发帖讨论 -> 碰撞思路 -> 方法内化为自己的判断
```

规则被设计成训练器。它把“等别人帮我看”改造成“我先做判断，再让别人校准方向”。讨论的价值不是让别人替作者琢磨作品，而是用外部反馈优化自己的思考路径。

### 期待链与情绪

```text
新人进入社群 -> 容易重复提问/直接求评 -> 规则要求先查先审 -> 问题更具体 -> 反馈更有效 -> 作者真正学到方法
```

情绪上是压实责任。帖子明确把写书的主要责任放回作者身上：写书大部分靠自己，社群只能提供思想碰撞。

### 关键判断

- 置顶帖和发帖规则是学习入口，不是形式要求。
- 搜索旧帖能减少重复消耗，也能先获得已有答案。
- 公开处刑前的自审，本身就是训练。
- 学的是方法，不是让别人代替你判断作品。
- 讨论的结果必须回到作者自己的掌握和执行。

### 可迁移写法

求评前自审表：

```text
我已读过的置顶/规则：
我搜索过的关键词：
旧帖里可参考的结论：
我自己判断的问题：
我已经尝试的改法：
仍然卡住的具体点：
希望别人讨论的范围：
反馈后我能沉淀的方法：
```

### 训练任务

下次发开篇或大纲求评前，先完成一张自审表。必须写出自己已经搜索过什么、已经判断出什么、仍然不确定什么，禁止只发“帮我看看行不行”。

### 读者承诺

作者提交小纲和表格，觉得表格有用但内容多而杂，核心困惑是：赌注和收益到底是什么、该怎么表现；“不战斗就会死”能不能算赌注；消灭敌人获得自由能不能算收益；开篇能不能出现高层博弈。

### 核心机制

```text
抽象设定过载 -> 转成主角不得不做的选择 -> 选错失去筹码 = 赌注 -> 选对获得新筹码 = 收益 -> 高层博弈后置为悬念
```

评论把赌注和收益还原成赌博类比：只有主角必须做选择时，赌注才成立。选错失去钱、宝物、权力、生命等筹码；选对获得新筹码。赌注大小可以通过紧迫性和重要性表现。

### 期待链与情绪

```text
大纲写得多且杂 -> 作者担心漏要点 -> 评论要求先写几章验证 -> 明确二元选择 -> 赌注/收益变具体 -> 开篇只露高层博弈痕迹
```

情绪上从兴奋混乱转为可执行纠偏。开篇不宜直接铺复杂高层博弈，因为博弈写简单不好看，写复杂又占篇幅；更合适的是先留蛛丝马迹，让主角带读者逐步探索棋局。

### 关键判断

- “不战斗就会死”可以是赌注，但必须绑定主角不得不做的选择。
- “消灭敌人、获得自由”可以是收益，但要具体成选对后新增的筹码。
- 大纲太干涩时，先写几章正文验证可读性。
- 开篇不适合直接展示复杂高层博弈。
- 现代、将军、枪械、盗墓、复辟帝制等元素叠加时，红线和影射风险会上升。
- 新人优先写低风险故事，比靠换说法规避敏感更稳。

### 可迁移写法

赌注收益选择表：

```text
主角当前处境：
不得不做的二元选择：
选项 A：
选项 B：
选错会失去的筹码：
选对会获得的新筹码：
紧迫性：
重要性：
高层信息是否只作为伏笔：
敏感元素叠加风险：
```

### 训练任务

选一个开篇，把主角遇到的第一个事件改写成明确选择题。写清选错损失、选对收益，再把所有高层博弈信息压缩成 3 个伏笔，不在开篇完整解释。

## 769：各位大佬们，不知道为什么，总感...

人工查看范围：正文、全部 4 条评论。无附件、无图片。

### 读者承诺

作者觉得自己知识储备不够，写大纲时无从下笔，想找大纲表格。回复给出两个方向：星球内可搜“剑来”找资料；更关键的是先区分自己卡在世界观设定，还是卡在剧情大纲。

### 核心机制

```text
大纲无从下笔 -> 区分设定卡/剧情卡 -> 设定先粗后细 -> 剧情用目标计划法和高潮逆推法 -> 先列最想写的情节再排序
```

评论强调，设定、金手指、升级体系、人物都为故事服务。世界观不必一开始过细，先有粗略大概，写到哪需要什么再加什么。剧情大纲则可以线性列故事，再用目标计划法、高潮逆推法和因果逻辑延伸。

### 期待链与情绪

```text
作者以为缺知识/缺表格 -> 评论拆成设定问题或剧情问题 -> 不再追求万能模板 -> 从最想写的情节启动 -> 排序、定开头结尾、补因果
```

情绪上把“我不会写大纲”的焦虑，转成“我先写故事需要的东西”。这类问题不能只靠表格解决，表格只能承载思考，不能替代故事顺序。

### 关键判断

- 大纲卡住时先判断是世界观卡，还是剧情卡。
- 世界观不要一开始写太细。
- 如非必要，勿增实体；设定为故事服务。
- 剧情大纲没有固定神表格，本质是线性列故事。
- 可先列最想写的几个情节，再排序、定开头结尾。
- 目标计划法、高潮逆推法、因果逻辑可以配合使用。

### 可迁移写法

大纲启动分诊表：

```text
我卡的是世界观还是剧情：
当前已有故事情节：
最想写的情节 1：
最想写的情节 2：
最想写的情节 3：
这些情节的合理顺序：
开头事件：
结尾高潮：
主角目标：
中间计划：
需要补的设定：
可以暂缓的设定：
```

### 训练任务

不要先找表格。先列 5 个最想写的情节，按因果顺序排成一条线，定出开头和结尾，再只补“为了让这 5 个情节成立必须存在”的设定。

### 读者承诺

作者分享一份从知乎看到的“设置世界观的方法”，核心价值不是让作者把三百问全部写进正文，而是作为世界观思考方向和大纲启发工具。

### 核心机制

```text
世界观资料 -> 提供大量问题 -> 作者选择性思考 -> 启发想不到的方向 -> 只把故事需要的部分写进正文
```

评论里出现两种判断：有人觉得太复杂、费力不讨好；也有人指出工具不必全用，作者可以选择性挑问题思考大纲。宝剑回复“挺好的，这个很有必要”，其他评论也强调这是给作者思考方向，不是全部展示给读者。

### 期待链与情绪

```text
作者遇到世界观空白 -> 资料给出问题池 -> 选择有用问题 -> 补足大纲盲区 -> 正文只露必要设定
```

情绪重点是从“资料很复杂”转成“工具为我所用”。世界观资料的作用是启发作者，不是制造设定负担，也不是让读者在开篇承受设定展示。

### 关键判断

- 世界观工具有价值，但不能全部机械套用。
- 复杂资料应选择性使用。
- 世界观问题主要给作者思考，不一定写给读者看。
- 好工具能启发作者想不到的方向。
- 设定资料要服务大纲和故事，而不是替代故事。
- 附件不可读时，不能假装已拆过三百问正文。
