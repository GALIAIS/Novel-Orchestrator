## 166：#公开处刑

人工查看范围：正文、全部 16 条评论、附件状态。附件 `《我在东京吸取妖怪基因》--公开处刑.docx` 尝试下载返回 401；本卡依据正文与评论精拆。

证据限制：正文前五章不可见；评论已集中指出开篇设定过多、节奏乱、金手指期待不足、主角转变突兀、官方女配动机降智、情绪能量过高等问题。

### 读者承诺

这篇的问题不在“有没有十八元素”，而在元素没有形成一个可代入的主轴。东京、灵气复苏、妖怪基因、机械风、恋爱、官方冲突都可能有卖点，但开篇五章如果把它们一起往读者脸上推，读者先感到乱，而不是期待。主角主动也不是“立刻杀人”或“立刻决裂”，而是在清晰困境里做出可信求变。

### 核心机制

```text
设定过载 -> 主角/金手指篇幅不足 -> 情绪未铺就爆发 -> 行为转变突兀 -> 代入和爽感同时下降
```

### 期待链与情绪

```text
读者想看东京妖怪能力 -> 先被大量设定分散 -> 金手指未建立期待 -> 主角快速动手/决裂 -> 读者质疑逻辑 -> 爽点失效
```

### 关键判断

- 开篇元素齐，不等于开篇成立。
- 设定太多会显得乱，尤其会挤掉主角、金手指和当前危机的篇幅。
- 前三章可以把主角和金手指相关文字扩写到总字数三分之二，其余内容压缩。
- 穿越原主承受的不公平伤害，不能天然转化为当前主角的强代入。
- 情绪没渲染到位时，马上动手打、马上杀人，会让读者出戏。
- 主角几秒内完成杀伐果断式思维转变，容易显得突兀僵硬。
- 金手指必须先拉起期待，再承担解决危机和制造爽点的功能。
- 官方与主角决裂可以写，但女配不能为了推动决裂而降智。
- 如果要写到 10-15 章才稳住基调，前五章硬塞完只会节奏乱。
- 东京、灵气复苏、机械风、恋爱可以热门，但必须由一个开篇主轴统一。
- 画面描述不清时，读者几眼看不明白就会弃。
- 不是所有题材都适合开局高情绪值；很多类型需要先稳定代入，再逐步抬矛盾。

### 可迁移写法

设定压缩表：

```text
开篇主轴：
主角当前困境：
金手指可见功能：
前三章主角+金手指占比：
必须前置的设定：
可后移的设定：
第一波情绪值：
主角转变是否有过渡：
配角行为是否有正常动机：
```

### 训练任务

把一个“都市异能/灵气复苏”开篇压成三章小纲：每章只允许一个新增设定，主角和金手指合计占三分之二篇幅，第一章不允许安排主角突然完成极端人格转变。

## 167：#公开处刑

人工查看范围：正文、全部 12 条评论、附件状态。附件 `因为我得了精神病，所以被拉入了冥界.docx` 只有 file id，未提供可下载链接；本卡依据正文与评论精拆。

证据限制：正文附件不可见；评论已给出关键判断，包括书名承诺和正文不符、精神病只是进入异界钥匙、恐怖求生节奏偏慢、平台风向风险等。

### 读者承诺

这篇的核心是“书名和正文承诺不能骗”。《因为我得了精神病，所以被拉入了冥界》承诺的是精神病认知与诡异世界之间的反差：别人怕鬼，主角不怕；别人觉得异常，主角觉得正常；幽默和恐怖都来自这种错位。如果主角脑子很正常，精神病只是一把开门钥匙，那读者冲着书名进来会发现承诺落空。

### 核心机制

```text
热词书名 -> 读者期待精神病反差 -> 正文只是群穿/异界钥匙 -> 承诺不兑现 -> 读者弃读或差评
```

### 期待链与情绪

```text
读者被精神病+冥界吸引 -> 期待主角异常认知破局 -> 正文主角表现正常 -> 卖点消失 -> 节奏慢问题被放大
```

### 关键判断

- 书名、简介、正文必须给同一批读者同一种承诺。
- 精神病热词不能硬蹭；正文没有对应体验，会反噬。
- “把读者骗进来杀”不可取，骗来的不是受众，留不住还可能差评。
- 如果卖点是精神病进诡异世界，主角就要真有精神病式认知差。
- 精神病题材常见爽点是反差：正常人恐惧，主角不怕；正常人死，主角活。
- 如果主角只是被拉入异界，精神病就不是卖点。
- “从小能看见脏东西的我被拉进冥界”反而更贴正文。
- 文笔能看不等于结构能留人；两万字才见鬼，对恐怖求生可能偏慢。
- 番茄灵异脑洞有平台风向，不能只看热词，要看收稿口径。
- 书名蹭校花、女友、女帝、女仆等词，但正文不提供对应内容，会错配读者。
- 简介可以恐怖、装逼或搞笑，但不能和正文体验相反。
- 同题材对标要拆承诺如何兑现，不只是拆哪个词多人看。

### 可迁移写法

书名承诺核对表：

```text
书名关键词：
读者会期待的核心体验：
主角第一章是否体现该体验：
简介是否同向：
正文第一事件是否兑现：
蹭词风险：
目标平台读者是否匹配：
如果删掉关键词，故事是否仍成立：
```

### 训练任务

给一个灵异书名做三版承诺校准：恐怖版、搞笑反差版、装逼求生版。每版都写出第一章必须出现的主角行为，不能只改书名和简介。

## 168：修改了一个开头，就不再发一遍公...

人工查看范围：正文完整开头片段、全部 24 条评论。重点查看“爆点前置+回忆解释”的插叙写法、巨狼危机、冰封背景、金手指手表、实验室自毁伏笔，以及评论中关于运镜、信息跳级、危机中插回忆的修正建议。

### 读者承诺

这篇不是不能用爆点前置，而是开篇的画面顺序必须让读者接得住。主角刚醒就遇巨狼，本该先形成清晰危机、主角反应、金手指解决和大目标，但正文插回十几分钟前解释冰封、实验室、装备、森林，会把紧张感切断。开头的任务不是证明大纲全都合理，而是让读者顺着镜头持续看下去。

### 核心机制

```text
爆点前置 -> 画面未铺 -> 插回忆解释 -> 危机被打断 -> 伏笔不显 -> 读者觉得乱和平淡
```

### 期待链与情绪

```text
巨狼扑来 -> 读者想看主角如何活 -> 正文跳回背景 -> 紧张下降 -> 设定疑问堆积 -> 主角目标不明
```

### 关键判断

- 当前-回溯-当前写法可以用，但开篇读者还没共情时风险很高。
- 上来一只巨狼扑来，如果没有铺垫和虚晃，画面像粗糙恐怖片。
- “我都被冰封五十年了”这类信息，在读者没接住身份时容易突兀。
- 回忆解释如果打断读者等危机解决，会削弱紧张感。
- 实验室自毁可以是伏笔，但读者看不出来时，只会觉得多余。
- 草原、一万年、五十年等信息需要足够引导，否则揭晓时变成“信息跳级”。
- 开局战斗容易喧宾夺主；真正重点可以是主角激活金手指。
- 金手指手表要从外貌、来源、功能、解决当下危机几个层面重点写。
- 主角醒来后的目标要明确：找人、找弟弟、活下去、弄清世界变化，至少要让读者知道下一步看什么。
- 运镜顺序可以改成：狼爪逼近、主角半醒、弟弟声音警告、扫视四周、巨狼逼近、逃跑中少量回忆。
- 战斗必须服务目的：体现敌人强、主角决断、金手指有效、第一波装逼。
- 爽点后可以用饥饿、虚弱、寻找食物等下行接下一章，避免单元断档。

### 可迁移写法

插叙爆点自检表：

```text
开头爆点画面：
读者此刻最想知道的问题：
是否立刻打断危机：
回忆必须解释的信息：
能否后移到危机解决后：
金手指第一次可见效果：
主角正面决断：
危机后的新目标：
```

### 训练任务

重写一个“苏醒后遇怪”的 1500 字开篇小纲：前 500 字只写当前危机和主角反应，中段激活金手指，最后 300 字再交代一个必要背景信息，并接出下一章目标。

## 169：#思考分析

人工查看范围：正文、全部 10 条评论。重点查看“挑剔读者”假设、弃读理由清单、教程与套路的定位、包装和气味类比，以及评论中关于创意、剧情和读者情绪的争论。

### 读者承诺

这篇把系统的自审视角说得很直接：检阅一章文，不是问自己套了多少套路，而是问一个挑剔读者有多少理由放下它换一本。这个读者没耐性、记不住长伏笔、只认书名承诺、讨厌偏题水文虐主、嫌解释长、嫌配角无用、嫌套路重复。作者要做的是在精彩剧情真正到来前，用包装、开篇、情绪和持续希望把他留住。

### 核心机制

```text
假设挑剔读者 -> 枚举弃读理由 -> 逐项堵破绽 -> 先留住读者 -> 再展开故事和套路
```

### 期待链与情绪

```text
读者点进同类书 -> 看包装和前几百字 -> 找到弃读理由就走 -> 找到希望/爽/惊喜才留下
```

### 关键判断

- 检查一章够不够好，要看读者有多少放下它的理由。
- 套路是辅助，自觉检查自己的文才是作者最重要的事。
- 挑剔读者只认书名定下的题材，偏题、水文、虐主都可能弃。
- 他没耐性，不愿等下一章，最好本章就看到希望或爽点。
- 伏笔超过数章不提醒，读者可能忘记，回收时反说突兀。
- 解释太长，会让读者忘记前面的剧情。
- 配角、支线、场景移动都必须有意义。
- 读者会觉得爽点还不够爽，套路套娃又不新。
- 金手指、敌人智商、巧合、名字、道具顺序都会成为弃读点。
- 读者有正常作息，会中断阅读；不够震撼的桥段他会忘。
- 教程和套路来自读者反馈总结，不能反过来压倒读者体验。
- 包装新颖等于金手指和卖点，气味浓郁等于书名简介开篇几百字能诱人。
- 剧情很重要，但精彩剧情出现前，仍要想办法留住读者。
- 真正学套路，是理解套路为什么能调动读者情绪。
- 要让读者猜到开头，却猜不到结尾；意外、惊喜、不毒同时成立。

### 可迁移写法

挑剔读者自审表：

```text
书名承诺：
本章是否偏题：
本章希望/爽点/惊喜：
解释是否过长：
伏笔是否 5 章内提醒：
配角是否有功能：
支线是否流回主线：
主角是否废话太多：
读者此刻最可能弃读的理由：
如何在本章堵住：
```

### 训练任务

拿自己一章正文做“弃读理由审判”：列出 20 个挑剔读者可能放下书的理由，并把其中最致命的 3 个改成同章可见的希望、爽点或悬念。

## 170：“只要用的对，就算双标情节也能...

人工查看范围：正文、全部 12 条评论、4 张图片。图片 1 为漫画完整桥段：男主帮女性拧开瓶盖，却对男性朋友拒绝并触发“重色轻友/算了算了”的喜剧反应；图片 2、3 为原贴文字说明和边界判断；图片 4 为“算了算了”表情图。

### 读者承诺

这篇讲的是轻度双标如何变成喜剧，而不是鼓励主角原则崩坏。拧瓶盖这种小事没有实质伤害，朋友之间插科打诨可以凸显“见色忘友”、沙雕、调侃氛围，读者笑的是人设反差和伙伴反应；但如果换成救人、重大利益、原则抉择，主角还双标，就会变毒。

### 核心机制

```text
无害小请求 -> 主角轻度偏心 -> 基友夸张控诉 -> 旁人劝架 -> 凸人设和调节节奏
```

### 期待链与情绪

```text
读者识别小事 -> 预判主角见色忘友 -> 基友反应放大 -> 旁人拦住 -> 轻松一笑 -> 人设更鲜明
```

### 关键判断

- 图片中的双标更准确说是借“重色轻友”之名，做“美女爽点”和喜剧反应。
- 这种桥段适合搞笑调侃，不适合重大原则场景。
- 拧瓶盖是小事，基友没受到实质伤害，所以读者不适感低。
- 如果换成救人二选一，主角见色轻友就会毒。
- 双标可以轻度，不能重度。
- 朋友间插科打诨本身能调节气氛和节奏。
- 这也是凸人设的惯用技巧。
- 女频团宠里父母宠女儿、调侃式区别对待儿子，也属于轻松基调下的双标喜剧。
- 轻松双标的前提是双方关系安全，实质上没有恶意和伤害。
- 被双标的一方最好有夸张吐槽、追打、被旁人拦住等反应，形成完整笑点。
- 主角不能以毫无悔意、以此为荣的态度造成严重后果。
- 读者接受的是轻松人设，不是道德破产。

### 可迁移写法

轻度双标边界表：

```text
请求事项：
是否小事：
被拒方是否受实质伤害：
偏心对象：
主角人设收益：
被拒方反应：
旁人是否劝架/吐槽：
是否触及生命/尊严/重大利益：
读者会笑还是会厌：
```

### 训练任务

写一个 600 字轻喜剧桥段：主角对朋友和暧昧对象轻度双标，但事件必须是小事，被双标者要有夸张反应，最后用吐槽收束，不能造成实质损失。
