## 091：本期视频地址：网文的“情绪拉扯...

人工查看范围：任务全文、三张可见图片、全部评论、作业样例与批改反馈。

证据限制：附件 `情绪拉扯的构建方式和案例解析+作业.docx` 外链返回 401，无法下载；本卡依据知识库正文、评论和可见图片精拆。

### 读者承诺

这篇是情绪拉扯训练的标准作业入口。它不是泛讲概念，而是把新人训练压成一个可验收格式：三选一题目、五折线、至少一个爽点、必须有虚晃一枪、下行必须符合“功都是主角的，锅都是别人的”。

### 核心机制

```text
明确题目 -> 确认目标 -> 画五折线 -> 逆推下行 -> 加虚晃一枪 -> 检查锅和功 -> 删除废戏
```

### 关键判断

- 严格审题是训练的一部分，写错题比写得差更浪费。
- 五折线是作业格式，不是所有开篇都必须五折线；评论里明确开头更常用三折线短平快。
- 作业只要求每段一句话梗概，长篇大论说明作者还没想清楚结构。
- 爽点要放在最后上升段，也可以在中间上升段制造小爽点。
- 虚晃一枪的重点是先让读者以为危险或冲突不会发生，随后意外下行。
- 下行不能由主角犯蠢、捅娄子、主动逃跑制造，否则会变成毒点。
- 同一结构能写多少字没有固定答案，要看文风、题材和是否加铺垫线。
- 批改样例反复强调：机器评分压大神歌喉会弱化金手指；校园主角主动跑会毒；混混袭警不符合常识；过度被动主角不建议。
- 新人准备开书要先看同题材榜单，重点拆金手指如何应用、如何贴合主线装逼，以及自己最迷茫的短板。

### 可迁移写法

五折线作业表：

```text
题目/题材：
主角金手指：
事件目标：
1 情绪上行：
2 拐点+下行：
3 情绪上行/小爽点：
4 拐点+下行：
5 爽点爆发：
虚晃一枪在哪一段：
下行的锅是否在别人：
主角是否主动且不犯蠢：
删除的废戏：
```

### 训练任务

任选 KTV、校门口混混、卡丁车三题之一，只写五句梗概。写完逐条检查：有没有虚晃一枪、下行是否锅在别人、最后爽点是否由金手指完成目标。

## 092：网文开头13项要点审核-【2023-5-4日更新版本】

人工查看范围：正文可见入口、全部评论、两张可见图片、关于 2024 新版入口、冲突、金手指、无敌文、慢开局、18元素篇幅和压抑/下行边界的讨论。

证据限制：知识库 Markdown 只保留正文开头和评论，主体以外链文章形式保存。

### 读者承诺

这篇是开篇审核工具的旧版总入口。它的价值不只在清单本身，更在评论里给出的边界：开篇要围绕主角、金手指、冲突、收益、爽点和期待，但清单不是绝对法规；书名、简介、题材和读者预期可以改变开篇节奏。

### 核心机制

```text
书名简介承诺 -> 主角聚焦 -> 冲突/目标 -> 金手指显形 -> 期待与收益 -> 首个爽点
```

### 关键判断

- 开篇冲突最好冲着主角本身，冲突只在朋友或配角身上会削弱代入。
- 思想品质通常不能直接当金手指，除非它能持续带来可见收益和期待。
- 无敌文不一定需要危机，但必须有冲突。
- 旧版 18 元素不是绝对标准，只是给新人降低失误率的建议。
- 《烂柯棋缘》这类慢开局能成立，是因为书名、简介已经提前建立“他怎么遇见棋局”的期待。
- 18 元素一般可在 1-3 万字内完成，但不是机械章数。
- 三次压抑不是所有爽点都必须有；可以一次到位，也可以曲折向上。
- 下行不是必填项，前世受苦、旧伤、既有压迫也可以承担下行功能。
- 用新工具规范写作初期会僵硬，必须靠持续写和练磨合。

### 可迁移写法

开篇审核边界表：

```text
书名简介已经承诺什么：
主角是谁：
冲突是否直接影响主角：
金手指是否可见：
读者期待的收益：
是否必须快开：
如果慢开，书名简介是否提前拉住期待：
首个爽点在哪：
哪些元素暂时不必强塞：
```

### 训练任务

拿自己的开篇做一次审核：只标三项，主角聚焦、金手指可见、首个期待。若三项不成立，先别补设定，直接重写第一单元目标。

## 093：#公开处刑 投稿被拒，编辑反馈...

人工查看范围：正文全文、全部评论、关于投稿被拒、缺代入感、剧情散、无爽点、主角沉睡、配角戏过多、18元素误判和重写方向的讨论。

证据限制：附件 `公开处刑-十六.docx` 外链返回 401，无法下载；本卡依据知识库正文与评论精拆。

### 读者承诺

这篇是新人开篇失败的典型诊断：作者觉得必要的背景、危机、配角打斗和世界设定，在读者眼里全是废话。真正要先想清楚的是三句话：主角面对什么冲突，获得什么金手指，如何解决冲突并装逼。

### 核心机制

```text
主角目标缺失 -> 配角戏吞掉开篇 -> 读者没有期待 -> 编辑判定散 -> 删除重写
```

### 关键判断

- “剧情散”不是文字乱，而是主角没有目标，也没有围绕目标行动。
- 主角介绍后沉睡，配角戏份过多，会直接破坏代入感。
- 开篇一整章写主角和父亲处境、悬浮背景关系，不符合商业开篇需求。
- 网文开篇要抛出“你来我往、蓄势待发”的情节，而不是作者自认为宏大的危机。
- 危机形式不是重点，危机带来什么期待、收获、爽点才是重点。
- 视角第一时间不聚焦主角，会让读者连谁是主角都混乱。
- 养崽、复仇、历劫、身体重塑等想法太多，开篇反而没有主线。
- 文笔流畅不能替代故事功能；配角打架写一章，和主角关系不强就是大雷。
- 改不如删，根本方向错了就别在局部修补。
- 后续评论补充：先设计最爽的画面，再围绕爽画面铺垫；期待感来自读者预知和信息差，爽感来自对比。

### 可迁移写法

删稿重写三问：

```text
主角面对什么冲突：
主角获得什么金手指/优势：
主角如何解决冲突并装逼：
读者最想看的爽画面：
谁不知道主角强：
谁会和主角形成对比：
哪些配角戏必须删：
第一章是否让主角行动：
```

### 训练任务

找一个失败开篇，不改原文。先用三句话重写故事核心：冲突、金手指、装逼解决。再把第一章里与这三句话无关的场景全部列为删除项。

## 094：目标平台，番茄。三张开头，请大...

人工查看范围：正文三章开头全文、全部评论、关于末日城、代理系统、主角性格毒点、情感行为矛盾和都市装逼支线的讨论。

### 读者承诺

这篇提供了一个有可取卖点但有人设毒点的开篇样本：现实负债主角绑定末日城市，像系统一样给幸存者发任务，从幸存者劳动中抽成换钱。这一核心很有爽感，但主角突然共情到哭、又马上盘算剥削幸存者，会让人物情感和行为打架。

### 核心机制

```text
现实债务压迫 -> 绑定末日城 -> 伪装系统发任务 -> 幸存者劳动产值 -> 主角抽成变现
```

### 关键判断

- “末世、系统、都市”融合思路被评论认可，两个世界转换也较流畅。
- 主角现实压力明确：父债三百万，催债上门，一个月期限。
- 金手指有新鲜感：主角不是获得系统，而是成为幸存者眼里的系统。
- 抽成、商城差价、任务奖励可以持续制造经营爽点。
- 主角发现幸存者是真人后立刻哭，情绪过快，会显得突兀。
- 先说一定送幸存者回家，后面又说让人累死累活给自己赚点数，行为和情感矛盾。
- 催债方人多势众却容易怂，可信度偏弱。
- 第三章同学聚会打脸俗套，但相比前面人设毒点，反倒正常。
- 幕后系统流要保持主角的价值观、收益逻辑和情绪表达一致。

### 可迁移写法

代理系统开篇表：

```text
现实压力：
系统代理权限：
幸存者为什么需要主角：
主角如何从任务中获益：
主角对幸存者的底线：
情绪表达是否过快：
收益行为是否和价值观冲突：
下一场现实爽点如何承接：
```

### 训练任务

设计一个“主角成为系统代理”的开篇。必须写清主角获利方式和道德底线，禁止前一段圣母落泪、后一段马上剥削。

## 095：大大，我想问一下，如何打破自己...

人工查看范围：正文提问、全部评论、关于固有思维、中庸、卖点中心、扫榜、灵感记录、矛盾和对比的讨论。

### 读者承诺

这条短问答解决的是“写不到点上”的根因：不是先追求花活，而是先围绕卖点构建剧情。系统文围绕系统，御兽文围绕宠兽，卖点从开篇就要明确，后续不要轻易变形。

### 核心机制

```text
确定卖点 -> 围绕卖点构建事件 -> 用卖点制造爽点 -> 扫榜记录亮点 -> 用矛盾/对比放大
```

### 关键判断

- 中庸常来自跑题：剧情东一笔西一笔，没围绕卖点推进。
- 系统文要用系统推进剧情，用系统能力创造爽点。
- 御兽流要以收服强力宠兽、宠兽对战实现爽点。
- 开篇卖点必须明确，后续尽量不要把读者吸引来的东西换掉。
- 想打破死板思维，要扫榜，看别人怎么写亮点，并记录功法、打斗、世界观、应对方式等可用灵感。
- 情绪起伏能弥补亮点不足，但不能替代卖点。
- 矛盾和对比是打破中庸的高效工具。
- 番茄代表性升级文的一个强项是对比：强弱、身份、资源、预期、反应之间的反差。

### 可迁移写法

卖点中心表：

```text
本书卖点：
开篇是否出现：
当前事件是否围绕卖点：
爽点由卖点的哪项能力产生：
读者点进来想看的东西：
有没有偏成别的题材：
本章对比点：
今日扫榜记录的一个亮点：
```

### 训练任务

选一个自己的题材，只写一句卖点。再列 5 个事件，每个事件必须说明“这个事件如何使用卖点制造爽点”。写不出来的事件直接删。
