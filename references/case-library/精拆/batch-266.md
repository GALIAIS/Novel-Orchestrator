## 1326：宝剑大大，请教一个问题。


人工查看范围：正文、全部 5 条评论。无附件、无图片。

证据限制：这是设定问答，没有具体正文样章；拆解重点是“身边卧底”不必成为毒点，关键是主角是否比其他人聪明一线并掌握主动。

### 读者承诺

这篇沉淀的是卧底危机可以写，但不能把主角写成全程被动挨打。提问者想在灵气复苏、连环杀人事件背景下设置一个敌方卧底，平时跟在主角身边，小高潮时作为小 boss 解决，用来提供生死危机、诬陷和不公平伤害。死翼的回答很明确：主角可以没立刻拆穿，但要察觉异样，并且最后先于其他人确认身份。

### 核心机制

```text
身边潜伏威胁 -> 主角察觉异常 -> 暗中准备 -> 小高潮确认身份 -> 主动反杀卧底
```

读者容忍危机，不容忍主角比配角还迟钝。主角比其他人聪明一线，读者就心安理得；和其他人一样，基本扑街；比其他人还弱，必然扑街。稳妥写法是前面埋“发现异样”的伏笔，小高潮揭示主角早有准备，借此彰显运筹帷幄。

### 期待链与情绪

```text
读者知道身边有问题 -> 主角发现细节 -> 表面继续同行 -> 暗中布置后手 -> 卧底暴露时主角反杀
```

卧底的价值不是让主角一直被动接招，而是制造“你以为主角不知道，其实主角早在布局”的期待回收。读者要看到主角在危险中领先其他人半步。

### 关键判断

- 身边卧底本身不是毒点。
- 主角长期毫无察觉才是毒点。
- 主角至少要比其他人聪明一线。
- 卧底造成的诬陷和不公平伤害要有逻辑来源。
- 伏笔要提前放，不能小高潮临时硬说早有准备。
- 主角掌握主动，比单纯被动反击更稳。
- “所有人没注意到的危机主角注意到了”是可复用开篇优势。

### 可迁移写法

```text
卧底身份：
卧底接近主角的理由：
第一处异常：
主角察觉但不说破的理由：
主角暗中准备：
卧底制造的伤害：
小高潮确认方式：
反杀/揭穿爽点：
```

### 训练任务

写一个 1000 字卧底小高潮：前 300 字用一个细节证明主角早有察觉，中段让卧底发动诬陷或袭击，结尾让主角拿出提前布置的证据或后手反杀。

## 1327：如何判定自己有没有涉及抄袭呢，...


人工查看范围：正文、全部 3 条评论。无附件、无图片。

证据限制：这是边界问答，没有具体对照片段；拆解重点是新人借鉴时要拆创意和结构，不要做“中译中”式复述。

### 读者承诺

这篇沉淀的是借鉴不是问题，照搬表达和剧情链才是问题。提问者担心写着写着脑海里出现其他书里的画面，死翼给出的边界很短：没搞中译中就没事；借鉴创意可以，别把句子、段落、剧情都抄。

### 核心机制

```text
看到作品画面 -> 提取创意/功能 -> 换人物欲望与情境 -> 重建事件因果 -> 避开句段剧情照搬
```

“中译中”不是只改词，而是把别人的句子、场景顺序、冲突过程换一层说法。真正可用的借鉴，是把一个功能拆出来：这里负责拉期待、制造误会、展示能力、兑现爽点，然后用自己的题材、人物、规则和因果重写。

### 期待链与情绪

```text
新人想借鉴好桥段 -> 担心抄袭 -> 学会抽功能不搬文本 -> 获得可操作边界 -> 敢写但保留自检
```

这条对拆书系统也重要。拆书不是复制原文，而是把桥段背后的承诺、机制、节奏、情绪功能拆出来，再生成自己的事件。

### 关键判断

- 借鉴创意不等于抄袭。
- 句子、段落、剧情链都照搬，风险最高。
- 只换词不换因果，就是中译中。
- 拆书要拆功能，不要拆成可替换模板句。
- 相似画面不可怕，连续相似事件链才危险。
- 角色欲望、规则、代价、结果都要重建。
- 写完后要反查是否保留了原作具体表达。

### 可迁移写法

```text
原作可学点：
它在原作中的功能：
不能照搬的句子/段落：
不能照搬的事件链：
我的人物欲望：
我的世界规则：
我的新因果：
最终差异点：
```

### 训练任务

选一个喜欢的桥段，只写“功能说明”，不写原剧情复述。再用完全不同的题材和人物写 500 字新桥段，最后标注与原作至少 5 个差异点。

## 1328：最近读了《如何创造炫人耳目的对...


人工查看范围：正文、1 张图片、全部 5 条评论。图片已下载到 `_source_cache\batch-266\1328-1.jpg` 并查看，内容为《如何创造炫人耳目的对话》思维导图，包含对话是什么、对话不是什么、用人物、针对对话练习、加强对话冲突、塑造对话技巧、对话写作十大议题等分支。微信读书搜索页 GET 200，幕布链接 GET 200，但均为前端页面，未额外取得可读大纲文本。

证据限制：图片较大且部分小字较糊，拆解依据图片可见主干、正文说明与评论反馈，不替代原书完整阅读。

### 读者承诺

这篇沉淀的是对话不是“角色轮流说话”，而是推进、叙事、传递信息、制造悬念和暴露人物的综合工具。作者读完书后做思维导图，死翼扫了提纲判断“应该挺有用”。评论补充：几乎每章都需要对话，如果其他方面拉胯，也可以试着用对话制造悬念。

### 核心机制

```text
人物欲望 -> 对话目标 -> 信息不完全 -> 冲突/潜台词 -> 推进剧情或暴露人物
```

图片主干里最可迁移的部分是：对话行动功能、用人物、加强冲突、塑造技巧、对话十大议题。对话要服务于人物想要什么、隐瞒什么、误解什么、压迫什么，而不是把背景资料平铺给读者。

### 期待链与情绪

```text
读者进入对话 -> 发现双方信息不对等 -> 听出潜台词/冲突 -> 获得新信息或新悬念 -> 期待下一步行动
```

好对话既能让读者看懂当下，又能保留未说出口的东西。它可以承担叙事，但不能变成说明书；可以传递信息，但要让信息带着人物立场和冲突进入场景。

### 关键判断

- 对话应承担推进、叙事、信息传递、悬念和人物塑造。
- 对话不是背景说明的替代垃圾桶。
- 每段对话都要有至少一方的目标。
- 信息差能让普通对话变成悬念。
- 冲突不一定是吵架，也可以是隐瞒、试探、误解。
- 人物说话方式要暴露身份、关系和压力。
- 工具书资料要转写成可执行训练，而不是只收藏导图。

### 可迁移写法

```text
对话场景：
角色A目标：
角色B目标：
双方信息差：
不能明说的信息：
冲突/试探点：
对话推进的剧情：
留下的悬念：
```

## 1329：内投编辑不回复是不是就凉了，因...


人工查看范围：正文、全部 15 条评论。无附件、无图片。

证据限制：这是投稿流程讨论，不是创作技法；拆解重点是新人投稿时不要因焦虑乱投、乱撤、破坏基本流程。

### 读者承诺

这篇沉淀的是内投等待期的纪律。提问者担心编辑不回复就是凉了，又因为听说不要一稿多投，所以不敢投下一家。评论共识是：不要一稿多投，等明确被拒再投下一家；没回可能是没来得及看，也可以顺着邮箱加 QQ 直接问；周末编辑不加班正常，但不同编辑处理速度差异很大。

### 核心机制

```text
投递稿件 -> 等待编辑反馈 -> 超时礼貌询问 -> 明确拒稿/撤回 -> 再投下一家
```

这个案例的关键不是判断某个编辑到底看没看，而是流程意识：新人不要在信息不全时用焦虑驱动决策。六天没回可能让人难受，但直接一稿多投风险更高；要么继续等，要么礼貌确认，要么明确撤回后再投。

### 期待链与情绪

```text
作者投出稿件 -> 等待变焦虑 -> 寻找外部确认 -> 获得流程规则 -> 用询问/撤回替代乱投
```

投稿是创作系统的一部分。作品写完不代表流程结束，稳定的投递纪律可以减少不必要的编辑关系风险和时间损耗。

### 关键判断

- 不要一稿多投。
- 没回复不一定立刻等于拒稿。
- 可以通过邮箱、QQ 等公开渠道礼貌询问。
- 周末和编辑个人节奏会影响回复速度。
- 明确拒稿或撤回后再投下一家。
- 不同编辑处理速度差异大，不能套单一标准。
- 新人投稿要把焦虑转成流程动作。

### 可迁移写法

```text
投稿平台/编辑：
投递时间：
约定等待期：
已等待天数：
可询问渠道：
询问信息：
撤回条件：
下一家投递时间：
```

### 训练任务

为自己的投稿建立一张表：每条记录写清投递对象、投递时间、预计等待期、询问时间、反馈状态、是否撤回。没有明确状态前不投同一稿件给下一家。

## 1330：#公开处刑 大家好，我又来了，...


人工查看范围：正文、文件记录、全部 11 条评论。文件记录包含“第一到五章.doc”和“开头小纲填写表格.docx”；导出页 `full-index.html` 中对应文件 `url` 为空，本地按文件名与文件 id 搜索未找到附件实体。

证据限制：无法查看五章正文与表格附件，只能依据作者自述和评论反馈拆解。拆解重点是赛博朋克/科幻开篇不能用信息轰炸替代人物关系、剧情线和情绪入口。

### 读者承诺

这篇沉淀的是科幻开篇尤其容易把背景、设定、人物关系、风格追求一次性塞给读者，但网文开篇仍然要先让读者看懂“谁、为什么被针对、凭什么强、接下来要做什么”。作者想写赛博朋克世界的硬汉角色，身手高强、表面诙谐、背负浪漫沉重故事，并加入威廉吉布森式严肃赛博朋克内涵；评论反馈却是没看懂人物为什么针对主角、主角凭什么厉害、警方为何像熟人。

### 核心机制

```text
赛博设定/硬汉气质 -> 信息量过大 -> 人物关系缺席 -> 情绪线不清 -> 开篇阅读体验混乱
```

作者解释很多内容在第三章、第四章、第五章才交代，但开篇读者已经需要基础抓手。Franz 的评论抓住重点：像用 30 章节奏写开局，信息轰炸感有代入，但阅读体验不好；宝剑理论里开头情绪可能比铺垫更重要，信息少也能先把矛盾冲突和代入写出来。

### 期待链与情绪

```text
读者进入陌生科幻世界 -> 先抓主角处境与冲突 -> 产生情绪代入 -> 再逐步理解设定背景 -> 接受更大信息量
```

科幻不是不能交代背景，而是背景要被行动、危机、人物关系带出来。赛博朋克的术语、世界观、风格可以成为质感，但不能挡住第一章最基本的剧情线。

### 关键判断

- 科幻开篇也要先有可理解的冲突。
- 世界背景不能替代人物关系建立。
- 主角强大必须有读者能接受的展示或暗示。
- “第五章会解释”救不了第一章的迷茫。
- 信息轰炸可以制造风格，但会牺牲清晰度。
- 新人写长后才更容易判断哪些信息可以后移。
- 开头情绪优先级常高于设定完整度。

### 可迁移写法

```text
第一章主角身份：
主角当下麻烦：
谁在针对主角：
针对理由的最小暗示：
主角强大的最小展示：

本章情绪入口：
读者下一章期待：
```
