## 1836：在知乎上经常看到蛤蟆等大佬说开...


人工查看范围：正文、全部 4 条评论。

证据限制：这是开局写法讨论帖，不含作者自己的开头样稿；本卡只拆“开局只写主角不是写人物介绍书，而是用选择和舞台展示主角魅力”的方法。

### 读者承诺

这篇沉淀的是“开局只写主角”的真正含义：不是三千字介绍主角履历，而是在冒险环境或压力场里只让主角成为读者注意力中心。提问者观察《诡秘之主》《大奉打更人》《夜的命名术》《这个明星很想退休》等开头，发现很多第一章基本只围绕主角，配角只承担引出剧情或末尾补环境的功能。大佬解释是防止新人写乱，但提问者自己尝试后，经常写成“主角介绍书”。

评论把问题定位到舞台和选择。别闹zzz 说自己开头加两个配角后，回头看主角成了路人，注意力被分散；后来发现不是不能写配角，而是出场场景太随便，没有足够展示主角选择，所以塑造苍白。开局要先确定主角性格，要讨喜，同时构架适合主角登场的舞台，让主角能充分展现魅力。Franz 则说第一章只写主角时，就不停给主角岔路让他做选择，这样不直接介绍性格也能表现人设，还能推动剧情。

### 核心机制

```text
开局聚焦主角 -> 设置压力舞台 -> 给主角连续选择 -> 通过选择显人设 -> 配角只服务主角
```

只写主角不是少写人物，而是把所有信息都变成展示主角的动作和选择。

### 期待链与情绪

```text
读者进入开局 -> 注意力集中在主角 -> 看主角面对岔路选择 -> 判断主角讨喜与否 -> 愿意追下去
```

如果主角没有选择，聚焦主角也会变成枯燥介绍。

### 关键判断

- 开局只写主角是为了防止新人写乱。
- 配角可以出现，但必须服务主角。
- 配角过多容易喧宾夺主。
- 只写主角不等于写主角介绍书。
- 主角性格要先确定。
- 主角要讨喜，能吸引读者。
- 开局舞台要适合主角登场。
- 选择比介绍更能表现人设。
- 岔路选择能同时推动剧情。
- 世界背景可以后置，先让主角成立。

### 可迁移写法

```text
主角核心性格：
开局压力舞台：
第一处岔路：
主角选择：
读者看到的人设：
配角是否服务主角：
背景后置位置：
本章推进结果：
```

### 训练任务

写一个只聚焦主角的第一章大纲：必须设计 3 次选择，每次选择都展示不同人设侧面；配角只能用来制造压力或反馈主角。

## 1837：大佬们，开局自带女友，后期分手...


人工查看范围：正文、全部 11 条评论。

证据限制：这是毒点判断问答，不含具体剧情样稿；本卡只拆“开局自带女友后期分手/死亡容易晃点读者”的风险。

### 读者承诺

这篇沉淀的是开局自带女友会自动建立强关系承诺，后期分手或死亡极易变成毒点。提问者问开局自带女友，后期分手或者死亡算不算毒点。评论反应很直接：算，死女主是大忌，分手也可能是剧毒。何天然给出少数可行条件：如果彻底死亡，前期铺垫和人物刻画都必须是为了她一死而准备，且死得有极大意义；关键她不能是真正女主，不然肯定翻车。真要死，一上来死反而更好接受。

奔跑的车轱辘追问“你为什么要这么做”，并指出任一配角都要有足够作用，更何况女友。提问者说前期没有过多情感描写，只靠“女友”概念，十章后掉线，想知道读者是否会无感。评论结论是：如果不是非得女友，可以换工具人，不要晃点读者。这说明读者对“女友”这个标签天然有预期：感情线、陪伴、归属、稳定关系或后续重要影响。作者若只是想制造死亡刺激或剧情推动，用女友身份包装工具人，就是拿强承诺骗读者。

### 核心机制

```text
开局女友标签 -> 读者建立关系承诺 -> 后期分手/死亡 -> 承诺落空 -> 毒点爆发
```

女友不是普通工具人，身份一给出就绑定读者期待。

### 期待链与情绪

```text
读者看到自带女友 -> 期待稳定关系或重要情感线 -> 女友掉线 -> 感到被晃点
```

死亡若无极大意义，前期情感铺垫越多，反噬越强。

### 关键判断

- 开局女友后期分手通常是毒点。
- 死女主是大忌。
- 女友死亡必须有极大意义。
- 若要死，最好她不是真女主。
- 一上来死亡比培养后死亡更容易接受。
- 女友身份会自动制造强关系承诺。
- 情感描写少不代表读者无感。
- 任一配角都要有足够作用。
- 不是非得女友，就换工具人。
- 不要用女友标签晃点读者。

### 可迁移写法

```text
角色身份：
读者默认承诺：
她的剧情作用：
是否必须是女友：
掉线方式：
死亡/分手意义：
替代工具人方案：
读者反噬风险：
```

### 训练任务

把一个“开局女友十章后死亡”的设定改成三版：工具人版、开局已死版、非女主但死亡有重大意义版。比较哪版承诺风险最低。

## 1838：今天听到一个说法。


人工查看范围：正文、全部 2 条评论。

证据限制：这是创作习惯反思帖，不含具体作品案例；本卡只拆“码字只是输出环节，不能用战术勤奋掩盖战略懒惰”的训练判断。

### 读者承诺

这篇沉淀的是新人不能把闷头码字当成全部努力。作者听到“不能用战术上的勤奋，掩盖战略上的懒惰”，于是决定每天不能只码字，还要抽时间看书、拆书、想剧情。正文说得很清楚：码字只是网文的一个输出环节，并不是码得越多越能出头，也要搞好输入和思考环节。

评论继续确认这一点。是康娜酱呀说闭门造车只会把自己弄自闭，写不出好作品；執着的小乐说“有道无术，止于术”。可迁移的判断是：写作能力不是单一字数堆出来的，而是输入、拆解、构思、输出、反馈的闭环。码字量当然重要，但如果题材判断、结构设计、读者承诺、爽点储备都没更新，码得越勤快，可能只是把错误路径走得更远。

### 核心机制

```text
输入阅读 -> 拆书分析 -> 剧情思考 -> 码字输出 -> 反馈复盘
```

码字是输出环节，不是完整创作系统。

### 期待链与情绪

```text
作者发现只码字没进步 -> 补输入和思考 -> 输出质量提升 -> 避免闭门造车
```

真正的勤奋要能改变判断，不只是增加字数。

### 关键判断

- 不能用战术勤奋掩盖战略懒惰。
- 闷头码字不是全部努力。
- 码字只是网文输出环节。
- 看书是输入。
- 拆书是结构化吸收。
- 想剧情是战略设计。
- 闭门造车会让作者自闭。
- 字数多不等于能出头。
- 输入和思考不足会放大错误路径。
- 写作训练要形成闭环。

### 可迁移写法

```text
今日码字：
今日阅读：
今日拆书：
今日剧情思考：
发现的问题：
明日调整：
输出是否受输入影响：
```

### 训练任务

设计一周写作记录表：每天必须有输入、拆书、构思、码字、复盘五栏；只记录字数的日子视为训练不完整。

## 1839：【分享一下我的开书流程】


人工查看范围：正文、全部 11 条评论。

证据限制：这是作者个人开书流程总结，不是唯一标准；本卡只拆其中“从总纲到卷纲、节点、章纲，再到反派、配角、主角、战斗库、对白检查”的流程价值。

### 读者承诺

这篇沉淀的是开书流程要从目标层级往下拆，而不是先想零碎剧情。作者说自己 2021 年初开始写作，第一年原地踏步，扑了三本后痛定思痛，看宝剑教程后形成写书流程。流程从构思大纲开始：只确认题材、卷数和字数范围，具体剧情先不想；根据分卷数构思每卷上下行，第一卷建议上行；先设计第一卷和最终卷方向，确定单卷目标和最终目标；再设计第一卷上下行波动顺序；根据上下行构思每个节点目标；再把段落拆成起因、阻碍、目标；最后根据具体目标细分每章章纲，要求每章都与段落目标有直接影响或关联。

后半部分是角色和执行层。完成第一章后，用开篇 18 个自查点自审，前 500 字不能吸引读者就建议修改或重写；第一章完成后，要回头核对段落目标、卷纲目标、总纲目标。节点完成后再设计反派，顺序是最终 boss、重要节点 boss、初期 boss、第一个敌人，反派必须围绕阻碍主角达成目标或致主角于死地。配角则按影响大节点、小节点、单章目标的顺序设计，再从中深挖主配或女主，女主必须对主角有深远影响。之后才设计主角和配角性格，建立互补或共鸣；战斗形式列成表格库，避免重复；对白检查重点不是直不直白，而是是否符合人物性格、当下环境，以及有没有用。

### 核心机制

```text
题材/卷数 -> 卷级上下行 -> 节点目标 -> 段落起因阻碍目标 -> 章纲 -> 反派/配角/战斗/对白
```

开书流程的关键是先定目标层级，再让人物和战斗都服务目标。

### 期待链与情绪

```text
作者先确定最终方向 -> 第一卷建立上行 -> 节点逐步推进 -> 每章都影响目标 -> 读者感到主线稳定
```

读者不会看见流程，但会感到每章都在推动事情。

### 关键判断

- 开书先确认题材、卷数、字数范围。
- 具体剧情不要过早乱想。
- 每卷要有上下行。
- 第一卷建议上行。
- 第一卷方向和最终卷方向要先确定。
- 节点目标要从卷级上下行拆出。
- 段落要有起因、阻碍、目标。
- 每章要和段落目标直接相关。
- 第一章要用开篇自查点反复审。
- 反派、配角、女主、战斗、对白都要服务目标。

### 可迁移写法

```text
题材：
卷数/字数：
最终目标：
第一卷目标：
第一卷上下行：
节点目标：
段落起因/阻碍/目标：
单章影响：
反派阻碍方式：
配角作用：
战斗库变化：
对白功能：
```

### 训练任务

按这个流程为一本新书写一页开书表：只允许先写目标、上下行、节点和段落目标，最后再写反派、配角和战斗形式。

## 1840：#思考分析


人工查看范围：正文、一张图片、全部 3 条评论。图片已下载到 `_source_cache\batch-368\1840-outsider-advice.jpg` 并人工查看。

证据限制：图片是知乎视频回答截图，标题为“网文新手去哪个网站？”以及“新手写网文第一戒律--不要相信外行，不要信亲...”，本卡只拆知识库正文、图片可见文字和评论中的“外行建议过滤”判断。

### 读者承诺

这篇沉淀的是新人写网文必须过滤外行意见，尤其是亲近的人不等于专业的人。正文说，写网文不要相信外行人，包括亲近的人、亲戚、朋友、同学、兄弟、好友，只要不是从事网文相关行业的，就不要听信。图片对应知乎问题“网文新手去哪个网站？”，回答者是流浪的蛤蟆，视频标题可见“新手写网文第一戒律--不要相信外行，不要信亲...”，视频字幕也能看到“他们如果不是从事我们相关行业的”。

评论补足了判断标准：隔行如隔山，看教程也是一样，所以宝剑老师会在每个视频开头贴成绩，没有成绩没有发言权。亲戚看到你写网文，可能只会说“都多大了，还写这些天话胡话的东西”。这不是说任何非作者反馈都没价值，而是要区分情绪支持、普通读者反馈和创作决策建议。题材选择、平台选择、开局写法、商业节奏这类高影响决策，应优先看有成绩、懂平台、懂类型的人；亲友的否定或热情都不能直接当创作依据。

### 核心机制

```text
外行意见 -> 来源资格检查 -> 区分情绪/专业建议 -> 只采纳有行业证据的创作决策
```

亲近不等于懂行，热心也不等于有效。

### 期待链与情绪

```text
新人听到亲友评价 -> 容易动摇 -> 检查对方是否懂行业 -> 回到专业反馈和市场证据
```

过滤外行意见能减少不必要的心态消耗。

### 关键判断

- 写网文不要轻信外行。
- 亲近的人也可能完全不懂行业。
- 隔行如隔山。
- 看教程也要看讲者成绩。
- 没有成绩，发言权要打折。
- 亲友常把网文看成胡话或不务正业。
- 外行可以提供情绪支持，但不适合决定创作策略。
- 平台、题材、开局等决策要看行业内反馈。
- 专业建议要有成绩、案例或平台经验支撑。
- 过滤意见能保护作者心态和方向。

### 可迁移写法

```text
收到的建议：
建议来源：
是否行业相关：
是否有成绩/案例：
属于情绪反馈还是专业反馈：
可采纳部分：
应忽略部分：
下一步验证方式：
```

### 训练任务

整理最近收到的 10 条写作建议，按“外行情绪反馈、读者体验反馈、专业创作建议”三类归档，只允许第三类直接影响开书决策。
