## 591：#公开处刑

人工查看范围：正文、全部 4 条评论、附件状态。附件 `被女帝流放后，我与失落的姬太子相遇了.docx` 下载返回 401，未能查看正文附件。无图片。

证据限制：无法查看投稿正文附件，本卡依据发帖者描述和评论诊断拆解。

### 读者承诺

这篇诊断系统流玄幻开篇的关键缺口：主角被污蔑、流放、吊打敌人，本来可以形成不公平伤害和反击爽点，但系统只像任务发布器，没有讲清奖励、赌注和风险。读者看不到主角为什么听系统，也看不到系统如何让他变强，开篇又长时间下行，导致牛逼行为和阅读体感脱节。

### 核心机制

```text
主角被污蔑 -> 系统发布任务 -> 未明示奖励/风险 -> 主角照做但动机弱 -> 情节持续下行 -> 读者难受 -> 应改为谷底起跑后快速上行
```

### 期待链与情绪

```text
读者期待不公平后的反击 -> 需要系统奖励支撑反击可信度 -> 需要赌注说明主角为何服从 -> 若下行写太细 -> 期待转为憋闷
```

### 关键判断

- 系统金手指不能只安排任务，必须点明奖励如何让主角变强。
- 主角为什么听系统，要靠赌注、风险或现实困境解释。
- 如果没有奖励和风险，系统会退化成“主线任务发布器”。
- 主角表现强，但情节一直下行，读者体感会难受。
- 开篇可以站在谷底开始上行，不要边放狠话边杀人边继续被流放。
- 下行阶段要压缩，尤其不能在开头写得过细。

### 可迁移写法

系统任务开篇检查表：

```text
当前困境：
系统第一次任务：
任务奖励：
奖励如何改变主角处境：
拒绝任务的代价：
接受任务的风险：
第一次反击对象：
下行压缩位置：
上行启动点：
```

### 训练任务

写一个系统流开篇前 1200 字：先让主角处在明确谷底，再给出系统任务、奖励和拒绝代价，最后让主角用奖励完成一次小反击。下行铺垫不得超过全文三分之一。

## 592：这个刊物有大佬看过吗？特好奇里...

人工查看范围：正文、图片 1 张。无评论，无附件。

图片证据：图片为起点读书界面截图，书架中可见多册《网络文学评论》，旁边有《术师手册》《解忧杂货店》等书，发帖者询问该刊物内容。

### 读者承诺

这篇不是具体作品诊断，而是资料入口意识：作者看到《网络文学评论》这类外部刊物，会自然产生“里面刊什么、能不能用于学习网文”的疑问。它提醒系统不要只收作品案例，也要收纳行业观察、理论刊物、榜单样本和作者访谈等外部资料，并把资料转成可执行问题，而不是堆书名。

### 核心机制

```text
看到外部刊物 -> 产生内容好奇 -> 判断是否和网文创作相关 -> 提取栏目/样本/理论 -> 转成拆书或训练问题
```

### 期待链与情绪

```text
作者缺少资料源 -> 偶遇刊物 -> 好奇其内容价值 -> 若能转化为问题清单 -> 资料入口变成学习入口
```

### 关键判断

- 外部刊物本身不是答案，必须先判断它提供的是理论、评论、行业观察还是作品样本。
- 对新人来说，资料入口过多会分散注意力，必须绑定当前训练问题。
- 《网络文学评论》这类刊物适合用来补充宏观观察，但不能替代扫榜和正文拆解。
- 有价值的资料应转化为“可验证写法”：标题、栏目、样本、观点、对应作品、可训练动作。
- 资料收集的终点不是收藏，而是进入创作、诊断或复盘流程。

### 可迁移写法

外部资料入口转化表：

```text
资料名称：
资料类型：
它讨论的对象：
可提取观点：
对应作品样本：
能解决的当前问题：
需要验证的正文证据：
可转成的训练任务：
```

### 训练任务

任选一份网文相关外部资料，只摘一个观点。用一本正在追读的作品验证这个观点是否成立，并写出一个能直接改稿或仿写的小任务。

## 593：说个坏消息，我谋划了挺长一段时...

人工查看范围：正文、全部 11 条评论、图片 1 张。无附件。

图片证据：图片为表达发蒙、挫败状态的表情包，与正文“斗破同人寄了”的情绪一致。

### 读者承诺

这篇处理同人长篇开题前的风险评估：同人不是“借原作流量就简单”，它同时受原作设定、人设、读者记忆、市场热度和长篇续航约束。开头容易写，但写到百万字需要故事能力、参照作品和题材热度共同支撑；只凭兴趣或只凭热度都会踩坑。

### 核心机制

```text
想写热门原作同人 -> 做大纲 -> 发现设定/人设约束 -> 缺少市场参照 -> 编辑反馈热度不足 -> 转向扫榜和风险收益判断
```

### 期待链与情绪

```text
作者期待借同人启动 -> 大纲暴露能力缺口 -> 市场反馈打击信心 -> 评论给出扫榜/换题材路径 -> 情绪从发蒙转向重新判断风险
```

### 关键判断

- 同人开头好写，长篇一百万字以上很难，根因仍是讲故事能力。
- 原创中能轻松解决的问题，在同人里可能因原作设定和读者记忆变成毒点。
- 斗破同人缺少当前市场参照，编辑建议若为赚钱可看斗罗。
- 原神同人流量大，但如果作者没玩明白原作，只查人物设定就写，容易留不住读者并招骂。
- 星主建议点开排行榜，找火题材直接研究；奇幻、悬疑不卷但流量小，风险和收益并存。
- 作者最终意识到自己仍想写竞争最激烈的方向，说明题材选择要同时面对兴趣和风险。
- 评论提醒可回看同人拆书帖，尤其火影同人拆解，说明同人写作需要专项样本研究。

### 可迁移写法

同人长篇开题风控表：

```text
原作名称：
当前平台热度：
榜单可参照作品：
原作核心爽点：
不能违背的人设/设定：
原创写法迁入后的毒点：
开头卖点：
百万字续航主线：
作者熟悉度：
风险收益判断：
```

### 训练任务

选一个想写的同人题材，先不写正文。列 3 本同平台参照作品，拆每本前 3 章的原作记忆点、主角切入点和长线主线，再判断自己是否能写到 30 万字不断档。

## 594：提问：主角展示自己的金手指，这...

人工查看范围：正文、全部 1 条评论。无附件，无图片。

### 读者承诺

这篇给出金手指首秀的最小闭环：不用纠结是另造小矛盾，还是等第一个主要矛盾解决，只要让读者看到“主角发现或遭遇麻烦，并主动使用金手指解决麻烦”。首秀的重点不是解释完整设定，而是让读者快速确认金手指能带来行动优势。

### 核心机制

```text
主角发现金手指 -> 眼前出现麻烦 -> 主角主动使用 -> 麻烦被解决 -> 金手指完成第一次展示
```

### 期待链与情绪

```text
读者知道主角有外挂 -> 等待它如何有用 -> 眼前麻烦制造验证场 -> 主角主动使用 -> 期待后续更大用法
```

### 关键判断

- 常见顺序是：发现有金手指、碰到当前麻烦、主动用金手指解决。
- “发现”和“碰到麻烦”的顺序可以互换。
- 如果第一个主要矛盾要六七千字才解决，中途应至少完成一次金手指有效性展示。
- 小矛盾不是必须另造，关键是麻烦要近、选择要主动、结果要清楚。
- 金手指首秀应服务第一个单元故事，而不是脱离主线做孤立说明。

### 可迁移写法

金手指首秀三步表：

```text
金手指名称：
主角何时发现：
眼前麻烦：
主角主动选择：
使用方式：
即时结果：
读者学会的规则：
下一次升级期待：
```

### 训练任务

把你的开篇前 3000 字标出来：金手指出现在哪一段，麻烦出现在哪一段，主角主动使用在哪一段。如果三项缺一项，补一个不超过 600 字的首秀桥段。

## 595：宝剑大我有个问题

人工查看范围：正文、全部 4 条评论。无附件，无图片。

### 读者承诺

这篇把爽点节奏和抽奖金手指放在一起看：三章小爽、十章大爽可以嵌入三幕式，但抽奖金手指若只是“主角越强，抽到的东西越好”，很快会失去吸引力。真正能撑长线的不是数值升级，而是奖励类型、使用场景、解谜条件和剧情功能发生质变。

### 核心机制

```text
阶段性爽点节奏 -> 抽奖提供新奇奖励 -> 奖励若只数值变强 -> 新鲜感衰减 -> 条件和奖励同步质变 -> 形成长线期待
```

### 期待链与情绪

```text
读者期待下一次抽奖 -> 随机性制造好奇 -> 无用物品可变伏笔/交易/笑点 -> 若奖励没有本质变化 -> 期待断档
```

### 关键判断

- 三章小爽、十章大爽可以放进三幕式，不是孤立机械节奏。
- 抽奖金手指常见问题是缺少新奇成长性。
- 只随着实力提升抽得越来越好，本质仍是常规升级。
- 很多类似设定不会把抽奖当主金手指，而是作为附带彩蛋。
- 特殊升级条件可以增加引导性，但奖励也要跟着发生本质变化。
- 随机物品可以用于伏笔、交易和趣味剧情，但必须在可控范围内服务主线。

### 可迁移写法

抽奖金手指质变表：

```text
初期抽奖功能：
初期奖励类型：
中期新增条件：
中期奖励质变：
后期核心玩法：
无用物品回收方式：
和主线矛盾的关系：
每次抽奖后的新期待：
失控随机性的限制：
```

### 训练任务

设计一个抽奖金手指三阶段版本：第一阶段解决眼前麻烦，第二阶段改变主角资源关系，第三阶段改变主线问题的解法。每阶段写 3 个奖励，不能只是数值变大。
