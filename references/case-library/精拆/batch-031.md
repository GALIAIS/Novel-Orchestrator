## 151：#思考分析 关于我自己扒文的一...

人工查看范围：正文、全部 2 条评论。重点查看扒文从混乱期、有目的零散积累到分阶段精扒的三阶段变化，以及“先总后分、全文逆推、按问题分批次收集”的方法。

### 读者承诺

这篇是拆书能力成长路线。新手一开始会觉得处处都是重点，扒得太细、太慢、太碎，最后半途而废；进阶后要先确定目的和主次，先拉全文框架，再按伏笔、配角、情绪、市场性等单项分层扒。拆书不是复制文本，而是为了把别人的处理方式转成自己能验证、能应用的方法。

### 核心机制

```text
混乱全扒 -> 问题驱动 -> 选同方向样本 -> 先总后分 -> 分层精扒 -> 用自己作品验证
```

### 期待链与情绪

```text
作者不知道怎么学 -> 先模仿扒法 -> 发现效率低 -> 改成带问题取证 -> 再升级成全文框架逆推 -> 形成个人 SOP
```

### 关键判断

- 第一阶段是混乱期，目之所及全是重点。
- 新手容易没有鉴别力，龙空、碧水、网站方法都觉得对。
- 一章扒几千字会导致工程量过大，只扒开头或几十章就半途而废。
- 混乱期虽然低效，但能解决一些具体问题，建立扒文概念。
- 第二阶段开始围绕“发现问题、寻找案例”来扒。
- 问题驱动的扒法效率高，但容易忽略整体框架。
- 第三阶段选文应选择自己喜欢且对应接下来写作方向的作品。
- 精扒要先确定目的主次和作品值得学习的点，列出排序。
- 第一遍先看完、总结、拉框架结构、分析结论，再用自己的文验证。
- 第二遍按问题要点分批收集，如伏笔、布局、处理方式。
- 第三遍可单拉配角，看为什么打动自己、如何塑造。
- 拉多少遍取决于需求和时间安排，像音轨或 PS 图层一样分层处理。

### 可迁移写法

分层扒文 SOP 表：

```text
本次扒文目的：
样本是否对口：
第一遍全文框架：
最值得学的 3 个点：
第二遍专项一：
第三遍专项二：
对应自己作品的问题：
可迁移结论：
验证方式：
是否需要继续下一层：
```

### 训练任务

选一本同方向作品，不许从第一章逐句全扒。先写 500 字全文框架，再列 3 个最想解决的问题，每次只围绕一个问题二刷取证。

## 152：我发现有些作者为了不写主角初次...

人工查看范围：正文、全部 18 条评论、两张图片。图片为同人开篇截图，重点显示主角穿越多年、仍难适应杀戮忍者世界，经历死亡战争后对生命麻木，并选择投靠大蛇丸等内容。

### 读者承诺

这篇延续人物成长问题，但焦点是“跳过成长”的代价。把开篇时间调到主角已经穿越多年，能避开初次杀人难写的问题，但也容易让读者觉得缺了一块，因为人物关键转变没有被看见。跳过不是不能用，关键是必须迅速拉出爽点、明确当前动机，并避免让主角出现“主动投靠敌人”“主动认怂”这类即时毒点。

### 核心机制

```text
跳过初次成长 -> 代入缺口 -> 当前关键节点 -> 立刻补动机/爽点 -> 避免主动认怂毒点
```

### 期待链与情绪

```text
读者看到主角已适应异界 -> 想知道他如何变成这样 -> 作品没有展示成长 -> 产生缺失感 -> 若当前爽点足够强可被带走 -> 若毒点先出现则流失
```

### 关键判断

- 把时间调到重大转变节点是可行的，但会跳过人物成长阶段。
- 这种写法本质是取巧圆设定，属于扬长避短。
- 若作者不擅长写初次杀人，跳过可能是当下最优解。
- 跳过成长会少一次折线和成长，代入感会降低。
- 跳过后必须迅速拉出爽点，不能只是为了跳过而跳过。
- 主动投靠敌人、主动认怂、主动投降这类表达容易成为毒点。
- 如果实际是潜入或把敌人当工具人，应在前文明确，不能让读者误读太久。
- 所有贬义词和行动可以加反话标记，强调这是表面“投靠”。
- 初次杀人反应不必写很长，十个字的作呕也能完成过渡。
- 同人类型多样：填补意难平、无限流、种田抢先机、YY 抢机缘等，主角功能和读者期待不同。
- 读者看太多“死亡考验”，新人写不好会流失，因此跳过有风险也有现实合理性。

### 可迁移写法

跳过成长风险表：

```text
跳过了哪段成长：
当前切入节点：
读者会缺哪块信息：
是否立刻给当前爽点：
主角当前动机：
是否出现主动认怂/投靠：
是否能改成潜入/利用：
需要补的十字反应：
后续是否回收成长空白：
```

### 训练任务

写两个版本的同一开篇：A 版写初次杀人 300 字过渡，B 版跳到穿越多年后。B 版必须在前 500 字内给出当前爽点和明确动机，避免读者觉得缺了一块。

## 153：#思考分析

人工查看范围：正文、全部 10 条评论、三张图片。图片为刺猬猫明日方舟同人《被踹出罗德岛，龙门桥洞盖小被儿》页面与正文截图，显示大段行文、台词后接描写、月票/推荐/打赏数据和同人标签。

### 读者承诺

这篇提醒不要把“网文行文规则”当成绝对铁律。段落长短、台词和描写组合方式，都要看平台、受众、题材和同人信仰加成。对同人文来说，原作受众、梗、角色熟悉度和平台阅读习惯可能比所谓“小段落规则”更重要。能把故事讲顺，才是第一位。

### 核心机制

```text
平台受众 -> 原作信仰/梗 -> 行文习惯 -> 故事流畅 -> 成绩成立
```

### 期待链与情绪

```text
作者看到不合常规的行文 -> 疑惑为什么成绩好 -> 发现平台和同人受众不同 -> 回到故事和受众适配 -> 放下硬套规则
```

### 关键判断

- “网文每段不能超过一百字”不是绝对规则。
- 刺猬猫读者可能更习惯某些大段阅读和同人文风。
- 台词后接一段描写并非天然错误，只要读起来顺、故事能推进。
- 对网文来说最重要的是故事，能讲出故事的方式就是好方式。
- 学自己看得懂的文，看不懂的不要硬懂。
- 读者受众千奇百怪，彼之毒草我之仙草。
- 明日方舟、崩坏三、原神等同人可能有信仰加成，外行很难用普通文笔/剧情标准理解。
- 同人风格跟着原作走没问题。
- 会玩梗是同人成绩的重要因素。
- 狼人杀等特殊题材读者能接受高密度发言和大豆腐块，说明阅读负担和受众训练有关。
- 不要因为别人某种行文有效，就机械效仿；先判断自己平台和受众是否一致。

### 可迁移写法

行文规则适配表：

```text
目标平台：
目标受众：
是否同人：
原作文风/梗密度：
读者是否习惯大段：
故事是否流畅：
当前行文规则来自哪里：
是否适合自己作品：
不能硬学的点：
```

### 训练任务

找一本你看不懂但成绩好的同人文，不评价好坏，只记录平台、原作受众、梗密度、段落习惯和你不能照搬的原因。

## 154：#公开处刑

人工查看范围：正文、全部 4 条评论、附件状态。附件 `六边形爆星强者.docx` 知识库只保留文件链接和 id，尝试下载返回 401；本卡依据正文和可见评论精拆。

证据限制：docx 附件无法下载，未能查看前五章全文；评论里包含对结构、书名、金手指、身份压迫、胎中之谜和转场的具体反馈。

### 读者承诺

这篇是无限流开篇修复样本：结构可以没大毛病，但如果题材和金手指都太标准，就必须靠书名筛选、概念解释、身份压迫回收和场景转场来提升吸引力。读者不怕套路，怕的是套路没有亮点、信息没解释、期待被轻飘飘放过。

### 核心机制

```text
标准题材框架 -> 书名筛选读者 -> 金手指明确 -> 压迫制造期待 -> 支线承诺回收 -> 场景转场装逼
```

### 期待链与情绪

```text
读者看到无限流/综漫/加点 -> 判断是否对口 -> 看到主角被压 -> 等待反差爽点 -> 身世支线形成期待 -> 转场装逼兑现
```

### 关键判断

- “结构没大毛病”不等于吸引人。
- 标准综漫流加深蓝加点金手指稳定，但缺乏出彩创新点。
- 书名要简单粗暴筛选读者，如题材加金手指形式。
- “胎中之谜”如果实际是记忆屏蔽、受刺激解开，就应该尽早说明。
- 给主角“娼妓之子”身份，会制造母亲亲情线期待。
- 如果母亲透明，不如改成孤儿、野种、没人要的废物，避免支线承诺落空。
- 身世压迫可以用于反差爽点，但必须后续回收。
- 胎中之谜解开的转场不能生硬切分隔线。
- 更好的做法是让敌人压近、主角意识模糊、介绍身世，再回到敌人下刀被主角挡下。
- 转场应通过同一场景延续视角，压迫不断，装逼才顺。

### 可迁移写法

无限流开篇吸引力表：

```text
题材标签：
金手指一句话：
书名是否筛选读者：
标准套路里唯一亮点：
需要解释的专有词：
主角身份压迫：
身份是否带支线承诺：
转场是否同场景延续：
第一章反差兑现：
```

### 训练任务

重写一个标准无限流开篇的前 800 字：书名用“题材+金手指”筛选读者，第一章解释核心专有词，并把身份压迫和金手指觉醒放在同一场景里完成。

## 155：转一篇知乎的《我不是药神》的编剧改变思路文章

人工查看范围：知识库正文、外链标题与可检索摘要。知识库正文说明作者转发知乎《原著改编剧本分析 11《我不是药神》》，关注《救猫咪》十个故事类型和《我不是药神》前半段借鉴好莱坞金羊毛类型思路。

证据限制：知识库正文未保存知乎文章全文，知乎专栏页当前未能直接打开完整内容；本卡只依据知识库可见转述、链接标题、可检索摘要和《我不是药神》公开剧情信息做有限吸收。

### 读者承诺

这篇可吸收的是“现实题材也能用类型片骨架”。《我不是药神》前半段借鉴金羊毛，不是把现实题材写成冒险游戏，而是让主角因现实压力踏上取药/卖药之旅，途中组队、过关、获利、扩大影响，再由道德代价和社会议题把故事推向更重的后半段。类型结构服务现实情绪，不替代现实困境。

### 核心机制

```text
现实困境 -> 金羊毛任务 -> 组队取物/卖药 -> 获利和危机并行 -> 道德代价升级 -> 主题转向
```

### 期待链与情绪

```text
主角缺钱且被现实压迫 -> 病人提出机会 -> 主角踏上非法但有收益的旅程 -> 队伍扩大 -> 观众看到希望 -> 代价逼近 -> 从赚钱故事转成救命故事
```

### 关键判断

- 《救猫咪》的十个故事类型可以作为商业故事骨架，但不能生搬硬套。
- 《我不是药神》前半段可理解为金羊毛：主角为了现实目标踏上取药/卖药之路。
- 金羊毛的“宝物”不是奇幻道具，而是低价仿制药、代理权、病人渠道和解决经济困境的机会。
- 主角程勇先有现实压力：店铺冷清、父亲手术费、抚养权等。
- 吕受益登门提出药源需求，是旅程召唤。
- 刘牧师、刘思慧、彭浩等人加入，形成队伍。
- 前半段的商业成功提供上行情绪，但违法风险、病人生死和医药公司压力一直存在。
- 类型骨架让故事好看，现实议题让故事变重。
- 改编现实事件时，要把真实困境转成可追的任务链。
- 现实题材不等于散文化记录，也需要明确目标、阻碍、队伍、代价和转折。

### 可迁移写法

现实题材金羊毛表：

```text
现实困境：
主角私欲目标：
金羊毛是什么：
召唤者：
旅程第一步：
队伍成员：
每个成员带来的功能：
获利上行：
违法/道德/情感代价：
主题转向节点：
```

### 训练任务

把一个现实题材故事改成金羊毛骨架：主角必须有现实私欲目标，旅程中获得一个具体“宝物/资源”，但后半段要让道德代价反过来改变主角目标。
