## 226：我决定了，以后我专攻二次元

人工查看范围：正文、全部 7 条评论。重点查看作者想转向二次元/轻小说，以及评论区对“概念先分清、扫榜拆书、平台差异、人设和轻松文风”的建议。

### 读者承诺

这篇的核心不是立刻决定“专攻二次元”，而是先分清二次元、轻小说、同人和平台频道的差异。轻小说是个框，不同网站装的东西不同；二次元常见受众偏青少年，重视新鲜、脑洞、轻松、有趣、人设和语言里的独特思维方式。

### 核心机制

```text
概念混淆 -> 平台扫榜 -> 识别受众 -> 人设优先 -> 轻松脑洞表达
```

### 期待链与情绪

```text
作者想转型 -> 发现频道名称不同 -> 评论区要求先搞清概念 -> 扫榜同人/轻小说 -> 建立目标受众和写法
```

### 关键判断

- 不要在概念没搞清时决定长期专攻。
- 二次元、轻小说、同人不是完全同一件事。
- 轻小说是个框，什么都能往里装。
- 不同网站的轻小说频道规则不同。
- 学习资料不如当前扫榜拆书直接。
- 当时热度样本包括原神、火影、海贼和国漫同人。
- 轻小说常见要求是人设优先。
- 文风轻松更适配二次元/轻小说受众。
- 二次元受众较多为青少年，要求新鲜、脑洞清奇、轻松有趣。
- 二次元很多时候要拐一个弯思考，靠语言体现人物与众不同。
- 青少年读者想象力强、发散性强，也容易钻牛角尖。
- 平台选择要和题材、IP、同人热度、目标年龄层一起判断。

### 可迁移写法

二次元轻小说定位表：

```text
目标平台：
频道名称：
是原创/同人：
目标受众年龄层：
当前热度 IP/题材：
核心人设：
轻松感来源：
脑洞拐弯点：
前三章必须展示：
```

### 训练任务

选一个二次元/轻小说榜单样本，拆出：人设钩子、轻松语气、脑洞点、受众年龄层和第一章承诺。

## 227：#思考分析

人工查看范围：正文、全部 15 条评论、1 张图片。图片为写作状态表情包；评论重点围绕晋江签约年限、未成年签约、监护人身份证、签人/签作品区别、纯爱收紧和合同风险。

### 读者承诺

这篇是新人把“暑假开写”和“签约赚钱”混在一起时必须先补的平台合同认知。准备人设、大纲、章纲之外，还要准备平台规则、签约门槛、未成年签约方式、合同周期、版权归属和投稿风险。目标是赚钱时，合同和平台适配本身就是创作准备的一部分。

### 核心机制

```text
写作计划 -> 平台选择 -> 签约规则 -> 合同风险 -> 再决定投稿路径
```

### 期待链与情绪

```text
新人想暑假写文赚钱 -> 想投晋江 -> 评论提醒长约/签人/未成年风险 -> 先查规则和合同 -> 再决定是否继续
```

### 关键判断

- 准备开文不只准备人设、大纲、章纲。
- 以稿费到账为目标时，平台规则必须提前查。
- 晋江被评论区描述为签约周期长、签人风险高。
- 签人的网站要比签作品的网站更谨慎。
- 未成年签约通常需要监护人同意和身份证信息。
- 不同平台未成年规则不同，应看官网签约规则。
- 若拿父母身份证签，成年后再用自己身份证可能有风险。
- 合同最好在签前找专业人士看。
- 纯爱/耽美平台选择受政策和平台收紧影响。
- 如果不是必须写耽美，女频风格也可考虑其他平台。
- 投稿前要先了解平台热门类型和优缺点。
- 新人不必急于签，先写到可签再问编辑也可以。

### 可迁移写法

平台签约准备表：

```text
目标平台：
目标类型：
是否签人/签作品：
合同周期：
最低更新/字数要求：
未成年签约规则：
版权/代理范围：
是否需要监护人：
是否需要法律审核：
替代平台：
```

### 训练任务

为一个目标平台做签约风险清单：至少查清签约主体、合同年限、版权范围、未成年规则和退路平台。

## 228：#思考分析 #推书试读/影视/...

人工查看范围：正文、全部 14 条评论、2 张图片。两张图片为作者自写开头与上架书开头对照；人工查看后可见两者都是长图文本样张，评论区据此判断左图青涩、右图成熟些但叙述仍散。

### 读者承诺

这篇围绕“自己的文风适合哪个网站”做样本文风诊断。评论区判断：文笔够用，但情节设置和情绪调动差距大；左图比较青涩，适合轻之文库或 SF 轻小说这类青少年向平台，右图更适合刺猬猫；两者共同问题是叙述散、开头目标不明确。

### 核心机制

```text
文本样张 -> 文风诊断 -> 受众年龄判断 -> 平台匹配 -> 补剧情和情绪短板
```

### 期待链与情绪

```text
作者拿开头求平台定位 -> 评论区看文风和冲突 -> 判断青少年向/同人向平台 -> 指出目标散和冲突升级问题 -> 回到剧情训练
```

### 关键判断

- 文风适合哪个网站，不能只听自我感觉，要拿文本样张判断。
- 文笔够用不等于能签约。
- 情节设置和情绪调动比单纯文笔更决定开篇有效性。
- 原神同人适合刺猬猫。
- 轻小说/同人可考虑刺猬猫、菠萝包等平台。
- 一开口就是老二次元时，平台受众要跟着选。
- 游戏同人有 IP 加成，剔除加成后要看开篇结构是否成立。
- 人设后立刻接冲突或金手指，格式可以是标准的。
- 一大一小两个冲突会圈定受众，可能偏学生党和自尊心强的读者。
- 成年读者可能对学生式冲突兴趣不大。
- 大妈下跪这类冲突升级过快，容易尴尬。
- 开局让主角任性有风险，除非精准符合目标受众口味。
- 可以先让主角理论，再发现对方不可理喻，再反击。
- 作者本人是青少年时，写给成熟读者会明显吃阅历亏。
- 平台匹配不能替代剧情能力训练。

### 可迁移写法

文风平台匹配表：

```text

文笔水平：
情节设置：
情绪调动：
目标受众年龄：
IP/同人加成：

冲突升级是否过快：
下一步训练：
```

### 训练任务

## 229：记录：通过学习，我把两门知识链...

人工查看范围：正文、全部 4 条评论。重点查看作者把画画的起型、修型、定型，与小说的框架、节奏、细化建立类比，以及评论区对框架稳定均订的补充。

### 读者承诺

这篇的核心是跨领域学习：画画不能沉迷局部，要先起型、修型、定型，整体塑造；小说也不能只执着某个好剧情，要先有框架、节奏、细化。对于非高手，大纲就像美术起型，是提前暴露问题、避免后期崩盘的规划工具。

### 核心机制

```text
急躁执着 -> 跨领域类比 -> 框架起型 -> 节奏修型 -> 细化定型 -> 稳定产出
```

### 期待链与情绪

```text
作者反复改局部却失败 -> 发现问题在整体框架 -> 用画画方法理解小说 -> 放慢节奏打磨 -> 建立长期稳定心态
```

### 关键判断

- 欲速则不达、慢工出细活、勤能补拙是理解后才有用的原则。
- 失败常常不是脑子笨，而是目的性过强导致急躁。
- 画画执着局部会忽略整体。
- 写小说执着个别剧情会越写越崩。
- 画画要起型、修型、定型。
- 小说要框架、节奏、细化。
- 非高手没有心中成像，更需要前期规划。
- 大纲会在建立过程中客观暴露毛病。
- 框架不仅稳定中后期，也稳定每一卷发挥。
- 每卷稳定不出大错，对整体均订有重要意义。
- 写作要看成有意思的事，才能承受上下行情绪。
- 初入门者不存在秒学绝世武功，必须反复练习。
- “望山跑死马”提醒作者别只盯远处目标而忽略脚下。

### 可迁移写法

框架节奏细化表：

```text
当前作品整体框架：
第一卷起型：
节奏修型问题：
需要细化的单元：
沉迷的局部：
被忽略的整体：
提前暴露的问题：
本轮修正动作：
```

### 训练任务

把一个已有大纲按“起型-修型-定型”重做一遍：先写整体框架，再查节奏问题，最后只细化第一单元。

## 230：投稿被拒了，大家帮忙点评一下，...

人工查看范围：正文、全部 25 条评论、附件状态。附件 `我是任务发布人内投版.docx` 仅有 file id，未提供可下载链接；本卡依据正文与评论精拆。

证据限制：附件正文不可见；评论已指出书名不够 YY、第一章无关内容太多、脑洞文应直接进核心事件、扶老人事件和系统承接弱、人设跃迁崩、金手指像主神缺创新、投稿应重点描述核心卖点和爽点等关键问题。

### 读者承诺

这篇是脑洞文投稿被拒后的开篇压缩案例。评论区认为第一章可以缩到 500 字以内，甚至直接从主角进入第一个世界开始。问题不是作者没有设计，而是设计没有引导读者理解：扶老人/叫救护车事件和获得系统缺少强联系，读者会觉得系统是天上掉下来的。

### 核心机制

```text
投稿被拒 -> 书名不够 YY -> 第一章冗余 -> 核心事件后置 -> 金手指承接弱 -> 重写开篇
```

### 期待链与情绪

```text
读者看到任务发布人书名 -> 不知道怎么 YY -> 前面坐车/老人事件拖慢 -> 系统获得缺关联 -> 进入世界太晚 -> 失去期待
```

### 关键判断

- 投稿被拒先对照开头 18 元素自查。
- 书名虽然点明金手指，但要让人能 YY。
- 脑洞文开篇要直接。
- 第一章无关内容可以大幅压缩，甚至 500 字以内。
- 可以直接从主角进入第一个世界写起。
- 更高级的写法是从世界剧情切入点写起，主角正在解决小冲突。
- 前文身份和金手指由来可以用几百字补。
- 事件切入必须和后续剧情有关联。
- 扶老人/叫救护车如果不能导向系统逻辑，就会被读者遗忘。
- 设计里的规则空子没有被文本引导出来，就等于不存在。
- 主角人设不能从普通小市民突然跃迁成天骄，再突然犯蠢。
- 几千字故事不行，人设也没立住，会像记叙文。
- 金手指要服务男主装逼。
- 任务发布人设定如果像主神，需要新的差异化意义。
- 投稿说明应重点写核心卖点、核心爽点种类、同类文相比读者为什么看这本。
- 参考样本可看 NPC 给玩家发任务、玩家/天选者任务流。

### 可迁移写法

脑洞文投稿开篇压缩表：

```text
书名 YY 点：
核心金手指：
第一世界切入点：
是否可从小冲突开始：
前 500 字保留信息：
必须删除的冗余：
系统由来关联：
主角人设是否稳定：
同类差异化：
投稿卖点说明：
```

### 训练任务

把一个脑洞文开篇压缩到 500 字：只保留主角身份、金手指来源、第一世界小冲突和一个核心爽点。
