## 171：提问:

人工查看范围：正文、全部 1 条评论。重点查看“大纲怎么写、有无模板”这类新人问题的处理方式。

### 读者承诺

这篇很短，但暴露出新人训练中的常见误区：把“大纲能力”理解为找一个现成模板。评论没有直接给模板，而是要求先用搜索栏找已有材料。对系统来说，这类问题应该进入“检索已有模板 -> 选一个匹配当前题材的模板 -> 用具体项目填充 -> 再诊断缺口”的流程，而不是重新讲一遍抽象大纲论。

### 核心机制

```text
新人要模板 -> 先检索已有知识库 -> 选择题材/阶段匹配模板 -> 填入当前项目 -> 根据缺口再学习
```

### 期待链与情绪

```text
作者不知道大纲怎么写 -> 想拿模板启动 -> 被提醒资料已有 -> 从提问转为检索和执行
```

### 关键判断

- 大纲问题不能只停在“有没有模板”。
- 知识库已有模板时，第一步是检索，不是重复造轮子。
- 模板只有套到具体题材、主角目标、阶段任务里才有用。
- 新人问大纲，往往真正缺的是项目当前层级：书纲、卷纲、单元纲、章纲。
- 不同层级的大纲不能混用。
- 大纲模板应先解决“接下来写什么”，再解决“整体多宏大”。
- 如果没有当前题材和篇幅目标，模板会变成空表格。
- 搜索已有材料本身也是训练的一部分。
- 直接白嫖答案容易跳过理解；自己检索能建立知识导航。
- 系统应把这类问题路由到 curriculum 和 templates，而不是拆书取证。

### 可迁移写法

大纲模板检索表：

```text
当前问题是书纲/卷纲/单元纲/章纲：
目标题材：
当前已有设定：
要检索的关键词：
找到的模板：
模板缺的栏目：
本项目必须补的栏目：
下一步输出物：
```

### 训练任务

从知识库或系统模板里选一个大纲模板，只填一个 3000 字单元故事：主角目标、阻碍、转折、兑现、新期待必须写满，其他宏大设定先不填。

## 172：#参考资料 逛了一下我的网盘，...

人工查看范围：正文、文件列表、附件状态。附件包括《救猫咪 1 电影编剧宝典》PDF、《救猫咪 3 反击战！》PDF和一个 file id；其中第一个 PDF 链接尝试下载返回 401，未能查看正文内容。

证据限制：附件无法下载，未能阅读 PDF 原文；本卡依据正文“资料很多但只看了很少一部分”和文件列表精拆。

### 读者承诺

这篇不是讲《救猫咪》的具体技巧，而是讲资料囤积的风险：资料和写作书很多，但真正看过、消化过、迁移到写作里的很少。对新人训练来说，资料库不能变成心理安慰，必须变成可执行的阅读顺序、摘取问题和应用任务。

### 核心机制

```text
资料很多 -> 阅读很少 -> 理论不进入写作 -> 形成囤积幻觉 -> 改为任务化阅读
```

### 期待链与情绪

```text
作者发现资料丰富 -> 意识到消化不足 -> 需要从收藏转向使用 -> 每次只读为当前问题服务的章节
```

### 关键判断

- 资料多不等于能力强。
- 写作书如果只收藏不阅读，会变成拖延借口。
- 《救猫咪》这类通用编剧书要通过网文问题转译使用。
- 阅读顺序要服务当前短板，不必一次读完整套。
- 每读一章理论，都要产出一个可用于当前作品的卡片。
- 资料库需要索引：书名、适用问题、可迁移概念、训练任务。
- 新人不该同时打开十几本理论书。
- 最小阅读单位可以是一章、一节、一个概念，而不是整本。
- 没有应用任务的读书笔记，很难转化为写作能力。
- 系统应把资料整理为“问题 -> 资料 -> 练习 -> 验收”，而不是纯书单。

### 可迁移写法

资料消化卡：

```text
资料名称：
当前要解决的问题：
只读章节：
摘出的一个概念：
对应网文场景：
今天要改的一段：
训练任务：
验收标准：
```

### 训练任务

选一本写作书，只读一个章节，写 200 字概念摘要，再把它转成一个网文章节修改任务；当天必须改一段正文或一个单元小纲。

## 173：在飞卢写起点风，在起点写飞卢风...

人工查看范围：正文、全部 20 条评论、3 张图片。图片 1 为知乎观点截图，强调飞卢不一样、起点作者去飞卢写起点风可能靠更新榜和长线稳定吃到收益；图片 2 为《我的战舰能升级》详情，446 万字、1857 万阅读、51 万粉丝、系统升级、科幻网游/星际科幻；图片 3 为该书目录，展示 12 卷以上的长期规划。

### 读者承诺

这篇讨论的是平台风格迁移，不是简单“换个平台就赢”。起点吃追读，飞卢吃前期、更新榜、上架爆发和持续更新；起点风在飞卢可能靠长线稳定和不烂尾形成稀缺，飞卢风在起点可能靠爆炸开篇提高 PK，但后续必须稍微放慢并稳住长线。迁移成立的前提是知道自己要吃哪个机制，而不是把两边缺点混在一起。

### 核心机制

```text
平台机制差异 -> 风格错位机会 -> 前期指标/更新节奏适配 -> 长线不崩 -> 收益放大
```

### 期待链与情绪

```text
作者看到起点追读压力 -> 想借飞卢快节奏破局 -> 又看到飞卢烂尾普遍 -> 起点风长线稳定成为差异化
```

### 关键判断

- 起点推荐晋级看追读，开头不够强容易过不了 PK。
- 飞卢前期曝光和更新榜机制不同，长线稳住可能吃到稀缺红利。
- 飞卢大量作品竭泽而渔、前期猛、后期崩，不烂尾本身能形成差异。
- 《我的战舰能升级》被作为飞卢写起点风的样本：首订低，但长期写到数百万字后收益稳。
- 图片显示该书有 12 卷以上目录规划，说明长线不是嘴上说说。
- 起点写飞卢风可以借爆炸开头，但不能全文照搬高耗竭节奏。
- 飞卢风长干容易血崩，必须降速、补长线结构。
- 飞卢并非必须日更两万；但新人没有存稿和判断力时，更新压力很大。
- 上架爆更、后续两三章吊住，是飞卢长写的一种执行方案。
- 起点大神日更也不低，不能把起点理解为低更新避难所。
- 跨平台写法要看运气和风向，但笔力及格是底线。
- 迁移不是“在 A 平台写 B 风”，而是“拿 B 风优势补 A 平台短板”。

### 可迁移写法

平台迁移判断表：

```text
目标平台：
核心指标：
原风格优势：
原风格劣势：
迁移后吃到的机制：
必须补的短板：
更新计划：
存稿量：
长线卷纲：
首订/追读观察点：
```

### 训练任务

做一份跨平台开篇方案：同一个题材分别写“起点版前三章”和“飞卢版前三章”的节奏差异，并说明后续 30 万字如何避免血崩。

## 174：有没有大佬总结过都市职业文的写...

人工查看范围：正文、全部 6 条评论。重点查看都市职业文的学习方式、行业经验门槛、副业写作者优势和《大医凌然》样本推荐。

### 读者承诺

都市职业文的承诺是“专业能力进入事件后产生爽感”。这类文不能只靠泛泛套路，作者最好足够了解行业，或者具备能快速拆行业流程、术语、职业伦理、晋升体系和真实冲突的能力。法医、警察、医生、道士、写小说、视频、直播等副业作者天然有素材优势。

### 核心机制

```text
行业认知 -> 职业流程 -> 专业问题 -> 主角能力解决 -> 外行看爽/内行不出戏
```

### 期待链与情绪

```text
读者想看职业新鲜感 -> 专业难题出现 -> 主角用职业能力破局 -> 获得认可/资源/地位 -> 下一个更难职业事件
```

### 关键判断

- 都市职业文一般需要足够了解行业。
- 自己拆同类作品比白嫖总结更有效。
- 副业写作者天然适合职业文，因为手里有真实细节。
- 法医、警察、医生、道士、作者、视频、直播等职业都能转化为事件源。
- 小众职业也能混小众头部，前提是专业感和故事性都够。
- 过气题材如黑客文，也可能靠职业真实和小众头部生存。
- 《大医凌然》可作为医生职业文样本。
- 职业文不是科普文，行业细节必须进入冲突和收益。
- 专业门槛越高，越要用读者能懂的任务呈现。
- 主角能力不能只靠作者说专业，要在一次次事件中证明。

### 可迁移写法

都市职业事件表：

```text
职业：
真实行业流程：
读者能懂的职业难题：
主角专业优势：
外行误判：
内行认可点：
解决后的收益：
下一次更高难度事件：
```

### 训练任务

选一个自己熟悉或能采访到的职业，写 5 个职业事件：每个事件必须包含专业难题、主角操作、外行误判和结果收益。

## 175：各位大佬们我想问一下，

人工查看范围：正文、全部 2 条评论。重点查看飞卢鲜花、月票、评价票、V 收与首订判断的关系。

### 读者承诺

这篇给的是飞卢数据判断的底线：鲜花、月票、评价票还可以，但 V 收少，不能只靠免费期互动数据决定生死。飞卢上架成本低，真正判断是否继续写，最直接的是上架看首订。对作者来说，这类问题要从焦虑转向数据节点：什么时候上架、首订多少、成本多少、存稿多少、是否值得继续。

### 核心机制

```text
免费期互动不错 -> V 收少产生焦虑 -> 飞卢上架成本低 -> 看首订验证 -> 决定继续/止损
```

### 期待链与情绪

```text
作者看见鲜花月票 -> 以为还有希望 -> V 收少又担心 -> 上架首订给出真实付费反馈
```

### 关键判断

- 飞卢免费期数据不能等同于付费能力。
- 鲜花、月票、评价票不错，只说明有人互动，不代表 V 章会买。
- V 收少是风险信号，但不是最终判决。
- 飞卢上架成本低，可以用首订做验证。
- 与其长期纠结，不如明确上架节点和观察指标。
- 首订是付费读者真实反馈，比免费互动更硬。
- 数据判断要结合更新成本、存稿、题材预期和个人承受能力。
- 如果首订差且后续没有明显改法，应及时止损。
- 如果首订尚可，可以按既定存稿和更新计划继续试。
- 平台数据问题应形成“观察表”，不要只凭情绪问能不能写。

### 可迁移写法

飞卢上架判断表：

```text
免费字数：
鲜花/月票/评价票：
V 收：
预计上架字数：
存稿量：
首订目标：
最低可接受首订：
更新成本：
继续条件：
止损条件：
```

### 训练任务

给一篇飞卢新书做上架观察表：列出免费期数据、预计首订、最低止损线和上架后三天更新安排；不允许只写“感觉还能写”。
