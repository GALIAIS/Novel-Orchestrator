## 1396：宝剑大大，构筑大纲的时候可以用...


人工查看范围：正文、全部 2 条评论。无附件、无图片。

证据限制：这是理论确认帖，没有完整正文样章；拆解重点是将“计划-意外式构建法”与“情绪折线”“手弩理论”统一为 2000 字小剧情的可操作结构。

### 读者承诺

这篇沉淀的是小剧情可以用计划意外式来搭情绪折线。提问者理解：计划意外式的上行来自接近目标，下行来自意外打断；情绪折线的上行下行来源更宽，可以是接近目标、得宝、内心情绪、内心矛盾等。例子是 2000 字内干掉反派：主角先开枪，敌人有防弹衣；主角想逃，出口被堵；最后主角搏命设陷阱让敌人坠落。死翼确认正确，并说写作理论很多时候就是套娃。

### 核心机制

```text
目标明确 -> 计划一推进 -> 意外打断 -> 计划二改向 -> 意外加压 -> 计划三搏命爆发
```

计划意外式适合短篇幅冲突，因为它让每一次情绪下行都有具体原因：不是作者硬压主角，而是主角方案被敌人或环境破解。最终爆发也因此更爽。

### 期待链与情绪

```text
读者相信计划可行 -> 发现敌人更强 -> 主角改计划 -> 退路被封 -> 读者紧张 -> 陷阱反杀释放
```

下行不能一路压死读者，所以中间要有小波折、小希望和临时选择。每个计划都要让读者短暂相信“这次有机会”，意外才有打击感。

### 关键判断

- 计划意外式可以作为情绪折线的局部实现。
- 上行可以来自接近目标，也可以来自新资源和新判断。
- 下行最好来自意外、阻碍和计划失败。
- 2000 字小剧情适合三轮计划。
- 手弩理论的爆发需要前面足够下压。
- 下压期间要留波折，不能把读者压死。
- 写作理论可互相套用，关键是服务具体冲突。

### 可迁移写法

```text
小剧情目标：
计划一：
意外一：
计划二：
意外二：
最低点：
最终计划：
爆发兑现：
```

## 1397：#公开处刑  都市异能的，之前...


人工查看范围：正文、附件标记、全部 22 条评论。附件 `一到四章.docx` 尝试下载但返回 403，无法查看文件正文。

证据限制：被处刑正文主要在 docx 附件中，当前只能依据评论判断问题：行文缺少逻辑关系和镜头转换，开头硬抛设定、硬打斗、主角视角与普通世界到非凡世界的过渡不足。

### 读者承诺

这篇沉淀的是都市异能开篇不能把设定、打斗和世界观硬塞给读者。评论集中指出：行文没有逻辑关系和镜头转换；一开始抛设定很不贴切；战力体系还没构建就打斗会显得生硬；主角从普通世界进入非凡世界应该有过程；开场人物没立起来，最好由主角视角、小事件引入大主线。最后的共识是，设定和升级要一点点透进去。

### 核心机制

```text
主角日常视角 -> 小事件异常 -> 逐步接触非凡 -> 设定随冲突透出 -> 升级/打斗获得代入
```

都市异能不是不能开篇打斗，而是打斗要有读者能跟上的视角、原因和情绪。读者还不知道主角是谁、世界怎么运作、能力意味着什么时，直接大段设定和战斗只会显得突兀。

### 期待链与情绪

```text
读者认识主角 -> 发现生活异常 -> 小麻烦升级 -> 非凡规则露出 -> 主角被迫选择 -> 大主线打开
```

开篇吸引力不是越非凡越好，而是让普通读者先站到主角身边，再和主角一起发现非凡世界。

### 关键判断

- 行文逻辑和镜头转换是基础问题。
- 都市异能设定要逐步渗透。
- 开场人物没立住，打斗爽点很难成立。
- 战力体系未搭好时，大打斗会生硬。
- 主角视角的小事件适合引入大主线。
- 不同平台节奏不同，不能照搬建议。
- 先修最严重问题，再谈其他优化。

### 可迁移写法

```text
主角普通身份：
开场小事件：
异常细节：
主角第一反应：
非凡规则第一次露出：
第一场小冲突：
设定延后内容：
大主线钩子：
```

## 1398：新一轮净网开始了。


人工查看范围：正文、全部 17 条评论、图片标记。图片已下载到 `_source_cache\batch-280\1398-1.jpg` 并查看。评论中的《反手充值一百万》《大英公务员》微信读书搜索链接按知识库可见外链标记查看，未展开外部作品正文。

证据限制：图片是群通知截图，不是完整平台规则文件；拆解重点是敏感题材风险识别：涉政、涉黄、未成年、骨科、炼铜、现实争议题材的尺度并非固定，发书时能过不代表后续安全。

### 读者承诺

这篇沉淀的是净网期的题材边界意识。图片提醒都市环境下师生恋、乱伦骨科、儿童色情、主角被炼铜等都过线；评论进一步讨论现实争议题材、女拳、涉政、历史政治尺度等。死翼说明：涉政大多指国内时事政治，二战英国这类不必然等同；但尺度从来没有固定标准，发书时行，发完不一定哪天还行。

### 核心机制

```text
净网信号出现 -> 识别敏感元素 -> 判断平台尺度 -> 降低现实争议/未成年/骨科/涉政风险 -> 调整题材表达
```

作者不能只问“别人写了为什么没事”。平台尺度有时间差、作品热度差、题材差和执行差。真正要做的是把高风险元素前置排查。

### 期待链与情绪

```text
作者看到风向 -> 担心作品风险 -> 明确高危元素 -> 修改或规避 -> 降低封章/限流损失
```

净网风险对创作的影响不是美学问题，而是连载存活问题。越靠近现实争议、未成年性化、亲属关系色情和国内时事，越需要谨慎。

### 关键判断

- 净网期要主动排查敏感元素。
- 未成年、炼铜、骨科、师生恋等是高风险。
- 涉政重点常在国内时事政治。
- 历史和海外政治也要看表达尺度。
- 尺度不是固定标准，会随时间变化。
- 火书能过不代表新人能过。
- 高风险题材要预留替换方案。

### 可迁移写法

```text
作品敏感元素：
是否涉及未成年：
是否涉及亲属/师生关系：
是否涉及国内时事政治：
是否涉及现实争议：
平台当前风向：
可替代表达：
最坏止损动作：
```

## 1399：大佬们，我重新拟大纲的时候在新...


人工查看范围：正文、全部 5 条评论。无附件、无图片。

证据限制：这是大纲取舍讨论，没有完整章纲；拆解重点是“同一卖点在不同平台风格下的入口选择”：底层奋斗更正统，女主捡到与家族产业更适合刺猬猫互动和噱头。

### 读者承诺

这篇沉淀的是平台风格决定开局入口。提问者的卖点是后启示录世界中用现代眼光和营销手段经营商业。旧大纲从底层人起步，联合被欺负的人反抗，优点是世界描绘和巨大矛盾；新大纲从被女主捡到、重整颓废家族产业开始，优点是省略势力积累，视角集中主线，更偏刺猬猫欢快和迪化。评论指出，第一种更正统、更起点风；第二种更适合刺猬猫，因为读者更喜欢男女主互动和噱头。

### 核心机制

```text
核心卖点固定 -> 判断目标平台 -> 选择入口风格 -> 保留爽点/互动 -> 处理后续主线接入
```

同一个题材可以写成底层奋斗，也可以写成被女主捡到后的家族经营。前者矛盾更硬，后者互动更快。平台不同，第一章要优先兑现的读者承诺也不同。

### 期待链与情绪

```text
读者进入平台熟悉入口 -> 看到对应爽点 -> 主角能力发挥 -> 关系/产业推进 -> 世界真相逐步打开
```

新大纲的问题是后期要接触底层和揭露世界真相，可能需要制造意外把主角丢到底层。这个风险要在大纲阶段提前设计，不要等后面强拐。

### 关键判断

- 大纲取舍要先确定平台。
- 底层反抗更正统、更起点风。
- 女主捡到和家族产业更适合互动噱头。
- 刺猬猫读者更吃男女主互动。
- 现代人眼光穿到未来/后启示录，需要额外设定支撑。
- 已写几万字不应成为唯一决策理由。
- 选轻入口时，要提前设计底层主线接入点。

### 可迁移写法

```text
核心卖点：
目标平台：
旧入口优势：
新入口优势：
平台读者最想先看：
后期主线接入方式：
已写存稿可复用部分：
最终入口选择：
```

## 1400：好家伙，末世，净网，都让我赶上...


人工查看范围：正文、全部 12 条评论。无附件、无图片。

证据限制：这是作者遇到净网和末世风险后的即时讨论，没有作品正文；拆解重点是已经写到 5-8 万字时如何止损：改、看首秀、切题材、转废土/仙侠/同人。

### 读者承诺

这篇沉淀的是敏感题材存稿期的止损判断。提问者末世写了 8 万字，遇到净网不知所措；评论建议上架前改总比上架后被净好。另有人说明末世不给推荐，而且多少沾血腥暴力和黄，起点七组编辑建议不要写末世，有人末世连审核都过不了。作者最后倾向番茄首秀凉了就切。

### 核心机制

```text
已写存稿 -> 遇到平台风向风险 -> 判断审核/推荐/首秀 -> 上架前改或观察 -> 失败快速切题材
```

已经写了几万字时，最危险的是被沉没成本绑架。末世本身常带血腥暴力、秩序崩坏、性威胁和阴暗内容，净网期更容易限流、审核不过或推荐减少。

### 期待链与情绪

```text
作者投入存稿 -> 风向突然变化 -> 焦虑上升 -> 制定首秀/审核止损线 -> 保住下一本机会
```

止损不是承认失败，而是职业化地保护时间。上架后被净、限流或大改，损失往往比 5-8 万字时调整更大。

### 关键判断

- 末世题材在净网期风险高。
- 上架前改比上架后被净更好受。
- 末世常伴随血腥暴力和擦边风险。
- 起点等平台可能不推荐或审核严格。
- 废土、仙侠、修真、同人可作为转向。
- 首秀数据可作为番茄止损线。
- 沉没成本不能替代平台风险判断。

### 可迁移写法

```text
平台：
审核风险：
推荐风险：
敏感元素：
可改成的相邻题材：
首秀/数据止损线：
切书触发条件：
```
