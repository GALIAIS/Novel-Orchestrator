## 2001：在知乎刷到了这个。


人工查看范围：正文、全部 35 条评论、1 张随帖截图；截图已缓存为 `_source_cache\batch-401\2001-zhihu-theory.jpg` 并按原图查看。

证据限制：截图只呈现一名提问者用“欲望—阻碍—行动—结果”分析丧尸小说中的无阻碍解决，以及答复者以人身贬损终止讨论，没有理论原文、作品章节或读者数据。评论对该公式是否普适、无敌流和盘点文是否存在阻碍、读者是否只为体验而来分歧明显。本卡不裁定某一理论绝对正确，而是把争论改造成“先声明模型适用域，再寻找功能等价结构”的诊断流程。

### 读者承诺

这篇沉淀的是写作公式应当承担可检验的诊断职责，而不是被当成能解释一切作品的自然定律。“欲望—阻碍—行动—结果”可以检查一个叙事单元是否存在人物追求、摩擦、选择和状态变化；它不能据此断言所有阅读快感都必须以同样篇幅、同样顺序显形，也不能因为某部作品表面没有战斗阻碍，就宣布它没有任何叙事压力。

遇到看似反例时，先把“阻碍”从战斗强弱中解耦。无敌主角可以瞬间消灭敌人，但仍可能受信息不足、到达时间、身份暴露、关系裂缝、规则限制、保护对象、社会后果或价值选择约束。真正需要检查的不是敌人能否打赢，而是主角珍视的状态能否无需选择和成本地获得。若答案仍是“可以”，该段的价值可能转向奇观、能力展示、情绪宣泄、关系陪伴、信息揭晓或模式识别，应改用相应指标评价。

还要区分作品整体、卷、场景和单次爽点。盘点、剪辑式高光或同人内容可能借用读者已有知识，省略目标和背景；省略不等于不存在，而是上下文被前置到共享认知中。相反，作者若错误假定读者知道前情，省略就会变成困惑。诊断时必须写明分析单位、目标读者已知信息和被省略环节由谁补足。

理论争论最终要落回文本预测：用该模型能否指出下一处读者会困惑、无聊或期待什么，修改后是否真的改善。如果理论只能事后给成功作品贴标签，无法提出可证伪判断，它最多是描述词汇；如果反例只能靠不断扩大概念边界被吸收，也应缩小适用域，而不是维护公式的万能性。

### 核心机制

```text
声明理论要解决的问题与分析尺度 -> 标出可观察的目标、摩擦、选择和结果 -> 对表面反例寻找信息/时间/身份/关系/规则等功能等价约束 -> 若仍不匹配则切换到奇观、陪伴或揭晓等体验模型 -> 提出修改预测 -> 用文本与读者反馈校正适用域
```

公式先生成问题，再接受反例约束；它的价值由诊断精度决定，不由覆盖范围决定。

### 期待链与情绪

```text
读者先看见主角珍视的状态 -> 表面优势让常规敌人迅速失效 -> 更深层约束暴露 -> 主角必须在两个代价间选择 -> 结果兑现能力优势又留下状态变化 -> 下一层约束自然抬升
```

无敌感来自常规障碍被稳定解决，牵引力来自仍有某种价值不能无成本获得。

### 关键判断

- 截图中的辱骂不能替代理论论证。
- 评论数量和个人收入经历不能证明公式真伪。
- 写作公式是模型，不是作品必须服从的法律。
- 分析前要声明题材、读者、尺度和问题。
- 欲望应落到人物珍视的可观察状态。
- 阻碍不等于战斗力更高的敌人。
- 信息、时间、身份、关系和价值冲突都可形成摩擦。
- 无敌主角可以解决战斗而保留其他约束。
- 高光或盘点内容可能由共享上下文补齐因果。
- 省略背景是否有效取决于目标读者已有知识。
- 真正的反例应促使模型缩小适用域或切换模型。
- 理论必须能产生文本层预测并接受反馈修正。

## 2002：前面那个讨论“理论”的贴，我还...


人工查看范围：正文、全部 25 条评论及嵌套回复。

证据限制：正文主张少数人可能凭直觉写好、多数人可借理论和练习接近市场需求，并指出理论往往是事后总结；评论继续讨论天赋、教育、经验、反馈和“读者想看什么”。帖子没有能力测量、训练对照、作品样本或长期数据，无法证明天赋比例、训练效率和商业结果。本卡回避不可验证的身份归因，只设计一套能留下预测、作品和反馈证据的训练闭环。

### 读者承诺

这篇沉淀的是理论与直觉不是二选一。理论把模糊感觉拆成可讨论的变量，帮助作者在尚未形成稳定直觉时提出假设；练习把假设放入具体文本；反馈暴露作者预期与读者实际体验的偏差；复盘再把有效判断压缩成更快的模式识别。成熟直觉可以减少显性推演，但仍需要在新题材、新平台或失效阶段重新接受验证。

训练时不要先判断自己“有天赋”还是“没天赋”。这类标签既难测量，又容易把一次顺利或一次失败解释成固定命运。更可执行的指标是：能否在写前预测读者会期待什么，能否按时完成样本，陌生读者实际复述了什么，修改是否改善同一问题，以及有效做法能否跨两个以上文本复现。

理论需要以假设形式进入创作。比如“本章末必须改变人物处境”不是教条，而可以转成预测：若章末只有信息重复，读者对下一章的问题会模糊；修改为不可逆选择后，读者能否更准确说出下一步期待。每次只改变一个主要变量，保留前后版本和反馈，才能区分有效机制与事后自圆其说。

反馈也要分层。个人偏好回答“我喜不喜欢”，理解反馈回答“我看懂了什么”，行为反馈回答“我在哪里停下或继续”，结构反馈回答“哪项承诺被兑现”。作者不必机械采纳读者提出的解决方案，但必须认真核对多人重复指出的症状。理论的终点不是背诵术语，而是在保持证据意识的前提下形成快速、可复查的判断。

### 核心机制

```text
用理论提出单一文本预测 -> 写出可完成的短样本 -> 记录作者预期 -> 收集理解/期待/行为反馈 -> 区分重复症状与个人方案 -> 定点修改并做前后对照 -> 把可复现判断压缩成直觉 -> 遇到新情境重新显性验证
```

理论负责提出可检验问题，实践负责制造证据，反馈负责纠偏，复现负责形成可靠直觉。

### 期待链与情绪

```text
作者从“有没有天赋”的笼统焦虑出发 -> 获得一个可操作假设 -> 完成短样本 -> 读者反馈暴露具体偏差 -> 单点修改产生可见改善 -> 多轮复现形成稳定判断
```

确定感不来自自我归类，而来自每轮都有作品、预测和结果可以核对。

### 关键判断

- 帖子没有证明天赋存在何种比例或来源。
- 会写但讲不清理论，与会讲但写不好可以同时存在。
- 理论是压缩经验的模型，不是成功保证。
- 直觉可能来自长期输入、训练和自动化判断。
- 不必先解决“我有没有天赋”才能开始训练。
- 写前预测比事后解释更能检验理论。
- 每轮应聚焦一个主要变量。
- 反馈中的症状与读者给出的方案要分开。
- 多人重复的理解偏差值得优先处理。
- 单次好评、签约或数据不能证明机制可复现。
- 形成直觉后仍要保留回到显性分析的能力。
- 市场、题材或读者改变时，旧直觉需要重新校准。

### 读者承诺

这篇沉淀的是 OCR 文档只能充当检索层，不能替代原始页。情感描写资料往往依赖相近词差异、程度、身体反应和语境，错一个否定词、形容词或标点就可能改变含义。可靠资料库应同时保留原始文件、OCR 文本、稳定页码和校对状态，让作者能先搜索，再回到原页确认。

建库前先做可用性抽检。均匀选取开头、中段、末尾和复杂排版页，人工核对标题、正文、表格、脚注、页码以及容易混淆的字；若错误会影响关键概念，就把整本标为“仅定位，不可直接引用”。不要为了看起来完整而让未经校对的文本混入正式方法库。

索引应服务场景决策，而不是堆情绪词。每条可按“情绪名称—触发事件—身体信号—即时想法—行动冲动—可见动作—关系对象—强度—场景功能—原页”记录。相近表达还要标出适用人物、语体和强度边界，避免把检索到的句子直接贴进角色身上，造成词藻丰富但人物反应失真。

引用和再利用必须留下来源链。任何进入创作方法卡的结论都要能回到具体页；OCR 与原页冲突时以人工核对的原页为准；无法打开原件时，只能记录书目信息和待核验任务。本案例目前正处于“资源已知、正文不可读”状态，不能假装完成内容吸收。

### 核心机制

```text
保存原始文件与来源信息 -> 生成带稳定页码的 OCR 检索层 -> 分层抽样人工校对 -> 标记可引用/仅定位/待核验状态 -> 按情绪触发身体行动与强度建立索引 -> 搜索命中后回查原页 -> 经角色和场景改写后使用
```

原页负责证据，OCR 负责速度，索引负责定位，作者仍负责把资料转成角色行为。

### 期待链与情绪

```text
作者遇到难以落笔的情绪 -> 用触发、强度或身体信号检索 -> 找到若干候选条目 -> 回查原页排除 OCR 错误 -> 按人物习惯选择反应 -> 用动作和选择推动场景
```

资料提供的是可验证候选，不是可以直接粘贴的成品情绪。

### 关键判断

- 两个附件本轮均未读到正文。
- 不得依据书名和评论推断书中具体内容。
- OCR 文本不是原件的证据替代品。
- 情感细微差异对识别错误尤其敏感。
- 页码或图像锚点必须在处理后保持稳定。
- 抽检要覆盖不同位置和复杂版式。
- 未校对内容应明确标为“仅定位”。

- 情绪词表不能替代人物一致性。

## 2004：心里装着很多东西，想说出来的也...


人工查看范围：正文、唯一 1 条评论。

证据限制：正文只描述“心里有很多东西、写了又删、觉得都是废话”，最后用一个自我贬低标签收束；评论只是旁观式玩笑。原文没有小说片段、具体事件或修改过程，不能判断作者实际表达能力，也不能据此作心理诊断。本卡只抽取一个写作问题：过早评价和抽象自我标签会删除可用于场景的具体材料。

### 读者承诺

这篇沉淀的是情绪不是一个形容词，而是一串受到触发后逐步泄露的变化。作者若一开始就用“矫情、难过、混乱”给自己或人物下结论，触发事件、身体反应、矛盾念头、想做又不敢做的事都会被压成概念，读者只能被告知情绪，无法亲历它。

转写时先把生成与评判分开。第一轮只收集事实：何时、何地、谁说了什么、身体先有什么反应、脑中闪过什么、想采取什么动作、为什么又停下。第二轮再找矛盾，例如想解释又怕显得在意，想离开又等待挽留，想删除又希望被看见。矛盾会自然产生动作延迟、口是心非和关系风险。

抽象材料进入小说还需要外部载体。选择一个人物会反复触碰、避开或误用的物件，一句没有说完的话，以及一个必须在本场完成的现实任务。让情绪改变人物完成任务的方式：倒水洒出、消息删改、门把握住又松开、该交出的东西被藏起。读者从可见偏差推断内在状态，比长段自我定义更有参与感。

删除并非问题，过早删除才会让素材消失。可采用“保留箱”：初稿里不确定的句子先移到场景后，完成事实和动作后再判断是否重复。最终文本可以克制，但克制应来自精准选择，不是因为羞耻先把所有具体感受抹去。

### 核心机制

```text
暂停自我标签与删改 -> 记录具体触发、身体信号、念头和冲动 -> 找出两种相反需要 -> 绑定现实任务、关系对象和物件 -> 让压抑通过动作偏差泄露 -> 选择产生关系后果 -> 完稿后再删重复解释
```

先保住材料，再把材料变成选择；情绪强度由行动成本和关系后果呈现。

### 期待链与情绪

```text
人物被具体事件触发 -> 表面继续完成日常任务 -> 身体和小动作泄露异常 -> 相反需要不断拉扯 -> 一次克制失败或主动选择 -> 对方反应改变关系 -> 未说出口部分继续牵引
```

读者的参与来自比人物更早发现泄露，并等待他最终说、做或失去什么。

### 关键判断

- 原帖太短，不能评价作者整体能力。
- 自我贬低标签不是有效的文本诊断。
- 本卡不对发帖者作心理或人格判断。
- 情绪需要触发、身体、念头、冲动和行动链。
- 生成阶段与编辑阶段应暂时分离。
- 两种相反需要比单一情绪词更能发动场景。
- 现实任务可以给抽象感受提供外部压力。
- 物件与动作偏差能承载未说出口的信息。
- 克制不等于删掉所有具体反应。
- 长段解释应在动作完成后再判断是否需要。
- 情绪必须改变选择或关系，才进入叙事因果。
- 留白应有可推断证据，不能只靠作者不写。

### 训练任务

任选一句“我很难过、很矫情、很混乱”式标签，禁止把它写进正文。先写 15 条未经删改的事实和反应，再写一个 800 字场景：人物必须完成一件普通任务，同时在两种相反需要之间作出有关系成本的选择。

## 2005：我在构思一个架空历史的故事背景...


人工查看范围：正文、全部 5 条评论、评论中外部视频的标题文字；按要求未打开视频或作品页面。

证据限制：正文只提出把多个历史政权、族群和影视人物放入同一架空世界，让主角种田争霸；没有时代基线、地图、角色名单、金手指或故事大纲。评论提出“能否驾驭”“人物要有共同核心”“先定金手指与风格”等经验判断，没有成稿验证。直接使用影视作品角色还可能涉及著作权、角色权益和平台规则，本卡不提供法律结论；实际使用前应按适用法、授权状态和发布平台规则另行核验。

### 读者承诺

这篇沉淀的是群英架空的吸引力不在“读者认识多少张脸”，而在不同政治能力、价值观和历史路径被放进同一套压力系统后，会作出怎样彼此冲突的选择。先确定作品的共同母题，例如秩序与自由、统一与地方利益、旧贵族与新生产方式，再选择能从不同角度回答该问题的人物功能，名单才会形成结构。

统合顺序应从世界而非客串名单开始。先设资源、交通、农业、军制、合法性和主要矛盾，明确哪些历史条件被合并、哪些仍不可同时存在；再为角色分配统帅、治理、情报、财政、技术、宗教或地方动员等功能。每个角色进入新世界后都必须重新拥有出身、利益、依附关系、信息边界和失败代价，不能只复制原作名场面。

风格决定导入机制。严肃争霸路线需要尽量消除“演员登场”感，让人物像世界内部成长出来，并接受制度和资源约束；轻松、元叙事路线可以使用召唤、声望或系统机制，但也要规定获取条件、忠诚边界和文化适应成本。无论哪种路线，金手指只解释人物为何相遇，不能替作者解决人物为何合作、为何背叛以及能力如何落地。

主角必须保持因果中心。引入名人后，每卷仍要由主角提出目标、争取资源、承担决策后果，并在群英不同方案间选择。角色数量由可管理的冲突槽位决定：同一阶段只保留少数拥有独立目标且会改变主线的人物，其余延后或合并。若删除某位角色只少一个熟悉名字而不改变任何决策，他只是彩蛋，不是结构成员。

### 核心机制

```text
确立共同母题与叙事风格 -> 建立资源制度和权力尺度 -> 按政治功能与价值立场筛选角色原型 -> 为每人重建新世界利益和限制 -> 让主角在互斥方案间决策 -> 合作或背叛改变争霸格局 -> 新秩序反过来检验共同母题
```

世界规则负责统一人物，价值分歧负责制造冲突，主角决策负责保持故事中心。

### 期待链与情绪

```text
读者认出人物能力或原型 -> 新世界利益迫使其偏离旧印象 -> 多名强者围绕同一难题给出互斥方案 -> 主角选择一条并承担代价 -> 联盟重组或制度变化 -> 更大尺度再次检验人物立场
```

熟悉感负责快速建立期待，真正的新鲜感来自角色在统一规则下产生不可预设的变化。

### 关键判断

- 原帖没有足够大纲，不能判断该设定是否可驾驭。
- 多历史元素并置前要建立统一时空基线。
- 角色选择需要共同母题和功能标准。
- 认识度不能替代人物在新故事中的欲望。
- 原作标签和名场面不能直接充当人物弧线。
- 严肃路线需要降低元叙事客串感。
- 轻松召唤路线也要有规则、忠诚与适应成本。
- 金手指不能自动解决合作和治理问题。
- 群英必须给出互斥方案，而非轮流赞美主角。
- 主角要承担选择和分配资源的责任。
- 角色数量由当前冲突槽位和可回收线索决定。
- 受保护角色、改编形象和平台合规要另设权利门禁。

### 训练任务

先不用任何现成角色名，围绕一个共同母题设计五个功能互异的原创角色原型；给每人写新世界利益、底线、能力盲区和一项与主角冲突的方案。再删掉两人，若首卷主要决策完全不变，继续合并角色，直到每人都能改变因果。
