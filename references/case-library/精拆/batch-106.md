## 526：#思考分析 宝剑巨您好，我现在...

人工查看范围：正文、全部 15 条评论。无附件，无图片。

### 读者承诺

这篇解决“开头拿住读者后，后续剧情变平淡”的问题：作者把主角内心转变误当三幕式，真正的一卷三幕需要冲突烈度升级或冲突对象升级。公主和亲不是三个冲突，而是一次冲突爆发；后面必须让异族、朝廷、权臣、感情线或体制线持续加码。

### 核心机制

```text
第一冲突爆发 -> 观念/立场冲突解决 -> 外部势力升级行动 -> 主角下一步保护行动 -> 多方最过激手段 -> 最终解决威胁
```

### 期待链与情绪

```text
读者看到公主被劫 -> 期待主角救人 -> 救完后期待矛盾升级 -> 异族/朝廷继续施压 -> 主角保护公主和建功立业并行 -> 一卷收束和亲危机
```

### 关键判断

- 发帖者设计的三点只是主角对和亲问题的内心转变，撑一幕可以，撑不起一卷。
- 星主指出发帖者对三幕式和冲突升级理解笼统。
- “劝公主和亲”到“行动阻止和亲”整体只是一次冲突的爆发和解决。
- 真正三幕式要求一卷里的冲突烈度升级，或冲突对象变化升级。
- 后续可以让异族采取新行动，或朝廷继续发布和亲命令。
- 第二幕应围绕主角如何保护公主继续行动。
- 第三幕则是异族和朝廷都使用最过激手段，主角最终解决全部威胁。
- 评论提醒剧情平淡还因为前一段结束前没及时安插后续伏笔，写一卷至少要想清下一卷大致提纲。
- 历史文还要考虑平台侧重、素材准备和基调，否则容易崩剧情、崩人设、崩细节。

### 可迁移写法

一卷冲突升级表：

```text
第一冲突：
第一冲突对象：
第一解决方式：
第二幕升级对象：
第二幕新增压力：
第二幕伪胜利/伪危机：
第三幕最过激手段：
最终解决：
下一卷伏笔：
```

### 训练任务

把一个“救公主”小单元扩成一卷：列出异族、朝廷、权臣、公主本人四方各自的利益，再设计三次升级。每次升级必须改变主角的行动方案。

## 527：#参考资料 从灵感脑洞到大纲细...

人工查看范围：正文、全部 3 条评论。无附件，无图片。

### 读者承诺

这篇把“灵感脑洞”推进到商业可行性验证：过稿不等于马上开书，几万字开头也不一定要立刻发。大纲阶段要验证这个创意是否能支撑完本、不崩、不写空；如果做完大纲就不想写，反而说明可行性不强。

### 核心机制

```text
灵感记录 -> 选定脑洞 -> 一句话核心梗 -> 简纲/粗纲 -> 设定落地 -> 细纲事件 -> 可行性验证
```

### 期待链与情绪

```text
作者过稿后仍担心写不长 -> 查大纲教程 -> 发现要验证创意能否完本 -> 从急着发书转为可行性研究 -> 避免开书后崩盘
```

### 关键判断

- 发帖者卡在过稿后写大纲阶段，越改越发现需要扩展的地方。
- 她强调几万字开头不必马上发出去。
- 大纲阶段是在验证一本书的创意是否能坚持到完本不崩。
- 视频概括的大纲步骤包括准备阶段、框架、设定落地、完善细纲。
- 框架从一句话核心梗，到 300-500 字简纲，再到 1000-5000 字粗纲。
- 设定落地包括世界观、人设、人物小传、人物线。
- 完善细纲包括阶段、事件、细节、人物弧光。
- 评论指出如果大纲做完不想写，那就不写，说明它不足以完结成一本商业化网文。

### 可迁移写法

大纲可行性验证表：

```text
一句话核心梗：
300字简纲：
粗纲主线：
世界观：
人物小传：
人物线：
阶段事件：
人物弧光：
是否还能写长：
是否仍想写正文：
```

### 训练任务

给一个脑洞写 1 句核心梗、300 字简纲、1000 字粗纲。写完后问自己：还想写正文吗？如果不想，记录是哪个环节暴露了可行性不足。

## 528：各位书友，有什么推荐的日常文吗...

人工查看范围：正文、全部 4 条评论。无附件，无图片。

### 读者承诺

这篇价值在于建立日常文样本入口：当作者想学日常文，不应空想“日常怎么写”，而要先收集同类样本。评论给出恋爱日常、长生日常、刺猬猫/起点样本，说明日常文内部也要按关系类型、平台气质和主角设定细分。

### 核心机制

```text
想学日常文 -> 收集样本 -> 按关系/设定/平台分类 -> 拆日常事件 -> 提炼节奏和看点
```

### 期待链与情绪

```text
作者不知道日常文参考 -> 读者推荐样本 -> 样本覆盖恋爱、长生、轻松平台向 -> 学习目标从泛日常变成同类拆解
```

### 关键判断

- 原帖只是询问日常文推荐，说明它是样本检索入口。
- 推荐包括《我家老婆来自一千年前》《我家女友没我可爱》《我加载了恋爱游戏》《我真的长生不老》《跳过谈恋爱不行吗》。
- 样本多与恋爱、时间差、长生、轻松互动有关。
- 刺猬猫和起点样本并列，提示平台口味不同。
- 日常文不能只理解为“没大事”，它常靠关系变化、人物反差、生活细节和轻冲突维持阅读。

### 可迁移写法

日常文样本表：

```text
书名：
平台：
主角设定：
关系核心：
日常看点：
轻冲突来源：
每章推进方式：
读者留下来的理由：
可迁移元素：
```

### 训练任务

选 3 本日常文，每本拆前 3 章。只记录“这一章发生了什么小事、关系有什么变化、读者为什么愿意看下一章”。

## 529：《自我反省》

人工查看范围：正文、全部 8 条评论、1 张图片。图片已下载并人工查看：`case-library\精拆\attachments\batch-106\case-529-image-1.jpg`。

图片证据：图片为派蒙表情包，文字“你小子”，用于配合发帖者自我反省后的调侃语气，不提供额外写作信息。

### 读者承诺

这篇的核心是同人主线不能只追原著剧情，必须围绕自己的金手指重建主线。原著萧炎搜集异火，是因为焚诀能吞异火变强，并服务救父主线；如果同人主角的金手指不是焚诀，就不能机械追着原著机缘走。

### 核心机制

```text
确定金手指 -> 明确如何变强 -> 设计解锁条件 -> 主线围绕解锁和升级 -> 原著事件只按功能取用
```

### 期待链与情绪

```text
作者以为截胡原著机缘就是同人 -> 发现只是换主角复刻原文 -> 回看金手指功能 -> 重新设计解锁金手指和升级主线 -> 同人获得差异化
```

### 关键判断

- 发帖者意识到自己以前的低级错误是没有围绕金手指写主线。
- 他原以为同人按原文剧情走、截胡原主角机缘和事件就够。
- 回看大纲后发现除了主角换人，与原著没区别。
- 斗破原著里异火主线成立，是因为焚诀能吞异火变强，并服务救父。
- 如果自己的主角身边是老爷爷，一堆斗技可用，就没必要去迦南学院复制原著路线。
- 解决办法是设置金手指 1 需要条件解锁金手指 2，每段剧情走向都为了解锁和升级。
- 评论指出金手指一定要一句话概括，关键不是“让主角变强”，而是“如何让主角变强”。
- 主线应围绕主角变强目的和金手指如何让他变强展开。

### 可迁移写法

同人金手指主线表：

```text
原著主线：
原著金手指：
原著为什么必须走这条线：
同人主角金手指：
同人主角如何变强：
需要解锁的条件：
哪些原著事件仍有用：
哪些原著事件必须放弃：
新主线一句话：
```

### 训练任务

给一个同人项目写“原著主线”和“我的主线”两张表。凡是不能服务自己金手指解锁或升级的原著事件，全部标记为可删或需重写。

## 530：大佬您视频里面展示的那本从这个...

人工查看范围：正文、全部 4 条评论。无附件，无图片。

### 读者承诺

这篇是拆书表格源文件定位问题：学习拆书工具时，不只要看视频展示，还要找到原文件和表格位置。星主给出的路径是文件包里的 Word 文档“拆书《这个明星很想退休》27万字”，表格在该文档内。

### 核心机制

```text
看到视频展示 -> 找参考表格 -> 定位文件包 -> 找 Word 原文件 -> 在文档中查看表格
```

### 期待链与情绪

```text
学习者想复用拆书表格 -> 文件包里一开始没找到 -> 星主补充准确文件名 -> 学习路径从模糊查找变成具体定位
```

### 关键判断

- 发帖者想要“这个明星很想退休”拆书表格版作为参考。
- 星主先给出文件包链接，说明原文件在几个文件里。
- 发帖者扫两遍没找到。
- 星主补充准确文件名：“拆书《这个明星很想退休》27万字”。
- 表格在这个 Word 文档里面。
- 这类知识库资料常不是单独表格文件，而是嵌在 Word 文档中。
- 对学习拆书来说，文件定位和命名记录本身也是知识库使用能力。

### 可迁移写法

拆书资料定位表：

```text
想找的资料：
视频/教程来源：
文件包链接：
准确文件名：
文件类型：
表格所在位置：
是否已下载：
可复用字段：
```

### 训练任务

找一个教程里提到的拆书表，把来源、文件名、文件类型、表格位置记录下来。下次复用时先打开原文件，不凭记忆重造表格。
