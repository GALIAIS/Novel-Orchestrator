## 1401：#公开处刑 这是我写的第四版开...


人工查看范围：正文、文件标记、全部 4 条评论。无图片；附件标记为 `太灵寺天师.docx`，id `421452488282518`，知识库未提供下载 URL。

证据限制：被处刑正文在 docx 附件中，当前只能依据评论判断问题：开篇回忆过多、人物前后言行不统一、对话为了有趣导致孤狼人设崩塌。

### 读者承诺

这篇沉淀的是开篇人设必须言行合一。作者把类型从都市异能改为东方玄幻后提交第四版开头，评论指出：前文用了很多文字塑造主角冷静、有危机感、渴望稳定生活的孤狼形象，但后面一开口就成了憨憨，人设直接崩塌。另一条评论强调开篇不要回忆，睡醒后直接和追兵开打更好，鞋子、师傅教功法等回忆都应后置。

### 核心机制

```text
开篇塑造人设 -> 行动/台词验证人设 -> 当前追兵冲突推进 -> 回忆压缩后置 -> 人物可信
```

读者认识角色不是看作者怎么描述，而是看角色在危险中怎么说话、怎么行动。冷静孤狼不能突然用憨憨式台词破坏气质，除非作者明确设计反差并能承担后果。

### 期待链与情绪

```text
读者看到危机感 -> 期待冷静应对 -> 台词崩人设会出戏 -> 追兵当前冲突兑现 -> 人设稳定
```

开篇回忆很容易让节奏发软。真正该放在第一屏的是醒来、追兵、选择、反击或逃生，而不是解释鞋子、师傅和功法来历。

### 关键判断

- 人设由行动和台词验证。
- 先描述孤狼，再写憨憨台词，会直接崩。
- 对话有趣不能压过人物一致性。
- 开篇回忆要极限压缩。
- 功法、师傅、物件来历可后置。
- 睡醒后直接面对追兵，比先回忆更有效。
- 修改时先定人设口径，再改台词。

### 可迁移写法

```text
主角核心人设：
第一句台词：
第一反应动作：
当前危机：
必须删除/后置的回忆：
保留背景句：
人设验证动作：
下一段冲突：
```

### 训练任务

选一段开篇，把所有回忆压缩成 3 句以内。再检查主角前 5 句台词，逐句标注是否符合“冷静/孤狼/危机感”等核心人设，不符合就重写。

## 1402：选2题回答，主角金手指为系统加...


人工查看范围：正文、全部 2 条评论。无附件、无图片。

证据限制：这是作业式剧情设计，未提供正文成稿；拆解重点是校园格斗中“群情激奋、临阵退缩、虚晃一枪、二次埋伏、金手指反杀”的成熟情绪折线。

### 读者承诺

这篇沉淀的是校园格斗小剧情的双重反转。事件是死党惹了校外混混，对方校门口堵人。第一轮上行是同学兄弟群情激奋要撑腰；第一轮下行是混混头恶名昭彰、带武器、放狠话，帮忙的人临阵退缩。第二轮上行是主角早已安排老师和警察，校门口果然没有混混；第二轮下行是对方埋伏小巷，老师被警告拖延，去警察局的同学被拦，主角和死党孤立无援。最后主角用系统加持格斗技巧打倒混混，围观者和老师赶来只看见主角带死党离开，身后全是倒地混混。死翼评价：没问题，合格，成熟剧情。

### 核心机制

```text
同学撑腰 -> 混混威慑退场 -> 主角预案奏效 -> 敌人反预判埋伏 -> 死党受辱 -> 主角反杀震撼围观
```

这套结构成熟，是因为每次上行都有具体方案，每次下行都有敌人更深一层的手段。最后不是主角随便开挂，而是在孤立无援和死党下跪求饶后，集中兑现战斗爽点。

### 期待链与情绪

```text
读者期待群架 -> 支援崩掉 -> 期待主角智取 -> 智取也被看穿 -> 情绪压到底 -> 格斗金手指爆发
```

围观群众迟到是一个好收尾：他们没有参与救援，却见证结果，从而放大主角的装逼余波。

### 关键判断

- 校园格斗可以用两次上行两次下行。
- 支援退缩能制造孤立感。
- 主角预案先成功再失败，比直接失败更有波动。
- 敌人要有经验，能看穿主角计策。
- 死党下跪求饶能把羞辱和情义拉满。
- 金手指反杀要在最低点兑现。
- 围观者见证结果，比现场解说更爽。

### 可迁移写法

```text
起因：
第一批支援：
敌人威慑：
主角预案：
敌人反预案：
最低点羞辱：
金手指爆发：
见证者反应：
```

### 训练任务

写一个 1800 字校园冲突段，必须包含两次“以为能解决”和两次“发现更糟”。结尾让旁观者只看到战斗后的结果，用反应放大主角形象。

## 1403：想探讨一下，关于网文中的【金钱...


人工查看范围：正文、全部 9 条评论、图片标记与外链标记。图片已下载到 `_source_cache\batch-281\1403-1.jpg` 并查看。评论中的《魔兽世界》微信读书搜索链接和 B 站货币设定视频链接均按可见外链标记检查。

证据限制：图片为帖子截图，非完整系统教程；拆解重点是网文金钱体系必须在动笔前设计收入、支出、消耗和等级，否则随手编会变成 bug。

### 读者承诺

这篇沉淀的是金钱体系不能无限膨胀。提问者指出，都市文主角有金手指后金钱不断积累，如果不控制，赚到几百亿后故事寿命就会崩。图片里的“财富消失术”总结了几种办法：用成功带来更快财富积累，再通过开新地图、作品魔改、荣誉、金手指剥削、冲动消费人设等方式让读者忽略或消耗主角财富。死翼补充《魔兽世界》的货币体系：打怪获得金币和修装备、买物资等损耗达到平衡；写小说也类似，《诡秘》的前中期在这方面做得好。

### 核心机制

```text
收入增长 -> 支出/损耗同步升级 -> 新目标消耗资金 -> 金钱不失控 -> 爽点持续
```

金钱体系和功法体系一样，需要开书前想清楚。什么等级能赚多少钱，什么事项必须花钱，哪些支出随主角升级而提高，都不能临时一高兴就写几个亿。

### 期待链与情绪

```text
主角赚到钱 -> 读者获得成功感 -> 新支出打开更大目标 -> 财富转化为行动力 -> 再次赚钱
```

钱不是数字收藏，而是推动剧情的燃料。只赚钱不花钱，主角会很快失去压力；只强行清空钱，又会像作者硬削。

### 关键判断

- 金钱体系要提前设计。
- 收入和支出要随等级同步增长。
- 修装备、买物资、建设、预测、加速等都是合理消耗。
- 金手指收费可控制主角经济。
- 冲动消费人设能制造坑，但不能变成降智。
- 读者不一定喜欢复杂货币设定，爽点仍要清晰。
- 随手编大额收入基本都是 bug。

### 可迁移写法

```text
当前阶段收入：
当前阶段必要支出：
升级后收入上限：
升级后支出上限：
金手指收费项：
建设/资源消耗：
财富失控风险：
下一阶段金钱目标：
```

### 训练任务

为一本都市/经营文设计 5 个阶段的金钱表。每阶段写主角能赚多少钱、必须花多少钱、花钱换来什么剧情推进，禁止只写存款增长。

### 读者承诺

这篇沉淀的是资料型外链的处理边界。原帖只有一个金山文档链接，没有正文说明，也没有评论可供交叉验证。虽然链接本身可访问到 kdocs 页面，但没有获取到文档正文。对知识库精拆而言，这类材料不能凭标题猜内容，也不能把外链可能包含的东西当成已读证据。

### 核心机制

```text
只有外链 -> 尝试访问 -> 无法取得正文 -> 标注证据限制 -> 不做内容性推断
```

拆书系统必须区分“链接存在”和“内容已读”。如果内容不可见，唯一可靠的信息就是链接地址、发帖人、时间和缺失状态。

### 期待链与情绪

```text
发现资料链接 -> 期待补充内容 -> 访问受限 -> 记录缺口 -> 等待可见材料再拆
```

这类案例的价值不在方法本身，而在防止知识库污染：不能为了凑满拆解而编造外链内容。

### 关键判断

- 外链存在不等于内容可用。
- 无正文、无评论时不能做主题推断。
- 可访问页面但看不到正文，仍是证据不足。
- 必须记录访问限制和检查方式。
- 不能用同作者前后文替代当前外链内容。
- 后续若能获得文档正文，可再补拆。
- 证据边界本身也是系统可靠性的一部分。

### 可迁移写法

```text

访问结果：
是否取得正文：
可用证据：
不可推断内容：
```

## 1405：#毒点解析 有个问题，最近我写...


人工查看范围：正文、全部 2 条评论。无附件、无图片。

证据限制：这是剧情设想问答，没有正文样章；拆解重点是“主角借金手指支棱后，被反派误判为傍大腿，从而让反派互斗”的可行条件。

### 读者承诺

这篇沉淀的是反派误判型装逼可以比单纯主角装逼更有趣。提问者写主角因金手指突然杀伐果断，但身边人和敌人都以为他不是真正强，而是傍上了别的大腿：反派 A 以为主角傍上反派 B，反派 B 又以为主角傍上反派 A，于是两个反派互斗。死翼判断：铺垫写到位、逻辑理顺就合适，比单纯主角装逼更有意思；如果写成主角有意造成这个局面，身边人不质疑主角，只是惊疑不定和嘀咕，接受度更高。

### 核心机制

```text
主角突然变强 -> 反派无法理解 -> 误判主角背后有人 -> 主角有意放大误会 -> 反派互斗 -> 主角收利
```

关键是误判要合理。反派不是蠢，而是基于信息不全做出看似有道理的分析。读者能看见反派逻辑，也知道主角真正底牌，所以会获得智斗和装逼的双重乐趣。

### 期待链与情绪

```text
读者知道主角有金手指 -> 反派误读成靠山 -> 反派自我脑补 -> 主角顺水推舟 -> 两边互咬 -> 读者暗爽
```

毒点风险在于身边人如果直接否定主角，会让读者不爽。更好的处理是身边人惊疑不定，而不是站出来贬低主角。

### 关键判断

- 误判剧情需要足够铺垫。
- 反派分析要自洽，不能纯降智。
- 主角最好有意推动误会。
- 身边人不宜公开质疑主角能力。
- 惊疑、嘀咕比贬低更容易被接受。
- 反派互斗能制造比单纯打脸更复杂的爽点。
- 读者要始终知道主角真实底牌。

### 可迁移写法

```text
主角真实底牌：
外界可见异常：
反派A误判：
反派B误判：
主角推动误会的动作：
身边人反应：
反派互斗结果：
主角收益：
```
