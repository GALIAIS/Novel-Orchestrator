# Batch 001：高权重案例人工精拆

本批次人工逐篇查看原文，先处理 5 个高权重案例。后续批次继续追加，不把本批次冒充全量完成。

## 001：为什么网络小说的核心不是“爽点”，而是“期待感”？

人工查看范围：正文主体、断期待六类、两长一短、单一目标多重期待、评论中对核心梗重复和两长一短的补充。

### 读者承诺

这不是单一案例，而是“追读维持机制”的总案例。它承诺解决的问题是：为什么爽点兑现后读者反而可能弃书，以及如何让读者在爽点之后继续保留阅读动力。

### 核心机制

```text
情绪拉扯 -> 期待形成 -> 期待兑现 -> 爽点达成 -> 新期待接上
```

文章最关键的反常识点是：爽点不是持续阅读的核心动力，期待感才是。爽点兑现的一瞬间，原有阅读动力会被消耗；如果没有新期待，读者会满足后离开。

### 断期待类型

1. 丢失目标：高潮或卷末后不知道下一步看什么。
2. 人际关系网破裂：换地图、刀配角、淡化旧角色导致见证体系消失。
3. 长期不升级/无收获：主线期待迟迟不兑现，支线又不能补足。
4. 期待满足程度极低：目标完成了，但爽点没写出层次。
5. 发展方向太平：核心梗运行方式重复，读者一眼看到头。
6. 毒点破坏承诺：虐主、偏题、人设崩、核心梗丢失等。

### 期待不断档方案

- 两长一短：最晚两万字内建立两个长线期待和一个短期期待。
- 短期期待要交错衔接，不能断档。
- 长线期待像任务列表，能随时转成短期目标。
- 单一目标可以拉多重期待，例如一个自救事件同时拉真相、势力评价、家人关系、人际网变化。

### 可迁移写法

任何题材都能使用“期待库存表”：

```text
当前短期期待：
长线期待 1：
长线期待 2：
当前爽点兑现前的新线索：
兑现后的下一目标：
见证体系是否还在：
```

### 训练任务

选自己当前开篇或大纲，列出所有期待。如果低于“两长一短”，补一个长线期待；如果当前爽点后没有新目标，在爽点兑现前插入新线索。

## 002：对于核心梗的理解

人工查看范围：正文完整阅读、核心梗提取流程、灵感/素材/核心梗关系、一级/二级/三级核心梗评论讨论。

### 读者承诺

解决“我该收集什么、怎样从故事里抽出可复用结构”的问题。它把“核心梗”定义成能复现固定读者情绪链条的故事结构，而不是一句流行梗、一个设定或一个题材标签。

### 核心机制

```text
原型故事 -> 环境/人物/起因/经过/结果 -> 抽象关键特征 -> 检查能否复现情绪链 -> 得到核心梗
```

核心梗要足够抽象，越抽象，可变化空间越大；核心梗要足够精简，能用少数构件复现情绪链，就不要保留多余元素。

### 关键判断

- 核心梗不是整本书才有，一级结构、二级结构、三级结构都可以有核心梗。
- 小说驱动力是矛盾，核心梗必须和具体矛盾结合才能展开成剧情。
- 收集具体情节没有尽头，收集核心梗才有复用价值。
- “英雄救美”不是固定场面，而是“对象遇到困难，主角解决，关系好感提升”的情绪链。

### 可迁移写法

拆任何样本时，不要停在“这里写了什么”，要继续追问：

```text
这个情节让读者产生了什么情绪？
这个情绪来自哪个环境/人物/起因/经过/结果特征？
删掉哪些具体外壳后，情绪链还能成立？
换成我的题材后，谁承担这个人物功能？
```

### 训练任务

从一个自己喜欢的爽点中抽取“环境、人物、起因、经过、结果”，删到只剩能复现情绪链的最小结构，再用另一个题材重写一次。

## 003：简单捡钱教程，最快的教会你什么叫故事循环！

人工查看范围：正文摘要、评论中关于“发生在别人身上，再发生在主角身上”的解释、玄幻换地图循环、诱惑/抵抗循环、双循环讨论。

### 读者承诺

解决新人“知道题材但不知道怎么循环写长”的问题。它把循环从抽象理论降成可执行框架：先让配角/环境演示一次后果，再让主角进入同类局面，读者自然期待主角如何应对。

### 核心机制

```text
别人先经历同类事件 -> 读者看到后果/规则 -> 主角遭遇同类事件 -> 主角用差异化方式应对 -> 对比产生爽点/期待 -> 换壳继续循环
```

### 典型循环

- 诱惑循环：配角没扛住诱惑死亡，主角遇到同类诱惑并成功抵抗。
- 换地图循环：山脉将军 -> 国王公爵 -> 教会 -> 神明 -> 位面，每层换外壳但保留结构。
- 副本循环：别人过不了副本，主角带着信息差或金手指通关。

### 可迁移写法

给主角上场前加一个“配角预演”：

```text
配角遭遇问题：
配角失败结果：
主角看到/听到/知道了什么：
主角遭遇同类问题：
主角差异化应对：
新循环外壳：
```

### 训练任务

用自己的题材写 3 轮循环。每轮保持同一核心情绪，但更换人物身份、场景、阻碍方式和主角解法。

## 004：宝剑刷三江：《科技入侵现代》

人工查看范围：知识库可见正文摘要和全部可见评论。

证据限制：该知识库文件只保留星球文章摘要，正文分析未完整落入 Markdown。本卡只基于可见信息精拆，不冒充完整拆解原文全文。

### 读者承诺

两界穿梭 + 科技入侵现代 + 学霸震惊。读者期待主角拿出异界/高维科技，在现代学术或产业场景中制造认知差和权威震惊。

### 核心机制

```text
现代权威认知边界 -> 主角掌握跨界科技 -> 用短平快事件展示 -> 学霸/专家震惊 -> 资源和身份反馈 -> 下一个科技场景
```

### 可见案例价值

评论强调“标准爽文震惊流程”“细节到位”“在意科学细节的读者被筛掉，吃这个爽点的读者留下”。这说明该类文的重点不是硬科普严谨，而是让目标读者相信“主角的科技优势能击穿现代权威”。

### 可迁移写法

科技震惊流必须有三个见证层：

1. 懂行的人看出离谱。
2. 权威机构或学霸给出震惊反馈。
3. 现实资源随之变化：论文、项目、投资、身份、舆论。

### 训练任务

写一个 1500 字短平快场景：主角拿出一项跨界技术，先让普通人不懂，再让专业人士震惊，最后给出资源反馈。

## 005：7000字详尽解析好评如潮的“黄金三章”

人工查看范围：知识库可见正文摘要、评论中对《太监能有什么坏心思》开篇结构、日记体、悬念抛收、金手指、信息拆分的讨论。

证据限制：该知识库文件只保留文章摘要和评论，未保留 7000 字主体分析。本卡只基于可见证据精拆。

### 读者承诺

用一个小众但有强戏剧性的开篇，证明“黄金三章”不是机械模板，而是高密度悬念、信息回收、人物趣味和金手指展示的组合。

### 核心机制

```text
书名/身份反差 -> 日记体轻松入口 -> 多个悬念抛出 -> 悬念分批回收 -> 金手指出现并解决问题 -> 第三章形成完整好评闭环
```

### 关键观察

- 评论指出“悬念的抛出+收回，一波一波节奏顺畅”。
- 第一章是精华，读起来顺，复盘后发现背后有逻辑。
- 日记体本来可能无聊，但通过悬疑、回收、诙谐基调和金手指出现形成吸引力。
- 新人可降低难度，不必一章抛无数悬念，做到“抛 3 收 2”也有训练价值。

### 可迁移写法

开篇信息不要一次讲完，而是拆成悬念批次：

```text
悬念 A：身份/处境
悬念 B：世界规则
悬念 C：金手指或异常
第一轮回收：
第二轮回收：
新悬念：
```

### 训练任务

用自己的开篇第一章设计 3 个悬念，其中至少回收 2 个，再在章末抛出一个更大的新期待。
