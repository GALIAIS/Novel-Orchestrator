## 686：#思考分析首先感谢宝剑大佬，萌...

人工查看范围：正文、全部 16 条评论、图片 1 张。无附件。

图片证据：图片为聊天截图，对方称“千字20买断”，并提醒可以往低了讲价，但要先确定书名和想法。评论中作者补充网站是纵横，编辑山猫给了修改意见。

### 读者承诺

这篇是新人买断过稿复盘。作者把开头主干稳定在“直面生死危机、主角靠手段活下来、立人设、稍微透露金手指并让它起关键作用、杂鱼打压后爆发”上，真正反复修改的是世界观信息如何让读者看懂。它承诺的是：新人开头先留人，再展示差异化设定，不能让读者云里雾里。

### 核心机制

```text
投稿不过/反复改稿 -> 保留生死危机和金手指主干 -> 世界观信息明牌化 -> 读者先看懂 -> 买断过稿
```

### 期待链与情绪

```text
主角遭遇危机 -> 用手段求生 -> 金手指产生关键作用 -> 读者看到差异化世界观 -> 编辑认可修改方向 -> 作者获得过稿反馈
```

情绪是从扑街焦虑到阶段性确认。作者原以为主角不知道的信息不能提，后来意识到旁白可以承担“读者明白、主角不明白”的说明功能。

### 关键判断

- 开头主干要抓住危机、人设、金手指、爆发。
- 新人不能指望读者耐心等世界观慢慢揭开。
- 新名词太多会让读者烦躁。
- 世界观区别点要展示，但要展示得明白。
- 旁白和 NPC 都可以作为低成本说明工具。
- 编辑反馈能帮助作者把“自己懂”改成“读者懂”。

### 可迁移写法

过稿开头检查表：

```text
开头危机：
主角活下来的手段：
主角人设：
金手指出现位置：
金手指关键作用：
杂鱼打压/爆发点：
差异化世界观：
读者是否看懂：
NPC/旁白说明：
新名词数量：
```

### 训练任务

把一个设定复杂的开头改成 800 字以内小纲：必须包含生死危机、主角求生手段、金手指关键作用、一次爆发，并用 NPC 或旁白在 100 字内解释一个核心世界观差异。

## 687：刺猬猫签约要拖三天吗？

人工查看范围：正文、全部 2 条评论。无附件，无图片。

### 读者承诺

这篇是刺猬猫签约等待的流程经验：首次签约可能比较慢，因为编辑人少，而且当时审核正忙；后续写完 10000 字可以直接发给编辑。它给作者的承诺是预期管理，不是官方时限。

### 核心机制

```text
签约申请/首次签约 -> 编辑人手少 -> 审核忙 -> 等待变长 -> 后续可直接发编辑
```

### 期待链与情绪

```text
作者等待签约 -> 担心是否拖三天异常 -> 评论解释人手和审核压力 -> 作者降低焦虑
```

情绪是流程焦虑被经验信息缓冲。评论没有给官方承诺，只给“习惯就好”和“第一次比较慢”的经验。

### 关键判断

- 首次签约慢不一定异常。
- 刺猬猫编辑人少会影响处理速度。
- 审核严查期间编辑更忙。
- 后续沟通路径可能更直接：写到 10000 字后发编辑。
- 不能把“三天”写成固定官方周期。

### 可迁移写法

签约等待 FAQ 表：

```text
平台：
当前阶段：
是否首次签约：
等待时长：
可能原因：
下一步动作：
是否官方承诺：
后续沟通方式：
```

### 训练任务

写一条 200 字以内的签约等待 FAQ：说明首次签约为什么可能慢、作者能做什么、哪些信息不能当官方承诺。

## 688：宝剑老师，我发现光有小纲表格还...

人工查看范围：正文、全部 4 条评论、图片 1 张。无附件。

图片证据：图片为修女角色插画，偏调侃配图；正文真正信息是“小纲表格不够，需要小纲+正文”。评论认同新人总结能力良莠不齐，正文才能看出切入视角、画面和重点是否写到点上。

### 读者承诺

这篇修正了“公开处刑只看小纲”的方法：小纲能表达作者意图，但正文才能验证执行。有些作者的小纲和实际想表达的东西完全不是一个世界，所以公开处刑最好采用“小纲 + 正文”模式。

### 核心机制

```text
小纲概括意图 -> 新人总结可能失真 -> 正文暴露执行偏差 -> 小纲正文对照 -> 更准确诊断
```

### 期待链与情绪

```text
评审者看小纲以为清楚 -> 正文发现视角/画面/重点偏移 -> 需要双证据判断 -> 公开处刑效率提高
```

情绪是尴尬和务实。评论里的复联 4 例子说明：同一个故事被错误概括后，会完全改变读者判断。

### 关键判断

- 小纲只能说明作者“以为自己写了什么”。
- 正文能说明作者“实际写出来了什么”。
- 新人常总结不清重点。
- 即使总结正确，正文的切入视角、画面、笔力也可能不到位。
- 小纲和正文对照比单看任一方更可靠。

### 可迁移写法

小纲正文对照表：

```text
小纲承诺：
正文实际呈现：
核心事件是否一致：
视角是否偏移：
画面是否支撑重点：
情绪是否兑现：
读者理解是否等于作者意图：
修改优先级：
```

### 训练任务

拿一组“小纲 + 正文”，分别判断小纲是否抓住重点、正文是否兑现小纲、正文是否有视角/画面/重点偏移，并写出一条修改建议。

## 689：啊，我的菈妮老婆，已经很旧没看...

人工查看范围：正文。无评论，无附件，无图片。

### 读者承诺

这篇从《艾尔登法环》菈妮的人设细节里提炼傲娇写法：真正好吃的傲娇不是只有“傲”，而是在关键亲密仪式里通过小动作露出“娇”。读者要的不是角色口头表白，而是高冷角色在克制动作中泄露期待。

### 核心机制

```text
高冷外壳 -> 交出关键物件 -> 任务/仪式推进 -> 小动作暴露期待 -> 读者确认她也愿意
```

### 期待链与情绪

```text
给钥匙 -> 去拿戒指 -> 戒指描述命中关系期待 -> 戴戒指时轻抬无名指 -> 玩家获得满足
```

情绪是“她嘴上不说，但身体细节已经承认”。作者因此直接修改自己作品女主人设，说明小动作比标签更能迁移。

### 关键判断

- 傲娇要同时有傲和娇。
- 亲密推进最好绑定物件或仪式。
- 小动作要克制、轻微、可观察。
- 角色不能直接表白，小动作才更甜。
- 物件描述能提前给读者关系期待。

### 可迁移写法

傲娇仪式推进表：

```text
关键物件：
女主表面态度：
男主任务：
物件描述：
仪式场景：
女主小动作：
读者识别出的真实期待：
关系推进结果：
```

### 训练任务

写 300 字以内男女主互动：女主表面冷淡，把关键物件交给男主；男主完成任务后归还或佩戴；女主不能直接表白，只能通过一个极小动作暴露期待。

## 690：今天谈到“公开处刑”发开头不如...

人工查看范围：正文、全部 13 条评论、图片 1 张、附件状态。附件 `开头小纲填写表格.docx` 仅有文件 id，未提供下载链接，本地导出目录未找到文件。

图片证据：图片展示“开头小纲填写表格”和“排查项目”。左侧要求填写预备书名、简介、第一章到第五章小纲，并按剧情发生顺序有逻辑简述。注意事项强调填写小纲后讨论、冲突带来的危机、主角行动决定的理由、金手指作用、凸显人设片段、夹点达成片段等。排查项目包括：第一章切实开始冒险、利用金手指解决冲突、金手指展示潜力、冲突发生在主角身上、冲突够激烈且原因合理、主角有主动行动选择、行动动机合理、行动必要且堵住其他退路、行动有赌注/收益、风险很大、遭受不公平伤害、利用金手指时用到手弩理论、帮主角达成转折、详细展示金手指作用方式、反转强烈、关键时刻展示人设、达成目标方式与人设价值观挂钩。

### 读者承诺

这篇是“公开处刑”结构化工具的原型。它把发正文求评改成先填开头小纲和 18 元素排查表，让评审者围绕剧情内容讨论，而不是在长正文里低效找问题。

### 核心机制

```text
下载表格 -> 填 5 章小纲 -> 按注意事项自查 -> 填 18 元素 -> 发公开处刑区 -> 聚焦剧情反馈
```

### 期待链与情绪

```text
新人想求评 -> 标准化表达开头 -> 评论者快速看懂剧情结构 -> 找出金手指/冲突/主动性问题 -> 表格迭代
```

评论整体认可“方便评论、清晰明了”。也指出无敌文、灵异文等可能套不上，作者回应后续可做无敌文版本，灵异文不是一个路数。

### 关键判断

- 开头小纲比直接发正文更便于讨论剧情。
- 表格至少覆盖前 5 章或第一个金手指装逼冲突。
- 新人少用倒叙、插叙、蒙太奇、大篇幅氛围开头。
- 18 元素不是所有类型通用，无敌文、灵异文需要变体。
- 结构化工具应随反馈迭代。

### 可迁移写法

开头结构化处刑表：

```text
预备书名：
简介：
前 5 章小纲：
第一场冒险：
金手指解决的冲突：
主角主动选择：
风险/赌注/收益：
不公平伤害：
反转：
人设展示：
目标达成方式与价值观：
不适用类型：
```

### 训练任务

为自己的开头填一份 5 章小纲，并标出第一个“使用金手指装逼的冲突情节”。再按“主角主动性、冲突强度、金手指作用、风险收益、人设展示”五项自查，每项只写通过或不通过及一句原因。
