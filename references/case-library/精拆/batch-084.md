## 416：最近在烦恼“战力”该如何做设定...

人工查看范围：正文、全部 6 条评论。无附件，无图片。

### 读者承诺

这篇讨论战力体系时，核心是复杂度和收益的取舍。新人不必追求《诡秘之主》式序列体系；那需要知识量、设定能力和探秘结构支撑。多数作者更适合先用简单、可控、不易崩的等级或尺度体系，把精力留给剧情、人物和爽点兑现。

### 核心机制

```text
战力设定焦虑 -> 区分有限制/无限制 -> 评估脑力成本 -> 选择可控体系 -> 避免复杂设定反噬
```

### 期待链与情绪

```text
作者想让战力真实又丰富 -> 复杂体系看起来高级 -> 意识到收益低成本高 -> 退回可控等级 -> 降低崩坏风险
```

### 关键判断

- 战力设定可粗分为有限制和无限制。
- 无限制体系如斗罗、斗破、JOJO 替身，能力空间大，方便剧情展开。
- 有限制体系不能死成游戏职业表，否则战斗会枯燥。
- 《诡秘之主》序列是在框架内固定能力，再用封印物扩展变化。
- 从某种意义上，封印物又让限制体系接近无限制。
- 复杂体系需要作者大量脑力维护。
- 做不好会有反效果。
- 做好了也可能只是满足部分设定党，属于锦上添花。
- 多数新人知识量不足，不适合硬做诡秘式序列。
- 宝剑判断：别说新人，绝大部分老手都整不了诡秘那种方式。
- 如果走诡秘式探秘，核心卖点是解密、探秘者和探秘者的自我牺牲。
- 探秘者更像解密眼睛，较难共情，需要牺牲和奉献桥段补共情。
- 十进制等级更好写，但真实感可能较弱。
- 更复杂的等级越容易崩，没有金刚钻不揽瓷器活。
- 可按对人/对楼/对军/对城/对国，或民用/警用/军用/战术/战略等尺度分级。

### 可迁移写法

战力体系取舍表：

```text
题材：
是否需要复杂战力：
有限制/无限制：
读者主要追设定还是剧情：
作者能维护的层级数：
能力变化来源：
是否有外部扩展物：
最容易崩的点：
可用简单等级：
复杂设定的实际收益：
```

### 训练任务

给自己的作品写两版战力体系：一版十进制简表，一版复杂序列表。分别写 3 场战斗，看哪一版更容易制造剧情和爽点。若复杂版不能明显提高体验，就用简表。

## 417：#参考资料 #拆书/影视作品 ...

人工查看范围：正文。原文无评论、无附件、无图片。

### 读者承诺

这篇很短，但给出青春校园素材的取样方向：韩国短剧《因为男高，所以幸福》适合写青春校园小说的人参考，因为它真实、轻松，并且是短剧结构，一个个桥段本身就是可截取的素材单元。

### 核心机制

```text
青春校园需求 -> 短剧样本 -> 桥段式素材 -> 真实轻松氛围 -> 转化为日常事件
```

### 期待链与情绪

```text
作者想写校园生活 -> 找到短剧桥段样本 -> 直接获得真实轻松的小事件 -> 降低素材截取成本 -> 转成章节日常
```

### 关键判断

- 推荐对象是青春校园小说作者。
- 样本是韩国短剧《因为男高，所以幸福》。
- 素材评价是真实、轻松。
- 短剧天然是桥段式结构。
- 桥段式素材省去从长剧中截取片段的功夫。
- 校园文不一定只学小说，也可学短剧中的生活桥段。
- 可迁移重点是素材单元，而不是照搬剧情。
- 轻松真实的校园桥段能服务日常、友情、尴尬、误会和小成长。

### 可迁移写法

校园短剧素材表：

```text
短剧名：
桥段名称：
人物关系：
校园场景：
真实感来源：
轻松感来源：
能转成的小说事件：
可替换角色：
可放入章节位置：
不可照搬部分：
```

### 训练任务

看一集校园短剧，拆出 5 个桥段。每个桥段只写“场景、关系、笑点/尴尬点、可转化事件”。不要复述整集剧情。

## 418：今天询问系统教程那位星友倒是提...

人工查看范围：正文、全部 2 条评论。无附件，无图片。

### 读者承诺

这篇是教程体系模块化的雏形：把拆书、冲突与矛盾、三幕式等内容拆成独立模块，再针对不同级别新人推荐阅读学习顺序。它的价值在于把散乱教程变成可路由的学习路径，根据反馈逐个系列完善。

### 核心机制

```text
教程内容散乱 -> 拆成专题模块 -> 匹配新人级别 -> 推荐学习顺序 -> 根据反馈迭代
```

### 期待链与情绪

```text
新人不知道先学什么 -> 教程拆块降低负担 -> 按级别进入对应模块 -> 学习顺序更清楚 -> 反馈推动系列完善
```

### 关键判断

- 询问系统教程的星友提醒了模块化需求。
- 计划把【拆书】单独成块。
- 【冲突与矛盾】单独成块。
- 【三幕式】也单独成块。
- 模块化后可针对不同级别新人推荐相应阅读顺序。
- 这被称为“宝剑侍从网文教程”的模块雏形。
- 后续计划按系列完善。
- 完善依据包括用户反馈。
- 教程系统不是一篇总论，而是入口、模块和学习顺序。

### 可迁移写法

教程模块路由表：

```text
新人级别：
当前问题：
先读模块：
后读模块：
必须跳过的超纲内容：
练习任务：
反馈收集方式：
下一轮调整：
```

### 训练任务

把一个网文教程拆成 5 个模块：拆书、冲突、结构、开篇、复盘。给 0-3 级新人分别安排阅读顺序和练习任务。

## 419：想问一下，如果要扫榜，像番茄这...

人工查看范围：正文、全部 3 条评论、2 次图片引用。2 次引用为同一张截图，已下载并人工查看：`case-library\精拆\attachments\batch-084-case-419\image-1.jpg`。

图片证据：截图里有人问番茄最有含金量的榜和单部作品最有参考价值的数据是什么；评论答复包括高分榜、追更榜、阅读榜，其中北渚回复说追更榜更多相当于推荐位，高分榜也是推荐位，阅读榜更值得看。

### 读者承诺

这篇和 410 互相印证：免费平台不能照搬起点“扫畅销”的逻辑。番茄这类平台，阅读榜被认为含金量最高；高分榜、追更榜可能更多受推荐位影响。扫榜时应优先找能代表真实阅读的榜，而不是只看平台展示的热闹位置。

### 核心机制

```text
免费平台扫榜疑问 -> 排除推荐位榜 -> 优先阅读榜 -> 结合数据算法 -> 选样本拆书
```

### 期待链与情绪

```text
作者不知道免费站该扫什么 -> 收费站畅销逻辑失效 -> 评论指出阅读榜更有价值 -> 榜单选择变清楚 -> 拆书样本更可靠
```

### 关键判断

- 起点类收费站扫畅销较有价值。
- 番茄类免费站没有直接对应畅销榜的简单指标。
- 图中提到高分榜、追更榜、阅读榜。
- 评论认为阅读榜含金量最高。
- 追更榜更多像推荐位。
- 高分榜也可能是推荐位。
- 免费平台的数据算法会影响榜单解释。
- 扫榜目标是找到真实读者在看的样本，不是找被平台推的位置。
- 本帖没有给出详细算法，只能确定“阅读榜优先”这一口径。

### 可迁移写法

免费平台扫榜表：

```text
平台：
目标题材：
可见榜单：
疑似推荐位榜：
优先阅读榜：
样本在读/阅读数据：
评论活跃：
是否平台强推：
是否适合拆书：
剔除理由：
```

### 训练任务

在番茄同题材里先看阅读榜，选 20 本，再剔除明显靠推荐位上榜、评论异常少、数据异常虚高的书，留下 5 本做拆书样本。

## 420：#公开处刑 认真写完开头，已仔...

人工查看范围：正文、全部 9 条评论、文件状态。原帖标注 1 个文件：《末日：恶役大小姐反攻指南》.docx，id `548285812822254`；知识库未保留下载链接，本地附件库未找到该文件。

证据限制：无法查看作者上传的正文文件，因此本卡依据帖子正文和评论区反馈提炼，不直接评价原文全文。

### 读者承诺

这篇公开处刑的核心问题是卖点错位。作者预设卖点是“与身世复杂的大小姐斗智斗勇的恋爱冒险”，但评论读到的是末世危机、无限重生金手指、大量数据、卖肉描写、日轻嘴炮和恶役大小姐标签。多个承诺同时出现且互相抵消，导致读者不清楚到底该看末世、恋爱、卖肉、金手指还是二次元人设。

### 核心机制

```text
题材标签过多 -> 核心卖点不清 -> 危机/金手指/卖肉互相抵消 -> 读者承诺错位 -> 开篇劝退
```

### 期待链与情绪

```text
读者看到末世期待危机 -> 无限重生削弱危机 -> 看到黑丝美腿期待卖肉 -> 主角濒死破坏代入 -> 看到恶役大小姐期待恋爱喜剧 -> 恶役恶感又阻碍恋爱
```

### 关键判断

- 评论者没明白核心卖点。
- 末世文危机感被无限重生金手指消除。
- 如果不走搞笑路线，危机被消除后张力不足。
- 前三章大量数据介绍，会压低阅读动力。
- 金手指过于复杂。
- 书名和两个简介都让评论者没看懂。
- 宝剑认为偏二次元人设和故事，自己不是受众，无法判断具体剧情行不行。
- 顶尖小说往往金手指、故事、人设都吸引人，三者契合。
- 灵感可以来自剧情、金手指或人设，方向本身没有高下。
- 如今市场是卖点尽早展示，但卖点不只等于新奇金手指。
- 《曹操穿越武大郎》《我师兄实在太稳健了》《打更人》都说明金手指不新也能靠剧情人设吸引。
- 二次元这类更适合 SF、刺猬猫等站，发错站风险高。
- 若开篇写美腿黑丝，就会被读者理解为卖肉。
- 卖肉前提是男主有恃无恐、至少死不了，读者才能代入观看。
- 主角濒死时细写黑丝美腿，会显得刻意。
- 日轻嘴炮式推动角色变化，中国网文受众较窄。
- 恶役大小姐标签要处理“恶役”是否让读者产生恶感；若恶感太强，恋爱喜剧难演。
- 丧尸危机爆发时，角色应对危机有反馈，否则环境违和。

### 可迁移写法

二次元开篇卖点校准表：

```text
预设核心卖点：
读者实际读到的卖点：
题材标签：
危机是否被金手指消除：
金手指是否复杂：
是否出现卖肉描写：
卖肉时主角是否安全：
人设标签是否适配平台：
恋爱对象是否会引发恶感：
应删/前移/后移的承诺：
```

### 训练任务

拿一个二次元开篇，列出读者第一章会看到的所有承诺。只保留一个主承诺和两个副承诺；其余承诺后移。检查危机、卖肉、恋爱、金手指是否互相拆台。
