## 1051：主角+主线剧情，主角+支线剧情...


人工查看范围：正文、全部 1 条评论。无附件、无图片。

证据限制：这是知识库查找问答，不含目标原帖内容；拆解重点是创作者如何维护可检索的知识资产。

### 读者承诺

这篇沉淀的是知识库检索与关键词找回：看到有用的方法帖，不收藏会导致之后无法复用；如果忘记收藏，至少要记住关键短语，并用星球搜索找回。

### 核心机制

```text
看到方法帖 -> 忘记收藏 -> 只剩关键词 -> 搜索找回 -> 建立个人索引
```

作者想找“主角+主线剧情，主角+支线剧情”的文段，因为忘记收藏找不到。评论建议记得关键词的话，用星球右上角搜索功能试试。这个案例本身很小，但它暴露了知识库使用的底层问题：写作方法如果没有收藏、标签、关键词和二次整理，就会在需要时断链。

### 期待链与情绪

```text
创作者记得有个方法 -> 当下需要复用 -> 找不到产生焦虑 -> 搜索关键词 -> 找回后纳入索引
```

对新人来说，方法帖不是看过就完事，而是要能在写到对应问题时快速回到原文。

### 关键判断

- 有用方法帖必须收藏或记入索引。
- 标题不完整时，关键词比印象更可靠。
- 关键词应记录原文核心短语。
- 平台搜索是补救，不是长期管理方案。
- 个人知识库要按问题场景分类。
- 找不到原帖时，也要记录本次搜索词。

### 可迁移写法

```text
方法帖主题：
原文关键词：
适用问题：
收藏位置：
个人标签：
复用场景：
搜索路径：
二次摘要：
```

### 训练任务

把最近 10 条有用写作帖整理成个人索引。每条必须写：原文标题、3 个关键词、解决的问题、适用章节场景、复用时要打开的路径。

## 1052：b站刷视频刷到的，本以为是营销...


人工查看范围：正文、全部 14 条评论及楼中楼、图片区 3 张、评论图片 1 张。图片 1-3 已下载查看，内容为雪花创作法 PPT 截图；评论图由只读代理临时下载查看，为幕布思维导图版摘要。无附件。

证据限制：未打开评论里的知乎外链；截图疑似 B 站/PPT/幕布搬运材料，不能证明原始出处、方法有效性或完整课程上下文。

### 读者承诺

这篇沉淀的是雪花创作法的大纲扩写流程：给“写大纲无从下笔”的作者一条可执行路径，从一句话卖点扩成五句主线，再扩人物、段落、情节表，最后进入正文。

### 核心机制

```text
一句话卖点 -> 五句话骨架 -> 人物小传 -> 段落扩写 -> 情节表 -> 正文
```

图片显示雪花创作法强调“以大见小、层层扩张”：先用一小时写一句话总结小说，30 字以内，不要出现人名，抓最核心卖点；再把这句话扩成一段，通常包含起始背景、主要灾祸、结局等五句；然后从每个人物角度整理故事线，包括姓名、一句话故事线、动机、目标、冲突、顿悟、结局；之后把每句总结扩成段落，为主要人物写几百字描述，把情节扩成场景清单，再回头修逻辑洞。

### 期待链与情绪

```text
作者面对空白大纲 -> 先压成一句话 -> 五句确定主线 -> 人物反推剧情 -> 扩成段落和场景 -> 逻辑漏洞变得可见 -> 可以开正文
```

评论讨论了雪花法与曼陀罗写作法的区别：曼陀罗适合发散思维和素材不足的新手，但失控概率高；雪花法的价值在于把发散想法拴回结构。也有人觉得复杂、实操别扭，说明它适合当脚手架，不适合机械照搬。

### 关键判断

- 雪花法先收束再扩写。
- 一句话大纲要短、准、能推销故事。
- 五句话骨架要覆盖主线转折。
- 人物不是孤立设定，要写动机、目标、冲突、变化。
- 扩写每一步都要回头修逻辑洞。
- 曼陀罗更适合发散，雪花法更适合控结构。

### 可迁移写法

```text
30字一句话：
五句话主线：
主角动机：
主角目标：
核心冲突：
人物变化：
五段详细大纲：
20行场景表：
```

### 训练任务

用同一个故事完成 5 轮压缩/扩写：30 字一句话、5 句话大纲、3 个角色小传、5 段详细大纲、20 行场景表。每轮只修逻辑漏洞，不另开新设定。

## 1054：每日突发奇想:


人工查看范围：正文、全部 1 条评论。无附件、无图片。

证据限制：这是作者个人思考，不是平台数据结论；拆解重点是日更章节如何分配设定、剧情、人物和观感价值。

### 读者承诺

这篇沉淀的是日更章节的价值分配：每天更新不是把字数随便填满，而是把有限精力分给观感、设定、剧情、人物。当天主打什么价值，就要明确兑现什么价值，避免东拼西凑导致整体体验反而更低。

### 核心机制

```text
日更字数 -> 选择主价值 -> 用剧情人物带出 -> 保留次价值 -> 完整吃下一分
```

作者把网文当成商品，直观价值分为设定、剧情、人物、观感。观感是门面，60 分文字功底是基础；每天动笔前要决定今天的日更重点。如果今天突出新颖设定，正文必须跟新设定有关，但不能把设定糊读者脸上，要借人物和剧情带出。同时，哪怕 60% 篇幅给设定，剩下 40% 也应给剧情和人物，而不是把用于引出设定的剧情人物重复计算。

### 期待链与情绪

```text
读者打开今日更新 -> 获得基本观感 -> 感知当天核心价值 -> 设定/剧情/人物至少有一项被吃透 -> 仍保留后续危机或人物期待
```

核心提醒是：一碗水不可能面面俱到。除非能力足够每天三个方面都爆满，否则要老实找重点，完整兑现一个价值。

### 关键判断

- 观感是基础门面。
- 日更前要确定当天主价值。
- 设定价值要借剧情和人物带出。
- 剧情人物不能被设定完全吞掉。
- 有限精力要有轻重。
- 零散拼凑会导致 1+1+1 小于 1。

### 可迁移写法

```text
今日主价值：
观感底线：
设定占比：
剧情占比：
人物占比：
当天必须兑现：
次价值保留：
章末期待：
```

### 训练任务

给 7 天日更做价值分配表：每天只选一个主价值，并写清如何用剧情和人物带出它、当天保留什么次价值、章末给读者留下什么期待。

## 1055：#毒点解析


人工查看范围：正文、全部 17 条评论及楼中楼。无附件、无图片。

证据限制：这是剧情设计问答，没有具体正文和目标读者反馈；建议来自评论经验判断，不能替代成稿测试。

### 读者承诺

这篇沉淀的是主角缺席段落的存在感与下行情绪释放：两三章不写主角，对新人风险很大。即便写配角被虐，也要缩短篇幅、穿插主角线、让配角嘴里念叨主角，并把舆论抨击的压力安排好释放对象。

### 核心机制

```text
配角出征 -> 主角被提及 -> 敌人展示强度 -> 主角线穿插 -> 真相反转 -> 情绪释放
```

作者想写高地位大佬带高手灭怪，主角因人设没去，普通人抨击主角“这么强却躲着”，接着用一两章写大佬和高手被虐杀，最后主角赶到救世界。评论指出：两三章不写主角，大神能把控，新人不建议；最好缩小到一章甚至一章之内。即使主角不出现，配角嘴里也要念叨主角，为后面装逼铺垫。普通人抨击主角属于下行情绪，释放压力不仅要主角上行，也要让制造压力的一方下行，或加水军带节奏等误导源。

### 期待链与情绪

```text
读者想看主角 -> 配角团出发制造危机 -> 主角被误解但存在感不断 -> 敌人逼格成立 -> 主角因意外或判断介入 -> 对比打脸 -> 读者情绪释放
```

配角可以惨，但不能废。敌人要强，不能靠把所有人写废来衬托；主角胜利最好有自身特质、金手指或反制逻辑，而不是轻松碾过去让前面战力体系崩掉。

### 关键判断

- 两三章无主角对新人风险太大。
- 配角战斗应压缩或穿插。
- 主角缺席时也要被念叨、被误解或被对照。
- 普通人抨击会制造下行情绪。
- 舆论压力要有误导源和释放对象。
- 配角可惨不可废，敌人强度要靠有效战绩证明。

### 可迁移写法

```text
配角出征目的：
主角缺席理由：
主角存在感来源：
敌人战绩：
舆论误导源：
主角线穿插点：
情绪释放对象：
配角不废证据：
```

### 训练任务

把“两三章配角团灭”改成一章内穿插结构：配角出发 200 字、主角线异常 300 字、配角受挫 500 字、舆论误导 300 字、主角赶赴 300 字、现场反推惨状 500 字。检查 6 项：主角存在感、篇幅比例、舆论来源、释放对象、敌人逼格、配角不废。
