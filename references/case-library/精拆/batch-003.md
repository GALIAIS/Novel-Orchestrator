# Batch 003：高权重案例人工精拆

本批次继续逐篇查看高权重知识库原文。若知识库 Markdown 只保留摘要、评论或外链提示，本批次会明确写出证据限制。

## 011：写铺垫还不会放钩子拉期待？86种拉期待手法

人工查看范围：正文可见摘要、外链说明、评论中对信息差、描述、故事卡系统和使用方式的补充。

证据限制：知识库文件没有保留 86 种手法全文，只保留摘要和评论。本卡只基于可见内容提炼“钩子库使用法”，不冒充已读取 86 条完整清单。

### 读者承诺

这篇解决的是“知道要铺垫，但不会具体拉期待”的问题。它把期待感从抽象要求拆成可翻用的手法库：某一章信息不够、钩子不够、爽感阈值抬不起来时，回来找一种信息差或反差手法补进去。

### 核心机制

```text
铺垫信息 -> 制造信息差/反差/危机/悬念 -> 读者预判兑现画面 -> 延迟兑现 -> 爽点阈值抬高
```

评论中最重要的补充有两点：一是先“描述”，让读者知道某物、某人、某事有多厉害，才会产生期待；二是让主角做出费解行为，也能用好奇心制造期待。

### 可迁移写法

把钩子当作章纲补丁，而不是一次性背完：

```text
这一章读者在期待什么：
读者已知信息：
主角/配角/反派未知信息：
是否有反差：
是否有危机：
是否有悬念：
兑现前还能补哪一个描述：
```

### 训练任务

找一章“写完但没追读感”的正文，只补 3 个钩子：一个信息差、一个反差、一个费解行为。补完后检查每个钩子是否都有后续兑现位置。

## 012：新人大精品感想：什么都想要，什么都得不到

人工查看范围：正文完整阅读、评论中对 86 种拉期待、期待循环和写作取舍的反馈。

### 读者承诺

这是一个新人实战反馈案例：不需要一次吃透所有理论，先把“拉期待 -> 满足期待 -> 再拉期待”练成可重复动作，也能把作品推到明显更高的成绩区间。

### 核心机制

```text
理论过多 -> 选择能用的一招 -> 拆书验证 -> 写作中反复使用 -> 追读稳定 -> 再扩展技巧
```

作者实际只重点用了信息差和反差两类拉期待方式，没有同时使用危机、悬念等全部工具。这不是降低标准，而是在时间紧迫时先抓住最能转化为章节动力的核心动作。

### 关键判断

- “什么都想要”会让新人迟迟不上手。
- “看在眼里”不如“抓在手里用着”。
- 期待循环比堆理论更先影响追订。
- 读懂一个方法后，要通过拆书和写作验证，而不是继续收藏下一篇。

### 可迁移写法

新人阶段可以先建一个极简期待循环：

```text
本章信息差：
本章反差：
本章满足点：
满足前的新期待：
下一章继续拉什么：
```

### 训练任务

连续写 5 章，每章只强制检查两项：信息差和反差。不要新增第三种理论要求。5 章后复盘读者是否愿意追“真相揭开”和“反差兑现”。

## 013：今天拆了一本文，突然知道怎么升级冲突来把文写长了

人工查看范围：正文完整阅读、评论中《大奉打更人》假死/吟诗装逼案例、震惊层级和二级结构补充。

### 读者承诺

这篇解决“一个爽点写完就没了，冲突不知道怎么升级”的问题。它给出的办法是把震惊放进关系层级里，让同层、中层、高层逐级知道、质疑、验证、震惊，再反过来制造新矛盾。

### 核心机制

```text
同层 A 震惊 -> 同层 B 不信 -> B 验证后震惊 -> 中层不信 -> 中层验证后震惊 -> 高层不信 -> 高层介入形成新冲突
```

评论中的《大奉打更人》例子说明：同一个事件可以通过不同关系人群多次释放。主角假死，现场、官员、女侠、京城同僚、上司、家人、朝堂都能给出不同反应；主角复活后，又能反向释放一次。

### 可迁移写法

把一个装逼点拆成三层见证：

```text
同层见证者：
同层质疑者：
中层权威：
高层权威：
敌对阵营反应：
友方阵营反应：
消息传播后引发的新矛盾：
```

### 训练任务

选一个主角胜利事件，写 6 个反应：友方同层、敌方同层、中立旁观者、中层权威、高层权威、主角亲近者。每个反应必须不同，不能全写“震惊”。

## 014：有关人际关系、信息差以及情绪

人工查看范围：正文可见摘要、评论中输电网比喻、情绪缺口/期待感讨论、感情线受众取舍、斗破案例补充。

证据限制：知识库文件只保留正文开头摘要和评论，未保留完整理论正文。本卡基于可见证据提炼。

### 读者承诺

这篇把爽点从“主角赢了”改成“信息差在人际关系网里释放”。爽点不是单一因素造成的，而是情绪缺口、信息差、人际关系三者互相成就。

### 核心机制

```text
关系网配置 -> 信息差配置 -> 情绪缺口形成 -> 信息差释放 -> 关系网传导反应 -> 爽点放大
```

评论里“输电网”的比喻很关键：人际关系不是装饰，而是传递爽点的网络。信息差释放的一瞬间，如果没有足够的人际节点接收和反馈，爽点就会停在主角自己身上。

### 关键判断

- 信息差未释放时，本质上就是期待感或情绪缺口。
- 人际关系决定爽感传播效率。
- 情绪、信息差、人际关系不能拆开看。
- 不同受众吃不同线，事业线读者未必愿意看大量感情日常，狗粮文读者也可能不吃长事业线。

### 可迁移写法

写爽点前先配置电网：

```text
谁知道真相：
谁不知道真相：
谁误判主角：
谁关心主角：
谁敌视主角：
谁有资格评价主角：
信息差释放后，每个人的反应是什么：
```

### 训练任务

用一个主角能力展示事件，先写“信息差表”，再写“关系网反应表”。要求至少有 5 个关系节点，并且每个节点的反应和立场不同。

## 015：【一级结构-基础】单元故事起承转合理论

人工查看范围：正文摘要、前置阅读说明、评论中人物功能、结局反应逆推、爽点差异化、金手指机制、练习熟练度等补充。

证据限制：该文原正文标注两万五千字，但知识库 Markdown 没有保留全文。本卡只基于可见摘要和评论精拆，不冒充完整读取两万五千字主体。

### 读者承诺

这是“单元故事构思兜底方案”。它面向新人最常见的问题：不知道一个小剧情怎么起、怎么承、怎么转、怎么合，也不知道人物功能和爽点应该如何安排。

### 核心机制

```text
宽泛人物功能 -> 单元目标/冲突 -> 起承阶段铺垫和小爽 -> 转合阶段升级和大爽 -> 结局反应反推前置安排
```

评论里有几个可见关键点：人物功能可以先宽泛确立；起承阶段的爽点不要和转合抢戏；同类型爽点可以连续用，但情绪细节要差异化；可以从结局后所有人物反应，逆推开始时他们需要处在什么位置。

### 可迁移写法

单元故事卡可以这样写：

```text
起：主角进入什么局面：
承：主角做什么选择/执行：
转：局面如何升级或反转：
合：主角如何解决并获得什么：
友方反应：
敌方反应：
中立方反应：
下一单元钩子：
```

### 训练任务

用一个 3000 字小单元练起承转合。先写“合”的收获和三类人物反应，再倒推“起”里必须提前出现的人物、误判、目标和信息差。
