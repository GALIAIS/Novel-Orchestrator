## 626：求推书

人工查看范围：正文、全部 4 条评论。无附件，无图片。

### 读者承诺

这篇处理扫榜后的偏好确认：作者扫榜后不再泛泛迷茫，而是明确自己想写“帅到爆炸的主角、弱者绝境反杀、迪化幕后黑手”等类型。求推书不是为了堆书单，而是为自己的 XP 找可拆样本；评论推荐《轮回乐园》，同时提醒扫榜仍然最有效。

### 核心机制

```text
扫榜 -> 从迷茫中确认 XP -> 用关键词找样本 -> 拆主角帅点/反杀/迪化 -> 反哺选题
```

### 期待链与情绪

```text
作者不知道写什么 -> 扫榜后明确想看帅主角 -> 求同类样本 -> 用样本验证升级节奏和人设爽点
```

### 关键判断

- 扫榜可以帮助作者从泛泛选择转向明确偏好。
- “帅到爆炸”需要拆具体表现：言行一致、强行动力、绝境反杀、幕后逼格。
- 《轮回乐园》可作为老阴毕、言行一致、升级节奏样本。
- 推荐书单只是入口，最有效的仍是自己扫榜。
- 男频女频都可拆，但要围绕核心爽点，而不是只按性别频道分类。

### 可迁移写法

帅气主角样本表：

```text
我想写的帅点：
榜单样本：
主角第一印象：
绝境反杀桥段：
迪化/幕后桥段：
言行一致证据：
升级节奏：
可仿写动作：
不可照搬部分：
```

### 训练任务

找 3 本“帅主角”样本，每本只拆前 5 章：主角在哪一段显得帅，读者为什么愿意相信他帅，是否有绝境反杀或幕后迪化。

## 627：教学专栏进行了更新。

人工查看范围：正文、全部 1 条评论、图片 1 张。无附件。

图片证据：图片为《宝剑网文教程》视频使用和学习教程页面，标题标注 2022-03-11 修订，重要前言强调网文教程不是义务教育教材，学习前要判断是否适合自己。

### 读者承诺

这篇强调新手学习必须先走系统入口：写网文问题多，不要东拼西凑知识碎片，先按顺序看置顶帖第一条教学专栏，再按发帖规矩提问。系统学习不是收集技巧，而是建立共同语言和问题路由。

### 核心机制

```text
新手问题多 -> 先读置顶专栏 -> 按顺序系统学习 -> 理解发帖规矩 -> 再带具体问题讨论
```

### 期待链与情绪

```text
新人想快速解决问题 -> 碎片知识越看越乱 -> 系统专栏提供路线 -> 重新刷一遍形成学习闭环
```

### 关键判断

- 教学专栏更新后，应作为新手第一学习入口。
- 系统学习好过东拼西凑的知识碎片。
- 置顶帖第一条是入口，发帖规矩也要看。
- 图片显示教程经过多次修订，说明学习材料是持续维护的。
- 教程前言强调学习者要先判断自己是否适合这套教程，避免盲学。

### 可迁移写法

系统学习入口表：

```text
当前问题：

是否做笔记：
是否看发帖规矩：
学完后的具体问题：
需要拆的样本：
下一步训练：
```

### 训练任务

## 628：前三章开头，有无大佬说下观感，...

人工查看范围：正文、附件状态。无评论，无图片。附件 `修仙，游戏罢了.doc` 原文仅有文件 id，未提供可下载链接，导出目录未发现本地文件。

证据限制：无法查看前三章正文附件，本卡仅依据可见问题拆解。

### 读者承诺

这篇是典型“前三章观感 + 平台推荐”求评帖。由于附件不可见，不能判断正文质量；但从问题本身看，它同时包含两个任务：一是前三章是否有开篇吸引力，二是这种修仙/游戏混合类型适合投哪个平台。两者都必须以正文证据和平台榜单为准，不能只凭题材名判断。

### 核心机制

```text
提交前三章 -> 请求观感 -> 判断开篇承诺/冲突/金手指 -> 再扫平台榜单 -> 给平台建议
```

### 期待链与情绪

```text
作者想确认开篇能不能留人 -> 同时想知道平台方向 -> 若正文不可见 -> 只能记录证据限制 -> 等正文后再评判
```

### 关键判断

- 评价前三章必须看到正文，不能只看书名或题材。
- 平台推荐也要结合正文风格、节奏、受众和榜单。
- “修仙 + 游戏”可能有多个平台方向，但要看是轻小说、起点升级、刺猬猫脑洞还是免费渠道节奏。
- 求评帖应同时提供正文、简介、目标平台和作者预期读者。
- 附件不可见时，只能建立评审框架，不做质量结论。

### 可迁移写法

前三章求评框架：

```text
书名：
题材混合：
前三章承诺：
第一冲突：
金手指/核心玩法：
主角目标：
文风节奏：
目标平台：
平台榜单对照：
求评问题：
```

### 训练任务

准备一次前三章求评时，正文之外必须附 5 项：书名、简介、目标平台、同平台对标书、你最想确认的问题。

## 629：凑一波感谢给宝剑大哥，星球的讨...

人工查看范围：正文、全部 1 条评论。无附件，无图片。

### 读者承诺

这篇虽然短，但体现社群型学习的价值：好的讨论环境让成员愿意提问、复盘、互相纠错，并把单个问题转成共同成长。星主回复“有用就行，共同发展”，说明社群的目标不是单向喂答案，而是让讨论成为可复用的学习场。

### 核心机制

```text
成员参与讨论 -> 获得反馈和启发 -> 表达感谢 -> 社群确认共同发展 -> 形成持续讨论动力
```

### 期待链与情绪

```text
新人带问题进入社群 -> 讨论环境给出反馈 -> 学习获得正向体验 -> 更愿意继续输出和复盘
```

### 关键判断

- 讨论环境本身是学习资源。
- 星球价值不只在教程，也在同伴提问、案例、处刑和补充。
- “共同发展”意味着成员也要贡献分析，而不是只消费答案。
- 感谢帖可以作为社群有效性的低权重证据，但不能替代具体方法论。

### 可迁移写法

社群学习反馈表：

```text
我提出的问题：
收到的反馈：
被纠正的认知：
可复用的方法：
我能反向贡献的案例：
下一次讨论目标：
```

### 训练任务

每次社群讨论后写 5 行复盘：别人指出了什么、我学到了什么、我要怎么改、这个结论能不能复用、我下次能贡献什么。

## 630：想问一下同学们，现在有两万字是...

人工查看范围：正文、全部 2 条评论。无附件，无图片。

### 读者承诺

这篇处理投稿前“两万字要不要发”和“大纲改动”的焦虑。评论给出的核心是：发给编辑的大纲，本质是推销作品，突出卖点和潜力；正文写作中细节变动正常，大框架不变即可。作者不必因为大纲每天微调而迟迟不发，但也不能让核心卖点和主线框架漂移。

### 核心机制

```text
已有两万字 -> 反复修改导致犹豫 -> 编辑要大纲 -> 大纲用于推销卖点 -> 保持大框架 -> 正文细节可变
```

### 期待链与情绪

```text
作者想投稿 -> 又觉得每天还能改 -> 担心发后再改影响 -> 担心大纲和正文不一致 -> 明确大纲用途后降低焦虑
```

### 关键判断

- 投稿前反复改 2 万字很常见，但不能无限拖延。
- 发给编辑的大纲是推销书的材料，不是逐字执行合同。
- 大纲要突出卖点，让编辑觉得作品有潜力。
- 正文写作中大纲内容肯定会有变动。
- 大框架不变即可，细节和桥段调整可以接受。
- 关键是核心卖点、主线方向、主要人设不要频繁漂移。

### 可迁移写法

投稿大纲卖点表：

```text
当前字数：
已修改幅度：
是否达到可读版本：
核心卖点：
主线框架：
前三章承诺：
大纲用于推销的亮点：
允许变动的细节：
不能变动的大框架：
投稿时间点：
```

### 训练任务

写一版投稿大纲，只保留卖点、主线、前三章承诺、前三个高潮。再列“可改细节”和“不可改框架”两栏，防止无限拖延。
