# Batch 007：高权重案例人工精拆

## 031：如何通过章纲控制【节奏】？（一）：大纲填充成小说

人工查看范围：问答全文。

### 读者承诺

这篇继续解决“同样一句章纲，别人能写 2000 字，我只能写 200 字”的问题。关键不是单纯素材少，而是没有把核心信息扩成信息团，也没有让扩充内容带上情绪拉扯和戏剧性。

### 核心机制

```text
概括原文 -> 找到核心信息 -> 还原信息团 -> 加入情绪拉扯 -> 重新扩写成正文
```

### 关键判断

- “看了四年网文”不等于有效积累，如果没有过脑子，过后想不起来很正常。
- 把他人文章概括，再扩成字数相近的文章，是有效训练。
- 在方法论里，概括就是搞清楚信息团里的核心信息。
- 扩写不能只恢复字数，还要恢复原文为什么可读：信息展示、情绪拉扯、戏剧性、读者期待。

### 可迁移写法

还原扩写表：

```text
原文段落：
概括成一句核心信息：
原文用了哪些信息团：
情绪拉扯在哪里：
人物反应在哪里：
我重新扩写的版本：
和原文差距：
```

### 训练任务

选 3 段标杆正文，每段先压缩到一句话，再不看原文扩写回接近原字数。最后对照原文，标出自己少了哪些信息团和情绪拉扯。

## 032：如何通过章纲控制【节奏】？（二）：节奏慢

人工查看范围：问答全文、评论中无效信息团、拆头部文、爽点爆发周期和持续对比补充。

### 读者承诺

这篇解决编辑反馈“节奏慢”时该怎么判断的问题。节奏慢不等于一定不好，关键看慢的过程中，信息团是否能提起注意力、拉起期待、产生满足。

### 核心机制

```text
单元进度 -> 信息团戏剧性 -> 情绪拉扯完整度 -> 满足兑现 -> 读者是否买单
```

### 关键判断

- 好的慢：单元故事不赶，每一步情绪拉扯到位，最后满足到位。
- 坏的慢：进度慢，信息团还不吸引人，读者没有期待和爽点。
- 同样进度下，信息团多但不吸引人，会被判断为节奏慢。
- 节奏理论必须结合拆文和题材素材积累，生搬硬套写不下去。
- 外站可能偏大爽点周期，起点常见小爽点和对比持续进行，直到高潮爆发。

### 可迁移写法

慢节奏诊断表：

```text
当前单元进度：
每个信息团是否有戏剧性：
是否目标读者感兴趣：
小爽点/对比是否持续：
高潮预计位置：
当前慢是铺垫，还是无效拖延：
```

### 训练任务

拿一章被认为节奏慢的正文，逐段标注“有效信息团/无效信息团”。无效信息团要么删除，要么补目标、冲突、对比或情绪拉扯。

## 033：【自我总结帖】对系统类金手指的思考

人工查看范围：正文完整阅读、评论中“系统替代故事步骤”和“套皮系统/道具核心”补充。

### 读者承诺

这篇把系统类金手指拆成一种运行仪式：特定行为、获得介质、通过介质变强。它解决的是系统如何倒推出主线，而不是只给主角一个面板。

### 核心机制

```text
主线行为 -> 获得介质 -> 消耗/转化介质 -> 主角变强 -> 行为正义性继续加强
```

### 典型例子

模拟器系统：斩妖除魔是行为，妖魔寿元是介质，消耗寿元提升武技是变强。链条完整后，主线天然清晰：继续斩妖除魔，并不断强化这个行为的正义性。

### 关键判断

- 系统本质是强化系统，在某方面加强主角。
- 有些系统只有“行为 -> 变强”两步，不一定都有介质。
- 从故事结构看，系统可以替代发现目标、制定计划、展开行动、突破困难、解决问题、获得收获中的一个或几个步骤。
- 从金手指看，可以根据系统运行机制倒推出故事结构。
- 套皮系统本质可能只是一个道具，但如果这个道具是剧情核心，也能成立。

### 可迁移写法

系统运行卡：

```text
主线行为：
介质：
变强方式：
系统替代了故事哪一步：
这个行为为什么正当：
该行为如何持续产生事件：
下一次升级需要什么：
```

### 训练任务

给一个系统设定补完整链条：行为、介质、变强、正义性、下一轮事件。缺任何一项，都先不要开书。

## 034：金手指的构型

人工查看范围：正文可见构型目录、评论中环境差、等级差、魅力差、构建/使用视角和信息差细分补充。

证据限制：知识库 Markdown 只保留正文开头和评论，没有保留完整构型正文。本卡基于可见内容精拆。

### 读者承诺

这篇尝试像故事卡一样，把金手指抽象成构型库。它的价值在于：不只问“这个金手指强不强”，而是问它制造了哪一种差异，以及这种差异能不能在环境中持续显出主角的优势。

### 可见构型

- 信息差：隐藏型、额外型、答案型、未来型、过去型。
- 资源差：平等型、悬殊型。
- 时间差：提前型、延后型。
- 角色差：赋予型、合体型。
- 评论补充：环境差、等级差、魅力差等。

### 核心机制

```text
环境中普遍难题 -> 主角拥有差异 -> 差异解决难题更快更强 -> 他人无法复制 -> 爽感成立
```

### 关键判断

- 环境不只是世界观设定，也包括人物和势力如何使用能力。
- 主角金手指必须在环境里稀缺，否则拉不起爽感。
- “我能你不能、我有你没有、我知道你不知道”是金手指差异的基础。
- 魅力型金手指可以通过人际关系网升级体现等级差。
- 构建金手指和使用金手指是两个视角：一个看它提供什么，另一个看主角拿它做什么。
- 隐藏型是在原有基础上挖出隐藏信息；额外型是提供新的正确信息；二者戏剧性不同。

### 可迁移写法

构型检查表：

```text
金手指属于哪种差：
环境里别人为什么没有：
它能解决什么普遍难题：
别人是否能轻易复制：
差异如何升级：
使用方式和构建方式是否一致：
```

### 训练任务

把自己的金手指归入至少一种构型。写出“别人为什么做不到”和“这个差异如何连续产生 3 个事件”。

## 035：核心梗和金手指有什么不同

人工查看范围：提问和全部评论。

### 读者承诺

这篇解决新人常见混淆：金手指是主角优势的具体设定，核心梗是利用这个优势反复满足读者情绪缺口的模板。

### 核心机制

```text
金手指提供手段 -> 主角解决阻碍 -> 产生固定戏剧性 -> 重复成核心梗 -> 读者获得核心情绪
```

### 关键判断

- 金手指是外挂、工具、具体设定。
- 核心梗是核心戏剧性、核心套路、核心剧情模板。
- 金手指为核心梗服务，核心梗常常需要依靠金手指运行。
- “变身铠甲”是金手指，“打怪兽”是核心梗。
- “杀怪就能变强”是金手指，“找怪、变强、装逼打脸”是核心梗。
- 核心梗可以抽象成核心情绪，也可以具象到一套固定剧情步骤。

### 可迁移写法

分工表：

```text
主角的具体优势/外挂：
它解决什么问题：
反复出现的剧情模板：
读者期待的核心情绪：
每次循环如何换壳：
金手指是否只是设定，没有形成核心梗：
```

### 训练任务

写 5 个例子，把金手指和核心梗分开。每个例子必须写出“具体外挂”和“重复剧情模板”两行。
