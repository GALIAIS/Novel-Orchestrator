### 读者承诺

这篇沉淀的是描写资料从“收藏品”变成“工具库”的关键：OCR 让 PDF 可搜索，作者写作时可以按关键词快速找女性描写，但真正可用还要与故事情节关联。正文强调原本女性宝典没法搜索，转成 OCR 后下次写作可以搜索。评论里有人提醒：无论女性还是男性，外貌描写都应与故事情节相关，不能只追求更美。比如写女角色天真无邪，之后最好让她因为纯真善良被骗，或无意帮助他人；这样外貌和气质不是白写，而是加深读者印象并推动逻辑。对写作系统来说，OCR 资料库的价值不是复制句子，而是让作者能按“天真、冷淡、病弱、疲惫、威压、伪装”等状态检索素材，再转写成场景里的功能细节。

### 核心机制

```text
不可搜索资料 -> OCR 化 -> 按角色状态检索 -> 改写成情节相关细节 -> 后文用行为兑现印象
```

描写资料只有进入剧情功能，才不是漂亮废话。

### 期待链与情绪

```text
读者看到外貌/气质线索 -> 形成性格预期 -> 后文事件验证或反转 -> 角色印象加深
```

外貌写法要给后文留钩子，而不是只停在第一眼好看。

### 关键判断

- OCR 让资料从收藏变成可检索工具。
- PDF 正文不可读时不能臆造内容。
- 描写素材要按状态和功能归类。
- 外貌描写不能只追求更美。
- 女性和男性角色都要遵守情节相关原则。
- 天真、善良等气质要有后续情节兑现。
- 描写可以成为伏笔。
- 搜到的句子要改写，不能直接照搬。
- 角色第一印象要服务后续行为。
- 描写库应记录关键词、状态、场景功能。

## 1767：大伙如何解决写文焦虑？


人工查看范围：正文、全部 6 条评论。

证据限制：这是经验交流短帖，不含具体作品正文；本卡只拆“焦虑时休息、强制产出、细纲复位”的操作路径。

### 读者承诺

这篇沉淀的是写文焦虑不能只靠情绪鸡汤，要分辨是脑子疲劳、逃避启动、还是大纲逻辑卡住。提问者心情不好就不想写、不想继续构思大纲，问有没有充电宝。评论给出三类处理：第一类是转移和休息，随便看别的类型文、剧、电影，出去走走呼吸空气，吃东西、看漫画、打游戏，写不下去就歇着，存稿就是这个时候用的；第二类是强制机制，锁字数软件、闯关模式、拼字房、倒计时、胜负欲，让自己先码下去；第三类是结构复位，卡住时重拉前面细纲，顺一遍逻辑后自然继续。底层判断是：码字磨心态，多数时间最终还是咬牙坚持；但强撑前要先知道自己卡的是心情、启动还是故事逻辑。

### 核心机制

```text
写文焦虑 -> 判断疲劳/逃避/逻辑卡住 -> 休息转移或强制码字或重拉细纲 -> 恢复产出
```

焦虑处理不是一种药，问题类型不同，处理手段不同。

### 期待链与情绪

```text
作者写不动 -> 暂时脱离或外部压力启动 -> 细纲恢复方向 -> 重新获得推进感
```

能继续写的充电宝，往往是休息、约束和逻辑复位三件事的组合。

### 关键判断

- 心情不好写不下去可以先休息。
- 存稿用于这种低谷期。
- 换类型阅读、剧、电影可让脑子转出去。
- 散步和新鲜空气有效。
- 锁字数和拼字房适合启动困难。
- 倒计时和胜负欲能逼出产出。
- 账户余额也是现实压力。
- 卡故事时重拉细纲更有效。
- 赚钱写作多数时间要咬牙坚持。
- 彻底放弃前，总会被迫继续面对稿子。

### 可迁移写法

```text
当前焦虑类型：
疲劳/逃避/逻辑卡住：
允许休息时长：
外部强制工具：
要重拉的细纲段：
恢复后的最低字数：
存稿是否可用：
```

### 训练任务

做一张“焦虑处理表”：写不动时先勾选疲劳、逃避、逻辑卡住，再分别执行休息 30 分钟、锁 500 字、重拉前 3 章细纲。

## 1768：有感而发，个人觉得同学们的处刑...


人工查看范围：正文、全部 1 条评论。

证据限制：这是学习心得帖，不含具体处刑样稿；本卡只拆“公开处刑文如何训练毒点识别和改写能力”。

### 读者承诺

这篇沉淀的是对新人来说，有毒点的公开处刑文往往比写得好的文更容易学，因为错误更可见、改写路径更短。作者认为好的文当然要学，但新人未必一年能学会；处刑文看多、挑错挑多、私下改写多后，类似毒点出现在自己小说里时，不用等第二天就能摘下滤镜，一眼看出错误并修改到位。评论确认这就是公开的目的。核心训练不是嘲笑别人，而是把他人的问题转化为自己的识别系统：毒点类型、出现位置、读者为何出戏、如何改成更顺的版本。尤其是私下改写很关键，只看错会变挑剔，动手改才能形成肌肉记忆。

### 核心机制

```text
阅读处刑文 -> 标出毒点 -> 分析出戏原因 -> 私下改写 -> 自己正文出现时快速识别
```

公开处刑的价值在于把别人的坑提前变成自己的预警器。

### 期待链与情绪

```text
新人看见明显错误 -> 低门槛理解问题 -> 改写得到解决方案 -> 自己写作时摘下滤镜
```

错误样本越具体，学习反馈越快。

### 关键判断

- 毒点文对新人很有训练价值。
- 好文难学，错文更容易定位。
- 挑错多了能形成识别敏感度。
- 私下改写比只评论更有效。
- 摘下滤镜是重要门槛。
- 处刑样本能训练自查。
- 公开处刑目的不是羞辱作者。
- 毒点要归类沉淀。
- 自己出现同类问题时要能当天发现。
- 学习仍需继续，不等于已经会写。

### 可迁移写法

```text
处刑样本：
毒点位置：
读者出戏原因：
同类风险：
我的改写方案：
可加入自查清单：
```

### 训练任务

选 3 篇公开处刑文，每篇只挑 1 个毒点，写出 200 字改写版，并把毒点归入自己的自查清单。

## 1769：写男频玄幻小白文的作家里谁的文...


人工查看范围：正文、全部 8 条评论。

证据限制：这是文笔学习求推荐帖，不含作者样稿；本卡只拆“小白文文笔底线与商业优先级”的判断。

### 读者承诺

这篇沉淀的是小白文不是不要文笔，而是文笔优先级低于情节、爽点和阅读体验。提问者觉得自己小说太白，像小学生作文，缺形容词、比喻、铺垫、伏笔，想学遣词造句。评论分两层：一层说小白文不在乎文笔，情节好就行，真想提文笔可看知秋、江南、总管，但他们不是小白文；另一层强调第一位是语句通顺、字词达意、断句分段、标点符号，很多人不是缺文笔，而是阅读体验差。再往后是故事性和爽点才是网文首要关注点，文笔起不到决定作用；新人文笔可能比老作者好，但书未必更赚钱。提升文笔靠多看多写，有意无意看名著和好作品，强熬出来的比喻和形容容易别扭、弄巧成拙。

### 核心机制

```text
通顺达意 -> 分段标点提升阅读体验 -> 情节爽点优先 -> 文笔自然积累 -> 少量好句服务关键场景
```

小白文文笔的底线是让读者顺着读，不是让句子显得高级。

### 期待链与情绪

```text
读者无阻力读懂 -> 情节和爽点连续推进 -> 关键句稍微提神 -> 阅读体验稳定
```

文笔如果拖慢爽点，就是反向加戏。

### 关键判断

- 小白文情节好更重要。
- 文笔不是完全不需要。
- 语句通顺是第一底线。
- 字词达意比华丽更重要。
- 断句分段和标点会影响阅读体验。
- 形容词、比喻不能强熬。
- 故事性和爽点是平台商业核心。
- 文笔好不等于更赚钱。
- 多看多写能自然提升。
- 可读非小白强文笔作者，但不要误套节奏。

### 可迁移写法

```text
本段是否通顺：
是否字词达意：
分段是否过长：
标点是否影响阅读：
情节爽点是否被文笔拖慢：
需要提神的关键句：
要删掉的强行比喻：
```

### 训练任务

拿 1000 字正文，只做底线文笔修正：断句、分段、标点、删别扭形容词。不要新增大段比喻。

## 1770：看了好几个作者的开头，发现都会...


人工查看范围：正文、全部 2 条评论。

证据限制：这是开头通病分析短帖，不含具体样稿；本卡只拆“主角选择必须由剧情条件和人设逻辑逼出”的方法。

### 读者承诺

这篇沉淀的是开头容易进入上帝视角：作者想让主角干嘛，主角就干嘛，而不是主角受到剧情影响自然行动。正文指出，如果主角有多条路线，读者甲觉得该选 A，读者乙觉得该选 B，作者却让主角选 C，读者会认为这是作者自以为是，主角行为不按逻辑走，留下不严谨印象，后面再出现类似问题就会弃书。解决办法是把所有可能分叉都堵死，只留一条活路，让主线清晰、阅读轻松。评论补充，类似金手指奖励多选项时，不选的几项要带一笔分析，告诉读者主角为什么不选；主角行为方式还要贴合人设，比如偏笨拙和偏机灵的人会做不同选择。核心是：选择不是作者替主角按按钮，而是压力、信息、资源、性格共同收束出的必然结果。

### 核心机制

```text
多条可选路线 -> 展示条件限制 -> 分析不选原因 -> 人设过滤选择 -> 只剩一条活路
```

读者接受选择，前提是看见其他路为什么走不通。

### 期待链与情绪

```text
读者预判可能路线 -> 作者逐一堵死分叉 -> 主角选择符合性格 -> 主线清晰推进
```

选择越像被局势逼出来，读者越省力。

### 关键判断

- 上帝视角会破坏代入。
- 主角行为应受剧情影响。
- 多路线时要解释不选的路。
- 作者给 C 路线必须证明 A/B 走不通。
- 分叉堵死后主线更清晰。
- 金手指多选奖励也要分析不选项。
- 选择要贴合人设。
- 笨拙型和机灵型主角选择不同。
- 不严谨印象会积累成弃书。
- 轻松阅读来自逻辑收束。

### 可迁移写法

```text
当前选择点：
路线A：
路线B：
路线C：
各路线限制：
主角已知信息：
主角人设倾向：
最终唯一活路：
读者可能质疑点：
```

### 训练任务

写一个主角三选一场景。先列 A/B/C 三条路线，再在正文里各用一句堵死 A/B，最后让主角选 C，并让选择符合人设。
