## 671：最近有什么比较火的动漫吗？我扫...

人工查看范围：正文、全部 2 条评论、图片 1 张。无附件。

图片证据：图片是“得了一激动连妈都不要了”的表情包，用来表达扫榜后准备开综漫试水的兴奋和冲动。

### 读者承诺

这篇是综漫起书前的选世界求助。作者扫榜后发现起点综漫除了《轮回乐园》外样本不多，想开一本综漫试水，但不确定最近哪些二次元世界够热。它承诺的是：综漫不是随便拼世界，而要先确认当下读者熟悉、正在讨论、能带来流量的作品池。

### 核心机制

```text
扫榜发现综漫样本少 -> 想开综漫试水 -> 不确定热番世界 -> 社群求推荐 -> 用热门 IP 降低进入门槛
```

### 期待链与情绪

```text
作者看到题材机会 -> 对综漫产生冲动 -> 担心世界选择过冷 -> 求助最近热番 -> 评论给出更衣人偶/鬼灭之刃
```

情绪是题材兴奋和信息焦虑。图片表情强化了“突然想冲”的状态，但正文仍在做基本市场验证。

### 关键判断

- 综漫世界选择要看读者熟悉度和近期热度。
- 起点综漫样本少，不能只看一个头部样本就贸然复制。
- 热番能降低解释成本，但也带来版权/同质化/读者预期风险。
- “现象级”与“近期火”不是同一件事，评论区把两者分开。
- 试水前先列世界池，比直接开写更稳。

## 672：刚在qq群看见一问题：

人工查看范围：正文、全部 12 条评论、图片 1 张。无附件。

图片证据：图片是二次元萝莉表情包，文字为“是数一数二的烧姬”。它强化了评论区调侃氛围，但正文讨论的是平台尺度和题材红线。

### 读者承诺

这篇讨论一个高风险设定：成年穿越者男主和外形一米二的仙帝萝莉双修。虽然设定上女主年龄是三千万岁，但读者和平台审核首先看到的是“萝莉外形 + 性关系暗示”。它承诺给作者的不是写法建议，而是红线提醒：有受众不等于能写，平台整治时这类内容死得最快。

### 核心机制

```text
萝莉外形 -> 双修/性暗示 -> 小众受众存在 -> 平台审核高压 -> 封禁风险高于收益
```

### 期待链与情绪

```text
作者提出猎奇设定 -> 社群震惊调侃 -> 有人承认可能有受众 -> 编辑/平台红线压下 -> 最终建议别碰
```

情绪是荒诞、调侃和风险警觉。评论里“猫站编辑已经通知不准写”“双修设定最近尽量别碰”把讨论从 XP 转回平台安全。

### 关键判断

- 外形年龄比设定年龄更影响审核风险。
- “三千万岁仙帝”不能抵消萝莉外观带来的问题。
- 双修、萝莉、近期严查三者叠加，风险极高。
- 有受众不等于有平台空间。
- 题材尺度问题要先问网站编辑，不要拿整本书赌。

### 可迁移写法

平台红线判断表：

```text
角色外形年龄：
角色设定年龄：
是否有性/双修暗示：
目标平台：
近期整治方向：
编辑是否允许：
读者受众是否足以抵消风险：
可替代设定：
是否应直接放弃：
```

## 673：#公开处刑

人工查看范围：正文、全部 4 条评论、附件状态。无图片。附件 `无.txt` 有下载链接，但请求返回 401，未能查看原稿正文；本地导出目录未找到该文件。

证据限制：无法读取附件原稿，本卡依据可见简介和评论拆解。

### 读者承诺

这篇是番茄综漫同人存稿求评，标题为《综漫:从获得吉尔伽美什能力开始》。它的承诺应当是：主角获得吉尔伽美什能力后，在综漫/万界场景中快速建立压迫感，满足二次元中二幻想和装逼爽点。但评论显示开篇被系统对话、空间设定分析和主角细节矛盾拖慢。

### 核心机制

```text
综漫同人 -> 吉尔伽美什能力模板 -> 系统/天理空间 -> 万界征战 -> 装逼爽点启动
```

### 期待链与情绪

```text
读者看到吉尔伽美什能力 -> 期待王之财宝式压场 -> 开篇却长聊系统和空间 -> 设定分析打断 -> 主角反应平淡 -> 情绪起不来
```

情绪断点在“解释先于事件”。评论指出第一章系统逼逼赖赖太长、天理空间分析不适合放第一章、主角遇到魔王/天使只冷淡命名不能引发情绪波动。

### 关键判断

- 同人能力模板开篇要先展示能力，而不是先解释系统。
- 中二可以保留，但必须绑定行动、压迫和反制。
- 主角动机细节要一致：复活爱人、壁纸、住所风格不能互相打架。
- 轮回乐园式空间分析难学，普通作者第一章不宜堆。
- 读者想看主角怎么爽，不想第一章看设定论文。

### 可迁移写法

能力模板启动章表：

```text
主角能力模板：
第一场压迫：
主角第一次反制：
旁人/敌人反应：
系统必须说明的信息：
应后置的设定分析：
主角动机细节：
中二台词是否服务行动：
第一章爽点：
```

## 674：#拆书/影视作品

人工查看范围：正文、全部 2 条评论、附件状态。无图片。附件 `-638ad4f94afa05f0.jpg` 有下载链接，但请求返回 401，未能查看思维导图；本地导出目录未找到该文件。

证据限制：无法读取附件思维导图，正文中书名写作《重生之似水年华》，评论指出应为《重生之似水流年》。

### 读者承诺

这篇拆都市生活万订作品的结构，核心收获是“一个早期成果可以多次变现”。作品第一章就能看到第四个主线的铺垫，主线之间衔接顺滑；主角为什么行动，前面一直在传递，真正实施时读者接受自然。

### 核心机制

```text
早期铺垫行动理由 -> 完成作文写作 -> 获得成果 -> 多角色多场景反复反馈 -> 引出后续主线
```

### 期待链与情绪

```text
读者知道主角为什么写作文 -> 作文完成 -> 监考老师震惊 -> 同伴眼神变化 -> 女主好奇 -> 回学校再兑现 -> 新主线被牵出
```

情绪是顺畅感和持续收获感。爽点不是“一次作文一次夸”，而是同一成果跨二十多万字持续外溢。

### 关键判断

- 主线衔接要提前铺行动理由。
- 过渡内容也要为后续临近主线服务。
- 一个小成果要设计多个验收场景。
- 权威者、同龄人、女主、学校场景可以从不同角度放大同一成果。
- 每次变现最好推动关系或下一主线，不能只是重复夸主角。

### 可迁移写法

早期成果多次变现表：

```text
早期成果：
首次完成场景：
权威者反应：
同伴反应：
关键关系人反应：
家庭/学校/职场二次反馈：
引出的下一主线：
持续变现跨度：
每次反馈推动了什么：
```

### 训练任务

选一个主角早期小成果，如作文、竞赛作品、商业点子、课堂发言。设计 5 次后续变现：权威者、同伴、关键异性、家庭/学校、下一主线。每次都必须推动人物关系或下一步行动。

## 675：上了个推荐，票也没起来。差不多...

人工查看范围：正文、全部 9 条评论、图片 2 张。无附件。

图片证据：第一张为 2022-03-13 作品日报，《我在漫威开监狱 等》打赏 0、月票 0、推荐票 57、收藏 170；第二张为起点通知，作品《我在漫威开监狱》将于 2022-03-13 14:00 获得新版网站-轻小说频道新书图文推推荐，提示非完结作品保持更新。

### 读者承诺

这篇是推荐位反馈和切书判断。作者上了推荐但票数没有起来，产生“差不多该切了”的判断；评论补充推荐类型、手机端/网页端差异，并建议在推荐位上先吃完推荐。作者进一步自省：越写问题越多，发现自己对爽点理解太浅，准备写到特定剧情约 20 万字再切。

### 核心机制

```text
获得推荐曝光 -> 数据未起 -> 作者想切 -> 评论校验推荐价值 -> 归因到爽点理解浅 -> 设定观察/止损节点
```

### 期待链与情绪

```text
上推荐期待票数上涨 -> 日报显示数据低 -> 失落想切 -> 他人提醒先吃完推荐 -> 作者把失败转为学习和止损计划
```

情绪从失望转为复盘。图片把落差具体化：推荐来了，但日报数据仍是打赏 0、月票 0、推荐票 57、收藏 170。

### 关键判断

- 上推荐不等于数据一定起飞。
- 判断推荐效果要区分网页端、手机端、栏目位置。
- 推荐期间不要立刻切，先吃完推荐再观察。
- 数据差不能只怪推荐，也要回看爽点和作品吸引力。
- 切书要设节点，如推荐结束、特定剧情、字数线，而不是纯情绪决定。

推荐反馈复盘表：

