### 读者承诺

这篇把《大奉打更人》前一百章拆成“舞台准备、登台唱戏、演出收尾”三段。它承诺的不是机械套十序列，而是每个事件先把问题、负面情绪、任务目标和期待感摆上舞台，再让主角通过破案、炼金、诗词或他人扬名完成显圣，最后用震惊、评价、奖励、关系和日常调剂收尾。

### 核心机制

```text
铺垫问题与情绪 -> 主角获得登台机会 -> 用能力解决真实难题 -> 旁观者震惊认可 -> 奖励/关系/新信息收尾
```

### 期待链与情绪

```text
读者看到别人焦虑或困局 -> 知道主角可能解决但尚未解决 -> 等主角登场证明能力 -> 问题被解决并说服场内角色 -> 评价、资源和关系上升
```

### 关键判断

- “舞台准备”主要铺地点、配角、主角欲望、任务目标、负面情绪、事件重要性、悬念和期待感。
- 负面情绪不一定来自主角本人，也可以来自朱县令、魏渊等被问题困住的旁人。
- 期待感不是已经解决，而是“主角即将解决但暂未解决”的吊胃口状态。
- “登台唱戏”可以是主角亲自大显身手，也可以是旁人帮主角扬名。
- 装逼不能空喊，破案、炼金、诗词等场景必须真的解决问题，并让场内角色信服。
- “演出收尾”常见功能是震惊、领导评价升高、功法武器等奖励，以及新信息铺垫。
- 日常收尾可以立人设、增关系、调情绪、解释世界观，喜剧角色能降低紧绷感。
- 世界观最好成为破局钥匙，例如通过税银案自然引出术士和炼金术，而不是干讲设定。
- 场景要有冲突和变化，说明性场景也可以通过抢答、追问、隐瞒答案制造冲突。
- 作者和读者之间也有隐性冲突：读者想知道答案，作者暂时扣住答案。
- 下行情绪不是罗列不利因素，而是让正负判断逐层波动，最终由负面占上风。
- 悬疑要先给疑点，再给线索；线索可以误导、指向新疑点，最后用突破点推翻旧解释。
- 评论里提醒，不必困在单一理论里，十序列、作家之旅、救猫咪等都应回到阶段功能。
- 更高阶的结构抓手是开场、惊人意外、中点、第二惊人意外、大高潮、结局，中间用情绪曲线填充。

### 可迁移写法

舞台唱戏收尾表：

```text
事件名称：
舞台准备-地点：
舞台准备-配角：
舞台准备-别人负面情绪：
舞台准备-主角欲望/任务：
期待感钩子：
登台方式：亲自显圣/他人扬名
必须解决的真实问题：
场内角色如何被说服：
收尾震惊：
收尾奖励：
收尾关系变化：
收尾日常/世界观信息：
下一事件钩子：
```

### 训练任务

## 367：各位大佬用什么软件写大纲？我之...

人工查看范围：正文、全部 6 条评论。无附件，无图片。

### 读者承诺

这篇讨论三条支线以上的大纲编排工具。真正需求不是某个软件本身，而是当故事复杂后，作者需要把主线、支线、情绪上行下行、删改备注和写作阶段的动态调整可视化，避免在长文档里反复翻找。

### 核心机制

```text
多线故事 -> 文档线性浏览吃力 -> 颜色/连接线/白板分区 -> 支线关系一眼可见 -> 写作时随时调整
```

### 期待链与情绪

```text
作者想写多线交织 -> 幕布式层级不够用 -> 需要外化复杂关系 -> 工具降低混乱感 -> 回到剧情因果与情绪走势
```

### 关键判断

- 三条支线以上时，问题从“记录大纲”变成“编排多线关系”。
- 纸笔、WPS 颜色标注、XMind、MindMaster、白板、便利贴、Excel、OneNote、ProcessOn、CAD 等都只是载体。
- 能用颜色区分主线和支线，比单纯层级折叠更适合横向比较。
- 每个剧情节点用连接线连起来，可以直接看出因果、并行和交叉。
- 标注上行下行，能让作者在大纲阶段看见情绪曲线。
- 旁边加备注框，适合记录删减理由、伏笔、角色状态和待补设定。
- 白板、便利贴、打印设定适合写作阶段，因为它们能在房间里持续提醒作者。
- 复杂工具不等于复杂故事，核心仍是让作者在写作时更快判断“这一段属于哪条线、服务哪个情绪”。

### 可迁移写法

多线大纲白板表：

```text
主线颜色：
支线A颜色：
支线B颜色：
支线C颜色：
节点编号：
节点所属线：
节点情绪：上行/下行/平缓
前置节点：
后续节点：
交叉线索：
备注/删改理由：
写作时必须看见的设定：
```

### 训练任务

把一个包含主线和两条支线的十章大纲画成节点图。每个节点必须标注所属线、情绪方向、前后连接，并用一句话说明这条支线为什么必须存在。

## 368：提两嘴今日里感悟最深的：

人工查看范围：正文。无评论，无附件，无图片。

### 读者承诺

这篇提供一种把视频和文字教程转成自身能力的方法：先隔段时间默写理论，再拿默写内容和原笔记或原视频对照，发现冲突就复看并判断是自己记错还是理念不同；如果没有冲突，再主动举例、扩展适用场景，把抽象理论落实到自己的写作卡点。

### 核心机制

```text
看教程 -> 间隔默写 -> 对照笔记/原视频 -> 处理记错或理念冲突 -> 自己举例扩展 -> 理论内化
```

### 期待链与情绪

```text
作者担心学了就忘 -> 用默写暴露空洞记忆 -> 对照产生修正 -> 举例转成可用经验 -> 卡文时有方法可调用
```

### 关键判断

- 听课和记笔记不等于真正记住理论。
- 隔段时间对着看过的视频列表默写观点，可以检验自己是否只留下模糊印象。
- 默写与笔记冲突时，不要急着判定自己错，要复看并区分“记岔”和“理念冲突”。
- 理念冲突本身也有收益，因为它迫使作者说清自己为什么不同意。
- 如果默写与原理论没有冲突，还要继续从理论上升到举例。
- 举例最好使用自己的大纲、卡文点或正在写的场景，这样理论才会进入工作流。
- 这类训练费时费力，适合工作后、卡大纲或卡码字时进行。
- 把学习当成卡文时的“娱乐项目”，能降低抗拒感，但仍要产出可检查结果。

### 可迁移写法

理论内化复盘表：

```text
理论来源：
隔多久后默写：
我默写出的观点：
原笔记/原视频观点：
差异点：
属于记错还是理念冲突：
复看后的修正：
我同意/不同意的理由：
可套用的自家例子：
下一次写作时使用位置：
```

### 训练任务

选一个写作教程观点，三天后不看笔记默写。对照原笔记后，写出一个自己记错的点、一个自己保留意见的点，以及一个能马上放进当前大纲的例子。

## 369：《查理的巧克力工厂》

人工查看范围：正文、全部 11 条评论、1 张图片。图片已下载并人工查看：`case-library\精拆\attachments\batch-074-case-369\image-1.jpg`。

图片证据：图片为《查理的巧克力工厂》中查理拿到最后一张金奖券的截图，字幕为“你找到旺卡的最后一张金奖券”。

### 读者承诺

这篇讨论开局连续两次没中奖后的第三次中奖。它承诺的是“虚晃一枪”可以增强兑现，但下行必须控制成本与合理性：第一次生日巧克力没中，第二次爷爷私房钱没中，第三次街上捡钱再中奖，形成两次失望后的意外上行和小高潮。

### 核心机制

```text
稀缺大奖 -> 贫穷限制 -> 第一次希望落空 -> 第二次希望再落空 -> 意外获得新机会 -> 真正中奖兑现
```

### 期待链与情绪

```text
观众知道主角终会中奖 -> 等待中奖方式 -> 两次不中奖拉低情绪 -> 捡钱制造惊人意外 -> 最后一张金券兑现小高潮
```

### 关键判断

- 全世界只有五张金券，先天然制造稀缺期待。
- 查理家很穷，生日才吃一次巧克力，让中奖机会带上阶层压力。
- 第一次生日礼物没中，是基础下行。
- 第二次爷爷用私房钱再买仍没中，是虚晃一枪，观众会下意识以为这次该中。
- 第三次街上捡到钱买巧克力，才把情绪反转成惊人意外和小高潮。
- 如果作者在第二次就让主角中奖，情绪波动会更浅。
- 但加码下行不能乱加，爷爷拿最喜欢的怀表去换钱会显得动机离谱，甚至劝退。
- “私房钱”是温情上行里的牺牲，“怀表”更像沉重下行，若再没中，成本过高。
- 若要使用怀表，必须补足它和工厂、爷爷梦想之间的因果，否则情怀换金钱不合设定。
- 网文读者通常比电影观众更缺耐心，重复同一种麻烦容易让人厌烦。
- 虚晃不能只重复同一个障碍，最好每次下行都产生新信息、新关系或新情绪。

### 可迁移写法

虚晃兑现控制表：

```text
最终必兑现结果：
稀缺性来源：
主角为什么特别需要：
第一次希望：
第一次落空：
第二次希望：
第二次落空：
每次落空是否有新信息：
第三次意外机会：
兑现方式：
牺牲成本是否过高：
网文读者是否会提前弃文：
```

### 训练任务

写一个“主角必然拿到资格”的开篇桥段。要求先设计两次落空，再设计第三次意外兑现；两次落空不能完全重复，而且牺牲成本不能高到让读者觉得角色动机离谱。

## 370：#思考分析 #推书试读/影视/...

人工查看范围：正文。无评论，无附件，无图片。

### 读者承诺

这篇从起点轻小说前排样本里观察到：轻小说文很重视人设，头部作品能同时兼顾男女角色、完整背景和复杂套路；中下游作品常以女性人设为主要卖点，即使逻辑较弱，只要人设足够突出也能跑出来；另一条路是披着日常皮的玄幻或都市异能文，用日常来调和角色魅力与主线推进。

### 核心机制

```text
轻小说入口 -> 强人设吸引 -> 角色关系形成持续期待 -> 日常调和节奏 -> 主线/设定提供长期支撑
```

### 期待链与情绪

```text
读者先被角色吸住 -> 期待角色互动和人设反应 -> 日常提供陪伴感 -> 主线维持方向感 -> 角色与剧情双线续航
```

### 关键判断

- 起点轻小说前排样本普遍重视立人设。
- 《术师手册》这类更高阶样本，男女角色、背景设定和复杂套路都比较完整，需要大量积累。
- 中下游轻小说常以女性人设为卖点。
- 强人设可以部分弥补逻辑弱的问题，但不能自动解决长期结构。
- 菠萝包平台的头部类型拿到起点也可能有一席之地，说明人设卖点可跨平台迁移。
- 除了强人设文，还有“披着日常皮的玄幻或都市异能文”。
- 这类作品用日常调和日常和主线关系，既要角色陪伴感，也要主线方向。
- 如果作者想同时写好人设和主线，难度明显高于只卖单一人设。
- 收集素材后仍要回头总结路线，否则容易只知道“别人都好”却不知道自己该学哪一种。

### 可迁移写法

```text
女性角色卖点：

主线复杂度：
逻辑要求高低：

不能照搬的高阶部分：
```
