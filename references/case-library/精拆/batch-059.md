## 291：#参考资料 多年前的资料，很大...

人工查看范围：正文、全部 3 条评论、附件状态。附件 `查理的编剧课学习笔记.docx`、`畅销小说作家培训教材2009.doc` 下载均返回 401，无法查看正文；本卡依据可见正文和评论精拆。

证据限制：两份资料正文不可见，无法验证其具体章节内容；可见评论强调其对情节、市场、二龙枪珠拆解、策略、市场定位、结构布局、开篇方法、编辑沟通和大纲范本有扫盲价值。

### 读者承诺

这篇是资料型案例。核心判断是：旧出版/编剧培训资料虽然不是网文原生，但对新人建立情节、市场、开篇、结构、编辑沟通和大纲范本的基础认知仍有价值；使用时不能照搬出版逻辑，要按网文节奏二次细研。

### 核心机制

```text
旧出版/编剧资料 -> 建立情节和市场基础认知 -> 学开篇/结构/沟通/大纲 -> 按网文特征重筛 -> 转成新人扫盲材料
```

### 期待链与情绪

```text
新人缺系统认知 -> 旧资料提供完整框架 -> 看到市场定位和结构布局 -> 获得开篇与编辑沟通模板 -> 再用网文案例校准
```

### 关键判断

- 多年前的资料不等于无用。
- 出版向资料可以建立最初的情节和市场认知。
- 情节策略、市场定位、小说结构布局值得揣摩。
- 旧资料里的开篇方法仍可能有借鉴意义。
- 新人扫盲需要系统性材料，而不只是零散技巧。
- 编辑沟通方法也属于作者职业能力的一部分。
- 大纲范本能降低新人从想法到执行的落差。
- 使用旧资料时必须根据网文特点细研。
- 出版节奏、网文追读和平台反馈不同，不能直接套。
- 资料分享类案例要标明附件可见性，不能替附件内容脑补。
- 这类资料适合纳入新人 L0-L1 的基础阅读路径。

### 可迁移写法

旧资料网文化转译表：

```text
资料名称：
资料来源/年代：
可用内容：
情节方法：
市场定位：
开篇方法：
编辑沟通：
大纲范本：
不适合网文的部分：
需要用哪个网文案例校准：
```

### 训练任务

选一份非网文写作资料，只摘 5 条可执行方法。每条都必须配一个网文案例验证，不能验证的先放入“待观察”，不直接写进自己的创作流程。

## 292：各位大佬，没有写作激情了怎么办...

人工查看范围：正文、全部 4 条评论。无附件，无图片。

### 读者承诺

这篇讨论开头反复修改后激情耗尽。核心不是强行鸡血，而是承认改文会消耗创作欲：短期可以休息和看书，长期要么按细纲硬推进，要么总结错漏后开新书，把旧开头的一万字当成低成本试错。

### 核心机制

```text
开头写一万字 -> 朋友反馈 -> 多次修改 -> 原始创作欲被磨损 -> 休息/输入/复盘 -> 继续硬写或开新激情
```

### 期待链与情绪

```text
作者期待开书顺利 -> 开头多次改动 -> 感到不是自己想写的东西 -> 激情下降 -> 暂缓恢复或总结重开 -> 保留学习收益
```

### 关键判断

- 写作激情强求不来。
- 没激情时可以歇着，看别人书补输入。
- 开头反复修改确实会伤害创作激情。
- 修改后作品不再像自己最初想要的样子，会导致心理脱节。
- 迎合市场不等于永远失去自我，可能是为了以后获得更大创作空间。
- 如果只有一万字，沉没成本很低。
- 可以先总结这一版错漏，再开一本新的。
- 没有激情时也可以创造新激情，而不是死磕旧稿。
- 如果必须继续，休息一两天后按细纲硬写。
- 激情不是唯一生产机制，细纲和习惯能承担低谷期推进。
- 新人要区分“作品问题”与“疲劳问题”。

### 可迁移写法

开头激情耗尽处理表：

```text
当前字数：
修改次数：
修改原因：
是否偏离原始兴趣：
市场迎合部分：
自己仍想写的部分：
休息/输入计划：
继续硬写条件：
重开条件：
本稿可总结错漏：
```

### 训练任务

对一个写崩的一万字开头做 30 分钟复盘：只写 3 个错漏和 3 个仍想保留的兴奋点。若兴奋点少于 2 个，就允许重开；若不少于 2 个，按细纲再写 3000 字再判断。

## 293：请问老师，去哪里投稿编辑会有反...

人工查看范围：正文、全部 2 条评论。无附件，无图片。

### 读者承诺

这篇讨论投稿被拒后想要具体反馈。核心认知是：多数编辑没有时间精力给详细修改意见，“未达签约标准”是常态；想要反馈，要去有评判传统的渠道、编辑或社群，但不能把编辑反馈当成默认服务。

### 核心机制

```text
投稿几万字 -> 收到未达签约标准 -> 不知道从哪改 -> 认识编辑无义务详评 -> 寻找有反馈渠道 -> 自建诊断清单
```

### 期待链与情绪

```text
作者期待编辑指出问题 -> 收到笼统拒稿 -> 迷茫 -> 评论降低反馈预期 -> 转向特定渠道/模板化自评
```

### 关键判断

- “未达签约标准”通常不会附带具体病因。
- 大多数编辑没有时间和精力主动详评。
- 投稿反馈不是写作教学服务。
- 想获得反馈，要主动寻找有评判机制的渠道。
- 起点部分编辑或投稿口可能会给简单评判。
- 简单评判、模板评判和深度诊断不是一回事。
- 作者不能把是否有编辑反馈作为唯一修改依据。
- 被拒后要用开篇六要素、读者承诺、期待链、金手指、主角目标等清单自查。
- 投稿前最好准备书名、简介、前三章、大纲和明确问题。
- 反馈稀缺时，案例库和同题材榜单是替代证据来源。

### 可迁移写法

拒稿后反馈寻找表：

```text
投稿平台：
投稿字数：
收到回复：
是否有具体意见：
可寻找反馈渠道：
反馈类型：
我的自查清单：
最可能问题：
下一轮修改范围：
是否需要重新投稿：
```

### 训练任务

把一次“未达签约标准”拆成自查任务：分别用书名简介、第一章、前三章、金手指、主角目标、期待感六项各写一句问题假设，再找同题材样本验证。

## 294：有没有懂保底和买断这种形式的老...

人工查看范围：正文、全部 3 条评论、1 张图片。图片已下载并人工查看：`case-library\精拆\attachments\batch-059-case-294\image-1.jpg`。

图片证据：图片为 2019 版起点分成订阅收入表，按日更新字数与均订估算月均订收入；评论据此判断纵横保底 20 元约对应分成均订 800-1000 左右。

### 读者承诺

这篇讨论保底/买断如何换算作者能力。核心不是“保底 20 就是什么段位”的绝对定级，而是用分成订阅收入表做粗略参照：按收入看，保底 20 大致对应分成 800-1000 均订级别，但平台、年份、合同口径都会影响判断。

### 核心机制

```text
拿到保底报价 -> 不知道能力位置 -> 用分成订阅收入表参照 -> 粗略换算均订区间 -> 形成能力和合同预期
```

### 期待链与情绪

```text
作者获得纵横保底 20 -> 想知道自己水平 -> 评论提供收入表 -> 换算不到一千均订/800-1000 -> 对自身商业能力有粗定位
```

### 关键判断

- 保底和买断是合同收入形式，不等于作品真实市场上限。
- 保底 20 的“20”通常要结合千字价格、更新量和结算方式理解。
- 用分成订阅收入表只能粗略换算。
- 2019 起点表不能无差别代表所有平台和年份。
- 按评论估算，保底 20 大概对应分成均订 800-1000 左右。
- 买断/保底报价说明平台愿意为确定产能和质量支付风险成本。
- 作者用报价判断能力时，要看同平台同题材样本。
- 分成收入受日更字数、均订、渠道、税、福利影响。
- 保底适合稳定现金流，但可能牺牲上限。
- 分成适合看好上限，但风险更高。
- 合同判断不能只看“水平”，还要看版权、更新压力、结算周期和违约条款。

### 可迁移写法

保底买断能力换算表：

```text
平台：
合同形式：
千字价格/保底标准：
预计日更：
月收入估算：
参考分成均订：
同题材样本：
现金流需求：
上限预期：
合同风险：
是否接受：
```

### 训练任务

拿一个保底报价做三种测算：保守日更、正常日更、爆更日更。分别估算月收入，再和分成均订收入表对照，写出接受或拒绝的理由。

## 295：#公开处刑 《大明：最强皇婿居...

人工查看范围：正文、全部 6 条评论、附件状态。附件 `大明最强女婿居然是农民工.docx` 下载返回 401，无法查看正文；本卡依据可见正文和评论精拆。

证据限制：附件正文不可见，无法核验完整开篇；评论显示问题集中在金手指介绍太急、装逼缺铺垫、自我吐槽出戏、简介人物关系门槛、危机期待被金手指说明打断。

### 读者承诺

这篇是历史金手指开篇诊断。核心问题是期待顺序错位：读者先被攻城危机和主角窘迫钩住，却突然转去介绍金手指，导致期待落空。更好的顺序是让城墙快破、敌人志得意满、主角和敌人同时懵，再让即时筑城金手指现场发挥作用。

### 核心机制

```text
历史危机开局 -> 读者期待主角如何破局 -> 金手指说明插入 -> 情绪断掉 -> 危机顶点再展示金手指 -> 即时破局形成爽点
```

### 期待链与情绪

```text
读者看到轰轰炮声/攻城危机 -> 期待主角窘迫和底牌 -> 画面转去解释金手指 -> 期待落空 -> 改为城墙将破时现场砌墙 -> 敌我震惊
```

### 关键判断

- 金手指开端读起来太急，会让读者没来得及理解。
- 金手指解释不清，读者会直接弃书。
- 装逼情节需要前置铺垫，否则情绪拉不起来。
- 自我吐槽太多太随意，容易出戏。
- 历史+金手指题材可能更贴近飞卢生态。
- 简介里使用温青青、阿九等人物，如果读者没看过《碧血剑》，会产生门槛。
- 公共认知不足的人物关系不适合放在简介核心位置。
- 正文到炮声危机处能形成期待。
- 读者期待主角在危机情境里的窘迫和即时破局。
- 如果在危机中突然介绍金手指，期待会断。
- 即时筑城金手指应在城墙快破时出现。
- 让敌人本来志得意满，再看到城墙突然多砖，能制造强反应。
- 主角也可以先懵，再试出能力，增强发现感。
- 金手指展示最好用现场效果代替说明。

```text

```
