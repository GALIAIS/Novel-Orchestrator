## 1296：看了好多球友积极的处刑，甚至同...


人工查看范围：正文、全部 7 条评论。无附件、无图片。

证据限制：这是投稿策略讨论，没有具体样章；拆解重点是公开处刑和内投反馈如何搭配，而不是判断某本文是否能过稿。

### 读者承诺

这篇沉淀的是新人不要把同一个开头反复拿来公开处刑，先投一次稿更有性价比。处刑能得到反馈，但如果没有编辑反馈作为参照，很容易在小方向上左右摇摆，把原本没大问题的开头改坏。更优路径是先投稿，拒了且编辑给理由，再拿来处刑；如果过了但想提升、编辑又没有更好建议，再处刑也更有效。

### 核心机制

```text
开头写完 -> 先内投/投稿 -> 拿到通过或拒稿反馈 -> 再针对性处刑 -> 判断是大方向错还是小修小补
```

星球的价值是提升创作理论、技巧和经验，但一本书的开头问题往往太特定。一个想法转变可能导致整本书大改，若只是小方向，过度纠结反而破坏作者自己的判断。

### 期待链与情绪

```text
作者焦虑想要反馈 -> 公开处刑得到零散意见 -> 先用投稿取得硬反馈 -> 反馈更具体 -> 修改更有方向
```

评论里有人说“发处刑其实只是想有反馈”，这正说明问题：反馈饥渴不等于有效修改。内投要求高，通过或不回都能形成一次外部筛选；再带着筛选结果求诊断，问题会更集中。

### 关键判断

- 公开处刑可以做，但不宜同一稿反复处刑。
- 先投稿能提供更硬的外部反馈。
- 拒稿且有理由，再处刑，诊断会更准。
- 过稿后再求提升，也比盲目重改更稳。
- 开头问题高度个性化，不要被零散建议牵着走。
- 大方向错要掰，小方向分歧未必值得大改。
- 新人需要建立自信，不能把所有判断权交出去。

### 可迁移写法

```text
当前稿件状态：
是否已投稿：
编辑反馈：
反馈类型：
大方向问题：
小方向分歧：
处刑要问的问题：
本轮修改边界：
```

### 训练任务

把一个准备公开处刑的开头先列出 3 个自己最不确定的问题。若还没投稿，先标记“缺编辑反馈”；若已有拒稿理由，只围绕拒稿理由写诊断提问。

## 1297：#公开处刑


人工查看范围：正文、文件记录。无评论、无图片。文件区只有附件名“第三章-9be96175-1c59-4774-a886-cd42ddb3d470-2022-03-23 20：38：6.txt”，知识库 Markdown 未保留下载链接；本地输出目录按文件名和关键词搜索未找到该附件。

证据限制：缺少公开处刑正文样章，无法判断“麻辣味的阳春面？”具体指什么文本问题。拆解重点只能落在“附件样章缺失时不强拆”的证据规则。

### 读者承诺

这篇沉淀的是公开处刑类材料必须以样章正文为证据。只有标题、标签和附件记录时，不能凭“麻辣味的阳春面？”这句话去猜文风、题材或问题。对系统来说，这类案例仍有价值：它提醒拆书流程要把“可见正文”和“不可访问附件”分开，避免把缺失材料当成已读内容。

### 核心机制

```text
公开处刑帖 -> 样章在附件中 -> 附件不可访问 -> 只能登记证据缺口 -> 等待补齐后再诊断
```

强行拆会污染方法库。公开处刑的核心证据通常是正文样章：主角、冲突、开篇节奏、语言、场景、信息密度都要从样章里来。附件不存在时，最多记录帖子元数据和缺口。

### 期待链与情绪

```text
作者提交样章求反馈 -> 读者需要看到文本 -> 附件缺失导致诊断中断 -> 系统记录待补证据 -> 后续拿到正文再拆
```

这不是“没有价值”，而是价值在证据治理。系统越大，越需要明确哪些案例已读、哪些只是看见标题、哪些附件缺失，避免后续误以为 1297 已完成文本诊断。

### 关键判断

- 公开处刑不能脱离样章正文诊断。
- 附件名不是正文证据。
- 无评论、无样章时，不应创造问题判断。
- 可以记录标题、附件名、缺失原因和待补状态。
- 方法库要允许“证据不足”的案例存在。
- 后续若找回附件，应重新做完整精拆。
- 不强拆比编造一个看似合理的结论更可靠。

### 可迁移写法

```text
帖子类型：
可见正文：
附件名称：
附件访问状态：
可诊断内容：
不可诊断内容：
待补证据：
重新拆解条件：
```

### 训练任务

挑 3 个只有附件记录、没有正文的公开处刑帖，为每个写一张“证据缺口卡”。只写可见事实和不可判断项，不写任何剧情诊断。

## 1298：我是都市重生文，想内投一下看看...


人工查看范围：正文、1 张图片、全部 16 条评论。图片已下载到 `_source_cache\batch-260\812548882421212-image-1-image-818845542514242.jpg` 并查看，内容为起点编辑组内投数量、通过数量、通过率与通过题材表。

证据限制：图片是 2021-9-28 的统计截图，只能作为当时的社群参考，不能当作当前编辑组规则。拆解重点是内投选择决策：责编、主编、时间、无回复止损。

### 读者承诺

这篇沉淀的是新人内投不要把精力耗在“男编辑女编辑、通过率高低、投主编还是责编”的玄学上。评论给出最简判断：责编，没区别，月初投；主编也可以，但要求可能更高，不回的概率更大。内投本身难度高，没回未必值得反复猜测。

### 核心机制

```text
准备内投 -> 选匹配题材的责编/月初投 -> 等待回复 -> 有回复按意见改 -> 无回复则止损重写或换稿
```

图片中的通过率表能提供一个历史参照：不同组、不同编辑、不同题材确实有差异。但对单个新人来说，最重要的是稿件质量和题材匹配，而不是试图用表格算出必中路线。

### 期待链与情绪

```text
作者想提高通过率 -> 搜集编辑组信息 -> 选择投递对象 -> 等待结果焦虑 -> 用回复/无回复做下一步决策
```

评论里“没回就扔了重写，别的都没用”虽然激烈，但体现了止损逻辑：内投没有给出明确正反馈时，不要无限等待、猜编辑没看到还是懒得回。把焦虑转成下一稿动作。

### 关键判断

- 都市重生文内投优先找题材匹配的责编。
- 月初投是社群经验，不代表硬规则。
- 主编要求可能更高，不回概率更大。
- 历史通过率表只能参考，不可迷信。
- 内投难度高，无回复不是可操作反馈。
- 没有反馈时，不要长期纠结，考虑重写或换稿。
- 投稿策略服务稿件，不应替代稿件打磨。

### 可迁移写法

```text
题材类型：
目标编辑/组：
选择理由：
投递时间：
等待期限：
收到反馈：
无回复处理：
下一稿调整：
```

### 训练任务

为一篇待内投稿写投递卡：题材、目标责编、选择依据、等待期限、收到回复后的动作、无回复后的止损动作。不得把“再等等”作为唯一计划。

## 1299：不会写低境界时候的打斗怎么办啊...


人工查看范围：正文、全部 7 条评论。无附件、无图片。

证据限制：这是参考对象问答，没有具体打斗文本；拆解重点是低境界打斗不能照搬高境界宏大叙事，要回到武侠式动作、距离、招式和身体代价。

### 读者承诺

这篇沉淀的是低境界打斗要找低尺度参考。作者入坑辰东，写高境界战斗还好，但低境界时继续模仿遮天式宏大气势就不合适。评论给出的方向很集中：看武侠、金庸、滚开的低境界打斗。也就是说，低境界要写手、脚、兵器、身法、判断、破绽、受伤，而不是动不动宇宙边荒、天地磨灭。

### 核心机制

```text
低境界角色 -> 能力尺度有限 -> 近身动作和战术选择 -> 破绽/代价/胜负手 -> 小场面写出紧张
```

低境界的爽点不是场面规模，而是具体、可感、能想象。拳脚怎么打中、刀剑怎么逼位、主角如何预判、对手如何露破绽、伤势如何影响下一招，这些才是读者能代入的细节。

### 期待链与情绪

```text
读者知道双方实力有限 -> 关注每个动作选择 -> 看到风险和破绽 -> 胜负手出现 -> 产生紧张和爽感
```

高境界可以靠意象、气势、法则、星空尺度制造震撼；低境界要靠空间、身体、兵器和心理压迫制造临场感。尺度错位会让两个低级角色像在打灭世之战，读者反而出戏。

### 关键判断

- 低境界打斗不宜照搬高境界宏大描写。
- 武侠是低尺度战斗的重要参考。
- 金庸适合学招式、心理、攻防转换。
- 滚开适合学身体压迫、力量质感和近身狠劲。
- 低境界要写距离、角度、破绽、受伤和体力。
- 场面小不等于不爽，关键是胜负压力清楚。
- 气势描写不能替代动作因果。

### 可迁移写法

```text
双方境界：
战斗空间：
武器/能力：
第一招目的：
对手反制：
露出的破绽：
身体代价：
胜负手：
```

### 训练任务

写一场 600 字低境界打斗，禁止使用“天地、宇宙、法则、灭世”等高尺度词。必须写清楚三次攻防和一次身体受伤带来的后续影响。

## 1300：想请四姨老师看一下，我对于情绪...


人工查看范围：正文、1 张图片、全部 3 条评论。图片已下载到 `_source_cache\batch-260\184215122885812-image-1-image-818845885412182.jpg` 并查看，内容为主角潜入邪教窝点的情绪折线图：主角行动推进，队友情绪数次上升/低落，目标线同步变化。

证据限制：图片是情绪折线草图，不是完整正文；拆解重点是读者情绪、队友情绪、主角状态三者不能混为一谈，以及为什么不该隐瞒金手指让主角显得心虚。

### 读者承诺

这篇沉淀的是情绪拉扯里可以让队友情绪下行，但读者代入的核心仍然是主角。死翼的回答很明确：队友黄线处的想法会导致剧情下行，但读者状态应是“虽然情绪跟着下行一点，但知道主角肯定有办法”。这就是下行加期待感。不要隐瞒金手指，应让读者代入主角，让主角始终沉稳、自信、靠谱。

### 核心机制

```text
队友误判/失望 -> 情绪下行 -> 主角保持可靠 -> 读者知道主角有办法 -> 反转兑现期待
```

队友情绪、角色视角情绪、读者情绪不是同一个东西。队友可以慌、可以以为失败、可以虚晃一枪；读者可以被带着轻微下行，但必须保留对主角的信心。若主角自己也刚发现能力、也心虚，就会损害代入安全感。

### 期待链与情绪

```text
行动看似失败 -> 配角失望 -> 读者短暂下行 -> 主角稳住局面 -> 读者期待反转 -> 爽点兑现
```

主角受挫不是不能写，但要区分不可抗力的降维打击和主角无能。读者希望主角保持同层次下的强大与可靠，挫折最好来自超出当前层次的外力，而不是主角不知道自己在干什么。

### 关键判断

- 队友情绪和读者情绪有关联，但不是同一回事。
- 队友误判可以制造下行。
- 主角必须维持沉稳、自信、靠谱。
- 读者可以短暂下行，但要知道主角有办法。
- 不要为了悬念隐瞒金手指到让主角显得心虚。
- 主角受挫最好来自不可抗力或降维打击。
- 情绪折线的下行要同时保留期待感。

### 可迁移写法

```text
当前行动目标：
队友看到的信息：
队友情绪变化：
读者知道的信息：
主角真实把握：
下行原因：
期待保留点：
反转兑现方式：
```
