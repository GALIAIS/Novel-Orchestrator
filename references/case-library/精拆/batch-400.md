## 1996：#思考分析


人工查看范围：正文、全部 9 条评论、文中作品搜索链接标题；按要求未打开作品链接。

证据限制：帖子以一名高智商高情商、低武力角色为例，讨论弱武是否适合网文主角；没有具体情节、读者数据或长篇结构。评论对“后期应补齐武力”“可依靠强力追随者”“卖点不在武力则无妨”等存在分歧，均是经验判断。本卡不把主角最终全能力拉满当成必然规则，只拆如何让非战斗优势保持主体性，并让身体脆弱成为真实限制而非反复绑架工具。

### 读者承诺

这篇沉淀的是主角不必在每个能力维度都强，但必须在作品核心问题上不可替代。智谋型主角的主能力可以是判断利益、读取关系、组织信息、设计协作、预判反应或说服关键人物；这些能力要通过别人无法完成的选择产生结果，而不是作者在事后宣布“他早就料到”。读者需要看到问题、可用资源、限制和计划关键点，才能把胜利归因于人物能力。

低武力可以提供持续紧张：主角不能亲自闯阵、承受正面追杀、靠拳头纠正每次误判，因此必须提前布局撤退、身份保护、情报冗余和代理行动。脆弱性需要偶尔造成真实损失，例如错过窗口、计划缩小、盟友受伤或不得不公开筹码；但若每一卷都靠被抓等待救援，主角会失去主动性。

代理战力也不能是无条件忠诚的“外接手脚”。每位强力盟友应有自己的欲望、底线、能力和对主角的合作理由，可能因信息不全、价值冲突或风险过高拒绝执行。主角通过信誉、共同利益、救助、分配和正确判断维持网络；必要时，他要为让别人冒险付出政治、情感或资源成本。这样团队强大仍然属于主角的组织能力成果。

成长目标不必变成文武全能。可以补充基础自保、识别埋伏、逃脱工具、可信护卫和身体恢复，让弱点从“一碰就死”进化为“仍不能正面战斗，但能承担更高层决策风险”。若作品后期核心承诺转成个人武力征服，才需要重新评估类型错配；若承诺始终是谋略、政治或组织胜利，保留弱武反而能维持辨识度。

### 核心机制

```text
主角以非战斗能力识别关键矛盾 -> 在身体脆弱和信息限制下设计方案 -> 说服有独立利益的强力盟友协作 -> 代理行动出现偏差与成本 -> 主角现场修正完成胜利 -> 增强自保但保留核心弱武边界
```

主角不可替代的是判断和组织，盟友不可替代的是具体执行，双方都有选择与代价。

### 期待链与情绪

```text
读者看见主角无法硬闯 -> 他从信息与关系中找到可行缝隙 -> 盟友是否接受形成不确定 -> 计划执行出现偏差 -> 主角用判断完成修正 -> 弱点仍在但影响范围升级
```

紧张来自计划和身体都可能失守，爽感来自主角用自身专长夺回主动。

### 关键判断

- 帖子没有证明低武主角的普遍市场表现。
- 主角不必所有能力最终全满。
- 核心能力必须在主要矛盾中不可替代。
- 智谋胜利需要展示问题、限制和关键判断。
- “早有预料”不能替代公平铺垫。
- 身体脆弱应改变行动方案和风险。
- 弱点需要造成真实成本，不能只是假威胁。
- 反复被抓等待救援会损害主体性。
- 强力盟友要有独立利益与拒绝权。
- 忠诚和合作需要通过事件建立。
- 成长可以补自保而不抹掉核心弱点。
- 卖点若转为个人武力征服才需要重构能力向量。


### 训练任务

设计一场主角不能亲自战斗的营救：先公开全部情报与资源，让主角说服两名利益不同的强者协作；执行中必须有一人拒绝或偏离计划，主角只能靠判断、谈判和预案修正。结尾补一项自保成长，但禁止获得正面碾压能力。

## 1997：分享一下分析间谍过家家的文章，...


人工查看范围：正文、全部 3 条评论、外链标题与 URL 文本；按要求未打开哔哩哔哩影评链接或作品页面。

证据限制：正文只有“日常与非日常二重奏”的影评链接，影评内容不可见；评论建议学习人设、故事设计和把平淡日常写出高潮，同时提醒长篇日系作品需要规划字数。这些都没有具体章节证据。本卡不复述《间谍过家家》情节，只构建一套让关系日常与外部任务相互制造事件的双循环。

### 读者承诺

这篇沉淀的是日常和非日常不能写成两条互不相干的栏目。非日常线提供明确任务、秘密、敌人、期限和失败后果；日常线提供共同生活、误解、照料、身份表演和关系变化。外部任务迫使人物进入某种日常关系，日常建立的依恋、习惯和软肋又不断改变任务选择，两条线互相供能才形成二重奏。

日常场景也需要小型戏剧结构。每场至少有一个生活目标、两人以上不一致的需要、一项不能直说的信息和一次关系状态变化。吃饭、上学、购物或家庭活动本身不是内容；人物为了维持秘密、获得认可、保护某人或隐藏失误而采取行动，普通事件才会带上期待。笑点和温暖应留下新认知、承诺或债务。

非日常事件则不能只在需要刺激时随机入侵。任务推进应周期性改变日常：带来新身份、新邻居、新规则、伤势、怀疑或时间压力；日常技能和关系也要在任务中真正发挥作用。这样读者不会觉得温馨章可以跳过，也不会觉得战斗章把此前关系全部冻结。

长篇长度要由事件发动机和关系阶段决定。先列秘密还能制造多少种合理误会、核心关系要经过哪些不可逆阶段、外部任务能提供多少不同压力；当每轮只能重复隐瞒和险些暴露时，应安排真相升级、关系转型或收束，而不是靠更多同类日常延长。字数规划的对象不是固定篇幅，而是可持续变化数量。

### 核心机制

```text
非日常任务迫使角色组成或维持关系 -> 日常目标与秘密制造小冲突 -> 关系变化形成新软肋与资源 -> 外部任务利用这项变化升级 -> 任务后果回流日常 -> 秘密与关系逐阶段接近揭露
```

任务提供压力，日常提供意义，双向回流保证两种内容都不可跳过。

### 期待链与情绪

```text
读者先理解外部任务与期限 -> 看角色在普通生活中维持伪装 -> 小冲突产生温暖或笑点 -> 新关系让任务选择更困难 -> 非日常危机检验感情 -> 回到日常后状态已改变
```

日常满足陪伴，非日常制造牵引，关系变化把两种体验锁在一起。

### 关键判断

- 影评外链未读，不能归纳其具体观点。
- 不应凭作品名复述原作情节。
- 日常和任务线需要双向改变。
- 日常场景也要有目标、阻力和关系变化。
- 秘密应影响人物选择而非只制造误会笑话。
- 非日常事件不能随机插入提神。
- 日常能力和关系应在任务中回收。
- 任务后果应改变后续共同生活。
- 温馨场景不能整体跳过仍无影响。
- 长度由可持续事件发动机决定。
- 重复险些暴露会快速耗尽新鲜感。
- 秘密揭露和关系阶段需要预先设计。

### 可迁移写法

```text
非日常主任务：
失败后果与期限：
角色必须维持的日常关系：
每人不能直说的秘密：
本场日常目标：
双方不一致的需要：
日常场景的关系变化：
新增软肋或资源：
下一任务如何利用它：
任务后果怎样回流生活：
关系阶段清单：
秘密升级或揭露节点：
重复风险已高的桥段：
```

### 训练任务

设计一个任务章和两个日常章：任务章必须产生一项生活后果，两个日常章分别产生关系资源与关系软肋，下一任务同时使用两者。若删除任一日常章仍不影响后续，重写该章的状态变化。

## 1998：现在B站也开始收小说了。


人工查看范围：正文、全部 9 条评论及嵌套回复；并参照相邻案例 1985 已人工查看的同日历史活动截图。按要求未打开公众号、平台或活动页面。

证据限制：本帖正文只有一句平台收小说，评论对保底价格、质量门槛、渠道、推广和“主要为了漫改文库”的判断均未附规则或合同。相邻案例截图只能证明 2022 年活动曾宣传小说/脚本征集、千字价格起点和同步漫改，不能证明评论者推测或实际交付。本卡不重复合同核验模块，专门把“改编向稿件可能重视情节/剧本设计”当作待测试假设，整理跨媒介可视化方法。

### 读者承诺

这篇沉淀的是改编向投稿不能被理解成“小说写差也没关系”。无论交小说、脚本还是故事提案，接收方首先需要无歧义地理解人物目标、场景冲突、转折和结果；语言可以不追求华丽，却必须清楚、稳定、可执行。所谓更看重情节，只意味着每一段应容易转成角色在具体地点做出的动作，而不是允许逻辑、人物和因果松散。

适配漫画或动画的故事可以提前做四项外化。人物外化：目标、身份、关系和辨识特征能通过行为与造型显出。冲突外化：关键矛盾不是全在内心解释，而有可画的对象、动作和空间。节奏外化：每个分集有明确任务、升级和页尾/集尾问题。能力外化：金手指的规则、代价和效果能被视觉重复识别，而非只能靠长篇旁白理解。

改编测试应从场景卡开始。为每场写地点、在场人物、各自目标、障碍、可见动作、转折、视觉记忆点和离场状态；再把连续场景分成单集，检查每集是否改变任务或关系。若最重要的内容只能写成几页内心独白，需要寻找行动、道具、对话或空间选择来承载，而不是简单删掉心理复杂度。

投稿包可在小说样章之外准备一句话核心、人物关系图、首卷梗概、前三集节拍和一个代表性分镜场景，但是否提交取决于征集规则。跨媒介适配是作品层能力，版权、价格和制作交付仍需独立合同审查；不能因为故事适合漫改，就默认平台一定会改编。

### 核心机制

```text
高概念给出可视核心 -> 人物目标与能力规则外化 -> 场景卡组织动作转折和视觉记忆点 -> 分集任务形成集尾钩子 -> 样章验证阅读因果 -> 再按规则提交改编资料
```

可改编性来自情节能够被看见和分段，不来自牺牲小说基本可读性。

### 期待链与情绪

```text
读者一眼理解核心设定 -> 人物在具体场景追求目标 -> 视觉化能力制造转折 -> 每集改变任务或关系 -> 集尾提出下一问题 -> 长线秘密和成长逐步展开
```

漫画读者需要翻页/下一集牵引，小说读者同样需要清楚因果，两者可以共用事件骨架。

### 关键判断

- 本帖没有活动规则，不能确认收稿标准。
- 评论对改编目的和质量门槛只是推测。
- 改编向不等于小说可以写得混乱。
- 情节优先仍要求人物和因果清楚。
- 重要冲突需要可见动作和空间承载。
- 人物辨识度不能只靠设定表。
- 金手指规则应可重复视觉识别。
- 每集都要改变任务或关系状态。
- 集尾钩子应从本集结果推出。
- 内心复杂度应被行动化而非粗暴删除。
- 投稿附加材料必须服从实际规则。
- 可改编性与合同交付是两套独立门禁。

### 可迁移写法

```text
一句话可视核心：
主角可见目标：
角色造型与行为辨识点：
金手指可视规则：
代表场景地点：
各方目标与障碍：
关键可见动作：
场景转折：
视觉记忆点：
离场状态变化：
前三集任务：
每集结尾问题：
只能靠内心说明的内容：
可外化的道具对话或选择：
```

### 训练任务

## 1999：这两天先发个文+视频，主要是聊...


人工查看范围：正文、全部 7 条评论。

证据限制：正文只预告将根据两年半交流经验讲“网文新人四个阶段”和各阶段任务，没有给出四个阶段的名称、标准或课程正文；评论主要表示期待，并讨论“现在入行是否来得及”无法保证。本卡不能冒充原视频内容，以下四级路由是基于帖子“先识别阶段、不要跳级”的原则新建的证据型训练框架。

### 读者承诺

这篇沉淀的是“新人”不是单一状态，当前任务应由已经能稳定证明的能力决定，而不是由写了几年、自我感觉或一次成绩决定。分级的目的不是贴身份标签，而是减少同时训练过多技能：先解决最阻碍下一次完整交付的问题，达到可观察门槛后再升级。

第一级是完成与因果。目标是能独立写完场景和三章短单元，人物目标、阻力、选择、结果不丢失；毕业证据是陌生读者能复述发生了什么，作者能按计划完成。第二级是承诺与单元。目标是明确题材读者、开篇卖点、章内变化和小高潮；毕业证据是读者能说出为什么追下去，连续若干章兑现同一核心乐趣。

第三级是长线与产能。目标是把多个单元组织成完整卷，管理伏笔、人物变化、升级和稳定更新；毕业证据不是累计字数，而是一卷结束时主要承诺回收、状态改变、下一卷问题自然推出，且作者产能没有靠透支维持。第四级是商业与复现。目标是根据真实读者数据、编辑反馈、合同和完本复盘，连续项目复现自己的有效机制；一次签约或爆款只算证据之一，不自动毕业。

每级只设一个主任务和一个维持任务。一级不急着研究复杂长线，二级不同时混多种冷门玩法，三级不靠日更数字掩盖结构断裂，四级不把市场波动解释成全部个人价值。至于“现在入行来不来得及”，缺少未来信息无法保证；更可执行的问题是：能否用可承受成本完成下一门槛，并根据反馈决定是否继续。

### 核心机制

```text
收集当前作品证据 -> 判定完成因果/承诺单元/长线产能/商业复现阶段 -> 选择一个主训练任务 -> 用明确门槛验收 -> 保留旧能力 -> 升级或回退补缺
```

阶段由作品证据决定，任务由最短板决定，升级由可重复结果决定。

### 期待链与情绪

```text
新人从笼统焦虑转向具体阶段 -> 单一任务降低复杂度 -> 小门槛提供真实进展 -> 完整卷和项目建立能力证据 -> 复盘让下一次选择更可靠
```

希望来自下一步可做、可验收，不来自他人保证一定成功。

### 关键判断

- 原帖未公开四阶段内容，本卡分级是新建框架。
- 写作年限不能单独证明能力等级。
- 一次签约或成绩不能自动跳级。
- 每级必须有作品层可观察门槛。
- 完成和因果早于复杂市场策略。
- 开篇承诺要通过连续章节兑现。
- 长线阶段需要完整卷级回收。
- 产能不能建立在长期透支上。
- 商业阶段要处理数据、合同和复现。
- 每级只设一个主任务以避免过载。
- 能力退化或缺口出现时允许回退补课。
- 没有人能保证“现在入行一定来得及”。

### 读者承诺

这篇沉淀的是重写慢开头时，不能把所有解释连同必要锚点一起删除。读者在金手指出现前至少要知道：主角与异常世界是什么关系，这次进入是主动还是被迫，失败会失去什么，眼前任务要求什么，以及普通参与者拥有怎样的基础能力或准备。没有基线，特殊能力再强也无法形成差异和期待。

惊悚任务开篇可以按“日常关联—异常转移—规则告知—准备窗口—能力偏差—第一任务”推进。日常关联只交代会影响选择的信息；异常转移让角色和读者同时确认处境变化；规则告知只覆盖生存、任务和禁区；准备窗口允许主角观察他人、资源和普通面板；在这个基线后出现额外提示、特殊能力或规则漏洞，读者才能立刻理解它为何有价值。

术语要符合世界内认知。若角色真的把经历当游戏、系统公开称呼“副本”，该词可以使用；若世界人物把它视为真实空间和死亡威胁，作者单方面用游戏术语可能削弱代入。规则也应由广播、契约、主持者、环境反馈或他人经验等世界内来源传达，而不是作者隔空说明。

节奏快的判断标准是每一段都改变处境或决策，不是金手指出现得越早越好。普通规则建立一拍、金手指偏差出现一拍、第一次应用再一拍，三者很近但不能颠倒。首轮任务开始前不必解释整个惊悚世界，只需让读者能预测错误选择的代价，并看懂主角为何采取当前方案。

### 核心机制

```text
日常关系提供最低锚点 -> 异常转移确认被卷入 -> 世界内来源告知生存与任务规则 -> 准备窗口建立普通能力基线 -> 金手指显示一项偏差优势 -> 第一任务立即验证并付出代价
```

规则基线负责理解，能力偏差负责期待，任务实证负责把信息变成剧情。

### 期待链与情绪

```text
读者先知道主角为何被卷入 -> 转移事件打破安全感 -> 最少规则明确生死边界 -> 普通参与者准备形成参照 -> 主角发现异常优势 -> 第一任务检验能否活用
```

信息顺序清楚时，压缩字数会提高速度；顺序缺失时，压缩只会提高困惑。
