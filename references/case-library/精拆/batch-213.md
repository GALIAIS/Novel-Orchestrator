## 1061：这个表格是从b站up和一本写作...


人工查看范围：正文、附件元数据。无评论、无图片。附件 `个人调查报告.xlsx` 有下载链接，但请求返回未授权，未能查看表格内容。

证据限制：表格真实字段、结构和完整训练价值不可核验；只能依据正文“让角色立体化、丰满”和附件名“个人调查报告”拆解。

### 读者承诺

这篇沉淀的是角色调查表的立体化补缺：把 B 站 UP 和写作书中的角色塑造方法表格化，像做个人调查一样补全角色信息，让作者发现角色哪里空、哪里没动机、哪里没有关系压力。

### 核心机制

```text
外部方法 -> 表格化 -> 填角色资料 -> 暴露缺项 -> 回填剧情
```

作者说表格来自 B 站 UP 和一本写作书，目标是让角色更立体、更丰满，且可能还不完善。这个工具的价值不是替代正文，而是在写正文前后检查角色是否只有标签：身份、欲望、恐惧、关系、过去、当前目标、行为习惯等项目若填不出来，角色大概率还没立住。

### 期待链与情绪

```text
作者觉得角色扁平 -> 下载调查表 -> 逐项填写角色 -> 发现空白和矛盾 -> 补动机/关系/经历 -> 角色写作更稳定
```

角色调查表适合做辅助，不适合变成设定堆砌。填出来的信息必须能回流到行动、对话、选择和冲突里。

### 关键判断

- 表格适合发现角色缺项。
- 角色资料要回到正文行动中。
- 填表不是堆设定，不能全塞给读者。
- 欲望、恐惧、关系、创伤比外貌标签更重要。
- 表格本身可能不完善，需要按题材调整。
- 写完章节后要反查哪些角色信息没有体现。

### 可迁移写法

```text
角色身份：
当前欲望：
最大恐惧：
关键关系：
过去创伤：
当前目标：
行为习惯：
正文体现位置：
```

### 训练任务

自制一份角色调查表，至少包含身份、欲望、恐惧、关系、过去创伤、当前目标、行为习惯。选一个扁平角色，先写 300 字出场，再填表，再重写 300 字，对比角色变化。

## 1062：#公开处刑新人一枚，宝剑大大的...


人工查看范围：正文、全部 3 条评论、附件元数据。附件 `赏罚录.docx` 有下载链接，但请求返回未授权，未能查看小说正文。

证据限制：无法复核附件正文；评论提到“第一章前面几百字”，说明评论者可能读过附件，本次拆解只能依据作者自述和评论反馈。

### 读者承诺

这篇沉淀的是新人开篇的主角前置与铺垫压缩：读者进入第一章，先要确认主角是谁、他在做什么、哪里不对劲；在读者认识主角前铺太多设定和背景，读者记不住，也很难产生期待。

### 核心机制

```text
新人反复写开头 -> 自觉拖沓 -> 评论指出铺垫过多 -> 主角提前登场 -> 信息后移
```

作者说自己看完教程、反复写开头，对照 18PLUS 元素删掉一些不合理内容，但仍觉得节奏不流畅或拖沓，不知哪里删改。死翼耐萨里奥反馈：太短看不出太多，但第一章前几百字要精简，直接让主角来；开头铺垫的东西读者记不住，他们要先确认主角是谁。

### 期待链与情绪

```text
读者打开第一章 -> 先找主角 -> 看见动作/目标/冲突 -> 再接受必要设定 -> 愿意继续读
```

新人常把“准备充分”误写成“开头铺满信息”。开篇真正要解决的是主角识别和当前场面吸引，而不是把作者准备的设定一次性交代完。

### 关键判断

- 开头前几百字要让主角尽早出现。
- 读者认识主角前，设定记忆效率很低。
- 铺垫可以后移到动作中。
- 18PLUS 自查有用，但不能替代节奏检查。
- 太短的投稿样本也会限制反馈质量。
- 公开处刑要遵守专区发言规则。

### 可迁移写法

```text
前三句主角是否出现：
主角当前动作：
主角当前目标：
当前异常/冲突：
前置设定：
可后移信息：
无效铺垫：
读者记忆负担：
```

### 训练任务

把第一章前 500 字逐句标注为：主角信息、环境信息、设定信息、无效铺垫。删除或后移所有“读者还没认识主角前必须记住”的设定，重写 300 字，要求主角在前 3 句内出现并立刻有动作、目标或冲突。

## 1063：一些关于金手指的遐思。


人工查看范围：正文、全部 2 条评论。无附件、无图片。

证据限制：这是作者个人归纳，评论认可其对新人有帮助，但不是严格分类标准；拆解重点是金手指功能分工。

### 读者承诺

这篇沉淀的是金手指的外挂、阅历、辅助三分法：一本书的主角优势不只来自系统、宝物、老爷爷这类外挂，还来自主角自身的阅历、人设、知识、性格，以及前中期获得的功法、宠物、资讯等辅助资源。三类优势各自承担不同叙事功能。

### 核心机制

```text
外挂金手指 -> 承载主爽点
阅历金手指 -> 保留主角能动性
辅助金手指 -> 丰富冲突和奖励
```

作者把金手指分为三类：第一是外挂金手指，如系统、老爷爷、异能、神秘宝物，最直观、具体、强效，往往体现在书名简介上；第二是阅历金手指，如性格、知识、世界观、技术、重生穿越的信息差，属于主角后天或人设带来的非物质力量；第三是辅助金手指，如早期获得的功法、宠物、知识、资讯资源，未必一路帮到最高点，但前中期能帮助发展和战胜危机，也可作为解决大冲突后的奖励或伏笔。

### 期待链与情绪

```text
读者先被外显外挂吸引 -> 看到主角靠阅历主动选择 -> 通过辅助资源扩大打法 -> 大冲突后获得新奖励 -> 期待继续升级
```

三类金手指同时存在时，要避免主角成为外挂工具人。外挂解决大问题，阅历让主角像个人，辅助资源让冲突不单调。

### 关键判断

- 外挂金手指承担最大爽点和书名卖点。
- 阅历金手指保证主角主观能动性。
- 辅助金手指丰富冲突、伏笔和奖励。
- 三类金手指可共存，但功能要分清。
- 只靠外挂会让主角像工具人。
- 辅助资源不一定长期最强，但要在前中期有用。

### 可迁移写法

```text
外挂金手指：
核心爽点：
阅历金手指：
主角主动选择：
辅助金手指：
前中期用途：
奖励/伏笔：
三者分工：
```

### 训练任务

给一个主角设计三层金手指：一个外挂、一个阅历优势、一个辅助资源。分别写出它们在前三章、中期危机、一次大冲突后的用途，确保主角至少一次靠阅历而不是外挂做出关键选择。

## 1064：不知道该说什么好，今天才知道群...

证据限制：这是群友拿真人素材写文后的结果反馈，不含作品正文、题材标签和平台信息；拆解重点是熟人关系素材和 CP 互动如何转化为读者兴趣。

### 读者承诺

这篇沉淀的是熟人 CP 素材的同人化爆点：真实群聊里的关系、调侃、磕 CP 和熟人原型，本身带有强互动性和低门槛情绪。如果能转写成角色关系和连续事件，可能比作者自己抽象设计的原创开篇更容易形成读者代入。

### 核心机制

```text

```

作者说群友拿自己做女主写文，剧情全是群里拿他磕 CP 的事，上架后初订 1400；再对比自己刚开书时个位数初订，心情复杂。死翼耐萨里奥调侃“同 人 逼 死 原 创”，评论也说明不是写这位作者小说的同人，而是把这位作者本人当女主原型。

这个案例的重点不是鼓励未经同意写真人，而是提醒：能被一群人反复磕的关系，往往已经具备可读的互动张力。

- 群体反复调侃的关系可能有天然戏剧性。

- 真人原型写作要注意边界和授权。
- 关系素材比抽象设定更容易被读者感知。
- 原创开篇如果缺互动张力，可能输给熟人梗。
- 同人化不是照搬现实，而是提炼互动机制。


## 1065：如果在文里写了主角未成年抽烟o...


人工查看范围：正文、全部 2 条评论。无附件、无图片。

证据限制：这是平台审核经验问答，只有简短回复；不同平台、时期和上下文的审核尺度可能不同，本拆解只保留低风险写法原则。

### 读者承诺

这篇沉淀的是敏感行为的弱化处理：未成年抽烟、喝酒这类行为，不要在正文中刻意强调年龄与行为绑定。若它不是剧情必需，最好避开；若必须写，就减少美化、炫耀和标签化表达。

### 核心机制

```text
敏感行为 -> 不强调未成年 -> 不美化 -> 情节必要化 -> 降低审核风险
```

作者问文里写主角未成年抽烟或喝酒，会不会不过审被和谐。死翼耐萨里奥回复：不强调未成年其实没人当回事。这里的实用原则是降低触发点：不要把“未成年”作为卖点、酷点或反复强调的信息。

### 期待链与情绪

```text
作者担心审核 -> 识别敏感组合 -> 去掉强调标签 -> 保留必要情节 -> 降低被读者/审核注意的风险
```

敏感行为如果只是为了耍酷，完全可以换成更安全的压力表现；如果是人物困境，应该写后果、环境和代价，而不是写成魅力展示。

### 关键判断

- 未成年与烟酒绑定属于敏感组合。
- 不要刻意强调未成年身份。
- 不要把抽烟喝酒写成酷点。
- 非必要情节可以替换。
- 必须写时要服务人物压力或环境，不要美化。
- 平台尺度会变化，低风险写法更稳。

### 可迁移写法

```text
敏感行为：
是否剧情必需：
是否强调年龄：
是否美化：
替代表现：
行为后果：
审核风险：
保留/删除判断：
```

### 训练任务

把一个“未成年主角抽烟耍酷”的桥段改成三版：删除版、替代表现版、保留但写后果版。每版标注审核风险和人物表达效果。
