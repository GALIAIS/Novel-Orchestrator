## 711：感叹一下。

人工查看范围：正文、全部 4 条评论。无附件，无图片。

### 读者承诺

作者感叹：有大纲后剧情已经明了，但写正文时仍会发现很多细节不完善，所以还需要细纲把细节补全。评论区把新人和大神的差异说透了：新人像刚学做菜的厨子，要照着菜谱一步步来；大神可能不是没大纲，而是心里有路有地图。

### 核心机制

```text
有大纲 -> 主线明确 -> 写作时暴露细节缺口 -> 补细纲 -> 降低跑偏和现想现写风险
```

大纲解决“往哪走”，细纲解决“每一步怎么走”。对新人而言，不能误以为大神不写外显大纲，自己也能无纲裸写；大神心里有地图，新人一激动可能直接跑偏。

### 期待链与情绪

```text
作者开始按大纲写 -> 发现剧情方向清楚 -> 正文细节仍卡 -> 产生对细纲需求 -> 评论确认新人要照菜谱
```

情绪是清醒后的敬畏：大纲让作者看到路线，但细节执行仍需要工具。评论也指出日常甜文等风格未必适合过度大纲化，说明细纲不是所有作品同一强度。

### 关键判断

- 大纲和细纲不是一回事。
- 新人需要外显路线图，不能照搬大神裸写。
- 没写大纲不等于心里没大纲。
- 细节缺口应在写前尽量补，不要全靠现想。
- 日常文可能更依赖味道和即兴，但剧情型作品更需要细纲。

### 可迁移写法

大纲细纲分层表：

```text
一句话主线：
卷级目标：
章节目标：
本章冲突：
本章人物动作：
本章信息释放：
本章情绪点：
本章收尾钩子：
仍未补全的细节：
```

### 训练任务

拿一个 5 章大纲，把每章扩成细纲。每章至少写清：主角目标、出场人物、冲突、关键动作、情绪点、结尾钩子。写完后标出哪些细节原大纲没有覆盖。

## 712：不知道大家会不会为主角的装逼而...

人工查看范围：正文、全部 7 条评论。无附件，无图片。

### 读者承诺

作者问是否会为了主角装逼特地安排龙套出场，并希望登场角色能多次出现、反复利用、不轻易死亡。宝剑指出核心前提：被装逼角色的计划要和主角计划多次冲突，而且冲突逐步升级，他才会多次出现被打脸。

### 核心机制

```text
配角有自己的计划 -> 计划与主角多次冲突 -> 冲突逐步升级 -> 配角反复出现 -> 多次打脸仍合理
```

作者提出从情敌冲突转为感激、敬佩和友方，宝剑提醒这种处理很容易成为毒点，新人不建议玩；把情敌这种感情冲突换成利益冲突就稳很多。评论还举了传统层层打脸套路：小配角叫大哥，大哥叫长辈，长辈叫祖宗，最后祖宗反过来跪主角。

### 期待链与情绪

```text
配角第一次冲突 -> 被主角打脸 -> 配角升级资源/关系 -> 再次冲突 -> 再次打脸 -> 更高层人物入局
```

情绪是反复压制和反复释放。关键不是让龙套硬活着，而是让他的目标、资源和误判持续制造冲突升级。

### 关键判断

- 被装逼角色要有计划，不能只是路过挨打。
- 多次复用要靠冲突升级，不靠硬留命。
- 感情冲突转友方很难拿捏，新人容易写毒。
- 利益冲突更稳，更适合反复打脸。
- 旧套路可用，但要保证每一轮管够。

### 可迁移写法

被装逼配角复用表：

```text
配角目标：
配角资源：
第一次与主角冲突：
第一次误判：
第一次打脸结果：
配角升级方式：
第二次冲突：
是否引出更高层人物：
最终处理方式：
```

### 训练任务

设计一个可复用配角：写出他的利益目标、三次与主角冲突、每次升级的资源，以及每次被打脸后为什么还会继续入局。禁止使用“单纯仇恨”作为唯一动机。

## 713：#公开处刑 新写了一个开头，希...

人工查看范围：正文、附件元数据、全部 12 条评论。无图片。

证据限制：附件 `怪兽高武抽奖开头小纲.docx` 仅有文件名、id、大小和下载数；知识库 Markdown 与本地导出目录未找到该 docx，因此不能查看完整小纲，只能依据正文问题与评论诊断拆解。

### 读者承诺

作者新写了一个怪兽高武抽奖开头，核心问题是金手指要不要蹭热门 IP、节奏是否太快、是否扩成十章并补伏笔。宝剑反馈总体肯定：剧情构建成熟，节奏没问题，金手指第一章末尾出现更合适，且肯定要蹭热点更好。

### 核心机制

```text
新人低位/不公平处境 -> 怪兽危机 -> IP 抽奖金手指 -> 第一次能力实战 -> 翻盘装逼 -> 背后势力升级冲突
```

评论强调：主角几次翻盘都和金手指挂钩，因此展示使用金手指的过程必须写详细。如果读者因 LOL 元素进来，就要多写和 LOL 相关的东西，因为这是卖点。爽点情节不要被小纲一章限制，轮到装逼可以多煽动情绪、强化爽点。

### 期待链与情绪

```text
被不公平对待 -> 危险任务/怪兽压迫 -> 金手指触发 -> 抽到熟悉 IP 能力 -> 主角详细使用 -> 翻盘救场/打脸 -> 更高势力露头
```

情绪是不平、紧张、识别快感和翻盘爽。后续评论也提醒，被不公平对待的理由要站得住，比如空降、队内派系、亲戚顶替、队伍内部不合，而不是单纯让新人送死。

### 关键判断

- 热门 IP 能力可以蹭，但必须把使用过程写足。
- 金手指最好第一章末尾出现，尽快亮卖点。
- 节奏快不一定错，核心爽点可以扩写。
- 翻盘必须和金手指强绑定。
- 不公平伤害的逻辑要成立，否则读者会质疑队伍机制。
- 局部冲突要早埋升级线：上级、家族、导师、势力背景。

### 可迁移写法

IP 抽奖翻盘表：

```text
高压环境：
主角低位原因：
不公平任务：
怪兽危机：
抽到的 IP 能力：
读者熟悉点：
能力限制：
实战使用过程：
翻盘结果：
下一层冲突线：
```

### 训练任务

写一个 1500 到 2500 字开头：前 500 字内出现新人被不公平安排到危险任务；第一章内出现金手指；第一次抽奖抽到读者熟悉、能立刻联想到画面的 IP 能力；用 600 字以上写清第一次能力实战过程，并在结尾留下队长背后的人、被打脸者上级或导师注意三选一的升级线。

## 714：视频里的黄金三章！只能用卧槽来...

人工查看范围：正文、全部 2 条评论、外部视频链接状态。无附件，无图片。

证据限制：正文包含 B 站短链 `https://b23.tv/l5TQVGG`，网页搜索可定位到相关哔哩哔哩视频页标题“他要歪嘴了，快，撕烂他的嘴”，但未取得视频逐帧内容；因此只能依据作者评价、评论和可见页面标题拆解。

### 读者承诺

作者强烈推荐一个能体现“黄金三章”节奏的视频，评论认为虽然浮夸，但节奏确实如此，并补充“一个又一个反转”。它承诺的是用短视频样本直观感受强节奏和连续反转：观众刚形成预期，下一秒就被反转牵走。

### 核心机制

```text
强情绪开场 -> 建立预期 -> 快速反转 -> 再建立新预期 -> 再反转 -> 形成追看爽感
```

这里的“黄金三章”不是字面三章文本，而是节奏感样本：开头抓住、连续变化、不断给新刺激。浮夸本身不是问题，只要节奏和反转能让读者持续被推着走。

### 期待链与情绪

```text
作者强烈背书 -> 读者点开样本 -> 看到夸张冲突 -> 预期不断被打断 -> 评论确认节奏和反转有效
```

情绪是好奇、惊讶、发笑和被连续反转带来的爽感。对写作迁移来说，重点是学习预期管理，而不是照搬浮夸表演。

### 关键判断

- 强节奏不等于乱快，要有预期建立和反转。
- 浮夸可以作为样本，但小说要转成可读情绪链。
- 连续反转能显著提高追看欲。
- 评论背书能强化样本可信度。
- 推荐案例时最好补出可拆的反转点。

### 可迁移写法

连续反转拆解表：

```text
样本标题：
第一预期：
第一反转：
第二预期：
第二反转：
第三预期：
第三反转：
每次反转情绪：
可迁移到小说的桥段：
```

### 训练任务

找一个短视频、广告或小说片段，记录至少 3 次“观众以为会 A，结果变成 B”的反转点。写出每次反转前的预期、反转动作、产生的情绪，再改写成一条 100 字以内的推荐动态。

## 715：爽点总结很重要

人工查看范围：知识库可见正文摘要、全部 3 条评论。无附件，无图片。

证据限制：正文以文章外链形式存在，知识库只保留开头摘要和链接；未取得完整文章内容，因此依据可见摘要与评论拆解。

### 读者承诺

作者分析《重生之都市修仙》后发现，爽点密度非常高：一章至少一个爽点，很多时候一章多个。它承诺的是：章节不能只是推进流程，而要持续给读者即时反馈、身份确认、冲突期待和后续爆点。

### 核心机制

```text
普通动作 -> 意外危机 -> 主角介入 -> 即时爽点 -> 新场景入口 -> 小反派/规则压迫 -> 未来回收点
```

作者举例：主角放学，本可平淡写成找兼职；原作却让他救下差点被侵犯的美女酒吧老板，顺势获得去酒吧兼职的入口，再接触垃圾人酒吧二老板，为以后爽点埋伏笔。评论里也有人意识到自己只积累人物和情节，却没有总结爽点。

### 期待链与情绪

```text
普通日常 -> 突然出事 -> 主角出手 -> 被救者认可 -> 新资源/场景打开 -> 小反派作死 -> 等待回收打脸
```

情绪从平淡进入紧张，再到安全感、认可感、资源获得和对小反派的不爽。爽点总结的价值，就是把过渡段改成“即时反馈 + 后续期待”的组合。

### 关键判断

- 新人不能只总结人物和情节，还要总结爽点。
- 一章至少要有一个可感知爽点。
- 普通动作也能爽点化。
- 小反派和新场景要承担未来回收。
- 建立自己的爽点词典也是可行路径。

### 可迁移写法

日常爽点链表：

```text
原本普通动作：
意外危机：
主角介入方式：
即时爽点：
认可/资源入口：
新场景：
小反派/规则压迫：
未来回收方式：
```

### 训练任务

找一本同类型爽文，连续拆 10 章，标出每章的即时爽点和未来伏笔。再把“主角去找兼职”改写成 3 个版本：救美版、打脸版、资源入局版，并删除所有只负责过渡、没有情绪反馈的段落。
