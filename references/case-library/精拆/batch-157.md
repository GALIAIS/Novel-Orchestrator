人工查看范围：正文、全部 1 条评论。无附件、无图片。

### 读者承诺

这篇是视频预告，承诺用《诡秘之主》的设定，讲“大纲、矛盾、主线”的确立。它不是单纯赏析名作，而是把成熟作品的设定系统转化为可学习的结构方法。

### 核心机制

```text
名作设定 -> 提炼矛盾关系 -> 映射主线方向 -> 形成大纲图表 -> 输出方法论
```

作者特别提到“图表”折腾很久，说明这类选题不能只靠口头概念。复杂设定、阵营、力量体系和人物目标，需要被整理成图表，才能看出矛盾如何推动主线。

### 期待链与情绪

```text
释放视频进度 -> 抛出大题目 -> 暗示图表难点 -> 承诺做完就发 -> 读者期待结构拆解
```

情绪是制作现场感：稿子基本完成，但图表还在攻坚。评论“太棒了”说明读者对名作结构拆解有明确期待。

### 关键判断

- 名作拆解的价值在于抽出结构，不是复述剧情。
- 《诡秘之主》这类复杂设定适合用来讲大纲、矛盾和主线。
- 题目越大，越需要图表化支撑。
- 设定不是孤立背景，而应连接矛盾和主线。
- 预告类内容也能通过“进度 + 题目 + 难点 + 发布时间”拉期待。

## 783：#思考分析

人工查看范围：正文、图片 1 张、全部 1 条评论。无附件。

图片证据：图片已下载至 `<TEMP_EVIDENCE>\zsxq_184221155425552_1.jpg` 并人工查看。图片列出“我写得还不错”的十项标准：人设不仅做事还要文字强调；主角主线任务原因铺垫充分；情感戏；整体围绕主线发展；每个场景设计行情绪；金手指有成长性；从起点到终点完成一个任务并不断告知收益、任务和责任；手弩理论，即提前告知解决问题的能力、三次压制、反转打脸/解决；完成任务有收益且收益帮助后续发展；世界是动态的，会反过来影响主角行动。

### 读者承诺

作者提出一个自检问题：一本书做到哪些，才能拍着胸脯说写得还不错？图片给出的不是单点技巧，而是一整套从人设、主线、情绪、金手指、任务收益到动态世界的综合标准。

### 核心机制

```text
人设动机 -> 主线任务 -> 场景情绪 -> 金手指成长 -> 任务收益 -> 动态世界反作用
```

这套标准把“写得不错”拆成可检查项目：主角为什么做事、主线是否贯穿、每个场景是否有情绪设计、任务完成后是否有收益、世界是否持续发展并反过来压迫主角。

### 期待链与情绪

```text
作者追求好书标准 -> 列出多维度自检 -> 发现门槛很高 -> 形成系统训练方向
```

情绪上有一点劝退感，评论也调侃“想劝退一大波作者”。但它的价值正在于把模糊的“不错”拆成具体可练的项目。

### 关键判断

- 人设要通过选择和文字强调共同建立。
- 主角主线任务必须有充分原因和动机。
- 整体结构要围绕主线发展。
- 每个场景都应设计情绪。
- 金手指要有成长性。
- 任务链要不断告诉读者收益、目标和责任。
- 完成任务的收益要反哺后续发展。
- 世界必须是动态的，会影响主角行动。

### 可迁移写法
