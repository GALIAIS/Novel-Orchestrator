### 读者承诺

这篇沉淀的是附件投稿的证据边界：当正文主体只在附件里，而附件不可下载、不可本地定位时，拆书不能臆造内容。能记录的只有投稿行为、章节阶段、文件名和证据缺失。


作者说明第三章已写，请人查看。知识库保留了 docx 文件名、链接、ID、大小和下载数，但没有正文转写。外链下载失败后，当前可见信息无法判断这章的题材、问题或亮点。

对创作系统而言，这类案例提醒：没有正文就没有精拆，不应为了补齐批次而假装读过。

- 附件不可读时，只能拆投稿流程和证据边界。

## 1192：选题2. 主角金手指：____...


人工查看范围：正文、全部 3 条评论。无附件、无图片。

证据限制：这是五折线作业梗概，不是完整章节；拆解重点是校园格斗爽点中“受害者同情心”和“主角正义性”的维护。

### 读者承诺

这篇沉淀的是系统格斗校园爽点的关键修正：主角上行做得可以，但下行原因不能让被救对象失去同情心。死党父亲如果是赌鬼欠高利贷，读者会降低同情，主角出手的正义感也会变弱；改成被胁迫签高利贷欠条，段落就顺了。

### 核心机制

```text
死党遭混混威胁 -> 主角机智化解 -> 更大债务危机暴露 -> 混混上门 -> 主角用格斗金手指爆发
```

作业按五段写：拖把战术逼退混混、发现死党家高利贷危机、报警暂时化解、混混带协议上门搬东西、主角靠系统格斗技巧打倒混混并夺回协议。评论肯定主角上行做得好，但指出“父亲是赌鬼”会破坏读者同情。

### 期待链与情绪

```text
校园小危机 -> 主角聪明脱险 -> 家庭危机升级 -> 恶人上门压迫 -> 主角正义爆发收获敬佩
```

这类爽点要让读者站在主角这边。被救对象越无辜，主角越有正当性；被救对象自作自受越多，爽点越泄气。

### 关键判断

- 主角上行可以靠机智，不一定一开始就开打。
- 两次危机升级能拉出折线。
- 家庭债务原因会影响读者同情。
- 赌鬼父亲会让读者觉得后果自找。
- 被胁迫签欠条更能保留受害者无辜性。
- 校园格斗爽点要维护主角出手的正义感。

### 可迁移写法

```text
主角金手指：
被救对象：
危机来源：
受害者是否无辜：
第一次上行：
第二次下行：
爆发爽点：
正义性风险：
```

### 训练任务

把一个“替朋友出头”的剧情重写两版：朋友家人自作自受版、朋友家人被胁迫版。比较哪一版更能让读者支持主角出手。

## 1193：#思考分析


人工查看范围：正文。无评论、无附件、无图片。

证据限制：作者只描述事件设计，没有完整副本正文；拆解重点是童话副本中倒计时、真假目标、私奔诱惑如何形成情绪拉扯。

### 读者承诺

这篇沉淀的是“模糊技法被课程清晰化”后的自我识别：作者原本在《灰姑娘》故事世界里设计找灰姑娘、倒计时、假灰姑娘、私奔诱惑、揭露真相，其实已经在使用套路拉扯情绪，只是之前没有明确意识。

### 核心机制

```text
主角保命立约 -> 找灰姑娘看似简单 -> 倒计时压迫 -> 假目标出现 -> 矛盾升级揭真相
```

主角先用口才保命，并约定找到灰姑娘。《灰姑娘》里找灰姑娘表面容易，形成“虚晃一枪”；主角判断没那么简单。倒计时结束前，灰姑娘似乎出现，但主角识别出她不是真正的灰姑娘。对方提出私奔，矛盾无法调和，主角赢得胜利并离开故事世界。

### 期待链与情绪

```text
保命成功 -> 任务看似简单 -> 倒计时制造压力 -> 假答案让读者放松/怀疑 -> 真相揭露带来上行
```

童话副本的拉扯来自读者对原故事的熟悉感：熟悉让任务看似简单，变体让真相变得危险。

### 关键判断

- 经典童话自带读者认知，可做虚晃一枪。
- 倒计时能给解谜任务加压。
- 假目标出现不是终点，而是第二层下行的触发。
- “私奔”让童话浪漫变成道德/任务冲突。
- 现实人物回忆能补幻境危险感。
- 技法自觉化能提升作者后续复用能力。

### 可迁移写法

```text
原型故事：
主角保命条件：
看似简单的目标：
倒计时压力：
假答案：
矛盾不可调和点：
真相揭露：
离开/奖励：
```

### 训练任务

选一个童话或民间故事，设计“看似一眼能解决”的任务，再加入一个假答案和一个倒计时，让主角必须识别真正目标。

## 1194：觉得挺有道理，大家看着疗法自医...


人工查看范围：正文、全部 4 条评论、1 张图片。图片已下载到 `_source_cache\batch-239` 并人工查看。

证据限制：图片是网文综合症梗图，不是正式教程；拆解重点是新人常见自我归因偏差。

### 读者承诺

这篇沉淀的是网文新人自我诊断：常见症状包括自认能开创流派、觉得编辑有眼无珠、把成绩差归因于推荐少或境界太高、认为全世界不懂自己。对应疗法是扑了就好、饿一两天就清醒、看五白数据、必要时去找更包容的圈层。

### 核心机制

```text
新人自我评价过高 -> 作品上传遭现实反馈 -> 归因外部 -> 数据/收入校正幻觉 -> 回到可学习问题
```

图片列出四类症状：题材前所未见、编辑读者不懂高深境界、成绩不好不是自己的原因、全世界该围着自己转。评论补充“初生牛犊就是莽”，网络时代怀才不遇的文青减少，因为作品一上传就会被现实打脸。

### 期待链与情绪

```text
作者自信爆棚 -> 市场反馈不佳 -> 倾向甩锅 -> 数据打脸 -> 接受现实后才可能改
```

这类“疗法”的核心不是嘲笑新人，而是让作者停止错误归因，重新面对题材、节奏、爽点、读者适配等实际问题。

### 关键判断

- “没人见过的题材”常是未扫榜的错觉。
- “编辑不懂我”多数不能解释长期成绩差。
- 推荐少可能影响曝光，但不能解释所有问题。
- 五白数据能校正对市场的想象。
- 上传即反馈，网络平台会快速拆穿自嗨。
- 先承认问题在作品里，才有修正空间。

### 可迁移写法

```text
自我归因：
实际数据：
读者反馈：
是否扫榜：
同类成功书：
可改问题：
不能甩锅项：
下一步验证：
```

### 训练任务

列出自己最近一次扑街或反馈差的 3 个解释，把其中所有“外部原因”改写成可验证问题：题材、开头、节奏、爽点、平台适配、更新。

## 1195：手把手教你网文情绪拉扯和爽点制造【附案例分析及解析】


人工查看范围：正文、全部 6 条评论。另人工查看同日任务帖 `01-按类型\任务\2022-04-19_212522814245581_本期视频地址：网文的“情绪拉扯.md` 的任务正文和两张结构图；任务帖文字版 docx 未另行下载。

证据限制：动态正文只保留文章开头和外链，完整文章未在该 Markdown 内展开；同日任务帖提供了五折线作业、虚晃一枪、下行规则、检查项，可作为同主题补充证据。

### 读者承诺

这篇沉淀的是情绪拉扯与爽点制造的基础训练框架：用五折线把主角情绪做上行、下行、再上行、再下行，最后用金手指完成目标爆发爽点。关键要求是有虚晃一枪、下行不能让主角背锅、最后爽点必须兑现目标。

### 核心机制

```text
目标确认 -> 画五折线 -> 对立情绪上下摆动 -> 虚晃一枪制造误判 -> 金手指爆发完成目标
```

任务帖给出三道题：大神级歌喉打脸公子哥、系统格斗技巧解决混混、赛车技能破圈速。回答格式固定为五段：情绪上行、拐点下行、情绪上行、拐点下行、爽点爆发。思考步骤要求先打框架，再确认对立情绪，最后用金手指制造爽点，并用虚晃一枪设计前几段。

### 期待链与情绪

```text
读者看到目标 -> 第一次希望上升 -> 意外压低 -> 短暂缓和 -> 更强危机 -> 爽点爆发
```

拉扯不是单纯折磨主角，而是控制读者预期：让读者以为危机缓解，再用意外下行扩大期待，最终通过主角能力把情绪抬到完成目标。

### 关键判断

- 情绪拉扯是写作生涯都要磨练的尺度问题。
- 五折线适合训练结构感，但开头可能三折线更短平快。
- 下行情节必须符合“功都是主角的，锅都是别人的”。
- 虚晃一枪能让读者先放松，再被意外调动。
- 同一个结构可写多少字，没有固定范围。
- 拉扯方法很多，要靠拆书看大神如何处理。

### 可迁移写法

```text
主角金手指：
事件：
目标：
对立情绪：
1 情绪上行：
2 拐点下行：
3 情绪上行：
4 拐点下行：
5 爽点爆发：
虚晃一枪：
下行背锅检查：
```

### 训练任务

任选一个主角能力和一个小事件，按五折线只写五句梗概。写完检查三项：是否有虚晃一枪，是否让主角背锅，最后爽点是否完成目标。
