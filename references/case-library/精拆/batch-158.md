## 786：想问大家几个基础的问题，

人工查看范围：正文、全部 5 条评论。无附件、无图片。

### 读者承诺

新人连续问三个基础问题：怎么发现适合自己的题材，怎么架构整本书背景框架，怎么更快入门，是否靠仿写。评论给出的共同方向是：先从自己喜欢、熟悉、常看的题材入手，拆自己爱看的同类书，建立素材库，然后动手写，在写作中暴露问题。

### 核心机制

```text
阅读偏好 -> 题材选择 -> 拆同类书 -> 建素材库 -> 试写 -> 暴露问题 -> 修改再写
```

评论反复强调，没有“最好的入门办法”。新人只有写了，才知道问题，然后慢慢改进。背景框架可以借鉴成熟作品；仿写前不能只抄表面，要先问为什么别人这样写、为什么不能换。

### 期待链与情绪

```text
新人怕选错题材 -> 回到最常看的类型 -> 拆喜欢的作品 -> 心里有东西 -> 开始写 -> 问题变具体
```

情绪上从抽象焦虑转为低门槛启动。评论里的“我也是新手，共勉”降低了新人羞耻感，核心落点是：先动手，不要把所有前置知识都补完才开始。

### 关键判断

- 适合自己的题材，优先看长期阅读偏好和熟悉度。
- 兴趣能支撑长期写作。
- 新人背景框架可以借鉴已有成熟框架。
- 入门靠写、改、再写的循环。
- 拆几本成绩好且自己爱看的书，比空想题材更有效。
- 仿写要拆功能，不能只换皮。

### 可迁移写法

新人入门启动表：

```text
最近一年看得最多的题材：
最喜欢的 3 个看点：
熟悉的同类样本：
可借鉴的背景框架：
要拆的 10 章：
素材库条目：
试写目标字数：
试写后暴露的问题：
下一步补课内容：
```

### 训练任务

列出最近一年看得最多的 3 个网文类型，每类写出 3 个喜欢看点。选一本同类型成绩好且自己喜欢的书，拆 10 章，再写 3000 字试稿，记录 5 个具体卡点。

## 787：诸位大佬，新人想请教一个问题，...

人工查看范围：正文、全部 6 条评论。无附件、无图片。

证据限制：评论中给出一个星球短链接 `https://t.zsxq.com/y7mqrfe` 作为大纲样例入口，但当前 Markdown 未包含链接目标内容；本次只依据本帖正文和评论拆解，不能把链接内样例当作已读证据。

### 读者承诺

新人被百度、知乎上的内投大纲模板搞乱：有的说随便写，有的说作者信息、章节大意、角色设定都要写。自己开头卷纲细纲写了上万字，但整本大纲只想得到几百字，后面没想好，不知道内投到底该写什么。

### 核心机制

```text
格式焦虑 -> 找已有范例 -> 写简略版整本大纲 -> 区分开头/卷纲/细纲/整本大纲 -> 再处理卷纲波折自查
```

宝剑回复很直接：前面有人发过一份大纲，照这个形式写个简略版即可。也就是说，内投大纲的第一目标是让编辑判断项目方向，不必一上来追求网上最复杂模板。

### 期待链与情绪

```text
新人被多套标准搞乱 -> 权威给最小可行格式 -> 格式焦虑降低 -> 追问卷纲平淡 -> 期待后续卷纲审查工具
```

情绪从“我不知道写多细”转向“我怎么检查卷纲是否有波折”。评论后半段显示，社群对开头审查、卷纲审查这类标准化工具有强需求。

### 关键判断

- 内投大纲不必照最复杂模板堆满字段。
- 有范例时优先照范例写简略版。
- 开头、卷纲、细纲、整本大纲是不同粒度。
- 新人写不出完整后文，可以先表达主线方向和核心冲突。
- 真正难点会从格式转向故事是否波折、意外是否有效。
- 卷纲自查需要专门标准，不能只靠主观感觉。

### 可迁移写法

内投简略大纲表：

```text
书名：
题材：
核心卖点：
主角：
核心矛盾：
主线方向：
第一卷目标：
阶段剧情：
升级/成长路径：
结局方向：
编辑需要判断的亮点：
```

### 训练任务

找一份已有内投大纲样例，拆成字段。再用 500 字写自己的简略整本书大纲，只写主线、核心冲突、升级路径和结局方向；另把已有卷纲按“计划 -> 意外 -> 损失/收益 -> 新目标”重排。

## 788：300M网文写作资料

人工查看范围：正文、全部 4 条评论。无附件、无图片。

证据限制：正文提供百度网盘链接和提取码 `26l4`；评论中有人补充提取码为 `26L4`，后续评论在 2023-08-07 反馈“链接挂了”。本次未能查看网盘内容，因此不能拆解资料包内部文件，只能拆资源分享和失效风险。

### 读者承诺

这篇是资源分享，承诺提供 300M 网文写作资料。正文极短，给出网盘链接和提取码；评论主要围绕提取码识别和链接失效。

### 核心机制

```text
资料包分享 -> 提取码获取 -> 读者下载 -> 链接可能失效 -> 资料需本地化/索引化才可长期使用
```

资源帖的即时价值在于入口，但长期价值取决于能否保存、整理、索引。评论“链接挂了”说明外部网盘链接不能作为稳定知识库依赖。

### 期待链与情绪

```text
新人看到资料包 -> 期待一次性获取大量教程 -> 提取码不清 -> 他人补充 -> 链接失效 -> 资料价值断档
```

情绪上是“资料焦虑”的典型形态：看到大包会兴奋，但如果不整理、不消化、不备份，最终只剩失效链接。

### 关键判断

- 外部网盘资料入口不等于知识吸收。
- 提取码、链接有效期、备份状态都应记录清楚。
- 大资料包需要索引，否则新人容易收藏后不用。
- 链接可能失效，关键资料应本地化或转成知识库条目。
- 未读取资料包内容时，不能假装已吸收内部精华。

### 可迁移写法

资料包入库表：

```text
资料名称：
来源链接：
提取码：
获取日期：
是否已下载：
是否已备份：
目录结构：
高价值文件：
待消化顺序：
失效风险：
```

### 训练任务

整理一个自己的写作资料包，不超过 20 个文件。为每个文件写一句用途、所属模块和消化优先级，删除重复或暂时不会用的资料。

## 789：请问 穷 算不算人设

人工查看范围：正文、全部 9 条评论。无附件、无图片。

### 读者承诺

作者问“穷算不算人设”。评论给出更精确的区分：穷首先是人物当前状态，爱财才更像稳定人设；如果主角长期因系统、修炼等原因不断回到赤贫，穷也可以变成长期标签和行为驱动。

### 核心机制

```text
外部状态 -> 稳定欲望/行为模式 -> 他人评价和自我选择反复出现 -> 才形成可用人设
```

宝剑的判断最关键：穷是一个人当前的状态，爱财才是人设。其他评论补充，穷可以作为回忆、评价、动机的来源，比如因为穷所以努力，或因为穷所以堕落。

### 期待链与情绪

```text
读者知道主角穷 -> 期待他如何反应 -> 若只是状态很快消失 -> 人设弱 -> 若转化为爱财/节俭/拼命/怨恨 -> 人设成立
```

情绪重点是把静态标签转成行动逻辑。穷本身不能自动让人记住，只有它持续影响主角选择、说话、价值观和别人评价，才有叙事作用。

### 关键判断

- 穷首先是状态，不天然等于人设。
- 爱财、怕穷、节俭、拼命赚钱、因穷堕落等更接近人设。
- 如果主角一直因机制回到赤贫，穷可以成为持续标签。
- 状态要通过行为和选择转化为人物特征。
- 他人回忆或评价可以用穷解释人物动机。
- 人设要能预测人物反应。

### 可迁移写法

状态转人设表：

```text
人物当前状态：
状态持续多久：
状态造成的欲望：
稳定行为：
说话习惯：
他人评价：
关键选择：
如果状态消失，人设是否还成立：
```

### 训练任务

把“穷、病、丑、弱、孤独”各写成一个人设。每个必须补三项：稳定欲望、稳定行为、关键选择。禁止只写状态标签。

## 790：大大，新人新书扑了，扑的很彻底...

人工查看范围：正文、全部 10 条评论。无附件、无图片。

### 读者承诺

新人新书扑街后，第一反应是换网站，想找更适合新手的平台。回复把问题从“去哪里发”拉回“为什么扑”：总结踩了几个坑，踏实学习、积累、拆书、做大纲，再一步一步来。

### 核心机制

```text
新书扑街 -> 想换平台止痛 -> 复盘踩坑 -> 选想写方向 -> 拆书 + 做大纲 -> 多写多发 -> 用平台节奏验证
```

宝剑强调突破不是容易的事，只能厚积薄发；越急越出乱。其他评论补充：喜欢在哪里看书，就在哪里发；飞卢新人要跟风热点、日更一万、开局爽、章章爽、数据不好就切，节奏要快。

### 期待链与情绪

```text
扑街痛苦 -> 归因到平台 -> 评论要求复盘作品问题 -> 接受突破要积累 -> 回到方向/拆书/大纲/实践
```

情绪是从挫败和逃离转为复盘。新人容易把失败归因于平台不适合自己，但评论提醒：没有绝对适合新手的平台，关键仍是方法论、执行量和作品适配平台节奏。

### 关键判断

- 新书扑街后先复盘，不要只换平台。
- 总结为什么扑、踩了几个坑，比立刻开新站更重要。
- 突破来自学习、积累和实践，不是换环境。
- 先琢磨想写方向，再拆书和做大纲。
- 平台选择可以参考自己喜欢在哪里看书。
- 飞卢类平台节奏要求快、爽、高更新、数据不好及时止损。
- 没有绝对适合新手的平台，因人而异。

### 可迁移写法

扑街复盘表：

```text
作品名：
平台：
题材：
开篇问题：
节奏问题：
爽点问题：
更新问题：
数据节点：
踩坑总结：
下一本方向：
要拆的样本：
新大纲改法：
```
