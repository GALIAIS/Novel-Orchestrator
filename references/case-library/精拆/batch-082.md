## 406：《曹操穿越武大郎》同人版《木叶...

人工查看范围：正文、全部 8 条评论、1 张图片。图片已下载并人工查看：`case-library\精拆\attachments\batch-082-case-406\image-1.jpg`。

图片证据：图片为《木叶：被蓝染教导的鸣人》书籍页，显示轻小说/衍生同人分类、31 万字、标签为综漫/成熟/冷静/嘴炮，简介强调鸣人上空、被蓝染式教导后改变木叶关系。

### 读者承诺

这篇讨论同人创新时，核心不是简单把两个大 IP 拼接，而是用“改变原作遗憾”制造读者点击理由。蓝染教导鸣人带来的看点在于：原著中被欺负、被洗脑、被工具化的鸣人，会不会因此变成另一种人，并报复或改写木叶秩序。

### 核心机制

```text
原作遗憾 -> 强人设外力介入 -> 主角性格改变 -> 原作关系重写 -> 读者看改命
```

### 期待链与情绪

```text
读者知道原著鸣人的委屈 -> 蓝染式教导打破原路线 -> 等待鸣人报复/反控木叶 -> 原作遗憾被改写 -> 同人爽感兑现
```

### 关键判断

- “蓝染教导鸣人”的创新能让部分读者产生点击意愿。
- 蓝染标签是多智近妖、冷酷无情、野心与能力匹配。
- 原著鸣人的标签是被欺负、被打骂、被火之意志洗脑。
- 两者组合的真正看点是鸣人会变成怎样。
- 同人最大的看点常是弥补原作遗憾。
- 《曹操穿越武大郎》也是同人逻辑，开局改变武大郎被毒死的命运。
- 不是人人同时看过《火影》和《死神》，双 IP 拼接会筛读者。
- 即使看过两部，也不一定认可蓝染式三观。
- 非穿越会降低一部分代入感。
- 黑暗流、非穿越都偏冷门，新人不宜盲学题材。
- 可学习的是“武大郎式剧情架构和改命逻辑”，不必学冷门黑暗流题材。
- 书名从一开始就在筛选读者。

### 可迁移写法

同人遗憾改写表：

```text
原作：
原作最大遗憾：
被改变的角色：
外来影响/新导师：
角色会发生的变化：
原作关系如何重写：
读者最期待的报复/补偿：
需要读者知道的原作门槛：
是否过度筛读者：
新人是否适合学习：
```

### 训练任务

选一个同人设定，不写“某角色遇到某角色”这种表层组合，只写三项：原作遗憾、改变力量、改写后的第一场关系冲突。若没有原作遗憾，不做长篇。

## 407：关于打斗情节，我觉得要分两种情...

人工查看范围：正文、全部 9 条评论。无附件，无图片。

### 读者承诺

这篇把打斗问题拆成两个判断：打斗有没有意义，以及打斗有没有角色辨识度。如果一段战斗换掉主角和招式名也不违和，它只传达“跟谁打、赢没赢”，读者自然会跳。多数网文不必追求古龙式个人痕迹，但必须让打斗目的、形式、战果和收获有区别。

### 核心机制

```text
打斗目的 -> 角色/能力痕迹 -> 形式变化 -> 战果差异 -> 收获/后果
```

### 期待链与情绪

```text
读者知道为什么打 -> 等待角色用自己的方式解决 -> 战斗形式不重复 -> 战果改变局势 -> 战后收益让打斗有价值
```

### 关键判断

- 外网吐槽玄幻战斗像双方互发不同名字的能量波，有参考价值。
- 检验打斗可把主角和招式名替换到别的战斗里，看是否违和。
- 不违和说明战斗缺少意义和辨识度。
- 西门吹雪式战斗带强烈个人痕迹，换成陆小凤就不成立。
- 很多网文打斗只提供“主角跟谁打、赢了还是输了”两种信息。
- 这种打斗容易被读者跳订。
- 大多数作者不以打戏为卖点，可以扬长避短。
- 打斗前的理由必要性、打斗后的收获总结和超额收益，比过程更容易制造爽点。
- 打斗是有目的的，无意义打斗不该写。
- 打斗形式要多变：以力压制、属性相克、场地优势、偷袭、下毒、话术扰心、要挟等。
- 直接战斗可分为离体道具伤害和肢体伤害。
- 离体道具可换飞剑、弓箭、剑气、拳压、法宝等皮肤，但本质相近。
- 如果无法写出个人痕迹，就应略写过程，重点写原因和结果。
- 低武打斗更容易写出力量感、画面感和人物特色。
- 来回对波若无法避免，就要精简。

### 可迁移写法

打斗意义辨识表：

```text
这场为什么必须打：
主角个人痕迹：
对手个人痕迹：
能否替换名字仍成立：
打斗形式：
是否重复上一场：
关键战果：
战后收获：
超额收益：
可略写的对波：
```

### 训练任务

拿一场自己的打斗，删除所有人名和招式名给自己重读。若看不出主角是谁，就重写主角的战斗方式；若只剩胜负信息，就压缩过程，把字数挪到打斗原因和战后收获。

## 408：本篇有部分对飞卢风格的理解，也...

人工查看范围：正文、全部 4 条评论。无附件，无图片。

### 读者承诺

这篇记录飞卢改革期的新书风险：原本飞卢吸引新人的是反馈及时、更新换曝光，但更新榜和推荐机制调整后，新书曝光不足，老书持续曝光，新人首日低订变常态。平台机制变化时，作者不能只沿用旧经验，必须重新评估曝光、更新、付费投流和中下层生存空间。

### 核心机制

```text
平台机制调整 -> 新书曝光下降 -> 老书透支曝光 -> 新人反馈变慢 -> 开书策略重估
```

### 期待链与情绪

```text
作者奔着飞卢及时反馈入场 -> 高更新换来部分追订 -> 推荐改革削弱新书曝光 -> 首订预期下调 -> 平台选择焦虑上升
```

### 关键判断

- 作者正式写飞卢书后，一周前上架，一百多订。
- 因为日更三万，后续追订尚可，每天日收约一百。
- 当时建议新人暂缓开新书，因为飞卢面临改革。
- 更新榜已被改动。
- 推荐机制也被改，导致新书曝光不足。
- 老书持续曝光，存在提前透支。
- 很多人新书首日几百 V、上架后没曝光、首日百订成为常态。
- 目前看似乎只有氪金才有曝光度，对新人不友好。
- 飞卢朝起点方向打造精品和 IP。
- 中下层作者在改革中更受冲击。
- 飞卢原优势是反馈及时，但改革后新书推荐机制还不完善。
- 与番茄相比，免费平台算法可能更稳定，但长期免费对作者和读者未必是好事。
- 平台选择应关注“当前机制”，不是历史印象。

### 可迁移写法

平台改革风险表：

```text
目标平台：
当前机制变化：
新书曝光入口：
旧书曝光占比：
是否需要氪金：
新人首订预期：
日更量要求：
反馈速度：
中下层机会：
是否适合当前开新书：
```

### 训练任务

准备开书前，连续 7 天记录目标平台新书榜、更新榜、推荐位变化。只看新书，不看老书神话。写出“现在开/暂缓/换平台”的判断。

## 409：#拆书/影视作品 《此间男神》...

人工查看范围：正文、全部 2 条评论、1 张图片。图片已下载并人工查看：`case-library\精拆\attachments\batch-082-case-409\image-1.jpg`。

图片证据：图片为《此间男神》恋爱日常脑图，分为前期卖点、故事框架和分析。可见前期卖点包括各有特色的美少女、主角凭未来知识弥补亲情和学习遗憾、跟美少女恋爱的遐想；故事框架大量围绕宋诗涵、江悦、沈佩佩等角色的学习、误会、家庭、校园、恋爱互动推进；分析强调每个女角色负责一个区域，校内恋爱、校外故事、家里亲情等功能区分明显。

### 读者承诺

这篇拆《此间男神》后得出的判断是：日常文代入感为王。日常文不一定适合硬套三幕式，更适合研究每段内容背后的卖点、对观众产生的效果，以及作者如何用角色分工、关系推进、校园/家庭/恋爱区域制造可代入的生活连续感。

### 核心机制

```text
代入身份 -> 多角色功能分区 -> 日常事件推进 -> 情感/亲情/学习遗憾补偿 -> 关系逐步加深
```

### 期待链与情绪

```text
读者代入主角生活 -> 各女角色提供不同区域的情绪 -> 日常小事件不断补偿遗憾 -> 关系一点点升温 -> 生活感形成追读
```

### 关键判断

- 原帖拆的是《此间男神》前十几万字。
- 同时比较了《养成女友从梦境游戏开始》的日常部分。
- 作者判断《此间男神》的日常写得过关且标准。
- 最大感受是日常文代入感为王。
- 评论指出该图更像提炼总结，不像传统拆书。
- 发帖者认为许多日常文不遵循三幕式。
- 研究重点应是作者写各种内容的动机和作用。
- 也就是研究内容背后的卖点，再总结制造卖点的技巧。
- 图片显示前期卖点包括美少女人设、未来知识弥补遗憾、恋爱遐想。
- 主角用未来知识补亲情和学习遗憾，是强代入补偿。
- 女角色承担功能分区：校内恋爱、校外故事、家庭亲情等区域。
- 对话和心理活动是加深代入感的重要途径。
- 家庭关系改善、学习成绩、校园关系、恋爱暧昧共同构成日常追读。

### 可迁移写法

日常文代入卖点表：

```text
主角代入身份：
主角要弥补的遗憾：
女角色A功能区：
女角色B功能区：
家庭/亲情线：
校园/工作线：
恋爱/暧昧线：
每段日常的卖点：
对读者产生的效果：
下一次关系升温：
```

### 训练任务

设计一个日常文前 5 章。每章必须写出“生活事件、角色功能、代入补偿、关系推进”。不要套三幕式，只检查读者是否愿意代入这一天。

## 410：朋友们，谁知道番茄小说上面的『...

人工查看范围：正文、全部 10 条评论、3 次图片引用。3 次引用为同一张聊天截图，已下载并人工查看：`case-library\精拆\attachments\batch-082-case-410\image-1.jpg`。

图片证据：截图中编辑口径为：看书类型火不火，番茄最好在读人数 10 万以上才算比较火；七猫看榜单即可；番茄数据存在刷量和推书导致的水分。

### 读者承诺

这篇讨论番茄榜怎么看。结论不是盲信“番茄榜”，而是优先看阅读榜和在读门槛：番茄在读 10 万以上且上榜才有参考价值，低于 10 万意义不大；但在读也可能有水分，收入还受类型和阅读时长等算法权重影响，不能直接等同收入。

### 核心机制

```text
平台榜单 -> 优先阅读/在读指标 -> 设置10万门槛 -> 判断榜单水分 -> 结合类型和收入权重
```

### 期待链与情绪

```text
作者想用榜单判断市场 -> 番茄榜规则不透明 -> 10万在读提供筛选阈值 -> 数据水分提醒降噪 -> 类型收入差异修正判断
```

### 关键判断

- 有人问番茄榜按什么排，并分日榜周榜。
- 评论建议看阅读榜，其他很多是推荐位。
- 番茄在读一般 10 万以上的书才比较值得参考。
- 不要低于 10 万，低于 10 万意义较弱。
- 在读 10 万以上且上榜最好。
- 番茄在读数据可能比较虚。
- 同样百万在读，不同种类收入可能差十几倍。
- 有人在读榜前 20，一个月不到 10 万，说明在读不等同收入。
- 番茄部分在读可能有刷量或平台推书因素。
- 七猫口径是看榜单即可。
- 番茄收入算法可能因类型不同而差异很大。
- 读者读一本书时间越长，收入可能越高。
- 都市、总裁等热门类型的在读转换收入可能偏少，武侠、科幻等冷门类型可能偏多。

### 可迁移写法

番茄榜单降噪表：

```text
目标书：
榜单类型：
是否阅读榜：
当前在读：
是否超过10万：
是否同时上榜：
题材类型：
在读是否异常虚高：
同类收入参考：
是否可能是推荐位推高：
是否值得拆作市场样本：
```

### 训练任务

选番茄同题材 10 本书，只保留在读 10 万以上且上榜的样本。每本记录题材、在读、榜单位置、评论热度和可能收入偏差，剔除疑似推书水分样本。
