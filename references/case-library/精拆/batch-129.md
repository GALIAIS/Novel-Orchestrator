## 641：#公开处刑

人工查看范围：正文、全部 7 条评论、附件状态。无图片。附件 `正文.doc` 有下载链接，但请求返回 401；本地导出目录未找到文件。

证据限制：无法查看附件正文，本卡依据可见设定和评论反馈拆解。

### 读者承诺

这篇是都市异能《这位法师不太正经》的公开处刑。可见设定承诺的是“从小修行异能的主角 + 意外获得系统 + 下山任务 + 魔都异能事件 + 打怪升级得宝”。但评论集中指出开篇问题：第一章甚至分不清大叔、黑衣人谁是主角，书名有法师，两章却没点出法师卖点，整体太慢热。

### 核心机制

```text
书名承诺法师 -> 开篇主角不清 -> 卖点迟迟不出现 -> 读者无法代入 -> 改成开篇讲明主角和法师/系统
```

### 期待链与情绪

```text
读者看到书名期待不正经法师 -> 进入正文却看不到明确主角和法师卖点 -> 疑惑大叔/黑衣人身份 -> 期待断档 -> 评论要求按审查表重改
```

核心情绪不是爽，而是迷糊、慢热和不讨喜。若读者无法确认谁是主角，就不会关心任务、师叔失踪或异能组织。

### 关键判断

- 第一章不见主角，签约风险极高。
- 主角伪装可以有，但开篇必须让读者知道这是主角。
- 书名承诺“法师”，前两章就应点出法师卖点或不正经特质。
- 大叔和黑衣人如果不讨喜，会拖累读者第一印象。
- 都市异能不能先铺任务链，必须先立主角、金手指、冲突和题材卖点。

### 可迁移写法

开篇主角显影表：

```text
书名承诺：
第一章主角是否明确：
主角第一动作：
读者为什么喜欢他：
题材卖点首次出现：
金手指首次出现：
第一冲突：
是否存在伪装/误导：
伪装是否伤害代入：
```

### 训练任务

把一个“主角伪装登场”的开篇改成明牌版：500 字内让读者知道谁是主角、他有什么卖点、他正在面对什么问题。伪装可以保留给配角不知道，但不能让读者也不知道。

## 642：#参考资料 看大家分享，我也来...

人工查看范围：正文、全部 2 条评论、图片 4 张。无附件。

图片证据：图片为《情节与人物：找到伟大小说的平衡点》封面、目录和内页。目录显示第一部分“令人难忘的人物”，含核心性格、生理与自然属性、重大事件、关键时刻、初始状态、诱发事件等；第二部分“精彩的情节”，含序幕、三幕式结构、故事核心、引入、高潮和结局；第三部分是情节与人物整合。内页强调人物心理历程的关键时刻应发生在故事靠后、高潮前，并影响高潮部分行为反应。

### 读者承诺

这篇推荐《情节与人物》作为小说写作补充资料：它比影视剧本书更适配小说，人物讨论更细，情节安排更适中，可补充《作家之旅》和《故事策略》。对作者的承诺是：按目录顺序推人物，主角会更完整；同时能帮助决定序幕取舍、人物型/情节型作者的注意点，以及三幕结构中人物如何服务情节。

### 核心机制

```text
人物维度拆细 -> 情节三幕推进 -> 关键时刻影响选择 -> 人物与情节整合 -> 主角小传/大纲更完整
```

### 期待链与情绪

```text
作者需要补人物小传和大纲 -> 找到人物/情节平衡工具书 -> 按目录推主角 -> 角色关键选择影响高潮 -> 提高大纲可执行性
```

情绪是资料分享的实用感：不是空谈大道理，而是能直接拿目录做人物和情节清单。

### 关键判断

- 人物型作者和情节型作者都要补短板。
- 主角小传不能只写标签，要写核心性格、重大事件、初始状态和关键时刻。
- 序幕是否要写，应服务情节引入，而不是为了炫设定。
- 关键时刻应影响高潮行为，人物变化要进入情节结果。
- 工具书是补充，不替代网文类型承诺和榜单验证。

### 可迁移写法

人物情节整合表：

```text
主角核心性格：
生理/自然属性：
重大事件：
初始状态：
诱发事件：
关键时刻：
关键选择：
高潮行为：
情节如何迫使人物变化：
人物变化如何改变情节：
```

### 训练任务

给当前主角做一页小传：核心性格、重大事件、初始状态、诱发事件、关键时刻、高潮选择。最后写 200 字说明：这个人物变化如何改变最终高潮。

## 643：#拆书/影视作品本人 纯纯小白...

### 读者承诺

这篇是新手第一次拆《剑道第一仙》1-15 章。作者选取纵横 24 小时畅销榜前十作品，判断为无敌文，主角一直强。核心承诺不是“这份拆书一定正确”，而是通过拆榜单无敌文，训练从读者视角切换到作者视角，理解作者为什么这样安排爽点、强弱关系和情节。

### 核心机制

```text
选择榜单样本 -> 判断类型/主角卖点 -> 手工拆前 10-15 章 -> 追问作者动机 -> 形成作者视角
```

### 期待链与情绪

```text
新手羡慕他人分析 -> 迈出第一次拆书 -> 担心拆得不对 -> 获得反馈：重点是思考动机和逻辑 -> 学习焦虑降低
```

情绪是试探、紧张、羡慕和被鼓励后的稳定。评论把“对不对”焦虑改成“坚持拆、训练视角”。

### 关键判断

- 新手拆书先不要追求理论完美。
- 拆书目的不是答案正确，而是训练作者视角。
- 榜单样本 + 类型判断 + 主角核心卖点，是有效起点。
- 无敌文要重点观察主角强势感如何建立和维持。
- 多拆几次后，作者视角会内化，不用表格也能维持判断。

### 可迁移写法

```text

作者为什么这样写：

信息差/身份/实力如何制造无敌感：
可迁移写法：
```

## 644：三万字90收在刺猬猫算什么水平...

人工查看范围：正文、全部 11 条评论。无附件，无图片。

### 读者承诺

这篇处理刺猬猫早期数据焦虑：作者用“三万字 90 收藏”询问水平，并透露想视情况决定切不切，希望月入三千，作品是同人。评论给出的核心是：单看早期收藏不能下结论，应等试水推荐或上架 24 小时后再判断；低收藏不一定等于不能签约，签约也不完全看收藏。

### 核心机制

```text
早期收藏低 -> 作者想切书 -> 评论校准平台经验 -> 等推荐/上架强信号 -> 再做决策
```

### 期待链与情绪

```text
作者看到 3 万字 90 收 -> 担心签约和收入 -> 有人建议切 -> 有人建议等推荐/上架 -> 保留试水窗口
```

核心情绪是“想切又不甘心”。数据制造现实压力，评论提供不同经验口径，让作者把判断节点后移到更有力的信号。

### 关键判断

- 三万字 90 收是早期弱信号，不能单独决定生死。
- 推荐、上架 24 小时、签约反馈比裸收藏更有判断力。
- 同人和原创的收藏标准不同，不能混用。
- 月入三千是收入目标，需要用上架数据验证，而不是只看收藏。
- 签约未必只看收藏，作品方向和编辑判断也重要。

### 可迁移写法

早期数据决策表：

```text
平台：
题材/同人或原创：
当前字数：
当前收藏：
是否签约：
是否上推荐：
上架 24 小时数据：
收入目标：
切书判断节点：
继续写的最低条件：
```

### 训练任务

写一个作者连载早期拿数据求助的片段：必须包含一个具体数据、一个收入目标、两个相反建议、一个暂缓切书的判断节点。目标是训练“数据压力如何制造期待链”。

## 645：刺猬猫什么更新引来的流量最大？

人工查看范围：正文、全部 6 条评论。无附件，无图片。

### 读者承诺

这篇处理平台更新时间误区。作者原本问“刺猬猫什么更新引来的流量最大”，评论先误解成题材流量，回答原神、方舟、FGO；作者补充是问什么时候更新。后续评论给出晚上 6 点、凌晨 12 点等经验，但最终判断是：没有推荐时更新时间差别不大，正常更新，专注内容即可。

### 核心机制

```text
作者寻找更新时间流量密码 -> 评论给出时间点 -> 降低预期 -> 识别推荐/内容才是主变量
```

### 期待链与情绪

```text
作者希望靠更新时间涨流量 -> 得到若干时段建议 -> 被提醒没推荐涨得慢 -> 回到内容质量和正常更新
```

情绪是轻度焦虑和试图找小技巧；评论区的经验判断把注意力从表层优化拉回推荐、题材、内容和稳定更新。

### 关键判断

- 更新时间可以试，但不是核心增长变量。
- 没推荐时涨得慢很正常。
- 刺猬猫题材热度、PY 曝光、推荐位和内容质量比时段更重要。
- 表层运营不能替代正文质量。
- 新人不应把大量精力耗在更新时间玄学上。

### 可迁移写法

平台流量变量表：

```text
平台：
题材热度：
是否有推荐：
是否有外部曝光/PY：
更新时间：
更新稳定性：
内容卖点：
当前最该优化的是运营还是正文：
```
