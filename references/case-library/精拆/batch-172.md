## 856：分享个网站


人工查看范围：正文、全部 1 条评论。无附件、无图片。

证据限制：正文只给出“全历史”网站和星球文章链接，本次未访问外部网站和文章链接；评论提到该网站与“回形针”有关并存在立场偏向，是评论者判断，未做外部核验。

### 读者承诺

这篇是历史资料入口推荐：想了解世界各国历史，可以用“全历史”。但唯一评论提醒资料来源可能有立场偏向。对创作者来说，它的价值不是“找到一个网站就能直接当史料”，而是提醒历史/架空写作要把资料入口和资料校验分开。

### 核心机制

```text
资料网站 -> 快速查背景 -> 检查来源立场 -> 多源交叉验证 -> 转成可写历史素材
```

历史资料工具能提高检索效率，但不能替代判断。尤其是涉及国家、民族、战争、制度、意识形态时，资料平台的叙述角度会影响创作者对事件因果的理解。如果直接照搬，很容易让作品的历史逻辑和价值判断被单一来源带偏。

### 期待链与情绪

```text
作者需要历史信息 -> 找到资料入口 -> 快速形成背景认知 -> 发现来源可能偏向 -> 追加校验 -> 写作更稳
```

读者期待历史题材有可信质感。可信不是每个细节都百科正确，而是关键事件、制度、人物动机和时代环境不能被明显错误或单一立场破坏。

### 关键判断

- 资料网站适合做入口，不适合做唯一依据。
- 历史素材要记录来源和使用范围。
- 评论中的立场提醒要写进证据限制。
- 重大历史判断至少做两到三个来源交叉。
- 架空历史也需要了解真实制度和事件逻辑。
- 创作时要把“事实资料”和“改编设定”分开。

### 可迁移写法

历史资料校验表：

```text
资料入口：
查询主题：
原始出处：
是否有平台立场：
可交叉来源：
能直接使用的事实：
需要谨慎处理的判断：
可改编成剧情的冲突：
```

### 训练任务

选一个历史事件，先用资料网站做快速背景，再找两个不同来源交叉核对。最后只提炼 5 条可用于剧情的事实和 3 条必须谨慎处理的争议点。

## 857：网文开头13项要点审核-【2023-5-4日更新版本】


人工查看范围：正文、全部 56 条评论、图片 4 次出现对应的 2 个唯一图片文件。图片已保存并查看：`<EVIDENCE_CACHE>\batch-172\857-image-218221811528581.jpg`、`<EVIDENCE_CACHE>\batch-172\857-image-812851215555282.jpg`。

证据限制：Markdown 正文未收录 13 项正文细则，只提示新版迁移到“网文开篇自我审核十问【2024-7版本】”并给出文章链接；本次未打开外链。图片内容分别为《烂柯棋缘》书页截图、以及一个女频 KTV 打脸情绪折线作业图，只能作为评论讨论证据。

### 读者承诺

这篇本体是开头审核清单入口，评论区反而保留了很多使用边界：开篇冲突最好冲着主角本身；优秀品质/思想能否当金手指，关键看它能否带来阶段性收益并拉期待；无敌文不一定要危机，但要有冲突；18 元素不是绝对公式，要结合书名、简介、题材、受众期待和作者人气基础判断。

### 核心机制

```text
开头审核清单 -> 检查代入/冲突/金手指/期待 -> 结合题材包装 -> 调整节奏 -> 新人降低试错
```

评论中《烂柯棋缘》的讨论很关键：它开篇先写穿越前过程，看似踩雷，但书名和简介已经提前制造了“他如何遇见棋局”的期待，所以读者会被期待感拉着走。这说明清单不是机械打勾，而是判断读者在进入正文前已经被什么承诺吸引。

### 期待链与情绪

```text
读者看见书名简介 -> 形成初始期待 -> 开篇冲突/金手指承接期待 -> 情绪上行下行拉扯 -> 继续追读
```

开头审核的目标不是让每本书长得一样，而是确保新人没有漏掉最基本的商业承诺。评论里有人写完前六章对照后发现踩雷无数，也有人问 18 元素是否必须一章完成，回答是开头范围可以约 1-3 万或 2 万字上下浮动，新人更快更稳。

### 关键判断

- 清单是新人降低风险的审核工具，不是所有作品的绝对规则。
- 冲突最好直接影响主角，至少要波及主角代入。
- 金手指不一定是物理外挂，但必须带来可见收益。
- 无敌文可以没有危机，但必须有冲突。
- 慢开局要靠书名简介和题材预期提前拉期待。
- 情绪下行不是每个爽点都必须做满三次，可曲折向上。

### 可迁移写法

开篇审核使用表：

```text
书名/简介给出的期待：
题材默认承诺：
主角代入点：
开篇冲突是否波及主角：
金手指/优势是否有阶段收益：
第一事件期待：
情绪上行/下行设计：
哪些清单项可延后：
延后的理由：
```

### 训练任务

拿一个开头 2 万字，对照清单不要只打勾。每一项都写“读者为什么会继续看”或“为什么可以延后”。最后找出 3 个最影响追读的问题优先改。

## 858：我看要离刺荆轲的四本书：皇帝、...


人工查看范围：正文、全部 3 条评论。无附件、无图片。

证据限制：正文提到《皇帝》《门阀》《魔神》《圣人》都是万订精品，但本次未核验作品数据；评论对抄袭嫌疑、蹭热点、后期啰嗦等评价均为个人印象。

### 读者承诺

这篇的问题是：为什么一些又长又啰嗦、车轱辘话多的作品，仍能成为万订精品？评论给出几个方向：题材在当时新，门阀开头强，稳定日更和剧情不崩本身就是门槛。如果还能兼顾剧情、人设、创意、脑洞，就能进一步冲榜。

### 核心机制

```text
新题材/强开头 -> 稳定更新 -> 剧情持续跑动 -> 读者形成惯性 -> 啰嗦被容忍
```

商业网文不只比“精炼”。对长篇连载来说，稳定供给、题材占位、开头模板、持续推进和不大崩，往往能抵消一部分语言啰嗦。读者可能会抱怨车轱辘话，但只要主线还在跑、期待还在、更新还稳，就可能继续读。

### 期待链与情绪

```text
读者被题材/开头吸引 -> 每天有稳定更新 -> 主线持续展开 -> 形成阅读习惯 -> 容忍重复表达
```

这里的情绪不是单章爆爽，而是长篇追更的惯性满足。很多新人只盯着文笔缺点，却忽略了对方在题材选择、开头结构、更新纪律和大方向稳定上的优势。

### 关键判断

- 啰嗦不等于不能赚钱，但会限制上限和口碑。
- 新题材和强开头可以提供早期优势。
- 稳定更新本身就是商业能力。
- 剧情不崩可以筛掉大量竞争者。
- 拆成绩书要找它做对了什么，不只看自己讨厌什么。
- 车轱辘话可被容忍的前提是主线还在推进。

### 可迁移写法

长篇稳定性复盘表：

```text
题材新意：
开头可学点：
更新稳定性：
主线是否持续推进：
读者可容忍的短板：
真正支撑成绩的能力：
我能迁移的部分：
我必须避免的部分：
```

### 训练任务

找一本你觉得啰嗦但成绩好的长篇，拆 20 章。只记录三件事：它每天/每章给读者什么新东西，主线是否持续移动，哪些短板被题材或更新惯性掩盖。

## 859：#公开处刑 新手第一次下决心写...


人工查看范围：正文、全部 20 条评论、附件元数据 2 条。无图片。

证据限制：附件 `两万字正文.docx` file id `241588412221151`、大小 `61402`、下载次数 `164`，直链下载返回 `403 Forbidden`；附件 `开头小纲填写表格（1）.docx` file id `548211485558824`、大小 `24459`、下载次数 `170`，Markdown 未提供直链。本次未读取正文和小纲，只依据帖子正文和评论拆解。

### 读者承诺

这篇公开处刑的新手作品，被评论判断为幕后流、轻小说味，平台建议偏刺猬猫、SF。更重要的反馈集中在金手指系统：作品刻意仿《术士手册》，手游元素没问题，但短时间抛出太多系统功能，抽卡、盲盒、签到、任务、培养等堆在一起，会让网文系统显得臃肿。

### 核心机制

```text
手游式复杂系统 -> 功能过多同时出现 -> 阅读负担变重 -> 合并/延后功能 -> 用任务和等级控制节奏
```

评论里给出的改法非常具体：去掉或合并盲盒抽奖，把道具作为任务奖励；第一次抽卡不十连，单抽送 SSR；经验道具从任务、等级奖励、日常周常里来；任务系统拆成新手任务、等级任务、每日任务、每周任务。这样系统不再是功能展示，而是节奏控制器。

### 期待链与情绪

```text
读者看到新系统 -> 期待角色/奖励兑现 -> 功能过多导致理解负担 -> 功能合并后每次行动一份收获 -> 期待稳定
```

系统流的爽感来自“做一件事，拿一个明确回报”。如果同时抛出太多功能，读者记不住，也不知道哪个功能推动主线。把奖励按任务和等级分发，能让主角成长、剧情推进和读者期待同步。

### 关键判断

- 手游元素可用，但不能把全部 UI 功能搬进开篇。
- 一个事件最好对应一份收获。
- 抽卡、盲盒、签到功能重叠时要删减合并。
- 等级任务适合控制主角成长节奏。
- 每日任务负责日常小事件，周任务负责重要剧情道具。
- 签到系统容易造成战力膨胀，只适合作为剧情保险。

### 可迁移写法

复杂系统删减表：

```text
现有系统功能：
哪些功能重复：
第一章必须出现的功能：
可延后解锁的功能：
可删除的功能：
主角等级节奏：
每次行动对应奖励：
奖励如何推动主线：
签到是否必要：
```

### 训练任务

把一个包含抽卡、签到、盲盒、任务、商城的系统开头压缩成 3 个核心功能。第一章只允许出现一个主功能，其他功能必须通过等级任务逐步解锁。

## 860：对于想要发开头求“公开处刑”的...


人工查看范围：正文、全部 24 条评论、附件元数据 1 条。无图片。

证据限制：附件 `公开处刑填写表格【2024-7月全新版】.docx` file id `581258154584814`、大小 `20656`、下载次数 `1397`，直链下载返回 `403 Forbidden`；本次未读取表格正文，无法展开“10 项内容”的具体条目。

### 读者承诺

这篇是公开处刑投稿规范：下载填写表格，填写书名、简介、小纲，并核查 10 项内容；至少 6000 字正文放在表格下方；上传文件并添加 `#公开处刑#` 话题。它把“求评价”从随手贴文，变成可审查、可对照、可节省评审成本的标准化材料。

### 核心机制

```text
求评需求 -> 表格收集书名/简介/小纲 -> 正文达到最低样本 -> 添加话题 -> 方便公开处刑
```

评论进一步补足使用边界：18 元素不是必须一章完成，开头范围 2 万字上下浮动几千，新人快一点更好；无敌文可以有部分不符，但代入感要足够；不公平伤害最好波及主角；爱情故事可以把吸引异性的特质视为金手指，把社会性死亡或感情破裂视为死亡压力。

### 期待链与情绪

```text
新人想求评 -> 材料按表格整理 -> 评审能看见承诺和执行 -> 评论更聚焦 -> 作者知道怎么改
```

公开处刑的价值不只是被骂或被夸，而是让作者把作品承诺先讲清楚。书名、简介、小纲和 6000 字正文放在一起，评审才能判断开头是否兑现了承诺、问题是设定错、结构错还是表达错。

### 关键判断

- 求公开处刑要按表格提交，不要只丢草稿。
- 书名、简介、小纲和正文必须放在同一材料里。
- 正文至少 6000 字，才有最低判断样本。
- 开头要素不一定一章完成，可放在 2 万字左右范围。
- 规则主要服务零人气新手降低风险。
- 老稿已有 20 万字时，也能用这些概念自查后续。

### 可迁移写法

公开处刑提交自检表：

```text
书名：
简介：
题材：
开头小纲：
正文样本字数：
主角承诺：
金手指/优势：
第一事件：
当前最想被点评的问题：
是否添加话题：
```

### 训练任务

准备一次公开处刑材料：写 200 字简介、1000 字小纲、6000 字正文，并在文档开头列出你最想被点评的 3 个问题。不要只发刚写完的裸正文。
