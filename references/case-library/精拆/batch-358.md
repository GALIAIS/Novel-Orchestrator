## 1786：俺感觉，新手一定要清晰且明确的...


人工查看范围：正文、全部 17 条评论、微信读书搜索外链标记。外链《这个明星很想退休》《太监》《斗破》《明星很想退休》只作为书名指向。

证据限制：这是公开处刑后的方法总结帖，不含具体样稿；本卡只拆“新人开头大方向：对话、主视角、先解矛盾、市场反馈”的原则。

### 读者承诺

这篇沉淀的是新人首先要明白写文是给人看的，读者要看故事、主角选择和选择结果，而不是复杂设定。爽点可概括为：现实中不可能经历的事和不可能做出的选择，读者代入主角后完成，并得到正向反馈、变强变牛。作者认为开头内核是“对话”：可以是人物对话，也可以是自言自语、日记、和读者交流。《这个明星很想退休》的自我思考、《太监》的日记，都在和读者互动；纯叙述则容易干。她还强调前 500 字最好有双引号或类似对话痕迹。第二个原则是主视角锁在主角身上，即使配角聊天也必须与主角相关；配角震惊最好通过台词表达，不必切入心理。第三个原则是开头先解决第一个矛盾，再插大段回忆和设定。危机临头时主角应先反抗求生，再接收原主记忆；如果别人巴掌要打脸，主角却先回忆五六百字，读者耐心会断。评论补充：市场是唯一老师，公开处刑能帮迷茫新人，个人风格不能当商业不成立的借口。

### 核心机制

```text
读者要故事 -> 开头用对话/交流建立代入 -> 主视角锁主角 -> 先处理第一矛盾 -> 再释放回忆设定
```

新人开头先让情节流动，再谈设定和个人风格。

### 期待链与情绪

```text
读者进入文本 -> 听见主角/人物交流 -> 看见主角选择 -> 危机被行动推进 -> 得到正向反馈期待
```

对话的价值是让读者进入关系和选择，不是形式上凑双引号。

### 关键判断

- 写文终极意义是给人看。
- 读者主要看故事和选择结果。
- 爽点是代入不可能选择并得到正反馈。
- 开头可用对话、自语、日记和读者交流。
- 纯叙述容易干。
- 前 500 字缺少交流痕迹有风险。
- 主视角要锁在主角身上。
- 配角视角必须服务主角。
- 第一个矛盾未解决前不要大段回忆设定。
- 穿越危机开局要先反抗求生。
- 市场是唯一老师。

### 可迁移写法

```text
开头第一矛盾：
主角当前选择：
对话/自语/日记入口：
主视角锚点：
配角信息是否服务主角：
必须先处理的危机：
可后置的回忆设定：
第一处正向反馈：
```

### 训练任务

重写一个 800 字开头：前 200 字出现交流痕迹，前 500 字只锁主角视角，前 800 字必须推进或解决第一个矛盾。

### 读者承诺

这篇沉淀的是老书大纲可以做结构参考，而不是当代写法模板直接照搬。正文说网络上的老书大纲可在结构上做参考。第一张图片展示《诛仙》式资料：人物关系、性格、法宝、卷章推进、设定、支线与作品点评，能看出长篇大纲如何把人物、势力、阶段剧情和主题意象放在同一资料表里。第二张图片展示《斗破苍穹》式分章细纲：按章记录事件、冲突、人物出场、升级、地图推进、情绪节点。这两类资料价值不同：大纲帮助看全局，细纲帮助看章节节奏。评论指出第二张图是斗破不是诛仙，说明资料混杂时更要标清来源。对现在写作来说，老书大纲不可直接替代当前市场节奏，但很适合训练“如何记录人物功能、章节任务、阶段目标、伏笔回收和地图推进”。

```text
老书大纲资料 -> 区分全局大纲/分章细纲 -> 提取结构字段 -> 改成当前题材检查表 -> 避免照搬旧节奏
```

参考老书大纲，学的是组织信息的方法，不是复制当年的读者耐心。

### 期待链与情绪

```text
作者看到长篇全貌 -> 学会分卷分章记录 -> 明确人物和事件功能 -> 自己大纲更可控
```

资料越老，越要只取结构，不取节奏惯性。

### 关键判断

- 老书大纲适合做结构参考。
- docx 不可读时不能臆造附件内容。
- 图片可见《诛仙》大纲和《斗破》细纲样式。
- 全局大纲看人物、势力、卷章和主题。
- 分章细纲看事件、冲突、升级和节奏。
- 资料混杂要标清来源。
- 老书节奏不一定适配现在。
- 可把大纲字段转成自己的检查表。
- 大纲不是正文，不能替代写作反馈。
- 结构参考要服务当前题材。

### 可迁移写法

## 1788：有时候想不出好的过度情节，不妨...


人工查看范围：正文、全部 5 条评论、外链标记。外链 `https://k.xiezuocat.com/` 只作为工具指向。

证据限制：这是工具使用建议短帖，不含具体 AI 生成结果；本卡只拆“AI 可用于过渡情节灵感，但不能替代卷纲和判断”的边界。

### 读者承诺

这篇沉淀的是 AI 写作可以在卡过渡情节时提供灵感和乐趣，但不能依赖。正文建议想不出好过渡时，可以让 AI 写作换角度给思路。评论提醒 AI 很容易歪，因为它不知道你的卷纲；也有人提到写福利段落时被 AI 的生成能力震惊。工具名为秘塔快写。对创作者来说，AI 的最佳定位不是替你写正文，也不是替你决定剧情，而是提供“可能的桥”：两个事件之间怎样转场、一个场景怎样有趣一点、某个角色能用什么小动作衔接。作者必须用自己的卷纲、人物目标、主线功能来筛选 AI 输出。过渡情节本身如果没有功能，就应压缩；如果有功能，AI 可以给角度，但最终要回到主线、爽点或期待。

### 核心机制

```text
过渡卡住 -> AI 提供多个角度 -> 用卷纲和人物目标筛选 -> 只保留服务主线的桥段
```

AI 适合破卡，不适合接管故事方向。

### 期待链与情绪

```text
作者卡在衔接 -> 外部生成带来新角度 -> 筛掉跑偏内容 -> 过渡重新服务下一场重点
```

灵感工具的价值在于松动思路，不在于替代判断。

### 关键判断

- AI 可用于找过渡灵感。
- 不要依赖 AI。
- AI 不知道你的卷纲。
- AI 容易跑偏。
- 生成结果要用主线功能筛选。
- 过渡无功能时应压缩。
- 有功能的过渡要服务爽点或期待。
- AI 可提供趣味角度。
- 工具输出不能直接当正文。
- 作者判断仍是核心。

### 可迁移写法

```text
前一事件：
后一事件：
过渡必须完成的功能：
AI 给出的方案：
跑偏处：
可保留的角度：
改写后的过渡：
```

### 训练任务

找一个卡住的过渡，让 AI 给 5 个方案。只选 1 个角度，改写成符合主角目标和卷纲的 200 字过渡。

## 1789：最近理清思路了，准备开新书，对...


人工查看范围：正文、全部 8 条评论。

证据限制：这是题材选择讨论帖，不含新书大纲；本卡只拆“新人选废土升级还是传统玄幻”的市场判断与素材迁移问题。

### 读者承诺

这篇沉淀的是新人选题材时不能只看自己已有大纲和喜好，还要看流量、竞争强度和题材入口。提问者在废土升级和传统玄幻之间犹豫，想选有一定流量且竞争不太激烈的方向，原本写奇幻但觉得容易扑。评论明确说传统玄幻更容易扑，大神多，市场变小但竞争依旧激烈，已经是新人的乱坟岗；废土升级如果配有有意思的金手指，爽点别太差，味道给够，机会还挺大。作者决定写废土，但遗憾已有热血流爽文大纲、喜欢的人物和梗用不上；评论建议换身份移植到新书，或留着回头再用。核心是：题材选择是商业入口决策，旧大纲沉没成本不能绑架新项目；但人物、梗、情绪功能可以拆出来迁移。

### 核心机制

```text
新人选题材 -> 比较流量和竞争 -> 选择市场缝隙 -> 配金手指和味道 -> 拆迁旧素材
```

放弃旧大纲不是浪费，而是把不适配的整体拆成可复用零件。

### 期待链与情绪

```text
作者舍不得旧项目 -> 市场判断要求转向 -> 新题材带来机会 -> 旧人物梗选择迁移或封存
```

新人题材要先活下来，再谈完整实现旧热爱。

### 关键判断

- 传统玄幻新人更容易扑。
- 传统玄幻市场变小但竞争强。
- 废土升级有机会。
- 废土需要有意思的金手指。
- 爽点不能太差。
- 废土味道要给够。
- 奇幻和传统玄幻都可能高风险。
- 旧大纲是沉没成本。
- 喜欢的人物可换身份迁移。
- 风格差异太大时应封存素材。

### 可迁移写法

```text
候选题材A：
流量：
竞争：
自身优势：
候选题材B：
金手指亮点：
旧素材可迁移：
必须舍弃的旧设定：
最终选择：
```

### 训练任务

把一个旧大纲拆成三类：可迁移人物、可迁移梗、必须封存设定。再为新题材写一个金手指亮点。

## 1790：#公开处刑


人工查看范围：正文、文件标记、全部 61 条评论。文件区显示 `开头小纲填写表格.docx`，导出 Markdown 无下载 URL，本地未检出，未能读取附件正文。

证据限制：附件正文不可读；本卡只拆作者自述和评论中明确指出的硬核玄幻开头问题。

### 读者承诺

这篇沉淀的是“硬核老白玄幻”很容易成为新人逃避商业化的借口。作者第一次去起点、第一次写玄幻，被拒多次，定位硬核老白文、英雄联盟世界观，认为 18 元素和情绪折线都尽量做了。评论集中指出：玄幻、硬核、老白、娓娓道来几乎把雷踩满；这种路子和商业化背道而驰。具体问题包括视角没锁死主角，写得杂碎，冲突出现太晚；设定过多，只增加阅读负担，不能引起期待；开局不要背景，先出危机、金手指、信息差和打脸；前 500 字没有对话/故事，编辑可能直接不过；主角没有危机和目标，读者看不到推进剧情；英雄联盟世界观自带粉丝流量，但简介写背景不是看点爽点，读者进来想看熟悉英雄/类似满足，却看了多章还没看到；非粉也看不到亮点、爽点、趣味点。最尖锐的判断是：前五章没有故事，全是背景、设定、铺垫、伏笔、细节描写；主角只是从牢笼逃到训练营，花了一万多字。另有评论说主角没有进入冒险世界，没有限定他为什么必须推进主线；开篇近百字华丽环境描写不影响剧情，不写也没损失，努力用错方向；第一章埋伏笔和氛围不如抓眼球拉期待。核心是：硬核可以写，但商业开头必须先给主角、危机、目标、金手指/亮点和故事推进。

### 核心机制

```text
硬核玄幻自我定位 -> 背景设定铺太久 -> 主视角和目标不清 -> 读者看不到爽点故事 -> 商业化失败
```

老白不是慢，不是无对话，不是先堆设定；老白也要故事先动。

### 期待链与情绪

```text
读者带着题材/同人期待进入 -> 看不到熟悉看点和主角目标 -> 设定负担增加 -> 期待落空退出
```

自带流量的世界观也要尽快兑现读者想看的东西。

### 关键判断

- 硬核老白玄幻对新人高风险。
- 老白不是娓娓道来堆设定。
- 视角要锁主角。
- 冲突不能太晚。
- 设定过多会增加阅读负担。
- 开局应先有危机、目标、金手指或亮点。
- 前 500 字足以判断开头功力。
- 英雄联盟世界观要满足粉丝期待。
- 简介要写看点和爽点，不只写背景。
- 无用环境描写应删。
- 伏笔太早埋可能没人看到回收。
- 网文没有小白老白借口，只有能不能赚钱。

### 可迁移写法

```text
商业化开头检查：
主角是否出现：
主视角是否锁定：
前500字是否有对话/行动：
第一危机：
主角目标：
金手指/亮点：
粉丝期待兑现：
可删设定描写：
是否已有故事推进：
```

### 训练任务

拿一个硬核设定开头，删掉所有不影响剧情的环境描写。前 500 字必须出现主角目标、危机、对话或行动。
