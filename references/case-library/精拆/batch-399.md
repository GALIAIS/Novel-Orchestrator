## 1991：在这里跟大家吐个槽，近几年起点...


人工查看范围：正文、全部 3 条评论。

证据限制：帖子以“孤儿主角”“精神病主角”等概括 2022 年平台趋势，并把精神异常与现实反差视为搞笑来源；没有榜单样本、书评数据、医学定义或具体桥段。评论对相关作品的判断也只是个人阅读感受。本卡不验证市场趋势，不把真实精神疾病当作喜剧标签，只提炼“人物按一套非常规但稳定的内部逻辑行动”这一可用机制，并补上非污名化门禁。

### 读者承诺

这篇沉淀的是非常规主角的喜剧不应来自一句“他有精神病”，而应来自读者能逐渐掌握的一套个人逻辑。角色可能极端认真、过度字面、风险偏好异常、把社会礼仪理解成效率问题，或坚持一条与常人优先级不同的原则；当正常情境遇到这套规则，他会作出意外却可回推的行动。读者笑的是预期错位和人物一致性，不是疾病本身。

内部逻辑至少要写清五项：角色最重视什么，遇到哪类刺激会启动特殊判断，哪些常识他理解不同，绝不跨越的底线是什么，行动后会承担怎样的现实后果。非常规不能等于随机；如果每一章只为笑点临时换一种疯法，读者既无法期待，也会把人物看成作者段子工具。

反差还要保留能力和主体性。角色的非常规判断有时能识破他人习惯忽略的问题，有时会伤害关系或让任务变复杂；同伴既可能被逗笑，也会愤怒、拒绝或要求他修正。连续反馈让人物学习如何与世界相处，而不是让所有旁人负责证明他“疯得可爱”。

若作品确实描写某种真实精神健康状况，就需要可靠研究、个体差异、治疗与生活影响等完整语境，不能把诊断当作“危险、天才、搞笑、无道德约束”的快捷包。喜剧桥段宜放在人物关系与处境错位上，不以痛苦症状、失控或污名刻板印象为笑点；缺少研究时，直接写“非常规思维方式”比随意贴医学标签更稳妥。

### 核心机制

```text
正常情境建立常规预期 -> 角色按稳定的非常规优先级解读 -> 作出意外但可回推的行动 -> 行动产生任务与关系后果 -> 同伴反馈迫使调整 -> 内部逻辑在新场景继续变体
```

稳定逻辑提供人设，场景错位提供笑点，现实后果保证角色不是无成本的随机制造器。

### 期待链与情绪

```text
读者先理解社会常规 -> 预见角色会用另一套逻辑处理 -> 具体行动超出预期 -> 后果既好笑又改变局面 -> 角色守住核心原则并学习边界 -> 下一场期待更复杂的变体
```

可预测的是思维根源，不可预测的是它在新问题中会导向什么结果。

### 关键判断

- 帖子没有平台趋势数据，不能确认题材流行度。
- 真实精神疾病不应被简化为搞笑标签。
- 非常规人物需要稳定的内部逻辑。
- 随机越界不等于反差喜剧。
- 角色应有欲望、能力、底线和主体性。
- 笑点来自处境与逻辑错位，不来自诊断名称。
- 行动必须产生任务或关系后果。
- 同伴反馈应多样，不能只负责捧哏。
- 非常规判断既可带来优势也应有成本。
- 人物需要在反馈中学习而非永远重置。
- 涉及真实病症时需要专业、可靠研究。
- 缺乏研究时应避免使用医学标签包装人设。

### 可迁移写法

```text
角色最重视的事：
非常规判断的触发条件：
与常人不同的优先级：
他理解错误或不同的社会规则：
绝不跨越的底线：
该逻辑带来的真实优势：
该逻辑造成的关系成本：
常规场景预期：
角色的意外行动：
同伴的不同反馈：
本轮学会的边界：
需要避免的医学污名标签：
```

### 训练任务

不用任何诊断词，为一名喜剧主角写“欲望、触发、优先级、底线、优势、成本”六项规则；围绕排队、谈判和救人写三场戏，行动必须都能从同一逻辑推出，却产生三种不同后果。删除只靠随机发疯成立的笑点。

## 1992：#公开处刑


人工查看范围：正文、附件元数据、全部 5 条评论；并核对知识库导出目录中的文件元数据、下载接口快照和本地实体。

证据限制：附件 `三国之风起汜水.docx`（ID `118212845251812`，21673 字节）没有落盘实体，导出远程链接实测返回 403；本地未发现同名、同 ID 或同大小 DOCX。本轮未打开评论中的作品链接，因此不能直接核对文白混杂、叙事重点、考据或具体句病。以下只综合评论者读到附件后的二手诊断，建立历史穿越开篇的最低锚点。

### 读者承诺

这篇沉淀的是“穿越成历史人物”与“穿越本身是金手指”都还只是前提，必须在当前历史节点中变成可执行优势。开篇先让读者知道主角是谁、身处哪一事件前后、原身份即将承担什么风险、他比周围人多知道哪一条有用信息，以及现在必须做出什么选择。读者不需要先背完整年代，但需要知道这次选择为什么会改变已知走向。

历史知识只有满足准确、具体、可用和有代价四项才算金手指。主角记得大势却不知道精确日期，知道某人未来立场却无法证明，掌握现代常识但缺少制度与资源，都能形成限制。第一次实证应让他用一项信息改变生存、军令、关系或战场判断，同时暴露新风险；若只是内心说“我知道历史”，知识差没有进入故事。

三国等高熟悉度题材还需要史实锚点和偏离台账。最低核对包括时间、地点、人物在场、官职称谓、军政关系、交通与消息速度；作者选择采用正史、演义或架空混合时，也要在文本承诺中保持一致。偏离发生后，后续人物就不能继续按原历史机械行动，作者需记录“改变了什么、谁最先受影响、主角原知识哪部分开始失效”。

语言风格应统一功能。叙述可以有历史质感，主角内心可以保留现代思维，人物对白则受身份和场景约束；文白混合不是天然错误，但同一角色不能在没有理由时从古雅公文跳成现代口语。评论所说“要么考据、要么创意”不必成为二选一，真正底线是：独特创意仍需足够锚点让读者看懂，严谨背景也要有当前冲突和人物选择。

### 核心机制

```text
历史节点给出既定风险 -> 主角确认身份与可用知识差 -> 在资源制度限制下作出选择 -> 第一次行动改变局部结果 -> 历史偏离削弱旧知识可靠度 -> 主角重新调查并扩大影响
```

史实锚点负责可信，知识差实证负责爽点，偏离反馈负责防止金手指无限预知。

### 期待链与情绪

```text
读者认出历史节点和人物困局 -> 主角知道一项别人不知道的后果 -> 现实条件阻碍直接改命 -> 冒险使用信息获得小胜 -> 偏离引来新的政治或军事反应 -> 期待熟悉历史怎样被逐步改写
```

历史读者获得锚点，普通读者获得清楚的眼前选择，两类入口才不会互相排斥。

### 关键判断

- DOCX 未读时不能确认评论中的具体文本问题。
- “穿越”本身不是可感的能力实证。
- 开篇要明确历史节点、身份风险和当前选择。
- 知识差必须准确、具体、可用且有代价。
- 大势记忆不等于掌握每项细节。
- 第一次知识应用应改变局部结果。
- 时间、地点、称谓和信息速度是最低锚点。
- 正史、演义和架空混合需要明确口径。
- 历史偏离后人物反应必须随之改变。
- 偏离台账要标记哪些旧知识开始失效。
- 文白混合应按叙述、内心和对白分工。
- 考据与创意都不能替代当前冲突。

### 可迁移写法

```text
采用的历史口径：
开篇时间地点：
主角原身份与既定风险：
当前在场人物和关系：
主角确定知道的信息：
主角可能记错的部分：
现有资源制度限制：
第一次可执行选择：
局部结果怎样改变：
最先受影响的人：
从此失效的历史知识：
叙述内心对白的语言分工：
需要核对的史实清单：
```

### 训练任务

为一个历史穿越开篇做一页锚点表，限制只用一条知识差破局；写 2000 字“节点—风险—选择—偏离”闭环。结尾列出三项因偏离而不再可靠的未来信息，迫使主角下一章重新调查，而非继续照历史答案行事。

## 1993：在起点作家助手看到的#思考分析...


人工查看范围：正文、唯一 1 条评论、文中作品搜索链接标题；按要求未打开作品链接或平台页面。

证据限制：正文转载一篇标为“网文全武行--开头，这么写就对了!(1)”的 2022 年教程片段，包含主角名字、单主角、第三人称、主角视角和代称等经验规则；没有当前平台数据，也只展示“未完待续”的第一部分。本卡不把“两字常见名”“男频都是单主角第三人称”等说法当普遍事实，只保留更稳定的原则：读者需要尽快获得明确叙事锚点，并知道当前信息属于谁的感知与判断。

### 读者承诺

这篇沉淀的是近距第三人称可以同时保留第三人称称谓和第一人称般的心理贴近。叙述写“他看见什么、误解什么、害怕什么、准备怎么做”，世界信息通过主角可接触的感官、记忆和推断进入；作者知道但主角不知道的结论不能突然混入。读者因此不只是旁观事件，而是在同一信息边界里参与选择。

开篇视角锚定至少回答三问：当前焦点人物是谁，他身处什么情境，他现在最在意什么。可以先用旁人讨论、宏观异常或短暂外部镜头烘托主角，但必须迅速证明这些内容怎样影响或指向主角；否则镜头越大，读者越不知道该把情感投给谁。离开主角视角的段落也要服务主线，并有清楚的切换边界。

人物命名的核心不是一定两字或常见姓，而是可读、可区分、符合世界和目标受众。主角不应长期被“少年、青年、神秘人”等代称遮住身份；配角首次以外貌代称进入后，应在叙事需要时尽快获得稳定称呼。频繁轮换“男人、青年、他、某某”会让读者误以为是不同人，也会制造不必要的距离。

心理活动应属于当前视角人物，并与动作互相验证。可以直接写贴近人物语言习惯的内心判断，也可以用自由间接引语让叙述染上人物语气；但不能一段内无提示进入多人内心。第三人称不是全知信息随意倾倒，贴近也不等于每件事都用内心独白解释。

### 核心机制

```text
快速确认焦点人物 -> 感官与当前需求建立信息边界 -> 世界设定通过观察记忆和推断进入 -> 心理判断推动动作 -> 结果修正人物认知 -> 视角切换时明确交接
```

称谓负责定位人物，信息边界负责代入，选择与后果负责让视角真正参与剧情。

### 期待链与情绪

```text
读者知道跟随谁 -> 共享他有限的信息和压力 -> 与他一起形成判断 -> 行动验证或推翻判断 -> 认知变化引出下一选择
```

代入感来自持续共享同一问题，不来自反复说“他很震惊”。

### 关键判断

- 2022 年教程经验不能直接代表当前平台规律。
- 常见两字名是选项，不是普遍硬规则。
- 单主角第三人称也不是所有类型唯一答案。
- 开篇需要尽快确认焦点人物。
- 外部镜头必须明确指向主角及其主线。
- 近距第三人称有稳定的信息边界。
- 主角未知结论不能无提示进入叙述。
- 人名应可读、可区分并符合世界。
- 同一人物不宜频繁更换代称。
- 心理活动必须属于当前视角人物。
- 无提示进入多人内心会造成视角跳跃。
- 内心判断要推动动作而非替代事件。

### 可迁移写法

```text
当前视角人物：
他此刻最在意的问题：
可感知的信息：
只能推断的信息：
作者知道但本段不能写的信息：
稳定称呼：
首次出现的临时代称：
人物语言化的内心判断：
该判断推动的动作：
结果怎样修正认知：
需要切换视角的原因：
切换边界与新锚点：
```

### 训练任务

取一章多人场景，用颜色之外的文字标记每句信息属于观察、记忆、推断还是作者全知；删除当前视角人物不可能知道的结论。再统一人物称呼，确保每次视角切换都有分段、时间地点或新焦点锚点。

## 1994：【进阶】写作技巧:网文全武行-...


人工查看范围：正文、唯一 1 条评论、文中三个作品搜索链接标题；按要求未打开作品链接。

证据限制：这是同一 2022 年开头教程的第三部分，作者明确称所给格式只是“之一”，但正文同时包含不要大背景、单成长主线、不要路遇、普通或弱势身份等强经验判断；评论进一步提出常见具体出生点和从环境/人物推断信息。没有实际投稿样本或对照数据。本卡把它整理成新人可用模板，不把路遇、倒叙、多线或特殊身份一概判为错误。

### 读者承诺

这篇沉淀的是穿越或陌生世界开篇可以从一个“常见而有特点”的出生点开始。常见意味着读者不用先理解复杂概念就知道空间用途，例如住处、病房、牢房、办公室或交通节点；有特点意味着光线、材料、器物、声音和人物态度中，至少有两项能暗示世界规则、主角身份或当前危险。场景不是风景展示，而是第一轮可交互情报。

出生点应选择主线的交汇点。主角为何在这里，身体或社会状态出了什么问题，谁正在影响他，离开或留下会造成什么；这些问题自然推出第一项行动。若主角被照料，照料者的态度、用词和权限可以暴露地位与关系；若房间异常，主角观察并试探的结果应帮助决定合作、逃跑、伪装或调查。浅层信息直接可见，深层信息通过互动获取。

开篇主线做减法，不等于整本只能一条线。第一章先让一个当前需求统领场景、身份回忆、金手指和配角；职业、家庭、情感和成长线以后可以加入，但必须与当前行动发生接口。身份或职业本身应持续生成事件，比无关的随机路遇更容易保持因果；路遇也不是禁用，而是要证明它为何只会落到这个主角及其主线。

第一章结束前至少发生一次不可撤销变化：主角接受任务、暴露异常、得罪某人、获得但必须使用能力，或作出离开安全区的决定。宏大背景只在这一变化需要时显影。高身份主角也可以代入，只要局势、目标和损失足够具体；低位身份也不自动有压力，必须真的存在选择成本。

### 核心机制

```text
选择常见而有特点的出生点 -> 环境和人物提供浅层信息 -> 主角根据当前需求观察试探 -> 深层身份与危险逐步显出 -> 作出第一项不可撤销选择 -> 章末接入主线
```

场景负责锚定，取证负责代入，选择负责把静态出生点变成故事起点。

### 期待链与情绪

```text
读者迅速理解空间用途 -> 特殊细节暗示陌生世界 -> 主角从互动中确认身份处境 -> 当前需求逼出计划 -> 第一选择改变局面 -> 期待主线后果
```

信息越能立刻影响主角决定，环境越不像设定说明。

### 关键判断

- 该格式是新人友好选项，不是唯一开头。
- 常见场景降低空间理解成本。
- 特殊细节要提示世界、身份或危险。
- 环境信息应能被主角用于判断。
- 出生点最好与第一主线事件相交。
- 浅层信息可见，深层信息靠互动获得。
- 第一章先由一个当前需求统领材料。
- 多条长线可以在后续逐步接入。
- 身份和职业应持续生成因果事件。
- 路遇不是禁用，但需与主角主线有必然接口。
- 高身份与低身份都要有具体弱势或损失。
- 章末至少发生一次不可撤销变化。

### 可迁移写法

```text
出生点的常见用途：
区别于普通场景的两个细节：
```

## 1995：【进阶】写作技巧:网文全武行-...


人工查看范围：正文、唯一 1 条评论、文中四个作品搜索链接标题；按要求未打开作品链接。

证据限制：这是同一 2022 年教程的第二部分，集中劝新人避开穿越适应过程、中二独白、床上醒来、系统戏多、无厘头名称和大量颜文字；这些是作者审稿经验，不是跨题材定律。评论询问把穿越前反转放到第二章梦境是否仍是毒点，但没有后续回答。本卡保留“当前困境优先、前史按因果后置、金手指服务人物目标”的核心，不绝对禁止床醒、系统人格、轻小说符号或穿越过程。

### 读者承诺

这篇沉淀的是穿越开局最先要交付的不是“主角如何确认自己穿越”，而是他现在是谁、正面对什么问题、能做出哪项选择。醒来、掐腿、询问年代和长篇适应过程之所以常显拖沓，不是动作本身有罪，而是它们没有改变当前困境。若醒来即发现有人要处置自己、身体异常或身份即将暴露，床上场景同样可以迅速进入主线。

穿越前经历只在三种情况下值得进入正文：解释当前欲望或恐惧，提供此刻可用的知识，或构成后续必须回收的关系与谜团。第一章可以只给一个触发片段，先完成当前冲突；之后由物品、梦境、伤口、相似场景或人物选择触发回忆。梦境不是天然安全的后置方式，若只是把被删说明原样搬到第二章，仍会停住剧情。

金手指应把人物目标变成更有趣的行动，而不是代替人物决定主线。系统提供能力、规则、反馈或限制，主角负责选择是否接受、如何利用、承担什么代价；任务若完全任意、系统对话占据篇幅、人格化助手持续抢戏，主角会退化成执行者。名称搞笑、系统吐槽或颜文字也不是普遍禁忌，但必须属于作品稳定风格，并且不能损伤关键危机和阅读流畅。

开篇爽点需要“困境—选择—能力—结果”闭环。系统出现之前先让读者理解什么问题难解，出现后立即展示它改变了哪项选择；奖励不能只写面板数值，应落到身体、资源、身份或关系。主角若只和系统斗嘴，却没有对外行动，金手指存在感很高，故事存在感仍然很低。

### 核心机制

```text
直接进入穿越后的当前困境 -> 最少信息确认身份与需求 -> 主角作出选择 -> 金手指提供规则或能力 -> 主角主动利用并承担代价 -> 结果改变外部局面 -> 前史按触发点后置回收
```

穿越负责制造信息差，金手指负责扩大行动可能，人物选择才负责让故事前进。

### 期待链与情绪

```text
读者先看懂主角眼前难题 -> 身份信息只补解决所需部分 -> 金手指打开一条新解法 -> 主角冒险运用 -> 外部结果完成首轮反馈 -> 前史碎片解释更深动机
```

能力越快进入人物行动，系统越不需要靠大量对白证明存在。

### 关键判断

- 2022 年审稿经验不能外推为所有题材规则。
- 穿越确认动作的问题在于是否推进困境。
- 床上醒来可以用，只要危机和选择立即成立。
- 前史应解释当前动机、能力或长期谜团。
- 后置不等于把整段说明搬到第二章。
- 梦境必须由当前事件触发并产生新意义。
- 金手指提供可能，主角负责选择。
- 任意任务会夺走人物因果归属。
- 系统人格不能持续盖过主角与现实关系。
- 搞笑名称和符号需服从稳定风格。
- 数值奖励要落到可观察结果。
- 首轮爽点应完整改变外部局面。
