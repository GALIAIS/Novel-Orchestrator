## 1131：#公开处刑 之前在起点写了一篇...


人工查看范围：正文、文件区、全部 9 条评论及楼中楼。文件区有 `暂定.docx`，只有 id、大小、下载数，无下载链接，未能读取。

证据限制：未读取正文附件，不能评价具体开头文本；拆解依据来自作者设想和评论反馈，集中在同人背景、力量体系互换、代入感和期待缺失。

### 读者承诺

这篇沉淀的是同人仙侠混搭的代入风险：用火影背景写仙侠，不能只靠“熟悉 IP + 修仙体系”成立；读者还要能代入主角，并且开头要先给明确期待，而不是堆角色、遭遇、目标、行动和阻碍。

### 核心机制

```text
失败旧稿 -> 同人混搭新稿 -> IP受众筛选 -> 代入感风险 -> 期待先行再铺设定
```

作者起点仙侠文 14 万字未签，想切掉重写，用火影忍者为背景板写仙侠。评论指出：博人传受众有限，大多数人对佐助的印象还停留在少年时期；以动漫角色为主角，代入感和人设符合度都是大麻烦；火影修仙只是力量体系互换，不如改成诸天流。另有评论说：人物、遭遇、目标、行动、阻碍都有，但没看头，原因是读者没期待、代入感缺失。

### 期待链与情绪

```text
读者看到熟悉 IP -> 期待熟悉人物或新爽点 -> 角色人设不匹配/主角难代入 -> 兴趣下滑 -> 需要先给可追期待
```

同人混搭的情绪难点在于：读者既会用原作印象审判角色，又需要新故事提供自己的期待。如果二者都没抓住，设定越多越像废话。

### 关键判断

- 同人题材高度依赖原作受众。
- 动漫角色当主角时，代入感比原创主角更难。
- 改动原作人物能力体系，要处理人设一致性。
- 力量体系互换只是创意壳，不等于故事钩子。
- 开头先给读者期待，再铺人物和遭遇。
- 若代入困难，可改诸天流或原创主角介入。

### 可迁移写法

```text
原作受众：
主角选择：
原作人设约束：
新体系改动：
读者代入点：
第一期待：
无关设定：
可替代结构：
```

### 训练任务

把一个“火影修仙”开头改成两版：一版保留原作角色但严格匹配人设，一版改成原创主角诸天流介入。每版写出前三章读者期待、主角代入点、原作元素使用边界。

## 1133：看了公开处刑里的一些开头后，就...


人工查看范围：正文、全部 11 条评论及楼中楼。无附件、无图片。

证据限制：这是星友方法论总结和评论讨论，不是严格分类理论；“人物推动型天赋专利”是评论中的经验判断。

### 读者承诺

这篇沉淀的是新人选主推进类型：网文可以粗分人物推动和情节推动。新人不要一开始两边都贪，先判断自己要靠角色互动带故事，还是靠固定主线推进故事。

### 核心机制

```text
人物推动 -> 人设互动生剧情
情节推动 -> 固定目标带节奏
新人先选一边 -> 避免贪多嚼不烂
```

正文认为人物推动型依靠主要角色之间的有趣互动生成故事，典型如日常、恋爱、慢节奏、家长里短；重点是人物设定。情节推动型则一开始有明确固定主线，如复仇、找亲人、救人、商战、平天下、生存、阻止末日等，重点是大纲和节奏。评论补充：人物推动型很难，是天赋型领域；人物行动要有逻辑、符合人设、还要有趣，角色还要成长并符合成长逻辑。

### 期待链与情绪

```text
新人看公开处刑 -> 发现开头混乱 -> 用推进类型归因 -> 先做好一项 -> 再考虑人物和情节平衡
```

人物推动给读者的是角色活起来的陪伴和互动期待；情节推动给读者的是目标清晰、节奏明确、阶段推进。

### 关键判断

- 人物推动和情节推动是两种不同难点。
- 人物推动重在人设、互动、对白、成长逻辑。
- 情节推动重在主线目标、大纲、节奏。
- 新人先写好一种，不要同时满配。
- 情节推动文先把主角写好就很重要。
- 群像和人物推动难度高，容易进入大神领域。

### 可迁移写法

```text
作品推进类型：
固定主线：
核心人物：
互动来源：
大纲节点：
人设差异：
新人先练哪边：
暂时放弃哪边：
```

### 训练任务

为同一个创意写两版开篇设计：人物推动版写 3 个角色的人设差异和互动冲突；情节推动版写 1 个固定主线目标和 5 个阶段节点。比较哪版更适合自己当前能力。

## 1134：《济公传奇》，我愿意称之为古代...


人工查看范围：正文、全部 4 条评论。无附件、无图片。评论含《康熙微服私访记》微信读书搜索链接。

证据限制：未展开外链，未逐回分析《济公传奇》文本；拆解依据来自原帖对济公模式的概括和评论对精神寄托/救世主情结的补充。

### 读者承诺

这篇沉淀的是古代都市救世爽文：读者看济公，不一定代入李修缘本人，而是期待在遭遇不公时，有一个高能力、高道德、低姿态的救世者出来惩恶扬善。

### 核心机制

```text
民间不公 -> 济公疯癫入场 -> 法力/智慧识破 -> 惩恶救善 -> 公道释放
```

原帖认为济公是活佛转世，能掐会算，疯癫外壳下其实聪慧善良，能让宰相佩服，用法术惩治恶人、帮助好人，还能结识官员员外。评论补充：古代人看济公更多是精神寄托，而不是代入李修缘；济公反映的是救世主情结，“济”字直接点明救济核心梗。康熙微服私访记也有异曲同工之处。

### 期待链与情绪

```text
小人物受冤 -> 恶人得势 -> 济公看似不靠谱 -> 一眼识破 -> 恶人自食其果 -> 群众获得公道
```

爽点来自低姿态入场和高位格裁决的反差：疯癫、落魄、闲散只是外壳，内在是洞察、法力、人脉和善心。

### 关键判断

- 济公模式更偏救世主期待，不是普通主角代入。
- 疯癫外壳能降低主角压迫感。
- 法力、智慧、人脉共同支撑裁决爽点。
- 单元故事适合民间冤屈和小人物困境。
- 惩恶扬善要有正能量和趣味。
- “济”字本身就是救济核心卖点。

### 可迁移写法

```text
民间冤屈：
恶人权势：
主角外在伪装：
主角内在权能：
识破方式：
戏谑惩戒：
好人获救：
群众情绪释放：
```

### 训练任务

写一个 3000 字单元：主角表面不靠谱，实际一眼看穿地方恶霸骗局。要求包含一次被误解、一次反转显能、一次恶人自食其果、一次群众情绪释放。

## 1135：内投在鹿鸣那过了，攒稿多少比较...


人工查看范围：正文、全部 9 条评论及楼中楼。无附件、无图片。

证据限制：这是个人经验问答，没有平台规则或编辑要求原文；存稿数字只能作为经验区间，不是固定标准。

### 读者承诺

这篇沉淀的是开书前存稿和更新节奏管理：内投通过后，存稿不是越多越好，而是用来救急的缓冲带。真正关键是日常码字速度能不能跟上更新速度。

### 核心机制

```text
内投通过 -> 估算存稿 -> 评估日码能力 -> 存稿作缓冲 -> 开书后持续补充
```

评论给出两个经验口径：死翼耐萨里奥个人 5-7 万字左右，另一位建议 3 万字左右，太多的话前面要改动会全寄。讨论进一步指出：写得慢的话囤多囤少都不顶用，还是得码字速度跟上更新速度；存稿只是卡文或特殊情况救急。作者 3-4 小时写一章，知道写什么但过程中纠结词句，评论认为上游工作没出问题，但纠结速度要靠自己克服，没有捷径。

### 期待链与情绪

```text
内投过稿兴奋 -> 担心开书跟不上 -> 寻找存稿数字 -> 暴露写作慢 -> 回到日更能力训练
```

存稿能缓解焦虑，但不能代替生产能力。真正安全感来自稳定产出。

### 关键判断

- 3 万或 5-7 万都是经验区间，不是硬规则。
- 存稿太多，前文大改时损失更大。
- 存稿只应作为卡文和突发情况缓冲。
- 开书前要计算真实日更能力。
- 写作慢要练完成度，不能每个词都卡。
- 知道写什么却纠结表达，说明上游规划尚可。

### 可迁移写法

```text
目标更新频率：
每日真实字数：
单章耗时：
最低缓冲天数：
计划存稿量：
可能大改风险：
卡顿原因：
开书条件：
```

### 训练任务

连续 7 天记录有效码字时间、字数、卡顿原因；每天执行“先写完一章，再允许改 3 处词句”。最后计算真实日更能力和最低安全存稿量。
