## 421：推一个我自己目前在用，也觉得比...

人工查看范围：正文、全部 8 条评论。无附件，无图片。

### 读者承诺

这篇提供一种比固定大纲模板更能推进剧情的纵横轴大纲法：先定 10 万字大目标，再纵向追问达成目标需要什么、需求从哪里来，横向列出主角已有底牌、身份、人脉、金手指等可能资源，然后随机组合成小目标，最后把小目标串向大目标。

### 核心机制

```text
10万字大目标 -> 纵向需求链 -> 横向底牌库 -> 组合小目标 -> 串成大剧情
```

### 期待链与情绪

```text
作者知道终点但不知道过程 -> 纵向拆需求 -> 横向补资源和可能性 -> 小事件自然生成 -> 大剧情变顺
```

### 关键判断

- 第一步先确定主线方向、吊点和 10 万字主角要达成的大目标。
- 大目标可以是十万字化神、去上界找人。
- 纵向提问是：达成目标需要什么、需求的获得途径是什么。
- 横向列出主角金手指、人脉关系、身份、性格等。
- 纵轴是需求和需求可能来源。
- 横轴是主角持有的底牌和达成需求的各类可能。
- 一纵一横随机组合，可以生成小目标。
- 小目标要有主角目标、阻碍、目标达成、收获、额外收获。
- 小目标组合在一起，通往最后大目标。
- 核心是先想目标和需求，也就是最后收获，再想如何达成。
- 手弩理论、倒计时、虚晃一枪等可插入小目标过程中。
- 横纵轴里剧情点种类越多，大情节越精彩。
- 看过多少书，决定能想到多少种可能来源和底牌。

### 可迁移写法

纵横轴大纲表：

```text
10万字大目标：
当前吊点：
纵轴需求1：
纵轴需求2：
纵轴需求来源：
横轴底牌1：
横轴底牌2：
横轴人脉/身份：
组合出的小目标：
阻碍：
收获：
额外收获：
如何通往大目标：
```

### 训练任务

给自己的作品写一个 10 万字大目标。列 5 个纵向需求、5 个横向底牌，随机组合出 5 个小目标。每个小目标必须有阻碍、收获和额外收获。

## 422：#思考分析 这几天拆书的一个感...

人工查看范围：正文。原文无评论、无附件、无图片。

### 读者承诺

这篇提醒拆书前先判断事件关联性。符合三幕式等故事结构的小说，通常存在一件事与其余事件都有不同程度关联；但日常文可能每件事关联不强，主角更多起串联作用，后期才可能把事件拧合起来。拆书方法不能一刀切。

### 核心机制

```text
拆书前判断类型 -> 事件强关联/弱关联 -> 选择结构拆法 -> 主角串联或核心事件串联 -> 再拆卖点
```

### 期待链与情绪

```text
拆书者想套统一结构 -> 先看事件之间是否互相关联 -> 结构文抓核心事件 -> 日常文抓主角串联 -> 拆法更贴合样本
```

### 关键判断

- 拆书前要思考小说中各事件的关联性。
- 一般符合故事结构的小说，会有一件事与其他事件发生不同程度关联。
- 三幕式结构可作为强关联故事的参考。
- 存在不符合传统故事结构的小说。
- 日常文常见每一件事关联性不大。
- 日常文里主角起到串联作用。
- 日常文可能后期才把所有事件拧合起来。
- 拆结构前要先判断样本是强事件线还是弱事件线。
- 对弱事件线作品硬套三幕式容易误拆。

### 可迁移写法

拆书前事件关联表：

```text
作品类型：
核心事件：
事件1与核心事件关系：
事件2与核心事件关系：
事件3与核心事件关系：
是否三幕式强结构：
主角是否只是串联者：
事件后期是否拧合：
应该用结构拆还是卖点拆：
```

### 训练任务

选一本结构文和一本日常文，各列前 10 个事件。标注每个事件是否指向同一个核心问题。结构文拆核心事件链，日常文拆主角如何串联体验。

## 423：反派不存在一生之敌的说法？

人工查看范围：正文、全部 7 条评论。无附件，无图片。

### 读者承诺

这篇讨论长期反派的显隐节奏。明确和主角爆发冲突的反派要尽快解决；如果想做一生之敌，就应先隐秘存在、用暗示不断揭露，彻底亮相后最多拖三卷。反派能否长期存在，取决于利益冲突是否贯穿全书、人设魅力是否足够，以及其他内容能否持续提供爽感。

### 核心机制

```text
长期利益冲突 -> 隐秘暗示 -> 中后期清晰认知 -> 亮相后限期解决 -> 避免狗皮膏药
```

### 期待链与情绪

```text
读者感到幕后威胁存在 -> 信息逐步揭露 -> 反派亮相抬高期待 -> 主角阶段性解决 -> 拖太久则转为烦躁
```

### 关键判断

- 长期反派可以一直隐秘存在，用暗示不断揭露。
- 彻底亮相后最多拖三卷，再多读者会腻。
- 如果一开始说明白又一直拖，读者等不了那么久。
- 《诡秘之主》天尊/上帝早期有影响，但清晰认知在中后期。
- 一生之敌需要和主角利益从开书冲突到完书。
- 是否能设计，要看是什么利益冲突能持续一整本。
- 反派人设足够有魅力时，读者能允许他活得久一点。
- 其他内容足够爽，也能提高读者忍耐。
- 阿蒙式一生之敌一般作者把握不了。
- 明确爆发冲突的反派要尽快解决。
- 如果近期不解决，不要让他明确和主角爆发冲突，只让观众知道他后期会是反派。
- 一直升级、一直逃跑、一直差点赢的反派会像狗皮膏药，后期烦躁。
- 宿敌也可以变成调节气氛的迫害对象，类似口袋妖怪玩家宿敌，被主角一直吊打。

### 可迁移写法

长期反派显隐表：

```text
反派身份：
贯穿全书的利益冲突：
早期暗示：
中期揭露：
首次明确冲突：
彻底亮相卷数：
亮相后预计解决卷数：
人设魅力：
其他爽点支撑：
是否会变狗皮膏药：
```

### 训练任务

设计一个长期反派。写出 5 次暗示、1 次清晰亮相和 3 卷内解决方案。如果无法在 3 卷内解决，就不要让他早期正面冲突。

## 424：我想问下，在番茄写不同题材的文...

人工查看范围：正文、全部 4 条评论。无附件，无图片。

### 读者承诺

这篇不是题材分析，而是新人和编辑沟通的执行问题：在番茄扑了一本传统仙侠，想换成都市脑洞末世神话，是否换编辑不必凭尴尬感决定。正确做法是直接问现有编辑或查作者群公告。切书再开是常规操作，不声不响跑掉反而更不礼貌。

### 核心机制

```text
旧书扑街 -> 题材方向调整 -> 编辑沟通/群公告确认 -> 决定是否换编辑 -> 新书重开
```

### 期待链与情绪

```text
作者因扑街产生尴尬 -> 想换题材又怕问编辑 -> 评论降低社交压力 -> 直接沟通恢复流程
```

### 关键判断

- 作者之前在番茄签过传统仙侠，有明确编辑。
- 旧书写得偏传统，人物成长慢，配角刻画过多。
- 想通过配角悲惨让主角成长和蓄势爆发，但首秀扑得厉害。
- 作者意识到写作思路不符合网文商业化。
- 新构思是都市脑洞末世神话。
- 核心问题是换题材是否要换编辑。
- 评论认为切书换一本是常规操作。
- 不声不响跑了比直接问编辑更不礼貌。
- 番茄作者通常有 Q 群，编辑会发公告。
- 可以先看往昔公告，通常会说明相关规则。
- 扑街后的沟通不要被尴尬情绪卡住。

### 可迁移写法

换题材编辑沟通表：

```text
旧书题材：
旧书问题：
新书题材：
是否同编辑负责：
作者群公告是否说明：
需要问编辑的问题：
是否已说明切书原因：
是否提交新书核心卖点：
下一步动作：
```

### 训练任务

写一条给编辑的简短说明：旧书为什么停、新书是什么题材、核心卖点是什么、是否需要换编辑。控制在 200 字内，直接发送，不反复内耗。

## 425：应该是近期翻的开局最直接、最明...

人工查看范围：正文、全部 11 条评论、2 张图片。图片已下载并人工查看：`case-library\精拆\attachments\batch-085-case-425\image-1.jpg` 到 `image-2.jpg`。

图片证据：图片为《圣女帮我模拟修仙》书页与第一章截图。书页显示仙侠/幻想修仙、71 万字、五万收藏、男生畅销榜、标签含幕后流/BOSS流/轻松；简介直接写主角觉醒修仙模拟器，模拟器内存活越久获得奖励越多。第一章截图直接出现“欢迎进入修仙模拟器”“第三十六次模拟修仙人生”“选择新的模拟修仙人生”“从十项初生天赋中选三项”等模拟器规则。

### 读者承诺

这篇评价的是开局直接、清楚知道读者想看什么。样本《圣女帮我模拟修仙》一上来就把模拟器题材的核心承诺摆出来：进入模拟、选择天赋、让工具人或角色在模拟中存活更久，从而给主角带来奖励。它不靠废话铺垫，而是第一章就展示玩法。

### 核心机制

```text
题材核心玩法 -> 第一章直接展示 -> 模拟选择 -> 工具人产出 -> 主角获得奖励 -> 下一次模拟期待
```

### 期待链与情绪

```text
读者冲模拟器玩法进来 -> 立刻看到规则和选择 -> 等待模拟人生结果 -> 工具人越努力主角收获越多 -> 失败后期待下一轮更优解
```

### 关键判断

- 宝剑评价这是近期翻到的开局最直接、最明白读者要看什么的小说。
- 图片显示书名《圣女帮我模拟修仙》，题材和玩法直接写在标题里。
- 简介直接说明主角穿越仙侠世界，觉醒修仙模拟器。
- 只要模拟器内存活时间够久，就能获得大量奖励。
- 第一章开头直接进入修仙模拟器。
- 已经是第三十六次模拟，说明玩法立即运转，不从零解释太久。
- 主角可选择继续上一次模拟或刷新新人生。
- 新人生可立即刷新初生天赋。
- 评论中有人不喜欢模拟器题材，宝剑和念之秋都建议看不下去就不要写。
- 作者都不知道题材哪里有趣，读者更不会觉得有趣。
- 模拟器题材是否火不是重点，样本价值在开头直接。
- 评论提到主角存在感低和割工具人韭菜可能引发读者分歧。
- 也有人认为工具人被主角强迫劳动，反而强化主角主动性，像被游戏代练带飞。
- 每次模拟都能期待工具人带来更多收获，工具人死了则期待下次模拟获得更高收益。

### 可迁移写法

```text
主角收获：
工具人作用：
下一轮模拟期待：
```
