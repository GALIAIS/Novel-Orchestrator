## 131：看宝剑老师讲简介，想到《救猫咪...

人工查看范围：正文、全部评论、四张图片。图片包括《救猫咪》片名/一句话故事例子、《我的魅力只对坏女人有效》书页、《我在精神病院学斩神》简介页。

### 读者承诺

这篇把书名简介当成“第一钩子系统”来理解：一句话简介要有意外反转、美妙脑补片段、目标观众筛选和成本/基调控制。书名最好同时具备钩子、脑补空间和读者筛选功能，先在读者点开前完成一次期待预热。

### 核心机制

```text
书名/简介 -> 意外反转 -> 脑补片段 -> 关键词筛选读者 -> 开篇迅速接故事矛盾
```

### 期待链与情绪

```text
读者看到反常信息 -> 自动提出问题 -> 脑补爽点场景 -> 确认自己是目标受众 -> 点击正文 -> 第一章兑现钩子
```

### 关键判断

- 一句话简介要包含意外，也就是意料之外的反转。
- “植物人老公突然站起来”“我真没绑架女明星”这类标题，靠反常信息制造问题。
- 美妙脑补片段是让读者自动想象后续场面。
- 女频简介常用小剧场，是为了让读者提前脑补关系张力。
- 书名简介也要筛选读者，明确是无敌、苟、马甲、萌宝、脑洞还是新媒体。
- 25 岁以上读者常受生活压力影响，更容易吃新媒体强情绪和职场/家庭境遇。
- 25 岁以下读者更容易吃爽文、脑洞文和噱头。
- 番茄等平台常在简介前用括号补关键词，帮助读者快速定位。
- 《救猫咪》的片名例子强调：片名和一句话故事要有反讽/意外、期待感、目标观众和成本意识。
- 注意力时代，网文书名简介要向自媒体标题学习。
- 书名、简介、开篇切入点都要第一时间抓住读者，然后马上拉故事和矛盾。

### 可迁移写法

书名简介四要素表：

```text
书名：
意外反转：
读者会脑补的片段：
目标读者：
筛选关键词：
平台/年龄层：
简介小剧场：
第一章兑现点：
```

### 训练任务

为同一个故事写 3 个书名简介版本：一个偏新媒体，一个偏脑洞爽文，一个偏女频小剧场。每版都标出意外、脑补片段和筛选读者的关键词。

## 132：#公开处刑

人工查看范围：正文、全部评论。重点查看《末世：巨人与丧尸同在》、巨人/丧尸借设定、绿帽压制主角、前身被绿是否仍毒、鬼屋类型三要素、丧尸老题材和书名看点问题。

证据限制：附件 `末世：巨人与丧尸同在【公开处刑】.docx` 只保留文件 id；本卡依据可见正文与评论精拆。

### 读者承诺

这篇的关键是“末世巨人丧尸”本该提供鬼屋式生存压迫，而开局绿帽会把读者注意力引到人格损伤和憋屈上。男频开篇就近时间线内不要让主角或主角身份沾上无法挽回的损伤，哪怕是前身被绿，读者也会把不适带到主角身上。

### 核心机制

```text
无法逃避空间 -> 可怕怪物 -> 原罪/真相 -> 生存紧迫 -> 主角破局
```

### 期待链与情绪

```text
读者看到巨人/丧尸威胁 -> 明白人类无法逃离 -> 期待主角在鬼屋规则中求生 -> 真相和格局改变推进 -> 不被绿帽等无关毒点打断
```

### 关键判断

- 绿帽是男频强毒点，不适合拿来压主角。
- 就算是前身被绿，读者仍然容易把人格损伤粘到主角身上。
- 就近时间线内不要有无法挽回的损伤，如父母刚死、被绿、人格受辱等。
- 压主角的理由很多，不缺绿帽。
- 如果主要故事是丧尸和巨人，绿帽和核心矛盾关联弱，就没有保留意义。
- “鬼屋”三要素：无法逃避的空间、可怕的鬼怪、导致这一切的原罪。
- 《进击的巨人》的人类墙内生存、吃人巨人、人为制造巨人的真相，就是典型框架。
- 人类、丧尸、巨人的三角关系有看点，但不要中途换成种田路子。
- 故事核心要凝练有力，中途换路子会废。
- 紧迫感不是全部，还需要钓读者的看点和亮点。
- 丧尸题材很老，除非有强脑洞，否则可考虑转向诡异或规则类变体。

### 可迁移写法

末世鬼屋框架表：

```text
无法逃避空间：
主要怪物：
怪物规则：
原罪/真相：
主角第一目标：
不该使用的无关毒点：
第一章看点：
第一卷格局变化：
是否中途换路子：
```

### 训练任务

把一个末世开篇里的“绿帽压迫”删掉，改成与生存鬼屋直接相关的压迫：资源、感染、墙内规则、基地驱逐或怪物追杀。

## 133：#思考分析 星友的讨论给我带来...

人工查看范围：正文、全部评论。重点查看小白文系统、系统给主角带来好处、系统不要背刺主角、反系统逻辑、系统辅助主角魅力和系统文边界的讨论。

### 读者承诺

这篇强调小白系统文里，系统出现首先应该帮主角打开局面。读者默认系统是主角的东西或助手，如果系统上来先嘲讽、制约、浪费钱、背刺主角，还让主角短期处理不了，读者会把它当成主角无能和吃亏。

### 核心机制

```text
主角低潮 -> 系统带来强积极意义 -> 外部第三方施压 -> 主角借系统打开局面 -> 代价被爽感冲淡 -> 主角仍有判断力
```

### 期待链与情绪

```text
读者期待开挂 -> 系统给出有效能力 -> 主角局面扭转 -> 外部压力被打回去 -> 读者确认系统属于主角 -> 后续看主角如何用系统
```

### 关键判断

- 系统出现必然要给主角带来好处。
- 系统不要上来什么都没做，先把主角整毛。
- 主角和系统很多时候是绑定、一体的，系统打主角脸等于自己打自己脸。
- 压力最好来自外界第三方，而不是系统内部背刺。
- 系统可以有使用代价，但好处要足够大，能冲淡负面影响。
- 系统和主角不要产生当前无法立刻解决的冲突。
- 恶意系统如果短期化解不了，会让读者觉得主角无能。
- 系统像天降网盘 VIP，读者要的是加速，不是个人资料泄露。
- 系统要搞事，也要有铺垫、无实质损失，并给高额补偿。
- 主角可以反系统，但必须明确知道这样做能得到更多好处。
- 主角地位应在系统之上，不能沦为系统舔狗或工具人。
- 小白文不等于系统文，系统只是降低门槛的金手指之一。

### 可迁移写法

系统关系边界表：

```text
主角低潮：
系统第一份好处：
压力来源是否外部：
系统代价：
爽感是否冲淡代价：
系统是否背刺：
主角是否能立刻处理：
反系统是否有更大好处：
主角主观判断：
```

### 训练任务

设计一个系统开局：系统第一章必须帮助主角扭转外部压力。若系统带来代价，必须同章给出更大的补偿或可见收益。

## 134：提问：关于情绪上下行

人工查看范围：正文提问、全部评论。重点查看情绪上下行是否等于读者情绪起伏、下行主体可以是主角/读者/配角/围观群众，以及相关教程指向。

### 读者承诺

这篇虽短，但给出了拆情绪上下行的重要校准：上下行不只是一条“读者心情曲线”，它可能是主角情绪、读者情绪、配角/围观群众情绪，具体看文章类型和场景功能。拆书时不能只写“读者爽/读者压”，要先说明是谁的情绪在升降。

### 核心机制

```text
场景类型 -> 确认情绪主体 -> 上行/下行 -> 交叉影响读者 -> 爽点或期待形成
```

### 期待链与情绪

```text
主角受压/配角误判/围观轻视 -> 某一方情绪下行 -> 读者形成期待 -> 反转或兑现 -> 多方情绪上行/下行换位
```

### 关键判断

- 情绪上下行不是只看读者心情。
- 情绪下行可以是主角的情绪。
- 情绪下行也可以是读者的情绪。
- 情绪下行还可以是配角、围观群众的情绪。
- 不同文章类型，对情绪主体的划分不同。
- 拆书时先标注主体，再标注上行/下行。
- 情绪拉扯、爽点制造、情节下行和节奏教程应合并理解。
- 围观群众的误判下行，往往会转化为读者的装逼期待。
- 主角的下行不一定让读者下行，读者可能因为知道底牌而上行。

### 可迁移写法

情绪主体标注表：

```text
场景：
主角情绪：
读者情绪：
配角/反派情绪：
围观群众情绪：
谁在上行：
谁在下行：
是否转化为期待：
兑现后谁反转：
```

### 训练任务

拆一段装逼桥段，分别标出主角、反派、围观群众、读者四方情绪。不要只写“上行/下行”，必须写明主体。

## 135：写明白提升前的憋屈压抑、没实力...

人工查看范围：提问、回答、全部评论、一张图片。图片为“欲望 + 阻碍 + 克服 = 达成”，克服处带来读者爽点，下方标注改变。

### 读者承诺

这篇回答的是男频爽文最基础的问题：实力提升为什么爽。爽感来自提升前的憋屈压抑、没实力寸步难行，提升本身对多数人很困难，提升后又能一雪前耻。没有前面的欲望、阻碍和无力感，提升只是数值变化。

### 核心机制

```text
欲望 -> 阻碍 -> 没实力寸步难行 -> 艰难提升 -> 克服阻碍 -> 达成目标/一雪前耻
```

### 期待链与情绪

```text
主角想要 -> 被现实/敌人卡住 -> 读者感到憋屈 -> 提升很难但完成 -> 回去解决原阻碍 -> 爽点成立
```

### 关键判断

- 提升前要写明白憋屈压抑。
- 没实力要造成寸步难行，而不是口头说弱。
- 实际提升对多数人都很困难，才显得主角提升有价值。
- 提升后要一雪前耻，回到原压迫点兑现。
- 这是男频爽文核心问题。
- 不懂升级爽感，就容易出现“套路都不会”的问题。
- 图片公式是欲望加阻碍加克服等于达成，克服处让读者感受到爽点。
- 克服阻碍后必须发生改变，否则提升没有意义。
- 这类基础问题靠拆书能看明白，关键是实际写作要把前后对应写清。

### 可迁移写法

升级爽感表：

```text
主角欲望：
当前阻碍：
没实力造成的具体憋屈：
提升难度：
提升方式：
回到哪个压迫点：
如何一雪前耻：
达成后的改变：
```

### 训练任务

写一个升级单元：前半段用 500 字写“没实力寸步难行”，中段写提升难度，后半段必须回到同一个阻碍点完成一雪前耻。
