## 1661：免（S）费（M）


人工查看范围：正文、3 张图片、全部 4 条评论。图片已下载查看，内容主要围绕免费网文平台、资本、稿费不可见、收费阅读与作者收益关系。无附件、无外链。

证据限制：这是行业观点截图与短讨论，不是平台合同全文；本卡只拆“免费平台下作者收益不可见、从读者付费转向资本分配”的风险意识。

### 读者承诺

这篇沉淀的是免费平台不是单纯“读者不花钱，作者照样赚钱”的中性模式，而是会改变作者收入逻辑。图片和评论共同指向：免费模式是资本游戏，可能为了抢市场、砸钱、制造赢家，但小作者能不能拿到钱，要看网站/资本是否发钱；收费网站里一个订阅就是一个订阅，收入路径有迹可循，而免费平台如果没有可查询的收入途径，就会变成平台说给多少都可以。评论进一步强调，过去作者服务好读者老爷就能吃饭，免费模式下作者更像看资本脸色，小作者没有资格和资本互相看脸色。这里的核心不是简单否定免费，而是提醒作者理解收益透明度、平台分配权和议价能力的变化。

### 核心机制

```text
收费阅读 -> 读者订阅可追踪 -> 作者服务读者换收入
免费阅读 -> 平台/广告/资本分配 -> 收入不可见 -> 作者看平台脸色
```

免费模式的关键风险不是“读者不付费”，而是作者与收入来源之间隔了一层平台分配黑箱。

### 期待链与情绪

```text
作者以为免费也能赚钱 -> 发现收入由平台发放 -> 缺少透明订阅凭据 -> 担心被克扣和被支配 -> 需要收益风险意识
```

当收入依据不可查，作者的安全感会从“读者买单”变成“平台赏饭”。

### 关键判断

- 免费平台吃的是网站给不给作者发钱。
- 收费阅读的订阅和收入相对有迹可循。
- 免费平台若没有透明收入查询，平台分配权会变大。
- 免费模式会把作者和读者的互惠关系改成作者依赖资本分配。
- 小作者在平台面前议价能力很弱。
- 扣稿费只是表层问题，真正问题是收益逻辑变化。
- 免费模式可能砸钱抢市场，但不等于长期对作者友好。
- 作者要看清合同、分成、广告收益、保底和结算口径。
- 选择免费平台前，要判断自己能否承受不可见收益风险。
- 平台模式判断属于作者权益意识的一部分。

### 可迁移写法

```text
目标平台：
收入来源：
读者付费是否可追踪：
广告/流量收益是否透明：
分成规则：
结算周期：
扣稿费/压价风险：
作者议价能力：
是否适合当前阶段：
```

### 训练任务

把收费平台和免费平台各选一个，做一张收益透明度对比表：订阅/广告、结算、分成、可查询数据、平台扣款空间、作者可申诉渠道。

## 1662：兄弟们，多个修炼体系算不算卖点...


人工查看范围：正文、全部 7 条评论。无附件、无图片、无外链。

证据限制：这是设定讨论帖，不是完整世界观方案；本卡只拆“多个修炼体系能不能成为卖点，以及如何避免混乱”的机制。

### 读者承诺

这篇沉淀的是多个修炼体系可以存在，但核心力量只能有一个，而且必须逐步引出。提问者每次毙掉开头会连世界观一起毙，因为世界观围绕金手指设计，这次已经是第四个世界观和开头，想问多个修炼体系算不算卖点。评论给出关键约束：多个体系可以，但核心力量只能有一个，比如灵气或原力，不能一股脑全放上来。另一个角度是，多修炼体系也代表多种角色性格：剑仙直来直去，道士谜语人，武夫豪爽，和尚阿弥陀佛，所以体系要匹配人物人设。提问者尝试用阶级区分体系：资源丰富者、底层拼命者、天赋异禀者因特点选择不同道路；评论提醒，除非体系之间有高低之分，否则按阶层划分会出现漏洞。最终可拆出原则：体系多不是卖点本身，卖点是它们围绕同一个核心力量，产生人物差异、资源差异和冲突差异。

### 核心机制

```text
多个修炼体系 -> 统一核心力量 -> 逐步引出 -> 绑定人物/阶层/资源差异 -> 形成冲突和选择
```

多体系不是设定堆叠，而是用不同道路制造人物和社会结构差异。

### 期待链与情绪

```text
读者先理解核心力量 -> 接触第一体系 -> 看到人物风格差异 -> 再认识第二体系 -> 期待体系碰撞和主角选择
```

如果核心力量不统一，读者会觉得世界观散；如果一次性倒出来，读者记不住。

### 关键判断

- 多个修炼体系可以写。
- 核心力量只能有一个，如灵气、原力、道源等。
- 多体系要逐步引出，不能开头一股脑放上来。
- 每个体系要匹配人物性格和群体气质。
- 剑仙、道士、武夫、和尚等体系标签应带来行为差异。
- 用阶层区分体系时，要解释资源、高低和上升通道。
- 如果体系各有所长，阶级划分要谨慎，否则会有漏洞。
- 第一卷引出几个体系，要看读者是否已吃透核心规则。
- 每次毙开头连世界观一起毙，说明设定和故事绑定过重。
- 多体系的卖点来自碰撞和选择，不来自数量。

### 可迁移写法

```text
统一核心力量：
体系一：
体系一代表人设：
体系二：
体系二代表人设：
体系间高低/各有所长：
资源分配逻辑：
主角为什么选这条路：
第一卷引出顺序：
```

### 训练任务

把所有修炼体系压成一张表：核心力量、使用方式、代表人群、性格气质、资源门槛、优劣势。删掉不能制造人物差异或冲突差异的体系。

## 1663：请问什么样的书在飞卢比较有市场...


人工查看范围：正文、全部 2 条评论。无附件、无图片、无外链。

证据限制：这是平台市场短问答，不是飞卢市场完整报告；本卡只拆“飞卢市场判断先扫销售天榜，再看脑洞大开”的方法。

### 读者承诺

这篇沉淀的是判断飞卢市场，最直接的方法不是泛问“什么有市场”，而是去飞卢销售天榜扫榜。评论给出两层答案：一是直接翻销售天榜；二是一言以蔽之，飞卢吃“脑洞大开”的小说。对作者来说，平台市场不是靠听一个抽象标签就能掌握，而是要把榜单上正在卖的书拆成题材、书名、简介、开局钩子、金手指、更新节奏、付费节点。飞卢的“脑洞”也不是随便怪，而是高概念、强标题、强爽点、快速兑现，能在短时间内让读者明白“这本有什么不一样”。

### 核心机制

```text
想判断飞卢市场 -> 扫销售天榜 -> 提取高频题材和脑洞包装 -> 拆书名简介开篇 -> 形成选题库
```

飞卢市场答案在榜单里，不在抽象口号里。

### 期待链与情绪

```text
作者看到销售天榜 -> 发现高频脑洞 -> 拆出读者第一眼爽点 -> 迁移到自己的选题
```

不扫榜就问市场，容易得到正确但不可执行的废话。

### 关键判断

- 飞卢市场判断优先看销售天榜。
- 榜单比个人印象更接近当前市场。
- 飞卢常见答案是脑洞大开。
- 脑洞要落实到标题、简介、金手指和开篇。
- 不能只看题材名，要看读者为什么点进去。
- 同一题材要看榜单重复率和变体。
- 扫榜时要记录付费节点和更新强度。
- 选题库要从正在卖的书里提炼。
- 问“什么有市场”不如问“榜上这批书共同点是什么”。
- 脑洞不是乱设定，而是能快速让读者理解的新爽点。

### 可迁移写法

```text
榜单日期：
销售天榜前二十：
高频题材：
高频金手指：
书名关键词：
简介承诺：
开篇钩子：
付费节点：
可迁移脑洞：
```

### 训练任务

扫一次飞卢销售天榜前 20，只记录书名、简介第一承诺、主角金手指、开篇事件；拆完再决定自己的题材。

## 1664：有没有在飞卢写的大佬，问个问题...


人工查看范围：正文、全部 5 条评论。无附件、无图片、无外链。

证据限制：这是旧书处理短问答，不是飞卢后台规则全文；本卡只拆“飞卢旧书太监后是否续写”的低风险判断。

### 读者承诺

这篇沉淀的是飞卢旧书断更太久、成绩一般时，通常不要纠结续写，直接开新书。提问者之前在飞卢写过两本，一本国运类同人，一本动漫同人，12 万字上架后首订接近 260，当时浮躁切了，现在想回去飞卢，问是否要把太监书完结、还能不能捡起来写。评论非常一致：在飞卢续什么，直接开新书；不用纠结以前成绩；这么久之后，在飞卢后台下续写想起来太难；260 订阅还可以，但继续捡旧书仍是浪费时间。这里的重点是平台节奏和机会成本：飞卢读者和推荐更吃新书爆发，旧书断更后冷启动困难，花时间补旧书不如用经验重开。

### 核心机制

```text
旧书断更/太监 -> 热度和推荐窗口消失 -> 续写唤醒困难 -> 机会成本高 -> 直接开新书
```

飞卢旧书处理要看平台节奏，不要被过去首订和沉没成本拖住。

### 期待链与情绪

```text
作者想弥补旧坑 -> 评论建议别续 -> 放下沉没成本 -> 用旧经验开新书
```

旧书成绩还行不等于值得续；断更后的恢复成本可能大于新书收益。

### 关键判断

- 飞卢旧书断更太久，一般直接开新书。
- 不用过度纠结以前成绩。
- 首订 260 说明有过基础，但不代表现在能唤醒。
- 飞卢后台和推荐节奏下，续写想起来很难。
- 太监旧书是否完结，不应占用新书准备时间。
- 旧书最大价值是复盘题材、开头、付费点。
- 旧书补完属于沉没成本，不能替代新项目。
- 新书更容易重新抢窗口期。
- 回飞卢前先扫榜，而不是先修旧书。
- 旧书若要处理，做最低限度收尾即可。

### 可迁移写法

```text
旧书平台：
断更时长：
上架字数：
首订/成绩：
是否还有推荐窗口：
续写唤醒难度：
旧书可复盘经验：
新书准备度：
是否直接开新书：
```

### 训练任务

给旧书做一页复盘：题材、书名、开篇、上架节点、首订、切书原因、可复用经验；复盘完直接列新书选题，不再补旧书正文。

## 1665：想问问各位，有没有了解的哪个网...


人工查看范围：正文、全部 5 条评论。无附件、无图片、无外链。

证据限制：这是投稿渠道问答短帖，不是平台征稿规则全文；本卡只拆“20 万字中篇要按渠道和流量来源匹配”的判断。

### 读者承诺

这篇沉淀的是 20 万字左右中篇小说，不一定要按长篇网文连载逻辑处理，应该按平台收稿习惯、字数要求和作者自带流量来匹配渠道。提问者问哪个网站要二十万字左右中篇。评论给出几类方向：起点短篇频道可能有责编收；晋江字数要求不高；磨铁一直收中短篇，男频女频都有，千字价较高但平台较糊，也有奖品；如果作者自己有微博、B 站等社交账号，可以引流到爱发电或冲鸭这类平台，爱发电服务费和税费相对低。这里的重点是：中篇不适合硬套百万字连载打法，要先判断自己是靠平台流量、编辑收稿、买断/千字价，还是靠自媒体引流付费。

### 核心机制

```text
20万字中篇 -> 长篇连载窗口不匹配 -> 按短篇频道/中短篇收稿/自带流量平台选择 -> 匹配收益方式
```

中篇投稿先选渠道模型，再考虑作品包装。

### 期待链与情绪

```text
作者有中篇项目 -> 查平台字数和收稿习惯 -> 选择短篇/中短篇/自媒体付费 -> 降低投稿错配
```

渠道错了，作品不是不好，而是篇幅和商业模式不匹配。

### 关键判断

- 20 万字左右属于中篇，不适合所有长篇连载平台。
- 起点短篇频道可能有责编收。
- 晋江字数要求相对不高。
- 磨铁收中短篇，男频女频都有。
- 磨铁平台声量较弱，但千字价可能较高。
- 有社交账号时，可考虑爱发电、冲鸭等自引流付费。
- 自引流平台适合有粉丝基础的作者。
- 投稿前要确认字数、题材、结算方式和版权要求。
- 中篇收益可能来自买断、千字、奖金、订阅或粉丝付费。
- 不同渠道需要不同简介和卖点包装。

### 可迁移写法

```text
作品字数：
题材类型：
是否有自带流量：
目标收益方式：
可投平台：
字数要求：
版权要求：
结算方式：
投稿包装：
```

### 训练任务

给 20 万字项目做渠道表：起点短篇、晋江、磨铁、自引流平台各列字数要求、题材适配、收益方式、版权风险，再选一个主投渠道。
