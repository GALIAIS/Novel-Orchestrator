## 211：#思考分析 为了写小说看了不少...

人工查看范围：正文、全部 11 条评论。重点查看作者从大量开头中得出的“第一章必须写卖点”，以及评论区对生死危机、不公平伤害作为新人稳妥方案的补充。

### 读者承诺

这篇的核心是把“生死危机/不公平伤害”放回辅助位置。第一章最重要的是让读者确认这本书的卖点：卖点是金手指，就写金手指；卖点是稳健流，就突出稳健；卖点是什么，就让读者先看到什么。生死危机能降低新人失败风险，但不能替代卖点本身。

### 核心机制

```text
题材卖点 -> 第一章展示 -> 读者确认值得看 -> 危机/不公平辅助拉期待 -> 进入故事
```

### 期待链与情绪

```text
读者点进开篇 -> 看到核心卖点 -> 想看主角怎么使用 -> 危机或伤害补强压力 -> 期待后续兑现
```

### 关键判断

- 读者不是为抽象危机留下，而是为有意思的卖点留下。
- 生死危机和不公平伤害是新人低风险开篇工具。
- 笔力足够时，可以用更强卖点替代固定危机模板。
- 第一章就写金手指，读者会期待主角怎么使用。
- 卖点是核心，其它都是辅助关系。
- 只写生死危机不写卖点，会让读者概念模糊。
- 新人容易因为表达目标不清，写正文时不断摇摆。
- 写打脸前要先明确想达到的效果：名声、反应、爽感还是节奏。
- 生死危机、不公平伤害能补铺垫能力不足的短板。
- 无限流等题材也可以用卖点前置，不必机械套同一种开篇。
- 判断第一章是否有效，要看读者是否知道“这本书最值得看的东西是什么”。

### 可迁移写法

第一章卖点优先表：

```text
本书最大卖点：
第一章可见画面：
金手指/能力是否出现：
危机或不公平是否必要：
读者看完想追的问题：
卖点兑现前的阻碍：
辅助压力：
下一章使用方式：
```

### 训练任务

给一个开篇重写第一章目标：第一段内写出题材，第一章内展示卖点，并说明危机/不公平只是辅助哪种期待。

## 212：#公开处刑  一直在和剑神、星...

人工查看范围：正文、全部 9 条评论、附件状态。附件 `新书16：我真的不想当渣男.docx` 仅有 file id，未提供可下载链接；本卡依据正文与评论精拆。

证据限制：附件正文不可见；评论已指出心理活动废话多、系统绑定前内容可删、2500 字没有目标女角色、系统介绍太多、第一异性不惊艳、不公平伤害没有事件化、主角遭遇不真实等关键问题。

### 读者承诺

这篇是都市日常、系统、神豪、暧昧后宫方向的开篇反复打磨。作者已经知道核心爽点是泡妹子的暧昧、日常不爽变爽、装逼和修罗场，但实际开篇仍然把大量篇幅花在男主惨、环境、心理活动和系统介绍上，导致标题吸引来的目标读者看不到目标女角色，也看不到暧昧爽点。

### 核心机制

```text
日常神豪暧昧承诺 -> 前文废话拖慢 -> 目标女角缺席 -> 情绪张力不足 -> 读者失去期待
```

### 期待链与情绪

```text
读者因渣男/神豪/暧昧点进来 -> 期待任务和女角色 -> 长段惨状与心理活动 -> 系统说明拖延 -> 第一章没有核心关系钩子
```

### 关键判断

- 都市日常主线淡，更需要密集爽点和明确标签。
- 日常、系统、神豪、暧昧不能摇摆，目标受众要早定。
- 这种一级结构的难点是人设把控。
- 男主核心设定不能崩、不能毒。
- 关键女配是否出彩会决定成绩上限。
- 纯日常暧昧比有专业优长的都市职业文更难写。
- 开头不要用大量心理活动解释男主生活多惨。
- 绑定系统可以更早出现。
- 系统介绍不要一次讲完，拆到后续使用中。
- 标题吸引的是“泡妹子/渣男/暧昧”受众，第一章不能 2500 字没有女角色目标。
- 第一位异性至少要有看点，普通清秀不够支撑承诺。
- 不公平伤害要用事件体现，不是用口水话堆背景。
- “前女友挽高帅富分手”这类事件，比一万句惨更能建立伤害。
- 主角被投诉、挨打、落魄要有可信由头。
- 长得帅是稀缺资源，写惨必须解释为什么资源失效。

### 可迁移写法

都市日常系统开篇表：

```text
标题承诺：
目标受众：
第一位女角色：
系统任务：
日常不爽事件：
不公平伤害事件化：
男主核心人设：
第一处暧昧/装逼：
系统说明延后项：
```

### 训练任务

把一个都市日常系统开篇压缩到 1500 字：前 500 字出现系统任务，前 1000 字出现目标女角色，第一章结束前完成一次小爽点。

## 213：#公开处刑 我想尝试一下类似于...

人工查看范围：正文、全部 11 条评论、附件状态。附件 `神圣起源(1).docx` 仅有 file id，未提供可下载链接；本卡依据正文与评论精拆。

证据限制：附件正文不可见；评论已指出硬汉/燕双鹰风格目标不清、卧底助手关系不合理、看不到卖点和欲望、战斗能力金手指弱、真正金手指宇宙飞船被藏、科幻背景破坏兵王代入、书名不直给等关键问题。

### 读者承诺

这篇是想写硬汉电影和燕双鹰风格，但读者没有看到持续追读理由。评论区把它拆成无敌战神文：读者要的不是“主角冷静能打”，因为这已经是很多主角默认配置；真正卖点如果是外星飞船和高科技武器，就不能藏在后面，必须在书名、第一章或第一困境中直接亮出来。

### 核心机制

```text
硬汉战神承诺 -> 能打默认化 -> 真金手指被藏 -> 开篇无卖点 -> 追读欲望弱
```

### 期待链与情绪

```text
读者看到卧底/硬汉开篇 -> 等待无敌爽点 -> 只看到普通打斗 -> 不知道飞船金手指 -> 放弃
```

### 关键判断

- 燕双鹰风格本质上接近无敌战神爽文。
- 卧底身份可以用，但人物关系要合理。
- 主角不说废话，不代表必须硬塞助手当解说员。
- 解说功能可以交给盟友、交易对象、敌人或被保护者。
- 开篇打斗很容易无聊，因为读者还不知道谁是谁。
- 战斗能力、冷静、杀伐果断不够新，很多主角默认如此。
- 真正金手指如果是外星飞船和高科技武器，就不要藏着掖着。
- 设定绕不能成为第一章不展示卖点的理由。
- 可以设计无法解决的困难，让主角用金手指解决。
- 第一章展示金手指作用，第二三章用它解决困境。
- 兵王类现代都市读者通常要代入现实场景，硬科幻背景会抬高门槛。
- 兵王类常见吸引点不只是能打，还包括妹子、身份反差、保护和打脸。
- 书名要粗暴直接告诉读者卖点。

### 可迁移写法

硬汉战神金手指直给表：

```text
硬汉卖点：
真正金手指：
第一困境：
主角默认能力：
必须展示的新能力：
解说功能承载者：
是否需要女角色钩子：
书名是否直给：
科幻设定是否抬高门槛：
```

### 训练任务

重写一个硬汉战神开篇：第一章必须让主角遇到无法靠普通战斗解决的困境，并用真正金手指制造第一次期待。

## 214：这是个公开处刑。希望大佬指点一...

人工查看范围：正文、全部 22 条评论、附件状态。附件 `开头小纲填写表格(2).docx` 尝试下载返回 401；本卡依据正文与评论精拆。

证据限制：附件正文不可见；评论已指出书名题不对意、主角行为尴尬不讨喜、系统无亮点、配角着墨太多、跳舞桥段太毒、情绪拉扯不到位、低级不公平伤害像过家家、转场和语病问题等关键问题。

### 读者承诺

这篇是“单个小剧情有爽感，但放进开篇会尴尬”的公开处刑。作者想写穿越玄幻加系统情绪值，但评论认为书名承诺和正文表现错位：读者期待“变态”卖点，实际看到的是主角脑子有问题、跳舞尴尬、冲突低级、打脸前压制不足。真正问题不是桥段不能沙雕，而是沙雕行为没有被包装成可接受的人设选择和情绪兑现。

### 核心机制

```text
沙雕/变态承诺 -> 人设包装不足 -> 低级伤害 -> 尴尬行为 -> 情绪值轻松获得 -> 爽点变毒点
```

### 期待链与情绪

```text
读者因书名点进 -> 期待离谱但好看的爽点 -> 看到主角尬舞/配角乱写 -> 压制和反应不足 -> 读者尴尬退出
```

### 关键判断

- 书名要和正文承诺一致。
- “变态”可以是卖点，但不能让读者觉得主角脑子有问题。
- 主角行为不讨喜会造成大面积点叉。
- 系统没有亮点时，老套路更难成立。
- 拆天榜不能只抄壳，必须理解爽点包装。
- 开篇配角着墨太多会稀释主角。
- 龙套不要上来介绍一堆。
- 主角尬舞前至少要给“豁出去了”的心理台阶。
- 五层之后应该有反派装逼、长老怒骂、众人嘲笑，再打脸。
- 情绪拉扯不到位，奖励就显得太轻松。
- 低级争风吃醋的不公平伤害爽感弱，像过家家。
- 可以把争风吃醋包装成原则、利用、强迫放弃等更高一层冲突。
- 对手先礼后兵，会让角色更立体，也留下洗白可能。
- 走轻松沙雕向可以，但更考验笔力和视角稳定。
- 语病和生硬视角转换会放大尴尬。

### 可迁移写法

沙雕系统开篇排雷表：

```text
书名承诺：
主角离谱行为：
行为前的合理台阶：
不公平伤害事件：
反派压制：
旁观者反应：
打脸方式：
系统奖励是否太轻：
尴尬风险：
可升级包装：
```

### 训练任务

把一个尴尬沙雕桥段重写成可接受版本：先补主角为什么必须这么做，再补反派压制和旁观者反应，最后才给系统奖励。

## 215：#思考分析 最近有了点新思考，...

人工查看范围：正文。重点查看作者从读者视角切换到作者视角，发现“高潮爽时反复研究其实是在回味”，因此改用反向阅读。

### 读者承诺

这篇的核心是拆书视角校正：读到高潮觉得爽时停下来研究，很多时候不是作者视角，而是读者在回味；真正能学到结构的地方，反而常在读者不爽、跳读、觉得慢的段落。反向阅读那些不爽处，才能看清铺垫、转场和伏笔。

### 核心机制

```text
读者视角沉浸 -> 爽点处回味 -> 学不到结构 -> 反向读不爽处 -> 看清铺垫转场伏笔
```

### 期待链与情绪

```text
读者高潮处爽 -> 想拆为什么爽 -> 只是在回味结果 -> 回头看不爽段 -> 发现前置结构 -> 建立作者视角
```

### 关键判断

- 读者视角和作者视角不是一回事。
- 读者爽的时候，容易沉迷结果。
- 反复研究高潮不一定是在拆结构，可能只是在回味。
- 下行、不爽、跳读处常藏着铺垫、转场和伏笔。
- 反向阅读可以帮助理解作者如何把读者带到高潮。
- 拆书不能只盯爽点爆发处。
- 结构学习要看爽点前的准备工作。
- 作者视角需要主动问：这一段承担什么功能。
- 读者记不住情节、拆不出结构，可能是取样位置错了。
- 不爽处不是全都没价值，有些是必要下行和铺垫。

### 可迁移写法

反向阅读拆书表：

```text
高潮爽点：
爽点前的不爽段：
不爽原因：
该段承担的功能：
铺垫信息：
转场方式：
伏笔/预告：
如果删除会少什么：
可仿写动作：
```
