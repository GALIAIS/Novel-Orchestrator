### 读者承诺

这篇记录小站作者福利变化带来的平台决策：榜单奖励缩水，作者要重新评估当前书的收益预期和下本去向。平台选择不是情绪化跑路，而是看福利、推荐、签约难度、题材适配和自己能不能拿到有效曝光。

### 核心机制


- 次元姬榜单奖励缩水，作者考虑写完当前书后转平台。
- 评论建议不行就看 sf 征文或刺猬猫。
- 刺猬猫被认为难度不高，但仍要面对签约等级、上架均订等门槛。
- 作者卡在榜单十多位，担心被宝箱怪和同人压住。
- 星主建议上推荐就行，大推可以翻倍；当前阶段尽人事好好写，不要过度焦虑。
- 另一个作者反复改开头，星主强调开头改多少次都不亏，因为短短几章决定路线、基调和大部分收入。
- 先符合开篇 18 个准则，再谈文笔和表现。

## 577：感谢宝剑老师，孩子已经签约上架...

人工查看范围：正文、全部 6 条评论。无附件，无图片。

### 读者承诺

这篇是小站签约上架后的心态校准：成绩不理想不等于这本没有价值。若平台流量小、规则特殊，当前书可以承担练笔、拿全勤、验证方向的功能，下本再用积累过的笔力和经验冲更稳的成绩。

### 核心机制

```text
签约上架 -> 数据不理想 -> 判断平台和流量背景 -> 当前书继续练手 -> 拿全勤/完结经验 -> 准备下本
```

### 期待链与情绪

```text
作者签约上架开心 -> 看到比例扑街 -> 产生失落 -> 星主指出小站练手也有价值 -> 下本更稳
```

### 关键判断

- 作者已签约上架，但比例 1/20，主观感觉扑街。
- 星主先问平台；若在起点，1:20 并不算差。
- 作者说明是在次元姬这类小站，流量以个位计数。
- 作者曾试起点，但跟不上高强度节奏，最后按能力和方向选择小站。
- 星主判断拿全勤练手也不错，下本会更稳。
- 评论还提到不同平台上架规则不同，有的 12 万字强制上架，有的可沟通再写一卷。

```text
是否能拿全勤：
本书训练目标：
下本要迁移的经验：

```

### 训练任务

## 578：既然已经发了“开头自我审核”的...

人工查看范围：正文、全部 5 条评论。无附件，无图片。

### 读者承诺

这篇规定公开处刑前的前置动作：先按开头自我审核表自查，再拿出来让别人看。社群规则不是限制讨论，而是提升学习效率；先自审能过滤明显问题，后续他人意见才能集中在盲区。

### 核心机制

```text
想求评开头 -> 先按表格自审 -> 修掉硬伤 -> 再公开处刑 -> 收集不同阅读取向 -> 迭代开篇
```

### 期待链与情绪

```text
新人急着求评 -> 规则要求先自审 -> 找到开篇硬指标 -> 公开讨论更有效 -> 稿费目标回到学习效率
```

### 关键判断

- 想发公开处刑的作者，应先按开头自我审核表审核。
- 表格可在之前文件或“参考资料”标签找。
- 规则存在的目的，是提升所有人的学习效率。
- 星主不管水平高低，只希望成员通过学习赚到更多稿费。
- 评论认为开篇 18 要素是硬指标，应尽量满足。
- 新人对自己的文有滤镜和认知盲区，发出来请星友提意见仍有必要。
- 广泛吸取意见有价值，但前提是先做基础自审。

### 可迁移写法

公开处刑前置表：

```text
开篇自审表是否填完：
已发现硬伤：
已修改内容：
仍不确定的问题：
希望别人重点看的点：
反馈来源：
下一轮修改动作：
```

### 训练任务

提交开头求评前，先写一页自审说明：你按哪个表检查过、改了什么、现在只想让别人看哪三个问题。

## 579：发现一个非常简单提升写作效率的...

人工查看范围：正文、全部 2 条评论、2 张去重图片。图片已下载并人工查看：`case-library\精拆\attachments\batch-116\case-579-image-1.jpg`、`case-library\精拆\attachments\batch-116\case-579-image-2.jpg`。

图片证据：两张图片均为稿酬收入截图，一张总收入约 25.6 万，另一张 2022 年 1 月稿酬收入约 48.7 万；评论建议用稿费图替代单纯背景图，作为更强的写作动机提示。

### 读者承诺

这篇不是作品结构，而是写作启动机制：把写作工具背景换成能激发打开欲望的图，让自己更愿意进入写作界面。评论进一步把刺激物从“好看”升级为“稿费目标”，用未来收益想象强化日常启动。

### 核心机制

```text
启动写作困难 -> 改写作环境背景 -> 增加打开欲望 -> 用稿费图强化目标 -> 降低开始成本
```

### 期待链与情绪

```text
作者不想打开软件 -> 背景图带来即时刺激 -> 更愿意点开看看 -> 稿费图提供长期奖励想象 -> 增加码字启动概率
```

### 关键判断

- 发帖者用写作工具背景图作为写作启动器。
- 背景图需要改透明度，避免影响正文阅读。
- 评论建议使用稿费图，相信自己以后也能达到。
- 每天读一篇从小白到月入过万的励志故事，也是在制造目标想象。
- 图片中的高收入截图提供具体目标，而不是抽象鸡血。
- 这个方法解决的是“打开软件和开始写”的摩擦，不替代剧情设计和笔力训练。

### 可迁移写法

写作启动环境表：

```text
当前启动阻碍：
可替换背景图：
刺激类型：
是否影响阅读：
每日打开触发器：
长期奖励图：
实际码字记录：
```

### 训练任务

设置一个写作启动器：背景图、稿费目标图或一句目标提示。连续 7 天记录它是否让你更快打开软件，不记录字数以外的空想。

## 580：疑问：在小说刚开始让主角主动向...

人工查看范围：正文、全部 4 条评论。无附件，无图片。

### 读者承诺

这篇解决“主角主动示弱会不会劝退”的关键：可以不告诉读者完整计划，但必须告诉读者主角有计划。读者可以和敌人一样不知道陷阱细节，但不能误以为主角真怂、真蠢、真被压制到没有反击。

### 核心机制

```text
强敌压制 -> 主角主动示弱 -> 明示主角有计划 -> 隐藏具体底牌 -> 敌人入套 -> 反杀兑现震惊
```

### 期待链与情绪

```text
读者看到主角示弱 -> 如果没有计划感会反感 -> 多次暗示有想法 -> 读者等待翻盘 -> 陷阱反杀形成震惊爽点
```

### 关键判断

- 主角方和敌方实力差距大，正面硬刚无胜算，示弱诱敌可以成立。
- 作者想让读者和敌人获取同样信息，后面通过陷阱反杀。
- 一般扮猪吃虎会让读者知道主角有反杀实力，因此读者不反感扮猪。
- 若完全不说明计划，读者可能把示弱理解为主角软弱或作者压主。
- 星主建议：不说真实想法可以，但一定要强调主角有计划、有想法。
- 从主角开始冒险诱敌深入，就要多次强调计划感。
- 遮掩过度，遇到一目十行读者，间贴会直接开喷。

### 可迁移写法

示弱诱敌计划感表：

```text
敌我实力差：
主角为什么必须示弱：
读者知道的计划感证据：
隐藏的具体底牌：
敌人误判点：
反杀触发条件：
示弱期间防反感提示：
```

### 训练任务

写一个主角向强敌示弱的 1000 字桥段。不能暴露完整陷阱，但至少三次暗示主角有计划，让读者等待反杀而不是讨厌主角。
