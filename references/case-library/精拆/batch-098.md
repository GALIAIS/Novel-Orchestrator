## 486：今天下午遇到这本小说，一气呵成...

人工查看范围：正文、全部 20 条评论、1 张图片。图片已下载并人工查看：`case-library\精拆\attachments\batch-098\case-486-image-1.jpg`。

图片证据：图片显示样本书《苟在游戏开服前一百年》，五万收藏、96 万字、76 月票，标签为游戏异界、轻松、搞笑、玩家、无节操，简介承诺玩家穿越到游戏《仙魔》背景世界、带着傀儡妹子苟全乱世。

### 读者承诺

这篇是毒点反面教材：一本书文风、主角人设、金手指、期待感都不错，却因为持续压主角、让天才配角抢风头、主角伏低做小舔配角、最后机缘还被配角拿走，直接劝退读者。主角可以当小弟、做谋士、暂时伏低，但前提是读者能看到主角的位格、收益、回馈和主动权。

### 核心机制

```text
优秀开局条件 -> 设定主角资质差 -> 强配角压风头 -> 主角伏低做小 -> 主角机缘被抢 -> 爽感断裂 -> 读者弃读
```

### 期待链与情绪

```text
读者被幽默文风和金手指吸引 -> 期待主角低调但有掌控感 -> 配角持续压主角 -> 主角资源供养配角却无回馈 -> 机缘被抢 -> 憋屈转毒点
```

### 关键判断

- 样本书有优秀项：文风幽默、主角讨喜、金手指有潜力、期待铺垫好。
- 劝退点不是单一设定，而是剧情节奏持续压主角。
- 主角资质差可以写，但不能同时安排资质优秀配角长期压风头。
- 主角主动舔配角、伏低做小，本身就容易消耗爽感。
- 更大的毒点是主角出去一趟，回来发现自己家的机缘被配角拿了。
- 评论延伸到女主被揩油、牛头人等边界：读者不爽点往往来自核心权益被别人侵犯。
- 稳健类主角可以给更高位存在当臣子，但要先把主角的谋士位格和对方明主位格立住。
- 主角当小弟不憋屈的关键是：主公足够强、足够尊重主角、给主角足够回馈。
- 本例中配角原地位不如主角，资源还由主角提供，主角却趴到地底下抱大腿，反差就成了憋屈。
- 配角拿了资源没有反哺主角，主角像工具人，这会直接破坏代入。

### 可迁移写法

压主角毒点自检表：

```text
主角当前位格：
配角当前位格：
主角是否主动伏低：
伏低理由：
配角是否值得追随：
主角付出资源：
配角回馈主角：
主角是否保留主动权：
是否发生机缘被抢：
读者感到低调还是憋屈：
```

### 训练任务

写一个“主角暂时给强者当谋士”的桥段。必须写清强者凭什么值得主角辅佐、主角保留什么主动权、这次伏低换回什么收益。禁止让主角资源被拿走却没有回馈。

## 487：首先，打不过就去逞英雄，这不是...

人工查看范围：正文、全部 6 条评论、3 张图片。图片已下载并人工查看：`case-library\精拆\attachments\batch-098\case-487-image-1.jpg`、`case-library\精拆\attachments\batch-098\case-487-image-2.jpg`、`case-library\精拆\attachments\batch-098\case-487-image-3.jpg`。

图片证据：图 1 讨论主角见义勇为受伤是否毒点，争议点在“见义勇为”与“受伤吃亏”的区分；图 2 指出被欺负者最好是主角寻找的关键人物、亲人或重要角色，否则无缘无故打人不推动故事；图 3 强调不是不能写正义，而是要让大部分读者感同身受，普通人多半会犹豫挣扎后因不得不做才出手。

### 读者承诺

这篇拆的是见义勇为桥段的爽文底线：见义勇为本身不是毒点，打不过硬上、受伤吃亏、没有必要赌注、没有剧情收益才是毒点。爽文可以让主角做现实里读者不敢做的好事，但要让主角聪明、潇洒、全身而退，或者至少因救人获得超额收获。

### 核心机制

```text
主角见到不平事 -> 建立必要赌注/关系 -> 主角用能力或计谋介入 -> 不吃亏救人 -> 获得线索/人情/资源 -> 推动后续剧情
```

### 期待链与情绪

```text
读者希望主角有善心 -> 又害怕主角降智吃亏 -> 主角聪明破局 -> 被救者回馈剧情 -> 好人有好报 -> 读者获得现实补偿爽感
```

### 关键判断

- 原帖质疑“打不过还去逞英雄”是毒点，并追问这段剧情有什么用。
- 只为了体现主角善心而牺牲剧情推进，会显得别扭。
- 评论指出新手容易做伟光正人设，然后硬塞剧情证明主角是英雄好汉。
- 更正确的做法是让人设服务剧情，而不是让剧情单纯证明人设。
- 见义勇为不是毒点，受伤且显得蠢才是毒点。
- 如果主角打不过，可以用计谋救人并全身而退。
- 被救的人应引发后续剧情或给主角好处，形成超额收获。
- 爽文底线是好人有好报、恶人有恶报。
- 现实中见义勇为已经可能受伤吃亏，读者不想在小说里再经历一次。
- 图中强调必须有前置条件：被救者是关键人物、亲人、前女友或后续事件节点。
- 另一个关键是“必要之赌注”：主角为什么不得不管，读者为什么接受他管。

### 可迁移写法

见义勇为爽文表：

```text
不平事件：
被救者身份：
和主角的关系/价值：
主角为什么必须出手：
主角当前能力：
敌方风险：
计谋/能力破局：
主角是否吃亏：
被救者回馈：
后续剧情钩子：
```

### 训练任务

设计一个见义勇为桥段。必须写出“不得不管”的原因、主角不吃亏的解决方式、被救者带来的后续收益。若主角受伤，必须让伤变成更大的线索或收获。

## 488：宝剑大大，我之前在番茄有一本签...

人工查看范围：正文、全部 6 条评论。无附件，无图片。

### 读者承诺

这篇回答章节规划和三幕式的尺度问题：三幕式通常用于 6-15 万字一卷，不是机械套到每一章；章节不必每章都定死三幕和爽点，而应服从整体情节连贯。一小段剧情结束就断章，有存稿再结合推荐、活动、上架节奏卡爽点爆发。

### 核心机制

```text
大纲/卷纲明确 -> 到章节迷惑 -> 回看系统教程 -> 卷级三幕安排主波动 -> 小段剧情自然断章 -> 爽点一浪接一浪
```

### 期待链与情绪

```text
作者知道卷纲但不会落章 -> 误以为每章都要三幕/爽点 -> 星主拉回教程体系 -> 章节服从情节连贯 -> 在卷级波动中不断上行
```

### 关键判断

- 作者有 40 万字签约经验，但章节层面仍迷惑。
- 问题包括：每章是否都要三幕式，一个三幕式写几章，每章是否都要爽点，一卷中爽点是否越多越好。
- 星主判断这些问题教程已有答案，说明作者还没系统学习。
- 三幕式通常用在 6-15 万字一卷，20 万字以上也可以，看剧情安排。
- 每章规划不用定太死，要跟整体情节走。
- 一小段剧情结束时断章，跟固定章节字数关系不大。
- 有存稿时可以结合推荐、py 推书、网站活动、上架节点规划发章。
- 上架时可卡爽点爆发，但不能牺牲剧情连贯。
- 卷纲写作中也会频繁修改。
- 作者回看《曹操魂穿武大郎》后领悟：情节一浪接一浪，在三幕式波动中不断上行，才能粘住读者。

### 可迁移写法

章节落点规划表：

```text
本卷三幕阶段：
当前小段剧情：
本章承接：
本章推进：
本章结尾期待：
是否自然断章：
是否服务卷级上行：
近期爽点爆发点：
是否为推荐/上架卡节奏：
卷纲需修改处：
```

### 训练任务

把一个 10 万字卷纲切成 12 个小段剧情。每段只写：目标、阻碍、转折、收获、断章点。不要先按“每章必须爽”切，先按剧情连贯切。

## 490：今天被读者说剧情无聊，看了开头...

人工查看范围：正文、全部 12 条评论。无附件，无图片。

### 读者承诺

这篇处理“打脸虐渣写得无聊、看开头知结尾”的问题。常规套路不是不能写，但读者太熟会提前猜到结局。解决方法只有两类：要么用伏笔支撑非常规反转，虚晃一枪打破预期；要么快速略过俗套过程，把重点放在这段剧情引发的新期待上。

### 核心机制

```text
常规打脸虐渣 -> 读者预判结尾 -> 选择虚晃反转/快进略过 -> 新期待接上 -> 避免无聊
```

### 期待链与情绪

```text
读者看到炮灰围观主角配置 -> 立刻知道打脸结果 -> 情绪没被调动 -> 反转有伏笔则惊喜 -> 快进后接新目标则不拖沓
```

### 关键判断

- 作者按打脸虐渣写，配置有炮灰、围观群众、主角，但读者觉得无聊。
- 评论指出可能是套路太常规，读者过于熟悉。
- 也可能是情绪没调动起来，读者压根在情节外边。
- 星主在策划时能感到哪段会“看了开头就知道结尾”。
- 解决方案一：搞非常规反转，打破读者预期，也就是虚晃一枪。
- 解决方案二：迅速略过，用最短过程写完俗套剧情，把重点放在其引发的新期待上，也就是快进。
- 大高潮最好有惊人意外，否则看头弱。
- 反转必须有明确伏笔，不能临时花活。
- 伏笔太深、读者半懂不懂、后续不给说明，会像跳了关键章节。
- 伏笔说太多会被喷当读者弱智，但被喷伏笔多通常好过被喷突兀。
- 没有阻碍，故事不成立；读者只靠作者给的信息就猜到结果，说明情节没有足够转折。

### 可迁移写法

俗套桥段处理表：

```text
桥段类型：
读者会猜到的结尾：
是否必须写完整：
可用伏笔：
非常规反转：
是否会太绕：
可快进部分：
快进后新期待：
阻碍是否成立：
情绪是否被调动：
```

### 训练任务

选一个打脸虐渣桥段，写两个版本：一版虚晃反转，必须提前埋 2 个伏笔；一版快进处理，300 字内结束打脸，并在结尾抛出新期待。
