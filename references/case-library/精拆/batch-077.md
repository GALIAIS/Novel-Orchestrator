## 381：星球里的同人大佬们，我写同人的...

人工查看范围：正文、全部 6 条评论。无附件，无图片。

### 读者承诺

这篇讨论同人文补充原作设定的边界。结论不是“同人读者一定爱看设定补完”，而是设定必须服务剧情、爽点和故事推动；只要不 OOC 可以补，但单独塞大量设定就像干讲大道理，作者觉得有趣，读者可能觉得水。

### 核心机制

```text
同人原作设定 -> 作者想补完 -> 读者期待剧情/爽点 -> 设定进入事件或战斗 -> 读者通过故事理解设定
```

### 期待链与情绪

```text
读者因原作进入 -> 期待熟悉角色和新故事 -> 大段设定打断期待 -> 设定化成行动差异 -> 重新服务爽点
```

### 关键判断

- 同人补设定的底线是不能 OOC。
- 作者自己说“很多设定”，说明已经有协调不过来的风险。
- 单插设定像干讲大道理，讲的人觉得有趣，听的人未必愿意听。
- 正常网文应由剧情、爽点、故事推动，设定为这些服务。
- 设定不应自己成为推动主体。
- 读者可能因为大段设定觉得水，进而影响订阅。
- 如果有职业体系差异，不要直接抛设定，最好放进一场战斗展示差异。
- 用故事透露设定，比说明书式补完更稳。
- 同人读者也有偏好差异，不能把“动漫同好应该不在意”当作安全假设。

### 可迁移写法

同人设定补完检查表：

```text
原作设定：
我要补充的设定：
是否会导致 OOC：
设定服务的剧情：
设定服务的爽点：
设定是否可放进战斗/选择/冲突：
读者若跳过设定是否影响理解：
是否显得水字数：
更好的故事化呈现：
```

### 训练任务

选一个想补完的原作设定，不允许用说明书段落解释。把它改写成一场战斗、一段冲突或一次角色选择，让读者从结果里看出设定差异。

## 382：我想问问，后面两个视频在哪里看...

人工查看范围：正文、全部 2 条评论、1 张图片。图片已下载并人工查看：`case-library\精拆\attachments\batch-077-case-382\image-1.jpg`。

图片证据：图片为“人设系列内容梗概”，列出 1 网文人设基本概述已发布，2 如何有目的创建配角本期，3 进一步创建立体生动的主要角色，4 如何利用人设让剧情更精彩，后续依评论反馈待定。

### 读者承诺

这篇本身是催更，但图片暴露出一个教程产品的内容结构：先讲网文人设基本概述，再讲有目的创建配角，然后进入立体主要角色和用人设推动剧情。它可迁移为“人设学习路径”：先定义人设功能，再分别处理配角、主角和剧情联动，不要一上来只堆性格标签。

### 核心机制

```text
人设基本概述 -> 有目的创建配角 -> 立体主要角色 -> 人设驱动剧情 -> 根据反馈补齐
```

### 期待链与情绪

```text
学习者看到系列目录 -> 知道当前学到哪一步 -> 等待后续补齐主角和剧情联动 -> 形成持续学习期待
```

### 关键判断

- 人设系列不是孤立讲人物卡，而是从概述到配角、主角、剧情应用逐步推进。
- “有目的创建配角”说明配角不是随机补人，而要服务主角、冲突、信息或情绪。
- “立体生动的主要角色”应在配角之后展开，避免主角只有标签没有层次。
- “利用人设让剧情更精彩”说明人设最终要回到事件和剧情变化。
- 后续依评论反馈待定，说明教程体系也需要根据学习者需求补齐。
- 评论确认系列视频在日程上，会挨个补齐，体现内容承诺需要兑现。
- 对写作系统而言，目录本身就是拆解路径：概念、配角、主角、剧情应用。

### 可迁移写法

人设学习路径表：

```text
人设基本功能：
配角创建目的：
配角服务谁：
主角立体维度：
主角行动证明：
人设如何制造冲突：
人设如何推动剧情：
当前缺失的一课：
读者/学习者反馈：
下一步补齐内容：
```

### 训练任务

给一个故事列人设四步：先写人设基本承诺，再写三个有目的配角，再写主角的三层立体维度，最后写一个只有这些人设碰撞才会发生的剧情事件。

## 383：卖点缺失、卖点转移+，这是一个...

人工查看范围：知识库正文、外链文章可见正文。无评论，无附件，无图片。

外链来源：`https://mp.weixin.qq.com/s/JBrWXky9ivUirojSLsgLqQ`

### 读者承诺

这篇讨论中后期写崩的常见原因：主线大方向不稳、吊点结束后没有新吊点、人物关系断裂、卖点缺失或转移、目标缺失、升级体系和地图体系没有提前规划。长篇要把开篇噱头高开写法和升级体系写长写法结合起来，不能只靠一个噱头跑完整本。

### 核心机制

```text
开篇噱头吸引 -> 明确吊点与主线 -> 升级/地图体系续航 -> 小高潮/大高潮节奏 -> 新期待接续旧期待
```

### 期待链与情绪

```text
读者被开篇卖点吸引 -> 追着吊点验证 -> 吊点兑现后需要新期待 -> 地图/等级/关系继续展开 -> 中后期不掉追读
```

### 关键判断

- 故事构思完成后，要先确立中后期大框架。
- 吊点是读者继续看的核心理由，例如父子相认、僚机身份、追上校花等。
- 吊点结束却没有新期待，文基本就进入结束或流失区。
- 主角行事不能没有目标，人物关系链不能崩裂，卖点核心不要轻易更换。
- 每一章都应尽量按第一章强度去写，心态疲惫也会导致质量下滑。
- 人际关系断裂是常见崩法，换地图后要想办法维持熟悉人物或势力。
- 宅猪“莲花地图法”可理解为建立基地，维持主角人际关系。
- 卖点缺失常见于恐怖文主角变强后不再怕鬼，惊悚感下降。
- 前后风格骤变会带来追订风险。
- 目标缺失就是主线偏移，本该去 A 地，却停在 B 地水文。
- 脑洞向高开题材特别容易中后期崩，因为新奇点写完后没有新抓点。
- 传统玄幻/修真升级体系和地图体系，是长篇续航的金模式，可迁移到都市文娱、科技等类型。
- 每个地图要预估字数、副本结束等级、下一地图等级，避免噱头写完无从展开。
- 毒点、力量体系突然更换、过度听单个读者建议，都可能导致追订崩盘。
- 学习长篇不能只看开篇，也要分析优秀作品中后期的期待线安排。

### 可迁移写法

长篇卖点续航表：

```text
开篇噱头：
核心卖点：
第一吊点：
第一吊点预计兑现位置：
兑现后的新吊点：
主线大方向：
升级体系：
地图体系：
每卷小高潮：
每三万字大高潮：
人际关系维持方式：
是否存在卖点转移：
是否存在目标缺失：
读者建议是否会冲突已有设定：
```

### 训练任务

给一个高开脑洞题材设计 30 万字续航。必须写出开篇噱头、前三个吊点、三个地图或阶段、每阶段等级/地位变化，以及吊点兑现后的下一期待。

## 384：写了本小说，主角穿成了夏洛克+...

人工查看范围：正文、全部 1 条评论。无附件，无图片。

### 读者承诺

这篇讨论主角穿成夏洛克但被读者评价“不够聪明”。宝剑的处理很务实：已经写到 11 万字，就让数据、章节评和读者反馈直接告诉你哪里有问题；既然反馈集中在聪明程度，下次就把聪明写出来。反馈不是否定作者，而是连载修正方向。

### 核心机制

```text
借用高智人设 -> 读者带原主期待 -> 正文表现不足 -> 章节评反馈 -> 下一轮写法修正
```

### 期待链与情绪

```text
读者看到夏洛克 -> 期待推理和高智表现 -> 剧情简单/铺世界观导致落差 -> 反馈主角不够聪明 -> 作者补高智场景
```

### 关键判断

- 借用夏洛克这类强标签人物，会自动带来读者对聪明、推理、洞察的期待。
- 神秘学背景和世界观铺设不能替代主角能力表现。
- 11 万字、1900 收藏已经有足够连载反馈，不必只靠作者自我判断。
- 数据和章节评能直接指向问题位置。
- 读者说不够聪明，意味着人设承诺和正文表现不匹配。
- “下次写聪明点”不是空话，而是要把聪明落实为判断、推理、布局、信息差或解决问题。
- 年龄层信息有参考价值，但不能用来抵消核心反馈。
- 反馈的意义是帮助作者下一轮写得更准确，而不是纠结“我这样有没有问题”。

### 可迁移写法

高智人设反馈表：

```text
借用/承诺的人设：
读者预期能力：
当前正文表现：
章节评集中反馈：
数据异常位置：
缺失的聪明场景：
下一章补法：
是推理、布局、洞察还是信息差：
补完后如何验证反馈变化：
```

### 训练任务

为一个“高智主角”补三种聪明表现：一次快速洞察、一次提前布局、一次用信息差破局。每次都写明读者为什么能看出他聪明。

## 385：写大纲的时候怎么不让战斗力崩坏...

人工查看范围：正文、全部 2 条评论。无附件，无图片。

### 读者承诺

这篇讨论长篇升级文战力崩坏的预防。方法不是只列主角等级，而是给每一卷规定等级区间和最终高潮战斗的合理上限；卷内所有战斗不能比卷高潮更离谱。若主角越级击杀，也要通过后续同阶强敌或特殊条件，证明不是体系崩，而是挂太强或对手太弱。

### 核心机制

```text
总等级体系 -> 分卷等级区间 -> 卷高潮战力上限 -> 卷内战斗不越界 -> 异常战绩后续校准
```

### 期待链与情绪

```text
读者跟随升级 -> 期待强敌逐步升级 -> 越级胜利带来爽点 -> 后续同阶苦战校准尺度 -> 战力体系恢复可信
```

### 关键判断

- 大纲阶段可以先划分最高 10 个大等级、100 个小等级等总框架。
- 每一卷设置对应等级区间，如第一卷 1-20，第二卷 20-30。
- 这种区间能避免长期写作时主角成长过快或对手乱跳。
- 除主角等级，还要考虑实际战力。
- 挂、异火、装备、血脉等会让名义等级和实际战力不同。
- 《斗破苍穹》里萧炎斗王击杀斗宗云山，越两个大阶显得离谱。
- 后面安排斗皇萧炎苦战斗皇凤清儿，可以校准读者认知：不是战力全崩，而是异火太强、云山太弱。
- 宝剑补充：可以用每卷最终高潮战斗的战力合理性作为框架。
- 卷内战斗不能比该卷高潮更离谱。
- 异常战绩必须有解释或补救，否则读者会认为体系失控。

### 可迁移写法

战力框架表：

```text
总大等级：
总小等级：
第一卷等级区间：
第一卷高潮战斗：
高潮战力合理性：
卷内最高战斗尺度：
主角实际战力加成：
对手实际弱点：
是否出现越级击杀：
后续校准战斗：
读者如何确认体系未崩：
```

### 训练任务

给三卷升级文设计战力表。每卷写等级区间、最终高潮敌人、卷内最高战斗尺度、一次越级爽点和一次校准战斗，确保卷内没有比高潮更离谱的战绩。
