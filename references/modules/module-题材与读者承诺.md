# Module：题材与读者承诺

## 使用场景

- 不知道写什么题材。
- 书名简介没有吸引力。
- 开篇写了很多设定，但读者不知道看点。
- 主角目标明确，但读者不关心。
- 想判断一个点子是否适合某个平台或某类读者。

## 输入

- 题材或灵感一句话。
- 目标平台。
- 主角身份和金手指。
- 预期读者体验。
- 同题材样本。

## 分析步骤

1. 定义读者承诺：这本书稳定提供什么情绪价值。
2. 定义主角起点：读者为什么愿意看他改变。
3. 定义核心变化：力量、身份、关系、资源、认知、地位如何变。
4. 定义题材必需元素：同题材读者默认想看什么。
5. 定义差异化表达：不是换皮设定，而是同一乐趣的新触发方式。
6. 定义第一单元承诺：开篇必须先兑现哪一小块体验。

## 输出：读者承诺卡

```text
作品名：
题材：
目标读者：
一句话承诺：
核心乐趣：
主角起点：
核心变化：
金手指功能：
第一单元要让读者期待什么：
长线期待 1：
长线期待 2：
不能踩的题材毒点：
同题材样本：
```

## 失败信号

- 只能说“主角要变强”，说不出变强后有什么具体好处。
- 只能说“有系统”，说不出系统如何制造事件。
- 书名像一个题材，简介像另一个题材，正文又是第三种题材。
- 开篇没有题材对应的核心乐趣。
- 主角困境和目标都成立，但读者没有代入收益。
