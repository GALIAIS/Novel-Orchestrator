# Module：开篇与钩子

## 使用场景

- 第一章或第一单元不吸引人。
- 开篇信息很多但不成故事。
- 主角、身份、目标、金手指交代不清。
- 第一章末尾没有继续阅读理由。
- 想复现高分开篇的钩子布局。

## 核心判断

开篇不是把所有设定讲清楚，而是让读者尽快明白：

```text
我在看哪类书 -> 主角处在什么局面 -> 他会怎样改变 -> 我为什么想看这个改变
```

## 开篇六要素

1. 主角是谁？
2. 主角身份是什么？
3. 主角目前在干什么？
4. 主角需要做什么，或有什么困境？
5. 主角社交关系怎么样？
6. 主角金手指是什么？

这六项不是填空表，而是诊断用。如果某项缺失但读者仍然被强期待拉住，可以暂不补。若读者看不懂、没期待、没代入，就回到六要素定位缺口。

## 钩子类型

- 反常行为钩子：主角做出不符合常规预期的行为。
- 危机钩子：主角即将失去生命、身份、资源、关系或机会。
- 金手指钩子：能力机制清楚，并能立刻制造事件。
- 信息差钩子：读者知道主角有底牌，角色不知道。
- 关系钩子：前任、亲友、仇人、上级、同伴即将重新评价主角。
- 世界钩子：规则、禁忌、秘密、组织露出更大空间。

## 分析步骤

1. 划定第一个单元故事范围。
2. 标出六要素。
3. 标出第一章末尾的短期期待。
4. 标出两万字内的两个长线期待。
5. 检查钩子是否与题材承诺同向。
6. 检查开篇信息是否在行动中释放。
7. 输出删、补、换、提前四类修改。

## 输出：开篇诊断报告

```text
范围：
题材承诺：
第一章短期期待：
长线期待 1：
长线期待 2：
六要素缺口：
钩子类型：
最大问题：
必须保留：
必须删除：
必须提前：
必须补强：
下一版第一章目标：
```

## 常见修复

- 设定太多：改成主角行动中遇到规则。
- 冲突太早但没意义：先铺出得失和关系。
- 金手指不清：先展示一次最小用法。
- 危机开篇太长：拆成阶段性伪胜利和新危机。
- 倒叙不稳：尽快回到主时间线，或让倒叙本身产生强期待。
