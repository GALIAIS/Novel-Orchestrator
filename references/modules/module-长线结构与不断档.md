# Module：长线结构与不断档

## 使用场景

- 一个单元结束后不知道写什么。
- 高潮后追读下降。
- 换地图后爽点来源消失。
- 支线很多但不回主线。
- 长线谜团、升级、关系推进失控。

## 基础原则

读者不是因为已经爽到而继续看，而是因为相信继续看还能爽到、惊到、甜到、燃到、怕到或得到答案。

所以长线结构的核心不是“安排更多剧情”，而是管理期待库存。

## 期待库存表

```text
当前短期期待：
长线期待 1：
长线期待 2：
正在铺垫的新期待：
即将兑现的期待：
延迟满足的期待：
已经兑现但可延伸的期待：
风险：哪里可能断期待？
```

## 三板斧

1. 规划：提前知道当前单元之后接什么。
2. 延伸：当前事件的收获、敌人、关系、秘密能延伸出新问题。
3. 系扣：在旧期待兑现前，把新期待扣上。

## 循环衔接

```text
旧目标接近兑现 -> 新线索提前出现 -> 旧目标兑现 -> 余波扩大 -> 新目标成为当前目标
```

不要等旧目标彻底结束后再从零找新目标。

## 嵌套结构

一个短线事件可以嵌在中线目标中，中线目标嵌在长线承诺中。

例：

```text
短线：解决本次规则怪谈副本
中线：找出系统为何选择主角
长线：规则世界背后的真实秩序
```

短线负责即时追读，中线负责阶段期待，长线负责全书牵引。

## 人际关系网

爽点需要见证体系。换地图时尤其要处理：

- 是否带走部分旧角色？
- 是否提前引入新地图角色？
- 是否让新地图的人听过主角传说？
- 是否保留远程关系网、组织、聊天群、榜单、舆论或敌对势力？
- 是否让旧关系变化成为新期待？

## 支线源流

支线不能只是旁逸斜出。有效支线应至少满足一项：

- 提供资源。
- 提供敌人。
- 提供关系变化。
- 提供世界秘密。
- 提供主角成长。
- 提供主线解决方案。
- 提供长线期待。

```text
旧地图核心爽点：
旧人际网：
必须保留的关系：
新地图提前铺垫：
新地图第一目标：
新地图见证体系：

补救扣子：
```
