# Module：单元故事起承转合

## 使用场景

- 大纲有，但章节写散。
- 事件没有因果。
- 主角被剧情推着走。
- 转折生硬。
- 合阶段只有解决，没有收获和余波。

## 核心模型

```text
起：制造期待
承：接近期待
转：放大期待
合：揭晓期待
扣：接上下一个期待
```

## 单元故事卡

```text
单元名称：
章节范围：
读者承诺：
主角初始状态：
起：主角将得到/失去什么？
承：主角如何选择和执行？
转：出现什么意外变化？
合：期待如何兑现？
主角收获：
他人反应：
遗留问题：
下个目标：
```

## 起

起不是背景介绍，而是得失启动。

有效的起应回答：

- 主角为什么必须动？
- 不动会失去什么？
- 动了可能得到什么？
- 读者为什么想看这个结果？

## 承

承不是拖字数，而是让主角接近期待。

承阶段应包含：

- 主角判断。
- 主角选择。
- 主角执行。
- 阻碍升级。
- 能力、信息、关系或规则逐步露出。

## 转

转不是随便反转，而是让期待变强。

有效转折：

- 新信息改变读者理解。
- 新阻碍提高代价。
- 旧铺垫突然有用。
- 反派行动超出预期。
- 主角选择产生意外后果。

## 合

合不是“问题解决”四个字，而是完整兑现：

- 目标结果。
- 铺垫回收。
- 情绪释放。
- 资源或关系收获。
- 他人反应。
- 新目标露出。

## 主角功能检查

如果主角像旁观者，检查：

- 关键转折是不是主角推动的？
- 主角是否用自己的能力、信息、身份或选择解决问题？
- 主角是否承担代价？
- 事件结束后，主角状态是否改变？
