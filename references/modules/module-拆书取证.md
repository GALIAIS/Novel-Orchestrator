# Module：拆书取证

## 使用场景

- 想分析一本书为什么好看。
- 想学习同题材开篇。
- 想找期待感、爽点、起承转合、长线衔接案例。
- AI拆书结果太散。
- 想完整精拆一本书，要求章、单元、卷、全书都有覆盖。
- 想拆轻小说的角色属性、对白、插画节点、卷尾钩子。
- 想拆漫画的分镜、页翻点、视觉钩子、格子节奏。
- 需要把已有拆书方法纳入系统。

## 定位

拆书是取证模块，不是总控入口。拆书前必须先写问题或拆书目标。

错误做法：

```text
泛泛总结一下这本书为什么好看。
```

正确做法：

```text
我要知道这本书第一单元如何建立两长一短。
我要知道它的规则怪谈副本如何起承转合。
我要知道它如何在高潮前提前铺下一个目标。
我要完整拆这本书，重点看章节期待链、卷级结构和可迁移写法。
```

完整拆书不是禁用项，但不能只给总评。完整拆书必须产出覆盖台账和分层文件。

## 拆书层级

1. 快速扫读：确定题材承诺、主角、金手指、核心循环。
2. 单元拆解：按事件闭环拆起承转合。
3. 期待拆解：拆期待形成、压制、兑现、新期待。
4. 情绪拆解：拆交锋、反应、见证。
5. 长线拆解：拆循环、嵌套、换目标、换地图。
6. 可迁移拆解：提炼能用于自己作品的结构功能。

## 全书精拆模式

用户给出整本书或本地 txt/md 路径时，优先运行 `scripts/scaffold_full_book_deconstruction.py <book-file>` 生成文件夹，不在聊天里塞完所有内容。docx 先转文本或按文档工具读取后再建索引。

文件夹结构必须按大类建独立文件夹；每个大类文件夹内再放细拆文件。不得把设定、角色、大纲、情绪、伏笔等内容混写在一个总表里。

生成的 md 内容格式必须统一：用 `# / ## / ###` 分层，用两列表格 `字段 / 内容` 承载待填写项；已有总表继续用多列表格。禁止把大量字段写成裸冒号列表。

执行顺序：

1. 读取并识别章节边界；识别失败时按字数切块，但在说明里标记。
2. 建立 `01-章节索引/`：卷、章号、标题、字数、起止位置、覆盖台账。
3. 粗扫全书，只定题材承诺、主角路线、核心循环、主要矛盾，写入 `02-全书总览/`。
4. 建立十三个专项文件夹：题材承诺、世界观设定、力量金手指、大纲主线、卷章结构、角色拆解、关系势力、情节冲突、期待情绪、伏笔信息、叙事技法、商业包装、迁移训练。
5. 逐卷拆，不跳章；每章至少填一行章节功能表。
6. 每 5-15 章归并成一个单元，写起承转合和期待接力。
7. 每卷写卷级复盘，确认高潮兑现和下一卷接档。
8. 每个角色必须先归入 `08-角色拆解/01-主要角色/`、`02-重要配角/`、`03-反派角色/` 或 `04-功能角色/`，再为该角色单独写一个 md。
9. 设定、势力、事件、伏笔、期待、爽点、场景等也按条目独立建 md；总表只做索引，不替代精拆。
10. 轻小说或漫画样本必须补 `18-媒介专项/`；纯文字小说可保留为空模板。
11. 总报告只能引用前面文件里的证据，不允许凭印象重写结论。

## 全书细分维度

题材承诺：

```text
目标读者：题材读者是谁，来买什么情绪、补偿、身份、关系或答案。
核心卖点：一句话承诺、核心循环、差异化钩子、同题材参照。
读者禁区：题材毒点、平台雷区、读者不能接受的慢热/降智/背刺。
开篇兑现：书名简介承诺是否在第一章/第一单元被证明。
```

世界观设定：

```text
基础规则：历史背景、地理空间、时代技术、社会秩序、禁忌常识。
设定入口：读者第一眼如何理解这个世界，主角如何被卷入。
设定用途：每条设定解决什么剧情问题，制造什么期待。
揭示顺序：第几章揭示，是否立刻进入行动或爽点。
```

力量金手指：

```text
力量来源：等级、晋升条件、消耗代价、限制、克制关系。
金手指规则：获得方式、可用范围、升级条件、奖励、滥用边界。
展示场景：首次证明、阶段扩容、关键破局、卷级高潮。
主线绑定：金手指如何持续制造事件，而不是只当外挂说明。
```

大纲主线：

```text
全书主线：终局想象、主要矛盾、主角最终变化、长期反派/长期问题。
阶段主线：每阶段目标、资源变化、敌我格局变化、主角状态变化。
支线源流：支线从哪里来，如何流回主线，何时删除或合并。
防烂尾：可删支线、可扩支线、终局回收池。
```

卷章结构：

```text
卷级结构：每卷承诺、核心矛盾、阶段目标、高潮、卷尾收获、下一卷接档。
单元结构：5-15 章一个事件闭环，记录起承转合、得失、反转、扣子。
章节功能：开钩、加压、解释规则、兑现、铺垫、回收、转场、反转、接档。
节奏曲线：高压/低压、解释/行动、爽点密度、高潮间距、断档点。
```

角色拆解：

```text
输出方式：每个角色一个独立 md，按主要角色/重要配角/反派角色/功能角色分文件夹；角色总表只放索引。
基础档案：姓名称号、首次登场、退场/当前状态、出场章节清单、身份定位、社会身份、阵营势力、职业能力标签、读者第一印象。
外貌识别：年龄阶段、性别称谓、体貌特征、服装道具、气质关键词、标志性动作、标志性语言、外貌变化、他人评价、镜头呈现。
性格心理：显性性格、隐性性格、核心欲望、核心恐惧、价值观、底线、软肋、执念、自我认知、外界误判、性格矛盾、压力下反应。
成长变化：初始状态、阶段目标、关键选择、成长节点、性格变化、能力变化、地位变化、关系变化、代价、最终状态、变化可信度。
剧情功能：主线功能、单元功能、期待功能、冲突功能、伏笔功能、见证功能、情绪功能、爽点功能、不可替代性、可合并风险。
关系网络：与主角/主要角色/配角/反派关系、推进节点、反转节点、利益绑定、情感绑定、冲突绑定。
表演证据：代表行动、代表台词、代表选择、代表失误、代表高光、他人反应、读者期待、兑现/反转。
写作拆解：登场设计、立住人物的第一场戏、持续记忆点、信息揭示顺序、人物弧线写法、可迁移写法、不可迁移原因、仿写训练。
```

条目精拆最低要求：

```text
设定：规则内容、边界、例外、代价、违反后果、揭示顺序、使用场景、相关角色、相关事件、相关伏笔、可删性。
力量：来源、等级、晋升、消耗、限制、克制、训练门槛、失败惩罚、展示顺序、首次爽点、阶段扩容、关键破局、反噬代价。
大纲：终局、阶段目标、敌我格局、资源变化、未回收问题、期待库存、支线来源、回流方式、删除代价、回收位置。
卷章：卷级承诺、单元承诺、起因、起承转合、代价新麻烦、关系变化、章节功能、新增信息、伏笔回收、章尾钩子。
关系势力：权力变化、情感变化、利益变化、断裂修复节点、内部层级、关键成员、资源清单、弱点、覆灭/升级条件。
情节冲突：事件源、参与角色、相关设定、相关伏笔、阻碍升级、转折机制、收益代价、期待变化、情绪变化。
期待情绪：期待来源、读者想看的画面、失败代价、延迟手段、压制强度、见证反应、主角状态变化、情绪转换、回落方式。
伏笔信息：埋设方式、伪装信息、当时理解、真实用途、答案分层、误导方式、揭晓节奏、回收收益、二次利用。
叙事技法：视角、场景压力、人物调度、信息增量、潜台词、攻防关系、说明触发点、行动承接、读者负担。
商业包装：目标读者、竞品参照、卖点排序、标题钩子、章末追读、承诺偏差、平台风险、不可照搬边界。
迁移训练：适用题材、前置条件、替换变量、改写步骤、限制条件、训练目标、验收标准、复盘问题。
```

关系势力：

```text
关系网：亲友、师徒、队友、敌人、暧昧/情感线、组织关系的变化和见证。
势力结构：阵营目标、资源、行动方式、和主角的利益关系、阶段变化。
社会反馈：谁见证主角变强，谁误判主角，谁因主角受益或受损。
资源流动：钱、身份、情报、装备、技能、地盘、人脉的获取、消耗、回收和升级。
```

情节冲突：

```text
事件源：麻烦从哪里来，为什么必须由主角处理。
主角行动：主角做了什么选择，是否主动推动。
阻碍升级：信息差、资源差、身份差、时间压力、道德压力、强敌压迫。
转折机制：新信息、新代价、新误判、新规则、新关系变化。
兑现方式：行动结果、收益、他人反应、关系变化、长期问题推进。
```

期待情绪：

```text
期待链：期待形成、压制、延迟、局部兑现、大兑现、新期待。
爽点链：铺垫、欠账、误判、底牌、爆发、见证、收获、余波。
情绪线：恐惧、好奇、憋屈、期待、震惊、爽、温情、遗憾等变化。
追读点：章末想看的下一件事，以及它是否在后文兑现。
```

伏笔信息：

```text
伏笔回收：伏笔位置、读者当时理解、真实用途、回收位置、回收收益。
信息差：读者知道但角色不知道、主角知道但他人不知道、作者暂时隐藏。
悬念管理：谜面、线索、误导、阶段答案、最终答案。
解释节奏：何时解释、解释多少、解释后立刻如何使用。
```

叙事技法：

```text
开篇技法：第一章钩子、六要素、题材承诺、第一单元闭合。
场景写法：场景目标、冲突、反应、信息展示、转场。
对话写法：话外意图、关系变化、信息差、节奏、角色声口。
说明写法：设定如何插入行动，何时解释，解释后立刻如何使用。
镜头/分镜：视角选择、信息先后、特写、反应镜头、高潮落点。
可迁移模板：能复用的结构功能；不可迁移处必须说明题材、平台或设定限制。
```

商业包装：

```text
书名简介：外部承诺是否准确命中核心循环和主角收益。
标签卖点：平台标签、题材关键词、差异化钩子、同题材参照。
章节标题：是否持续制造期待、提示收益或隐藏悬念。
平台适配：起点/番茄/飞卢/晋江等平台的节奏、题材、读者差异。
风险边界：哪些是平台红利、题材红利、作者个人表达，不能直接照搬。
```

迁移训练：

```text
可迁移模板：能复用的结构功能、适用条件、改写步骤。
不可迁移原因：题材限制、平台限制、设定依赖、作者风格依赖。
仿写任务：拆一个结构后，必须设计一个自己的桥段或章节。
回写大纲：把学到的结构落回自己的设定、第一卷或前三章。
```

媒介专项：
